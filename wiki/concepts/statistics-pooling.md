# Statistics Pooling · 统计池化

## 一句话
沿时间轴把不定长的逐帧序列，用均值+标准差汇成一个固定长度的向量。

## 直觉
这是 [[x-vector]] 里"变长语音→定长嵌入"的那一步魔法。

痛点：网络前半段（[[time-delay-neural-network]]）逐帧处理，输入 100 帧出 100 帧、输入 500 帧出 500 帧——还是不定长。但最后要的是一个**定长**向量去算距离。怎么把"长度不定的一摞帧"塌成"一个固定大小的东西"？

答案朴素得意外：**对整段求统计量**。100 帧也好 500 帧也好，求个均值就只剩一个数（每维一个）。均值天然不在乎你有多少帧——这就是"变长→定长"。

为什么不只取均值、还要标准差？均值只说"这段语音平均长什么样"，标准差还带了"这段里嗓音波动有多大"——快慢、高低起伏的幅度，对区分人很有用。所以 x-vector 把**均值和标准差拼起来**，维度翻倍。

类比：一个班 30 个学生（不定长，换班可能 40 个）的成绩，你没法直接拿"30 个分数"和别的班"40 个分数"比。但你可以算这个班的**平均分 + 标准差**两个数——不管多少人都是两个数，拿去和别的班比就对齐了。statistics pooling 就是干这个，只不过对的是 1500 维的逐帧向量。

## 怎么做的
```
输入: 帧级序列 h[1..T]，每帧是 D 维向量（x-vector 里 D=1500）
      T 是帧数，不固定

mean = (1/T) Σ_t h[t]          # D 维
std  = sqrt( (1/T) Σ_t h[t]² − mean² )   # D 维，逐维标准差

输出 = concat(mean, std)       # 2D 维，固定！（x-vector: 1500+1500=3000）
```

不管 T 是多少，输出永远 2D 维。后面接全连接 → 512 维 = x-vector。

## 数字例子
3 帧，每帧 2 维（真实是 T 帧 × 1500 维，这里缩小手算）：

```
h1 = [2, 0]
h2 = [4, 0]
h3 = [6, 0]
```

**均值**（逐维求平均）：
- 维度 1: (2+4+6)/3 = 12/3 = **4**
- 维度 2: (0+0+0)/3 = **0**
- mean = [4, 0]

**标准差**（逐维）：维度 1 的偏差是 2−4=−2, 4−4=0, 6−4=+2
- 方差 = (4+0+4)/3 = 8/3 ≈ 2.67 → std ≈ **1.63**
- 维度 2 全是 0 → std = 0
- std = [1.63, 0]

**拼接输出** = [4, 0, 1.63, 0] → **4 维定长**。

现在换一段 5 帧的语音进来，无论那 5 帧是多少，pooling 出来**还是 4 维**（mean 2 维 + std 2 维）✓。长度被这一步彻底"吃掉"了。维度 2 全程是 0（这一维没信息），均值和标准差都是 0——也如实反映出来了。

## 跟普通 pooling 的对比
- 普通 [[pooling]]（CNN 里的 max/avg pool）沿**空间**降采样，还是保留一个缩小的网格。
- statistics pooling 沿**时间**直接塌成一个向量，把整段压成定长——目的是"消灭长度维度"，不是降采样。
- 进阶版 attentive statistics pooling（ECAPA）：求加权均值/标准差，让网络自己学"哪些帧更重要"，而不是每帧等权。
