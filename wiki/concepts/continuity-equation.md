# Continuity Equation · 连续性方程

## 一句话
"粒子不凭空冒、不凭空消"的数学化身。**密度变化 = 净流入** —— 仅此而已。

## 直觉

想象一座城市的某条街区（地图上一个格子）。

- `p(x, t)` = 这街区在 t 时刻有多少人（人口密度）
- `v(x, t)` = 这街区里人们走动的平均方向和速度（风场 / 速度场）
- `p · v` = 流量（多少人 × 多快走）

街区的人口怎么变？只能靠走进来和走出去。每一秒变多少：

```
人口变化率 = (进来的人) − (走掉的人)
          = 净流入
```

把"净流出"取个名字叫 `∇·(p·v)`（散度 · divergence），把"人口变化率"叫 `∂p/∂t`，等式就是：

```
∂p/∂t + ∇·(p·v) = 0
←─────→   ←─────────→
密度变化率   净流出率
```

挪到一边：**∂p/∂t = -净流出 = 净流入**。

就这。整个连续性方程其实在说一句废话：**你这点多了多少人 = 走进来的人 − 走出去的人**。因为人不会从空气里凭空蹦出来。把"人"换成水分子、电荷、粒子云、概率质量，结构完全一样。

## 三个图像化的例子

把 `∇·(p·v)` 取 0 / 正 / 负 拆开看：

**例 1 · 流过**

```
→ → → → →
→ → → → →
→ → → → →
```

所有粒子向右等速走。你站在中间任一点：左来的 = 右走的，净流出 = 0，所以 `∂p/∂t = 0`。粒子在流，但你这点密度不变。

**例 2 · 发散（喷泉中心）**

```
↖   ↑   ↗
←       →
↙   ↓   ↘
```

所有方向都在往外流。净流出 > 0 → `∂p/∂t < 0` → 这点密度下降（粒子从你脚下散开）。

**例 3 · 汇聚（黑洞中心）**

```
↘   ↓   ↙
→       ←
↗   ↑   ↖
```

所有方向都在往里流。净流出 < 0 → `∂p/∂t > 0` → 这点密度上升（粒子往这点汇聚）。

这三种就是散度取 0 / 正 / 负 的物理直觉。

## 跟 flow matching 的关系

[[flow-matching]] 要做的事：给一族 [[probability-path]]（每个时刻一份分布 `p_t`），找出对应的 [[velocity-field]] `v_t`，让粒子沿 `v_t` 流动时，密度演化正好就是 `p_t`。

连续性方程告诉你：**给定 p_t，对应的 v_t 是唯一的**（必须满足这条方程）。所以"选 path = 选 velocity"，flow matching 直接在 path 这一层做选择，velocity 自然定下来。

## 为什么它叫"守恒律"

把方程对整个空间积分（散度定理）：

```
∂/∂t ∫ p dx + ∫ ∇·(p·v) dx = 0
                          ↓ 散度定理
              = 整个空间边界上的流量
              无穷远处 p=0 → 这项也是 0

⇒  ∂/∂t ∫ p dx = 0
```

也就是：**总质量 / 总概率不随时间变化**。这就是 "`p_t` 永远积分为 1" 的数学保证 —— 它是连续性方程的副产品。

## 编程角度

如果你写过粒子模拟器：

```python
# 初始化 N 个粒子位置
particles = sample_from(p_0)

# 每一步沿 velocity field 推进
for t in timesteps:
    for i in range(N):
        particles[i] += v(particles[i], t) * dt

# 任意时刻 p_t 的密度 = 这点附近的粒子数密度
```

连续性方程就是这段代码的**连续极限**。粒子数永远 N 个不变 → 总概率守恒。某点密度变化 = 这步走进来的粒子 − 走出去的粒子 → `∂p/∂t = -∇·(p·v)`。

## 符号速查

- `∂p/∂t` · 偏导数：密度怎么随时间变（位置不动，只让 t 动）
- `∇·` · 散度（div）：一个向量场在某点"净流出 / 汇聚"的程度
- `p·v` · 概率流（probability flux）：密度 × 速度，方向是 v 的方向
