# 测试时训练 TTT · 每道题临时调一下参数，答完即弃

## 一句话
推理时对**当前这一个输入**用一个无监督辅助损失临时更新参数（通常只动 LoRA），调好后作答，然后把更新**复位**——下一题重来。

## 直觉 · 在线给自己开小灶
普通推理：参数固定，直接前向出答案。TTT 多一步：**先针对手头这道题微调几步再答**。微调用的不是标签（推理时没有），而是一个**自监督辅助损失**——只要能从输入本身构造、且"调好它有助于真任务"即可。直觉像考前临时针对这一题"找找手感"，考完忘掉、不带进下一题。

跟 [[in-context-learning]] 的关键区别：ICL 把示例塞进上下文、**权重一个字不改**；TTT **真的更新权重**（只是临时、每实例一份、用完丢）。

## 怎么做的
```
两阶段（每个测试实例独立）：
1. 优化: Δθ* ≈ argmin_Δθ  L_aux(输入; θ+Δθ)     ← 无监督辅助损失
2. 作答: y = Generate(θ+Δθ*, 输入)；然后丢弃 Δθ*
更新限制在 LoRA: W = W₀ + BA（W₀ 冻结，只学低秩 B、A）→ 便宜、可复位
```
辅助损失怎么选是关键。[[relat]] 选的是**重建输入**（[[reconstruction-as-fidelity]]）：能把原问题从潜在状态重建回来，就说明潜在没跑偏——一个输入锚定的信号。别的 TTT 变体用过熵最小化等，但那调的是"输出自信"，跟"忠于输入"无关。

## 数字例子 · ReLAT 的 TTT 设定
```
冻结主干 θ；临时 LoRA φ 初始化
for n in 1..N(=16):
    用当前 φ 生成潜在 → 重建原问题 → 算掩码交叉熵 L
    AdamW 更新 φ（lr η=2e-5 最优, θ 不动）
N 步后: 用 (θ, φ*) 对【原始问题】作答
作答完: φ 复位 → 下一题从头来
```
✓ 自检：全程没用任何标签——损失来自"重建输入"这个自监督信号。只动 LoRA（不动 20 亿级主干）所以 16 步很便宜、且**复位干净**：每道题拿到的是"为它专门调过、又互不污染"的模型。代价是每题多 N 步前向+反传的开销。
