# 单阶段检测 · 一次前向直接出框,不先提候选

## 一句话
两阶段检测先"提一堆候选框、再逐个分类";单阶段(YOLO 这类)把这两步拍成一次前向——一个网络直接从整张图回归出所有框 + 类别,快得多。

## 直觉 · 为什么两阶段慢
两阶段(R-CNN 家族)像"先海选再面试":① 用某种方法提出上千个可能有物体的**候选区域**;② 把每个候选区域各自送进分类器判类别 + 微调框;③ 最后去重。慢在**逐个候选重复跑分类**(上千次),而且分类器只看候选框里那一小块,**看不到全图上下文** → 容易把背景纹理误当物体。

单阶段的转念:**别提议了,直接让网络一口气把整张图的框都回归出来**。把检测当成一个回归问题——输入图,输出"每个位置有没有物体、框在哪、是什么"。一次前向搞定,天然快、天然看全图。

## 怎么做的 · 位置分工 + 一次输出
```
两阶段:  图 →[提~2000候选]→ 每个候选各跑分类+回归 →[去重]→ 框   (慢,看局部)
单阶段:  图 →[一个CNN一次前向]→ 所有位置的框+类别一起出 →[NMS去重]→ 框  (快,看全图)
```
单阶段靠"**空间分工**"避免重复:把图划成网格 / 铺满预设位置,每个位置只负责它那一块,一次性并行预测。[[yolo]] 用 7×7 网格,每格负责中心落在自己格里的物体。[[yolov2-yolo9000]] 再把每格两个自由框改成 5 个 [[anchor-boxes]],候选更多、召回更高,仍然只跑一次网络。

## 数字例子 · 前向次数差多少
```
两阶段:  1 次提候选 + ~2000 次逐候选分类 ≈ 上千次网络评估 → 一张图几秒
单阶段:  1 次前向,输出 7×7×30 = 1470 个数(含所有框和类别)→ 一张图 ~22ms(45 FPS)
```
✓ 关键差别不是"网络更小",而是**把上千次"逐候选评估"换成一次"整图评估"**。代价:一次性回归比"提议—验证"定位更糙(YOLO 的主要误差就是 localization),但换来实时速度和全图上下文(背景误检更少)。
