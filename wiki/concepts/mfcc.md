# MFCC · 梅尔频率倒谱系数

## 一句话
深度学习之前最标准的语音特征：log-mel 频谱再做一步 DCT，取前 ~13 个系数，抓住频谱的"大形状"（音色）。

## 直觉
它就是 [[log-mel-spectrogram]] 再多走一步。前面那几步（分帧 → FFT → mel 滤波 → log）合起来就是 log-mel；MFCC 的灵魂在最后那个 **DCT**。

DCT 在干嘛？把 log-mel 那条"频率 → 能量"的曲线，**本身当成一个信号，拆成几个'形状基'**：
- 第 0 个系数 = 整条曲线的**平均高度**（总能量）
- 第 1 个 = 曲线**往左高还是往右高**（低频重还是高频重的倾斜）
- 第 2 个 = 中间**鼓不鼓**（有没有弯）
- 越往后 = 越细的波纹

取前 13 个，就抓住了频谱的大致形状——这个大形状就是**音色 / 共振峰**（决定"这是啊还是哦""这是谁的嗓子"）。后面高阶的细波纹主要是音高的谐波，通常直接扔掉。

类比：描述一条起伏的山脊线，不用报每一点的高度，说"整体多高 + 往哪边斜 + 中间有个包"就够还原大形状了。DCT 头几个系数就是这几个数。

> "cepstral（倒谱）"是 "spectral（频谱）"前几个字母倒过来拼的——因为它是对频谱再做一次类傅里叶变换，"频谱的频谱"。名字本身是双关。

## 怎么做的
```
原始波形
 ① 分帧      25ms 一帧, 每 10ms 挪一下(帧间重叠)
 ② FFT       看这帧各频率有多少能量 → 频谱
 ③ Mel 滤波   按人耳分桶压成 ~26-40 个 band(低频细,高频糙)
 ④ log       取对数(人耳对响度也是对数感知)
   ────────── 到这步 = log-mel
 ⑤ DCT       对这条曲线拆"形状基", 取前 ~13 个系数 = MFCC
```

DCT 用的"形状基"是一组余弦波：第 k 个系数 = 把 log-mel 曲线和"第 k 条余弦波"逐点相乘求和（看这曲线含多少那个形状）。

## 数字例子
取一段 4 个 band 的 log-mel（真实有 ~40 个，这里缩小手算）：

```
x = [4, 3, 2, 1]   ← 低频能量高、往高频递减
```

**第 1 步 · 代 n 造波**。基函数 `cos[π/4·(n+0.5)·k]` 不是直接作用在 x 上的——给定一个 k，把 `n=0,1,2,3` 代进去，得到 4 个数，这 4 个数就是第 k 条"基波"（一个长度 4 的向量）。`N=4` 所以 `π/N=π/4`；`(n+0.5)` 是让取样落在每个 band 正中间（0.5,1.5,2.5,3.5），DCT 的约定：

```
k=0: cos(0)=1            → [ 1,    1,    1,    1  ]   一条平线
k=1: cos(π/4·(n+0.5))                                半条余弦,左正右负
     22.5° 67.5° 112.5° 157.5° → [ 0.92, 0.38,−0.38,−0.92]
k=2: cos(π/2·(n+0.5))                                一整条余弦, +−−+
     45°  135°  225°   315°    → [ 0.71,−0.71,−0.71, 0.71]
```

页里那几行 "0.92, 0.38…" 就是这么来的，纯粹是把 n 代进公式。

**第 2 步 · 系数 = 把波跟 x 点积**。`c_k` = x 和第 k 条波逐位相乘再相加（就是 [[dot-product]] / 相关）：「x 长得有多像这条波?」

```
x   = [ 4,    3,    2,    1  ]
─────────────────────────────────────────────
波0 = [ 1,    1,    1,    1  ] → c₀ = 4+3+2+1                  = 10
波1 = [0.92, 0.38,−0.38,−0.92] → c₁ = 4(.92)+3(.38)+2(−.38)+1(−.92) = 3.16
波2 = [0.71,−0.71,−0.71,0.71] → c₂ = 0.71·(4−3−2+1)           = 0
```

**c₀ = 10** → 跟平线重合度 = 总能量。
**c₁ = 3.16**（大正）→ x 正好也左高右低，跟"左高右低"那条波**高度吻合** → 量出"频谱往低频斜"。
**c₂ = 0** → x 是直线，跟"两头翘中间凹"那条 U 形波**完全不沾边**，正负相乘抵消。✓ 自检：直线频谱的"鼓包"系数必然为 0。

所以三个数 `[10, 3.16, 0]` 已经把这帧讲清楚了：**总量 10、向左斜、笔直不鼓**——足够还原 [4,3,2,1] 的大形状，不用存 4 个原始值。每条波是一把专量某种形状的尺子，`c_k` 就是"x 含多少那个形状"。这就是 MFCC 降维 + 抓形状的本事。

## 为什么当年多走这步（不直接用 log-mel）
1. **去相关**：相邻 mel band 能量高度冗余，老式模型（GMM 假设特征不相关）受不了，DCT 把它们压成互不相关的系数。
2. **降维**：40 个 mel → 13 个系数，省存储省算力。
3. **分离音色/音高**：低阶系数 = 频谱包络（音色），高阶 = 精细谐波（音高），一刀切开。

## 跟 log-mel 的对比 / 时代注脚
| | MFCC | log-mel |
|---|---|---|
| 最后一步 | 多一个 DCT | 没有 |
| 维度 | ~13 | ~40-80 |
| band 间相关性 | 去掉了 | 还在 |
| 谁爱用 | GMM-HMM / i-vector / x-vector | 现代深度模型 |

现在的深度模型越来越多直接喂 log-mel、甚至原始 filterbank，**跳过 DCT**——神经网络不在乎输入相不相关，反而那步 DCT 丢掉的信息它想自己留着学。[[x-vector]]（2018，Kaldi 血统）用 MFCC 是因为那是当时标准管线；[[whisper]]、[[qwen3-asr]] 就直接 log-mel / Fbank 了。
