# 连续 Mesh 连接表示 · 不直接预测面，给每个顶点一枚“连边磁铁”

## 一句话

每个顶点除了坐标和法线，再带一个连续向量；两枚向量的连接分数过线就连边，三条边围成一圈就恢复三角面。

## 为什么不直接生成面

顶点坐标是连续数，能平滑加噪、去噪。面却是三个整数索引，例如 `(1,2,3)`；索引从 2 跳到 3，不存在“跳了一半”的中间状态。扩散模型要处理这种离散拓扑很别扭。

MeshFlow 把离散问题往连续空间挪：顶点 `i` 带边嵌入 `h_i`，两顶点先算连续连接分数，再过阈值得到 0/1 边。扩散模型只负责生成连续的 `h`；最后一步才切回离散图。

## 数字例子 · 四个顶点恢复两个三角面

拿一张正方形薄片：

~~~text
v4 (0,1) —— v3 (1,1)
 |          / |
 |        /   |
 |      /     |
v1 (0,0) —— v2 (1,0)
~~~

目标有两个三角面 `(1,2,3)` 和 `(1,3,4)`，因此应有五条边：

~~~text
E = {12, 13, 14, 23, 34}
唯一不该连的是 24
~~~

为了把阈值算清，先用普通二维嵌入做玩具版：

~~~text
h1=( 0,0)   h2=(1,0)
h3=( 0,1)   h4=(-1,0)
阈值 τ=1.42，距离不超过 τ 就连边

d12=1.000  ✓     d13=1.000  ✓     d14=1.000  ✓
d23=1.414  ✓     d34=1.414  ✓     d24=2.000  ✗
~~~

接着找“三条边都在”的三元组：

~~~text
(1,2,3): 12、23、13 都在 → 收下这个面
(1,3,4): 13、34、14 都在 → 收下这个面
(1,2,4): 缺 24              → 不是面
(2,3,4): 缺 24              → 不是面
~~~

✓ 自检：恢复出的面正好是原来的两张，内部对角线 `13` 被两张面共用。

真实 MeshFlow 用的不是上述欧氏距离，而是 SpaceMesh 的 signed spacetime score。论文主文式 (2) 写 `d≤τ` 连边；但边损失、补充算法和官方代码提交 `55f56f6` 都使用 `d>τ`。因此读实现时，应把 `d` 当“越大越像边”的连接分数，别按主文那句普通距离理解。

## 法线为什么也要预测

边只能告诉你哪三个顶点围成面，不能告诉你这个面朝里还是朝外。MeshFlow 再预测每个顶点的外法线。三个顶点法线取平均，与几何叉乘得到的面法线比较：点积为负，就交换面里的两个顶点，把朝向翻过来。

上例全部顶点法线都取 `(0,0,1)`。面 `(1,2,3)` 的两条边是 `(1,0,0)` 与 `(1,1,0)`，叉乘得到 `(0,0,1)`，与平均法线同向，不用翻面。

## 边界

- 只找三边环，因此天然只生成三角面。
- 若几条错误边恰好组成三环，会凭空长出面。
- 小洞要靠边界环检测和三角化补丁修；生成质量不够时，拓扑后处理仍跑不掉。
