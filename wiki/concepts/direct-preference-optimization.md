# DPO · 直接偏好优化（跳过奖励模型对齐人类口味）

## 一句话
不训奖励模型、不跑 PPO，直接拿"赢/输"成对数据当分类题优化模型：把赢的样本概率顶上去、输的压下去，一步到位对齐人类偏好。

## 直觉 · 它把 RLHF 的两步压成一步
[[rlhf]] 的老套路是两步：先用人类偏好训一个**奖励模型**（打分器），再用 [[ppo]] 拿这个奖励去调模型。两步、要额外一个网络、还容易训崩。

DPO 的洞见：**偏好这件事，不必先变成一个分数再去优化**。给一对样本（同一个 prompt 下，人选出来的赢家 `y_w` 和输家 `y_l`），直接让模型"更想生成赢家、更不想生成输家"就行——写成一道**二分类**题：模型给赢家的相对概率应该高于输家。

关键巧思是"相对谁"：相对一个**冻结的参考模型** `π_ref`（通常就是没对齐前的自己）。DPO 不看绝对概率，看**你比参考模型多偏向某个样本多少**，这个量就是它的**隐式奖励**：

```
隐式奖励  r(y) = β · log[ πθ(y) / π_ref(y) ]
                  = β · ( logπθ(y) − logπ_ref(y) )
```

`β` 是温度（管偏离参考的力度）。直觉：`r(y)` 大 = "我比原来的自己更爱生成 y"。

## 怎么做的 · 一道 sigmoid 二分类
DPO 的损失就是"赢家隐式奖励要高过输家"的逻辑回归：

```
loss = − log σ( r(y_w) − r(y_l) )
     = − log σ( β·[(logπθ(y_w) − logπ_ref(y_w)) − (logπθ(y_l) − logπ_ref(y_l))] )
```

`σ` 是 sigmoid。赢输的奖励差越大 → `σ` 越接近 1 → loss 越小。梯度会**抬高赢家的对数概率、压低输家的**。

## 数字例子 · 算给你看，顺带看出它的坑
取 `β=1`，参考模型对赢家、输家初始都给 `logπ_ref = −2.0`（即一开始一样喜欢）。

**第 0 步（还没动）**：模型 = 参考，`r(y_w)=0`、`r(y_l)=0`，差 = 0。
```
σ(0) = 0.5      loss = −ln(0.5) = 0.693
```

**一步"好"的更新**——赢家概率抬上去、输家压下去：
```
logπθ(y_w) = −1.5   → r(y_w) = −1.5 −(−2.0) = +0.5   （比参考更爱赢家 ✓）
logπθ(y_l) = −3.0   → r(y_l) = −3.0 −(−2.0) = −1.0   （比参考更厌输家 ✓）
差 = 0.5 −(−1.0) = 1.5
σ(1.5) = 0.818      loss = −ln(0.818) = 0.201   ↓ 变小，对
```

**一步"偷懒"的更新**——这就是坑：赢家也被压低了，但只要差距更大，loss 照样降：
```
logπθ(y_w) = −2.5   → r(y_w) = −2.5 −(−2.0) = −0.5  （比参考更【不】爱赢家 ✗！）
logπθ(y_l) = −4.5   → r(y_l) = −4.5 −(−2.0) = −2.5
差 = −0.5 −(−2.5) = 2.0
σ(2.0) = 0.881      loss = −ln(0.881) = 0.127   ↓ 反而更小！
```
✓ 自检：第二步的 loss（0.127）比第一步（0.201）还低，DPO "更满意"——可赢家的对数概率从 −2.0 掉到了 −2.5，**模型其实更不会生成那张好图了**。DPO 只盯"赢减输"的差，没人管赢家的**绝对**概率，于是模型学会一个偷懒解法：**把两个都往下压、只要把输家压得更狠**，差距就拉大、loss 就降。结果"为了拉开差距，把好的也练差了"。这就是**策略发散**。

## STPO · Krea 2 怎么堵这个坑
[[krea-2]] 在偏好优化阶段自创了一个 DPO 变体，叫 **STPO**（报告没给全称）：**加一个辅助损失 + 改 DPO 公式**，专门盯住"赢家的绝对概率别往下掉"——不许靠单方面压低赢家来制造差距，逼模型真的把赢家顶上去。落到上面的例子，就是给"赢家 `r(y_w)` 掉到负数"这种情况额外加一笔惩罚，把第二步那种偷懒解法堵死。

> 类比：评委不能靠"把两个选手都压分、只要把差选手压更低"来拉开名次；STPO 等于规定"好选手的绝对分不许掉"。
