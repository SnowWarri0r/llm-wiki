# DINOv3 多样性奖励 · 给"组内整齐划一"扣钱，防风格塌缩

## 一句话
一条 prompt 生成一组图，用 DINOv3 把每张压成向量，算组内两两的平均距离当"多样性分"，加进奖励——逼模型每次给的几张互不雷同。

## 直觉 · 质量奖励单独跑会塌
RL 阶段如果只用"质量/偏好"奖励，模型会发现一个偷懒解：**每次都输出那一种最讨喜、最稳拿高分的风格**。结果分很高，但所有图都一个样——[[krea-2]] 把这叫**多样性塌缩**（house-style collapse），恰恰违背它"撑开审美空间"的初衷（见 [[generative-data-curation]]）。

对策：再加一个**多样性奖励**，专门给"这一组图太像"扣分。**DINOv3** 是自监督 ViT（[[dino]] 家族），把图压成对**风格/结构**敏感的向量；同一 prompt 那组图的向量越分散，多样性分越高。

## 怎么做的
```
1. 一条 prompt 生成一组图（如 4 张），每张过 DINOv3 → 一个向量
2. 算组内两两的"距离"（如 1 − 余弦相似度），取平均 = 这组的多样性分
3. 把多样性分加进总奖励：总 = 质量 + 对齐 + λ·多样性
4. RL（GRPO/GDPO）照常优化总奖励
```
关键经验（报告原话意思）：**这个 λ 一调小，模型很快又塌回单调**——所以整个训练全程都得让多样性奖励**保活**，不能训到后期撤掉。

## 数字例子 · 塌缩组 vs 多样组
图向量简化成 3 维，多样性分 = 组内两两 `1−cos` 的平均（越大越多样）：
```
塌缩组（三张几乎一样）:
  [0.90,0.40,0.10]  [0.89,0.42,0.12]  [0.91,0.39,0.08]
  两两 cos 都 ≈0.999 → 1−cos ≈0.001
  多样性分 ≈ 0.001   → 几乎拿不到多样性奖励

多样组（三张各走一个方向）:
  [0.90,0.40,0.10]  [0.20,0.95,0.10]  [0.10,0.30,0.95]
  两两 cos 低 → 1−cos 大
  多样性分 ≈ 0.564  → 多样性奖励高
```
✓ 自检：塌缩组质量分可能很高，但多样性分只有 0.001，总奖励被这一项压下来；多样组多样性 0.564 把总奖励顶上去。于是优化器**只有同时保住质量和多样性才划算**，"全输出同一种"这条偷懒路被堵死。λ 就是"多看重多样"的旋钮——它太小，惩罚不痛，模型照塌不误。
