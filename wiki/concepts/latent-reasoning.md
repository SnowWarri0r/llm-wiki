# 潜在推理 · 把"想"压进连续向量，不摊成语言

## 一句话
不把推理的中间步写成自然语言（CoT），而是产出一串连续潜在向量当"思考"，被下游当条件反复用——省 token、但不透明，还可能悄悄丢了原问题的约束。

## 直觉 · 为什么不写成话
显式思维链（CoT）把"想"摊成一长串语言 token：清楚、可读，但**又长又慢**（几千 token 起）。潜在推理的主张是：思考没必要落成人话，**压成 K 个连续向量**（ℝ^d，[[relat]] 默认 K=16）就行，下游模型直接拿这串向量当条件接着算。好处是**省 token、快**。

代价是两条：**不透明**（向量没法像句子那样读懂在想什么）；以及 [[relat]] 点出的**开环**风险——

## 开环：生成完没人验
```
潜在生成:  ℰ = g(Q)              把问题 Q 压成 K 个潜在向量
下游消费:  z = f([Q; ℰ])          直接拿 ℰ 当条件接着推
```
问题在于：`ℰ` 一旦生成就被下游消费，**没有任何环节回头检查它是否还忠于 Q 的约束**。潜在状态要是悄悄漂移（丢了"3 岁""红色"这种关键约束），错误会顺着推理链传下去、还发现不了。这叫**开环（open-loop）**。[[relat]] 的整篇就是给它"闭环"——用 [[reconstruction-as-fidelity]] 在作答前把潜在拽回忠于原题。

> 注意它跟"单次前向里的隐藏状态"不同：隐藏态是一次性内部量；这里的 ℰ 是**被显式存下、反复当条件用**的，所以一旦带病就会持续传染。

## 数字例子 · 省在哪、险在哪
```
显式 CoT:   ~2000 token 的推理文字 → 可读、可查，但长、慢
潜在推理:   K=16 个向量          → 等效"思考"，token 量小两个数量级
              但: 16 个向量里到底编了 Q 的哪些约束？没人验 = 开环
```
✓ 自检：把"想"从 2000 token 压到 16 个向量，省的是显式生成那串字的成本；丢的是可读性和"中途自检"的机会。ReLAT 的贡献不是放弃潜在推理，而是在保留这份省的同时，**补一道输入保真的闭环检查**。
