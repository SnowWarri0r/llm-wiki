# SigLIP 语义去重 · 删的是"长得像"，不是"字节一样"

## 一句话
把每张图用 SigLIP 压成一个向量，向量方向接近（余弦相似度高过阈值）就当近重复，每簇只留一张。

## 直觉 · 为什么不用哈希
按文件哈希去重只能抓**逐字节相同**的图。但训练集里真正的冗余是**语义重复**：同一张图被裁过、压过、改过水印、调过色——字节全不同，可它们对模型是同一个样本，留着只会让这个概念**过度重复**、淹没长尾（正是 [[generative-data-curation]] 要避免的）。

[[krea-2]] 用 **SigLIP**（CLIP 的变体，把对比损失换成 sigmoid，见 [[contrastive-learning]] / [[clip]]）把图压成向量：语义相近的图，向量方向也相近。于是"长得像"变成了"向量夹角小"，可量、可批量删。

## 怎么做的
```
1. 全库每张图过 SigLIP → 一个向量（如 768 维），方向编码"这图讲什么"
2. 算两张图向量的余弦相似度 cos（1=方向完全一致 / 0=无关）
3. 设阈值，如 cos > 0.95 算近重复
4. 近重复的成簇，每簇只留一张（通常留分辨率/质量最好的）
```
全库两两比太慢，实际用**近似最近邻索引**（[[faiss-ann-search]]）只查每张图的少数近邻，把 O(N²) 压到近线性。

## 数字例子 · 余弦怎么判重
把图向量简化成 3 维。余弦 = 点积 ÷ 两者长度（见 [[dot-product]]）：
```
a = [0.90, 0.40, 0.10]   （原图）
b = [0.88, 0.45, 0.08]   （同图改了水印/压缩过）
c = [0.10, 0.25, 0.96]   （另一张完全不同的图）

cos(a,b) = (0.90·0.88 + 0.40·0.45 + 0.10·0.08) / (|a|·|b|)
         = 0.9944 / (0.997·0.997) ≈ 0.998   > 0.95 → 判近重复，删一张
cos(a,c) = 0.29                              < 0.95 → 不同，两张都留
```
✓ 自检：a 和 b 字节完全不同（水印都改了），哈希去重抓不到；但语义上是同一张，余弦 0.998 一抓一个准。c 虽然第三维也有值，但整体方向差很远，余弦只有 0.29，不会被误删。**阈值就是"多像才算重复"的旋钮**：调高（如 0.98）只删几乎一模一样的，调低（如 0.9）连"很像"的也并掉。
