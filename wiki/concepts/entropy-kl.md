# 熵 / KL 散度 · 多不确定 · 用错分布多浪费多少

## 一句话
**熵** = 一个分布平均有多"不确定 / 惊讶"(`H = −Σ p·log p`)。**KL 散度** = 拿一个错分布 Q 去描述真分布 P 时**多花的惊讶** = 交叉熵 − 熵,恒 ≥0、相等才 0。都是 `−log` 在打底,跟 [[cross-entropy]] 一家。

## 直觉 · 惊讶度 → 熵 → KL 三连

**① 惊讶度 = −log p。** 一件事越不可能(p 小),它真发生时你越惊讶。`p=1`(必然)惊讶 0;`p` 越小 `−log p` 越大。([[cross-entropy]] 的 `−log` 就是这个。)

**② 熵 = 惊讶度的平均** `H(P)=Σ p·(−log p)`。一个分布越"摊平、说不准"熵越高;越"集中、笃定"熵越低。
- 公平硬币(0.5/0.5):最说不准 → 熵最高。
- 几乎总正面(0.99/0.01):基本笃定 → 熵很低。
- 熵也是"平均最少要几位编码"的下限(信息论本义)。

**③ KL 散度 = 用错分布的额外浪费。** 真相是 P,但你**误以为**是 Q 去编码 / 下注:
```
KL(P ‖ Q) = Σ p·log(p/q) = [用 Q 描述 P 的平均惊讶] − [P 自己的熵]
          = 交叉熵 H(P,Q)        −        熵 H(P)
```
- **恒 ≥ 0**:用错分布只会更浪费,不会更省;`P=Q` 时正好 0。
- **不对称**:`KL(P‖Q) ≠ KL(Q‖P)`——"把谁当真相"不一样。
- 直觉:KL 就是交叉熵里**纯属浪费**的那部分(扣掉 P 本身不可避免的不确定性)。所以训练最小化交叉熵 ≈ 最小化 KL(P 的熵是常数)。

## 数字例子 · 手算(用 log₂,单位 bit)

**熵**:
```
公平硬币 P=[0.5, 0.5]:
  H = −(0.5·log₂0.5 + 0.5·log₂0.5) = −(0.5·(−1) + 0.5·(−1)) = 1 bit   ← 最不确定
偏硬币 P=[0.9, 0.1]:
  H = −(0.9·log₂0.9 + 0.1·log₂0.1) = −(0.9·(−0.152) + 0.1·(−3.32)) ≈ 0.47 bit  ← 笃定多了，熵低
```

**KL**(真相 P=[0.5,0.5],你误用 Q=[0.9,0.1]):
```
KL(P‖Q) = 0.5·log₂(0.5/0.9) + 0.5·log₂(0.5/0.1)
        = 0.5·(−0.848)      + 0.5·(2.322)
        = −0.424            + 1.161   = 0.737 bit   ← 用错分布每次平均多花 0.74 bit
```
反过来 `KL(Q‖P) = 0.9·log₂(0.9/0.5)+0.1·log₂(0.1/0.5) = 0.9·0.848+0.1·(−2.322) ≈ 0.53 bit` —— **数不一样,印证不对称**。

## 落点
- **[[cross-entropy]]**:分类 / 预测下一 token 的损失就是交叉熵 = 熵 + KL;最小化它在把模型分布 Q 推向真分布 P。
- **[[closed-form-kl]]**:两个高斯的 KL 有闭式;同协方差时塌成均值差²。
- **[[rl-for-llm-people]] / RLHF**:KL 当"橡皮筋"惩罚,拽着新策略别离原模型太远。
- **[[softmax]]**:这些概率分布大多由 softmax 产生。
