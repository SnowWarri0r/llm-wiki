# SVD · 奇异值分解 · 任何矩阵 = 转一下·各轴拉伸·再转一下

## 一句话
任何矩阵都能拆成 **`W = U Σ Vᵀ`**:先换个角度看(`Vᵀ` 旋转)、沿各主轴**各自拉伸**(`Σ`,对角线上是奇异值 σ)、再换个角度摆好(`U` 旋转)。**σ 排序后就是各方向的"强度";留前 r 个 = 最佳低秩近似**——这是 [[matrix-rank]] / 压缩 / PCA 背后的同一台机器。

## 直觉 · 把一个变换拆成"旋转-拉伸-旋转"

一个线性变换 `y = W x` 看着复杂,SVD 说它本质上只是三步:
```
x ──Vᵀ──> 换一组正交坐标轴看 ──Σ──> 沿每根新轴各自缩放 ──U──> 再旋到输出坐标
```
- `V`、`U`:两组**正交方向**(纯旋转/反射,不改长度)。
- `Σ`:对角矩阵,对角线 `σ₁ ≥ σ₂ ≥ … ≥ 0` 是**奇异值**——第 i 根主轴被拉伸多少倍。

关键直觉:**σ 大的那根方向,是这个变换"使劲"的方向;σ 小的几乎被压扁、可以忽略**。所以一堆奇异值从大到小排开,就是"这个矩阵的信息都集中在哪几个方向"的体检表。

## 怎么做的 · 截断 = 最佳低秩近似

把矩阵写成 **r 个"秩 1 块"按强度相加**:
```
W = σ₁ u₁v₁ᵀ + σ₂ u₂v₂ᵀ + … + σ_n u_nv_nᵀ      （σ 从大到小）
```
只**留前 k 个、丢掉长尾**(σ 小的),得到的 `W_k` 就是 W 的**最佳秩 k 近似**(Eckart–Young 定理:在所有秩 k 矩阵里,它跟 W 的误差最小)。这正是:
- **[[matrix-rank]] 低秩**:秩 = 非零 σ 的个数;低秩 = 长尾 σ 都是 0 或近 0。
- **压缩 / 降维**:图像、embedding 只留前几个 σ,省存储、去噪声(长尾常是噪声)。
- **PCA**:对（中心化）数据矩阵做 SVD,前几个主轴就是主成分。
- **[[lora]]**:微调增量近似低秩,所以拿秩 r 的 `B·A` 就能逼近。

一个直觉数字:一张图的像素矩阵做 SVD,常常**前 5%~10% 的 σ 就还原出肉眼几乎一样的图**——剩下 90% 的奇异值贡献的都是细枝末节。

## 跟特征分解的区别(顺带)
特征分解 `W = QΛQ⁻¹` 只对**方阵**、且需要可对角化;SVD **任何形状矩阵都行**(d×k 也可),而且 U、V 一定正交、σ 一定非负实数——更稳、更通用。对称正定矩阵时两者重合。
