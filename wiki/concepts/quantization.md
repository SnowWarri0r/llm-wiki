# 量化 · WxAy · 用更少比特存数字和算

## 一句话
用**更少的比特**存权重、做计算，省显存 + 提速度。命名 **WxAy** = 权重 x 位、激活 y 位;**W8A8**(都 8 位)能让矩阵乘直接跑在 INT8 上、吞吐翻倍,但激活的"离群值"会坑精度,要靠 SmoothQuant 之类的招。

## 直觉 · 菜谱和食材都四舍五入

LLM 默认权重和激活都是 **16 位(BF16)**。量化就是把它们压到 8 位、4 位的整数。命名规则:

- **W8A16** = 权重 8 位、激活 16 位(weight-only,只压权重)
- **W8A8** = 权重 + 激活**都** 8 位(INT8)
- **W4A16** = 权重 4 位、激活 16 位

一层的计算本质是 `输出 = 权重 W × 激活 A`(矩阵乘)。关键区别:
- **只压权重(W8A16)**:省了权重显存/带宽,但**矩阵乘还得在 16 位里算**(激活 16 位,得把权重升回去) → 主要救内存,不太提算力。适合 batch 小、被显存卡住的解码。
- **W8A8**:两个操作数都 INT8 → **矩阵乘直接跑在 INT8 张量核** → 吞吐约 ×2、带宽减半。**这才是真·降延迟**(Lumine 200ms 出一帧靠它)。

## 坑在激活:离群值

权重数值乖、分布均匀,好压。**激活不行——有少数"离群值"(几个通道数值巨大)**。naive INT8 一压:这几个巨值把量化范围撑爆,其余正常值全被压成约等于 0 → 精度崩。

**SmoothQuant** 的招:把"难压的责任"**从激活搬一部分到权重**——激活按通道除以系数 `s`、对应权重乘回 `s`(数学上输出不变),让两边都变乖、都能干净 INT8:
```
Y = (X / s) · (s · W)      # 输出不变，但 X、W 都更平滑 → 都能 INT8
```

类比:**权重是菜谱**(固定,四舍五入容易);**激活是流进来的食材**(每次不同,偶尔某样爆量)。SmoothQuant 把爆量那样**预先缩小**、把缩放系数**烤进菜谱**,菜谱和食材就都能安心约成整数。

## 怎么做的(几种主流)
```
weight-only:  GPTQ / AWQ        # W4A16，救显存，解码常用
weight+act :  SmoothQuant       # W8A8，激活难压先平滑搬给权重，提算力
KV-cache 量化: 把 KV 也压到 8/4 位，长上下文省显存（与上面正交）
```
