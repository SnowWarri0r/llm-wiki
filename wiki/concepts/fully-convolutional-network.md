# Fully Convolutional Network · 全卷积网络 · 输入变大，输出网格跟着变

## 一句话

网络里没有要求固定展平长度的全连接层；卷积核能在任意大小的空间网格上复用，所以输入宽高改变，输出特征图宽高也跟着改变。

## 直觉

固定全连接层像一张写死 100 个插孔的接线板。输入必须正好 100 维，101 维就插不进去。

卷积核更像一把 `3×3` 小尺子。图片宽一点，它就多滑几步；高一点，也多滑几行。尺子自己的 9 个权重不用改。

## 怎么做的

一个不补边、步长 1 的 `3×3` 卷积：

```text
n_out = n_in - 3 + 1 = n_in - 2
```

同一层权重：

```text
输入 8×8  → 输出 6×6
输入 10×10 → 输出 8×8
```

卷积核参数量只由通道和核大小决定：

```text
参数量 = 3×3×C_in×C_out
```

它和输入是 `8×8` 还是 `1000×1000` 无关。

## 数字例子

假设网络只有两层 `3×3 valid conv`，都不接全连接：

```text
12×12 → 10×10 → 8×8
20×20 → 18×18 → 16×16
```

换输入尺寸后，两层权重形状完全不变，输出只是多出位置。

若在末尾展平并接 `8×8×64 → 10 类` 的全连接矩阵：

```text
12×12 输入产生 8×8×64，能接上；
20×20 输入产生 16×16×64，长度从 4096 变成 16384，矩阵立刻对不上。
```

这就是“没有固定长度全连接层”让空间输入可变的具体原因。

## U-Net 的额外限制

“全卷积”不等于任意边长都自动合法。原版 U-Net 有四次 `2×2` 池化，还用 valid convolution；每次池化前的边长要能整除 2。因此作者选择与结构兼容的 tile 大小。

## 代码出处

- `nn.Conv2d`、`nn.MaxPool2d`、上采样与逐点卷积都可作用于可变空间尺寸
- 固定输入限制通常来自 `Flatten + Linear`
