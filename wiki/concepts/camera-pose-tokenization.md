# Camera Pose Tokenization · 相机位姿 Token 化

## 一句话
把旋转四元数、平移方向和厘米尺度量化成 `[-1000,1000]` 的专用 token，让语言解码器像写结构化文本一样写相机位姿，再按确定公式还原成米制平移。

## 表示拆分
相对位姿分成三块：

- `q=(qx,qy,qz,qw)`：单位四元数，表示旋转；
- `u=(ux,uy,uz)`：单位方向向量，只说明往哪移动；
- `s_cm`：移动尺度，单位厘米，范围 1–1000，对应 0.01–10m。

四元数与方向分量都乘 1000 并取专用数值 token；`<1>` 在 scale 字段里表示 1cm。边界标签 `<frame> / <quat> / <offset> / <scale>` 负责结构。

## 完整数字例
第二帧相对第一帧不旋转，向 `(0.6,0,0.8)` 方向移动 `1.25m`：

```text
q = (0,0,0,1)           → <0><0><0><1000>
u = (0.6,0,0.8)         → <600><0><800>
s = 1.25m = 125cm       → <125>

<frame><quat><0><0><0><1000></quat>
<offset><600><0><800></offset><scale><125></scale></frame>
```

解码先除以 1000：`u=(0.6,0,0.8)`。再把厘米转米：

```text
t = u × s_cm / 100
  = (0.6,0,0.8) × 125 / 100
  = (0.75,0,1.00) m
```

自检：方向长度 `√(0.6²+0.8²)=1`；平移长度 `√(0.75²+1²)=1.25m`，与 scale 一致。

## 为什么不直接输出三个平移数
方向和尺度拆开后，方向永远落在 `[-1,1]`，能复用固定 `[-1000,1000]` 量化表；尺度则用厘米 token 覆盖 1cm–10m。代价是自回归解码必须同时保证四元数归一、方向单位长度和标签格式正确，模型生成的整数也不天然满足几何约束。

## 工程证据
论文为相机位姿复用基础模型词表最后 2,009 项：2,001 个数值 token 加 8 个结构 token。官方当前源码的解析器将 `<quat>` 与 `<offset>` 整数除以 1000，再按 `translation = offset × scale / 100` 还原米制平移。
