# Time-Delay Neural Network · TDNN · 时延网络

## 一句话
逐帧处理序列、每往上一层"看"的时间窗口翻倍的网络——本质就是带膨胀（dilation）的一维卷积。

## 直觉
痛点：一帧语音只有 10 毫秒，单看一帧什么都听不出来。要判断声学特征，得看一小段上下文（前后几帧）。但又不想像 RNN 那样一帧一帧串着算（慢、难并行）。

TDNN 的办法：每层只看**局部几帧**，但一层叠一层，越上层一个输出帧"间接看到"的原始帧越多。这跟 CNN 的感受野（[[receptive-field]]）是一回事，只不过卷的是时间轴。

[[x-vector]] 用的 TDNN 还有个聪明处：上层不是看相邻帧，而是**跳着看**（膨胀）。比如第 3 层看的是 {t−3, t, t+3}——隔 3 帧取一个。这样少算很多帧，却能把感受野撑得很大。等价于现在说的 dilated conv1d。

类比：看一篇文章，第一遍读你只看相邻的词（局部）；但你脑子里逐层抽象，到"理解整段意思"时，其实已经把很远的词的信息汇进来了——不需要每层都把所有词重读一遍。

## 怎么做的
x-vector 的 5 个帧级层（Table 1），"上下文"指这一层看输入的哪几帧：

```
层        看的帧（相对当前 t）      累计看到的原始帧数
frame1    [t−2, t+2]  连续 5 帧       5
frame2    {t−2, t, t+2}  跳着 3 个     9
frame3    {t−3, t, t+3}  跳着 3 个     15   ← 一个 frame3 输出帧已聚合 15 帧
frame4    {t}  只看自己               15
frame5    {t}  只看自己               15
```

到 frame3，每个输出帧已经"听过"15 帧（约 150ms）的上下文——足够刻画音素级声学了。frame4/5 不再扩上下文，只做逐帧的非线性变换、把维度抬到 1500，喂给 [[statistics-pooling]]。

## 数字例子
看感受野怎么逐层叠加。假设每层看相邻 3 帧（{t−1, t, t+1}）：

```
frame1 输出第 t 帧 ← 原始帧 t−1, t, t+1      （看 3 帧）
frame2 输出第 t 帧 ← frame1 的 t−1, t, t+1
                  = 原始帧 (t−2..t), (t−1..t+1), (t..t+2)
                  = 原始帧 t−2 ... t+2          （看 5 帧）
frame3 输出第 t 帧 ← frame2 的 t−1, t, t+1
                  = 原始帧 t−3 ... t+3          （看 7 帧）
```

✓ 每加一层，感受野两边各扩 1 帧：3 → 5 → 7。如果像 x-vector 那样**跳着取**（膨胀 d），扩得更快——隔 3 取一个时一层能让感受野多扩 6 帧，这就是为什么它只用 3 层就把上下文撑到 15 帧。

## 跳着取，不会丢掉相邻帧的关联吗？
不会。关键认知：**跳着取的是"已经聚合过的下层节点"，不是原始帧**。

两个设计保证了不丢：
1. **第一层是密集连续的，不跳**。x-vector 的 frame1 看 `[t−2, t+2]` 连续 5 帧，一个不落——相邻帧之间的细关联在这层就吃干净了。只有上层才开始跳。
2. **上层跳着取的每个点，本身已经是一个重叠窗口的总结**。等到 frame2 去取 frame1 的节点时，那个 frame1 节点已经装着一段相邻帧的信息了。

走一遍看有没有洞。frame1 在位置 p 覆盖原始帧 `{p−2..p+2}`。frame2 在 t 处取 frame1 的 `{t−2, t, t+2}`：

```
frame1[t−2] → 原始帧 t−4 .. t
frame1[t]   → 原始帧 t−2 .. t+2
frame1[t+2] → 原始帧 t   .. t+4
─────────────────────────────────
合起来      → 原始帧 t−4 .. t+4   连续 9 帧,无洞 ✓
```

frame2 虽然跳过了 frame1[t−1]、frame1[t+1] 这两个节点，但原始帧 t−1、t+1 的信息早被装进 frame1[t−2]、frame1[t]、frame1[t+2] 的窗口里了（窗口互相重叠）。拼起来铺满整段，没漏。

类比：写读书报告，复习层你只看 3 份章节摘要（跳着取），但每份摘要是有人逐页读完那一章写出来的（底层密集）——你没漏任何一页，只是在"摘要"这层跳，不在"读页"那层跳。

**trade-off（要诚实）**：上层看到的是邻居的"摘要"而非原始细节，所以极细粒度的逐帧分辨率确实被压掉了——但这是故意的，认人要的是整段嗓音特征不是每 10ms 的细节，还省算力。真出问题的是**底层也跟着跳**或几层跳步凑不好，覆盖会出现周期性空洞，叫 gridding（网格/棋盘伪影）；"第一层密集 + 上层重叠跳"恰好避开它。

## 跟普通卷积 / RNN 的对比
- vs RNN：RNN 串行，第 t 帧要等第 t−1 算完；TDNN 各帧并行，快得多。
- vs 朴素 conv1d：TDNN 加了膨胀（跳着取），同样层数下感受野更大，省算力。现代框架里就叫 dilated conv1d。
- vs Transformer 自注意力：注意力一步看全序列但是 O(T²)；TDNN 靠堆层逐步扩，线性，但看不到太远。
