# PageRank 实体覆盖 · 用链接结构挑出"必须见过的概念"

## 一句话
拿维基百科的链接图跑 PageRank，给每个条目算个"重要度"，取排名前 90% 当"必须覆盖的概念"（约 500 万个），再回头查训练集有没有图文对覆盖到它们。

## 直觉 · 覆盖全不全，谁说了算
[[krea-2]] 中训要保证模型"常识实体没盲区"——可"重要实体"有几百万个，怎么定哪些必须覆盖？人工列不现实。维基百科的**链接结构**天然是个投票系统：一个条目被越多、越重要的条目链接，它就越"中心"。PageRank 就是把这个"被谁链接、链接者本身有多重要"递归地量出来。

## 怎么做的 · 随机冲浪者的平稳分布
PageRank 把网页图看成一个**随机冲浪者**：站在某页，以概率 d（阻尼，常取 0.85）随机点一条出链跳走，以概率 1−d 随机瞬移到任意页。冲浪足够久后，**停在每页的概率**就是它的 PageRank。这正是一个 [[markov-chain]] 的**平稳分布**。

平衡式（每页的分 = floor + 指向它的页分给它的份额）：
```
PR(p) = (1−d)/N  +  d · Σ_{q→p}  PR(q) / 出链数(q)
        └ floor ┘     └ 上游按出链数均分它的分给下游 ┘
```
反复代入直到不变（幂迭代），就得到稳态。

## 为什么非要"瞬移"那一下（1−d）
先看**只跟着链接走、永不瞬移**会坏在哪：
- **死胡同**：有的页没有任何出链（图片页/PDF）。冲浪者跳进去就出不来，下一步没链可点 → 概率漏光，分加不回 1。
- **陷阱（rank sink）**：两个页只互相链、不链外面（A→B、B→A）。冲浪者一旦进去就永远打转，迭代到最后**全网概率都堆进这两页**、其它全饿成 0——可它俩根本不重要，只是把人困住了，结果完全失真。
- **不连通**：图分成互不相连的几块，有些页永远访问不到 → rank 恒为 0。

**瞬移就是给每一步装个逃生舱**：每步有 1−d 概率不点链接、随机跳到全网任意页。死胡同被弹出去、陷阱关不住你（概率能漏回外面）、每页都够得着。

直觉记 d=0.85：把它当**"会腻的冲浪者"**——真人点链接点着点着就腻了，关掉去搜个新词（=瞬移）。d=0.85 即"85% 再点一条、15% 腻了跳走"，平均点 `1/(1−d)≈6~7` 条就跳一次，贴近真人习惯。d 太大（0.99）→ 陷阱化解慢、收敛慢；d 太小（0.5）→ 瞬移太频繁，链接结构的信号被冲淡、排不出高下。

更深一层：PageRank 是 [[markov-chain]] 的平稳分布，而"平稳分布存在且唯一、幂迭代必收敛"要求这条链**遍历得到每个状态、不困在子集、不周期打转**（ergodic）。瞬移让任意页一步能到任意页，正好满足这个前提——所以它不只是工程补丁，更**在数学上保证这个数算得出、且只有一个解**。

## 数字例子 · 4 个页面
A→B→C→A 构成一个环，D→A（D 谁都没指向它）。`d=0.85, N=4`，floor=(1−0.85)/4=**0.0375**。
```
平衡式（代入链接关系）：
  PR(D) = 0.0375                     （没人指向 D → 只有 floor）
  PR(A) = 0.0375 + 0.85·(PR(C)+PR(D))   （C、D 都指向 A）
  PR(B) = 0.0375 + 0.85· PR(A)
  PR(C) = 0.0375 + 0.85· PR(B)
解出（反复迭代收敛）：
  PR(A)≈0.33   PR(B)≈0.32   PR(C)≈0.31   PR(D)=0.0375
```
✓ 自检：D 没有任何入链，永远卡在 floor `0.0375`——**孤儿页拿不到任何"链接传来的分"**，这 0.0375=(1−d)/N **全来自瞬移给的保底**（没有瞬移它就是 0、还会卡死冲浪者）。环里的 A/B/C 互相喂分，都涨到 ~0.32。把页面换成维基条目就是：被很多重要条目链接的"中心概念"分高，没人提的冷门词分低。取**排名前 90%** 当"必须覆盖"，约 **500 万**个实体，再去训练集里用全文检索查每个有没有被图文对覆盖到，没覆盖的就补采。

> 类比：朋友圈里"谁重要"不看自封，看**被多少重要的人提起**——PageRank 把这条递归算到底。
