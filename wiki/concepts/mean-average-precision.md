# AP / mAP · 检测器既不能乱报，也不能漏报

## 一句话

AP 是某个类别的“精确率—召回率”曲线面积；mAP 再把所有类别的 AP 求平均。COCO 常说的 AP 还会平均多个 IOU 命中门槛，因此比只看 AP50 严得多。

## 为什么“框对了几个”不够

检测器会给每个框一个置信度。阈值设高，只留下最有把握的框，通常误报少但容易漏；阈值设低，找回更多真物体，也会混进更多背景框。

所以要同时看两件事：

- **Precision，精确率**：报出来的框里，有多少是真的？<code>TP / (TP + FP)</code>。
- **Recall，召回率**：所有真物体里，找回了多少？<code>TP / (TP + FN)</code>。

其中 <code>TP</code> 是正确命中，<code>FP</code> 是误报，<code>FN</code> 是漏掉的真值。把置信度阈值从高往低扫，会得到一串 precision / recall 点；连起来就是 PR 曲线，曲线下面积就是 AP。

## 第一步：一个框怎样才算命中

预测框必须同时满足：

1. 类别正确。
2. 与某个尚未匹配的真值框达到指定 [[iou-intersection-over-union|IOU]] 门槛。

例如 AP50 使用 IOU≥0.50；AP75 使用 IOU≥0.75。第二个门槛更严：框大致盖住物体可能过 AP50，却过不了 AP75。

同一个真值只能匹配一次。后面再来一个高度重叠的重复框，会算误报，所以评测前通常先做 [[non-max-suppression|NMS]] 去重。

## 数字例子 · 从四个预测算到 AP=0.833

只看“狗”这一类。数据里共有 3 只狗，模型按置信度从高到低给出 4 个框。固定 IOU 门槛为 0.50，逐个判断结果是：

~~~text
预测顺序：  1    2    3    4
结果：      ✓    ✗    ✓    ✓
            TP   FP   TP   TP
~~~

每加入一个预测，就重算一次 precision 和 recall：

~~~text
第 1 个 ✓：TP=1, FP=0
Precision = 1/(1+0) = 1.000
Recall    = 1/3       = 0.333

第 2 个 ✗：TP=1, FP=1
Precision = 1/(1+1) = 0.500
Recall    = 1/3       = 0.333

第 3 个 ✓：TP=2, FP=1
Precision = 2/(2+1) = 0.667
Recall    = 2/3       = 0.667

第 4 个 ✓：TP=3, FP=1
Precision = 3/(3+1) = 0.750
Recall    = 3/3       = 1.000
~~~

第二个框是误报，只拉低 precision，没有找回新真值，所以 recall 仍是 0.333。

计算 AP 时，会先做 precision envelope：在每个 recall 位置，取“从这里往右还能达到的最高 precision”。这样可以消掉局部锯齿。三个真正让 recall 上升的位置变成：

~~~text
Recall 区间       最佳 Precision    面积
0     → 1/3       1.000             1/3 × 1.000 = 0.333
1/3   → 2/3       0.750             1/3 × 0.750 = 0.250
2/3   → 1         0.750             1/3 × 0.750 = 0.250

AP50 = 0.333 + 0.250 + 0.250 = 0.833
~~~

✓ 自检：三个 recall 区间宽度相加正好是 1；每段高度都在 0～1，因此总面积 0.833 也必然在 0～1。

真实 COCO 代码会在 101 个 recall 位置取样；上面用阶梯面积，是为了把“曲线下面积”拿笔算清楚。

## AP、mAP、AP50、AP75 到底差在哪

- **某类 AP50**：固定 IOU=0.50，算这个类别的 PR 曲线面积。
- **mAP50**：所有类别的 AP50 再平均。早期 VOC 常报这个口径。
- **AP75**：把命中门槛提高到 IOU=0.75，更看重定位是否贴合。
- **COCO AP**：分别在 IOU=0.50、0.55、…、0.95 十个门槛下算 AP，再跨门槛、跨类别平均。通常直接简称 AP 或 AP@[.5:.95]。

因此，YOLOv4 的 <code>43.5 AP / 65.7 AP50 / 47.3 AP75</code> 不是三个重复分数：65.7 是宽松门槛，47.3 是较严门槛，43.5 还把更严格的 0.80～0.95 一起算进去了。

## 最容易误读的三点

1. **AP 不是单个阈值下的精确率**：它扫过整条置信度排序，兼顾误报和漏报。
2. **AP50 与 COCO AP 不能直接横比**：一个只用 0.50 门槛，一个平均 0.50～0.95。
3. **mAP 的 m 是类别平均**：COCO 论文表格常把 mAP 简写成 AP；它不是只算某一个类。
