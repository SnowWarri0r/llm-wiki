# 强化引导 · 正负样本之差就是"该往哪走",CFG 不过是它的离线版

## 一句话
把高奖励样本的去噪方向减去低奖励样本的去噪方向,得到一个向量 `Δ`——它指向"怎么改能更高分",就是强化引导;而 [[classifier-free-guidance]] 的"条件−无条件"其实是同一种引导,只不过 CFG 那个方向是**离线固定**的,强化引导是**随奖励在线更新**的。

## 直觉 · CFG 和 RL 引导原来是一回事
[[classifier-free-guidance]] 每步算 `v = v_uncond + w·(v_cond − v_uncond)`,核心是那个差向量 `v_cond − v_uncond`——"因为给了条件而多走的那部分",把它放大就更贴 prompt。

[[diffusionnft]] 指出:这个"差向量当引导"的套路不止 CFG 能用。**用奖励把样本切成正(高分)负(低分)两堆**,正堆的去噪速度 `v⁺` 减去负堆的 `v⁻`,得到的 `Δ ∝ (v⁺−v_old) ∝ (v_old−v⁻)` 同样是个引导方向——只不过它说的是"怎么改能更高分",不是"怎么改更贴条件"。于是一句深刻的话:**CFG 是一种"离线的"强化引导**——cond−uncond 是预先固定好的方向,而真正的强化引导会随着奖励和训练在线变。

## 怎么做的 · 一个模型内建引导,替掉 CFG 的两模型
CFG 要同时养"条件模型"和"无条件模型"两套(或一套但跑两遍),推理每步两次前向。强化引导把方向 `Δ` 直接**学进单个模型**:[[negative-aware-finetuning]] 用隐式参数化让一个 `v_θ` 同时表达正负速度,采样时只取被引导的正速度 `v_θ⁺ = v_old + (引导)`,一遍前向出"已引导"结果。
```
CFG:        v = v_uncond + w·(v_cond − v_uncond)     方向固定、两模型/两遍前向
强化引导:   v_θ⁺ ≈ v_old + (2/β)·Δ                   方向随奖励在线学、单模型一遍
            Δ = 正样本方向 − 负样本方向(由 reward 切出)
```

## 数字例子 · 引导都是"外推一个差向量"
沿用 CFG 的向量直觉(标量简化):
```
CFG:   v_uncond=[1,0]  v_cond=[1,0.5]  差=[0,0.5]  → w=3: [1,0]+3·[0,0.5]=[1,1.5]
强化:  v_old=1.0       Δ=0.1(奖励切出的改进量) → β=0.5: v_θ*=1.0+(2/0.5)·0.1=1.0+0.4=1.4
                                                  采样用 v_θ⁺=(1−0.5)·1.0+0.5·1.4=1.2
```
✓ 自检:两边都是"在基准方向上,沿一个差向量外推"。CFG 的差是 `cond−uncond`(固定)、外推系数是 `w`;强化引导的差是 `Δ`(奖励切出、在线更新)、放大藏在 `2/β` 里。把它们看成同一族,就懂了为什么 NFT 能用一个在线学的引导替掉 CFG 那套离线两模型。
