# 扩散 RL 的似然墙 · 为什么不能照搬 GRPO,以及绕反向的代价

## 一句话
[[policy-gradient]]/[[ppo]]/[[grpo]] 都要"模型给这个样本的概率",LLM 逐 token 现成、扩散却算不出来(似然只能靠昂贵近似);主流绕法是把**反向去噪**切成多步马尔可夫过程让每步概率可算,但这条路锁死采样器、还可能让模型偏离前向过程。

## 直觉 · LLM 能直接做 RL,扩散卡在哪
RL 微调 LLM 顺手,是因为策略梯度要的 `π(动作)` 唾手可得:模型对每个 token 输出的就是概率分布,一条回答的概率=逐 token 概率连乘。奖励一来,乘上对数概率的梯度就能更新。

扩散没有这种现成概率。一张图是从噪声经几十步去噪来的,"模型生成这张图的概率"要对所有可能路径积分——只能靠**概率流 ODE** 或 **SDE 的变分下界**昂贵地近似。没有可算的似然,policy gradient 这一系就直接卡住。这就是似然墙。

## 怎么做的 · 把"去噪几十步"当成"几十个动作"
FlowGRPO / DanceGRPO 的转念:**生成一张图 = 从噪声 x_T 走到图 x_0 的一条几十步轨迹** `x_T→x_{T−1}→…→x_0`。把它看成一局游戏,每一步去噪就是一个"动作",整件事就成了一个**多步马尔可夫决策过程(MDP)**——跟 LLM 逐 token 生成同构,只是 token 换成"一步去噪":
```
状态  = 当前半成品 latent x_t（+时间步、prompt）
动作  = 你采出的下一步 x_{t−1}
策略 π = 采样器从哪个分布抽 x_{t−1}
```
**前提**:用随机采样器(SDE)——每步"往预测方向走一步、再撒一点噪声",于是每步抽 x_{t−1} 服从一个高斯:
```
x_{t−1} ~ N(μ, σ²)，  μ = x_t + v_θ(x_t,t)·Δt（走一步）， σ² = 撒进的噪声方差
```
高斯密度有闭式公式,抽出某个具体 x_{t−1} 就能直接算它的概率:
```
π(x_{t−1}|x_t) = 1/√(2πσ²) · exp( −(x_{t−1}−μ)² / (2σ²) )
```
**算不出的"整张图的概率"被拆成几十个能算的"每步概率"**(整条轨迹概率 = 各步连乘,log 域连加)——似然问题就这么绕过去了,GRPO 那套(组内归一优势、推 logprob)直接照搬。

### 数字例子 · 每步高斯可算
```
某一步：μ=0.50（走一步该到的位置）, σ=0.10, 实际抽到 x_{t−1}=0.55
密度 = 1/√(2π·0.01) · exp(−(0.55−0.50)²/(2·0.01))
     = 3.989 · exp(−0.125) = 3.989 × 0.8825 ≈ 3.52
一条 30 步轨迹的 log 概率 = 这 30 步各自 log 密度相加 → 完全可算
```
✓ 跟 LLM"一句话概率 = 各 token 概率连乘"一模一样的结构,只是 token 换成"一步去噪"。

## 代价 · 为了"每步可算"落的三个坑
能套 GRPO 了,但这套"在反向上逐步算概率"的做法继承三个毛病:
```
① 锁死 solver:要"每步高斯可算"就必须用"走一步+撒高斯"的一阶 SDE 采样器;扩散默认更快的 ODE/高阶 solver 每步不是能写出密度的简单高斯 → 用不了,被钉在最慢的采样器上
② 前后向不一致:只盯反向动态,丢掉对唯一前向过程的忠实,模型可能退化成"级联高斯"
③ CFG 难搞:扩散重度依赖 CFG,RL 里硬塞 → 变成又慢又繁的两模型优化
```
[[diffusionnft]] 正是冲着这三点来的:既然**前向加噪只有一条、反向去噪有无数条**,就把 RL 搬回唯一的前向过程,似然墙、solver 锁死、CFG 两模型一次性全绕开。
