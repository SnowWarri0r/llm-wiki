# Speaker Embedding · 声纹嵌入

## 一句话
把一段任意长的语音压成一个固定长度向量，代表"是谁在说"，不管说了什么。

## 直觉
类比：给每个人的**嗓音拍一张身份证照片**。照片尺寸固定（比如 512 维），不管你录 3 秒还是 30 秒。判断两段录音是不是同一个人，就是量两张照片的距离——近就是同一人，远就是不同人。

它替代的是"逐帧比对"这种行不通的做法：两段语音帧数不一样、内容也不一样，没法直接对齐着比。嵌入这一步先把"内容、长度、信道"这些噪声都甩掉，只留"谁"，再比就干净了。

关键分工：[[x-vector]] 这类模型抽的是**身份**；ASR（[[whisper]] / [[qwen3-asr]]）抽的是**内容**。同样喂音频，目标正交。

## 怎么做的
三代演进，骨架越来越像"神经网络嵌入 + 后端打分"：

```
i-vector (2011)：GMM-UBM 统计 + 因子分析 → 低维向量。纯统计，不学。
x-vector (2018)：TDNN 逐帧 → statistics pooling 沿时间汇成定长 → 段级全连接 → 512 维嵌入。
                 训练当分类任务（认 N 个说话人），用时扔掉分类头取中间层。
ECAPA-TDNN (2020)：x-vector + SE 注意力 + 多尺度聚合 + attentive pooling。
```

拿到两个嵌入怎么判同不同人（后端打分）：
- **余弦相似度**：最简单，量两个向量夹角。
- **PLDA**：再训一个概率模型，把"说话人间差异"和"同一人内部波动（不同录音/信道）"分开建模，打一个对数似然比分。比裸余弦更鲁棒，是 x-vector 时代标配后端。

## 数字例子
两段录音各自得到 2 维嵌入（真实是 512 维，这里缩小手算）：

```
录音 A（张三，安静）: a = [0.8, 0.6]
录音 B（张三，带噪声）: b = [0.6, 0.8]
录音 C（李四）:         c = [-0.7, 0.7]
```

余弦相似度 = 点积 /（各自模长），这里三个向量模长都≈1（0.8²+0.6²=1），省去分母：

- A·B = 0.8×0.6 + 0.6×0.8 = 0.48 + 0.48 = **0.96** → 很近 → 判同一人 ✓
- A·C = 0.8×(−0.7) + 0.6×0.7 = −0.56 + 0.42 = **−0.14** → 接近 0（夹角约 98°）→ 判不同人 ✓

✓ 同一个人即使一段带噪，嵌入还是靠在一起（0.96）；换个人立刻拉开（−0.14）。这就是"好嵌入"的标准：类内紧、类间散。数据增强（往 A 里掺噪声造出 B 一起训）就是逼网络学会"带噪也认得出是张三"。

## 跟 ASR 嵌入的对比
| | 声纹嵌入 | ASR 表示 |
|---|---|---|
| 抽什么 | 谁说的（身份） | 说了什么（内容） |
| 输出 | 一整段一个定长向量 | 逐帧/逐 token 序列 |
| 怎么用 | 算两向量距离判同异 | 解码成文字 |
