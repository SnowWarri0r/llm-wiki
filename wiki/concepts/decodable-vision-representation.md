# Decodable Vision Representation · 可解码视觉表示

## 一句话
把视觉标注编码成模型容易生成的文本或图像，同时保留一条确定的逆变换，使生成结果还能还原成 benchmark 要的框、点、深度、法线或 3D 坐标。

## 为什么“能生成”还不够
模型画出一张“看着像深度图”的灰度图，不等于能算 AbsRel；写一句“人在左边”，也不等于能算检测 F1。评测要求机器可读，所以编码必须同时满足：

1. 模型原生通道能学；
2. 输出格式有明确边界；
3. 数值能按确定规则还原；
4. 格式出错时解析器能识别，而不是默默猜。

## 数字例子 · 框坐标往返
图像宽 `W=640`、高 `H=480`，像素框为 `[64,96,448,384]`。按宽高分别归一化：

```text
x0' = x0 / W = 64 / 640  = 0.100
y0' = y0 / H = 96 / 480  = 0.200
x1' = x1 / W = 448 / 640 = 0.700
y1' = y1 / H = 384 / 480 = 0.800

文本 = <p>person</p><bbox>[0.100,0.200,0.700,0.800]</bbox>
```

其中 `x0,y0` 是左上角，`x1,y1` 是右下角；`W,H` 是输入图像像素尺寸；带撇号的是 `[0,1]` 归一化坐标。解码时反乘宽高，正好回到 `[64,96,448,384]`。三位小数意味着 640 宽图上的横向量化间隔约为 `0.001×640=0.64 px`。

## 图像编码也是协议

- 深度：有效 metric depth 先转 inverse depth，再归一化成灰度；近处更亮、远处更暗。
- 法线：单位向量 `(nx,ny,nz)` 的三个分量映射到 RGB 三通道。
- 二值 mask：固定前景色与背景色。
- point map：归一化 XYZ 分别写入 RGB。

论文没有公开深度归一化的完整端点公式，所以不能把任意 min-max 公式冒充官方实现。只看“逆深度”这一步的直觉例：三个像素深度 `d=[1,2,4]m`，逆深度 `1/d=[1,0.5,0.25]m⁻¹`；若仅为演示而在这三个值内做 min-max，灰度会是 `[255,85,0]`。这说明为什么近处更亮，但真实数据的归一化边界必须以转换脚本为准。

## 失败边界
可解码不等于无损：坐标有量化误差，生成文本可能漏标签，生成图像可能出现边缘模糊或颜色漂移。协议把错误变得**可定位、可度量**，并没有消灭错误。
