# SPP + PAN · 先把视野拉宽，再让高低层双向汇合

## 一句话

SPP 扩大最深层视野，PAN 让语义与位置在特征金字塔里来回流动。

## 直觉

最深层知道“这是什么”，浅层知道“边缘在哪”。只做自顶向下融合，语义能送到细网格，但细位置回不到深层。YOLOv4 先用 SPP 让深层同时看近处与远处，再用 PAN 补一条自底向上的回程路。

## 怎么做的

~~~text
最深层 13×13
  → 原特征直通 + 5/9/13 三档 max-pool 并排拼接（SPP）
  → 上采样 13→26→52，拼接浅层特征
  → 下采样 52→26→13，再向深层回传（PAN）
  → 52/26/13 三个检测头
~~~

三个 SPP 池化的 stride 都是 1，并配 padding，所以感受野变大，空间尺寸不变。直通支路常被画成“1×1”视野，但它并没有实际执行一次 1×1 最大池化。YOLOv4 的 modified PAN 用 concat 代替 shortcut 相加，先保留两路通道，再让后续卷积混合。

## 数字例子

官方结构进入 SPP 前是 <code>13×13×512</code>。一路保留原特征，另外三路分别做 5×5、9×9、13×13 池化：

~~~text
每一路输出 = 13×13×512
concat 后通道 = 4×512 = 2,048
输出尺寸 = 13×13×2,048
~~~

✓ 空间仍是 13×13，但每个位置同时带四档上下文。后续 1×1 卷积再压缩通道，避免 2,048 通道一路拖到底。
