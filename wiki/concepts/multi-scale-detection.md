# 多尺度检测 · 同一张图铺粗中细三层探测网

## 一句话

一次前向在多张不同分辨率的特征图上直接出框，让小、中、大物体各用合适的网格。

## 直觉

只用 13×13 网格，像拿粗方格纸描所有东西：大汽车还行，小路牌可能只占半格。多尺度检测同时铺三张方格纸：粗网格看全局，细网格抠位置，中间层接住两头之间的物体。

它和多尺度训练不是一回事：多尺度训练是不同 batch 换输入尺寸；多尺度检测是**同一次前向**同时在多层输出。

## 怎么做的

YOLOv3 对 416×416 输入使用三层：

~~~text
深层 13×13，stride 32，分配大 anchor
  ↑ 上采样2倍 + 拼接骨干26×26特征
中层 26×26，stride 16，分配中 anchor
  ↑ 上采样2倍 + 拼接骨干52×52特征
浅层 52×52，stride 8，分配小 anchor
~~~

上采样后的深层特征带强语义，骨干较早的同尺寸特征带位置细节；拼起来再卷积，三层各自输出预测。

## 数字例子

COCO 有 80 类，每个尺度每格预测 3 个 anchor：

~~~text
每个 anchor = 4坐标 + 1物体性 + 80类 = 85个数
每层通道 = 3 × 85 = 255

13×13层候选 = 13×13×3 = 507
26×26层候选 = 26×26×3 = 2,028
52×52层候选 = 52×52×3 = 8,112
总候选 = 507 + 2,028 + 8,112 = 10,647
~~~

✓ 三层都输出 255 通道，但空间位置数量不同；相加正好得到一次前向的 10,647 个候选框。

## 跟多尺度训练的对比

| 名称 | 变的是什么 | 一次前向有几个检测层 |
|---|---|---:|
| 多尺度训练 | 输入可能是 320、416、608 等 | 可以仍只有 1 层 |
| 多尺度检测 | 同时使用粗中细特征图 | YOLOv3 有 3 层 |
