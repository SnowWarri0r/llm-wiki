# ISG · 让一个稀疏 anchor 代表整段去噪区间

## 一句话

ISG（Intra-Segment Guidance，段内引导）在相邻两个少步 anchor 之间随机抽一个中间时刻，用“老师走前半段、学生走后半段”的路线，监督学生一次跨完整段的直接跳转。

## 为什么固定四个时间点还不够

四步生成器只在四个 coarse timestep 上接受监督。例如 SD 3.5 Large 使用 `{.25,.5,.75,1.0}`。但老师在相邻时间点上的重建能力并不平滑：SenseFlow 测得的一步重建误差会局部振荡，尤其在高噪声的 `.8–1.0` 区间。

所以一个手工选中的 anchor 可能恰好落在老师不可靠的位置；而两个 anchor 之间的大量时间点完全没有直接把信息送给学生。

## 三条路线

设当前段是 `τ_i→τ_{i-1}`：

1. 随机抽 `t_mid∈(τ_{i-1},τ_i)`；
2. 冻结老师，从 `x_{τ_i}` 走到 `x_{t_mid}`；
3. 冻结这条目标支路里的学生，从 `x_{t_mid}` 再走到 `x_tar`；
4. 活跃的学生从 `x_{τ_i}` 直接一步走到 `x_{τ_{i-1}}`；
5. 让直接结果贴近 `stopgrad(x_tar)`。

正式目标是：

\[
\mathcal L_{\mathrm{ISG}}^{(i)}
=\mathbb E_{\epsilon,t_{\mathrm{mid}}}
\left[
\left\|
x_{\tau_{i-1}}-\operatorname{stopgrad}(x_{\mathrm{tar}})
\right\|_2^2
\right].
\]

- `i`：当前是哪一段；
- `τ_i`：这段的高噪声端；
- `τ_{i-1}`：这段的低噪声端；
- `t_mid`：段内随机采样的中间时刻；
- `x_{τ_{i-1}}`：学生直接跨完整段得到的结果；
- `x_tar`：老师走前半段、学生走后半段得到的目标；
- `stopgrad`：目标数值参与比较，但目标支路不接收梯度；
- `E`：不断更换噪声和中间时刻后取平均。

## 一维数字例

取 `τ_i=.75、t_mid=.60、τ_{i-1}=.50、x_{τ_i}=.20`。为方便手算，把速度都当常数。

```text
老师前半段速度 vr = -2.0
xmid = .20 + (.60-.75)×(-2.0)
     = .50

目标支路的学生速度 vG(mid) = -1.5
xtar = .50 + (.50-.60)×(-1.5)
     = .65

学生直接跨段速度 vG(anchor) = -1.0
xdirect = .20 + (.50-.75)×(-1.0)
        = .45

LISG = (.45-.65)² = .04
```

反向传播只推动直接路线的 `.45` 靠近冻结目标 `.65`。下次训练会重新采 `t_mid`，于是同一个 anchor 逐渐吸收整段里不同位置的信息。

## 一个容易误读的地方

`x_tar` 不是“老师独自跑完整段”的答案。老师只走 `τ_i→t_mid`，后半段仍由学生完成，只是这条目标支路被 `no_grad / stopgrad` 冻结。这样既把老师的局部行为带进来，又避免要求学生严格复刻老师完整轨迹。

论文公式写平方 L2；2026-07-24 检查到的官方代码 commit `fafc81b7` 使用平滑绝对误差：

\[
\sqrt{(x_{\mathrm{direct}}-x_{\mathrm{tar}})^2+10^{-6}}-10^{-3}.
\]

它在误差很小时平滑，在误差很大时更接近绝对值，对异常大误差没有平方损失那么敏感。
