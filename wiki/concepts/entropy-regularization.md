# Entropy Regularization · 花钱买"别太早笃定"

## 一句话
在 loss 里加一笔"保持一点随机性就给奖励", 逼策略多探索一会儿、别一发现点甜头就锁死一招撞死。

## 直觉 · RL 的"过早笃定"病

策略会从"乱试"(高熵, 各动作概率差不多)慢慢变"果断"(低熵, 某个动作接近 100%)。这本身是好事 —— 学会了当然该果断。

坏就坏在**它容易笃定得太早**: 刚发现"一直往右冲"能蹭点分, 就立刻 all-in 这一招, 然后一头撞死在第一个障碍前, **再也不肯试别的**。这叫**过早收敛到局部最优**, 是 RL 最常见的翻车方式。表现就是某个指标(比如每局得分)冲一下就**躺平成一条直线** —— 策略完全确定化, 每局一模一样, 出不来了。

熵正则就是反制: 给策略**保留好奇心**, 别急着锁死, 多试试别的动作, 说不定能发现更好的玩法(比如"跳过那个障碍")。

## 怎么做的

在要最小化的 loss 里**减去**一项熵(等于鼓励熵变大):

```
total_loss = 策略损失 + 价值损失 − c · 熵
                                    ↑
                              c 就是 ent_coef, 探索力度旋钮
```

`ent_coef` 这个系数就是旋钮:

- `0` (关掉) → 完全贪心, 发现啥好立刻锁死 → **容易钻死胡同**
- `0.01` (常用默认) → 轻轻逼它多探索一会儿
- `0.05+` (很大) → 拼命逼它瞎试 → **可能永远不肯定下来**, 一直乱按学不精

最优在中间: 前期多探索找好策略, 后期收敛到果断执行。

**一个隐藏前提**: 熵奖励要起作用, **奖励的数值尺度得正常**。如果环境给的 reward 是几百上千, 那一点点熵奖励会被**淹没**, 熵照样秒崩。所以实战里熵正则常和**奖励归一化**(把 reward 压到 ±1 尺度)配套用 —— 尺度不对, 调熵也白搭。

这套思想往大了讲叫**最大熵强化学习** (maximum entropy RL): 在追高分的同时尽量保持策略多样性, 代表算法是 SAC。`ent_coef` 是它最朴素的一个旋钮。

## 代码出处

stable-baselines3 `ppo.py`: `entropy_loss = -th.mean(entropy)`, 再 `loss = policy_loss + ent_coef * entropy_loss + vf_coef * value_loss`。默认 `ent_coef=0.0`(图像任务常手动调到 0.01)。日志里的 `entropy_loss` 就是当前熵的相反数, 看它有没有过早塌到 0。
