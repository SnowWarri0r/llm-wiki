# YOLOv3 · 一张图铺三层探测网

YOLOv2 已经有 anchor、位置限位和多尺度输入，但真正出框的地方仍只有一层。YOLOv3 没推翻这套框参数化，而是把“在哪一层找物体”也纳入分工：深层粗网格找大物体，中层和浅层细网格接住中小物体，再用更深的残差骨干撑住语义。它不是一次范式革命，却把后来 YOLO 系最耐用的检测头骨架定了下来。

## 一句话

**保留 YOLOv2 的 anchor 解码，在 13×13、26×26、52×52 三层同时出框，让不同大小的物体去合适的特征层报到。**

## 它要解决的痛点

- **YOLOv2 只有一个检测头**：passthrough 把 26×26 细节搬到 13×13，却仍在粗网格上统一出框；COCO 小目标 AP 只有 5.0。
- **分类不一定互斥**：softmax 强迫一个框只能选一个类，但 Open Images 里同一物体可以同时是 Woman 和 Person。
- **骨干要更强，但不能把实时性吃光**：Darknet-19 很快，分类精度却落后更深的 ResNet；直接换 ResNet-152 又太慢。
- **密集候选的负样本太多**：一个真值只该交给最合适的 anchor，其他与真值重叠较高的候选不该被硬打成背景。

## 核心贡献

1. **三尺度检测**：[[multi-scale-detection]] 在 13×13、26×26、52×52 三张特征图分别预测，每层分到 3 个 anchor。
2. **新骨干**：[[darknet-53]] 用 1×1 / 3×3 卷积与 23 个残差块，在分类精度接近 ResNet-152 时快约 2 倍。
3. **多标签分类**：[[multi-label-classification]] 去掉类间 softmax，每个类别单独做 sigmoid + [[cross-entropy|二元交叉熵]]。
4. **更清楚的真值分配**：[[anchor-truth-assignment]] 只让全局最佳 anchor 当正样本；与真值重叠较高但没胜出的预测忽略物体性负损失。
5. **完整失败记录**：论文明确报告线性中心偏移、普通 anchor 中心公式、focal loss、双 IOU 阈值都没带来收益。

## 从 YOLOv2 继承了什么

框的四个解码公式没有变：

~~~text
b_x = sigmoid(t_x) + c_x
b_y = sigmoid(t_y) + c_y
b_w = p_w × exp(t_w)
b_h = p_h × exp(t_h)
~~~

<code>(c_x,c_y)</code> 是负责格子的整数坐标，<code>(p_w,p_h)</code> 是 anchor 宽高，<code>t_x,t_y,t_w,t_h</code> 是网络输出，<code>b_x,b_y,b_w,b_h</code> 是解码后的框。中心仍被 sigmoid 限在本格，尺寸仍表示相对 anchor 的倍率。YOLOv3 的关键不是换公式，而是让 9 种 anchor 分散到三张网格上。

## 三尺度预测 · 哪一层负责，不再一刀切

以 416×416 输入为例，总下采样 32、16、8 倍，得到三张检测网格：

| 检测头 | stride | 每格覆盖的原图步长 | 分配的 anchor | 候选数 |
|---|---:|---:|---|---:|
| 13×13 | 32 | 32 px | (116,90)、(156,198)、(373,326) | 13×13×3=507 |
| 26×26 | 16 | 16 px | (30,61)、(62,45)、(59,119) | 26×26×3=2,028 |
| 52×52 | 8 | 8 px | (10,13)、(16,30)、(33,23) | 52×52×3=8,112 |

总候选数是 <code>507+2,028+8,112=10,647</code>。每层输出通道都是：

~~~text
3 anchors × (4 坐标 + 1 物体性 + 80 类) = 255 channels
~~~

深层 13×13 语义强、位置粗，分给大 anchor；浅层 52×52 位置细、语义弱，分给小 anchor。网络还会把深层特征上采样 2 倍，与骨干中较早的同尺寸特征拼接：深层带“这是什么”，早层带“边缘具体在哪”。这个过程做两次，最终三层都直接预测，不再像 YOLOv2 那样只把细节搬到唯一检测头。

## 一条贯通的框分配例子

设输入仍是 416×416，一个行人真值框为：

~~~text
中心 = (200,168) px
宽高 = (60,120) px
~~~

先忽略中心，把 60×120 与 9 个真实 anchor 中心对齐，只计算 [[iou-intersection-over-union|形状 IOU]]。最接近的几项：

~~~text
与 30×61：交集=1,830，并集=7,200，IOU=0.254
与 62×45：交集=2,700，并集=7,290，IOU=0.370
与 59×119：交集=7,021，并集=7,200，IOU=0.975  ← 最高
与 116×90：交集=5,400，并集=12,240，IOU=0.441
~~~

因此全局最佳 anchor 是 59×119。它属于中号组，物体自动交给 26×26 检测头；不是先由人工判断“这是中目标”。该层 stride=16：

~~~text
中心网格坐标 = (200/16, 168/16) = (12.5, 10.5)
负责格索引 = (12,10)
格内小数 = (0.5,0.5)

t_x = logit(0.5) = 0
t_y = logit(0.5) = 0
t_w = ln(60/59) ≈ 0.0168
t_h = ln(120/119) ≈ 0.00837
~~~

宽高修正几乎是 0，说明聚类 anchor 给了很贴近的起点。正向解码：

~~~text
b_x = (sigmoid(0)+12) × 16 = 200 px
b_y = (sigmoid(0)+10) × 16 = 168 px
b_w = 59 × exp(0.0168) ≈ 60 px
b_h = 119 × exp(0.00837) ≈ 120 px
~~~

✓ 中心和宽高都回到真值。这个例子把“选 anchor、选检测层、选格子、预测四个数”串成了同一个动作链。

## 正样本、忽略、负样本怎么分

每个真值框只把**全局形状 IOU 最高**的一个 anchor 设为正样本，只有它学习坐标和类别。其余候选再看当前预测框与真值的重叠：

- 最佳 anchor：正样本，物体性目标为 1，并学习框与类别。
- 不是最佳，但与某个真值 IOU 高于阈值：忽略，不把它当背景处罚。
- 与所有真值都不够重叠：负样本，只学习物体性为 0。

论文写的忽略阈值是 0.5；公开 <code>yolov3.cfg</code> 使用 <code>ignore_thresh=.7</code>。两者不要混写成同一个数。这个忽略带的作用是：同一物体附近常有多个合理候选，最佳者负责就够了，其他好框不奖励，但也别硬骂成背景。

## 独立 logistic 分类 · 一个框可以同时有多个名字

YOLOv2 的 WordTree 已经处理过标签层级；YOLOv3 进一步去掉普通类别 softmax，让每个类独立做 sigmoid。假设一个框的三个类别 logit 是：

~~~text
Person = 2.2   → sigmoid = 0.900
Woman  = 1.4   → sigmoid = 0.802
Dog    = -2.0  → sigmoid = 0.119
~~~

Person 和 Woman 可以同时为高分。若硬做 softmax，三类会被迫分一块总和为 1 的蛋糕，大约变成 0.683、0.307、0.010，“女人也是人”反而互相抢概率。独立 sigmoid 配二元交叉熵，目标可以直接写成 <code>[1,1,0]</code>。

这并不表示最终一定保留所有标签：推理时仍可按阈值和数据集规则筛选。改变的是训练假设——类别不再被结构强迫互斥。

## Darknet-53 · 把残差接进 YOLO 骨干

Darknet-53 延续 Darknet-19 的 1×1 / 3×3 交替卷积，又加入 ResNet 式 shortcut。五个 stage 的残差块重复次数是 <code>1、2、8、8、4</code>，共 23 个残差块；每块两层卷积，主干深度明显增加，但恒等捷径让梯度能跨层直达。

论文在 256×256 ImageNet 单裁剪、同训练设置下给出的结果：

| 骨干 | Top-1 | Top-5 | Bn Ops | FPS |
|---|---:|---:|---:|---:|
| Darknet-19 | 74.1 | 91.8 | 7.29 | 171 |
| ResNet-101 | 77.1 | 93.7 | 19.7 | 53 |
| ResNet-152 | 77.6 | 93.8 | 29.4 | 37 |
| **Darknet-53** | **77.2** | **93.8** | **18.7** | **78** |

Darknet-53 的 Top-5 与 ResNet-152 相同，速度约 <code>78/37=2.11×</code>；Top-1 与 ResNet-101 接近，速度约 <code>78/53=1.47×</code>。论文还报告它达到最高的实测 BFLOP/s，意思不是运算更少就完事，而是结构更能把 GPU 喂饱。

## 训练配方

论文正文只交代：整图训练、不做 hard negative mining，继续使用多尺度训练、大量数据增强、BatchNorm 和 Darknet 框架，没有给完整 epoch 表。官方配置补出了可执行细节：batch 64、subdivisions 16、momentum 0.9、weight decay 0.0005、初始学习率 0.001、burn-in 1000 batch、在 400k 和 450k 步各乘 0.1，最多 500,200 batch；饱和度和曝光增强 1.5、hue 0.1，并开启随机输入尺寸。

这些配置是官方实现证据，不应冒充论文正文的消融结论。论文没有给“去掉某项增强掉多少 AP”的表。

## 实验 · AP50 很强，框的精修仍弱

COCO test-dev 上，YOLOv3-608 得到：

| 指标 | YOLOv2 | YOLOv3-608 | RetinaNet ResNet-101 |
|---|---:|---:|---:|
| AP@[.5:.95] | 21.6 | 33.0 | 39.1 |
| AP50 | 44.0 | 57.9 | 59.1 |
| AP75 | 19.2 | 34.4 | 42.3 |
| AP-small | 5.0 | 18.3 | 21.8 |
| AP-medium | 22.4 | 35.4 | 42.7 |
| AP-large | 35.5 | 41.9 | 50.2 |

小目标 AP 从 5.0 升到 18.3，三尺度检测确实补中了旧短板。但 AP50=57.9 接近 RetinaNet 的 59.1，综合 AP 却差 6.1；这说明 YOLOv3 很会给出“差不多盖住物体”的框，IOU 门槛升高后，边界贴合度仍落后。

还要把一个容易漏掉的口径说清：论文 Figure 3 的图注写明，作者修复了 YOLOv2 的数据加载 bug，这件事本身带来约 2 mAP。因此 v2→v3 的总提升不能全部记到三尺度检测或 Darknet-53 头上；论文没有给出足够消融，无法继续精确拆账。

速度曲线也要按输入尺寸读：YOLOv3-320 是 28.2 AP / 22ms，416 是 31.0 / 29ms，608 是 33.0 / 51ms。论文把它和不同 GPU 上报告的同期方法放在一起比较，并说明 M40 与 Titan X 速度接近；这些毫秒数适合读趋势，不该当成跨硬件绝对排名。

## 哪些尝试没用

论文专门留了一节失败实验：

1. **普通 anchor 的线性中心偏移**：用框宽高乘中心偏移，训练稳定性变差。
2. **直接线性预测 x、y**：去掉 sigmoid 后，mAP 掉了约 2 点。
3. **Focal loss**：mAP 约掉 2 点。作者猜测独立物体性已经挡掉大量易负样本，但明确承认原因没完全搞清。
4. **双 IOU 阈值分配**：模仿 Faster R-CNN 的正/忽略/负双阈值，没调出好结果。

这节很有价值：YOLOv3 不是把当年所有热门组件都装进去，而是保留在自己训练体系里确实稳定的组合。失败数字也有限，不能补写成不存在的完整消融表。

## 局限与作者的最后一问

- 严格定位仍弱：AP75 和综合 AP 明显落后 RetinaNet。
- 三尺度让小目标翻身，却没有让中大目标同步领先；论文自己也说需要继续调查。
- 9 个聚类中心、3 个尺度以及均分 3 个 anchor，是作者“差不多这么选的”，不是系统搜索出的最优结构。
- 论文最后没有停在榜单，而是点名视觉系统可能被用于隐私收割、监控与军事用途，提醒研究者对技术落地的伤害负有责任。这不是架构细节，却是论文真实的最后一节。

## 我的批注

- v3 最重要的变化不是“网络更深”，而是把**框大小与特征层级绑定**。anchor 匹配的结果同时决定负责形状和负责检测头。
- “多尺度训练”和“多尺度检测”最容易混：前者是同一网络换输入大小，后者是一次前向在三张特征图上同时预测。
- 10,647 个候选里，一个真值只分给一个正 anchor。候选变多不代表正样本也泛滥，ignore 规则正是密集预测能训稳的关键之一。
- v3 的分类头为后续开放标签与重叠标签留下了更自然的接口，但它仍是固定 80 类检测器，不等于开放词表检测。
- 论文语气像随手写的技术报告，内容却很诚实：哪几招借来的、哪几招失败、指标哪里不好看、技术可能伤害谁，都没有藏。
