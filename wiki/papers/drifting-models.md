# Generative Modeling via Drifting · 把反复修图留在训练时，生成时一步出图

## 一句话
Drifting 不让模型在推理时沿 ODE 反复去噪，而是在训练时不断把当前生成样本推向一个临时目标：这个目标由“靠近真实样本”和“远离当前生成样本”两股力合成。训练收敛后，生成器一次前向就能把噪声映射成图像。

## 先看完整拼图

### 训练时
1. 噪声 `ε ~ p_ε` 经过生成器，得到 `x=f_θ(ε)`；生成器把噪声分布搬成当前生成分布 `q_θ=[f_θ]#p_ε`。
2. 冻结的特征网络 `φ` 把生成图、真实图都变成更容易比较语义的特征。
3. 对每个生成特征，核函数给附近样本更大的权重：吸向真实特征 `V⁺_p`，同时减去指向其他生成特征的 `V⁻_q`，得到净漂移 `V_{p,q}`。
4. 把 `φ(x)+V` 当成本轮临时目标，并用 `stopgrad` 冻住；梯度只负责让生成器下一轮更接近这个目标。
5. 训练迭代反复重算 `q` 和 `V`，逐渐把整个生成分布推向数据分布。

### 推理时
只保留 `ε → f_θ(ε) → 图像`。特征网络、核函数、正负样本和漂移场全部删除，因此是 1 NFE。

## 1. Pushforward：`q=[f]#p_ε` 到底说了什么
`#` 不是乘法。它表示“把一批概率质量经过函数 `f` 搬到新位置”。例如噪声只有 `-1`、`+1` 两种结果，各占 50%；若 `f(-1)=0`、`f(+1)=2`，输出分布 `q` 就是在 `0` 和 `2` 各有 50% 的概率。这里没有先规定 `q` 长什么样；`q` 完全由噪声分布和生成器共同决定。

见 [[pushforward-distribution]]。

## 2. 漂移场怎么计算

对生成点 `x`，先算两股局部平均位移：

```math
V_p^+(x)=\frac{1}{Z_p}\mathbb E_{y^+\sim p}[k(x,y^+)(y^+-x)]
```

```math
V_q^-(x)=\frac{1}{Z_q}\mathbb E_{y^-\sim q}[k(x,y^-)(y^--x)]
```

```math
V_{p,q}(x)=V_p^+(x)-V_q^-(x)
```

- `x`：当前生成样本的特征。
- `y⁺ ~ p`：从真实数据分布抽到的正样本特征。
- `y⁻ ~ q`：从当前生成分布抽到的负样本特征。
- `k(x,y)`：相似度权重；越近，影响越大。
- `Z_p=E_p[k(x,y⁺)]`、`Z_q=E_q[k(x,y⁻)]`：把权重归一化，避免样本数量或权重总量改变步长。
- `V⁺`：指向附近真实样本的加权平均位移。
- `V⁻`：指向附近生成样本的加权平均位移；总公式把它减掉，所以实际效果是远离当前生成团块。

论文使用的核是：

```math
k(x,y)=\exp\left(-\frac{\lVert x-y\rVert_2}{\tau}\right)
```

`τ` 是温度：小 `τ` 只看非常近的邻居，大 `τ` 会让更远的样本也参与。最终模型同时使用 `0.02、0.05、0.2` 三个尺度。

### 一维数字例
设 `x=0`、`τ=2`，真实点是 `2、4`，生成点是 `-1、0.5`：

- 真实点核权重归一化后是 `0.731、0.269`，所以 `V⁺=0.731×2+0.269×4=2.538`。
- 生成点核权重归一化后是 `0.438、0.562`，所以 `V⁻=0.438×(-1)+0.562×0.5=-0.157`。
- 净漂移 `V=2.538-(-0.157)=2.695`，当前点向右移动。

注意：`V⁻` 本身仍是“指向生成邻居”的向量；真正的排斥来自总公式前面的减号。

## 3. 为什么要同时吸引和排斥
- 只有吸引：很多生成点可能一起扑向少数真实样本，覆盖面会塌缩。
- 只有排斥：样本会散开，却不知道真实数据在哪里。
- 两者成对：真实分布决定“该往哪去”，当前生成分布决定“哪些地方已经挤得太满”。

它能降低 mode collapse 风险，但不能据此宣称“天然杜绝 mode collapse”；结果仍依赖特征、核、采样和优化。

## 4. 反对称只证明了哪一步

论文构造满足：

```math
V_{p,q}(x)=-V_{q,p}(x)
```

因此当 `q=p`：

```math
V_{p,p}(x)=-V_{p,p}(x)\quad\Rightarrow\quad V_{p,p}(x)=0
```

这严格证明的是 **`q=p ⇒ V=0`**：匹配时一定停。

反过来的 **`V=0 ⇒ q=p`** 并不只靠反对称就成立。论文附录给出的是可辨识性的讨论：经过平滑后分布有足够支撑、有限维展开中的探针点足够多且相互作用向量不退化时，零漂移才有望唯一锁定匹配分布。作者也明确承认一般条件仍是开放问题。因此正文不能把两边写成无条件等价。

## 5. `stopgrad` 不动点损失：数值相等，梯度不相等

```math
\mathcal L(\theta)=\mathbb E_{\epsilon}\left[
\left\lVert f_\theta(\epsilon)-
\operatorname{sg}\big(f_\theta(\epsilon)+V_{p,q_\theta}(f_\theta(\epsilon))\big)
\right\rVert_2^2
\right]
```

- `f_θ(ε)`：当前生成结果。
- `V_{p,qθ}`：按当前生成分布算出的漂移。
- `sg(·)` / `stopgrad`：前向时保留数值，反向时把括号内部当常数。
- 损失在数值上等于 `E||V||²`，但训练时**不会穿过 `V`、核权重或 `q_θ` 求导**。

标量例：令 `f_θ(1)=θ`、当前 `θ=1`、本轮算出 `V=0.4`。临时目标冻结为 `1.4`，损失是 `(1-1.4)²=0.16`，梯度为 `2(1-1.4)=-0.8`。学习率 `0.25` 时，`θ` 更新成 `1.2`，朝冻结目标走了一段。下一轮再用新的生成分布重算漂移和目标。

所以“训练迭代替代推理时 ODE 迭代”是分布层面的直觉，不是说优化器在逐步数值求解同一条 ODE。

## 6. Mini-batch 里如何估计漂移
真实分布和当前生成分布都拿不到完整积分，训练时只能用一批样本近似：

1. 生成特征与正样本、负样本两两计算距离，得到一个相似度矩阵。
2. 屏蔽生成样本与自己的负样本配对，避免“自己排斥自己”。
3. 对每一行做 softmax：对当前查询点来说，哪些候选更近。
4. 再对每一列做 softmax：避免某个候选同时吸走所有查询点。
5. 两个权重取几何平均 `A=sqrt(A_row A_col)`，再加权求位移。

消融中，仅做行归一化的 FID 是 `8.92`，行列都做是 `8.46`，完全不归一化是 `10.54`。这说明双向归一化不是排版细节，而是可测量的训练稳定因素。

## 7. 为什么在特征空间漂，而不是像素空间硬比
像素欧氏距离把“移动几个像素”和“语义完全不同”都当作数值差；高维像素空间里的核权重也容易变得近乎一样，漂移方向失去语义。

因此论文用冻结的 latent-MAE / ConvNeXtV2 特征 `φ_j(x)`，在多个尺度 `j` 上计算：

```math
\mathcal L=\sum_j \mathbb E\left[
\left\lVert \phi_j(x)-\operatorname{sg}(\phi_j(x)+V_j)\right\rVert_2^2
\right]
```

`φ` 的参数不更新，但梯度会穿过 `φ` 和 VAE decoder 回到生成器；`V_j` 仍然冻结。它不是“把预训练特征当标签”，而是借特征空间定义什么叫语义上接近。推理时整个 `φ` 被删掉，没有额外成本。

直接去掉特征编码器，ImageNet 训练无法正常工作。像素模型从弱 MAE 特征的 FID `32.11`，换强 MAE 到 `9.35`，再加入 ConvNeXtV2 特征到 `3.70`，说明特征质量是核心条件之一。见 [[perceptual-loss]]。

## 8. CFG 为什么被搬进训练时
普通 [[classifier-free-guidance]] 在推理时需要条件、无条件两次网络前向。Drifting 改造负分布：

```math
\widetilde q(\cdot|c)=(1-\gamma)q_\theta(\cdot|c)+\gamma p_{data}(\cdot|\varnothing)
```

平衡时令 `\widetilde q=p_data(·|c)`，整理得：

```math
q_\theta(\cdot|c)=\alpha p_{data}(\cdot|c)-(\alpha-1)p_{data}(\cdot|\varnothing),
\qquad \alpha=\frac{1}{1-\gamma}\ge 1
```

例如 `α=2` 对应 `γ=0.5`：训练负样本一半来自当前条件生成，一半来自真实无条件数据；平衡关系变成 `q=2p_cond-p_uncond`，类似传统 CFG 的条件外推。模型还把 `α` 当条件输入，所以推理时只跑一次，不需要额外无条件分支。

这只是分布层面的外推表达，不能把右边逐点当成始终非负的普通混合分布。

## 9. 模型和训练配方

### Latent 版
- SD-VAE 把 `256×256×3` 图像压成 `32×32×4` latent；生成器直接从同形状高斯噪声预测 latent。
- patch size 2 后得到 `16×16=256` 个 token；DiT 输出 latent，再由 VAE decoder 还原图像。

### Pixel 版
- 输入、输出都是 `256×256×3`；patch size 16，同样得到 256 个 token；不使用 VAE。

### 共同结构
- DiT-B/2：宽度 768、12 层；DiT-L/2：宽度 1024、24 层。
- 16 个 register token；32 个随机 style token，来自 64 个可学习向量的码本。
- 使用 SwiGLU、RoPE、RMSNorm、QK-Norm 和 AdaLN / in-context conditioning。

SOTA latent L/2 使用 200k step、约 1280 epoch、AdamW、学习率 `4e-4`、10k warmup。有效生成 batch 是 8192：每步 128 个类别，每类 64 个生成样本；正样本按类从队列/数据中取，另有全局无条件队列。项目 README 报告该规模使用 128 张 TPU v6e，说明“一步推理”没有消除训练成本，只是把成本前移。

## 10. 实验：论文真正证明了什么

### ImageNet 256×256
- latent：B/2 `FID 1.75`，L/2 `1.54`，均为 1 NFE。
- pixel：B `1.76`，L `1.61`，均为 1 NFE。
- 在论文列出的一步原生生成方法中，L/2 的 `1.54` 最好；但不是所有方法中的总体最佳：多步/其他设定的 RAE+DiT 报 `1.13`，SiD2 pixel 报 `1.38`。
- 一些 diffusion/flow 基线表中写作 `250×2` NFE，因为常规 CFG 每步要跑条件与无条件两次；不能简化成“所有扩散都固定 250+ NFE”。

### 关键消融
- 打破吸引/排斥系数对称：默认 `8.46`；吸引乘 1.5 变 `41.05`，排斥乘 1.5 变 `46.28`；只留吸引是 `177.14`。
- 每类正样本从 1 增到 16、32、64：FID `20.43 → 10.39 → 8.97 → 8.46`。
- 单温度 `τ=.02/.05/.2`：`10.62/8.67/8.96`；三温度合用 `8.46`。
- 随机 style token：`8.86 → 8.46`，有帮助但不是主要增益来源。

### 机器人控制
把 Diffusion Policy 的多步动作去噪器换成 Drifting 后，100 NFE 变 1 NFE。部分任务提升，如 ToolHang state `0.30→0.38`、BlockPush phase-1 `0.36→0.56`；但也有退步，如 ToolHang visual `0.73→0.67`、PushT state `0.91→0.86`。因此应写“多数任务接近或部分超过”，不能写成无条件追平。

## 11. 它和 GAN、DMD、MMD、Flow Matching 的关系
- **GAN**：都在做分布间对抗式纠偏，但 Drifting 没有可学习判别器，局部力来自固定特征空间和核权重。
- **[[dmd-distillation]]**：都有“吸向目标、排斥当前模型”的分布匹配味道；DMD 使用扩散 score 差并依赖老师，Drifting 直接用样本间核漂移，不需要预训练生成老师。
- **MMD**：特定核下可以诱导类似漂移，但本文加入核归一化、显式漂移步和训练时 CFG；作者报告直接按 MMD 框架没有得到可用结果。
- **[[flow-matching]]**：都学习噪声到数据的映射；Flow Matching 学连续时间速度场并在推理时积分，Drifting 让参数更新承担跨轮分布演化，部署时只保留终态生成器。

## 12. 局限与开放问题
1. `V=0 ⇒ p=q` 的一般可辨识性没有完整理论保证。
2. ImageNet 成功高度依赖强特征提取器；这会把其偏差带进训练目标。
3. 训练要构造大规模正负配额、队列和两两距离矩阵；推理便宜不等于训练便宜。
4. 本文主要验证 ImageNet 256 和有限机器人任务，尚不足以推出文本到图像、视频或更高分辨率同样稳定。
5. CLIP 最近邻检查只能说明没有发现明显近邻复制，不能证明模型绝不记忆训练数据。
6. 官方仓库复现指标与论文有轻微差异，例如 latent L FID `1.53` vs 论文 `1.54`，应注明运行版本和评测口径。
