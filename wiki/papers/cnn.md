# CNN · 卷积神经网络

## 一句话
用一个小核滑遍整张图、一路共用同一套权重，把"局部性 + 平移不变"两条视觉先验焊进网络结构 —— 参数比全连接少几个数量级，且天生认得出挪了位置的目标。

## 它要解决的痛点
图像拉平喂全连接（MLP）有两个致命浪费：① 参数爆炸（224×224 → 第一层就上千万）；② **位置写死**——同一个特征出现在不同位置要重新学。CNN 用局部连接 + 权重共享一次解决两者。

## 核心贡献（这页讲的几件事）
- **卷积**：小核滑窗，每处逐元素相乘求和 → feature map。stride/padding 控尺寸。详见 [[convolution]]。
- **两条先验焊进结构**：局部连接（只看小窗口）+ 权重共享（一套核滑遍全图）→ 参数暴降 + 平移不变。这是 CNN 的 [[inductive-bias]]。
- **多核 → 层级特征**：一层多核各抓一种模式，层层组合：边缘 → 纹理 → 部件 → 物体。
- **pooling + 感受野**：下采样把分辨率砍半，让 [[receptive-field]] 指数级扩张，高效从"看个边"撑到"看整只猫"。详见 [[pooling]]。
- **谱系**：LeNet → AlexNet → VGG → [[resnet]]（残差破退化、做到上百层）→ 被 ViT 用 [[patch-embedding]] + 纯注意力填平护城河。

## 关键概念 → 概念页
- [[convolution]] · 滑窗点乘求和（原子操作）
- [[pooling]] · 下采样、扩感受野
- [[receptive-field]] · 一个输出能看到原图多大一片
- [[inductive-bias]] · 局部性+平移不变是焊进结构的先验
- [[patch-embedding]] · ViT 用它替掉卷积进视觉
- [[resnet]] · 卷积网络做深的关键
- [[self-attention]] · 一层即全局，对照卷积"靠深度慢慢撑"

## 我的批注 / 疑问
- CNN vs Transformer 是"先验 vs 通用"两端：卷积省、快、小数据友好；注意力天花板高但要更多数据。这正是 [[inductive-bias]] 主线——偏置在小数据是宝、大数据是包袱。
- "感受野靠深度慢慢撑" vs 注意力"一层即全局"是理解 ViT 为什么能换掉 CNN 的关键。
- ResNet 既是 CNN 做深的终点，又给两年后 Transformer 的 sublayer 留好了残差模板。
