# LTX-2 · 让声音和画面在同一段去噪过程中彼此校正

LTX-2 是一个开源的联合音视频生成模型。它没有先做视频再补音效，也没有把音频和视频硬塞进同一种 token；它保留两套适合各自模态的 VAE 与 Transformer 流，只在每一层用双向交叉注意力交换信息。这样，嘴型可以追着台词调整，碰撞声也能反过来参考画面里的接触时刻。

这篇技术报告的架构信息很有价值，但训练配方和评测披露明显不足。下面会把**论文明确写出的事实**、**官方代码补充的实现细节**和**为了讲清机制而造的小数字例子**分开，不把三者混在一起。

## 一句话

**视频负责空间和运动，音频负责时间和声场；两条流各自去噪，又在每一层沿共同时间轴双向对答案。**

## 先看总路线

```text
prompt ─→ 冻结 Gemma 3-12B ─→ 全层特征聚合 ─→ 视频 / 音频各自的双向 connector
                                                            │
随机 video latent ─→ 14B video stream ─┐                    │
                                       ├─ 一次前向含 48 个双流 block
随机 audio latent ─→  5B audio stream ─┘   每层做 self → text → A↔V → FFN
                ↑                              │
                └──── 采样器按 σ 更新两块 latent，重复若干步 ─────┘

最终 video latent ─→ Video VAE decoder ─→ RGB frames
最终 audio latent ─→ Audio VAE decoder ─→ mel ─→ vocoder ─→ 24 kHz stereo
```

上面是纯文本生成路径。训练时才会把真实视频、音频送进各自的 VAE encoder 得到干净 latent，再加噪学习速度；生成时没有 raw video / raw audio，起点是随机 latent。48 层是**一次模型前向**的深度，不是 48 个采样步。

论文要学的是三层分工：

1. **表示层**：视频与音频用不同 VAE，分别压缩空间—时间和声音频谱。
2. **生成层**：14B 视频流与 5B 音频流同时做 flow matching 去噪。
3. **对齐层**：每个 block 里的双向交叉注意力只共享时间坐标，让两边在事件发生时刻互相校正。

## 它要解决的痛点

### 先视频、后配音为什么不够

`T2V → V2A` 看似合理：先把画面生成出来，再让声音模型看视频配音。但它只能让声音服从已经定稿的画面。若台词节奏需要嘴型延长半秒，视频已经无法回头修改。

反过来的 `T2A → A2V` 也一样：语音能决定嘴型，却无法事先知道最终房间有多大、地面是什么材质、物体在哪里碰撞，因此混响、脚步和 Foley 音效缺少视觉依据。

同一个事件通常是双向约束：

- 台词的音素和节奏约束嘴型；
- 嘴唇、说话人位置和镜头切换约束声音应该来自谁、来自哪里；
- 物体接触画面约束撞击声的时刻；
- 房间大小和材质约束混响；
- 音乐节拍也可能反过来影响剪辑和动作节奏。

所以真正的目标不是分别拟合 `p(video|text)` 和 `p(audio|video,text)`，而是尽量建模联合分布 `p(video,audio|text)`：在生成过程中，两边都还可以改。

### 为什么不把音视频做成完全相同的一条流

视频 token 要表示 `x、y、时间`，而音频主要沿时间展开；视频空间细节的容量需求也更高。强行共用一套宽度、VAE 和位置编码，会让音频承担不需要的空间计算，或让视频容量不足。LTX-2 因而采用 [[asymmetric-dual-stream-transformer]]：结构深度一致，但宽度不同，论文给出的容量是视频 14B、音频 5B。

## 一块双流 Transformer 到底做什么

每个 dual-stream block 依次做四件事，两条流都有一套自己的参数：

```text
① 模态内 self-attention
   video token 互相看：建立空间布局和运动
   audio token 互相看：建立语音、音乐、环境声的时间连续性

② text cross-attention
   两条流分别读取为自己准备的文本条件

③ audio ↔ video cross-attention
   video query 读取 audio K/V；audio query 同时读取 video K/V

④ FFN
   对每个 token 做通道方向的非线性变换
```

论文正文没有写 block 数；当前官方 `ltx-core` README 与实现给出 **48 层**。这是代码补充，不应误写成论文表格中的公开训练配置。

### “双向”不是串行改两遍

官方实现先保存交换前的 `video`、`audio` 隐状态快照，再分别计算 A→V 与 V→A。第二个方向不会读取第一个方向刚写回的结果。因此一层里的两次交换是对称的：都从相同的层起点出发，再各自加回残差。

## 交叉注意力怎么把撞击声对准画面

[[audiovisual-cross-attention]] 的两个方向是：

```text
A → V：Q 来自 video，K/V 来自 audio
V → A：Q 来自 audio，K/V 来自 video
```

以 A→V 为例：

\[
A_{V\leftarrow A}
=\operatorname{softmax}\!\left(\frac{Q_VK_A^\top}{\sqrt d}\right)V_A
\]

- `Q_V ∈ R^(N_v×d)`：`N_v` 个视频 token 提出“我现在需要什么声音信息”的查询；
- `K_A ∈ R^(N_a×d)`：`N_a` 个音频 token 提供可匹配的索引；
- `V_A ∈ R^(N_a×d)`：真正被加权取回的音频内容；
- `d`：共同的注意力通道宽度；除以 `√d` 是为了防止维度大时点积过大、softmax 过早饱和；
- softmax：把每个视频 query 对所有音频 token 的分数变成和为 1 的权重。

反方向只需交换 A/V 下标。

### 一个时间对齐的玩具例子

某个视频 token 表示 `0.40s` 的杯子碰桌面。它对 `0.36s / 0.40s / 0.44s` 三个音频 token 的缩放后分数是 `[1,3,1]`：

```text
softmax([1,3,1]) ≈ [0.107, 0.787, 0.107]
三个音频 value 的“碰撞强度” = [0.2, 1.0, 0.1]
取回的强度 = .107×.2 + .787×1.0 + .107×.1 ≈ .819
```

权重集中到同一时刻的强碰撞声。真实模型当然是高维向量、多头和多层；这个例子只解释一次 attention 的算术，不是论文实验数据。

## 为什么跨模态时只用 1D RoPE

视频 self-attention 使用 3D [[rotary-position-embedding]]，给 token 注入 `(x,y,time)`；音频 self-attention 使用 1D 时间 RoPE。音视频互相注意时则只保留共同的**时间轴**：

```text
video 内部：这是谁、在画面哪里、什么时候发生
audio 内部：这是什么声音、什么时候发生
跨模态：   两件事是不是发生在同一个时刻
```

音频没有天然的画面 `x,y` 坐标。如果把视频空间坐标硬塞进跨模态位置比较，就等于要求“左上角画面”与某个不存在的“左上角声音”对齐。1D temporal RoPE 只回答时间是否接近；至于视频 query 应该关注嘴、车还是手，仍由内容相似度学习。

论文 Figure 3 还显示：V2A 注意力可以随说话人移动到相应人物和嘴部。不过图中热力图平均了 heads 与 layers，且 V2A/A2V 分别取推理前、后 1/3 步，并不能据此断言每一层都形成同样的定位模式。

## Cross-modality AdaLN：为什么还要让“噪声进度”开门

两条流未必处于相同噪声强度。比如做 V2A 时，给定视频可以是干净条件（`σ_video=0`），待生成音频仍很嘈杂。若不告诉交叉注意力双方各自有多可靠，它可能让一团未成形的音频过度改写干净视频。

论文和当前官方代码合起来给出两层控制：

1. **自己的时间步负责整理自己。** 视频、音频分别用本流的 diffusion timestep 生成 scale/shift，再对送入 cross-attention 的隐藏状态做 AdaLN。
2. **另一模态的噪声进度负责控制写回量。** 交叉注意力输出再乘一个 gate；这个 gate 由对方模态的 `σ` 生成。对方当前是否可靠，决定它的话应该加回多少。

简化写法是：

\[
h'_V=h_V+g(\sigma_A)\odot
\operatorname{CrossAttn}\!\left(
\operatorname{AdaLN}(h_V;\sigma_V),
\operatorname{AdaLN}(h_A;\sigma_A)
\right)
\]

- `h_V,h_A`：交换前的视频/音频隐藏状态；
- `σ_V,σ_A`：两条流当前的噪声级别；
- `AdaLN(h;σ)`：先归一化，再由 `σ` 生成逐通道缩放和平移；
- `g(σ_A)`：由音频噪声级别算出的门；
- `⊙`：逐通道相乘；
- 残差 `+h_V`：即使门很小，视频原有信息仍能通过。

论文第 5 页一句话把 scale/shift 描述成“由另一模态隐藏状态条件化”，紧接着 3.1.2 又说由各流 diffusion timestep 调制、由另一流 timestep 控门。当前开源实现与后者一致，因此这里按可核对的实现解释，并把前一句视为不够严谨的概括。

## 文本为什么不能只拿 Gemma 最后一层

LTX-2 冻结 Gemma 3-12B，但不只读取最后一层。论文假设：较早层更保留字形、音素等局部信息，较深层更偏向复杂语义；语音生成既需要“说什么”，也需要“怎么准确发音”。

所有层输出堆成：

\[
H\in\mathbb{R}^{B\times T\times D\times L}
\]

- `B`：batch 大小；
- `T`：文本 token 数；
- `D`：Gemma 每层的隐藏维度；
- `L`：抽取的层数。

然后：

1. 每一层跨 `T,D` 做 mean-centered scaling；
2. 把层维并进特征维，得到 `[B,T,D×L]`；
3. 用可学习投影 `W∈R^(D×L × D_target)` 压回目标宽度。

当前官方 V1（19B LTX-2）代码把有效 token 的特征减均值、除以该层最大值与最小值之差并乘 8，padding 清零，再拼层和过线性层。论文只写了 mean-centered scaling，没有披露这个精确公式；它属于实现补充。

### 一个只演示形状的小例子

假设一个 token 在三层的 2 维表示分别为 `[1,0]、[0,2]、[3,1]`。拼接后是 `[1,0,0,2,3,1]`。若教学用的 `W` 只是把第 1/5 维加到输出 1、把第 2/4/6 维加到输出 2：

```text
output = [1+3, 0+2+1] = [4,3]
```

只用最后一层会得到 `[3,1]`；聚合后可以同时保留前层信号。真实 `W` 是训练得到的稠密浮点矩阵，不是这个手写选择器。

`W` 在一个短暂初始阶段与 LTX-2 一起用 diffusion MSE 优化，Gemma 始终冻结；随后 `W` 也冻结。报告没有给出这个阶段的步数、数据量或学习率。

## Thinking tokens 不是“模型在偷偷写思维链”

Gemma 是 decoder-only 模型，原始文本 token 的因果注意力只能看左侧。LTX-2 在后面加一个全双向 text connector，让整句重新混合；视频和音频各有自己的 connector。

[[thinking-tokens]] 实际是可学习 registers：用它们替换固定长度序列里的 padding 槽。比如 8 个位置里 prompt 只占 5 个：

```text
原来：[猫][跳上][桌子][杯子][碎裂][PAD][PAD][PAD]
改后：[猫][跳上][桌子][杯子][碎裂][R1 ][R2 ][R3 ]
```

双向 connector 跑完后，`R1-R3` 可以聚合整句信息，给后续 DiT 多几个“摘要槽”。它们不是自然语言词，不会输出可读推理步骤，也不能据此宣称模型学会显式 reasoning。当前代码还会把 mask 打开，让正文 token 与 registers 全部互相可见。

论文说 thinking tokens “可能携带聚合上下文或语义上补出的缺失细节”，但没有给独立消融结果，因此这是机制解释与作者假设，不是已量化证明。

## 音频 VAE：为什么先变 mel，再压成 25 token/s

[[audio-vae]] 的输入不是原始波形直接切 token：

```text
stereo waveform
→ 重采样到 16 kHz
→ 左右声道各算 mel-spectrogram
→ 沿通道拼接
→ causal audio VAE
→ 约 25 token/s，每个 token 128 维
```

4 秒音频约 100 个 token，20 秒约 500 个 token。这里的 25 token/s 是时间密度；不能仅凭 `采样点数 ÷ latent 标量数` 宣称感知压缩率，因为中间 mel 变换、频率维和量化方式都不同。

解码反向走：128 维 token → audio VAE decoder → 双声道 mel → 修改版 HiFi-GAN vocoder → 24 kHz 双声道波形。vocoder 的工作是把频谱表示合成为可播放的相位连续波形；它不是 DiT 的另一轮去噪。LTX-2 将 HiFi-GAN V1 generator 通道数加倍，以承担立体声联合重建。

当前官方实现进一步给出音频 latent 形状 `[B,8,T/4,16]`：固定的 16 个频率槽与 8 个通道合起来正好是每个时间位置 `8×16=128` 维。这是代码说明，不是论文中的独立实验。

## 视频 VAE：算力为什么能降下来

当前官方实现的 causal video VAE 形状是：

```text
[B,3,F,H,W] → [B,128,F',H/32,W/32]
F' = 1 + (F-1)/8
```

例如 `33×512×512` 的视频变成 `5×16×16` 个空间—时间位置，每个位置 128 维。位置数量从 `33×512×512=8,650,752` 变为 `5×16×16=1,280`；DiT 在后者上工作。通道数从 3 增到 128，不能把 6758 倍位置缩减直接叫作文件压缩率，但它解释了注意力为什么不用面对每个像素。

## 训练目标：两条 latent 速度场一起学

补充材料 Figure A1 说明模型用 [[flow-matching]] velocity loss 训练。对视频或音频任一模态，可写成：

\[
x_\sigma=(1-\sigma)x_0+\sigma\epsilon,
\qquad
v^*=\epsilon-x_0,
\qquad
\mathcal L=\|v_\theta(x_\sigma,\sigma,c)-v^*\|_2^2
\]

- `x_0`：VAE 编码后的干净 latent；
- `ε`：与 latent 同形状的随机噪声；
- `σ∈[0,1]`：插值位置，0 靠近数据、1 靠近噪声；
- `x_σ`：训练时送进模型的带噪 latent；
- `v*=ε-x_0`：这条直线路径的真实速度；
- `v_θ`：参数为 `θ` 的双流 DiT 预测速度；
- `c`：文本以及另一模态的信息；
- `‖·‖²₂`：所有预测误差平方后求和。

为什么目标是 `ε-x_0`？因为对插值式关于 `σ` 求导：`dx_σ/dσ=-x_0+ε`，正好是从数据指向噪声的恒定速度。推理从噪声端反向积分，就沿相反方向回到数据。论文没有给总损失如何平衡 video/audio 两项、噪声采样分布或优化器。

## Modality-CFG：文本和另一模态分开加力

标准 CFG 只有“有文本/无文本”两个预测。LTX-2 的每条流还有另一模态条件，因此增加第三种分支。为避免论文把 `t` 同时让人联想到 timestep，这里把原文的文本条件 `t` 改记为 `c`：

\[
\widehat M(x,c,m)=M(x,c,m)
+s_{text}[M(x,c,m)-M(x,\varnothing,m)]
+s_{cross}[M(x,c,m)-M(x,c,\varnothing)]
\]

- `x`：当前要生成的那条流的 noisy latent；
- `c`：文本条件；`∅` 表示拿掉文本；
- `m`：另一模态特征；第二个 `∅` 表示拿掉跨模态条件；
- `M(x,c,m)`：文本和另一模态都给全时的速度预测；
- `M(x,∅,m)`：只保留另一模态，用来测“文本带来了多少变化”；
- `M(x,c,∅)`：只保留文本，用来测“另一模态带来了多少变化”；
- `s_text`、`s_cross`：分别放大两种条件产生的差值方向。

论文的基点是**完整条件输出**，再额外加两条差值，不是标准 CFG 常见的“从无条件输出出发”。[[modality-aware-cfg]] 会完整对照两者。

### 一个标量手算

假设某一维速度预测：完整条件 `2.0`、去文本 `1.5`、去另一模态 `1.2`。

```text
文本贡献 = 2.0 - 1.5 = 0.5
跨模态贡献 = 2.0 - 1.2 = 0.8

video: s_text=3, s_cross=3
2.0 + 3×0.5 + 3×0.8 = 5.9

audio: s_text=7, s_cross=3
2.0 + 7×0.5 + 3×0.8 = 7.9
```

论文所有结果采用视频 `(3,3)`、音频 `(7,3)`。更大的音频文本系数可理解为更强地强调台词、音素和声音描述，但报告没有给 scale sweep 或消融，不能断言 7 是普适最优值。CFG 还需要额外前向，提升控制力的代价是推理计算增加。

## 1080p 怎么避免一次塞满显存

这一节只处理高分辨率视频支路。video latent 是 VAE 压缩后的 `(time,height,width)` 特征网格，不是 RGB 像素；tile 是从这张网格切出的一小块。MP 是“百万像素”，所以 0.5MP 约等于 50 万像素。[[multi-scale-tiled-inference]] 分四段：

1. **约 0.5MP base generation**：先确定全局构图、运动和音视频同步；
2. **latent upscaler**：增加视频 latent 高、宽方向的网格点，只准备更大的高分辨率画布；
3. **overlapping tiles refinement**：把放大后的 latent 沿空间和时间切成重叠块，用同一基础模型逐块补高频细节；
4. **latent blend + decode**：把各块在重叠位置的预测加权融合，整张 latent 拼好后再由 Video VAE 解码成 1080p 帧。

为什么不能直接从头跑 1080p：`1920×1080=2,073,600` 像素，约为 `0.5MP` 的 `4.1×`。帧数和 VAE 压缩率不变时，视频 latent 的空间位置也约增至 4.1 倍，必须同时保存的激活随之上涨。先跑 0.5MP 能让模型在较小网格上看完整段视频；upscaler 只增加位置，真正缺失的皮肤、头发和环境纹理仍由精修阶段生成。

重叠是为了给边界像素足够上下文；融合是为了让相邻块从两次预测平滑过渡。如果无重叠硬拼，接缝两侧会分别在不同上下文中生成，容易出现纹理跳变。

这项技术降低的是**峰值显存**，不是让总计算免费消失。重叠区会重复计算，分块也可能削弱跨越很远区域的一致性。报告没有给 tile 大小、overlap、blend window、upscaler 训练方法、相对整图推理的质量/速度消融，也没有说明精修阶段音频流是否重新运行。原文只说 upscaler 维持时间一致性与音频对齐，不能据此补出一条未公开的音频执行路径。

## 数据与训练：公开了什么，没有公开什么

公开内容：训练数据取自 LTX-Video 数据集的子集，重点保留音频“显著且有信息”的片段；新 captioner 同时描述镜头运动、灯光、人物行为、音乐、环境声，并准确转录对话及说话人、语言、口音。caption 被要求事实化，不做情绪臆测。

没有公开：

- 视频数量、总时长、语言和领域分布；
- 去重、版权、安全过滤和质量阈值；
- optimizer、学习率、batch、训练步数、分辨率/时长课程；
- video/audio loss 权重、flow 的 `σ` 分布；
- 训练 GPU 数、时间、成本、失败恢复；
- “brief initial stage”和结论所称“progressive joint training”的具体阶段。

因此这篇报告足以理解模型结构，却不足以独立复现预训练。

## 实验到底证明了什么

### 速度：真实数字强，但不是纯架构消融

论文 Table 1 在单张 H100、121 帧、720p、单步 Euler、CFG=1 下测每个 diffusion step：

| 模型 | 输出 | 参数量 | 秒/步 |
|---|---:|---:|---:|
| Wan 2.2-14B | 仅视频 | 14B | 22.30 |
| LTX-2 | 音频+视频 | 19B | 1.22 |

`22.30/1.22=18.28`，所以“约 18×”算术成立，而且 LTX-2 还同时产生音频。但两者不是只替换某个模块的同架构对照：VAE 压缩、token 数、Transformer、kernel 和实现都不同。这个表证明的是**端到端每步速度差**，不能把 18× 全归功于双流不对称。

### 质量：结论比证据强

- 报告称内部人类偏好评测显著优于 Ovi，并可比 Veo 3、Sora 2；但没有公布 prompt 集、样本数、受试者数、胜率、置信区间或原始表。
- 报告称 2025-11-06 的 Artificial Analysis 排名为 I2V 第 3、T2V 第 4；没有列出分数或评测配置，且榜单会随新模型变化。
- 报告称最长生成 20 秒同步双声道，并对照当时 Veo 3 12s、Sora 2 16s、Ovi/Wan 2.5 10s；这些是版本相关的论文时点数字，不是永久产品上限。

所以可以说 LTX-2 展示了很强的效率和有竞争力的公开排名；“开源 SOTA、媲美闭源”的证据在这份报告里不够完整。

## 局限与失败方式

- 训练不足的语言或方言：发音、语音内容和嘴型对齐都会下降；
- 多说话人：可能把某句台词分配给错误人物；
- 超过约 20 秒：可能出现时间漂移、同步下降、场景多样性降低；
- 分块 1080p：论文没有量化长距离一致性与接缝失败；
- thinking tokens：没有独立消融，无法知道改进来自全层特征、双向 connector 还是 registers；
- 无显式 world model/reasoner：长叙事和事实可靠性依赖上游 LLM 写出的条件文本；
- 合成媒体风险：逼真语音与人物视频可用于冒充，部署需要标识合成来源、溯源和安全策略；
- 训练数据偏差会同时出现在人物外观、口音、语言和声音环境中。

## 一个不能忽略的报告内部矛盾

摘要、3.1、Table 1 和当前官方 README 都写 **14B video + 5B audio = 19B**；但结论却写“13B video + 3B audio”。两组数字无法同时为真。这里采用前者，因为它被正文多个位置和开源实现交叉支持，同时保留矛盾，不能悄悄把结论那句抹掉。

## 核心贡献

1. **不对称联合生成**：[[asymmetric-dual-stream-transformer]] —— 两条模态专用流按信息密度分配容量，而不是复制两套同宽 backbone。
2. **逐层双向校正**：[[audiovisual-cross-attention]] —— 用共同时间轴交换声音与画面信息。
3. **把文本拆成两种条件接口**：[[thinking-tokens]] —— 全层 Gemma 特征经过视频/音频各自的双向 connector 与 registers。
4. **紧凑立体声音频 latent**：[[audio-vae]] —— 约 25 token/s、每 token 128 维，再由立体声 vocoder 重建 24kHz 波形。
5. **独立控制提示与同步**：[[modality-aware-cfg]] —— 分开放大文本方向和另一模态方向。
6. **受限显存生成 1080p**：[[multi-scale-tiled-inference]] —— 低分辨率定全局、latent 放大、重叠分块精修。

## 关键概念

- [[diffusion-transformer]] · 双流 backbone 仍是在 latent 上预测速度的 DiT。
- [[flow-matching]] · 训练两条流从噪声到音视频数据的速度场。
- [[adaptive-layernorm]] · 用噪声进度调节跨模态输入和输出门。
- [[rotary-position-embedding]] · 模态内用 3D/1D，跨模态只用共同的时间维。
- [[classifier-free-guidance]] · modality-CFG 的单条件前身。
- [[video-vae]] · 把空间 32×、时间约 8×压到深 latent。
- [[log-mel-spectrogram]] · 音频 VAE 之前的时频表示。

## 我的批注

- 最值得借鉴的不是“再加一个音频塔”，而是**保留模态专用表示、只共享必须共享的坐标**：视频与音频不共享空间，只共享时间。
- Cross-modality AdaLN 解决的是可靠性不对称。A2V/V2A 中一边可能是 `σ=0` 的干净条件，另一边仍在去噪；门控让同一架构自然支持联合生成和单向编辑。
- thinking tokens 这个名字容易制造过度联想。代码里的 `learnable_registers` 更准确：它们是可写的隐藏槽，不是语言化思考过程。
- 论文把“联合模型”与“真实 joint distribution”绑得很紧，但没有似然、互信息或同步消融足以证明逼近质量；更稳妥的说法是架构允许双向依赖。
- 18× 每步速度是本报告最硬的数字，也是最容易被错读的数字：它是整套系统对整套系统，不是 dual-stream 对 symmetric-stream 的单变量实验。
- 报告正文对架构很具体，对训练和评测却很薄。开源权重可以验证“能跑”，不能自动补齐数据来源、训练成本与人评可信度。

## 跟 wiki 里其他 paper 的关系

- LTX-Video · LTX-2 延续其深时空 video latent 与实时生成取向，再加入并行音频流。
- [[vits]] · 都把声学中间表示交给神经 vocoder，但 VITS 以 TTS 为中心，LTX-2 要同时建模画面、语音、音乐和 Foley。
- [[cosmos-3]] · 两者都让不同模态在统一 Transformer 框架内交互；Cosmos 3 更强调理解—生成权重接力，LTX-2 更强调音视频联合去噪与时间同步。
- [[sensenova-vision]] · SenseNova 用单一生成接口统一视觉任务；LTX-2 选择模态专用 latent 与不对称双流，代表另一种“统一但不抹平差异”的路线。
