# YOLOv2 / YOLO9000 · 给 YOLO v1 逐项校准

YOLO v1 证明了检测可以一次前向做完，但它的框容易偏、召回不够高、类别表也被检测数据集的规模卡住。YOLOv2 没有靠“堆更深、做集成”硬冲，而是把训练、框参数化、特征分辨率和输入尺度逐项理顺；YOLO9000 再把检测数据与分类数据接进同一棵标签树，第一次把实时检测扩到 9000 多类。

## 一句话

**YOLOv2 把 v1 的自由回归改成有先验、有限位、能变尺度的密集检测；YOLO9000 再用 WordTree 把“会框”与“认识更多类别”拼到一起。**

## 它要解决的痛点

- **定位与召回**：YOLO v1 的主要错误是框偏，召回也低于候选框路线。作者明确把这两项列为 Better 章节的主攻方向。
- **训练起点别扭**：v1 先在 224×224 上做分类预训练，再突然切到 448×448 学检测；网络既要换任务，又要适应分辨率。
- **框没有合适的起跑线**：v1 直接回归坐标；Faster R-CNN 的 anchor 虽然给了先验，手选形状与无限制中心偏移又会带来新问题。
- **小目标信息被压扁**：13×13 深层特征适合大物体，却容易丢细节。
- **检测类别太少**：COCO 只有常见大类，ImageNet 分类却有大量细粒度类；检测框标注贵，没法指望直接补齐同等规模的数据。

## 核心贡献

1. **逐项校准**：BatchNorm、高分辨率分类预训练、全卷积预测、[[anchor-boxes]]、[[dimension-clustering]]、直接位置预测、passthrough 与 [[multi-scale-training]]，把 VOC 2007 mAP 从 63.4 推到 78.6。
2. **更轻的骨干**：Darknet-19 用 19 个卷积层、5 个池化层和 5.58B 次运算，在 ImageNet 上拿到 72.9% top-1 / 91.2% top-5。
3. **标签树**：[[hierarchical-classification]] 把互相重叠的“狗 / 梗犬 / 诺福克梗”放进路径，而不是硬塞进一个互斥 softmax。
4. **联合训练**：检测图更新框、物体性与类别；分类图只更新有标签的类别路径，并借最高分候选框提供弱定位信号。
5. **大词表实时检测**：COCO 检测数据 + ImageNet 分类数据合成 9418 类 WordTree，得到 YOLO9000。

## Better · 不是一招封神，而是一串小修正

### BatchNorm 与高分辨率分类器

所有卷积层加 BatchNorm 后，mAP 从 63.4 到 65.8，提升 2.4；作者因此移除了 dropout。随后先把分类网络放到 448×448 的 ImageNet 上微调 10 个 epoch，再学检测，mAP 到 69.5。关键不是“高分辨率天然更准”，而是先让卷积核适应新像素尺度，别把换分辨率和换任务挤在同一步。

### 全卷积 anchor：mAP 小降，召回先涨

YOLOv2 去掉全连接层，把输出改成 13×13 网格；每格预测 5 个 [[anchor-boxes]]。输入选 416 而不是 448，是因为总下采样 32 倍后正好得到奇数边长 13，图像中心只有一个中心格。

这一步不是立刻涨分：中间模型从 **69.5 mAP / 81% recall** 变成 **69.2 mAP / 88% recall**。框从 98 个增到一千多个，误报机会也多了，所以 mAP 略降；但少漏了 7 个百分点，给后续修正留出空间。论文的消融恰好提醒：召回和 mAP 不是一回事。

### 框形状聚类 + 直接位置预测

手选 anchor 未必贴数据。论文用 <code>distance = 1 - IOU</code> 做 k-means：VOC 上 5 个 IOU 聚类中心的平均最近 IOU 是 **61.0**，已经追平 9 个手选 anchor 的 **60.9**；9 个聚类中心能到 **67.2**。[[dimension-clustering]] 的价值是给回归一个更顺手的起点，不是替网络完成最终定位。

中心坐标也加了限位。若格子左上角是 (c_x,c_y)，先验框是 (p_w,p_h)：

~~~text
b_x = sigmoid(t_x) + c_x
b_y = sigmoid(t_y) + c_y
b_w = p_w × exp(t_w)
b_h = p_h × exp(t_h)
objectness = sigmoid(t_o)
~~~

取 <code>c_x=3, t_x=0</code>，则 <code>sigmoid(0)=0.5</code>，所以中心 <code>b_x=3.5</code>，一定落在第 3 个格子的横向范围 [3,4]。再取 <code>p_w=2, t_w=ln(1.5)≈0.405</code>，则 <code>b_w=2×1.5=3</code>。反算 <code>ln(3/2)=0.405</code>，正好回到 t_w。中心用 sigmoid 管住位置，宽高用指数表达相对倍数，两种变量各走各的合适坐标系。

消融表把“新骨干 + 维度先验 + 直接位置预测”合在一步，从 69.6 推到 74.4；因此可以说这组设计贡献接近 5 mAP，不能把 4.8 全单独记到聚类或位置限位头上。

### Passthrough：把 26×26 细节搬到 13×13

YOLOv2 把较早的 26×26×512 特征按相邻 2×2 重排成 13×13×2048，再与深层特征拼接。空间尺寸减半、通道数乘四，总元素数没变：

~~~text
26 × 26 × 512 = 346,112
13 × 13 × (512 × 4) = 13 × 13 × 2,048 = 346,112  ✓
~~~

它不像后来的多层特征金字塔那样在多个尺度直接出预测，只是把细粒度信息送进同一个检测头。消融提升 1.0 mAP：74.4 → 75.4。

### 多尺度训练：同一套权重，速度精度现场换挡

每 10 个 batch，网络从 {320,352,…,608} 中随机换一个训练尺寸；这些数都是 32 的倍数。测试时同一套权重可以按设备和延迟预算选分辨率：

| 测试尺寸 | VOC2007 mAP | FPS |
|---|---:|---:|
| 288 | 69.0 | 91 |
| 352 | 73.7 | 81 |
| 416 | 76.8 | 67 |
| 480 | 77.8 | 59 |
| 544 | 78.6 | 40 |

训练下限是 320，论文仍额外测了 288；这说明网络能向更低分辨率外推一次，但不等于任意尺寸都稳。[[multi-scale-training]] 真正留下的工程价值，是把“部署模型选型”变成同一模型的运行时旋钮。

## Faster · Darknet-19 把算力花在刀刃上

当时常见的 VGG-16 卷积部分需要 30.69B 次运算；YOLO v1 自定义骨干是 8.52B；Darknet-19 降到 **5.58B**。它主要用 3×3 卷积提特征，在其间插 1×1 卷积压通道，最后用全局平均池化做分类。结构不花哨，重点是让 3×3 别总在最宽的通道上干重活。

分类训练共 160 epoch：初始学习率 0.1，四次幂多项式衰减，weight decay 0.0005，momentum 0.9，并做裁剪、旋转、色相/饱和度/曝光增强。随后在 448×448 上再训 10 epoch，学习率 0.001，分类精度到 **76.5% top-1 / 93.3% top-5**。

改检测器时去掉最后分类卷积，加 3 个 3×3×1024 和最终输出层。VOC 有 20 类、每格 5 个框，每个框要 4 坐标 + 1 物体性 + 20 类，所以最终通道数是：

~~~text
5 × (4 + 1 + 20) = 125
~~~

检测训练同样 160 epoch，初始学习率 0.001，在第 60、90 epoch 各除以 10；weight decay 0.0005，momentum 0.9。

## Stronger · WordTree 把“会框”和“认识”拆开学

扁平 softmax 默认类别互斥，但“狗”和“诺福克梗”显然可以同时成立。[[hierarchical-classification]] 把类放进树里，每个分叉只在兄弟节点之间做 softmax，叶子概率等于整条路径的条件概率相乘。

取一棵小树：

~~~text
实体
├─ 动物 0.80
│  ├─ 哺乳类 0.75
│  │  ├─ 狗 0.90
│  │  └─ 猫 0.10
│  └─ 鸟 0.25
└─ 交通工具 0.20
~~~

兄弟概率各自加起来为 1：0.80+0.20=1、0.75+0.25=1、0.90+0.10=1。于是：

~~~text
P(狗) = 0.80 × 0.75 × 0.90 = 0.54
~~~

若检测框的物体性只有 0.60，最终“这个框里是狗”的分数是 0.60×0.54=0.324。反过来把叶子除掉最后一段：0.54/0.90=0.60，正好等于到“哺乳类”的路径概率 0.80×0.75，链条闭合。

WordTree1k 把 ImageNet 1000 个叶类加上中间节点，输出扩到 1369；top-1 / top-5 只从普通 Darknet-19 的 72.9/91.2 降到 **71.9/90.4**。树的额外好处是“不确定品种时还能答狗”，错误会停在较粗的父类，不必硬猜叶子。

## 联合训练 · 两种图片，各更新自己知道的部分

最终 WordTree 有 **9418 类**。训练把 ImageNet 分类图和 COCO 检测图混在一起，并过采样 COCO，使两者数量比例约为 4:1。

- **检测图**：正常更新框坐标、物体性和有标签的类别路径。
- **分类图**：没有真值框，只找对该类别概率最高的预测框，更新那一个框的类别树；论文还假设该框与潜在真值至少有 0.3 IOU，据此给物体性弱监督。
- **粗标签**：若标签只有“狗”，只更新到“狗”这一层，不惩罚下面“德牧还是金毛”，因为数据没提供这条信息。

YOLO9000 用 3 个先验框而不是 5 个，以控制 9418 类带来的输出规模。这里最漂亮的拆法是：COCO 教模型**东西在哪**，ImageNet 教模型**东西叫什么**；WordTree 决定不同粒度的名字怎样共存。

## 实验 · 成绩要按指标口径看

### VOC：实时线内的平滑换挡

VOC 2007 上，YOLOv2 416 是 **76.8 mAP @ 67 FPS**，544 是 **78.6 @ 40 FPS**。同表 Faster R-CNN ResNet 是 76.4 @ 5 FPS，SSD500 是 76.8 @ 19 FPS。VOC 2012 上，YOLOv2 544 得到 **73.4 mAP**；论文表中说明它比对照方法快约 2–10 倍。

### COCO：AP50 不差，严格定位和小目标仍弱

COCO test-dev 2015 上：

| 指标 | YOLOv2 |
|---|---:|
| AP@[.5:.95] | 21.6 |
| AP50 | 44.0 |
| AP75 | 19.2 |
| AP-small | 5.0 |
| AP-medium | 22.4 |
| AP-large | 35.5 |

AP50=44.0 看着接近同期方法，但一换成更严格的综合 AP 就只有 21.6；AP75 也低于 AP50 一大截。小目标 AP 只有 5.0，passthrough 缓解了细节损失，却没有根治。这组数据比“v2 更准”更有信息：它明显提升了 v1，但定位精细度和小目标仍是软肋。

### ImageNet detection：会迁移动物，不等于万类都框得好

ImageNet detection 200 类里，只有 44 类与 COCO 重叠。YOLO9000 总体 **19.7 mAP**；156 个没有检测框监督的类仍有 **16.0 mAP**。但误差很有结构：red panda 50.7、fox 52.1、koala bear 54.3、tiger 61.0、armadillo 61.7；diaper、horizontal bar、rubber eraser、sunglasses、swimming trunks 都是 0。

原因不是标签树失效，而是 COCO 的物体性经验更容易从常见动物迁到新动物；COCO 没有服饰框，模型只靠分类图很难学会“墨镜该框在哪”。所以“能预测 9000 多类”描述的是输出词表，不代表 9000 多类都有同等可靠的定位质量。

## 关键概念

- [[anchor-boxes]] · 先给框一个宽高起点，再回归相对偏移
- [[dimension-clustering]] · 用 IOU 距离从数据里选先验框
- [[multi-scale-training]] · 同一权重适配多档输入尺寸
- [[hierarchical-classification]] · 路径概率让粗类与细类共存
- [[one-stage-detection]] · YOLOv2 仍是一阶段，只是从 98 个自由框变成密集 anchor
- [[iou-intersection-over-union]] · 既做框评价，也做聚类距离
- [[mean-average-precision]] · VOC mAP@0.5 与 COCO AP@[.5:.95] 不能混为一谈

## 我的批注

- 最值得学的不是某个组件，而是消融顺序。anchor 让 mAP 从 69.5 降到 69.2，却把召回从 81 拉到 88；作者没有删掉它，而是继续修参数化，最后把“更多候选”兑现成精度。
- Table 2 不是每项独立实验。维度先验和位置预测与新骨干一起出现，不能从总增量里硬拆单项功劳。
- 多尺度训练把部署选择从“训练五个模型”变成“一套权重换输入”，这是很有工程味的设计。
- WordTree 解决的是标签语义冲突与数据合并，不是凭空创造定位监督。服饰类 0 AP 把边界说得很清楚。
- YOLOv2 仍只在一个 13×13 检测头上预测。细粒度特征只是搬进来，还没有形成真正的多尺度输出；这正是 YOLOv3 会继续补的地方。
