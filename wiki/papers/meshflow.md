# MeshFlow · 把"谁连谁"改写成连续数,整张网格一次并行长出来

## 一句话
生成 3D 网格最难的不是"顶点摆哪儿",而是"哪两个顶点该连一条边"——前者是连续坐标、后者是离散开关。MeshFlow 给每个顶点挂一枚连续的"连边磁铁",让扩散模型只生成连续量,最后一步才把磁铁的吸力切成 0/1 的边;再配上 VAE 压缩,整张网格一次并行生成,不用一根一根边地吐。

## 它要解决的痛点
一张三角网格 = 一份顶点表 + 一份面表。顶点 `(0.31,−0.42,0.08)` 是连续数,可以平滑地加噪声、去噪声——扩散模型最擅长这个。可面 `(17,42,91)` 是三个整数索引,把 `42` 改成 `43` 没有"改一半"的中间态。**连续坐标好办,离散连接才是硬骨头**,几条老路都各有别扭:

- **先做隐式表面、再抽网格**(SDF + Marching Cubes):形状能出,但三角面又密又糊,锐边被磨平,还常要求封闭表面,美术师拿到手很难改。
- **把坐标和面全切成离散 token、自回归逐个吐**:拓扑清楚,但坐标被量化到每轴 128 档(细节发抖),而且网格越大、串行解码越慢,长网格还容易提前"说完了"。

MeshFlow 换个思路:**别让扩散模型碰离散的东西**。把"有没有边"也改写成连续数,让模型全程只在连续空间里玩,离散化推到最后一步再做。

## 核心贡献
- **连续连接表示**([[continuous-mesh-connectivity]]):每个顶点带三样连续量——坐标、外法线、一枚"连边磁铁"(边嵌入)。两枚磁铁的吸力分数过阈值就连一条边,三条边围成一圈就补一个三角面。离散拓扑被这枚磁铁"翻译"成了连续向量。
- **MeshVAE 压缩**([[mesh-token-merge]]):把 N 个顶点压成约 N/4 个 latent。压法叫 TokenMerge——不是丢点,而是把相邻几个顶点的通道拼一块、再让短 query 回头用 cross-attention 看全部顶点,所以"变短"不等于"看不见"。
- **并行生成**([[flow-matching]] + [[diffusion-transformer]]):在压好的 latent 上跑整流流 DiT,一次采样步同时更新整串 latent(默认 28 步),而不是一根边一根边地自回归。
- **点云指路**([[voxel-rope-conditioning]]):把条件点云先落进 32³ 的粗体素格,再用 3D RoPE 把"点在哪块空间"写进注意力——只记大致位置、不记采样细节,缩小训练和推理的口音差。

## 方法主线

### 1. 先把网格改写成"一张连续的顶点表"
不直接生成面,而是给每个顶点一条连续向量 `(v, n, h)`:`v` 是坐标、`n` 是外法线(指向表面外)、`h` 是连边磁铁。扩散模型只管生成这些连续量;解码最后一步才把任意两枚磁铁的吸力算出来,过线的连成边,三条边成环补成一个三角面。法线在这里当"朝向裁判":几何算出的面法线要是跟三个顶点法线的平均方向相反,就把面的两个顶点交换一下,把正反面翻回来。好处是不挑输入——不要求封闭流形;代价是只做三角面。

### 2. MeshVAE 把顶点表压短
逐顶点跑 DiT 太贵,先用 VAE 压。编码器给每个顶点拼上坐标、法线、邻居坐标(都过一遍 Fourier 编码),然后 **TokenMerge**:把相邻 4 个顶点 token 的通道拼成一个宽 token、再用一层线性压回原宽度,序列就短了 4 倍。关键是这批短 token 随后用 **cross-attention 回看全部原始顶点**——它们不是空白工位,而是抱着原料上桌,所以压短没有立刻丢掉信息。解码端做反向的 TokenSplit 把序列摊回来。

### 3. 从一堆候选边拼回三角面
解码后先按 mask 删掉补齐用的空位,再两两算磁铁吸力。对每条边 `(i,j)` 找它俩的共同邻居 `k`,如果 `ij`、`jk`、`ki` 三条边都在,就收下三角面 `(i,j,k)`,最后用法线校正朝向。麻烦点:三条"错误边"要是恰好凑成一个环,也会凭空长出一个错误面。

### 4. 整流流 DiT 并行生成 latent
训练时把干净 latent `x₀` 和高斯噪声 `ε` 用一条直线连起来:中间点 `x(t)=(1−t)x₀+tε`,这条直线的速度恒定是 `ε−x₀`。DiT 看着带噪点、时间 `t` 和点云条件,去回归这个速度。推理反过来:从纯噪声出发,顺着预测速度一步步走回 `t=0`,再交给 VAE 解码成网格。这里的"并行"指**每一步同时更新整串 latent**,砍掉了自回归那条越来越长的串行链;采样轮数固定(默认 28 步),不随网格变大而变长。

### 5. 点云怎么给 DiT 指路
难点是训练看的是网格顶点、推理看的是表面均匀采样点,两者分布不一样(美术师爱在锐边多放顶点,均匀采样只按面积撒)。直接把精确坐标喂进去,模型会记住"顶点怎么排",一到推理就"换了口音"。解法:先把点落进 32³ 的粗体素格,同格的点共用一个位置编号,再用 3D RoPE 把这个粗位置写进注意力。模型只抓"点在哪块空间",不纠结采样细节,训练推理的差就小了;代价是条件几何变粗。论文里对比过另一条路(点云先过预训练 shape encoder 变成 2048 个条件 token 做 cross-attention),那条推理更自然但训练收敛慢,最后没选。

## MeshVAE 的五笔损失
VAE 要同时学好几件事,损失拆成五笔各管一摊:

- **mask BCE**:这个位置是真顶点,还是补齐 batch 长度用的空位?
- **顶点 MSE**:预测的 xyz 离真值坐标多远。
- **法线 MSE**:外法线方向偏没偏。
- **边对比损失**:把"真边"的磁铁吸力推过阈值,把"非边"往反方向推开。
- **KL(权重 10⁻³)**:让 latent 靠近高斯,后面 Flow 才好从噪声生成。

几个补充材料给的具体值:连边阈值 `τ=0.6`;非边样本数量巨大,用 `λ_neg=0.01` 压低它的权重;`λ_KL=10⁻³`。权重设小不是"不重要",而是防止"非边"和 KL 这种数量多的项淹没坐标、法线的重建信号。

## 训练配方
- **数据**:约 60 万个专业美术师做的**私有**网格,归一化到 `[−1,1]³`,**不量化坐标**,过滤掉最大顶点度数 >50 的网格。数据私有,外部能跑公开 checkpoint 但没法完整复现训练分布。
- **MeshVAE**:8 层编码器 + 8 层解码器,隐藏维 1024,共 **233M** 参数;32 张 H100 训 3 天,每卡 batch 64,AdamW,学习率 `5e-4`。
- **Flow DiT**:约 4–9 亿参数(**论文自相矛盾**:主文写 18 层 / 1024 / 427M,补充材料写 16 层 / 1536 / 895M);64 张 H100 训 10 天,学习率 `1e-4`,用 BF16 + Flash Attention + EMA,后期改 logit-normal 时间采样,推理 timestep shift = 3.0。

## 实验结果

### VAE 重建:用短得多的表示,做到接近最优的重建
比的是"原网格 vs VAE 重建"的 Chamfer 距离([[mesh-quality-metrics]],越小越好),外加"表示比"——latent 占原表示长度的多少(越小压得越狠):

| VAE 重建 | CD×100 ↓ | 表示比 ↓ |
|---|---|---|
| TreeMeshGPT | 1.63 | 0.22 |
| EdgeRunner | **1.21** | 0.47 |
| **MeshFlow** | 1.29 | **0.014** |

MeshFlow 的表示只占原长度约 **1.4%**(别人 22%–47%),重建 CD 却只比最好的 EdgeRunner 差一点点。所以准确说法不是"重建全场最低",而是**"用短一个数量级的表示,做到接近最优的重建"**。

### 点云条件生成:又快又准(Toys4K)
从生成网格的表面采 5000 点算 CD/HD([[mesh-quality-metrics]]),再看耗时和顶点数:

| Toys4K 生成 | CD ↓ | HD ↓ | 耗时 | 顶点数 |
|---|---|---|---|---|
| FastMesh-V1K | 4.09 | 10.32 | 3.41s | 467.2 |
| **MeshFlow** | **2.33** | **4.23** | **1.53s** | 459.75 |

### "18× 加速"要看口径
两个数都真,但分母不同,别混着说:
- **主表(batch 均时)约 2.2×**:`3.41s ÷ (1.06s 采样 + 0.47s 抽 mesh = 1.53s) ≈ 2.23×`。
- **摘要里的 18×(单物体)**:补充材料单物体计时 FastMesh ~21s、MeshFlow ~1.2s,`21 / 1.2 ≈ 17.5× ≈ 18×`。

### 消融:TokenMerge 的起手 query 极关键
边 F1(×100)同时查乱连(precision)和漏连(recall),两头都要好才高。换 MeshVAE 里"短 query 从哪儿起步":

| 初始化 query | 边 F1 |
|---|---|
| Q-Former(随机可学) | 49.47 |
| FPS(最远点采样) | 60.18 |
| **TokenMerge** | **99.78** |

差距大得反常:因为 TokenMerge 的 query 一开始就带着**原输入内容**,随机 query 和最远点采样都没有这份信息。论文没报方差,稳妥结论是"在这套设置里明显更好",不是"所有 mesh VAE 的普遍定律"。另外两组:压缩倍率 2× / 4× 的边 F1 是 92.65 / 88.82(压得越狠拓扑越难恢复);最大顶点数 2048 / 4096 / 8192 是 99.84 / 99.78 / 99.71(这组重建没随规模明显崩)。

## 局限
- **只会三角面**:不生成美术工作流常用的 quad,也没有 UV 和纹理。
- **指标看不见拓扑**:CD/HD 低不代表没洞、没翻面、好编辑;换个顶点顺序几何位置一模一样(CD/HD=0)但面朝向已经翻了。论文因此主动**不报** Normal Consistency——模型直接预测法线,拿这项高分说明不了拓扑更好。
- **错误边会长出错误面**:三条误边凑成一个三环,恢复器照样接受。
- **训练数据私有**:60 万 artist mesh 外部无法完整复建分布。
- **图像条件不是替代品**:README 有可选 DINOv3 图像输入,但发布流程仍需要点云提供几何位置。
- **全注意力仍贵**:并行采样消掉的是"随输出变长的串行链",没消掉标准 DiT 每层对 token 数的二次注意力成本。

## 我的批注 / 疑问
- 最值钱的不是"又一个 flow DiT",而是**先发明了一种扩散模型吃得下的网格**:坐标管形状、边嵌入管连接、法线管朝向。表示一理顺,VAE 压缩和 Flow 并行才顺理成章。
- "18× 加速"要留个心眼:那是**单物体**口径;主表按 batch 均时只有约 **2.2×**。引用时把分母写清楚。
- "非自回归 = 对网格大小线性"说得太满:它把**串行深度**降成固定采样步数(真优势),但每层全注意力对 token 数还是二次。稳妥理解是"生成轮数不随输出变长",不是"总计算严格线性"。
- **复现坑(codex 查证)**:论文、补充材料、公开代码有三处对不上——① 连边阈值主文写 `d≤τ`,代码实际是 `d>τ`(代码里的 `d` 是"越大越像边"的带符号分数,不是普通距离);② DiT 规模主文 18 层/427M、补充材料 16 层/895M,两套冲突;③ 补洞默认关闭。复现时把论文版本 `v2` + 代码提交 `55f56f6` + gated 模型配置一起记下来。
