# YOLO · 看一眼就把框和类一起吐出来

## 一句话
把目标检测从"先找候选框、再一个个分类"的多步流水线,改成**一个神经网络、一次前向**直接从整张图回归出所有框的坐标 + 类别——"You Only Look Once",于是能做到实时(45 FPS)。

## 它要解决的痛点
DPM 把分类器滑过整张图;原始 R-CNN 先用 Selective Search 提约 2000 个候选框,再逐框提 CNN 特征。Fast R-CNN 已把整图卷积共享起来,但仍要等 Selective Search;Faster R-CNN 再用 RPN 生成候选。YOLO 真正砍掉的不是某一个慢算子,而是“先提候选、再逐候选判断”的分段流水线。

整图上下文还让 YOLO 比只看局部候选块更少把背景误当物体。

## 核心贡献
- **检测 = 单次回归**:一个 CNN 一次前向,直接从整张图输出所有框 + 类别概率,没有候选框、没有逐块重复分类。
- **网格分工**:把图切成 S×S=**7×7** 网格,物体中心落在哪个格子,就由哪个格子负责预测它。每格预测 B=**2** 个框(各 5 个数:x,y,w,h,置信度)+ C=**20** 类概率 → 输出张量 7×7×**30** = **1470** 个数。
- **置信度 = Pr(有物体) × IOU**:框的置信度既表达"这里有没有东西",又表达"框得准不准"。
- **全图上下文**:一次看整张图 → 比只看候选块的 R-CNN **背景误检少得多**(但定位更粗、localization 错误多)。
- **实时**:base YOLO 63.4% mAP @ **45 FPS**;Fast YOLO 52.7% @ **155 FPS**(是当时其它实时检测器 mAP 的两倍多)。对比 Faster R-CNN VGG-16 73.2 mAP 但只 7 FPS。
- **艺术画迁移**:Picasso 实验用 VOC 2012 训练,People-Art 实验用 VOC 2010 训练;都在自然照片上训好后直接迁移。YOLO 在两个人体艺术数据集上明显好过 DPM、R-CNN,但这两组结果不能外推成普遍域泛化保证。

`x,y` 不是整图像素坐标,而是框中心相对当前格子的偏移;`w,h` 则相对整张图宽高归一化。两个框各有自己的几何和置信度,但同一格只共享一套 20 类概率——这也是一格挤进多个不同类别小物体时容易漏的根源。

## 责任分配与类专属分数
用同一只狗从训练贯穿到推理:

- 框 A 与真值 IOU = 0.70
- 框 B 与真值 IOU = 0.30
- 框 A 成为责任框
- `Pr(狗 | 有物体) = 0.80`
- 框 A 置信度目标 = 0.70
- 狗的类专属分数 = `0.80 × 0.70 = 0.56`

训练时,同格两个框中当前 IOU 更高的那个负责这个物体;这里就是框 A。推理时,每个框都用共享的条件类概率乘自己的置信度,所以框 A 的狗分数是 0.56。

## 训练损失

```text
① 责任框中心 x,y:× λcoord=5
② 责任框尺寸 √w,√h:× λcoord=5
③ 责任框置信度:目标 = 当前 IOU
④ 非责任框与空格框置信度:目标 = 0,× λnoobj=0.5
⑤ 有物体网格的一套 20 类条件概率
```

误差平方和容易优化,但它不直接优化 mAP,对定位和分类误差的加权也不够理想。网格里空框远比有物体框多,`lambda_noobj=0.5` 用来压低这批负样本的梯度;`lambda_coord=5` 则强调框的几何。

### 数字例子·归一化宽度先开根再算损失

```text
同样相差整图宽度的 0.01:
小框 0.01 → 0.02:√0.02 - √0.01 = 0.0414
大框 0.10 → 0.11:√0.11 - √0.10 = 0.0154

平方根残差比:0.0414 / 0.0154 ≈ 2.68
真正进入损失的是残差平方:2.68² ≈ 7.20

✓ 同样偏 1% 图宽,小框收到的尺寸损失约是大框的 7.2 倍。
```

## 训练配方
先用前 20 个卷积层和分类头在 ImageNet 224×224 上预训练,再去掉分类头,加 4 个卷积层 + 2 个全连接层,把输入放大到 448×448 训检测。检测训练约 135 个 epoch;训练开头缓慢升到 `1e-2`,论文没有给这段升温单独指定 epoch 数。

| 项目 | 配方 |
|---|---|
| 检测训练 | 约 135 epochs |
| Batch | 64 |
| Momentum | 0.9 |
| Weight decay | 0.0005 |
| Learning rate | `1e-3` 缓慢升到 `1e-2`;之后 `1e-2` 训 75 epochs,`1e-3` 训 30,`1e-4` 训 30 |
| 正则化 | 第一层全连接后 dropout 0.5 |
| 数据增强 | 缩放/平移最多 20%;曝光/饱和度最多 1.5× |

## 推理与 NMS
`7×7×2=98` 个框各自拿共享的 20 类条件概率乘自己的置信度,得到类专属分数。沿用前面那只狗:框 A 的狗分数是 `0.56`,旁边两个高度重叠框的狗分数是 `0.48` 和 `0.41`。对每类框按分排序后做 NMS:留 0.56 的框 A,删掉和它高度重叠的两个低分框。手算例子可用阈值**例如 0.5**;这是讲解用取值,不是论文固定的超参数。

论文还给了一个容易漏掉的量化结果:YOLO 的网格本身已经减少重复框,NMS 不像在 R-CNN/DPM 里那么关键,但仍能增加约 2-3 mAP。

## 实验：快只是半个答案，关键是它错得不一样

速度—精度正面账里,YOLO 是 63.4 mAP @ 45 FPS,Fast YOLO 是 52.7 mAP @ 155 FPS;Fast R-CNN 是 70.0 mAP @ 0.5 FPS,Faster R-CNN VGG-16 是 73.2 mAP @ 7 FPS。YOLO 跨过了实时线,但更关键的证据不是“快”,而是它与 Fast R-CNN 的错误结构不同。

### 错误画像 · 定位差，背景误检少

论文把 VOC 2007 每一类分数最高的 N 个检测拿出来,再在 20 类上平均:

| 模型 | 正确 | 定位 | 相似类 | 其他 | 背景 |
|---|---:|---:|---:|---:|---:|
| Fast R-CNN | 71.6% | 8.6% | 4.3% | 1.9% | 13.6% |
| YOLO | 65.5% | 19.0% | 6.75% | 4.0% | 4.75% |

YOLO 定位错误是 **19.0% vs 8.6%**;背景误检却只有 **4.75% vs 13.6%**,约为 Fast R-CNN 的三分之一。它更常见的错法不是把背景认成物体,而是物体看对了、框的位置没卡准。

### 模型互补 · 不是普通集成

**最佳 Fast R-CNN 单独为 71.8 mAP**。把不同模型加到它旁边:

| Added model | Added model mAP | Combined with best Fast R-CNN | Gain |
|---|---:|---:|---:|
| Fast R-CNN (2007 data) | 66.9 | 72.4 | +0.6 |
| Fast R-CNN (VGG-M) | 59.2 | 72.4 | +0.6 |
| Fast R-CNN (CaffeNet) | 57.1 | 72.1 | +0.3 |
| YOLO | 63.4 | 75.0 | +3.2 |

再加一个 Fast R-CNN 变体最多涨 0.6,加 YOLO 却涨 3.2。收益不是因为“又添一个模型”,而是 YOLO 背景误检少、定位误差多,正好和 Fast R-CNN 互相兜底。

### VOC 2012 · 小物体短板坐实

**YOLO：57.9 mAP；Fast R-CNN：68.4 mAP；Fast R-CNN + YOLO：70.7 mAP（+2.3）**。互补收益换数据集依旧存在。分到具体类别,bottle、sheep、tv/monitor 比表中邻近方法低 8-10 点,cat 和 train 反倒更强。这组证据不必铺成完整 20 类榜单:它最有用的地方,是把成群小目标与细小目标容易漏的短板坐实。

### 跨域泛化 · 从照片迁到艺术画

两组实验用的训练集不同:**Picasso 用 VOC 2012 训练,People-Art 用 VOC 2010 训练**;都是在自然照片上训好后直接迁移:

| 模型 | Picasso AP | People-Art AP |
|---|---:|---:|
| YOLO | 53.3 | 45 |
| R-CNN | 10.4 | 26 |
| DPM | 37.8 | 32 |

这只说明两个人体艺术数据集上的迁移结果,不能外推成普遍域泛化保证。定性艺术画示例不复制进 wiki,可到[论文官方 arXiv v5](https://arxiv.org/abs/1506.02640v5)查看。

作者还把 YOLO 接到摄像头,把取帧和显示也算进整条链路,验证它确实能实时运行。

四个局限都能从机制直接倒推:

- **每格只有两个框和一套类别概率** → 成群小目标容易漏。
- **框形状从训练数据中学** → 罕见长宽比和布局不稳。
- **多次下采样** → 定位特征偏粗。
- **误差平方和不等于 mAP** → 主要错误仍是定位。

YOLO v1 留下的长久贡献,是把通用目标检测重新表述成**一个联合训练的单一回归系统**。

## 我的批注 / 疑问
- YOLO 的漂亮是**把"检测"重新表述成"回归"**:不再"提议—验证"两步,而是让一个网格直接对号入座。这种"把多步流水线拍成一次前向"的思路,跟端到端 TTS(见 [[vits]] 治两段失配)是同一种品味。
- 代价很诚实:每格只出 2 个框 + 1 套类别 → **一群小物体(如成群的鸟)会漏**,定位也偏粗。VOC 2012 的逐类结果与错误画像都把这点坐实了。
- 置信度里塞进 IOU 这一手很关键:让"有没有物体"和"框得准不准"共用一个训练信号,推理时按 Pr(类)×IOU 排序 + [[non-max-suppression]] 去重就得到干净结果。
