# Cosmos 3 · 把“看懂、想象、行动”塞进同一个世界模型

Cosmos 3 面向机器人、自驾和智能空间。它不是简单地把五种输入都接到一个 Transformer 上，而是把两套工作方式装进同一个 checkpoint：Reasoner 用自回归方式理解和推理，Generator 用 rectified flow 生成图像、视频、音频和动作。两边参数分开，注意力仍能交换信息。

## 一句话

**同一个模型里放两套 Transformer 塔：Reasoner 负责“看懂和说”，Generator 负责“画出来和做动作”；语言、图像、视频、音频、动作经过各自的入口后，被排进同一条序列。**

## 它要解决的痛点

- 一个机器人要收拾桌子，旧方案常要串起 VLM、视频世界模型和动作策略。每个模型各说各话，信息在接口间来回翻译。
- 理解和生成并不是两件互不相干的事。决定“下一步该抓杯子还是盘子”，本身就要先想象动作后果，再判断哪种后果更合理。
- 语言推理适合自回归：后一个词不能偷看未来。图像、视频和动作生成更适合去噪：所有待生成位置应该一起协调。硬让一套权重同时适应两种注意力规则，容易互相牵制。

## 整体方案

1. 每层都有两套参数：AR token 走 Reasoner，扩散 token 走 Generator。MoT 解决的是任务冲突，不是 MoE 那种省算力路由。
2. Reasoner query 只能因果地看 AR 子序列；Generator query 可以双向看完整的 `[AR; DM]` 序列。AR 表示不会被尚未生成好的扩散 token 反向污染。
3. 图像理解走 ViT；图像/视频生成走冻结的视频 VAE；音频走冻结的 Audio VAE；不同维数的动作先经过各自的线性投影。
4. 任务由 token 排列和加噪位置决定：文生图、图生视频、前向动力学、逆动力学和策略任务都能写成“干净条件 + 带噪目标”。
5. 3D MRoPE 同时编码时间、高度和宽度。视频的时间位置按真实 FPS 缩放，而不是简单使用第 0、1、2 帧。

## 五种模态如何进入模型

- **语言**：普通文本 token，放在 AR 子序列。
- **图像/视频理解**：ViT 使用 16×16 patch，再用两层 MLP 合并相邻 2×2 token。
- **图像/视频生成**：冻结 Wan2.2-TI2V-5B 的视频 VAE；时间压缩 4 倍，空间先压缩 16×16，再合并 2×2 latent token，最终每个 token 约对应原图 32×32 像素区域。
- **音频**：冻结 Audio VAE；48 kHz 双声道音频约变成每秒 25 个 audio token。
- **动作**：动作维数随设备变化。单臂机器人一步是 10 维，双臂是 20 维，人形机器人是 29 维。每种设备先用自己的输入投影映射到统一隐藏维度，输出时再投影回原动作空间。

## MoT 和联合注意力

MoT 每层为两塔分别保存 LayerNorm、Q/K/V/O 投影和 MLP。Reasoner 和 Generator 没有共享一套 Transformer 权重，但它们的 key/value 会在联合注意力里拼到一起。

- Reasoner query：只看此前的 AR token。
- Generator query：双向看全部 AR token 和全部扩散 token。
- 结果：生成端能读取文字和视觉条件；推理端不会读到尚未完成的生成结果。

Super 的每塔约等于一个 32B 稠密模型，因此总参数约 64B。每个 token 只经过自己所属的塔，所以每 token 激活的主干参数仍约为 32B；代价是部署时必须把两塔权重都放进显存或跨设备切分。

## 一个贯穿例子：机器人把杯子放进托盘

输入包含一句目标、两路相机画面和机器人状态。不同任务只改变哪些 token 是条件、哪些 token 要去噪：

- 前向动力学：给定当前视频和动作，去噪未来视频，回答“这样动会看到什么”。
- 逆动力学：给定前后视频，去噪动作，回答“中间大概做了什么”。
- 策略：给定目标和当前视频，同时去噪未来视频与动作，回答“接下来会看到什么、机械臂该怎么动”。

这里的“同一个模型”不等于一次前向完成任务。语言仍逐 token 生成；图像、视频、音频和动作仍要跑多步去噪。

## Rectified flow 到底在学什么

Generator 先把干净目标 `x₀` 与高斯噪声 `ε` 混合：

```text
xσ = σ·ε + (1-σ)·x₀
目标速度 v* = ε - x₀
```

取 `x₀=2`、`ε=-1`、`σ=0.25`：

```text
xσ = 0.25×(-1) + 0.75×2 = 1.25
v* = -1 - 2 = -3
```

模型看到位置 1.25、噪声强度 0.25 和条件信息，学习预测这条直线路径的速度 -3。训练时可以在任意噪声位置抽一点监督；推理时从噪声端出发，多步沿预测速度走回干净数据端。

## 为什么 FPS 不能只写进提示词

视频 VAE 每 4 帧压成一个时间 token。基础视频是 24 FPS，因此基础 token 速率是 `24/4=6 token/s`。目标 FPS 不同时，模型用：

```text
Δt = TPS_base / TPS
```

调整相邻 token 的时间位置：

- 24 FPS：`0, 1, 2, 3, 4`
- 16 FPS：`0, 1.5, 3, 4.5, 6`
- 30 FPS：`0, 0.8, 1.6, 2.4, 3.2`

这样模型知道“相邻 latent 帧之间过去了多久”，不只知道它们谁先谁后。消融里，无控制的综合分是 8.51，只加文字 FPS 提示是 9.28，只加 MRoPE FPS 调制是 9.63，两者一起用达到 9.81。

## 数据

Reasoner 预训练使用 22,002,013 个样本，其中图文约 1881 万、视频文本约 102 万、纯文本约 217 万；SFT 使用 2,171,673 个样本。

数据先做语义去重，再让 AI judge 从忠实、完整、正确三个维度各打 1–5 分。去重删掉 4.23% 的近重复样本。预训练阈值设为 2，保留 78%，优先保住覆盖面；SFT 阈值设为 5，只保留 46%，优先保证监督精度。

Generator 使用图像、视频、音频、动作和视频 transfer 数据。视频训练覆盖 256p、480p、720p，最长分别为 400、400、300 帧；不同长度样本被拼进固定 74,000-token 上下文，尽量少做 padding。

## 训练流水线

1. Reasoner 从语言模型与 ViT 出发，做两轮多模态预训练；Nano/Super 分别由 Qwen3-VL-8B/32B 初始化，Edge 使用 NVIDIA 自己预训练的模型。
2. 用 Physical AI 数据做 SFT。训练 8200 步，全局 batch 512；同时按 1:4 混入高质量预训练数据，减轻专项训练造成的能力遗忘。
3. 把训练好的 Reasoner 权重复制给 Generator，作为 Generator 的初始化。
4. Generator 做图像、视频、音频联合预训练，再在 mid-training 中加入动作和 transfer 数据。
5. 最后分支做文生图、图生视频和机器人策略 post-training。论文中的榜单冠军并非全部来自一个未经专项训练的通用 checkpoint。

## QKV 复制过去，怎么从“预测下个词”变成“预测去噪速度”

它不会在复制的一瞬间变成去噪器。复制只是给 Generator 一个比随机数更懂语义的起点，后面的生成预训练才真正改造这些参数。

先区分两种容易混在一起的“attention 权重”：

- 会复制的是模型参数：LayerNorm、`W_Q/W_K/W_V/W_O` 投影矩阵和 MLP。
- 不会复制的是每次前向临时算出的注意力图 `softmax(QKᵀ)`。输入和 mask 一变，注意力图就会重新计算。

QKV 参数本身并没有写着“只能预测下一个词”。Transformer block 做的事一直是把一串隐藏向量重新混合、提取关系。任务由输入、attention mask、输出头和损失共同决定：

| 部分 | Reasoner | Generator |
|---|---|---|
| 输入 | 文本 token、ViT 特征 | 带噪 VAE / Audio / Action latent |
| 输入适配 | token embedding、ViT projector | 模态线性投影、模态 embedding、噪声强度条件 |
| Attention | 因果，只能看前文 | 双向，能看完整生成区和 Reasoner 条件 |
| 输出 | 词表 logits | 连续速度 `v̂` |
| 训练目标 | 下一个 token | `v*=ε-x₀` |
| 损失 | Cross-Entropy | masked MSE |

因此真正的过程是：把 Reasoner 的 Transformer 主干参数复制过来；视频、音频、动作先通过自己的投影进入同一个隐藏维度；Generator 改用双向 attention 和连续速度目标；Flow 训练产生的梯度再继续修改复制来的 QKV、MLP 和 Norm。论文明确说 VAE 与 Audio VAE 是冻结的，latent 通过线性层进入 Transformer；动作的输入、输出投影从随机初始化开始联合训练。

沿用前面 `x₀=2、ε=-1、σ=0.25` 的例子：Generator 看到 `xσ=1.25`，正确速度是 `v*=-3`。下面不是 Cosmos 3 的真实网络结构，而是把几百亿参数压成一个只有 `w` 的玩具模型，专门看“新损失怎样修改旧权重”。

先把公式写全。玩具模型拿输入 `xσ` 乘可训练参数 `w`，输出预测速度：

```text
模型：v̂(w) = w·xσ
```

训练要让预测 `v̂` 靠近正确答案 `v*`，所以用平方误差。当前例子只有一个数，不需要再求平均：

```text
损失：L(w) = (v̂(w) - v*)²
            = (w·xσ - v*)²
```

这里 `w` 是唯一要学习的变量；`xσ=1.25` 和 `v*=-3` 是这条样本给定的常数。求导是在问：“`w` 稍微变大一点，损失会朝哪个方向、变多快？”

为了不跳链式法则，先把预测误差记成 `e=w·xσ-v*`，于是 `L=e²`：

```text
dL/dw = (dL/de) · (de/dw)
       = 2e · xσ
       = 2(w·xσ - v*)·xσ
```

第一项 `2e` 来自平方 `e²` 的导数；第二项 `xσ` 来自 `e=w·xσ-v*` 对 `w` 的导数。两段乘起来，就是“损失 → 误差 → 参数”的链式法则。

现在才代数字：

```text
w=2，xσ=1.25，v*=-3

初始预测：
v̂ = 2×1.25 = 2.5

初始损失：
L = (2.5-(-3))² = 5.5² = 30.25

梯度：
dL/dw = 2×(2.5-(-3))×1.25
       = 2×5.5×1.25
       = 13.75
```

梯度是正数，说明继续增大 `w` 会让损失更大，因此梯度下降要反方向减。更新公式是：

```text
w_new = w - η·dL/dw
```

`η` 是学习率。取 `η=0.1`：

```text
w_new = 2 - 0.1×13.75 = 0.625

新预测：
v̂_new = 0.625×1.25 = 0.78125

新损失：
L_new = (0.78125-(-3))²
      = 3.78125²
      ≈ 14.30
```

自检也对得上：这条样本的最佳参数应满足 `w·1.25=-3`，所以 `w_best=-3/1.25=-2.4`。第一次更新把 `w` 从 `2` 降到 `0.625`，确实朝 `-2.4` 移动；损失也从 `30.25` 降到 `14.30`。一步以后仍然不会去噪，真实模型还要用大量样本反复更新所有 QKV、MLP、Norm 与模态投影，才把“理解世界的初始化”改造成真正的去噪器。

复制值得做，是因为 Reasoner 已经学会物体、动作、空间和语义之间怎样关联；Generator 不必从随机参数重新学“机械臂、杯子、托盘分别是什么”。但报告没有给出“随机初始化 Generator vs 复制 Reasoner 初始化”的严格消融。附录 E.1 的实验让两边 Generator 都从头训练，只替换条件侧的理解塔；它证明更好的 Reasoner 表征能提高 Physical AI domain 分数，不能单独证明复制初始化本身带来多少收益。

## 实验结果

- 文生图 UniGenBench：专项后训练的 Cosmos3-Super 为 91.36，Qwen-Image-2512 为 84.25。
- PAIBench-G 文生视频：Cosmos3-Super 80.0，Wan2.2-A14B 78.0。
- PAIBench-G 图生视频：Cosmos3-Super 82.8，Wan2.2-A14B 81.3。
- Physics-IQ 图生视频：Super 直接生成 43.8；配合 WMReward best-of-N 重排为 48.9。
- RoboLab 特定指令成功率：Cosmos3-Nano-Policy-DROID 为 39.7%，π0.5 为 28.1%。
- LIBERO-10 新设备适配：500 步时 mid-training 初始化达到 24.6%，普通 pre-training 初始化仍为 0%；2000 步时分别为 97.4% 和 95.2%。

## 消融告诉了什么

- 换成 Cosmos Reasoner 后，T2V Physical AI domain 分数从 73.7 升到 75.7；主要收益来自机器人、自动驾驶和工业场景，画面质量基本持平。
- 联合音视频训练没有拖垮视频：T2V overall 从 78.6 到 79.1，I2V 从 81.7 到 82.2。
- 三种动作模式联合训练后，逆动力学 MSE 从 `1.11×10⁻³` 降到 `3.09×10⁻⁴`，策略覆盖率从 74.1% 升到 77.3%；但前向动力学 PSNR 从 27.13 降到 26.22，统一训练不是白捡。

## 局限与边界

- “统一”指架构、表示和训练起点统一，不代表一个通用 checkpoint 无需专项后训练就能同时拿下所有榜单。
- 总参数约为单塔两倍，部署仍要承担两塔的权重显存；报告没有给出与单流统一模型的严格端到端延迟对照。
- 图像、视频、音频和动作仍是迭代去噪，不能把“同一个模型”理解成“一次前向完成所有生成”。
- 多项视频指标依赖 VLM judge。论文也说明公开 PAIBench-G 的部分结果无法复现，因此另用 Qwen2.5-VL-72B 重新评测；排名比单个绝对分更稳妥。
- RoboLab 的 39.7% 虽领先，但也说明复杂操作离可靠部署还很远。

## 我的批注

- Cosmos 3 最有价值的不是“五模态”这个标签，而是把同一个任务改写成“哪些是干净条件、哪些是带噪目标”。这让前向动力学、逆动力学和策略真正共用一套训练接口。
- MoT 的代价很直接：用两套参数换任务隔离。它不像 MoE 那样主要追求稀疏计算；更像让理解工程师和生成工程师各用一套工具，但仍在同一张白板上交换信息。
- 3D MRoPE 的 FPS 调制看起来只是位置编码小改动，却是把“帧编号”升级成“真实时间”的关键。消融也表明它比只在提示词里写 FPS 更有效。

## 历史定位

- 2025 Cosmos 2.x：Reason、Predict、Transfer、Policy 分成不同模型。
- 2026 Cosmos 3：用 MoT 把理解、生成、控制与动作统一到一个模型架构和训练体系。
- 下一步真正值得看的不是还能塞多少模态，而是统一训练能否稳定带来跨任务迁移，并把推理成本压到机器人可实时运行的范围。
