# ReLAT · 给潜在推理"闭环"：测试时用重建当保真检查

## 一句话
潜在推理把中间思考从自然语言搬进不透明的潜在向量里图省事，但没人检查这些向量是否还忠于原问题（开环）。ReLAT 在答题前先做一圈 **问题→潜在思考→重建问题**：如果从潜在向量重建得回原问题，说明它没丢信息；用这个重建损失在测试时临时调一组 LoRA，再正式作答，答完即弃。

## 它要解决的痛点
- **潜在推理是"开环"**：为效率，很多系统把推理的中间步从语言 token 换成连续潜在向量（K 个 ℝ^d 向量，被当条件反复用）。可一旦生成，下游直接消费，**没人验证它是否还保留了原问题的约束**——潜在状态悄悄漂移，错误顺着推理链传下去都发现不了。
- **缺一个"输入锚定"的检查**：以往的测试时方法（如熵最小化）调的是"输出自信度"，跟"有没有忠于输入"无关。需要一个**把潜在状态拽回原问题**的信号。

## 核心贡献
1. **把潜在推理看成"输入保真"问题**：[[reconstruction-as-fidelity]] —— 用原问题 Q **自己**当可观测代理：若潜在 ℰ 真的编码了 Q 的约束，那就该能从 ℰ **重建出 Q**；重建失败=丢了任务相关信息。这是**必要非充分**的检查（重建对≠答案对，但重建错一定丢了信息）。
2. **可微思考表示**：[[differentiable-thought-representation]] —— 潜在每步不硬选一个 token，而是对词表 logits 做 softmax 得分布 α，再算**期望嵌入** `ê = αᵀ W_emb`（各 token 嵌入的加权混合）。这样潜在向量留在嵌入空间里、且**对重建损失可微**，能反传。
3. **ReLAT 测试时训练框架**：[[test-time-training]] —— 冻结主干 θ、只更新一组**临时 LoRA φ**；损失是**对原问题 token 的掩码交叉熵**（只算真正的问题词，模板/padding/格式不算）。跑 N=16 步 AdamW 后用调好的模型对**原始 Q** 作答，然后 **LoRA 复位**，下一题重来。

## 它怎么跑（Algorithm 1）
```
冻结主干 θ；临时 LoRA φ
重复 N=16 步：
  嵌入 Q → 逐步生成 K=16 个可微潜在思考 ê_j (= softmax(logits/τ)ᵀ W_emb)
  以这串潜在为条件，逐 token 重建问题 Q：P(Q|ℰ)=∏ P(q_k|ℰ,q_<k)
  算掩码交叉熵 L_ReLAT（只对原问题 token）
  AdamW 更新 φ（θ 不动）
N 步后：用 (θ, φ*) 对【原始 Q】生成答案 y
作答完 → φ 复位
```
关键：重建出的 Q̂ **只用于算训练损失**；答案来自"调好参数后的模型对原问题"作答，不是从 Q̂ 来。

## 关键数字（§5 实验）
- Qwen3-8B 平均：Single **50.9** → LatentMAS（开环潜在）**61.1** → **ReLAT 67.1**。
- AIME 2024：50.0 → 56.7 → **73.3**（比 Single **+23.3**、比开环 LatentMAS **+16.6**）。
- 跨 Qwen3-4B/8B/14B、DeepSeek-R1-Distill-Qwen-7B 一致提升。
- 效率：ReLAT **10,280 token @63.3%**（AIME2025）vs Self-Refine **64,381 token @53.3%** → **少 84% token 还更准**。
- 消融：lr 最优 η=2e-5；潜在长度 K=16 够用、K=32 不再涨。

## 关键概念 → 概念页
- [[reconstruction-as-fidelity]] · 能重建回输入 = 潜在没丢信息（必要非充分的闭环检查）
- [[differentiable-thought-representation]] · softmax 期望嵌入让"选 token"可微、能反传
- [[test-time-training]] · 每道题临时调参（LoRA）再复位；区别于不改权重的 [[in-context-learning]]

## 我的批注 / 疑问
- 最漂亮的一招是**把"保真"变成"自编码"**：不需要标签、不需要知道正确答案，只问"你还原得回题目吗"——一个纯自监督、输入锚定的信号。这跟 [[generative-data-curation]] 里"caption 准不准"当关口、跟 autoencoder 的母题一脉相承。
- "**必要非充分**"这个分寸很关键：重建对不保证答案对，但重建错铁定丢了信息。所以它是个**过滤器**（把已经跑偏的潜在状态在作答前拦下），不是正确性证明。
- 反直觉的是它**比迭代式自我修正（Self-Refine）省得多**：Self-Refine 反复用语言长篇重写，token 爆炸；ReLAT 在潜在空间里调一组小 LoRA，84% 更省还更准。把"想清楚"从"写更多字"换成"调一下内部状态"。
- 待查：N=16 步的测试时开销 vs 直接多采样几条 CoT 的性价比；K 个潜在向量到底编码了 Q 的什么（可解释性）。
