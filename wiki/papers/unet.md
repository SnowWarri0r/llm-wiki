# U-Net · 一边看清是什么，一边记住它在哪

U-Net 的价值不只是画了一个 U。它把像素级分割里互相打架的两件事装进同一个网络：深层特征负责看懂上下文，浅层特征负责保留边界位置。编码器一路缩小特征图，换来更大的感受野；解码器一路放大，再通过跳跃连接取回对应尺度的细节。此后“下采样提语义、上采样还原空间、同尺度特征横向汇合”成了分割、修复、扩散生成里最常见的骨架之一。

## 一句话

**把整张图一次编码成多尺度语义，再把浅层的精确位置直接拼回解码器，让每个像素同时拿到“这是什么”和“它具体在哪”。**

## 先把任务说清：分类和分割不是一个输出

给一张 `572×572` 的显微图：

- 图像分类只回答一个标签，例如“有细胞”；
- [[semantic-segmentation]] 要为每个像素回答类别，例如“这个像素是细胞，旁边那个是背景”。

所以分割网络不能在最后把整张特征图展平后只吐一个类别。它要保留空间结构，最后得到一张与图像区域逐点对应的类别图。

## 它要解决的痛点

### 痛点 1：逐像素滑窗会把同一片区域重复算很多次

早期做法以某个像素为中心裁一块 patch，让分类器判断中心像素；然后窗口移动一格，再跑一次网络。相邻 patch 大部分内容重合，卷积却从头重算，速度慢得离谱。

### 痛点 2：上下文和定位互相扯后腿

- patch 小：边界位置看得细，但不知道这一小块属于哪个大结构；
- patch 大：上下文够了，却需要更多池化，空间坐标被压粗。

一个像素要分对，既要知道“这是一只细胞”，也要知道“细胞边缘落在哪一列”。

### 痛点 3：医学标注少，而且相邻细胞容易粘成一团

原论文的三套数据只有 30、35、20 张有标注训练图。更麻烦的是，两只紧挨着的细胞中间只隔几像素背景；普通逐像素损失很容易把这条细缝忽略掉。

## 先看完整拼图

U-Net 训练和推理都走同一条主线：

```text
输入图像
→ 编码器：卷积提特征，池化降分辨率
→ 最低分辨率：看见更大范围的上下文
→ 解码器：上采样恢复分辨率
→ 同尺度跳跃连接：裁剪编码器特征并按通道拼接
→ 1×1 卷积：把每个位置的 64 维特征映成 K 个类别分数
→ 每个像素做 softmax，得到类别概率
```

这不是“压下去再放回来”这么简单。**解码器单靠放大只能恢复尺寸，恢复不了池化前已经丢掉的边界坐标；跳跃连接才把这些高分辨率细节重新送回来。**

## 核心贡献

1. **对称编码器—解码器**：深层拿上下文，解码器恢复空间分辨率。
2. **同尺度跳跃连接**：[[skip-connection]] 不是相加，而是裁剪后按通道拼接，让语义与定位在每一层汇合。
3. **整图卷积推理**：[[fully-convolutional-network]] 没有固定长度全连接层，一次前向同时预测一大片像素，并可配合重叠切块处理超大图。
4. **小数据训练**：用平移、旋转、灰度变化和弹性形变扩充数据；原文尤其强调随机弹性形变。
5. **边界加权损失**：[[weighted-pixelwise-cross-entropy]] 把相邻细胞之间的窄缝权重调高，逼模型学会把粘连目标分开。

## 原版结构：尺寸为什么从 572 变成 388

今天很多实现用 `padding=1`，每次 3×3 卷积后尺寸不变。**原论文不是这样。**它使用不补边的 valid convolution：

```text
n_out = n_in - k + 1
```

当卷积核 `k=3`：

```text
572 - 3 + 1 = 570
570 - 3 + 1 = 568
```

每个 3×3 卷积让宽和高各减少 2；连续两次就各减少 4。四次 `2×2, stride=2` 最大池化再把尺寸减半。

### 一条尺寸链从头算到底

```text
输入                    572
编码第 1 层：两次 3×3   572 → 570 → 568
2×2 池化                568 → 284

编码第 2 层             284 → 282 → 280
池化                    280 → 140

编码第 3 层             140 → 138 → 136
池化                    136 → 68

编码第 4 层             68 → 66 → 64
池化                    64 → 32

瓶颈两次 3×3            32 → 30 → 28

上采样 1                28 → 56
两次 3×3                56 → 54 → 52

上采样 2                52 → 104
两次 3×3                104 → 102 → 100

上采样 3                100 → 200
两次 3×3                200 → 198 → 196

上采样 4                196 → 392
两次 3×3                392 → 390 → 388

1×1 分类卷积            388 → 388
```

最终只预测输入中央的 `388×388`。每边少掉：

```text
(572 - 388) / 2 = 92 像素
```

这些边缘位置没有完整上下文，所以原版干脆不输出它们。

## 编码器：缩小空间，扩大视野

每一级执行：

```text
3×3 valid conv + ReLU
→ 3×3 valid conv + ReLU
→ 2×2 max pool, stride 2
```

池化把宽高减半，通道数翻倍：

```text
572²×1
→ 568²×64
→ 280²×128
→ 136²×256
→ 64²×512
→ 28²×1024
```

空间越小，每个深层位置对应的原图范围越大，也就是 [[receptive-field]] 更大。模型因此能根据周围组织判断一个模糊像素属于哪类结构。

## 解码器：上采样只恢复格子，跳跃连接才恢复位置

解码每一级先把特征图放大 2 倍，并把通道减半。论文称它为 `2×2 up-convolution`；今天通常用 [[transposed-convolution]] 或“插值 + 普通卷积”实现。

但放大不是时光倒流。池化前一块 `2×2` 里究竟哪个像素最亮，池化后通常只剩最大值。把这个值复制回 `2×2`，无法知道原来亮点在哪。

于是 U-Net 从编码器横向复制同尺度特征：

```text
解码器上采样特征：56×56×512
编码器同尺度特征：64×64×512
```

原版尺寸不同，先从编码器四周各裁 4 像素：

```text
(64 - 56) / 2 = 4
```

得到 `56×56×512`，再按通道拼接：

```text
56×56×512 concat 56×56×512
= 56×56×1024
```

后续两次 3×3 卷积把这两份信息揉在一起。深层分支回答“这里大概是细胞”，浅层分支补上“边缘具体落在这里”。

### 为什么用 concat，而不是加法

相加会先把两份通道一一混在一起；拼接则暂时保留两份完整证据，让后面的卷积自己决定怎么融合。代价是拼接后的通道数更多，显存和计算量更大。

## 最后一层 1×1 卷积到底做了什么

到输出前，每个位置有 64 个特征。`1×1` 卷积不看邻居，只把这个位置的 64 维向量线性映射成 `K` 个类别分数：

```text
输入：388×388×64
权重：1×1×64×K
输出：388×388×K
```

假设只有“细胞 / 背景”两类，某个像素输出 logits `[2, 0]`：

```text
exp(2)=7.389, exp(0)=1
总和=8.389

p(细胞)=7.389/8.389=0.881
p(背景)=1/8.389=0.119
```

这一步仍然在每个空间位置独立执行，因此不会把整张图压成一个标签。

## 全卷积为什么能吃不同尺寸

全连接层要求固定长度输入。例如把 `32×32×64` 展平成 65536 维后接一个权重矩阵，输入改成 `40×40×64`，矩阵维度立刻对不上。

卷积核没有绑定某个绝对位置。同一组 `3×3` 权重能从左上滑到右下；输入变大，核只是多滑几步，输出特征图也跟着变大。U-Net 又没有把特征图展平进固定长度全连接层，所以空间尺寸可以调整。

但“可变”不等于“任意整数都直接合法”。原版有四次除以 2 的池化，还要求每次池化前尺寸为偶数。兼容的输入边长每增加 16，输出边长也增加 16：

```text
572 → 388
588 → 404
604 → 420
```

## 超大图怎么切：overlap-tile

一张显微图可能大到 GPU 放不下。原版把它切成输入 tile：

- 每块输入 `572×572`；
- 只取中央 `388×388` 的可靠预测；
- 相邻输出块首尾相接；
- 因为每块需要四周 92 像素上下文，相邻输入块会重叠 `92+92=184` 像素；
- 整张图最外缘缺少上下文，就镜像外推边界像素。

这和今天常见的“带 halo 的分块推理”是一回事：重叠区不是重复输出，而是给每块中央结果提供完整上下文。

## 损失：为什么相邻细胞之间那条缝要更贵

普通逐像素交叉熵：

```text
L(x) = -log p_{l(x)}(x)
```

- `x`：某个像素坐标；
- `l(x)`：这个像素的真实类别；
- `p_{l(x)}(x)`：模型给真实类别的概率；
- `-log`：概率越接近 1，损失越接近 0；越自信地答错，惩罚越大。

U-Net 再乘一个位置权重 `w(x)`：

```text
L = - Σ_{x∈Ω} w(x) log p_{l(x)}(x)
```

- `Ω`：输出区域里所有像素的集合；
- `Σ`：把所有像素损失加起来；
- `w(x)`：该像素的重要程度。

论文给的权重图为：

```text
w(x) = w_c(x)
     + w_0 · exp( - (d_1(x)+d_2(x))² / (2σ²) )
```

- `w_c(x)`：类别平衡权重，防止面积大的背景淹没小目标；
- `d_1(x)`：像素到最近细胞边界的距离；
- `d_2(x)`：到第二近细胞边界的距离；
- 两个距离都小，说明像素夹在两个细胞之间；
- `w_0=10`：论文设置的额外权重幅度；
- `σ≈5` 像素：高权重带扩散多宽。

### 用一个边界像素从头算到损失

取 `w_c=1, d_1=1, d_2=1, w_0=10, σ=5`：

```text
d_1+d_2 = 2
(d_1+d_2)² = 4
2σ² = 2×25 = 50

exp(-4/50) = exp(-0.08) ≈ 0.9231
w(x) = 1 + 10×0.9231 = 10.231
```

若这个像素真实类别是背景，模型却只给背景概率 `0.119`：

```text
普通交叉熵 = -log(0.119) ≈ 2.127
加权损失   = 10.231×2.127 ≈ 21.76
```

同样的错误发生在普通区域，权重若为 1，损失只有 2.127。这样训练会优先修正“把两个细胞粘在一起”的错误。

### 原论文公式少了负号

论文 Eq. (1) 印成 `Σ w log p`，但文字说它是要惩罚错误的交叉熵，Caffe 的 softmax loss 也是最小化负对数似然。两种一致写法是：

- 最大化 `Σ w log p`；
- 或最小化 `-Σ w log p`。

正文采用第二种，避免出现“正确概率越高，loss 反而越大”的方向矛盾。

## 初始化：`√(2/N)` 里的 N 是什么

论文使用适合 ReLU 的 He 初始化。每个权重从均值 0 的高斯分布采样，标准差：

```text
std = √(2/N)
```

`N` 是一个输出神经元接收的输入数。若上一层有 64 个通道，卷积核为 3×3：

```text
N = 3×3×64 = 576
std = √(2/576)
    = √(1/288)
    ≈ 0.0589
```

它不是说所有权重都等于 0.0589，而是从 `N(0, 0.0589²)` 中随机抽。目标是让信号经过卷积和 ReLU 后，方差不要一层层爆炸或消失。

## 小数据怎么训：弹性形变不是随便扭图

原文只有几十张训练图，所以增强承担了很重的职责：

1. 平移与旋转：同一组织换位置、换朝向；
2. 灰度变化：模拟显微镜成像亮度差异；
3. 弹性形变：先在粗糙 `3×3` 控制网格上采样随机位移，位移服从标准差 10 像素的高斯分布；
4. 用双三次插值把 9 个控制点的位移平滑扩展到每个像素；
5. 对图像和标注 mask 使用同一份形变，否则输入和答案会错位。

它模拟的是组织被挤压、拉伸后的平滑变形，不是每个像素各自乱跑。论文还在编码器末端使用 dropout，提供额外随机扰动。

## 训练配方

- 框架：Caffe；
- 优化：随机梯度下降；
- batch size：1，作者宁可把 GPU 显存留给更大的输入 tile；
- momentum：0.99，用历史梯度平滑 batch=1 带来的抖动；
- 卷积层：23 层；
- 训练时间：NVIDIA Titan 6 GB 上约 10 小时；
- 推理速度：当时 GPU 上处理一张 `512×512` 图少于 1 秒。

这些是 2015 年硬件和 Caffe 实现的数，不能直接拿来代表现代 PyTorch U-Net 的速度。

## 实验到底证明了什么

### EM 神经结构分割

- 训练集：30 张 `512×512` 果蝇幼虫神经组织电镜图；
- 推理时平均 7 个旋转版本；
- 不做额外预处理或后处理；
- warping error：`0.000353`；
- Rand error：`0.0382`；
- pixel error：`0.0611`。

此前滑窗 CNN IDSIA 的 warping error 为 `0.000420`，U-Net 更低。但不同指标并非全部第一：DIVE-SCI 的 Rand error `0.0305`、pixel error `0.0584` 都低于 U-Net。

### PhC-U373 细胞分割

- 35 张部分标注训练图；
- 平均 IoU：`0.9203`；
- 2015 年第二名：`0.83`。

IoU 就是预测区域和真实区域的交集除以并集。若重合 80 个像素，并集 100 个像素：

```text
IoU = 80 / 100 = 0.8
```

### DIC-HeLa 细胞分割

- 20 张部分标注训练图；
- 平均 IoU：`0.7756`；
- 2015 年第二名：`0.46`。

这组差距最大，也最能支持“少量标注 + 强增强 + U 形结构”的组合有效。

## 实验没有证明什么

论文没有给现代意义上的逐项消融：

- 没有单独移除跳跃连接；
- 没有单独移除弹性形变；
- 没有比较普通交叉熵和边界加权损失；
- 没有对照 same padding；
- 没有多随机种子的均值与方差。

因此只能说整套方案在三项任务上有效，不能从这篇论文的表格精确分摊“每个零件贡献几分”。

## 原版 U-Net 与今天常说的 U-Net

| 项目 | 2015 原版 | 今天常见实现 |
|---|---|---|
| 3×3 卷积 | valid，不补边 | 常用 padding=1 保持尺寸 |
| 跳跃连接 | 先裁剪，再 concat | 尺寸相同，直接 concat |
| 上采样 | upsampling + 2×2 up-conv | 转置卷积或插值 + 卷积 |
| 归一化 | 没有 BatchNorm | 常见 BN / GN |
| 损失 | 边界加权像素交叉熵 | 常混合 Dice / CE / focal |
| 任务 | 生物医学像素分割 | 分割、修复、扩散去噪等 |

把扩散模型的“U-Net”理解成同一个思想家族即可，不要以为 Stable Diffusion 原封不动用了这套 23 层 Caffe 网络。扩散 U-Net 保留多尺度下采样—上采样和横向跳连，但加入残差块、时间条件、注意力，并把输出改成噪声或速度预测。

## 我的批注

- **真正的突破是信息分工，不是字母造型。**编码器负责语义，跳连负责定位，解码器负责把两者逐级合并。
- **上采样不会自动找回细节。**把 28×28 放大成 56×56 只增加格子，不会凭空知道边界原先在哪；跳连才是关键数据源。
- **原版的 crop 不是 U-Net 永恒教义。**它只是 valid convolution 造成的尺寸账。换成 same padding 后不需要裁剪，U-Net 的核心仍然成立。
- **边界权重是把任务结构写进损失。**作者知道“分开相邻细胞”比普通背景像素更重要，于是直接让这些像素的错误更贵。
- **少数据并非只靠架构。**几十张图能训练，强数据增强和任务先验占了很大功劳；不能把结果全归给跳跃连接。
- **原文没有 Dice loss。**“医学分割就用 Dice”是后来实践，不是这篇 2015 论文的训练目标。

- 2014 FCN · 把分类 CNN 改成整图像素预测
- 2015 **U-Net** · 对称解码器 + 同尺度跳连 + 小数据增强
