# VITS · 文本进去，波形出来

## 一句话
VITS 把“声学模型先做 mel、声码器再做波形”改成了一套联合训练、并行生成的系统：训练时从真实语音里学习“文字应该对应什么声音”，推理时只给文字，就能一次并行地产生整段波形。它用条件 VAE 管声音潜变量、归一化流连接文本与音频分布、MAS 学对齐、随机时长预测器学节奏，最后用 GAN 把波形细节磨真。

这里的“端到端”要说准确：VITS 消掉的是**两个独立模型之间固定的 mel 接口**。训练内部仍然会计算线性谱、mel 频谱和潜变量，只是它们不再是需要单独训练、落盘再喂给下一个模型的产品接口。

## 先看全局：五个模块各管一件事

假设文本是“你好”，经过音素化得到 3 个音素，而真实录音被切成 8 个声学帧。VITS 要回答五个问题：

1. **后验编码器**：这段真实录音里，到底藏着什么音色、音高和发音细节？把线性谱编码成潜变量 `z`。
2. **MAS**：8 帧声音分别属于哪个音素？找到一条单调、不跳音素的对齐路径，例如时长 `[3, 2, 3]`。
3. **文本先验 + 归一化流**：推理时没有录音，怎样只看文本也能采到像 `z` 的潜变量？让文本分布经过可逆流后贴近音频后验。
4. **随机时长预测器**：推理时没有 MAS 可以看真实音频，3 个音素各说多久？学习时长分布，而不是死记一个固定答案。
5. **波形解码器 + 判别器**：怎样把 `z` 直接变成听起来像真人的波形？生成器负责说，判别器负责挑毛病。

训练路线是：`真实音频 → 线性谱 → 后验 z → 波形重建`，同时用文本先验、MAS、时长预测和判别器从旁约束。推理路线更短：`文本 → 时长 → 文本先验 → 流的逆变换 → z → 波形`。后验编码器和判别器都只在训练时使用。

## 01 · 它究竟改进了两段式 TTS 的什么

传统两段式系统通常这样做：

```text
文本 ──声学模型──> 预测 mel ──声码器──> 波形
                    ↑
声码器训练时看到的是“真实语音算出的 mel”
推理时看到的却是“声学模型猜出的 mel”
```

两个模块各自训练时都可以很好，但接起来以后，声码器突然要处理一类训练时没见过的、有误差的 mel。这叫 **exposure mismatch / 中间表示失配**。论文中 Tacotron 2 + HiFi-GAN 直接拼接的 MOS 是 `3.77`，用声学模型生成的 mel 再微调声码器后升到 `4.25`，就是这类失配存在的直接证据。

VITS 把文本侧、潜变量和波形生成器放进同一个训练目标里。生成器训练时处理的潜变量，就是整套系统共同学出来的潜变量，因此不用再给两个独立模型补接口。它并没有证明 mel “完全没用”：训练重建损失仍然在 80 维 mel 上比较，后验编码器还读线性谱；真正被拿掉的是**推理链路上那个必须由前级模型预测、再交给后级模型消费的固定 mel 瓶颈**。

## 02 · 条件 VAE：为什么训练看音频，推理只看文本也能说

直接让一段文本对应唯一波形并不合理。同一句“今天下雨了”，可以快说、慢说、升调、降调，也可以由不同说话人来说。条件 VAE 把这份没有写在文字里的变化收进潜变量 `z`。

- `x_lin`：真实语音的线性频谱，后验编码器的输入。
- `q_φ(z | x_lin)`：后验分布。因为看过真实声音，所以知道这一次具体怎样说。
- `c_text`：文本编码器输出的音素条件。
- `p_θ(z | c_text)`：文本先验。推理时只能看文字，要靠它产生可能的声音。
- `G(z)`：解码器，也就是波形生成器；把潜变量变成原始波形。

训练时，后验编码器输出每个潜变量维度的均值 `μ_q` 和标准差 `σ_q`，再用重参数化采样：

```text
ε ~ N(0, I)
z = μ_q + σ_q ⊙ ε
```

`ε` 是标准高斯噪声，`⊙` 是逐元素相乘。这样既能采样，又能让梯度穿过 `μ_q`、`σ_q` 回到编码器。

VAE 的核心有两股力：

```text
L_recon = || x_mel - mel(G(z)) ||₁
L_kl    = log q_φ(z | x_lin) - log p_θ(z | c_text)
```

- `x_mel`：真实语音的 mel 频谱。
- `mel(G(z))`：把生成波形也转成 mel 频谱。
- `||·||₁`：把每个时频格子的绝对误差相加。它逼生成器把内容和声学轮廓还原对。
- `L_kl`：同一个 `z` 在后验下有多合理，减去它在文本先验下有多合理。平均以后就是 KL 散度，逼文本先验覆盖后验能产生的声音。

一个只看均值、并假设两边方差都为 1 的简化例子：后验均值 `μ_q=1.2`，文本先验均值 `μ_p=0.8`，那么一维高斯的 KL 是

```text
KL = 1/2 · (μ_q - μ_p)²
   = 1/2 · (1.2 - 0.8)²
   = 0.08
```

如果先验均值从 `0.8` 移到 `1.1`，KL 就变成 `0.005`。直觉上，它在教文本侧：“以后没录音可看时，也要把 `z` 放到后验经常出现的位置。”真实公式还会比较方差；这里固定方差只是为了看清“两个分布离得越远，罚得越重”。

## 03 · 归一化流：文本先验太简单，就把坐标系掰弯

单个对角高斯适合计算，却很难表示真实语音里弯曲、多峰、互相关联的潜变量分布。若硬用简单先验去贴复杂后验，KL 会逼后验也变简单，生成细节容易丢。

VITS 在两者之间插入可逆函数 `f`：

```text
训练：后验 z ──f──> z_p      在文本先验 p(z_p | c_text) 下算概率
推理：采样 z_p ──f⁻¹──> z    再交给解码器生成波形
```

- `z`：音频后验空间里的潜变量，适合解码器。
- `z_p`：文本先验空间里的潜变量，适合用高斯描述。
- `f` / `f⁻¹`：同一组可逆耦合层的正向与反向。

为了建立直觉，假设有个玩具流 `z_p = z + 2`。后验里 `z=1.5`，正向得到 `z_p=3.5`；推理时先验采到 `3.5`，逆变换减 2，又回到解码器认识的 `1.5`。平移不改变体积，雅可比行列式为 1，所以不需要额外修正密度。VITS 的真实流当然是多层非线性仿射耦合，但论文也特意使用了**体积保持**版本：耦合层不预测缩放，`|det J|=1`，让似然计算更轻。

这不是可有可无的装饰。论文的 30 万步消融实验里，完整基线 MOS 为 `4.50`，拿掉归一化流后只剩 `2.98`，下降 `1.52`。注意这组数来自专门的 30 万步消融实验，不能和主实验训练到 80 万步的 `4.43` 直接横比；但同组内的巨大落差足以说明流在连接文本先验和音频后验时很关键。

## 04 · MAS：不知道谁对应谁，就用动态规划找一条最佳路径

训练数据只有“整句文字 + 整段录音”，通常没有标出第几个音素从第几帧开始。假设 3 个音素要占满 5 帧，合法对齐必须满足：

- **单调**：时间只能从前往后，不能说完第三个音素又回到第一个。
- **不跳过**：每个音素至少拿到一帧。
- **全覆盖**：每个声学帧都分给一个音素。

先计算每个“音素 i — 帧 j”组合的匹配分数 `s(i,j)=log p(z_j | c_i)`：这帧潜变量放在这个音素的先验下，概率越高，分数越大。用一个可手算的例子：

```text
              帧1  帧2  帧3  帧4  帧5
音素 A         4    3    1    0    0
音素 B         0    2    4    3    1
音素 C         0    0    1    3    5
```

动态规划状态 `Q(i,j)` 表示“用前 i 个音素解释前 j 帧”的最高累计分数：

```text
Q(i,j) = s(i,j) + max(Q(i,j-1), Q(i-1,j-1))
```

到当前帧只有两种合法来路：`Q(i,j-1)` 表示继续说当前音素；`Q(i-1,j-1)` 表示从上一个音素切过来。比如 B 在帧 2 的累计分是 `4+2=6`；走到 B 的帧 3，继续 B 得到 `6+4=10`，从 A 切来得到 `7+4=11`，所以选 11。回溯整张表得到路径 `A,A,B,C,C`，也就是时长 `[2,1,2]`，累计分数 `4+3+4+3+5=19`。

这就是 MAS（Monotonic Alignment Search）。它不靠外部强制对齐器，而是在训练中反复做两件事：当前模型先找最像的对齐；新的对齐再反过来训练文本先验。最终得到的每音素帧数，也正好成为时长预测器的训练标签。

## 05 · 随机时长预测器：随机到底从哪来

推理时没有真实录音，MAS 无从搜索，因此模型必须只看文字，决定每个音素占多少帧。这里的 `d` 不是一句话的总秒数，而是**逐音素的整数序列**。例如三个音素的 MAS 对齐结果是：

```text
d = [2, 4, 3]    # 第一个音素 2 帧，第二个 4 帧，第三个 3 帧
```

普通时长预测器学的是“文字 → 一个答案”，同一句话每次都会得到相同的 `[2,4,3]`。VITS 想学的是 `p_θ(d | c_text)`：给定同一份文本表示 `c_text`，哪些时长组合常见、各有多常见。这里 `p_θ` 不是一张手写概率表，而是一套参数为 `θ` 的 Flow 网络。

### 先分清两种随机：训练辅助随机，不等于推理采样随机

Flow 擅长处理连续向量，真实时长却是 `1、2、3…` 这样的整数。VITS 在训练时引入 `u` 和 `ν`，只是为了把整数标签改造成 Flow 能计算密度的连续对象：

- `u`：去量化变量，与 `d` 同形状，每项都在 `[0,1)`。`d-u` 会落在能唯一还原 `d` 的区间内。
- `ν`：增广变量，也与 `d` 同形状。它给每个标量时长添一个辅助维度，让可逆 Flow 不必只在一条线上塑形。

沿用 `d=[2,4,3]`，训练时辅助后验采到：

```text
u = [0.2,  0.6, 0.1]
ν = [0.5, -0.7, 0.2]

d-u = [1.8, 3.4, 2.9]
拼接后 = [[1.8, 3.4, 2.9],
          [0.5,-0.7, 0.2]]
```

`u` 没有篡改时长答案：`ceil([1.8,3.4,2.9])=[2,4,3]`，正好还原原标签。它只是把三个孤零零的整数摊成三个连续小区间。`ν` 最后也不会被当作时长输出；它只是训练 Flow 时借来的一条辅助坐标轴。

训练时的 `u、ν` 来自 `q_φ(u,ν | d,c_text)`。这个 `q_φ` 是一个**只在训练时上班的辅助后验**：它既看文本，也看 MAS 给出的正确时长 `d`，再挑出适合当前样本的 `u、ν`。推理时没有正确时长可看，所以 `q_φ` 整个被拿掉。

### 那一长串公式到底在算什么

真正想提高的是：只给文本时，模型把真实时长 `d` 判得有多靠谱。

```text
log p_θ(d | c_text)
```

但把所有可能的 `u、ν` 都积分一遍才能精确算它，代价太高。于是训练一个容易采样的 `q_φ`，用它抽几组 `u、ν`，计算下面这个可训练的下界：

```text
log p_θ(d | c_text)
≥ E_{q_φ(u,ν | d,c_text)} [
      log p_θ(d-u, ν | c_text)
    - log q_φ(u,ν | d,c_text)
  ]
```

每个符号的岗位是：

- `d`：MAS 数出的真实逐音素时长，例如 `[2,4,3]`。
- `c_text`：文本编码器给出的条件；它告诉 Flow 当前是什么音素、上下文是什么。
- `θ`：负责生成时长的主 Flow 参数；`p_θ` 给连续二通道表示打密度分。
- `φ`：训练辅助后验的参数；`q_φ` 负责提出值得检查的 `u、ν`。
- `E_q[...]`：从 `q_φ` 多采几次再取平均；代码通常用少量蒙特卡洛样本近似。
- 第一项 `log p_θ(...)`：主 Flow 觉得这组“连续时长 + 辅助坐标”有多像真实数据，越大越好。
- 第二项 `-log q_φ(...)`：校正“样本是由 q 挑出来的”这件事，不能把 q 的偏好冒充成 p 的能力。
- `≥`：右边只是左边的可计算下界，不是精确相等；把下界抬高，也会推动真实时长的对数似然提高。

用一个标量密度例子看清符号。假设这一组 `[d-u,ν]` 在主 Flow 下的密度是 `p=0.20`，在辅助后验下的密度是 `q=0.50`：

```text
单次下界分数 = log 0.20 - log 0.50
              = -1.609 - (-0.693)
              = -0.916

L_dur = -下界分数 = 0.916
```

为了单看主 Flow 的进步，暂时固定 `q=0.50`。若训练后 `p` 从 `0.20` 升到 `0.40`：

```text
新分数 = log 0.40 - log 0.50 = -0.223
新 L_dur = 0.223
```

损失从 `0.916` 降到 `0.223`，意思就是：主 Flow 更愿意给这组真实时长附近的连续表示高密度。正式训练里 `p_θ` 与 `q_φ` 会一起更新；`L_dur` 定义为整条下界的负数，因此优化器仍然只需做熟悉的“最小化损失”。

### 推理时，真正的随机从高斯噪声来

推理时固定同一份 `c_text`，重新采一份高斯噪声 `e∼N(0,I)`，再让主 Flow **反向**运行：

```text
高斯噪声 e → 主 Flow 逆变换 → log w → exp → 连续时长 w → ceil → 整数帧数
```

官方实现确实先输出 `log w`，再执行 `w=exp(log w)` 和向上取整。下面的数字只用来演示这条计算链：

```text
噪声 A → log w=[0.59,0.92,0.69]
       → w≈[1.80,2.51,1.99]
       → ceil(w)=[2,3,2]

噪声 B → log w=[0.80,1.31,0.92]
       → w≈[2.23,3.71,2.51]
       → ceil(w)=[3,4,3]
```

文本没有变，噪声变了，时长组合就可能变；这才是“随机时长”的直接来源。`noise_scale_w` 调节采样噪声的幅度，`length_scale` 则在取整前整体拉长或压短时长。论文对同一句话采样 100 次，VITS 得到了一段时长分布；确定性 Glow-TTS 每次只有同一个长度。

文本条件在 SDP 入口做 stop-gradient：时长模块可以学习自己的 `θ、φ`，但不能反过来拖着文本编码器改表示来迁就它。

## 06 · GAN：先把 G、D 和三条损失认清

只用 mel L1，模型容易把多种可能的高频细节平均掉，听感会发闷。VITS 沿用 HiFi-GAN 的思路，用一个会造波形的网络和一组会挑波形毛病的网络互相训练。

这里的括号只是“把输入送进网络”，不是默认读者已经学过一套 GAN 黑话：

- `z`：后验编码器从真实语音得到的潜变量。
- `G`：生成器，也就是 VITS 的波形解码器；`G(z)` 是它根据 `z` 造出的假波形。
- `y`：数据集里的真实波形。
- `D`：判别器；`D(y)` 是它给真波形的分数，`D(G(z))` 是它给假波形的分数。

VITS 的 `D` 实际是六个子判别器的集合，输出的也是局部得分图，不是一个严格的“真假概率”。为了看懂公式，可以先把它们压成一个分数：least-squares GAN 训练 `D` 把真声推向 `1`、假声推向 `0`，训练 `G` 把自己的假声也推向 `1`。

它使用周期 `[1,2,3,5,7,11]` 的多个判别器。以周期 3 为例，一维波形会按每 3 个采样点折成二维：

```text
[x1 x2 x3 x4 x5 x6 x7 x8 x9]
              ↓ period = 3
[x1 x2 x3]
[x4 x5 x6]
[x7 x8 x9]
```

卷积沿列比较相同相位的位置，更容易发现嗓音的周期结构有没有破。`period=1` 相当于直接看原始波形；其余 `[2,3,5,7,11]` 来自 HiFi-GAN 的多周期判别器。VITS 为了效率没有保留 HiFi-GAN 里多个池化尺度的全部判别器。

三条损失分别训练不同对象。公式里的 `E[...]` 就是对 batch、局部得分以及多个子判别器求平均或求和。

```text
L_adv(D) = E[(D(y)-1)² + D(G(z))²]
L_adv(G) = E[(D(G(z))-1)²]
```

`L_adv(D)` 只更新判别器。若 `D(y)=0.8`、`D(G(z))=0.3`：

```text
L_adv(D) = (0.8-1)² + 0.3²
         = 0.04 + 0.09
         = 0.13
```

第一项催 `D(y)` 从 `0.8` 靠近 `1`；第二项催假声分数从 `0.3` 靠近 `0`。

轮到 `L_adv(G)` 时，先把 `D` 的参数冻结，只沿着 `D(G(z))` 把梯度传回 `G`：

```text
L_adv(G) = (0.3-1)² = 0.49
```

如果 `G` 改进后能让固定的 `D` 给出 `0.7`，它的损失就会降为 `(0.7-1)²=0.09`。看似同一个假声分数一会儿要去 `0`、一会儿要去 `1`，并不矛盾：前一步更新裁判 `D`，后一步更新选手 `G`，两边轮流走。

只看最终真假分数仍然太粗，于是 `G` 还要承担特征匹配损失：

```text
L_fm(G) = E[ Σ_{l=1..T} (1/N_l) · ||D_l(y)-D_l(G(z))||_1 ]
```

- `l`：判别器的第几层；`T` 是拿来比较的总层数。
- `D_l(x)`：输入 `x` 经过判别器第 `l` 层后得到的中间特征图，并不是另一个新判别器。
- `N_l`：第 `l` 层特征元素个数；除以它是为了算该层的平均误差，避免大特征图天然占便宜。
- `||a-b||_1`：把对应元素的绝对差全部加起来。

假设只比较两层：第一层真特征 `[2,-1]`、假特征 `[1.5,-0.5]`，第二层真特征 `[0.4]`、假特征 `[0.1]`：

```text
第 1 层平均差 = (|2-1.5| + |-1-(-0.5)|) / 2 = 0.5
第 2 层平均差 = |0.4-0.1| / 1 = 0.3
L_fm(G) = 0.5 + 0.3 = 0.8
```

它等于告诉生成器：“最终分数骗过去还不够，裁判逐层看到的周期、局部纹理和波形形状也要接近真声。”训练 `G` 时，VITS 同时使用对抗损失和特征匹配；训练 `D` 时不使用特征匹配。

官方训练代码的顺序可以压缩成：

```text
① G(z).detach() → 算 L_adv(D) → 只更新 D
② 重新算 G(z)  → 算 L_adv(G)+L_fm(G)+其他生成侧损失 → 更新 G 与生成侧模块
```

## 07 · 五项损失：每一项到底在管谁

生成侧总损失是：

```text
L_vae = L_recon + L_kl + L_dur + L_adv(G) + L_fm(G)
```

- `L_recon`：真实 mel 与生成波形的 mel 做 L1；管“内容和声学轮廓对不对”。
- `L_kl`：后验与经过流连接的文本先验做 KL；管“推理时只看文本能不能采到可解码的 z”。
- `L_dur`：随机时长模型的负变分下界；管“每个音素说多久”。
- `L_adv(G)`：生成器骗判别器的损失；管“波形像不像真的”。
- `L_fm(G)`：真假波形在判别器中间层的特征差；给对抗训练更稳定、更细的方向。

判别器另有自己的 `L_adv(D)`。所以“一个端到端模型”不等于“只有一条简单损失”：VITS 的推理链路很干净，训练系统却相当复杂，后验编码器、流、对齐、SDP、生成器和判别器要同时配合。

## 08 · 模型与训练配方

- **后验编码器**：16 个 WaveNet 残差块；输入线性频谱，潜变量 192 通道。
- **文本先验**：文本编码器预测高斯均值与方差，再接 4 层体积保持的仿射耦合流；每层含 4 个 WaveNet 残差块。
- **解码器**：基本沿用 HiFi-GAN V1 生成器，用转置卷积逐级上采样，再用多感受野残差块补局部细节。
- **随机时长预测器**：4 层神经样条耦合层，用 192 维隐藏状态与深度可分离卷积。

数据与预处理：

- LJ Speech：单说话人，13,100 段、约 24 小时、22.05 kHz；训练/验证/测试为 12,500/100/500。
- VCTK：109 位说话人，约 44,000 段、44 小时；从 44.1 kHz 降到 22.05 kHz，切分 43,470/100/500。
- 文本先转成 IPA 音素，并在音素之间插 blank token。
- 后验输入是线性谱，STFT 的 FFT/window/hop 为 `1024/1024/256`；重建损失用 80 维 mel。
- 不整段解码训练，而是从潜变量随机截 32 帧，再只生成对应波形片段，显著省显存和时间。
- AdamW，`β₁=0.8`、`β₂=0.99`、weight decay `0.01`，初始学习率 `2×10⁻⁴`；4 张 V100、混合精度，最多训练 80 万步。

## 09 · 实验：音质、多说话人和速度

LJ Speech 单说话人 MOS（越高越好，括号是 95% 置信区间）：

| 系统 | MOS |
|---|---:|
| 真人录音 | 4.46 ± 0.06 |
| Tacotron 2 + HiFi-GAN | 3.77 ± 0.08 |
| Tacotron 2 + 微调 HiFi-GAN | 4.25 ± 0.07 |
| Glow-TTS + HiFi-GAN | 4.14 ± 0.07 |
| Glow-TTS + 微调 HiFi-GAN | 4.32 ± 0.07 |
| VITS（确定性时长） | 4.39 ± 0.06 |
| **VITS（随机时长）** | **4.43 ± 0.06** |

VCTK 多说话人 MOS：VITS 为 `4.38 ± 0.06`，真人录音为 `4.38 ± 0.07`；Glow-TTS + 微调 HiFi-GAN 为 `3.82 ± 0.07`。这说明同一套方法不只适用于单一声音。论文的 CMOS 仍显示听众对真人有小幅偏好：LJ Speech 为 `-0.106`，VCTK 为 `-0.262`，因此更准确的说法是“主观质量非常接近”，而不是“已经听不出差别”。

单张 V100、batch 1、100 句测试文本的合成速度：

| 系统 | 采样吞吐 | 实时倍数 |
|---|---:|---:|
| Glow-TTS + HiFi-GAN | 606.05 kHz | ×27.48 |
| VITS | 1480.15 kHz | ×67.12 |
| VITS（确定性时长） | 2005.03 kHz | ×90.93 |

这里“×67.12 实时”是说：生成 1 秒语音只需大约 `1/67.12 ≈ 0.015` 秒，不是“音频播放速度变成 67 倍”。随机时长预测器更慢一些，但主体仍是非自回归并行生成。

## 10 · 消融到底证明了什么

论文另训了一组 30 万步模型做消融：

| 版本 | MOS | 相对完整基线 |
|---|---:|---:|
| 完整基线 | 4.50 ± 0.06 | — |
| 去掉归一化流 | 2.98 ± 0.08 | −1.52 |
| 后验输入改用 mel 谱 | 4.31 ± 0.08 | −0.19 |

- **流是关键桥梁**：简单文本先验直接贴复杂音频后验，质量大幅崩掉。
- **线性谱保留的信息更足**：mel 轴会压缩频率细节，拿它做后验输入会有可测下降；不过训练的重建目标用 mel 仍然有效。
- **随机时长改善不只看 0.04 MOS**：对同一句话采样 100 次，VITS 的时长分布接近自回归 Tacotron 2 的自然波动，而确定性 Glow-TTS 没有变化；F0 曲线也显示音高和节奏都在改变。多说话人模型还学到了不同说话人的时长习惯。

## 11 · 额外能力：同一潜变量还能做声音转换

多说话人版本里，文本编码器不接收说话人 ID，因此流空间中的表示更偏向“说了什么”。附录据此做 voice conversion：

```text
源说话人音频
  → 后验 z（带源说话人条件）
  → 正向流，得到较独立于说话人的表示
  → 逆向流（换成目标说话人条件）
  → 解码器（目标说话人条件）
  → 内容不变、声音换人
```

这不是论文主实验的核心贡献，但很能说明“可逆流”不只是为了算 KL：同一个潜空间还提供了把内容与说话人条件重新组合的通路。

## 12 · 局限与今天该怎样看它

- **训练复杂**：后验、流、动态对齐、时长流、生成器和多个判别器一起优化；推理简单不代表训练容易稳定。
- **对齐假设有边界**：MAS 强制单调且不跳音素，适合正常朗读；对重复、漏读、倒序或非常规演唱式发声并不天然合适。
- **依赖音素化**：论文明确把文本转 IPA，系统仍需要语言相关的前端，并非原始字符拿来就能跨语言工作。
- **多样不等于可控**：换噪声能改节奏和音高，但用户很难精确指定“第三个字慢 20%、句尾升两个半音”。
- **实验范围有限**：主要是 22 kHz 英语朗读数据，使用主观 MOS；没有验证噪声文本、长篇稳定性、跨语言或零样本说话人泛化。
- **时代位置**：VITS 是连续潜变量端到端 TTS 的代表作。后来的系统常把语音离散成 codec token，再交给语言模型或非自回归模型；范式变了，但“联合训练消接口失配、显式处理对齐与一对多”仍然是非常可复用的设计思想。

## 我的批注

VITS 最漂亮的地方不是把 VAE、Flow 和 GAN 三个热词堆在一起，而是先承认 TTS 同时有五种不同问题，再给每种问题安排合适的工具：VAE 容纳同一句话的多种声音，Flow 解决文本先验表达力，MAS 解决未知对齐，SDP 解决节奏的一对多，GAN 解决波形锐度。把训练/推理两条路线真正走一遍，会发现每个模块都有明确岗位，没有谁只是为了让架构图更热闹。
