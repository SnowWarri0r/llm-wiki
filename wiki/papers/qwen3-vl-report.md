# Qwen3-VL · 一张图片怎样进入语言模型

## 一句话

Qwen3-VL 先用视觉编码器把图片或视频变成视觉 token，再用 merger 把这些 token 改成 Qwen3 能接收的维度，最后让视觉 token 与文字 token 进入同一个自回归语言模型。它并不是让 LLM 直接读取 JPEG，也不是单独训练一个“看图答案头”。

## 先拼完整的数据流

以“这张图里有几只猫？”为例：

1. 图片进入继续训练过的 SigLIP-2 视觉编码器，也叫 Qwen3-ViT。
2. 视觉编码器输出一张二维特征网格；每个格子是一个向量，代表局部画面。
3. 两层 MLP merger 每次取相邻 `2×2=4` 个视觉特征，合成一个视觉 token，并把向量宽度改成 Qwen3 LLM 的隐藏维度。
4. 视觉 token、问题文字 token 被排进同一条序列。
5. Qwen3 像普通语言模型一样逐 token 生成答案，例如“有 3 只猫”。

所以它的主体仍是“预测下一个文字 token”的 decoder-only LLM。视觉编码器负责看，merger 负责接线，LLM 负责综合图像和文字后作答。

## 1. 三个主体模块

### 视觉编码器：把像素整理成局部特征

Qwen3-VL 采用 SigLIP-2 架构，并继续用动态分辨率数据训练，报告把得到的视觉骨干称为 Qwen3-ViT。默认使用 SigLIP2-SO-400M；2B 和 4B 小模型使用 SigLIP2-Large 300M。

“动态分辨率”表示图片不必全部缩成同一个正方形。长图会产生更宽的特征网格，大图通常会产生更多视觉 token。报告架构图给出的三个例子分别是：

- 极长页面：11,427 个视觉 token；
- 很小的横幅图：8 个视觉 token；
- 普通横图：1,125 个视觉 token。

这保住了小字和版面细节，但代价也很直接：视觉 token 越多，LLM 的上下文越长，注意力计算和 KV cache 越贵。

视觉编码器同时使用 2D-RoPE 和按输入尺寸插值的绝对位置 embedding。前者告诉 attention 两块特征相隔几行几列；后者把预训练时学到的位置表调整到新的网格尺寸。

### Merger：既降 token 数，也改向量维度

这一步要完成两件事：先把视觉 token 数降下来，再把每个 token 的宽度改成语言模型能接收的宽度。先约定：

- `D_v`：视觉编码器输出的每个格子有多少个数；
- `D_l`：Qwen3 每个文字 token 的隐藏宽度；
- `v_{ij}`：视觉网格第 `i` 行、第 `j` 列的 `D_v` 维向量。

假设视觉编码器输出 `12×16=192` 个特征。merger 每次合并一个 `2×2` 小方块：

```math
12\times16\quad\xrightarrow{\;2\times2\text{ merge}\;}\quad6\times8=48\ \text{个视觉 token}
```

具体做法是把 4 个 `D_v` 维向量拼成一个 `4D_v` 维向量，再通过两层 MLP 映射到语言模型宽度 `D_l`：

```math
[v_{00};v_{01};v_{10};v_{11}]\in\mathbb R^{4D_v}
\quad\xrightarrow{\mathrm{MLP}}\quad
z\in\mathbb R^{D_l}
```

- 方括号加分号表示“把四个向量首尾拼起来”，不是相加。
- 192 变 48，token 数减少 75%；但每个新 token 汇总了原来四格的信息。

这就是 [[modality-projector]] 在 Qwen3-VL 里的具体形态：不只换维度，还顺便做空间合并。

### Qwen3 LLM：把视觉 token 当作上下文继续写答案

merger 输出已经和文字 embedding 一样宽，因此可以直接插入文字序列。LLM 的 self-attention 能让问题 token 读取视觉 token，也能让后续答案读取前面的图文上下文。

模型有四个 dense 版本 `2B / 4B / 8B / 32B`，以及两个 [[moe]] 版本：

- `30B-A3B`：总参数约 30B，每个 token 实际激活约 3B 参数；
- `235B-A22B`：总参数约 235B，每个 token 实际激活约 22B 参数。

`A3B / A22B` 不是显存占用，也不是模型压缩后的大小；它只说明每个 token 经过路由后，本轮参与计算的专家参数规模。完整权重仍需存储。

## 2. DeepStack：不让最后一层视觉特征包办一切

普通 VLM 常把 ViT 最后一层输出交给 LLM。最后一层偏向已经整理好的高级语义，但小字、边缘和局部纹理可能在逐层抽象时被弱化。

Qwen3-VL 从视觉编码器三个不同深度取特征，每一组经过独立 merger，随后加到 LLM 前三层对应视觉位置的 hidden state 上：

- `l∈{1,2,3}`：三处注入位置；
- `f_l^{ViT}`：第 `l` 处抽出的视觉中间特征；
- `m_l`：该位置专用的 merger，负责把 token 数和向量宽度对齐；
- `h_l^{LLM}`：LLM 第 `l` 层原本的 hidden state；
- `h_l^{new}`：视觉补充写入后的新 hidden state。

```math
h_l^{\text{new}}=h_l^{\text{LLM}}+m_l\!\left(f_l^{\text{ViT}}\right),\qquad l=1,2,3
```

为什么是加法：两边已经被 `m_l` 对齐成同样的形状，逐元素相加能把视觉补充写进原位置，而不新增 token。若改成沿序列拼接，上下文会变长；若沿隐藏维拼接，后续层的输入宽度也要跟着改。

一个两维教学例：若某个视觉 token 在 LLM 第 1 层是 `[1.0, 2.0]`，对应 DeepStack 特征是 `[0.2, -0.5]`，注入后就是：

```text
[1.0, 2.0] + [0.2, -0.5] = [1.2, 1.5]
```

如果采用拼接，序列会增加 token 或隐藏维度；DeepStack 选择相加，因此上下文长度不增加。代价是多跑三套轻量 merger，并在前三层多做几次加法。

消融使用内部 15B-A2B LLM、200B 预训练 token，且没有后训练：平均分 `74.7→76.0`，InfoVQA `71.9→74.2`，DocVQA `89.5→91.1`；TVQA 则从 `80.6` 微降到 `80.5`。因此可以说它整体有效，不能说每项任务都提升。

## 3. Interleaved-MRoPE：让时间、行、列都拿到快慢不同的频率

[[mrope]] 给每个视觉 token 三个位置数：时间 `t`、行 `h`、列 `w`。RoPE 会按位置旋转 query 和 key；位置差最后会影响 attention 分数。

频率有快慢之分：慢频率转得慢，适合区分相隔很远的位置；快频率对小位移更敏感，适合辨别近处细节。原版 MRoPE 把旋转维度连续切成三段，分别交给 `t/h/w`。这样每个轴只拿到一段频率，频谱不均。

用 12 对旋转维度做教学演示：

```text
旧：t t t t | h h h h | w w w w
新：t h w | t h w | t h w | t h w
    慢频率 ─────────────────→ 快频率
```

交错后，三个轴都同时拿到慢、中、快频率。它没有增加维度，也没有增加 attention 次数，只改变“哪一对维度使用哪个坐标”。

拿行坐标 `h=2` 看一次真实旋转。RoPE 会把一对二维数 `[1,0]` 按角度 `hω` 旋转，其中 `ω` 是这一对维度的频率：

```math
\operatorname{RoPE}([1,0],h,\omega)
=[\cos(h\omega),\sin(h\omega)].
```

若行轴拿到一个慢频率 `ω=.1` 和一个快频率 `ω=1`：

```text
慢频率：角度 = 2×.1 = .2 rad → [cos .2, sin .2] ≈ [.980, .199]
快频率：角度 = 2×1  = 2 rad  → [cos 2,  sin 2 ] ≈ [−.416, .909]
```

同一个“向下两行”，一组维度变化很小，保留粗粒度远程关系；另一组变化很大，分辨近距离位移。Interleaved-MRoPE 的意义，就是不让 `t/h/w` 中某一根轴只能使用频谱的一小段。

报告把长视频能力提升归因于 Interleaved-MRoPE、文字时间戳和视频数据共同作用，但没有给 Interleaved-MRoPE 单独开关的消融。因此不能从报告中拆出它独自贡献了多少分。

## 4. 文字时间戳：把真实时间从位置编号改成模型能读的内容

Qwen2.5-VL 的 T-RoPE 把绝对时间绑定到时间位置 ID。长视频会得到数值大、间隔稀疏的位置编号；为了让模型适应不同帧率，还需要构造覆盖各种 fps 的训练数据。

Qwen3-VL 改成在每组视频帧前写一段文本：

```text
<0.0 seconds> [第 1 组帧]
<3.0 seconds> [第 2 组帧]
<00:01:12>    [第 3 组帧]
```

模型不必从一个巨大位置 ID 猜真实时间，而是像读字幕一样直接读“3.0 秒”。训练同时使用秒和 `时:分:秒` 两种格式。

这个改法有三个边界：

1. 时间戳会占少量额外上下文；
2. 它只说明采样帧的时间，不能恢复采样时漏掉的事件；
3. 它解决时间表示，不替代帧间动作理解，后者仍靠视频数据和 Transformer 学习。

## 5. 四阶段预训练：先接通，再逐步拉长上下文

| 阶段 | 更新哪些参数 | 训练 token 预算 | 序列长度 | 主要目的 |
|---|---:|---:|---:|---|
| S0 | 只训练 merger；ViT 与 LLM 冻结 | 67B | 8,192 | 先把视觉特征接入语言空间 |
| S1 | ViT + merger + LLM 全部更新 | 约 1T | 8,192 | 大规模图文联合预训练 |
| S2 | 全部更新 | 约 1T | 32,768 | 增加长文本、视频和 agent 数据 |
| S3 | 全部更新 | 100B | 262,144 | 专门适配长视频和长文档 |

这里的 token 预算是训练过程中消费的 token 总量，不等于去重后的独立数据量。

从训练成本上看，逐步拉长上下文很合理：self-attention 的 token 配对数随长度平方增长。把 8K 直接换成 32K，序列长度扩大 4 倍，朴素 attention 配对数会放大约 16 倍。报告没有把“为什么这样排阶段”写成公式，这个成本解释属于基于 Transformer 计算规律的推断。

报告还说训练目标从 per-sample loss 改为“平方根归一的 per-token loss”，目的是平衡纯文本与多模态样本。报告没有给精确公式、归一长度和 packing 细节，因此不能自行写出唯一实现；页面只保留作者明确给出的目的，不伪造公式。

## 6. 预训练数据：能力不是只靠三处结构升级

报告用了九大类数据管线：

| 数据类别 | 它在教什么 | 报告披露的代表细节 |
|---|---|---|
| 图文 caption / 交错文档 | 看图描述、跨页图文关联 | 专用 Qwen2.5-VL-32B 重写 caption；书籍页可合并到 256K |
| 视觉知识 | 识别长尾实体及属性 | 十余类实体；按重要度采样并重写稀疏说明 |
| OCR / 文档 | 读文字、还原版面、跨页问答 | 30M OCR 样本；3M Common Crawl PDF + 4M 内部文档；扩到 39 种语言 |
| grounding / counting | 输出框、点和数量 | 框、点、计数三类任务；二维坐标统一缩放到 `[0,1000]` |
| 空间 / 3D | 相对位置、可操作性、三维框 | 3D 框采用 9-DoF JSON，并统一到虚拟相机坐标系 |
| 视觉代码 | 截图转 HTML、图转 SVG、图表转代码 | 同时混入 Qwen3 / Qwen3-Coder 纯文本代码数据 |
| 视频 | 事件顺序、时间定位、长视频总结 | 短片段 caption 合成长故事；按阶段调整 fps 与最大帧数 |
| STEM | 几何感知和视觉推理 | 1M 点定位、2M 感知 VQA、6M 图注、60M 练习、12M 长 CoT |
| agent | GUI 操作、函数调用、搜索 | 桌面/手机/网页轨迹；多轮函数调用和图文搜索轨迹 |

二维坐标归一化可以手算：若原图宽 2,000 像素，目标点横坐标是 500，那么输出坐标是 `500÷2000×1000=250`。换成宽 4,000 的图，只要目标仍在四分之一处，归一化坐标仍是 250。这样下游不必先猜原图尺寸。

大量数据由旧模型、检测器、规则和伪标签生成，再经过规则或模型过滤。报告披露了流程和部分规模，但没有公开完整数据清单与配比；复现者无法只靠报告重建训练集。

## 7. 后训练：SFT、蒸馏和 RL 各补不同短板

### SFT：先学会按指令回答

SFT 约 120 万条样本，其中三分之一纯文本，三分之二图文或视频。先用 32K 上下文训练一轮，再用混合 32K 数据和 256K 长文档、长视频训练第二轮。

Instruct 版使用直接回答格式；Thinking 版使用显式长 CoT。长 CoT 冷启动数据里，文字与多模态约为 `1:1`。一个重要过滤规则是：若 Qwen3-30B-nothink 不看图也能答对一道视觉数学题，就删掉它，防止模型靠题面文字“蒙对”而没有真正读图。

### 强到弱蒸馏：先学老师答案，再学老师概率分布

蒸馏分两段：

1. **off-policy**：强老师先生成答案，小模型直接学习这些外部答案；
2. **on-policy**：学生自己生成回答，再让老师对学生走过的序列给出每个 token 的概率分布，用 KL divergence 拉近两者。

第一段覆盖老师擅长的解题路径；第二段专门训练学生自己真正会走到的状态。这里没有换一种神秘损失，只是**先由学生决定当前上下文来自哪里**，再在这个上下文上计算老师到学生的 KL：

```math
D_{\mathrm{KL}}(p_T\Vert p_S)
=\sum_i p_T(i)\log\frac{p_T(i)}{p_S(i)}.
```

- `i`：词表中的一个候选 token；
- `p_T(i)`：老师给该 token 的概率；
- `p_S(i)`：学生给该 token 的概率；
- `Σ_i`：把整个词表的贡献相加；
- 方向 `T||S`：用老师概率加权，惩罚学生没有覆盖老师高概率答案。

若老师给 `[.8,.2]`，学生给 `[.5,.5]`：

```text
KL = .8×ln(.8/.5) + .2×ln(.2/.5)
   = .8×ln(1.6)   + .2×ln(.4)
   ≈ .3760 − .1833
   ≈ .1927
```

训练会把学生的 `[.5,.5]` 往老师的 `[.8,.2]` 推。真实词表有数万项，并且回答中的每个位置都重复这项计算。on-policy 改变的是“在哪些学生自己会走到的前缀上算 KL”，不是 KL 公式本身。见 [[on-policy-vs-off-policy]]。

### RL：可验证题和开放题分开给反馈

Reasoning RL 覆盖数学、代码、逻辑、grounding 和视觉谜题。每题先采 16 个回答；全错的题在数据准备阶段被删，正式训练时又删掉通过率高于 90% 的简单题，留下“模型有机会学会、但还没完全会”的题。最终约 30K 条 RL query。训练使用 SAPO；报告只给算法名称与引用，没有在本报告中展开公式。

General RL 覆盖 VQA、caption、OCR、文档解析、grounding、时钟识别和指令遵循：

- 有明确答案的格式、计数、定位任务使用规则奖励；
- 开放回答由 Qwen2.5-VL-72B-Instruct 或 Qwen3 充当 judge；
- 对语言混杂、重复、格式错误等稀有坏习惯，另建专门提示集提高它们在训练中的出现频率。

## 8. Thinking with Images：模型可以边看边调用视觉工具

这部分不是“脑内凭空生成一张新图”，而是一个多轮视觉 agent：

```text
看原图 → 思考下一步需要什么 → 调用工具 / 环境动作
      → 读取返回的局部视觉证据 → 继续思考 → 最终回答
```

训练分两代：

1. 用约 10K grounding 样本把 Qwen2.5-VL-32B 训练成视觉 agent，再做多轮工具 RL；
2. 用这个 agent 蒸馏约 120K 多轮交互，随后给 Qwen3-VL 做冷启动 SFT 和工具 RL。

RL 同时使用最终答案正确性、多轮推理质量、工具调用次数三类奖励。早期模型学会了一个漏洞：无论题目多难都只调一次工具，因为这样可能骗过前两项奖励。团队于是加入“调用次数是否接近专家估计”的奖励，迫使模型按任务难度决定探索次数。

报告没有完整枚举可调用工具及其接口，因此不能把常见的裁剪、放大、搜索工具全部当成官方实现细节。

## 9. 基础设施：为什么需要五种并行

预训练基于 Megatron-LM，规模最高达到约 10,000 张 GPU。五种并行切的是不同东西：

- TP（Tensor Parallel）：把同一层的大矩阵乘切给多张卡；
- PP（Pipeline Parallel）：把连续 Transformer 层分到不同卡组，像流水线一样传激活；
- CP（Context Parallel）：把一条超长序列的 token 分给多张卡；
- EP（Expert Parallel）：把 MoE 的不同专家放到不同卡；
- ZeRO-1 DP：数据并行副本处理不同 batch，同时切分优化器状态。

256K 长序列首先压到 CP；235B MoE 的专家分布依赖 EP；单层矩阵仍太大时再叠 TP；层数沿 PP 分段。它们不是五个同义词。每多一种并行，也多一种通信和负载不均问题。见 [[distributed-training-parallelism]]。

推理与训练不是一套系统：报告使用 vLLM 或 [[sglang-inference]] 部署。vLLM 的 PagedAttention 主要管理不等长请求的 KV cache；SGLang 侧重结构化生成、复杂 prompt 和请求调度。

| 通用 VQA | MMBench-EN | 88.8 | 89.3 | Instruct 略强 |

| 视频 | MLVU avg | 83.8 | 84.3 | Instruct 略强 |
| GUI agent | OSWorld | 38.1 | 31.6 | Thinking 更强，但不是绝对高成功率 |

Thinking 并非所有任务都更好。需要长链推理的题通常受益，OCR、直接定位、部分视频问答反而常由 Instruct 版领先。

工具增强结果要单独看。表中 V* 的 `93.7+`、HRBench-4K 的 `85.4+`、HRBench-8K 的 `82.4+` 都带 `+`，表示推理时调用了工具，不能和纯模型一次前向混为一谈。

### 1M 上下文实验到底证明了什么

Needle-in-a-Haystack 把一帧关键证据插进长视频不同位置，再让模型找到并回答问题。报告在 1 FPS 下测试，并动态降低帧分辨率以保持视觉 token 预算：

- 30 分钟、256K 训练上下文内：100%；
- 约 2 小时、1M token 的 YaRN 外推：99.5%。

这证明模型在这个合成检索任务里能跨很长上下文找到显眼证据，不等于它能以原始清晰度完整理解两小时视频。帧率只有 1 FPS，分辨率还会随长度调整。

### 横向比较也有不完全公平之处

视频评测中 Qwen3-VL 最多使用 2,048 帧，而受 API 与资源限制，Gemini 2.5 Pro、GPT-5、Claude Opus 4.1 分别只输入 512、256、100 帧。作者明确写明无法保证完全公平。因此“超过闭源模型”必须连同不同帧预算一起读。

纯文本能力也不是全面超过 Qwen3 文本版。235B Instruct 在 AIME-25 上是 `74.7 vs 70.3`，但 MMLU-Pro 是 `81.8 vs 83.0`；Thinking 版 AIME-25 也低于对应 Qwen3 文本模型 `89.7 vs 92.3`。更准确的结论是：加入视觉后语言能力总体保住了，部分任务因新数据与后训练得到提升，但并非每项都反超。

## 11. 消融真正证明了什么

### Qwen3-ViT 不是所有指标都涨

与原始 SigLIP-2 相比，Qwen3-ViT 在 CLIP 阶段的内部 OmniBench 从 `36.9→45.5`，但 ImageNet-R 从 `96.1→95.7`、ImageNet-S 从 `76.2→74.5`。接入同一个 1.7B LLM、训练 1.5T token 后，RealWorldQA `58.7→66.1`，InfoVQA `65.3→67.0`。它更适合报告目标中的综合视觉语言任务，不代表所有传统分类指标无条件提高。

### DeepStack 有独立消融，另外两项没有

DeepStack 平均分 `74.7→76.0`，并在 InfoVQA、DocVQA 等细粒度文档任务明显提高，所以这项因果证据相对扎实。

报告没有给 Interleaved-MRoPE、文字时间戳、平方根归一损失的单独开关实验。长视频结果同时受结构、视频数据、采样方法和上下文扩展影响，不能把全部增益只归给某一个部件。

## 12. 局限与一句话带走

1. 训练数据大量来自内部收集、合成和伪标签，完整数据与配比未公开。
2. 平方根归一 per-token loss 没有精确公式，无法仅凭报告复现。
3. 1M 是借 YaRN 外推得到，不是原生训练长度；长视频测试还降低分辨率维持 token 预算。
4. 多项开放任务使用模型 judge；结果会继承 judge 的偏差。
5. 工具增强、Thinking 与纯 Instruct 是不同推理预算，横向比较时必须分开。
6. 动态高分辨率能保细节，也可能产生上万视觉 token，延迟与显存并不便宜。

一句话记住：**Qwen3-VL 的主线不是三个新名词，而是一套完整系统——动态分辨率 ViT 把图变成 token，merger 降采样并接入 Qwen3，DeepStack 补回中间视觉细节，位置与时间机制服务长视频，再靠大规模专门数据、蒸馏、RL 和分布式训练把能力真正做出来。**
