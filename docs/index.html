2026-07-01paperTurboQuant · 随机旋转把"任意向量"变成"分布已知",在线量化逼近理论最优在线向量量化(边来边压、看不到数据统计)老做不到最优失真。TurboQuant 一个几何洞察破局:**先随机旋转向量,它的每个坐标就服从一个固定已知的分布(集中的 Beta),而且高维下坐标之间近独立**([[random-rotation-quantization]])——于是可以**离线就设计好一个最优标量量化器、在线只旋转+套用**,不看数据。逼近信息论下界只差 ~2.7×。再补一手两阶段:MSE 量化器会让内积估计有偏,后接 1-bit [[quantized-jl]] 残差纠成无偏([[mse-vs-inner-product-bias]])。用在 [[kv-cache]] 压缩:3.5-bit 质量无损、2.5-bit 微降,≥6× 省内存、H100 上 attention 快 8×。 2026-07-01conceptMSE 最优 ≠ 内积无偏 · 压得最像,点积却系统性偏按"重建误差最小(MSE)"设计的量化器,把向量压得最像原样;但拿压完的向量去算内积(点积),结果会**系统性地偏**(通常偏小)——因为量化误差跟向量本身相关。两个目标不是一回事,得分开治。 2026-07-01conceptQJL · 1-bit 符号随机投影,给内积一个无偏估计Johnson-Lindenstrauss(JL)说随机投影能保住向量间的距离/内积;QJL 更狠——投影后**只留符号(1 bit)**,靠一个缩放系数,仍能给两个向量的内积一个**无偏**估计(平均意义上不偏不倚,只是有点方差),存储便宜到每维 1 bit。 2026-07-01concept随机旋转量化 · 把"不知道数据长啥样"变成"精确知道每个坐标的分布"在线量化的死结是"看不到数据统计,不知道该把量化格子切在哪";随机旋转一下向量,它每个坐标就服从一个**固定、已知**的分布(集中的 Beta),而且高维下坐标之间**近独立**——于是最优量化器能**离线预制好、在线只旋转+套用**,完全不看数据。 2026-06-30paperDiffusionNFT · 把扩散的 RL 从"反向采样"搬回"前向加噪"给扩散模型做在线 RL 的老路是把反向采样切成多步 MDP 再套 GRPO,但继承一身毛病(锁死一阶 SDE solver、前后向不一致、CFG 要两套模型)。DiffusionNFT 换思路:**一条前向加噪过程是唯一的、反向去噪有无数条**,那就把 RL 直接做在前向上——用奖励把样本切成正/负两堆([[negative-aware-finetuning]]),正负速度之差就是隐式的改进方向([[reinforcement-guidance]],本质是把 CFG 当成"离线版的奖励引导")。整个目标退化成一个**监督式 [[flow-matching]] 损失**:免似然、任意 solver、只要干净图不要轨迹,比 FlowGRPO 快 3–25×。 2026-06-30concept扩散 RL 的似然墙 · 为什么不能照搬 GRPO,以及绕反向的代价[[policy-gradient]]/[[ppo]]/[[grpo]] 都要"模型给这个样本的概率",LLM 逐 token 现成、扩散却算不出来(似然只能靠昂贵近似);主流绕法是把**反向去噪**切成多步马尔可夫过程让每步概率可算,但这条路锁死采样器、还可能让模型偏离前向过程。 2026-06-30concept负样本感知微调 · NFT · 用奖励切正负,在前向过程上把 RL 折成监督损失给每张生成图打个奖励 `r∈[0,1]`,按 `r` 把样本软切成正堆(高奖励)和负堆(低奖励);让一个模型同时学"正速度"和"负速度",正负之差自然指向"该往哪改"——于是扩散的在线 RL 退化成一个**加权的监督 [[flow-matching]] 损失**,不碰似然、不碰反向采样轨迹。 2026-06-30concept强化引导 · 正负样本之差就是"该往哪走",CFG 不过是它的离线版把高奖励样本的去噪方向减去低奖励样本的去噪方向,得到一个向量 `Δ`——它指向"怎么改能更高分",就是强化引导;而 [[classifier-free-guidance]] 的"条件−无条件"其实是同一种引导,只不过 CFG 那个方向是**离线固定**的,强化引导是**随奖励在线更新**的。 2026-06-29paperGRAPE · 最好的指令微调数据，是"合身"的那条同一条指令、好几个老师给的候选回答，别默认挑最强老师那条；让**目标模型自己**给每个候选打分（长度归一概率 = 困惑度），选它觉得**最自然/最合身**（困惑度最低）的那条来做 SFT。结果超过最强老师(405B)、超过 3× 大的数据集，还省算力。NeurIPS 2025 spotlight。 2026-06-29paperOmniEraser · 用视频帧白送的 ground-truth 教模型"连影子一起抹掉"对象消除的真痛点不是"把物体抠掉",而是**连它的影子/反光一起干净抹除、且别凭空补出个新东西**。OmniEraser 两手都解决:数据上用**视频帧白送真 ground-truth**([[video-frame-paired-supervision]])——物体在场的帧当输入、物体离开后的帧当目标,影子在后一帧里真的没了;模型上用**双条件输入**([[object-background-guidance]])把"要删的物体"也喂进去,让 [[flux-1]] DiT 明确知道哪块该清空,不再瞎补。 2026-06-29concept背景建模(MOG) · 固定机位下,谁老不变就是背景,谁突然变就是前景固定机位拍的视频里,背景的每个像素长期稳定、前景(走动的物体)经过时才突变;给每个像素建一个"它平时是什么颜色"的概率模型(高斯混合),某帧某像素偏离太多就判为前景——这样不用人工就能从视频自动挑出"有物体 / 没物体"的帧。 2026-06-29concept分布对齐的 SFT · 数据好不好，是相对你这个模型说的微调时别只挑"绝对最强"的回答，挑**贴合目标模型自己分布**的那条（它给高概率/低困惑度）——既学到任务，又少把模型从预训练分布上拽走。 2026-06-29concept双条件输入 · 不光告诉模型"周围长啥样",还告诉它"要删的是这个"普通 inpaint 只把**背景上下文**喂给扩散模型,模型不知道挖空的洞里"本该是空的",于是照周围猜着补出个新物体;OmniEraser 把**要删的物体本身**也作为一路条件一起喂进去,模型就明确知道"这块是要清掉的东西、那块是要保留的背景",不再瞎补。 2026-06-29concept对象效果消除 · 抹的不只是物体,还有它在世界上留下的痕迹真正的"删除一个物体"要连它的**影子、反光、倒影**一起抹掉,并把那块还原成合理的空背景;只抠掉物体本体、留下一摊孤零零的影子,看着比不删还假。
