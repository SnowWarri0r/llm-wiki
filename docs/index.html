2026-07-22topicPPO → GRPO → Dr.GRPO / DAPO → GSPO · LLM 强化学习算法地图这几种算法并不是五套互不相干的训练方法：它们都在做“采样回答 → 打分 → 提高好回答概率、降低坏回答概率”，主要分歧只有三处——**用什么估计 advantage、按回答还是按 token 汇总 loss、用 token 级还是序列级的重要性比率做 clip**。 2026-07-22conceptClassifier-Free Guidance · CFG让模型同一步算两遍 —— 一遍看着 prompt、一遍蒙着眼，再把"看着 prompt 多走的那部分"放大，逼它更听话。 2026-07-22conceptDAPO · Decoupled Clip and Dynamic Sampling Policy OptimizationDAPO 是面向长 CoT 在线 RL 的 GRPO 系统配方：放宽上侧 clip 保探索、动态过滤零梯度题、按有效 token 汇总 loss，并把超长截断的硬惩罚改成平滑信号。 2026-07-22conceptDiffusion Transformer · DiT把扩散模型里那个老 U-Net 换成 Transformer —— 图像切成 latent token，像处理一串 token 一样去噪。 2026-07-22conceptDMD 蒸馏 · NFE · 把 40 步压成 4 步把多步扩散老师蒸成少步学生，目标不是抄老师的轨迹，而是让学生的**出图分布**匹配老师的分布；NFE（跑几次网络）从几十降到几步 → 近实时生成。 2026-07-22conceptDr.GRPO · Group Relative Policy Optimization Done RightDr.GRPO 保留“同题多回答、减组均值”的无 critic 优势估计，但去掉**除组内标准差**和**除每条回答实际长度**，避免原始 GRPO 暗中重加权题目与回答。 2026-07-22conceptFlow Matching · 流匹配不直接预测音频样本，而是学一个**速度场**，从噪声出发沿场积分若干步到目标音频。 2026-07-22conceptGRPO · Group Relative Policy OptimizationGRPO 保留 PPO 的“新旧概率比率 + clip”，但不再训练 critic；它对同一道题采样一组回答，用各回答相对组均值的得分作为 advantage。 2026-07-22conceptGSPO · Group Sequence Policy OptimizationGSPO 沿用 GRPO 的组相对 advantage，但让整条回答共用一个长度归一的新旧策略比率，并在序列级做 clip；它修的是更新粒度，不是 reward 的来源。 2026-07-22concept对数 · 把乘除变成加减对数不是什么新运算，它只是把指数倒过来问：\(\log_b x=y\) 的意思就是“\(b\) 的 \(y\) 次方等于 \(x\)”；正因为指数相乘时幂次相加，对数才会把里面的乘除变成外面的加减。 2026-07-22concept数学符号速查ML paper 里反复出现的数学符号 + 它们对应的"工程意思"。看到不认识的就翻这页。 2026-07-22conceptPerceptual Loss · LPIPS / DINO · 在"特征空间"里比两张图像不像不逐像素比两张图，而是把它们都喂进一个**预训练的"懂图"网络**、比深层激活的距离 —— 这比"看不看得像"更接近人眼。**LPIPS** 和 **DINO loss** 就是用两种不同网络特征来比。 2026-07-22conceptQK-Norm · 给 Q/K 做归一化稳住训练在算 attention 分数之前，先对 Query 和 Key 各做一次 RMSNorm —— **防止点积越训越大、attention 塌成 one-hot 而炸训练**。 2026-07-20paperHDR · 先规划整段轨迹，再逐层生成每一步HDR 不让流式视频模型看到第一步就立刻押死整条路线，而是先生成一棵从粗到细的时间层级：上层用少量、仍带噪的 latent 表示可修改的整体方案，下层再把方案细化成连续画面；SHAP 只连接同层前一步、父节点和相邻父节点，把时间注意力从平方级降到线性级。
