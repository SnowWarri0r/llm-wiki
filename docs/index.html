2026-07-22topicPPO → GRPO → Dr.GRPO / DAPO → GSPO · LLM 强化学习算法地图这几种算法并不是五套互不相干的训练方法：它们都在做“采样回答 → 打分 → 提高好回答概率、降低坏回答概率”，主要分歧只有三处——**用什么估计 advantage、按回答还是按 token 汇总 loss、用 token 级还是序列级的重要性比率做 clip**。 2026-07-22conceptDAPO · Decoupled Clip and Dynamic Sampling Policy OptimizationDAPO 是面向长 CoT 在线 RL 的 GRPO 系统配方：放宽上侧 clip 保探索、动态过滤零梯度题、按有效 token 汇总 loss，并把超长截断的硬惩罚改成平滑信号。 2026-07-22conceptDr.GRPO · Group Relative Policy Optimization Done RightDr.GRPO 保留“同题多回答、减组均值”的无 critic 优势估计，但去掉**除组内标准差**和**除每条回答实际长度**，避免原始 GRPO 暗中重加权题目与回答。 2026-07-22conceptGRPO · Group Relative Policy OptimizationGRPO 保留 PPO 的“新旧概率比率 + clip”，但不再训练 critic；它对同一道题采样一组回答，用各回答相对组均值的得分作为 advantage。 2026-07-22conceptGSPO · Group Sequence Policy OptimizationGSPO 沿用 GRPO 的组相对 advantage，但让整条回答共用一个长度归一的新旧策略比率，并在序列级做 clip；它修的是更新粒度，不是 reward 的来源。 2026-07-22concept对数 · 把乘除变成加减对数不是什么新运算，它只是把指数倒过来问：\(\log_b x=y\) 的意思就是“\(b\) 的 \(y\) 次方等于 \(x\)”；正因为指数相乘时幂次相加，对数才会把里面的乘除变成外面的加减。 2026-07-22concept数学符号速查ML paper 里反复出现的数学符号 + 它们对应的"工程意思"。看到不认识的就翻这页。 2026-07-20paperHDR · 先规划整段轨迹，再逐层生成每一步HDR 不让流式视频模型看到第一步就立刻押死整条路线，而是先生成一棵从粗到细的时间层级：上层用少量、仍带噪的 latent 表示可修改的整体方案，下层再把方案细化成连续画面；SHAP 只连接同层前一步、父节点和相邻父节点，把时间注意力从平方级降到线性级。 2026-07-20concept流式 AR vs 双向视频扩散双向扩散让整段视频在每个去噪 step 互相修改，推理强但代价高；流式 AR 从左到右提交，延迟低但早期错误无法撤回。 2026-07-20conceptConditional Flow Matching · 条件流匹配把难算的全局 velocity field 训练目标，换成"给定一对噪声和数据样本时该往哪走"的可监督回归。 2026-07-20conceptHierarchical Latent Denoising · 层级 latent 去噪先用少量粗时间节点决定整段轨迹，再用更多细节点逐层落实；粗层故意不完全去噪，避免太早把一种方案锁死。 2026-07-20conceptKV CacheTransformer 每层 attention 算过的 Key / Value 矩阵存下来，下一个 token 直接复用，不重算前面所有 token。 2026-07-20conceptSHAP · 稀疏层级注意力树中每个节点只读同层前一个节点、自己的父节点、父节点左右邻居和层首参考帧，用固定数量的连接传递长程规划。 2026-07-17paperLTX-2 · 让声音和画面在同一段去噪过程中彼此校正**视频负责空间和运动，音频负责时间和声场；两条流各自去噪，又在每一层沿共同时间轴双向对答案。**
