2026-06-30paperDiffusionNFT · 把扩散的 RL 从"反向采样"搬回"前向加噪"给扩散模型做在线 RL 的老路是把反向采样切成多步 MDP 再套 GRPO,但继承一身毛病(锁死一阶 SDE solver、前后向不一致、CFG 要两套模型)。DiffusionNFT 换思路:**一条前向加噪过程是唯一的、反向去噪有无数条**,那就把 RL 直接做在前向上——用奖励把样本切成正/负两堆([[negative-aware-finetuning]]),正负速度之差就是隐式的改进方向([[reinforcement-guidance]],本质是把 CFG 当成"离线版的奖励引导")。整个目标退化成一个**监督式 [[flow-matching]] 损失**:免似然、任意 solver、只要干净图不要轨迹,比 FlowGRPO 快 3–25×。 2026-06-30concept扩散 RL 的似然墙 · 为什么不能照搬 GRPO,以及绕反向的代价[[policy-gradient]]/[[ppo]]/[[grpo]] 都要"模型给这个样本的概率",LLM 逐 token 现成、扩散却算不出来(似然只能靠昂贵近似);主流绕法是把**反向去噪**切成多步马尔可夫过程让每步概率可算,但这条路锁死采样器、还可能让模型偏离前向过程。 2026-06-30concept负样本感知微调 · NFT · 用奖励切正负,在前向过程上把 RL 折成监督损失给每张生成图打个奖励 `r∈[0,1]`,按 `r` 把样本软切成正堆(高奖励)和负堆(低奖励);让一个模型同时学"正速度"和"负速度",正负之差自然指向"该往哪改"——于是扩散的在线 RL 退化成一个**加权的监督 [[flow-matching]] 损失**,不碰似然、不碰反向采样轨迹。 2026-06-30concept强化引导 · 正负样本之差就是"该往哪走",CFG 不过是它的离线版把高奖励样本的去噪方向减去低奖励样本的去噪方向,得到一个向量 `Δ`——它指向"怎么改能更高分",就是强化引导;而 [[classifier-free-guidance]] 的"条件−无条件"其实是同一种引导,只不过 CFG 那个方向是**离线固定**的,强化引导是**随奖励在线更新**的。 2026-06-29paperGRAPE · 最好的指令微调数据，是"合身"的那条同一条指令、好几个老师给的候选回答，别默认挑最强老师那条；让**目标模型自己**给每个候选打分（长度归一概率 = 困惑度），选它觉得**最自然/最合身**（困惑度最低）的那条来做 SFT。结果超过最强老师(405B)、超过 3× 大的数据集，还省算力。NeurIPS 2025 spotlight。 2026-06-29paperOmniEraser · 用视频帧白送的 ground-truth 教模型"连影子一起抹掉"对象消除的真痛点不是"把物体抠掉",而是**连它的影子/反光一起干净抹除、且别凭空补出个新东西**。OmniEraser 两手都解决:数据上用**视频帧白送真 ground-truth**([[video-frame-paired-supervision]])——物体在场的帧当输入、物体离开后的帧当目标,影子在后一帧里真的没了;模型上用**双条件输入**([[object-background-guidance]])把"要删的物体"也喂进去,让 [[flux-1]] DiT 明确知道哪块该清空,不再瞎补。 2026-06-29concept背景建模(MOG) · 固定机位下,谁老不变就是背景,谁突然变就是前景固定机位拍的视频里,背景的每个像素长期稳定、前景(走动的物体)经过时才突变;给每个像素建一个"它平时是什么颜色"的概率模型(高斯混合),某帧某像素偏离太多就判为前景——这样不用人工就能从视频自动挑出"有物体 / 没物体"的帧。 2026-06-29concept分布对齐的 SFT · 数据好不好，是相对你这个模型说的微调时别只挑"绝对最强"的回答，挑**贴合目标模型自己分布**的那条（它给高概率/低困惑度）——既学到任务，又少把模型从预训练分布上拽走。 2026-06-29concept双条件输入 · 不光告诉模型"周围长啥样",还告诉它"要删的是这个"普通 inpaint 只把**背景上下文**喂给扩散模型,模型不知道挖空的洞里"本该是空的",于是照周围猜着补出个新物体;OmniEraser 把**要删的物体本身**也作为一路条件一起喂进去,模型就明确知道"这块是要清掉的东西、那块是要保留的背景",不再瞎补。 2026-06-29concept对象效果消除 · 抹的不只是物体,还有它在世界上留下的痕迹真正的"删除一个物体"要连它的**影子、反光、倒影**一起抹掉,并把那块还原成合理的空背景;只抠掉物体本体、留下一摊孤零零的影子,看着比不删还假。 2026-06-29concepton-policy vs off-policy · 学自己刚做的，还是学别人/旧的on-policy = 拿**当前策略自己刚生成**的数据学（信号对得准、稳，但得不断现采）；off-policy = 拿**别的策略**（旧的自己/别的模型/人类）的数据学（能复用、便宜，但分布错位、要纠偏）。 2026-06-29concept并行 Transformer 块 · attention 和 MLP 不接力,同时算再相加普通 Transformer 块是"先注意力、再前馈"两路接力;并行块让两路**读同一份归一化输入、各算各的、最后一起加回主干**,省一次归一化、还能把两路的第一个大矩阵乘拼成一个 GEMM 一次算完——快 ~15%,表达力几乎不掉。GPT-J / PaLM / ViT-22B / [[flux-1]] 单流块都用它。 2026-06-29concept困惑度 Perplexity · 模型对一段话有多"意外"把模型在一段文本上的平均交叉熵取指数，得到一个"等效分支数"：困惑度 = exp(平均每 token 的负对数概率)；越低 = 模型觉得这段话越自然、越在意料之中。 2026-06-29concept视频帧白送的成对监督 · 物体自己会走,等于免费拍了 before/after要教模型"把物体连影子一起抹掉",最难的是拿到 ground-truth(抹干净后的真实画面);视频里物体本来就会移动,**有物体的那一帧当输入、物体离开后的那一帧当目标**,影子/反光在后一帧里真的没了——监督信号白送,不用手工修图。
