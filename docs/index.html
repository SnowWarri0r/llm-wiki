2026-06-28concept分层图像生成 · 产出一摞可编辑图层，不是一张拍平图生成的不是糊在一个平面上的像素图，而是一摞带透明通道（RGBA）的图层（画布+背景+K 个前景），每层能单独挪/改/重组——图像版的"逐元素编辑"。 2026-06-28concept选择性 token masking · 用"哪些给定/哪些待生成"当任务开关同一个扩散模型，靠"哪些 token 从干净 latent（给定、不预测）初始化、哪些从噪声（待生成）初始化"这个开关，把生成/分解/编辑统一成一个模型——换 mask = 换任务，权重不动。 2026-06-26conceptDINOv3 多样性奖励 · 给"组内整齐划一"扣钱，防风格塌缩一条 prompt 生成一组图，用 DINOv3 把每张压成向量，算组内两两的平均距离当"多样性分"，加进奖励——逼模型每次给的几张互不雷同。 2026-06-26conceptFAISS / 近似最近邻 · 用一点精度换十亿级的速度精确最近邻要拿查询跟全部向量逐个比，十亿级太慢；FAISS 用"先分桶只翻几桶"(IVF)+"把向量压成几字节"(PQ)两招，换来毫秒级近似检索。 2026-06-26concept层级 k-means 策展 · 把"逐张审查"压成"逐簇审查"十亿张图没法一张张看；先用 k-means 把它们按视觉语义聚成簇、再递归切子簇，每个簇心附近挑几张代表图让 VLM 审一眼，按簇整批保留/丢弃/补采。 2026-06-26conceptPageRank 实体覆盖 · 用链接结构挑出"必须见过的概念"拿维基百科的链接图跑 PageRank，给每个条目算个"重要度"，取排名前 90% 当"必须覆盖的概念"（约 500 万个），再回头查训练集有没有图文对覆盖到它们。 2026-06-26conceptPrompt Expansion · 把短输入扩成富 caption（还得防塌成一种风格）模型训在信息密集的长 caption 上，用户却爱打短句；前面挂一个会写字的 LLM 把短输入扩成富 caption，用 SFT 学会扩写、再用 GDPO + 多奖励把扩写调到"提质又不跑题、还别塌成一种风格"。 2026-06-26conceptSigLIP 语义去重 · 删的是"长得像"，不是"字节一样"把每张图用 SigLIP 压成一个向量，向量方向接近（余弦相似度高过阈值）就当近重复，每簇只留一张。 2026-06-26conceptStyle Reference · 只借参考图的"风格"，别把它的"内容"也搬来用文字生成、再拿一/多张参考图导风格；核心难题是别让参考图的主体/物件钻进结果（内容泄漏），靠自监督训练 + 偏好优化把"风格"和"内容"两条通道拆开。 2026-06-25paperKrea 2 · 不追一种"标准好看",撑开整个审美空间**主流模型都在卷"单一默认审美"并卷到饱和;Krea 2 反过来——数据上拒绝按美学分过滤、拒绝 AI 合成图,训练上用六段流水线(预训→中训→SFT→偏好优化→多奖励 RL→蒸馏)把审美空间撑宽,还自创 STPO(稳住 DPO 的偏好优化)和 prompt 级 rubric 奖励,目标是"多样且可控",不是"一种好看"。** 2026-06-25paperQwen-Image-Bench · 文生图评测从"生成"升到"创造"**老基准只问"画得对不对",这题已满分;Qwen-Image-Bench 加两根新支柱——真实世界保真 + 创意表达——把模糊的"好不好"拆成 56 条可逐项判定的细则,再训一个跟人类专家 ρ=0.92 对齐的判官自动打分。** 2026-06-25conceptDPO · 直接偏好优化（跳过奖励模型对齐人类口味）不训奖励模型、不跑 PPO，直接拿"赢/输"成对数据当分类题优化模型：把赢的样本概率顶上去、输的压下去，一步到位对齐人类偏好。 2026-06-25concept生成模型的数据策展 · 别在数据端先把多样性筛没了训文生图模型时，用"美学打分器"过滤数据会把模型口味偷偷锁死；与其追"每张都漂亮"，不如守住"caption 准确"，并且**一张 AI 合成图都不掺**。 2026-06-25conceptLLM-as-Judge · 拿模型当判官用一个模型给别的模型的输出自动打分/排名，替掉又贵又慢的人工评测——前提是先把这个判官校准到跟人类口味一致。
