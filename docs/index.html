Adam · 自适应矩估计优化器

给梯度记两个 EMA——一阶动量 m（记方向、抹抖）+ 二阶 v（梯度平方的 EMA，记大小），更新量 = lr·m̂/(√v̂+ε)，每个参数都拿到自己的自适应步长。动量 + RMSprop 合体，几乎不用调参，Transformer/LLM 默认优化器。

缠论 · 动力学篇 · 背驰与区间套

形态学（分型→笔→线段→中枢）只告诉你走势"长什么样"，动力学回答"还有没有劲、什么时候转"。核心是**背驰**（后一段力度比前一段弱 = 趋势要尽），力度用 **MACD 面积**量；再用**区间套**（大级别背驰里套小级别背驰）把买卖点精确到一个点。一条定理把两者钉死：**任何背驰必制造某级别买卖点，任何买卖点必源自某级别背驰**。

缠论 · 操作篇 · 走势分解与同级别分解

缠论操作的地基是一条极简定理：**任何级别的走势，要么是趋势，要么是盘整**（走势类型定理）。把走势在**同一个级别**上拆成"盘整 + 趋势"的接龙（同级别分解），一个走势类型走完就是一个操作点。难点是**走势的多义性**（同一张图能有多种合法分解），缠论用"走势终完美 + 只做当下"来化解。

§ 04Concepts · 概念

Actor-Critic · 一个动手, 一个打分

一个网络管"做什么"(actor), 一个管"这局面值多少"(critic), critic 算出的"实际比预期好多少"反过来教 actor —— PPO 就跑在这套结构上。

MLconceptupdated 2026-06-09auto HTML ✓

Advantage Function · 跟基线比,不是看绝对值

"这一步赚了多少"比"这一步总共拿了多少"信号更稳定 —— **优势函数 = 实际回报 − 预期回报**, 让训练不会因 reward 整体偏移就乱掉。

MLconceptupdated 2026-05-28auto HTML ✓

Agent Reflection · 反思 · 从碎片到认知

定期把大量低层观察总结成高层认知 —— "我看到 A/B/C/D → 我觉得 X"。反思结果存回 [[memory-stream]]，下次检索时跟原始观察一起参与排序。

MLconceptupdated 2026-05-27auto HTML ✓

Batch-Invariant Kernel · 批大小不变算子

同一条样本不管跟谁凑 batch，kernel 都尽量按同一顺序做浮点运算，结果保持 bit-for-bit 一致。

MLconceptupdated 2026-05-22auto HTML ✓

Batch Normalization · BN

对一个 mini-batch 在每个 channel 内做归一化（减均值/除方差），让中间层的分布**不要剧烈漂移**，间接稳住训练。

MLconceptupdated 2026-05-21auto HTML ✓

Bin Quantization · 等距分箱量化

把连续值（如某个频道的能量）切成 K 个等距 bin，每个值映射到它所在 bin 的索引 —— 1 行 numpy 代码搞定。

MLconceptupdated 2026-05-21auto HTML ✓

Bitter Lesson · 苦涩教训

长期看，能利用算力和数据持续 scale 的学习方法，会吃掉大量人手写规则。

MLconceptupdated 2026-05-28auto HTML ✓

Bitwise Determinism · 位级确定性

同样的输入，训练 kernel 和推理 kernel 跑出来的浮点数在**二进制位上完全相同**。

MLconceptupdated 2026-05-20auto HTML ✓

Bottleneck Block · 瓶颈残差块

用 `1×1 → 3×3 → 1×1` 先降维、再做空间卷积、再升维，让深层 ResNet 便宜很多。

MLconceptupdated 2026-05-22auto HTML ✓

Causal Language Model · CLM · 自回归语言模型

给定前 N 个 token，预测第 N+1 个 token。训练时每个位置都做这个预测 —— 每个 token 都贡献一份 loss。

MLconceptupdated 2026-05-21auto HTML ✓

Classifier-Free Guidance · CFG

让模型同一步算两遍 —— 一遍看着 prompt、一遍蒙着眼，再把"看着 prompt 多走的那部分"放大，逼它更听话。

MLconceptupdated 2026-06-04auto HTML ✓

Clipped Surrogate Objective · PPO 的核心一行 clip

不让新策略的概率比"老策略大太多 / 小太多" —— PPO 把这个约束**写进 loss 函数本身**, 一行 `torch.clamp` 干掉了 TRPO 那套复杂的 KL 约束 + 共轭梯度。

MLconceptupdated 2026-05-28auto HTML ✓

闭式 KL · 同协方差高斯 KL = 均值差²

"闭式"= 有个干净公式能直接代数算（对立面是撒随机点模拟着估）；两个**同协方差高斯**的 KL 散度就有闭式解，直接塌成两个均值差的平方（MSE）。

MLconceptupdated 2026-06-15auto HTML ✓

Conditional Flow Matching · 条件流匹配

把难算的全局 velocity field 训练目标，换成"给定一对噪声和数据样本时该往哪走"的可监督回归。

MLconceptupdated 2026-05-22auto HTML ✓

Continuity Equation · 连续性方程

"粒子不凭空冒、不凭空消"的数学化身。**密度变化 = 净流入** —— 仅此而已。

MLconceptupdated 2026-05-22auto HTML ✓

Contrastive Learning · 对比学习

不告诉模型"这张图是什么类别", 只告诉它"这张图配这句话, 不配那 N 句话" —— **靠正样本互相吸引、负样本互相推开**, 让模型自己学出语义对齐。

MLconceptupdated 2026-05-28auto HTML ✓

Convolution · 卷积 · 一把小尺子滑过整张图

拿一个小窗口（卷积核）滑过整张图，每停一处就把窗口盖住的像素跟核**逐元素相乘再求和**得到一个数 —— 滑遍全图得到一张新图（feature map）。

MLconceptupdated 2026-06-12auto HTML ✓

Cross-Attention · 交叉注意力

Decoder 用自己的 Query，去查 Encoder 输出的 Key/Value，让生成过程能看见输入序列。

MLconceptupdated 2026-05-22auto HTML ✓

Cross-Entropy · 交叉熵 · −log(你押对的概率)

模型对一堆候选输出一个概率分布，交叉熵只看一件事：**你给"正确答案"分了多少概率，取 −log**。押对的概率越高 loss 越低，越低 loss 暴涨。分类 / 预测下一个 token 的标准损失。

MLconceptupdated 2026-06-12auto HTML ✓

Decoder-Only Paradigm · 解码器路线

只用 Transformer 的 decoder 部分（带 causal mask 的 self-attention），目标是<strong>预测序列里每个位置的下一个 token</strong> —— 训练完后天然能做生成。

MLconceptupdated 2026-05-21auto HTML ✓

Degradation Problem · 深网络反而变差

深网络（比如 56 层）在**训练集**上的表现比浅网络（20 层）还差——不是过拟合，是 SGD 学不到"白送的好解"。

MLconceptupdated 2026-05-21auto HTML ✓

Diffusion Transformer · DiT

把扩散模型里那个老 U-Net 换成 Transformer —— 图像切成 latent token，像处理一串 token 一样去噪。

MLconceptupdated 2026-06-15auto HTML ✓

DMD 蒸馏 · NFE · 把 40 步压成 4 步

把多步扩散老师蒸成少步学生，目标不是抄老师的轨迹，而是让学生的**出图分布**匹配老师的分布；NFE（跑几次网络）从几十降到几步 → 近实时生成。

MLconceptupdated 2026-06-15auto HTML ✓

dMel · 离散 Mel 频谱

把 log-mel 频谱图按预设 bin 量化成离散 token，让音频能像文本一样直接喂进 transformer —— 不用训 codec。

MLconceptupdated 2026-05-21auto HTML ✓

Dual-AR · 双自回归

慢 AR 写主旋律（语义骨架），快 AR 写配器（声学细节），主从串成两层 AR。

MLconceptupdated 2026-05-20auto HTML ✓

Dual-Model Architecture · 双模型架构

**前台 always-on 低延迟模型** + **后台异步深推理模型**，共享 context，前台委托后台干慢活。

MLconceptupdated 2026-05-20auto HTML ✓

Dual-Tower Architecture · 双塔架构

两个独立的 encoder, 一个跑图、一个跑文本, **各跑各的最后把输出投影到同一个向量空间** —— 工程上两边可以独立放大缩小, 推理时一边可以预先算好缓存。

MLconceptupdated 2026-05-28auto HTML ✓

Early Fusion · 无独立编码器的早融合

各模态用**轻量预处理**直接进同一个 transformer，不用 Whisper / ViT 那种独立大编码器再拼。

MLconceptupdated 2026-05-20auto HTML ✓

Elastic Inference · 一个模型，多档算力

一次训练出一**族**模型，推理时按手头算力**临时挑一档**（快但糙 ↔ 慢但好），不必为每个预算单独训一个模型。

MLconceptupdated 2026-06-10auto HTML ✓

EMA · 指数滑动平均 · 一行就够

一行更新规则 `新 = m·旧 + (1−m)·新值`，一个数原地滚 —— 把抖动磨平、追出一条更稳的"趋势线"，不用存任何历史。

MLconceptupdated 2026-06-11auto HTML ✓

Emergent Abilities · 涌现能力 · 小模型 ≈ 0 大模型突然能

某些任务上小模型的表现接近随机，到了某个模型 size 阈值后<strong>突然</strong>跳到接近完美。这种"相变"叫 emergence。

MLconceptupdated 2026-05-21auto HTML ✓

Encoder-Only Paradigm · 编码器路线

只用 Transformer 的 encoder 部分（双向 self-attention，没有 causal mask），目标是<strong>产生表征</strong>而不是<strong>生成 token</strong>。

MLconceptupdated 2026-05-21auto HTML ✓

Entropy Regularization · 花钱买"别太早笃定"

在 loss 里加一笔"保持一点随机性就给奖励", 逼策略多探索一会儿、别一发现点甜头就锁死一招撞死。

MLconceptupdated 2026-06-09auto HTML ✓

Few-Shot Learning · 少样本学习

在 prompt 里给几个输入输出例子，让模型不改权重就照着模式完成新样本。

MLconceptupdated 2026-05-22auto HTML ✓

Flow Matching · 流匹配

不直接预测音频样本，而是学一个**速度场**，从噪声出发沿场积分若干步到目标音频。

MLconceptupdated 2026-06-04auto HTML ✓

Forced Alignment · 强制对齐 · 把已知文字钉到时间轴上

文字已经给你了（所以叫"强制"），只求每个字/词在音频里**什么时刻**出现（起止时间戳）——就是卡拉OK逐字高亮要算的那个"何时点亮"。

MLconceptupdated 2026-06-14auto HTML ✓

GAE · 把"超出预期多少"算得又稳又准

优势(实际 − 预期)有"看到底"和"只看一步"两种极端算法, 一个太抖一个太偏 —— GAE 用 λ 在两者之间插值, 取个甜点。

MLconceptupdated 2026-06-09auto HTML ✓

Grouped GEMM vs GEMV

MoE 推理里，小 batch 常像很多个 GEMV；把同 expert 的 token 凑起来可变成 grouped GEMM，吞吐更好但会增加调度等待。

MLconceptupdated 2026-05-22auto HTML ✓

GRPO · Group Relative Policy Optimization

PPO 的简化版：不用 value network，用**同 prompt 一组采样**之间的相对 reward 当 advantage。

MLconceptupdated 2026-05-31auto HTML ✓

GSPO · Group Sequence Policy Optimization · 把重要性比率提到序列级

GRPO 在 **token 级**算重要性比率（每个 token 一个 `r`）；GSPO 把比率提到**整条序列**——因为每个 token 只采一次、token 级比率是高方差噪声、容易把训练带崩，而奖励本来就是给整条序列的。

MLconceptupdated 2026-06-14auto HTML ✓

hMLP · Hierarchical MLP

TML 用来处理视频 patch 的轻量预处理器：先保留空间局部性，再把图像块压成 transformer 能吃的 token。

MLconceptupdated 2026-05-22auto HTML ✓

图像质量指标 · PSNR / SSIM · 重建得像不像

两把尺子量"重建图 vs 原图"差多少：PSNR 死磕逐像素误差（取对数变 dB，越高越好），SSIM 看局部亮度/对比度/结构（0~1，越接近 1 越像）。

MLconceptupdated 2026-06-15auto HTML ✓

In-Context Learning · ICL · 在 prompt 里现学

**模型权重一个不动**，在 prompt 里给几个 (input, output) 示例，模型就能从这些示例里"现学"任务规则，然后在新输入上应用。

MLconceptupdated 2026-05-21auto HTML ✓

Inductive Bias · 归纳偏置

模型架构里**预先写死的"对世界的假设"** —— 你越早写得对, 数据少时越省; 数据多时反而是包袱。

MLconceptupdated 2026-05-28auto HTML ✓

Inline Emotion Tags · 内联情绪标签

在输入文本里直接插入 `[whisper]`、`[laugh]` 这类标签，让 TTS 模型控制语气、情绪或副语言声音。

MLconceptupdated 2026-05-22auto HTML ✓

Input Transformations · 用输入格式喂任务

不为每个下游任务设计新的网络结构，而是<strong>把任务结构编码进输入文本</strong> —— 用分隔符、拼接顺序告诉模型"这是哪种任务"。

MLconceptupdated 2026-05-21auto HTML ✓

KL-VAE · 把图压进 latent 的那层地基

一个轻度正则的自编码器，把 512×512×3 的像素图压成 64×64×4 的小 latent —— **扩散模型不在像素上画画，在这个压缩空间里画，省 48 倍计算**。

MLconceptupdated 2026-06-15auto HTML ✓

K 线买点形态 · 抛压衰竭的几种长相

缩量下影线 / 箱体底缩量十字星 / 等量反包阳线 / 揉搓线 —— 都在回答同一个问题：**在这个震荡区间里，砸盘的还有没有力气了？** 它们是"抛压衰竭、有人接"的不同长相，是缠论横盘龙头低吸、爱在冰川低吸战法里的买点信号。

金融conceptupdated 2026-06-05auto HTML ✓

KV Cache

Transformer 每层 attention 算过的 Key / Value 矩阵存下来，下一个 token 直接复用，不重算前面所有 token。

MLconceptupdated 2026-05-20auto HTML ✓

Language Modeling as Multitask Learning · LM 即多任务学习

GPT-2 论文标题点破的核心 thesis：训练 LM 预测下一词的过程<strong>已经在学无数个具体任务</strong>，多任务能力是预训练的天然副产品。

MLconceptupdated 2026-05-21auto HTML ✓

LayerNorm · 层归一化

对每个 token 自己的 hidden 维度做归一化，不依赖 batch 里其他样本。

MLconceptupdated 2026-05-22auto HTML ✓

Log-Mel Spectrogram · Fbank · 对数 Mel 频谱图

把 1D 音频波形转成 2D `[频道 × 时间]` 的能量矩阵 —— 跟人耳听到的"哪个时刻、哪个频段、有多响"对应。ASR 圈管它叫 **Fbank**（log mel filterbank energies），同一个东西。

MLconceptupdated 2026-06-14auto HTML ✓

Looped Transformer · 把同一个 block 循环套用

不堆很多**不同**的层，而是把**同一个** transformer block 反复套 L 圈 —— 在深度上共享权重，把"算多深"和"用多少参数"解耦。

MLconceptupdated 2026-06-10auto HTML ✓

Markov Chain · 马尔可夫链 · 只看现在、不看历史

一串状态，下一步**只取决于你现在在哪、跟怎么走到这儿无关**（无记忆）；每步用一条"转移核"规则跳。扩散去噪就是它的高斯版。

MLconceptupdated 2026-06-12auto HTML ✓

Masked Language Model · MLM

随机把句子里 15% 的 token 换成 `[MASK]`，让模型用<strong>剩下所有位置</strong>（左右都行）的信息预测被遮住的词。

MLconceptupdated 2026-05-21auto HTML ✓

数学符号速查

ML paper 里反复出现的数学符号 + 它们对应的"工程意思"。看到不认识的就翻这页。

MLconceptupdated 2026-05-22auto HTML ✓

Memory Stream · 记忆流

Agent 看到的、听到的、做的所有事，按时间顺序存成一条流 —— 每条记忆带时间戳和重要性评分，需要的时候检索出来塞进 prompt。

MLconceptupdated 2026-05-27auto HTML ✓

Micro-Turn · 200ms 微回合

把时间切成 200ms 一片，每片里**输入和输出同时发生**，模型看到的是一条单一交错 token 流。

MLconceptupdated 2026-05-20auto HTML ✓

Modality Projector · 把别的模态翻译进 LLM 的隐空间

一个小 MLP，把**冻结的外部编码器**吐出的语音/图像特征，投影成 LLM 听得懂的 hidden 向量，塞进序列里的占位符位置 —— 让一个纯文本 LLM 不改主干就能"看图听音"。

MLconceptupdated 2026-06-14auto HTML ✓

MoE · Mixture of Experts

模型养着许多"专家"子网，每次推理一个 token 只激活其中少数几个 —— 总参大但计算路径小。

MLconceptupdated 2026-05-20auto HTML ✓

M-RoPE · 多模态旋转位置编码

把 RoPE 的"一个位置数字"升级成**三元组 (时间, 行, 列)** —— 让同一套 attention 既能数文字的先后，又能知道图像 patch 在画面的第几行第几列。

MLconceptupdated 2026-06-05auto HTML ✓

Multi-Head Attention · 多头注意力

把 attention 跑 **h 次**（论文 h=8），每次用不同的 Q/K/V 投影 → 每个 head 学不同的关系模式 → concat 后再线性。

MLconceptupdated 2026-05-20auto HTML ✓

Multi-Token Prediction · MTP · 一步预测好几个

**每个解码步一次吐多层 code**，而不是一次一个 —— minimind-o 的 Talker 用它配合"延迟错位"把 8 层 Mimi codebook 摊成阶梯并行，序列不膨胀，又不丢残差依赖。

MLconceptupdated 2026-06-08auto HTML ✓

Multitask Speech · 一个模型干多件事

Whisper 一个模型同时做语音转录、语音翻译、语言识别、时间戳对齐 —— 全靠输入序列里的**特殊 token** 区分任务，不改架构。
