Concept

多尺度训练 · 一套权重适配多档输入

一句话

训练时不断切换输入分辨率，让同一个全卷积检测器在部署时能按速度预算换挡。

直觉

固定 416 训练的模型像只练一种配速：换成低分辨率，虽然能跑，未必稳。多尺度训练每隔一会儿就换跑道，让网络习惯“画面粗一点要快、画面细一点要准”。

它不是图像金字塔推理：一次推理仍只选一个尺寸。也不是训练多套权重：尺寸变，权重不变。

怎么做的

YOLOv2 总下采样倍数是 32，所以训练尺寸必须是 32 的倍数：

每 10 个 batch:
  size ← random choice {320, 352, 384, ..., 608}
  resize network input to size × size
  继续训练同一套权重

全卷积结构没有固定长度的全连接层，因此能在训练中直接改空间尺寸。

数字例子

同一个目标原本在 416×416 图上宽 52 像素，占图宽：

52 / 416 = 0.125

若换到 320×320，保持相同比例后宽度约为：

320 × 0.125 = 40 像素

换到 608×608：

608 × 0.125 = 76 像素

✓ 归一化宽度始终是 0.125，但卷积看到的纹理细节从 40 像素涨到 76 像素。训练时反复经历这些档位，部署时才能平滑换挡。

速度—精度取舍

论文用同一套权重测试：

尺寸 VOC2007 mAP FPS
288 69.0 91
416 76.8 67
544 78.6 40

训练档位是 320–608；288 是额外低分辨率测试点，不能误写成训练集合的一部分。
