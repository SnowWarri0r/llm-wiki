Concept

分层图像生成 · 产出一摞可编辑图层，不是一张拍平图

一句话

生成的不是糊在一个平面上的像素图，而是一摞带透明通道（RGBA）的图层（画布+背景+K 个前景），每层能单独挪/改/重组——图像版的"逐元素编辑"。

直觉 · 拍平图改不动

普通文生图产出拍平（flat）的图：所有元素压成一个像素平面。想挪个 logo、换句文案、重排版，只能整张重画。设计真正要的是图层：画布、背景、每个前景元素各一层、透明可叠，单独编辑、可复用。mrt 把分层生成推到了大规模。

怎么做的 · 表示与三个硬骨头

表示成 {I_canvas, I_bg, {I_fg^i}_{i=1..K}}：

画布层 canvas   全尺寸、透明底（承载溢出，见下）
背景层 bg       半透明 RGBA
前景层 fg^i     各自一块 RGBA(带 Alpha 通道存透明/羽化)

每层用 VAE（mrt 用 video-vae，即 WAN-2.1-VAE）编码，Alpha 通道让"半透明/镂空"能被表示和合成。难点有三个，都来自"图层"本身：

  • 图层定义本就 ill-posed：一张图"该分成几层、怎么分"没有唯一正确答案（没有干净的 ground-truth）。
  • 遮挡补全：被上层挡住的部分要补全，半透明叠加时尤其难。
  • 溢出（overflow）：元素常超出可见画布边界（mrt 说 60%+ 设计有），旧做法在边界截断丢像素 → 挪回来缺一块；overflow 画布层把溢出也完整留住。

具体例子 · 一张促销图拆成什么

拍平图: [背景渐变 + 一张产品照 + "5折" 大字 + 角标logo] 糊成一张
分层后:
  canvas  : 全尺寸透明（含被裁出可见区的产品照边角）
  bg      : 渐变背景（半透明）
  fg¹     : 产品照（可单独换图/挪位）
  fg²     : "5折"文字（可改文案/字色）
  fg³     : 角标 logo（可替换/缩放）

✓ 自检：要把"5折"改成"3折"，分层下只重生成 fg² 一层，其余原封不动；拍平图则得整张重画、还可能改坏背景。这就是分层的价值——可编辑性 + 可复用性，代价是模型要学会"拆得干净 + 补得上遮挡 + 留住溢出"。

局限（mrt 自己列的）

设计数据训出来的模型对真实照片泛化弱（缺真实光影）、阴影/反射这类效果学不好、复杂半透明叠加的遮挡补全仍难、图层粒度因 ill-posed 而有歧义。
