Concept

重建即保真 · 能还原回输入，就说明中间表示没丢信息

一句话

要检查一个中间表示是否忠于输入，不必有标签——只问"能不能从它重建回原输入"：还原得回，信息就在；还原不回，关键信息已经丢了。

直觉 · 把"保真"变成"自编码"

relat 面对的难题：潜在推理（latent-reasoning）把问题压成一串潜在向量再往下用，但没人知道这串向量是否还带着原问题的约束。要验证它"忠不忠于输入"，通常得有标准答案——可推理时没有。

巧解：拿输入自己当可观测代理。如果潜在向量真的编码了问题 Q 的信息，那"从潜在向量重建出 Q"就该成功；重建不出来，说明任务相关信息在压缩中丢了。这是个纯自监督信号——不需要标签、不需要知道正确答案，只需要原问题(本来就在手上)。

必要非充分 · 它是过滤器，不是正确性证明

要点：重建成功不保证最终答案对（你可能记住了题目却推错）；但重建失败铁定意味着丢了信息。所以这是个必要条件：

潜在忠于输入  ⟹  Q 可重建      （逆否：Q 重建不出 ⟹ 潜在已跑偏）
Q 可重建      ⇏  答案一定正确   （只是没丢题，不代表算对）

用法因此是过滤/拦截：在正式作答前，把那些"已经从原问题漂移掉"的潜在状态先调回来，而不是去证明答案正确。

数字例子 · 重建损失怎么当信号

重建用对原问题 token 的掩码交叉熵（cross-entropy，只算真正的问题词，模板/padding 不算）：

问题 Q = "甲比乙大 3 岁"（关键 token: 甲, 乙, 大, 3, 岁）
从潜在 ℰ 逐 token 预测，看模型给真值 token 的概率：

潜在忠实:   P(甲)=.8 P(乙)=.7 P(大)=.9 P(3)=.85 P(岁)=.9
  loss = -[ln.8+ln.7+ln.9+ln.85+ln.9]/5 = 0.236   ← 低 → 信息在
潜在漂移(把"3"丢了):  P(3)=.05 其余不变
  loss = -[ln.8+ln.7+ln.9+ln.05+ln.9]/5 = 0.715   ← 高 → "3"没编进去

✓ 自检：漂移版唯一变化是"3 岁"那个关键约束没被潜在记住，重建损失立刻从 0.236 跳到 0.715——损失高 = 潜在丢了约束。relat 就用这个损失（在测试时调一组临时 lora）把潜在拽回忠于原题，再作答。
