Concept

Anchor boxes · 先给框一个形状起点，再预测怎么改

一句话

不让网络从零猜完整框；先在每个位置摆几种预设宽高，再回归中心和尺寸的相对变化。

直觉

直接报一个框，像让人凭空写出房子的经纬度和长宽。Anchor 先在地图每个格子上摆几块不同形状的模板：高瘦、扁宽、近方形。网络只回答“用哪块模板、往哪挪、放大多少”。

这降低了回归难度，也把一个位置能提出的框从少数几个扩到多个形状。代价是候选数暴涨、正负样本更不平衡，而且模板选得差时，网络得先费力把它拧成目标形状。

怎么做的

YOLOv2 在 13×13 的每个格子预测 5 个 anchor。对一个 anchor：

中心: b_x = sigmoid(t_x) + c_x
      b_y = sigmoid(t_y) + c_y
尺寸: b_w = p_w × exp(t_w)
      b_h = p_h × exp(t_h)

(c_x,c_y) 是格子位置，(p_w,p_h) 是 anchor 宽高。中心的 sigmoid 把结果限制在当前格；尺寸的指数把网络输出解释成相对 anchor 的倍数。

数字例子

取横向格子起点 c_x=3，网络输出 t_x=0：

sigmoid(0) = 0.5
b_x = 3 + 0.5 = 3.5

中心一定在当前格 [3,4] 内。

再取 anchor 宽 p_w=2，希望预测宽度 b_w=3：

需要放大倍数 = 3 / 2 = 1.5
t_w = ln(1.5) ≈ 0.405
b_w = 2 × exp(0.405) = 2 × 1.5 = 3

✓ 反算 ln(b_w/p_w)=ln(3/2)=0.405，正好回到网络输出。

跟直接回归的对比

方案 网络学什么 优点 代价
YOLO v1 直接报框 输出少、结构直 定位起点自由，召回低
YOLOv2 anchor 报相对模板的变化 候选多、召回高 模板和匹配规则很关键
