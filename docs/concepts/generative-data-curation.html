Concept

生成模型的数据策展 · 别在数据端先把多样性筛没了

一句话

训文生图模型时，用"美学打分器"过滤数据会把模型口味偷偷锁死；与其追"每张都漂亮"，不如守住"caption 准确"，并且一张 AI 合成图都不掺。

直觉 · 过滤器即偏见

直觉上，训练集越"漂亮"模型越漂亮，所以业界常拿美学打分模型（aesthetic score）和画质评估模型（IQA）去筛数据，低分的扔掉。

krea-2 的反对意见：这些打分器本身有口味。拿它当门卫，等于把它的偏见焊进你的数据分布——所有"不讨喜但真实"的图（朴素的纪实照、奇怪的构图、低饱和的旧照片）被成批删掉。模型从没见过这部分分布，以后也就永远画不出来。你以为在提质，其实在剪掉分布的尾巴 → 审美收敛到打分器喜欢的那一小撮。

它的替代信条很简单："只要 caption 准确描述了这张图，哪怕图本身不讨喜，也对训练有用。" 把关口从"好不好看"换成"配文对不对"。

怎么做的 · 只删四类 + 0 合成图

Krea 的过滤不按美学分，只剔除这几类：


1. 重复 / 过度重复的概念（去重，别让常见东西淹没长尾）
2. VLM 看不全要点的图（说明信息没法被 caption 准确捕捉）
3. 带有害偏见和明显伪影的
4. 视觉过于复杂、不适合低分辨率阶段建模的
5. 【硬规则】AI 生成的图 —— 一张都不要

注意分寸：这不是完全不过滤。低分辨率阶段用便宜的 CPU 过滤（拉普拉斯、熵）剔废图；高分辨率阶段也会上画质/美学过滤——但只用来删掉"极差"的，绝不用来 oversample"更美"的。差别全在意图：「删极差」是清噪声，「挑更美」才是把打分器口味焊进分布。Krea 只做前者。

配文侧：OCR 抽字 + captioner 结合元数据写信息密集长描述 + 便宜 LLM 改写成不同长度/格式，训练时以长 caption 为主（密集监督→收敛快），但也喂短/中 prompt，让模型推理时短句也能用。

为什么 0 AI 数据 · 合成图会给质量"封顶"

很多管线拿模型自己生成的图回填训练集。Krea 说哪怕掺一点点都不行，逻辑是：合成图更好学（它本来就来自一个简单些的分布），模型会优先把 loss 降在这些"软柿子"上，慢慢往合成分布塌——相当于给模型质量设了个上限。

一个玩具账本看"软柿子"效应。两类样本，模型在它们上的 loss：

            真实图(难, 信息丰富)   合成图(易, 已被某模型简化过)
初始 loss        2.0                    1.2
练一会儿         1.6                    0.4   ← 合成图掉得快得多

梯度下降天然偏爱"掉得快"的方向。混入合成图后，模型把劲都使在合成图上（回报立竿见影），真实图那条学得敷衍 → 输出越来越像"AI 味"的合成分布。用模型的输出喂模型，是在给后代近亲繁殖：每一代都比上一代更"好学"、更窄。所以 Krea 把合成图从预训练集里彻底清零。

自检：这不是"合成图质量差"那么简单——而是"合成图太好学"，会把优化的注意力从真实分布的难样本上吸走，等于自设天花板。

跟其它策展层的关系

预训练（自下而上、广度优先）之后，Krea 还有中训做自上而下的按域策展：层级 k-means 聚类 + VLM 复核、SigLIP 语义去重、用 PageRank 按"实体覆盖"补齐约 500 万个维基概念——保证常识实体都见过。这一层管"覆盖全不全"，跟预训练那层"别筛掉尾巴"互补。
