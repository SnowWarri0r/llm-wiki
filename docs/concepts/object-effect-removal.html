Concept

对象效果消除 · 抹的不只是物体,还有它在世界上留下的痕迹

一句话

真正的"删除一个物体"要连它的影子、反光、倒影一起抹掉,并把那块还原成合理的空背景;只抠掉物体本体、留下一摊孤零零的影子,看着比不删还假。

直觉 · 为什么"抠掉物体"远远不够

人眼对"物理一致性"极其敏感。一个人站在阳光下,地上有他的影子、旁边玻璃里有他的倒影。你把人 P 掉、影子和倒影还在——大脑立刻觉得"这画面有鬼":一摊没有主人的影子。所以对象消除的真实目标不是"把 mask 区域填上",而是把这个物体存在过的一切证据都清理干净。

难点在于:影子/反光通常不在你框的 mask 里。用户(或自动工具)框的是物体本体,影子在物体之外、形状还不规则。模型得自己意识到"这片暗区是那个物体投的,得一起清",这不是填洞能解决的,需要它理解物体和效果之间的因果。

两个常见翻车(见 omnieraser):

  • 影子赖着不走:只学会抠物体、没学会清效果(训练数据里压根没有真实光影)。
  • 凭空补新物体:模型不知道这块该空,照周围猜着补出个怪东西(只给背景上下文的锅,见 object-background-guidance)。

怎么做的 · 数据教因果 + 条件给意识

对付这两个毛病,要两条腿:

影子清不掉  → 用带真实光影的成对数据,且 mask 故意不圈影子,
              逼模型学到"物体↔效果"的关联   (<a href="../concepts/video-frame-paired-supervision.html">video-frame-paired-supervision</a>)
凭空补物体  → 把"要删的物体"也作为条件喂进去,
              让模型明确"这块要清空"        (<a href="../concepts/object-background-guidance.html">object-background-guidance</a>)
