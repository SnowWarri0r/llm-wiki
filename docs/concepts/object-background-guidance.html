Concept

双条件输入 · 不光告诉模型"周围长啥样",还告诉它"要删的是这个"

一句话

普通 inpaint 只把背景上下文喂给扩散模型,模型不知道挖空的洞里"本该是空的",于是照周围猜着补出个新物体;OmniEraser 把要删的物体本身也作为一路条件一起喂进去,模型就明确知道"这块是要清掉的东西、那块是要保留的背景",不再瞎补。

直觉 · 为什么只给背景会"凭空补物体"

inpaint 的标准做法:把要删的区域抠成洞,让模型"根据洞周围的内容把洞填上"。但"填上"这个目标本身是模糊的——

洞周围是草地和栏杆,模型很自然地觉得"这里应该也有点东西",于是补出一只新的狗、一个新的影子、或者一团 shape-like 的怪东西。

根子在于:模型对洞里"应该变空"这件事毫无意识。它只会延续周围的视觉线索。你越想删干净,它越觉得"这么大块地方总得长点啥"。这就是 omnieraser 说的 "synthesize a new object rather than faithfully removing"。

怎么做的 · 把"物体"和"背景"当两路独立条件拼进去

拼接(在 latent 通道维) 四样:
  ① 被抠出来的物体 latent   ← "这是要删的东西"
  ② 背景 latent             ← "这是要保留的上下文"
  ③ 噪声                    ← 扩散的起点
  ④ 二值 mask               ← "要动的位置在这"
拼好 → 过一个可训练 linear 投影 → 进 DiT 去噪

关键差别:物体不再是"被挖掉的洞",而是一路显式条件。模型同时看到"要删什么"和"要留什么",任务从含糊的"把洞填得像周围"变成明确的"把这个物体抹成背景"。

数字例子 · 加了"物体条件"FID 砍多少

RemovalBench 消融(越低越好):

只给背景(background-only)   FID 119.94   LPIPS 0.241   CMMD 0.509
加物体条件(object-background) FID  39.52   LPIPS 0.133   CMMD 0.208
                              ▲ FID 砍 66%

✓ 自检:FID 从 119.94 掉到 39.52,降了三分之二——这说明"凭空补物体"是主要失分项,一旦让模型明确知道"要删的是这个、别再创作",生成质量立刻上一个台阶。
