Concept

困惑度 Perplexity · 模型对一段话有多"意外"

一句话

把模型在一段文本上的平均交叉熵取指数，得到一个"等效分支数"：困惑度 = exp(平均每 token 的负对数概率)；越低 = 模型觉得这段话越自然、越在意料之中。

直觉 · "平均下来它在几个里猜"

模型每步预测下一个 token，给真值 token 一个概率。概率高 = 不意外；概率低 = 意外。把整段的"意外程度"汇总成一个数，就是困惑度。

它有个好懂的解读：等效分支数。困惑度=10 ≈ "模型平均每步像在 10 个等可能选项里猜"；困惑度=2 ≈ "平均每步像在 2 个里猜"——越小越笃定。所以困惑度低 = 这段话对模型很顺。

怎么做的

一段 N 个 token，模型给每个真值 token 的概率 p₁…p_N：
平均 NLL = −(1/N) Σ log pₜ        （= 平均每 token 的交叉熵）
困惑度  PPL = exp( 平均 NLL )

关键是那个 1/N：困惑度是逐 token 的几何平均，已经按长度归一了。所以它能公平比较不同长度的回答——长回答不会因为 token 多就吃亏，这正是 grape 用它给候选回答排序的原因。

数字例子 · 两段回答比"合身"

同一指令的两个候选，看目标模型给每个真值 token 的概率：

候选 A（合身）:   逐 token 概率 [0.5, 0.4, 0.5]
  平均 NLL = −(ln0.5+ln0.4+ln0.5)/3 = 0.768
  PPL = exp(0.768) = 2.15      ← 模型平均像在 ~2 个里猜，很顺

候选 B（强老师但不合身）: [0.2, 0.1, 0.3]
  平均 NLL = −(ln0.2+ln0.1+ln0.3)/3 = 1.705
  PPL = exp(1.705) = 5.50      ← 平均像在 ~5.5 个里猜，别扭

✓ 自检：A 的困惑度 2.15 < B 的 5.50。哪怕 B 来自更强的老师，目标模型觉得 A 更自然——grape 就选 A。注意用困惑度(而非总对数概率)是关键：总概率会偏向短回答，逐 token 几何平均把长度拉平了，比的是"平均每 token 多顺"。
