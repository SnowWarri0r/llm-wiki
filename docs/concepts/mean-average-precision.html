Concept

mAP · 检测的通用分数(每类的 PR 曲线面积再平均)

一句话

目标检测的总分:对每个类别画一条"准确率 vs 召回率"曲线、取曲线下面积(AP),再对所有类别求平均(mAP)。判一个预测算不算命中,靠 iou-intersection-over-union ≥ 阈值(常用 0.5)。

直觉 · 为什么不能只看"对了几个"

检测既怕乱报(把背景当物体 → 准确率 precision 低),又怕漏报(真物体没框出来 → 召回率 recall 低)。只看一个不行:全都不报,precision 满分但 recall=0;疯狂乱报,recall 高但 precision 稀烂。得同时看两者怎么权衡——这就是PR 曲线:把预测按置信度从高到低走,边走边记 precision 和 recall,画出一条线。曲线越往右上鼓(高准 + 高召回),检测越好,曲线下面积(AP)就越大。

怎么做的 · 命中判定 → PR 曲线 → AP → 平均


1. 每个预测框:和真值 IOU ≥ 0.5 且类别对 → 命中(TP),否则误报(FP);漏掉的真值 = FN
2. 按置信度从高到低扫,累计算 precision=TP/(TP+FP)、recall=TP/(TP+FN)
3. 画 PR 曲线,取曲线下面积 = 该类的 AP(平均精度)
4. 所有类别的 AP 求平均 = mAP

YOLO 论文里 63.4% 就是 PASCAL VOC 20 类的 mAP(IOU 阈值 0.5)。

数字例子 · 一条小 PR 曲线取 AP

某类 5 个预测,按置信度排序,命中/误报为 [✓ ✓ ✗ ✓ ✗],共 4 个真值:

第1个 ✓: TP=1 P=1/1=1.00 R=1/4=0.25
第2个 ✓: TP=2 P=2/2=1.00 R=2/4=0.50
第3个 ✗: TP=2 P=2/3=0.67 R=2/4=0.50
第4个 ✓: TP=3 P=3/4=0.75 R=3/4=0.75
第5个 ✗: TP=3 P=3/5=0.60 R=3/4=0.75
AP ≈ 在各召回水平上取"往右能达到的最高 precision"再平均
   ≈ (1.00 + 1.00 + 0.75) / …  → 这一类的 AP;20 类平均 = mAP

✓ 直觉:排在前面的预测越"又准又能覆盖真值",PR 曲线越贴右上、AP 越高。mAP 把 20 个类的这种表现汇成一个数,好横向比不同检测器(如 YOLO 63.4 vs Faster R-CNN 73.2)。
