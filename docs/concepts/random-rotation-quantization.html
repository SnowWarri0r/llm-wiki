Concept

随机旋转量化 · 把"不知道数据长啥样"变成"精确知道每个坐标的分布"

一句话

在线量化的死结是"看不到数据统计,不知道该把量化格子切在哪";随机旋转一下向量,它每个坐标就服从一个固定、已知的分布(集中的 Beta),而且高维下坐标之间近独立——于是最优量化器能离线预制好、在线只旋转+套用,完全不看数据。

直觉 · 旋转不是压得更狠,是把未知变已知

量化就是"把连续值塞进有限几个格子"。切格子要切在数据多的地方才省 bit——可在线场景看不到全量数据(KV 向量边生成边存、检索向量边来边压),不知道数据往哪偏,只能保守地均匀切,浪费。

随机旋转破局。关键几何事实:把任意向量乘一个随机正交矩阵(旋一下,长度不变),它的坐标就被"搅匀"成各向同性——单位向量的每个坐标,统计上都长一个样、服从同一个已知分布。原来"数据往哪偏我不知道",旋转后变成"每个坐标都服从我算得出来的那个分布"。未知 → 已知,这才是杠杆。分布已知,就能离线把最优标量量化器设计好;在线只需旋转 + 套这个预制量化器,零数据依赖(data-oblivious)。

怎么做的 · 旋转 → 每坐标最优标量量化 → 转回

离线(一次性):按已知的坐标分布,设计一个最优标量量化器(格子切在概率密度高处)
在线(每个向量 x):

  1. x' = R·x           # R 随机正交矩阵,旋转,长度不变
  2. 每个坐标 x'_i 各自套那个预制标量量化器 → 量化码
  3. 解码时反旋转 Rᵀ 转回

两个性质让它成立:① 坐标分布已知(旋转后是集中的 Beta,见下);② 高维近独立——不同坐标几乎不相关,所以"每坐标各自最优"复合起来 ≈ 整个向量的最优(turboquant 证明只差信息论下界 ≈2.7×)。

数字例子 · 旋转后坐标全挤到 ±1/√d 附近

单位向量(长度 1)随机旋转后,每个坐标平方 x_i² 服从 Beta(1/2, (d−1)/2),期望 E[x_i²]=1/d。所以典型坐标幅度 ≈ 1/√d——维度越高,所有坐标越挤在 ±1/√d 附近这一薄壳里。

d=4:   每坐标 ~ ±1/√4 = ±0.5      （例:旋转后 x'≈[0.5, −0.5, 0.5, −0.5]，长度 √(4·0.25)=1 ✓）
d=100: 每坐标 ~ ±1/√100 = ±0.1    每个都 0.1 上下,分布又窄又已知
d=1000:每坐标 ~ ±0.0316           越高维越集中

✓ 含义:旋转前坐标可能一个 0.98、其余接近 0(能量全堆一处,量化器不知道堆哪);旋转后每个坐标都是同一个窄分布,一个离线预制的量化器对所有坐标、所有输入都好使。这就是"未知变已知"的数字面孔——E[x_i²]=1/d 是拿到手就能算的常数,不用看任何数据。
