Concept

Mesh 质量指标 · CD 看平均偏差，HD 盯最坏一点，拓扑还得另查

一句话

Chamfer Distance 汇总两组点互找最近邻的平均距离；Hausdorff Distance 取最坏的最近邻距离，两者都不直接检查面朝向和连接结构。

为什么要两种距离

只看平均值，小块严重破损会被大量正常点冲淡；只看最坏点，又可能被一个离群点绑架。CD 和 HD 因此常一起报：

  • CD：整体形状平均差多少。
  • HD：最糟糕的局部差多少。

数字例子 · 三个一维点

把 3D 先压成一条线，真值点和预测点分别是：

GT   = [0, 2, 4]
Pred = [0, 3, 4]

预测点各自找最近真值：

0→0: 0    3→2或4: 1    4→4: 0
平均 = (0+1+0)/3 = 0.333

真值反过来找最近预测：

0→0: 0    2→3: 1       4→4: 0
平均 = (0+1+0)/3 = 0.333

若采用双向均值相加的定义，CD=0.333+0.333=0.666。最坏的最近邻距离是 1，所以 HD=1。

✓ 自检：若 Pred 也换成 [0,2,4]，所有最近邻距离都为 0，CD 和 HD 同时归零。

不同代码库可能对距离先平方、再开方，或把双向均值取平均而不是相加。比较论文前必须先对齐实现；MeshFlow 表里的 CD/HD 统一乘了 100。

为什么 mesh 不能只看这两个数

拿同一张三角面，把顶点顺序从 (1,2,3) 改成 (1,3,2)。几何位置完全没动，所以采样出的表面点也没动，CD=0、HD=0；但面法线已经从朝外翻成朝里。

洞、非流形边、细长三角面和难编辑的拓扑，也可能只带来很小的点距离。MeshFlow 直接预测法线，因此作者没有报告 Normal Consistency：这项分数会天然占便宜，却不能证明拓扑更好。

MeshFlow 的评测口径

  • 重建：比较原连续 mesh 与 VAE 重建，量化方法的坐标离散误差也会被算进去。
  • 生成：Toys4K 上从 mesh 表面采样 5,000 点，再算 CD/HD。
  • 缺口：没有一个表格指标专门检查洞、翻面、quad 布局或美术可编辑性，定性结果仍不可替代。
