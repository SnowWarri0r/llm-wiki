Concept

GAE · 把"超出预期多少"算得又稳又准

一句话

优势(实际 − 预期)有"看到底"和"只看一步"两种极端算法, 一个太抖一个太偏 —— GAE 用 λ 在两者之间插值, 取个甜点。

直觉 · 两种极端都不好

advantage-function 要算"实际比预期好多少", 难点在"实际接下来到底拿多少分"怎么估:

  • 看到底 (蒙特卡洛): 把这局剩下每一步的真实分全加起来。准, 但噪声极大 —— 后面发生啥全看运气, 抖得厉害。
  • 只看一步 (TD): 只看下一步的分 + critic 对下一个局面的估值。稳, 但偏 —— 太依赖 critic, critic 不准就全错。

一个太抖, 一个太偏。GAE 就是这两者之间的调和旋钮。

怎么做的

先认识最小的原子 —— 每一步的"惊讶值" (TD error, 记作 δ):

δ = 这步实际拿的分 + critic对"落脚处"的估值 − critic对"出发处"的估值

直白说: "这一步, 比 critic 当场预期的, 好了还是差了一点点。" 每一步都有一个这样的小惊讶。

GAE = 把未来所有惊讶加起来, 但越远的越打折:

优势 = δ_当前 + (γλ)·δ_下一步 + (γλ)²·δ_下下步 + ...

那个打折率是 γ × λ 相乘。比如 γ=0.99、λ=0.95 → 每往未来一步, 权重乘 0.94:

当前步 1.00 → +1步 0.94 → +2步 0.88 → +5步 0.73 → +10步 0.54 → +20步 0.29 ...

越远的惊讶, 指数级地越不算数。

为什么是这两个数相乘? 因为它俩打折的理由不一样:

  • γ (折扣): "晚拿到的分, 不如现在到手的值钱" —— 真实的时间折扣, 就算 λ=1 它也在
  • λ (GAE): "越远的惊讶越靠 critic 瞎猜、噪声越大" —— 额外再压一压, 纯粹的"信现实 vs 信 critic"旋钮

两个极端正好是 λ 的端点:

  • λ = 1 → 只剩 γ 打折, 远处惊讶照单全收 → 退化成"看到底"(准但抖)
  • λ = 0 → γλ=0, 除当前步全乘 0 → 退化成"只看一步"(稳但偏)
  • λ = 0.95 → 甜点, 主要看现实但让 critic 帮忙抹平噪声

代码出处

stable-baselines3 buffers.py 的 compute_returns_and_advantage: 从后往前累加 last_gae = delta + gamma * gae_lambda * last_gae, 正是上面那条递推。gae_lambda=0.95、gamma=0.99 是默认。
