Concept

点积与投影 · dot product · 衡量"方向多一致 + 多长"

一句话

两个向量点积 a·b = Σ aᵢbᵢ = |a||b|cosθ——对应分量乘起来加总,几何上等于"方向多一致(cosθ)× 各自多长"。它是相似度、注意力分数、检索打分、整个矩阵乘法的最小积木。

直觉 · 一个数说清"像不像 + 多强"

点积把两个向量压成一个数,这个数同时编码了两件事:

  • 方向一致吗:cosθ——同向为正、垂直为 0、反向为负。
  • 各自多大:|a|·|b|——向量越长,点积越大。

所以点积大 = "方向一致且都不小"。如果只想要"方向像不像"、不管长短,就除掉长度 → 余弦相似度 cosθ = a·b / (|a||b|)。

落点几乎无处不在:

  • self-attention 分数:Q·Kᵀ 就是"当前 token 的问"和"每个位置的答"做点积,点积大 = 越该关注。
  • retrieval-scoring / contrastive-learning:query 向量和候选向量点积当相关性分。
  • dual-tower-architecture:两塔末端就是一个点积。

投影 · a 在 b 上的影子

点积还直接给出投影:a 在单位方向 b̂ 上的影子长度 = a · b̂。

b̂ = b/|b|                       # 单位方向
proj_b(a) = (a · b̂) · b̂          # a 投到 b 方向上的分量（向量）
影子长度  =  a · b̂  =  |a|cosθ

直觉:拿手电筒沿 b 方向垂直照 a,墙上影子的长度就是 a·b̂。注意力、PCA 主轴投影、把一个向量"分解到某个方向有多少"全是这个动作。

怎么做的 · 一个数字例子

a = [3, 4]          |a| = 5
b = [4, 0]          |b| = 4
a·b = 3×4 + 4×0 = 12
cosθ = 12 / (5×4) = 0.6   →  θ ≈ 53°
a 在 b(x轴)上的影子 = a·b̂ = 12/4 = 3   （正好是 a 的 x 分量）

矩阵乘法 C = AB 的每个元素 Cᵢⱼ,就是 A 的第 i 行和 B 的第 j 列做一次点积——所以"矩阵乘 = 一大批点积",matrix-rank / GPU 的 GEMM 都建在它上。
