Concept

MAS · 动态规划找文本↔音频的单调对齐

一句话

文本有 N 个音素、音频有 T 帧(T 远大于 N),谁对应谁不知道;MAS 用动态规划找一条单调(不回头)的对齐——每个音素占连续的若干帧、依次铺满全部 T 帧——且让整体似然最大,不需要外部对齐器。

直觉 · 为什么难,为什么必须单调

一个音素会持续好几帧(拖长音就更多),但训练时没人告诉你"第 3 帧属于第几个音素"。要学好,得知道这个对齐。两个约束让问题可解:

  • 单调:说话是从左念到右的,对齐不能倒退、不能跳过——第 i 帧对应的音素,不会比第 i−1 帧的更靠前。
  • 满覆盖:每个音素至少占 1 帧,所有帧都被覆盖。

在这两个约束下,"最优对齐"能用动态规划高效搜出来(类似 DTW 找最优路径),避免了穷举天文数字种对齐。

怎么做的 · DP 在对齐矩阵上找最优单调路径

建一张 N×T 的打分表,格子(i,j)= "第 i 个音素配第 j 帧"的对数似然
只允许两种走法(保证单调):
  → 同一音素多占一帧(右走)     ↗ 换到下一个音素(右上走,音素轴自下而上)
DP 累计每条路径的总分,回溯出总分最大的那条阶梯路径 = 最优对齐

训练时用当前模型算这张表、MAS 定对齐,再拿对齐去优化模型——迭代收敛。承自 Glow-TTS。

数字例子 · 3 音素对齐 8 帧

音素 [a, b, c],音频 8 帧。单调对齐 = 把 8 帧切成 3 段连续块:

一种候选:a→帧1-3   b→帧4-5   c→帧6-8     时长 [3,2,3] 和=8 ✓
路径(阶梯,只右走/右上走):
音素c            ■ ■ ■
音素b        ■ ■
音素a  ■ ■ ■
      帧1 2 3 4 5 6 7 8
DP 会在所有"和=8 的单调切法"里,挑总似然最高的那种切法

✓ 关键是只准右走(同音素续帧)和右上走(换下一音素,音素轴自下而上),所以路径必是不回头的阶梯——这就是"单调"。挑出的时长 [3,2,3] 之后还拿去训 stochastic-duration-predictor,让推理时不看音频也能预测节奏。
