Concept

背景建模(MOG) · 固定机位下,谁老不变就是背景,谁突然变就是前景

一句话

固定机位拍的视频里,背景的每个像素长期稳定、前景(走动的物体)经过时才突变;给每个像素建一个"它平时是什么颜色"的概率模型(高斯混合),某帧某像素偏离太多就判为前景——这样不用人工就能从视频自动挑出"有物体 / 没物体"的帧。

直觉 · 为什么用"概率模型"而不是"减一下"

最朴素的想法:拿当前帧减去一张"空背景图",差大的地方就是前景。但现实里背景自己也会轻微抖:树叶晃、光照变、相机噪点。硬阈值一减,满屏都是假前景。

所以给每个像素单独学一个"它平时的颜色分布"。一片墙的像素常年是 灰=128 上下小幅波动,就用一个均值 128、方差小的高斯描述它。来了新一帧,这个像素如果还是 ~128,落在分布里 → 背景;如果突然变成 60(有人穿黑衣走过)→ 离分布老远 → 前景。用分布而不是单个值,就能容忍背景的正常抖动,只对真正的闯入者报警。

MOG(Mixture of Gaussians,高斯混合)再进一步:一个像素平时可能在几个状态间切换(树叶晃过来是绿、晃过去露出后面的天是蓝),所以用几个高斯一起描述它的常态,任意一个匹配上都算背景。

怎么做的 · 每像素几个高斯 + 占比阈值挑帧

对每个像素维护 K 个高斯(均值 μ、方差 σ²、权重 w):
  新像素值落进某个高斯(|值−μ| < 2.5σ) → 背景,更新该高斯
  都不匹配 → 前景,顶掉权重最低的那个高斯换新的
一帧里前景像素占比 > 0.15 → 判定"这帧有物体"(前景帧)
            占比很低     → "这帧物体已离开"(背景帧)

omnieraser 就用这个 0.15 阈值把视频帧粗分成前景帧 / 背景帧,再去配对。

数字例子 · 一个像素判前景还是背景

某墙面像素,常态高斯 μ=128、σ=4:

匹配带 = μ ± 2.5σ = 128 ± 10 = [118, 138]
新帧该像素 = 130  → 在带内 → 背景 ✓(只是正常抖动)
新帧该像素 = 55   → 离带老远 → 前景 ✓(有人走过遮住墙)

整帧统计:1024×1024≈100 万像素,若 18 万个像素被判前景(占比 0.176 > 0.15)→ 这帧标"有物体"。 ✓ 自检:用 2.5σ 的带宽(覆盖正态约 99%),背景的正常波动几乎都落带内不误报;只有真被遮挡、颜色大跳的像素才算前景。阈值 0.15 是为了滤掉零星噪点——得有成片像素变了才算"真有个物体"。
