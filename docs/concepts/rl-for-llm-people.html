Concept

RL 直觉打底 · 给懂 LLM 不懂 RL 的人

一句话

强化学习里那套"policy / rollout / advantage"的词, 翻译成 LLM 自回归就全是你已经会的东西 —— policy 就是模型那个 softmax 分布, 训练就是在拧这个分布。

直觉 · 为什么 PPO 那页看着云里雾里

PPO / RLHF 那几页默认读者已经有 RL 脑回路。但 RL 跟监督学习不是一套思维: 监督学习数据是死的, RL 的数据是模型自己生出来的、还会过期。这页把地基补上, 全程用 LLM 续写当例子, 不用想"游戏"。

§0 顿悟点: LLM 本身就是一个 policy

policy π(a|s) 听着玄, 其实就是: 给一个局面 state, 吐出"下一步每个动作的概率"。它就是个神经网络, 输入 state, 输出一个 softmax 分布。

对 LLM 直接对号入座:

RL 术语 LLM 里是什么
state s 目前为止的 token 串 (prompt + 已生成部分)
action a 下一个 token
policy π(a\|s) 模型最后那层 softmax 输出的分布
更新 policy 调权重, 让这个分布变形

举个具体的。state = "今天天气真", 模型输出下一个 token 的分布:

好  : 0.6
差  : 0.2
蓝  : 0.05
... 词表里其它几万个 token 摊掉剩下的 0.15

这个分布就是 π。 你训练 LLM, 本质就是调权重让它变形。RL 说的"概率分布变化"指的就是这个 —— 没有任何新东西。

§1 rollout 与"数据会过期"

rollout = 让模型真的按自己的分布走一遍, 把走过的路记下来。

给个 prompt, 模型按 π 采样: 采到 好 → state 变成 "今天天气真好" → 再采下一个 token → …… 直到 <eos>。这整条轨迹 + 最后的打分, 就是一次 rollout。跑很多次, 攒一堆 (state, action, 这条路最终多少分)。

关键的耦合 —— 这是 RL 跟监督学习最大的不同:

这批 rollout 是用当时那个模型 (叫 π_old) 采的。它记录的是"在 π_old 的口味下, 这些路各自值多少分"。

监督学习里数据是死的 (猫的图永远是猫), 你怎么更新模型都不影响数据。但 RL 里数据是模型自己生出来的。你一旦更新权重, 模型口味变了 (π_new), 它压根不会再走当初那些路 —— 那批旧数据就开始过期:

  • π_new 跟 π_old 还像 (小步) → 旧数据描述的路, 新模型大致还会走 → 数据还能用
  • π_new 跟 π_old 差很远 (大步) → 旧数据全是"新模型不走的路" → 拿它算梯度纯属误导 → 越训越飞

所以 PPO 的全部纠结就一句: 别让一步更新把分布推太远, 否则手里的数据立刻作废。 clip 就是这个刹车 (见 clipped-surrogate-objective)。

§2 它到底怎么就"增加 reward"了

光看 clip 那张图看不出 reward 怎么涨 —— 因为中间隔了一条因果链:

  1. 把"想要 reward"翻译成"调每个 token 的概率", 靠的是 advantage A (见 advantage-function): - A > 0: 这个 action 比平均好 → 想让模型下次更可能选它 (概率往上推) - A < 0: 比平均差 → 想让它更不可能 (概率往下压)
  2. 怎么推概率? 看简化目标 L = r · A, 其中 r = π_new(a)/π_old(a) = "新模型把这个 token 的概率, 相对老模型放大/缩小了多少倍"。A>0 时把 r 调大 (概率调高) → L 变大 → 梯度上升就去顶大 L → 它就真把这个 token 概率往上拧; A<0 反过来往下压。
  3. clip 只是限幅器: 想往上拧?行, 拧到 1.2 倍就够了, 再拧 loss 就 flat、梯度归零、不再给动力。防的就是 §1 那个"推太远数据作废"。
  4. reward 怎么涨的? 不是某一个 token 让它涨, 而是成千上万条 rollout, 每条里好 token 往上推一点、坏 token 往下压一点, 统计平均下来整个分布慢慢往高分区域挪。reward 曲线往上爬, 是这个挪移累积的结果。

一句话: advantage 决定每个 token 往哪拧, clip 限制拧多猛, 海量样本平均后分布整体偏向高分。 详见 policy-gradient。

§3 KL(LLM ‖ SFT) 是个啥

KL 散度 = 量两个概率分布差多远的尺子。一模一样 → KL = 0; 越不一样 → 越大 (只增不减, 没有负数)。

KL(LLM ‖ SFT):

  • SFT = PPO 开训前那个冻结的模型 (出发点)
  • LLM = 正在被 PPO 拧的这个
  • 这一项 = "现在的模型, 输出分布离出发点漂了多远"

为什么要它? 因为 PPO 一门心思冲 reward model 高分, 会把模型训畸形 (只会开头喊 "Certainly!"、疯狂凑字数 —— 这叫 reward hacking, 模型在钻 RM 的 bug)。

把它想成一根拴在 SFT 上的橡皮筋: reward - β·KL。你想往高分跑可以, 但离基线越远橡皮筋拽得越狠; β 是橡皮筋松紧。结果是模型只能"在正常说话的范围内尽量讨喜", 而不是变成只会刷分的怪物。详见 rlhf。

§4 这几个 loss 函数的走势

你说对这些函数"没概念", 这是最该补的。形状其实很简单:

  • ① cross-entropy -log(p): p 是模型给正确答案的概率。p=1 → loss=0; p→0 → loss 冲上天 (没上限)。一条从右下飙到左上的曲线。直觉: 答对越没把握, 惩罚越狠。
  • ② policy gradient -log(p)·R: 跟 ① 同形状, 只是整体乘了权重 R。R 大 = 这条样本特别值得学 → 整条曲线拉高 R 倍, 梯度也放大 R 倍。cross-entropy 管"对不对", R 管"这条值不值得使劲学"。
  • ③ KL: 两分布重合 = 0, 越分叉越大 (想成两条概率曲线不重叠的那块面积)。KL(LLM‖SFT) 就是 §3 那根橡皮筋的拉力。
  • ④ PPO 的 clipped L: 本该一路线性涨 (r·A), 但 clip 在 1±ε 把它削平, 削平处梯度=0、不再推。它是 ① 那种"无限放大"的反面。走势见 ppo 页 Fig 03。
