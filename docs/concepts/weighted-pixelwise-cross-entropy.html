Concept

Weighted Pixel-wise Cross-Entropy · 逐像素加权交叉熵

一句话

每个像素仍用 -log(真实类别概率) 计错，但关键像素再乘更大的权重，让模型优先修正更重要、也更容易被多数像素淹没的错误。

直觉

一张细胞图里，背景可能占 90%，两只细胞之间的窄缝不到 1%。所有像素一视同仁时，模型把窄缝错成细胞，对总损失影响很小；结果两只细胞粘成一团。

权重图相当于在答案纸上标出：“这几道边界题分值更高。”

公式回答什么

L = - Σ_{x∈Ω} w(x) log p_{l(x)}(x)
  • Ω：所有输出像素的位置集合；
  • x：其中一个像素坐标；
  • l(x)：该像素真实类别；
  • p_{l(x)}(x)：模型给真实类别的概率；
  • w(x)：这个像素的权重；
  • Σ：把所有像素的损失相加；
  • -log：正确概率越小，惩罚越大。

U-Net 的边界权重

w(x)=w_c(x)+w_0 exp(-(d_1(x)+d_2(x))²/(2σ²))
  • w_c 平衡类别数量；
  • d_1,d_2 分别是到最近、第二近细胞边界的距离；
  • 两者都小，说明像素夹在两个细胞之间；
  • 原文取 w_0=10, σ≈5 px。

数字例子

取一个夹在两只细胞间的背景像素：

w_c=1, d_1=1, d_2=1, w_0=10, σ=5

w = 1 + 10×exp(-(1+1)²/(2×5²))
  = 1 + 10×exp(-4/50)
  = 1 + 10×0.9231
  = 10.231

模型给真实背景的概率只有 p=0.119：

普通损失 = -log(0.119) = 2.127
加权损失 = 10.231×2.127 = 21.76

同一预测错误在普通区域若 w=1，只有 2.127。边界像素于是获得约 10.23 倍梯度权重。

自检：若模型把真实类别概率提高到 0.9：

加权损失 = 10.231×[-log(0.9)]
         = 10.231×0.1053
         ≈ 1.08

预测变好，损失从 21.76 降到 1.08，方向正确。

易混点

原论文 Eq. (1) 印成正的 Σw log p。若把它当“要最小化的 loss”，方向会反；实际等价做法是最大化这条加权对数似然，或最小化带负号的交叉熵。现代框架采用后者。

代码出处

  • PyTorch 可先用 F.cross_entropy(..., reduction="none") 得到逐像素损失，再乘位置权重图
  • 类别权重和位置权重不是一回事：前者按类别固定，后者可随像素位置变化
