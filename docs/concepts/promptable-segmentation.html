Concept

Promptable Segmentation · 可提示分割

一句话

把分割重新定义成"给任意提示（点/框/掩码/文字）返回一个合理掩码"，哪怕提示有歧义也得给出至少一个说得通的答案。

直觉

类比 LLM 的"下一个 token 预测"——那个任务本身平平无奇，但它通用到能撑起所有下游任务。SAM 想给分割找同一个"通用预训练任务"，答案就是可提示分割：不绑定"分割人/车/猫"这种固定类别，而是"你给个提示，我返回对应的掩码"。

它替代的是"一个数据集训一个专用分割模型"。一旦任务是"提示→掩码"，模型就能像 LLM 被 prompt 一样，被组合进各种下游任务（zero-shot-transfer）：想做物体检测？把检测框当 prompt。想做实例分割？让检测器吐框、SAM 抠掩码。

关键的拧巴点是歧义：你点在一个人的衬衫上，到底要衬衫、这个人、还是这群人？以前的模型被逼着猜一个。SAM 的任务定义网开一面——"返回至少一个合理掩码就算对"，并且让模型一次吐 3 个（整体/部分/子部分），再用 IoU 分数挑。把歧义当 feature，不当 bug。

怎么做的

输入: 图像 embedding(编码器跑一次) + 一个 prompt(点/框/掩码/文字)
   ↓ 轻量掩码解码器(2 层 transformer, 双向注意力)
输出: 3 个候选掩码  +  每个掩码的 IoU 预测分
   ↓
取 IoU 分最高的那个(或按下游需要全要)

训练时：对同一个 prompt，3 个预测里只对 IoU 最高的那个算 loss（反向传播只走最好的那条），这样 3 个 head 自然分化成"整体/部分/子部分"三种粒度，而不是互相打架。

数字例子

一个点落在「穿格子衬衫的人」的衬衫上。模型输出 3 个候选掩码 + 自评 IoU：

掩码 A = 整件衬衫        IoU 预测 0.92
掩码 B = 整个人          IoU 预测 0.64
掩码 C = 衬衫上一个口袋  IoU 预测 0.41
  • 默认取 IoU 最高的 A（衬衫） 交付 ✓
  • 但三个都留着——下游要"人"就用 B，要"口袋"就用 C，一个点同时覆盖了三种合理解读。
  • 训练时若这次的标注真值是"整个人"，就只对 B 算 loss、把 B 往真值推，A/C 不动。久了 3 个 head 就稳定分工到三种粒度。✓ 自检：如果只输出 1 个掩码，遇到这种点必然有 2/3 概率"猜错粒度"；输出 3 个把"猜"变成"覆盖"。
