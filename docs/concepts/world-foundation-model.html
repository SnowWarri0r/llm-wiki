Concept

World Foundation Model · WFM · 世界基础模型

一句话

一个学会"世界接下来会怎么变"的大模型——给它当前画面和一个动作，它能模拟/预测未来的画面，当 Physical AI 的训练场和推演器。

直觉

physical-ai 的智能体（机器人、自动驾驶）要在真实世界里学，可真实世界又慢又贵又危险——撞一次车、摔一次机械臂都是代价。所以需要一个能模拟世界的模型当训练场：智能体在里面"想象"地试，安全又能无限快进。

世界基础模型就是干这个的：它内化了世界的动态规律（物体怎么掉、人怎么走、车怎么开），所以能预测/生成可信的未来。给定"现在 + 一个动作"，它续出"接下来会发生什么"。这跟视频生成模型像，但目标不同——视频模型要好看，世界模型要符合物理、可被动作控制、能推演后果。

它替代的是"只能在真实世界里采数据"和"手写仿真器"。手写仿真器（如游戏引擎）画面假、覆盖窄；世界模型从海量真实视频里学，画面真、泛化广，还能批量造合成训练数据（synthetic data generation）。

怎么做的

输入: 当前观察(图/视频) + 文字指令 + (可选)动作
  ↓ 世界模型(通常是视频扩散 / 自回归 backbone)
输出: 未来的视频帧(+音频/+下一步动作)

用法三件套：

  • 模拟器：给动作 → 生成未来，让智能体在"梦里"试错（forward dynamics）。
  • 合成数据工厂：批量生成带标注的场景视频，喂给下游感知/策略模型，省真实采集。
  • 策略评估：拿世界模型推演一个动作序列的后果，挑最好的。

cosmos-3 把这套和"理解（VLM）+ 动作（策略）"用 mixture-of-transformers 缝进一个模型，所以它既是世界模型，也是 VLM，也是策略模型。

跟视频生成模型的对比

普通视频生成 世界基础模型
目标 画面好看、符合 prompt 符合物理、可被动作控制、能推演后果
控制 文字/图 文字/图 + 动作，可逐步交互
用途 内容创作 Physical AI 训练场 / 合成数据 / 策略评估
