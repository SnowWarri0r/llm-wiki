Concept

DualCodec · 低帧率 + 语义增强的双流音频码

一句话

把 24kHz 音频压成很少的离散码(12.5 帧/秒),而且第一个码本不是普通声学码、是从自监督模型蒸馏来的语义码——一条流管"说什么"、另一条流管"听起来啥样",低比特率还保音质。

直觉 · 为什么要"双流"

普通神经音频码(Encodec/DAC)把波形硬压成码,码里全是声学细节,帧率还高(~75Hz)→ 序列长,下游 LLM 难学、生成慢。DualCodec 的洞察:语音里"内容"和"音色细节"是两回事,应该分开编码。

  • 第 1 码本(RVQ-1)= 语义:直接从自监督模型 w2v-BERT-2.0 的第 16 层特征蒸馏。这层特征天然编码"说了什么内容",拿它当第一码本,下游模型只要先把这条短序列预测对,内容就对了。
  • 第 2–8 码本 = 声学残差:量化"波形减掉语义那部分"剩下的细节(音色、韵律、环境),把声音还原得像。

低帧率(12.5Hz)是另一个杠杆:同样一段话,码的条数少 6 倍(对比 Encodec 75Hz),下游序列短、跑得快。

怎么做的 · 双流编码 + RVQ

24kHz 波形
 ├─ SSL 流:  w2v-BERT-2.0 第16层特征 → RVQ-1(码本16384)= 语义码
 └─ 波形流:  CNN 编码器(4 层,strides 4·5·6·8·2)→ 降 1920 倍 → 12.5 帧/秒
             波形特征 − RVQ-1特征 = 残差 → RVQ 2..8 = 声学码
解码: (RVQ-1 语义 + RVQ 2..8 声学) 相加 → 解码器 → 波形

一共 N=8 个码本(rvq-codec 的残差递推),比特率约 0.75–0.93 kbps,质量胜过 Mimi / SpeechTokenizer / DAC / Encodec。

数字例子 · 12.5 帧/秒的账

输入 24000 样点/秒
编码器总下采样 = 4×5×6×8×2 = 1920
帧率 = 24000 / 1920 = 12.5 帧/秒        （25Hz 版少最后那个 ×2：4×5×6×8=960 → 24000/960=25）
对比 Encodec 24kHz：下采样 320 → 75 帧/秒
省了 75 / 12.5 = 6 倍的码条数 → 下游序列短 6 倍

✓ 一句 2 秒的话:DualCodec 只要 2×12.5=25 帧、每帧 8 个码;Encodec 要 2×75=150 帧。下游 LLM 面对的序列直接短一个数量级。
