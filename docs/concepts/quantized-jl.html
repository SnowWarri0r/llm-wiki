Concept

QJL · 1-bit 符号随机投影,给内积一个无偏估计

一句话

Johnson-Lindenstrauss(JL)说随机投影能保住向量间的距离/内积;QJL 更狠——投影后只留符号(1 bit),靠一个缩放系数,仍能给两个向量的内积一个无偏估计(平均意义上不偏不倚,只是有点方差),存储便宜到每维 1 bit。

直觉 · 为什么"只留符号"还能估内积

JL 变换:拿一个随机矩阵把高维向量投到低维,内积近似不变。QJL 把每个投影坐标压成 1 bit(只记正负号)。听着丢了很多信息,但关键在无偏:对一堆随机方向取符号、再按合适系数放大,sign(⟨R,x⟩) 跟另一个向量配起来,期望正好等于真内积——单次噪声大,多个方向一平均就准。省 bit 又不引入系统性偏差,正是它值钱的地方。

怎么做的 · 符号 + 缩放 = 无偏内积

给随机方向 r（高斯或 ±1）：
  存 x 时只存 1 bit：b = sign(⟨r, x⟩)
  估 ⟨x, q⟩ 时：用 c · b · ⟨r, q⟩ 这类项,对多个随机方向取平均
    → E[估计] = ⟨x, q⟩     （无偏；c 是与维度有关的缩放常数）

在 turboquant 里 QJL 不是单独用,而是接在 MSE 量化器后面纠偏:MSE 量化器压完主体、内积估计会偏(mse-vs-inner-product-bias),对残差 r=x−x̂ 再做 1-bit QJL,把那点偏差无偏地补回来——最终既低 MSE、内积又无偏。

数字例子 · 符号项平均出真内积

设真内积 ⟨x,q⟩=0.8。单个随机方向的符号项估计噪声很大(可能给 +2.1、−0.5、+1.3…),但期望对准 0.8:

方向1: +2.1   方向2: −0.5   方向3: +1.3   方向4: +0.3   …（单次乱跳,但无系统偏差）
取 4 个平均:(2.1−0.5+1.3+0.3)/4 = 0.80   ✓ 逼近真值,且不偏
方向越多 → 方差越小 → 越准；每方向只花 1 bit

✓ 对比:一个有偏的估计器可能永远给 0.72(系统性偏小),加再多样本也纠不回来。QJL 的价值是无偏——加方向就能收敛到真值,这正是 TurboQuant 拿它做残差纠偏的原因。
