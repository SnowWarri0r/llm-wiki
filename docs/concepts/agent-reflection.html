Concept

Agent Reflection · 反思 · 从碎片到认知

一句话

定期把大量低层观察总结成高层认知 —— "我看到 A/B/C/D → 我觉得 X"。反思结果存回 memory-stream，下次检索时跟原始观察一起参与排序。

直觉

你每天看到几百件事，但真正影响你决策的不是那几百件碎片，是你从碎片里提炼出来的判断。

比如你新入职一个月，每天看到的碎片是：

  • 周一：组长开会迟到 15 分钟
  • 周二：组长在群里发了个没什么内容的周报
  • 周三：组长当面被总监问进度答不上来
  • 周四：隔壁组的人跟你说"你们组长快被换了"
  • 周五：组长请全组吃饭但气氛很怪

这些碎片单独看都是小事。但你的大脑会在某个时刻把它们合成一条高层判断："我组长可能要被换掉了，我该提前想想怎么应对。"

这就是反思 —— 从碎片里提炼出碎片本身没有的信息。

怎么做的

Generative Agents 里反思的触发和流程：

触发条件：最近一批记忆的重要性分数加起来超过阈值（比如累积到 150 分）。不是定时触发，是"积累够了才反思"。

流程：


1. 从最近的高重要性记忆里提出 3 个问题
   LLM prompt: "给定这些记忆，你最想搞清楚的 3 件事是什么？"

2. 针对每个问题，从 memory stream 检索相关记忆

3. 让 LLM 基于检索到的记忆回答问题，生成 insight
   "基于以下记忆，你能得出什么结论？"

4. 把 insight 存回 memory stream（带时间戳 + 高重要性分数）

关键：反思的结果也是记忆。下次检索时它会跟原始观察一起参与排序，而且因为重要性分数通常更高，它更容易被捞出来。

反思的层级

反思可以基于之前的反思再次反思 —— 形成一棵树：

叶子（原始观察）:
  "看到 Klaus 在画画" · "Klaus 提到他喜欢印象派" · "Klaus 的房间挂满画"

第一层反思:
  "Klaus 对绘画很有热情"

第二层反思（基于多个第一层）:
  "Klaus 是一个有艺术追求的人，跟他聊天可以多聊艺术"

层级越高，信息越抽象但越有用 —— 做决策时你不需要知道"Klaus 房间挂了几幅画"，只需要"Klaus 是艺术爱好者"。
