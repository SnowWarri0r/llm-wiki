Concept

SHAP · 稀疏层级注意力

一句话

树中每个节点只读同层前一个节点、自己的父节点、父节点左右邻居和层首参考帧，用固定数量的连接传递长程规划。

连接规则

对第 ℓ 层第 i 个节点：

  • 同层前一个 (ℓ,i-1)：保持局部时间连续。
  • 父节点 π(ℓ,i)：知道自己正在细化哪段粗计划。
  • 左右相邻父节点：知道这段前后边界。
  • 每层第一个节点读取干净第一帧；之后通过同层链继续传播。

父层必须先生成，所以这些连接都指向已经存在的节点，不会看到未来。

为什么是线性复杂度

如果最终有 N 个细节点，完整二叉树约有 2N-1 个节点。每个节点只读固定的 4–5 个位置，因此注意力条目数量与节点数成正比，为 O(N)；全注意力或越来越长的完整前缀是 O(N²)。

KV cache 负责保存已经生成节点的 K/V，SHAP 决定当前 query 从 cache 里取哪几项。两者不是同一个概念：cache 避免重算，稀疏 mask 减少读取数量。

容易误读

SHAP 不是让最终第 100 帧直接看所有前 99 帧。它靠“父层总结长时间段 + 同层一步接一步”间接传信息；代价是摘要若丢了精确状态，细层可能出现墙体消失、物体变色等漂移。
