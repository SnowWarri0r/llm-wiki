Concept

Weak Supervision at Scale · 大规模弱监督

一句话

不花钱请人标数据，用互联网上天然存在的"音频 + 自动字幕"配对 —— 质量差但量大，最终效果碾压精标小数据。

直觉

传统 ASR 训练方式：请一群人听录音打字，一小时录音标注成本几十到几百美元。标得很准（强监督），但量上不去 —— 世界上最大的公开精标数据集 LibriSpeech 也就 960 小时。

Whisper 换了一套：互联网上到处是"视频 + 自动生成字幕"的配对（YouTube / 播客 / 新闻），字幕可能有错、不全、甚至是机器翻译的。但它们免费、海量、多样 —— 最终收了 68 万小时，比 LibriSpeech 多 700 倍。

类比：

  • 强监督 = 请了一位米其林厨师教你做 10 道菜，每道教得很细 → 你做这 10 道很好，换一道就不会
  • 弱监督 = 看了 10000 个民间厨师的短视频，很多步骤拍糊了、调料量说不清 → 你每道都不完美，但什么菜你都敢下手，鲁棒性极强

工程上更准的类比：在有噪声的数据上训大模型 = 用 dropout 训神经网络 —— 噪声本身就是正则化手段。

怎么做的

  1. 从互联网上收集"音频 + 文本配对"（不人工标注）
  2. 过滤明显不靠谱的（语言不匹配、文本太短、音频静音）
  3. 保留的配对里文本可能有错 → 没关系，量足够大时模型能学出"多数正确"的分布
  4. 用标准 encoder-decoder Transformer 在这些数据上直接训
  5. 最终模型不在任何目标测试集上 finetune → zero-shot 跑

为什么管用

噪声在大规模下是可以被平均掉的。

如果 1000 条训练样本里有 50 条字幕是错的（5% 噪声率），模型在这 50 条上学到的是噪声，在其他 950 条上学到的是信号。只要信号量 >> 噪声量，模型的期望输出就趋向正确。

精标小数据反而有另一个问题：不够多样。LibriSpeech 主要是美式英语的朗读语音 —— 模型在这上面训得再好，碰到印度口音 / 街头采访 / 方言 / 背景噪声就完蛋。Whisper 的 68 万小时覆盖了 99 种语言 + 各种说话场景，模型被迫学会应对所有情况。
