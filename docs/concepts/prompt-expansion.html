Concept

Prompt Expansion · 把短输入扩成富 caption（还得防塌成一种风格）

一句话

模型训在信息密集的长 caption 上，用户却爱打短句；前面挂一个会写字的 LLM 把短输入扩成富 caption，用 SFT 学会扩写、再用 GDPO + 多奖励把扩写调到"提质又不跑题、还别塌成一种风格"。

直觉 · 训练分布和用户分布对不上

文生图模型见过的是"光影、材质、构图都写满"的长描述，可用户输入往往短、模糊、各人习惯不同（"画只猫" vs 一长串 JSON）。这中间的落差就由 krea-2 的 prompt expander 补：它是个 LLM，把用户的短 prompt 改写成模型爱吃的富 caption，再喂给生成模型。

怎么做的 · 两段训练

① SFT（学会扩写）：用另一个 LLM 从长 caption 反向造"用户 caption"（短、口语、半指令、故意省细节），配成 (短输入 → 富 caption) 的训练对让 expander 模仿；还合成思考轨迹（中间一步"先重建用户意图"，能提升下游）。再做分布塑形——oversample 艺术性图扩大创意覆盖、给该写实的 prompt 加轻量"摄影介质"偏置；目标不是强加 house style，而是让扩写分布同时覆盖"艺术导演范"和"直白写实"。

② RL（GDPO + 多奖励）：目标从"模仿目标 caption"转成"扩写要让出图更好、同时保住用户意图"。难点是没法便宜地让人对"哪个扩写更好"做海量配对标注，所以用 GDPO（Generative DPO，direct-preference-optimization 的生成式变体）：偏好对不是人标的，而是模型自己生成候选、用奖励自动判出来的。

GDPO 配多奖励 · 偏好对怎么自动来

对一条用户 prompt：

1. expander 生成多个候选扩写
2. 每个扩写出图，再打多奖励：
     图级奖励      = 出图质量/偏好
     prompt 级可验证 = 扩写有没有忠于原请求（程序可查）
     安全闸        = 别引入没请求的/不可接受内容（稀疏约束）
     多样性        = DINOv3 组内多样性分（见 <a href="../concepts/dinov3-diversity-reward.html">dinov3-diversity-reward</a>）

3. 合成总分 → 分高的当"赢"、分低的当"输" → 自动凑成偏好对
4. 拿这些自动偏好对跑 DPO

数字例子 · 一条 prompt 凑出偏好对

三个候选扩写，各项打分（图级 0–1、忠实 0/1、安全 ok=不扣）：

候选   图级   忠于原意   安全    总分(简单相加, 忠实/安全当门)
 E1    0.8      1         ok     0.8
 E2    0.5      1         ok     0.5
 E3    0.9      0(跑题)   ok     —   （忠实=0 直接出局：再好看也不能改用户意图）
→ 赢家 = E1(0.8)   输家 = E2(0.5)   组成偏好对喂 DPO

✓ 自检：E3 出图最好看（0.9）但改了用户意图（忠实=0），被可验证奖励一票否决——这正是"提质但不能跑题"的约束。剩下 E1>E2 形成偏好对，无需任何人工标注。多样性分则跨这一组算（见 dinov3-diversity-reward），防止 expander 把每条 prompt 都扩成同一种风格。RL 的 prompt 还混了硬例（内部踩、bug 报告、手改 prompt，按失败类分桶），挑"难但不绝望"的——综合分中等、有真实方差，DPO 才拿得到有效信号。
