Concept

扩散 RL 的似然墙 · 为什么不能照搬 GRPO,以及绕反向的代价

一句话

policy-gradient/ppo/grpo 都要"模型给这个样本的概率",LLM 逐 token 现成、扩散却算不出来(似然只能靠昂贵近似);主流绕法是把反向去噪切成多步马尔可夫过程让每步概率可算,但这条路锁死采样器、还可能让模型偏离前向过程。

直觉 · LLM 能直接做 RL,扩散卡在哪

RL 微调 LLM 顺手,是因为策略梯度要的 π(动作) 唾手可得:模型对每个 token 输出的就是概率分布,一条回答的概率=逐 token 概率连乘。奖励一来,乘上对数概率的梯度就能更新。

扩散没有这种现成概率。一张图是从噪声经几十步去噪来的,"模型生成这张图的概率"要对所有可能路径积分——只能靠概率流 ODE 或 SDE 的变分下界昂贵地近似。没有可算的似然,policy gradient 这一系就直接卡住。这就是似然墙。

怎么做的 · FlowGRPO 的绕法与三个坑

FlowGRPO / DanceGRPO 的招:把反向采样(噪声→图,逐步去噪)看成一个多步马尔可夫决策过程,每一步的状态转移是个高斯——高斯概率可算,于是每步就有了可算的"动作概率",能套 grpo 了。代价是继承三个毛病:

① 锁死 solver:训练损失耦死一阶 SDE 采样器,用不了扩散默认更高效的 ODE / 高阶 solver
② 前后向不一致:只盯反向动态,丢掉对唯一前向过程的忠实,模型可能退化成"级联高斯"
③ CFG 难搞:扩散重度依赖 CFG,RL 里硬塞 → 变成又慢又繁的两模型优化

diffusionnft 正是冲着这三点来的:既然前向加噪只有一条、反向去噪有无数条,就把 RL 搬回唯一的前向过程,似然墙、solver 锁死、CFG 两模型一次性全绕开。
