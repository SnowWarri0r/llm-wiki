Concept

LLM-as-Judge · 拿模型当判官

一句话

用一个模型给别的模型的输出自动打分/排名，替掉又贵又慢的人工评测——前提是先把这个判官校准到跟人类口味一致。

直觉

评 1000 张图、几十个模型,全靠人打分又贵又慢又难复现。自然想法:让一个模型当裁判(judge),自动打分。

但有个坑:随便抓个通用多模态大模型(MLLM)当裁判,它的口味跟人类专家对不齐——它可能偏爱艳丽配色、看不出专业创作者在乎的细节。所以关键不在"用不用模型当判官",而在把判官校准到人类偏好:拿大量人类标注训这个判官,直到它的排名跟人类专家高度一致,才敢放它自动评。

qwen-image-bench 的 Q-Judger 就是这么炼的:基于 Qwen3.6-27B,用 80 位美院标注员盲标、13 万+人标对训练,最后跟人类专家排名 Spearman ρ=0.92(几乎一致),才拿它去自动评 56 条细则。这跟 rlhf 训 reward model 是同一个母题——想规模化判断,先用人把判官喂准。

判官值不值得信,就看它的排名跟人类排名有多吻合,常用 Spearman 等级相关 ρ:ρ=1 完全一致、0 没关系、−1 完全相反。

怎么做的


1. 收集人类偏好标注(成对比较 / 逐条打分),量要大
2. 拿这些标注训判官(或微调一个 MLLM), 让它的打分逼近人类
3. 验证: 在留出集上算判官排名 vs 人类排名的 Spearman ρ, 够高才敢用
4. 部署: 判官自动给海量输出打分, 人只抽查

数字例子

判官信不信得过,核心看它给模型排的名次跟人类排的有多像。Spearman ρ 就量这个:

ρ = 1 − 6·Σd² / (n·(n²−1))     d = 同一个模型在两套排名里的名次差, n = 模型数

4 个模型 A/B/C/D。人类专家排名(1=最好): A=1, B=2, C=3, D=4。判官排名: A=1, B=3, C=2, D=4。

名次差 d:  A: 1−1=0   B: 2−3=−1   C: 3−2=+1   D: 4−4=0
d²:        0          1            1            0      → Σd² = 2
ρ = 1 − 6·2 / (4·(16−1)) = 1 − 12/60 = 1 − 0.2 = 0.8

✓ 自检:判官只把 B、C 调了个个,其余全对 → ρ=0.8(高度相关但没满分)。Q-Judger 的 ρ=0.92 比这还高,意味着它给一堆模型排的名次跟人类专家几乎一模一样,所以可信。要是 ρ 只有 0.3,这判官就是瞎打,不能用。
