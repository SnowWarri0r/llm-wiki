Concept

语义码 vs 声学码 · 把"说什么"和"听起来啥样"分开

一句话

现代 TTS 把语音拆成两类离散码:语义码只管内容(说了哪些字、什么韵律骨架)、声学码管音色和细节(谁在说、什么情绪、什么环境)。分开之后,内容用短序列先定,细节再并行补——又快又可控。

直觉 · 为什么要拆

如果只有一种码(全是声学),那"说什么"和"听起来啥样"混在一起,模型得同时学两件难事,序列还长。拆开后各管一段:

  • 语义码:从自监督模型(w2v-BERT / HuBERT 这类)蒸馏,天然编码"内容"。序列短、信息密度高、好预测。
  • 声学码:量化波形残差,还原音色/情绪/环境。信息量大但可以并行补(不必逐帧慢慢来)。

好处很实在:① 克隆音色 = 换声学码那一路,内容不变;② 情绪控制 = 在声学侧加 CFG;③ 内容用短语义序列先定,延迟低。

怎么做的 · 两级码的分工

文本 ──(LLM,自回归/短序列)──▶ 语义码   ← "说什么",内容骨架
参考音 ─▶ 声学 prompt
语义码 + 声学 prompt ──(NAR 并行)──▶ 声学码  ← "听起来啥样",音色/细节
声学码 + 语义码 ──(codec 解码)──▶ 波形

dualcodec 就是这么落地的:RVQ-1 = 语义码(w2v-BERT 蒸馏),RVQ 2–8 = 声学码(残差)。

数字例子 · 序列长度对比

一句 2 秒话,帧率 12.5Hz → 25 帧
语义码:  25 帧 × 1 码/帧 = 25 个码     ← LLM 只预测这条短序列(定内容)
声学码:  25 帧 × 7 码/帧 = 175 个码    ← NAR 一次性并行填(补细节,不逐帧)

✓ 内容(25 个码)交给擅长序列的 LLM 逐步定;细节(175 个码)交给 NAR 并行补——各用各的长处,总延迟被压下来。若全塞给一个自回归模型逐帧吐 200 个码,就是又慢又长。
