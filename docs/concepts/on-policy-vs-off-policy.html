Concept

on-policy vs off-policy · 学自己刚做的，还是学别人/旧的

一句话

on-policy = 拿当前策略自己刚生成的数据学（信号对得准、稳，但得不断现采）；off-policy = 拿别的策略（旧的自己/别的模型/人类）的数据学（能复用、便宜，但分布错位、要纠偏）。

直觉 · policy 是什么

策略（policy） = 智能体当前的行为方式：给定状态，它会做什么。放到 LLM 上，策略就是模型本身——给定上下文它吐出什么 token，就是它的策略。

  • on-policy：学习用的数据，正是这个模型现在自己会产生的。边走边学。
  • off-policy：学习用的数据来自另一个分布——旧版自己、别的模型、人类标注、回放池。

核心张力一句话：对得准 vs 能复用。

  • on-policy：数据 = "模型此刻真实的做法"，更新信号贴合、稳；代价是得一直现采（模型一更新，旧数据就过期，基本不能复用）。
  • off-policy：能复用旧数据/蹭别人的数据（省、样本效率高）；代价是分布错位——数据来自别的分布，直接学有偏、不稳，要纠偏。

开车类比：on-policy = 学自己开车（反馈贴合你的手感，但得一直开）；off-policy = 看 F1 录像/自己几年前的旧录像学（素材便宜又多，但是别人的/旧的开法，照搬未必迁得到你现在的车）。

数字例子 · 分布错位为什么有偏，重要性采样怎么纠

两个动作 A、B，奖励 r(A)=1, r(B)=10。

  • 当前策略 π（我们关心的）：π(A)=0.8, π(B)=0.2
  • 数据来源 μ（旧的/别的策略，off-policy 数据从它来）：μ(A)=0.5, μ(B)=0.5
我们想要的:  E_π[r] = 0.8·1 + 0.2·10 = 2.8

直接拿 μ 的数据平均(off-policy 不纠偏):
  E_μ[r] = 0.5·1 + 0.5·10 = 5.5   ✗ 偏了(μ 比 π 多采了高奖励的 B)

重要性采样纠偏: 每条样本乘权重 w = π(a)/μ(a)
  A: w = 0.8/0.5 = 1.6   B: w = 0.2/0.5 = 0.4
  Σ μ(a)·w(a)·r(a) = 0.5·1.6·1 + 0.5·0.4·10 = 0.8 + 2.0 = 2.8   ✓ 回到真值

✓ 自检：μ 过采了 B（高奖励），所以裸平均 5.5 偏高；重要性采样把"π 比 μ 更/更不偏好某动作"折算回来（B 的权重 0.4 把它压下去），就还原成 π 的真实期望 2.8。一句话：用 π/μ 这个比率把"别人的数据"重新加权成"像我自己采的"。

代价（为什么不能随便 off-policy）：当 π 和 μ 差很远，比率 π/μ 会在某些动作上爆掉（μ 很少采、π 很想要的地方，权重巨大）→ 估计方差爆炸、训练不稳。所以离当前策略越远越危险。ppo 的 clip、clipped-surrogate-objective 正是"只允许小步 off-policy"的纠偏阀门：把比率裁在 1 附近。

接回 wiki

  • ppo / grpo / rlhf 是 on-policy：模型自己 rollout、自己学；clip/重要性采样让它能安全地复用几步内的旧数据。
  • 普通 SFT 拿外部强模型的回答训 = off-policy（数据来自别的分布）。
  • grape / distribution-aligned-sft：选"目标模型自己给高概率(perplexity 低)"的回答 = 把 SFT 往 on-policy 拉——挑贴合自己分布的素材，少漂移。
  • direct-preference-optimization：在固定偏好集上训(偏 off-policy)，所以才有 on-policy 变体/STPO 来缓解错位。
