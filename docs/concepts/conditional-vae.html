Concept

条件 VAE · 学一个潜变量,把数据编进去再还原(带条件)

一句话

VAE 学一个中间"潜变量 z":编码器把数据压进 z(而且逼 z 服从一个规整分布)、解码器从 z 还原数据。条件 VAE 就是让这套过程带上条件(比如文本),训练目标是最大化 ELBO = 还原得像 − 潜变量别偏离先验太远。

直觉 · 为什么要个"潜变量"

直接学"文本→波形"太难:一段文本对应无数种说法。VAE 的招是引入一个中间桥 z——把音频的信息(音色、韵律、细节)压进一个规整的潜空间,解码器只需从 z 还原波形。训练时:

  • 后验编码器 q(z | 音频):看真实音频,给出该用哪个 z。
  • 解码器 p(波形 | z):从 z 还原波形。
  • 先验 p(z | 条件):不看音频、只看条件(文本),猜 z 大概长啥样。推理时没有音频,就从这个先验取 z。

关键张力:后验(看了音频,信息足)和先验(只有文本,信息少)要尽量接近,推理才能拿先验代替后验。

怎么做的 · 最大化 ELBO(两项拔河)

ELBO = E[ log p(数据 | z) ]  −  KL( q(z|数据) ‖ p(z|条件) )
        └─ 重建项:还原得像 ─┘     └─ 正则项:后验别偏离先验太远 ─┘
  • 重建项拉高 → 还原逼真(vits 里是 mel 频谱的 L1 log-mel-spectrogram)。
  • KL 项拉低 → 后验贴近先验,推理才能用先验采样。 两项拔河,学出一个"信息够还原、又贴合条件先验"的潜空间。

数字例子 · KL 项在算什么

设某维后验 q=N(均值 1.0, 标准差 0.5)、先验 p=N(均值 0.8, 标准差 0.5)。两个同方差高斯的 KL:

KL = (μ_q − μ_p)² / (2σ²) = (1.0 − 0.8)² / (2·0.25) = 0.04 / 0.5 = 0.08

✓ 后验均值离先验越远,KL 罚越大 → 逼编码器别把 z 放到先验够不着的地方。若两者重合(μ_q=μ_p),KL=0,推理用先验采样和用后验一样好。VITS 的痛点正是"文本先验太简单、够不着音频后验",所以要 normalizing-flow 把先验掰复杂。
