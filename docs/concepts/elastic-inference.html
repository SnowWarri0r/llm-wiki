Concept

Elastic Inference · 一个模型，多档算力

一句话

一次训练出一族模型，推理时按手头算力临时挑一档（快但糙 ↔ 慢但好），不必为每个预算单独训一个模型。

直觉 · 别为每块设备各训一个

部署一个模型要面对千差万别的预算：手机要快、服务器能慢工出细活。传统做法是训一堆不同大小的模型（small/base/large），各训各的、各存各的，贵且难维护。

Elastic（弹性）的思路：训一个，内部自带多档。推理时一个旋钮一拨，就在"快而糙"和"慢而精"之间滑动，沿质量-算力的 Pareto 前沿走。Matryoshka（套娃）系列把这个旋钮做在"宽度/维度"上；looped-transformer 的 ELT 把它做在循环圈数 L 上——多套几圈就多算、更准。

怎么做的 · 训练时就把每一档都练好

难点不在推理，在训练：只有一套共享 block，怎么保证"中途退出"的浅档也像样、而不是只有满配能用？如果只拿满圈 L_max 训，这套权重只学会"套满才好"，第 3 圈停下就是半成品。

naive 想法的坑：把所有圈数（2/4/6/8）都跑一遍各算真值 loss——① 太贵；② 各圈数可能各出互不相干的答案，没人保证"圈数越多越好"。

ELT 的 Intra-Loop Self Distillation（ILSD） 在一步训练里解决。拿 ELT 是图像生成想最顺：循环 = "在同一张画布上反复精修"，圈数越多修得越细。

每个训练步：

  1. 随机抽一个中途圈数 L_int（这步抽到 3，满配 L_max=8）。每步抽的不一样。
  2. 前向跑循环，用同一个输出头读出两个结果： - teacher = 满 8 圈输出 F_max（精修到底） - student = 第 3 圈退出输出 F_int（只修 3 遍的粗版）
  3. 三个 loss，全灌进同一套共享 Θ：
loss 干什么 为什么要它
真值(F_max) 满配对齐真实目标 锚住"最好那档"是真的好
λ·真值(F_int) 中途档也对齐真实目标 让粗版自己就能用，不只是半成品
蒸馏(F_int→stopgrad F_max) 把粗版拉向精修版 让"多修几圈"=朝同一张画稳定精修，不是各画各的
  1. 反向传播 → 更新那唯一一套 block 参数。

三个设计的"为什么"：

  • 随机抽 L_int：一步只多读一个退出点（不是全跑），很多步下来每个深度都被训到 → 便宜（类似 stochastic depth 的抽样）。
  • 蒸馏 + stop-grad 是灵魂：蒸馏让"第 3 圈的画"长得像"第 8 圈的低细节版"，于是循环变成一条精修轨迹（每多一圈朝 F_max 推进），保证"圈数越多越好"；stop-grad 冻住 teacher 那侧，让 student 往 teacher 靠、而不是 teacher 被弱 student 拖down成平庸中间值——强的拉弱的上来，弱的不拖强的下去。

净效果：这套共享 block 被逼成一个"单调精修器"——任意一圈停下都是张连贯（只是更粗）的输出，多套一圈就更精。弹性就这么来的。

它换来什么

  • 一次训练，覆盖多端：一个权重文件，手机跑 2 圈、服务器跑 10 圈。
  • Any-Time 推理：算力紧就早退、宽裕就多算，运行时动态权衡。
  • 代价：训练更复杂（要同时照顾所有档），而且每一档未必比"专门为这档训的单模型"更强——换的是部署的灵活和省事。
