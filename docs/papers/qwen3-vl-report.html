Qwen3-VL ·
给大模型装一双能看图的眼睛。

Qwen Team, Alibaba · Qwen3-VL Technical Report · arXiv 2511.21631 · 2025-12

一个 VLM = 眼睛(视觉编码器) + 翻译插头(merger) + 大脑(LLM)：图按原生分辨率变成变长 token，和文字 token 进同一条序列让 LLM 一起读。Qwen3-VL 在这套上做了三处升级——Interleaved-MRoPE、DeepStack、文字时间戳——把长视频、细粒度感知、视频定位顶上去。

TL;DR

Qwen3-VL = 三件套 VLM：SigLIP-2 ViT(眼睛) → MLP merger(翻译插头，2×2 patch 压成 1 token)1 → Qwen3 LLM(大脑)，图按原生动态分辨率变成变长 token2。dense 2/4/8/32B + MoE 30B-A3B/235B-A22B，各出 non-thinking/thinking。三处架构升级(vs 2.5)：① Interleaved-MRoPE3——把 t/h/w 交错铺满高低频(老 MRoPE 分块→频谱不均、长视频差)；② DeepStack4——抽 3 个中间 ViT 层经各自 merger 注入 LLM 前 3 层(多层视觉特征，不增上下文)；③ 文字时间戳5——视频每组帧前缀 <3.0 seconds> 文字 token 代替 T-RoPE。训练：预训练四阶段(S0 只训 merger→S1/2/3 全参，上下文 8K→32K→256K) + 后训练三步(长 CoT SFT→强师蒸馏→RL)。能力：原生 256K(→1M)、OCR 32 语种、2D+3D grounding、GUI agent、视觉数学，纯文本还反超同级文本模型。

§ 01VLM 三件套：眼睛 → 插头 → 大脑

视觉语言模型(VLM)的标准结构就三块（跟你在 minimind-o/Qwen3-ASR 见过的模态编码器→projector→LLM一个套路）：

  • SigLIP-2 ViT（眼睛）：把图切 patch 编码；继续在动态分辨率上训，用 2D-RoPE + 插值绝对位置适配任意尺寸。旗舰用 SigLIP2-SO-400M，小模型(2B/4B)用 SigLIP2-Large(300M)。
  • MLP merger（翻译插头）：两层 MLP 把 2×2 个视觉特征压成 1 个 token、对齐到 LLM 维度——既降 token 数又接进语言空间。
  • Qwen3 LLM（大脑）：图像 token 和文字 token 进同一条序列一起 attention。dense 2/4/8/32B + MoE 30B-A3B / 235B-A22B(后者激活 22B)。

§ 02三处升级(vs Qwen2.5-VL)

① Interleaved-MRoPE · 把频谱铺均匀

多模态位置编码 MRoPE 给每个 token 标 (时间 t, 高 h, 宽 w) 坐标。老做法把嵌入维度切成 t / h / w 三大块各给一段旋转频率 → 频谱不均（t 全占低频、w 全占高频），长视频位置建模差。Qwen3-VL 把 t/h/w 交错铺满高低频 → 每个轴都均匀覆盖全频段 → 长视频长程位置强多了。

② DeepStack · 不只取最后一层 ViT

普通 VLM 只把 ViT 最后一层的视觉 token 喂给 LLM。DeepStack 抽 3 个不同深度的 ViT 层（浅层细节 → 深层语义），各过一个专用 merger，残差加到 LLM 前 3 层的 hidden state 上——多层视觉信息都进来，而且不增加上下文长度。

眼熟吗

这跟 Ideogram-4 把 Qwen3-VL 当文本编码器时"取 13 个中间层拼起来"是同一个思想：多层特征 > 只取最后一层，不同深度编码不同抽象度，全要。一个发生在 ViT 侧(DeepStack)、一个在 LLM 侧(Ideogram)。

③ 文字时间戳 · 视频帧前缀 <3.0 seconds>

Qwen2.5-VL 用 T-RoPE 把时间塞进位置编码，长视频时位置 id 又大又稀、还要密集采各种帧率训（背景见 M-RoPE 页）。Qwen3-VL 改成：给每组视频帧前缀一个文字时间戳 token（如 <3.0 seconds>，秒和 HMS 两种格式都训）→ 简单、精确，视频定位/dense caption 更强。（优化上还把 per-sample loss 换成平方根归一的 per-token loss，平衡纯文本 vs 多模态。）

§ 03训练：四阶段预训练 + 三步后训练

阶段训什么token / 上下文
S0 对齐只训 MLP merger（ViT + LLM 冻结）67B / 8K
S1 多模态预训练全参（ViT+merger+LLM 一起）~1T / 8K
S2 长上下文全参，加大视频/agent 数据~1T / 32K
S3 超长适配全参，主攻长视频/长文档100B / 256K

后训练三步：① 长链思考(CoT) SFT → ② 强师知识蒸馏 → ③ 强化学习(RL)。分流出 non-thinking（直答）和 thinking（显式推理，复杂任务更强）两版。

Qwen3-VL = 你学的 模态编码器→projector→LLM 在视觉上的旗舰落地（同 minimind-o/Qwen3-ASR 骨架）：眼睛 SigLIP-2(CLIP 家族 ViT) + merger + Qwen3 解码器，位置用 Interleaved-MRoPE。它也是 Ideogram-4/Qwen-Image-2.0 当文本/条件编码器用的那个大脑。一句话：VLM 就是给 LLM 接个 ViT 眼睛 + merger 插头；Qwen3-VL 靠 Interleaved-MRoPE + DeepStack + 文字时间戳把长视频和细粒度感知顶上去。

引用 / 术语表

  1. 三件套 + MLP merger · 视觉编码器(SigLIP-2 ViT) + MLP merger(2×2 视觉特征压成 1 token、对齐 LLM 维度) + Qwen3 LLM。merger = 视觉版的 projector。
  2. 动态分辨率 → 变长 token · 图按原生尺寸进，token 数随分辨率变；ViT 用 2D-RoPE + 插值绝对位置适配任意尺寸。
  3. Interleaved-MRoPE · MRoPE 给 token 标 (t,h,w)。老做法分块致频谱不均、长视频差；新做法把 t/h/w 交错铺满高低频 → 均衡 → 长程视频位置强。
  4. DeepStack · 抽 3 个中间 ViT 层、各过专用 merger、残差加到 LLM 前 3 层 → 多层视觉特征(浅细节→深语义)都进，不增上下文。同 Ideogram-4 "取 13 中间层"的思想。
  5. 文字时间戳 · 视频每组帧前缀 <3.0 seconds> 文字 token 代替 T-RoPE 的绝对时间位置 → 简单精确、利于视频定位/dense caption。
  6. 四阶段预训练 + thinking · S0 只训 merger 对齐 → S1/2/3 全参，上下文 8K→32K→256K；后训练长 CoT SFT→强师蒸馏→RL，出 non-thinking / thinking 两版。
