一张图片，怎样进入语言模型？

Qwen3-VL 不是让 Qwen3 直接读取 JPEG。图片先被视觉编码器切成特征，再由 merger 合成语言模型能接收的视觉 token；从这一步开始，图和文字才进入同一条序列。理解这条数据流，DeepStack、MRoPE、长视频和后训练才能各归其位。

Qwen Team · Qwen3-VL Technical Report · arXiv:2511.21631v2 · 2025-11-27
TL;DR
先记主线

视觉编码器负责看，merger 负责接线，Qwen3 负责理解并写答案。DeepStack 把中间视觉特征补进 LLM 前三层；Interleaved-MRoPE 让时间、行、列都覆盖快慢频率；文字时间戳让模型直接读出视频的真实秒数。真正把能力做出来的还包括约 2T 级分阶段预训练、九类数据、SFT、强到弱蒸馏、两套 RL、视觉工具训练和万人 GPU 并行。

§ 01先拼完整：从图片和问题，到一句文字答案

输入“这张图里有几只猫？”时，模型实际走四段路。视觉编码器只负责把像素整理成向量；它不会自己生成答案。答案仍由一个 decoder-only Qwen3 逐 token 写出来。

最容易混淆的一点：视觉 token 不是“猫、桌子、文字”这些离散标签，而是一串连续向量。LLM 通过训练逐渐学会这些向量和语言含义之间的对应关系。

Dense、MoE、Instruct、Thinking 是两组不同维度

§ 02图片怎样变成视觉 token：动态分辨率与 2×2 merger

“动态分辨率”不是一句画质口号。图片越大、版面越长，视觉编码器产生的网格通常越大，最终进入 LLM 的视觉 token 也越多。小字更容易保住，计算账单也跟着上涨。

Merger 到底“合并”了什么

假设 ViT 输出 12×16=192 个特征，每格都是宽度 Dv 的向量。merger 每次拿相邻 2×2=4 格，先首尾拼接，再用两层 MLP 映射到 LLM 的隐藏宽度 Dl。

12\times16=192\quad\xrightarrow{\;2\times2\text{ 合并}\;}\quad6\times8=48\ \text{个视觉 token}
Dv：视觉编码器每个格子的特征宽度。
Dl：Qwen3 LLM 的 hidden size。
2×2 merge：空间高宽各减半，token 总数变为四分之一。
MLP：两层小网络，学习视觉空间到语言空间的转换。

merger 是 模态投影器1，但比“只换维度”的 projector 多做了一步空间合并。代价也要说清：合并减少计算，却可能把很小的细节揉在一起；DeepStack 正是用中间层特征补这部分信息。

§ 03DeepStack：最后一层看语义，中间层补细节

普通 VLM 常把 ViT 最后一层输出一次性交给 LLM。问题是，视觉网络越往后越偏高级语义；小字、边缘和局部结构可能在抽象过程中变弱。DeepStack 不替换主路，而是额外从三个深度抽特征。

h_l^{\mathrm{new}}=h_l^{\mathrm{LLM}}+m_l\!\left(f_l^{\mathrm{ViT}}\right),\qquad l=1,2,3
fᵥⁱᵀ_l：第 l 个抽取位置的视觉中间特征。
m_l：该位置专用的 merger，负责数量和宽度对齐。
hᴸᴸᴹ_l：LLM 第 l 层原本的 hidden state。
+：逐元素残差相加，不是把 token 接到序列尾部。

消融使用内部 15B-A2B、200B 预训练 token、没有后训练：平均分 74.7→76.0，InfoVQA 71.9→74.2，DocVQA 89.5→91.1；TVQA 则 80.6→80.5。证据支持“整体有帮助，文档细节任务尤其受益”，不支持“每项都涨”。

§ 04Interleaved-MRoPE：三把位置尺都要有粗刻度和细刻度

M-RoPE2 不再只给 token 一个序号，而是给视觉 token 三个坐标：时间 t、行 h、列 w。RoPE 用这些位置旋转 attention 的 query 和 key，让相对时间与空间距离影响注意力。

旋转频率有快慢：慢频率转得慢，适合保留远距离关系；快频率对小位移敏感，适合分辨近处细节。原版连续切三段，每个轴只能拿到一段频率。Interleaved 改为交错分配。

报告认为它能缓解原版频谱偏置、改善长视频位置建模，但没有给单独关闭 Interleaved-MRoPE 的消融。长视频进步还同时来自时间戳、视频数据和长上下文训练，不能把全部增益算给这一项。

§ 05文字时间戳：别让模型从巨大位置编号里猜秒数

Qwen2.5-VL 的 T-RoPE3 把绝对时间绑进时间位置 ID。视频很长或采样不均时，ID 会又大又稀；训练还得覆盖很多 fps，才能学会位置编号与真实时间的对应。

它没有做到的事：时间戳只标记已经采到的帧，不能恢复采样时漏掉的动作；它解决“何时发生”，帧间发生了什么仍由视觉特征和 Transformer 学习。

§ 06四阶段预训练：先把线接通，再逐步拉到 256K

逐步拉长是训练成本上的合理安排：self-attention 需要比较 token 对，长度放大四倍，朴素配对数约放大十六倍。这是基于 Transformer 计算规律的解释；报告只明确给出了阶段配置。

报告没有给公式的地方：作者说把 per-sample loss 改成“平方根归一的 per-token loss”，目的是平衡文本与多模态样本，但没有披露精确归一对象、packing 方式和公式。这里不拿常见实现替作者补空白。

§ 07九类预训练数据：三项结构改动不会凭空长出能力

OCR、3D 定位、视觉数学和 GUI 操作不是同一种任务。报告为它们分别造数据、打伪标签和做过滤。把这部分删掉，只剩“模型结构先进”，会错过技术报告的大半信息。

数据大量来自旧模型、检测器、规则和伪标签，再经过规则或模型过滤。报告披露了流程与部分规模，没有公开完整数据清单和混合比例，因此外部团队无法只靠报告重建训练集。

§ 08后训练：SFT 教格式，蒸馏教解法，RL 纠正行为

长 CoT 数据为什么先检查“没有图还能不能答”

视觉数学题有时题面文字已经泄露答案。报告用 Qwen3-30B-nothink 在不提供图片的情况下试答；若仍能答对，就删除样本。这样留下的训练题才真的要求模型看图，而不是靠文字模式蒙答案。

蒸馏的 on-policy 到底多做了什么

Reasoning RL 最终约 30K 条 query。每题先采 16 个回答：全错的题在准备阶段被删，训练时又删掉通过率超过 90% 的简单题，留下“不是完全不会，也没有已经学会”的区间。SAPO5 是报告采用的策略梯度算法；本报告只给名称与引用，没有展开它的完整公式。

§ 09Thinking with Images：不是脑内生图，而是多轮调用视觉工具

这里的 “with Images” 指模型能根据当前证据决定下一步视觉动作，读取工具或环境返回的新证据，再继续回答。报告没有完整列出工具接口，所以不能把所有常见裁剪、放大、搜索动作都当成已公开实现。

训练先用约 10K grounding 样本把 Qwen2.5-VL-32B 做成第一代视觉 agent，再蒸馏出约 120K 多轮交互训练 Qwen3-VL。早期模型发现了奖励漏洞：无论难题简单题都只调一次工具，仍可能骗过答案与推理 judge。第三项调用次数奖励就是专门堵这个漏洞。

§ 10万人 GPU 怎样分工：五种并行切的不是同一件东西

预训练最高扩到约 10,000 张 GPU。只写“用了 Megatron-LM”没有解释力：235B 参数、256K token 和 MoE 专家分别压在不同瓶颈上，必须从不同方向切开。

训练与部署不是一套系统。报告用 vLLM 或 SGLang6 做推理：前者用 PagedAttention 管理不等长请求的 KV cache，后者侧重结构化生成、复杂 prompt 和请求调度。完整并行背景见 分布式训练并行。

1M 上下文实验到底证明了什么

排行榜还有两条必须连着看的脚注

视频横评时 Qwen3-VL 最多使用 2,048 帧；受 API 与资源限制，Gemini 2.5 Pro、GPT-5、Claude Opus 4.1 分别只输入 512、256、100 帧。作者明确承认比较不能保证完全公平。

纯文本也不是“全面反超 Qwen3”。235B Instruct 的 AIME-25 是 74.7 vs 70.3，但 MMLU-Pro 是 81.8 vs 83.0；Thinking 版 AIME-25 也低于对应文本模型 89.7 vs 92.3。更准确的说法是：加入视觉后，语言能力总体保住，部分任务因新数据与后训练得到提升。

§ 12消融与边界：哪些因果证据扎实，哪些还不能拆账

没有单独消融：报告没有分别关闭 Interleaved-MRoPE、文字时间戳或平方根归一损失。长视频结果同时受结构、数据、采样和上下文扩展影响，不能把全部进步归给一个零件。
01数据不可完整复现内部收集、合成、伪标签与具体混合比例没有全部公开。
021M 不是原生训练长度原生训练到 256K，1M 来自 YaRN 位置外推。
03动态高分辨率很贵官方示例可达 11,427 个视觉 token，prefill 和 KV cache 都会增长。
04部分评测依赖模型 judge开放回答和 VideoMMMU 等口径会继承 judge 偏差。
05推理预算必须分开Thinking、工具调用与纯 Instruct 不是同等成本。
06训练目标披露不完整平方根归一 per-token loss 没有精确公式与 packing 细节。
一句话带走：Qwen3-VL 的主线不是三个新名词，而是一套完整系统——动态分辨率 ViT 把图变成 token，merger 降采样并接入 Qwen3，DeepStack 补回中间视觉细节，位置与时间机制服务长视频，再靠专门数据、蒸馏、RL 和分布式训练把能力真正做出来。

术语表 · 点正文角标即可回看

  1. Modality projector / merger：把视觉特征改成 LLM hidden size 的小网络。Qwen3-VL 的 merger 还会把相邻 2×2 特征合成一个 token。
  2. M-RoPE：多模态旋转位置编码。用时间、行、列三个坐标表示视觉 token 的位置。
  3. T-RoPE：Qwen2.5-VL 把视频绝对时间绑入时间位置 ID 的方案；Qwen3-VL 改用文字时间戳。
  4. KL divergence：衡量两组概率分布差异。蒸馏时让学生对每个候选 token 的概率靠近老师。
  5. SAPO：Smooth and Adaptive Policy Optimization，本报告 Reasoning RL 使用的策略梯度算法；详细目标函数来自其独立论文。
  6. SGLang：LLM / VLM 推理服务框架，负责请求调度、KV cache 和结构化生成等工程工作。
