不追一种
标准好看

Krea AI · 2026-06-23 · K2 Raw + K2 Turbo 开放权重

主流模型都在卷"单一默认审美"并卷到饱和。Krea 2 反着来：数据上别把不讨喜的图筛掉、一张 AI 合成图都不掺，后训练上用 STPO + 多奖励 rubric，把审美空间撑宽。

TL;DR

现在的文生图模型技术都很强，但都收敛到一小撮默认审美（一看就"AI 味"）。Krea 2 主张图像生成应是探索性媒介——多样且可控。做法分两头：数据端不按美学分过滤（那会把模型口味焊死）、全程 0 AI 合成图4；训练端走六段流水线，自创 STPO（稳住 DPO 的偏好优化）+ 多奖励 GRPO 配 prompt 级 rubric 奖励，最后用 TDM 蒸出少步快出的 K2 Turbo。定位：Artificial Analysis 文生图榜前十、独立实验室第二。

§ 01不是"更好看",是"更多样"

这几年文生图一代比一代强，可它们在审美上越长越像——高饱和、强对比、油润讨喜的那种"AI 味"。Krea 的判断是：技术指标饱和之后，把所有 prompt 都优化成同一张"精致默认图"是走窄了。图像生成应该是探索性的媒介1：表达力要广（能横跨很多种审美），还要可控（创作者能在里面导航）。所以这篇报告的目标函数不是"一种好看"，而是"撑开一个能探索的审美空间"。

§ 02架构：一套标准的现代 MMDiT

骨架本身不猎奇，是一套调到位的现代配置：多模态扩散 Transformer（MMDiT / DiT），用 flow matching 训。文字进来由 Qwen3-VL 编码（多层特征聚合）；图像在 latent 空间生成，再经 VAE 解回像素。注意力用 GQA + 门控 sigmoid，归一化用零中心 RMSNorm + QKNorm，MLP 是 SwiGLU 4×，位置编码 3D 轴向 RoPE6。差异化不在这层，在数据和后训练。

真正费功夫的是把这套标准配置训稳、训快。几个不显眼但实用的选择：256px 的第一个 epoch 上 iREPA 加速收敛，之后撤掉让模型长自己的表征；低/中分辨率用 8-bit 训练（256px tensorwise、512px rowwise，1024px 起回 bf16），换来 15–20% 提速且几乎不掉点；每个分辨率单独扫一个时间步偏移（shifted logit-normal）的最优值，但推理偏移保持不变；学习率走 warmup-stable-decay，权重平均用 PMA（参数合并近似）替 EMA——效果相当但省下一大笔显存。

§ 03数据哲学：别在源头把多样性筛没了

这是全篇的灵魂。业界常拿美学打分器（aesthetic score）和画质评估（IQA）去筛数据、按分数挑"更美的"。Krea 反对3：这些打分器自带口味，拿它塑造数据分布等于把它的偏见焊进去——"不讨喜但真实"的图被系统性压低，模型从没见过、以后也画不出来，审美就这么收敛了。Krea 把关口从"好不好看"换成"caption 准不准"：只要配文准确描述了图，哪怕图本身不讨喜也留着。

注意这里有个分寸——Krea 不是完全不过滤：低分辨率阶段用便宜的 CPU 过滤（拉普拉斯、熵之类）剔废图，高分辨率阶段也会用画质/美学过滤，但只用来删掉"极差"的，绝不用来 oversample"更美"的。差别在意图：删极差 vs 挑更美。再加一条硬规则——预训练里一张 AI 合成图都不掺4。详见 generative-data-curation。

为什么 0 AI 数据这么硬：合成图更好学（本就来自更简单的分布），模型会优先把 loss 降在这些"软柿子"上、慢慢往合成分布塌，相当于给质量封了个顶。用模型的输出喂模型，是给后代近亲繁殖——每代更"好学"、更窄。

§ 04配文与覆盖：喂进去的不是图，是图文对

留下哪些图只是一半，另一半是怎么描述它——训练信号是图文对，配文不准等于信号脏。Krea 的配文是条流水线：先 OCR 抽出图里的可见文字，再让一个 captioner 结合 OCR 结果 + 元数据（相机参数、已知实体）写出信息密集的长描述，最后用一个便宜 LLM 把它改写成不同长度/格式。训练以长 caption 为主（密集监督 → 收敛更快、loss 更低），但也喂短/中 prompt，让模型推理时短句也能用。

预训练是"自下而上"广撒网；之后的中训反过来"自上而下"补覆盖：用 FAISS 做层级 k-means 聚类、让 VLM 审查每个簇心附近的图，用 SigLIP 相似度做语义去重，再用维基百科 PageRank 排序（取 top 90%、对照 Wikidata）保证约 500 万个概念都被覆盖到——常识实体别有盲区。SFT 阶段则是小而精的人工集：量到一定程度后，质量远比规模重要。机制详见 SigLIP 语义去重 · 层级 k-means 策展 · PageRank 实体覆盖。

§ 05六段训练流水线

差异化的另一半在后训练。从随机权重到能少步出图，Krea 2 走六段，每段各司其职：先把广度铺开（预训），再补域覆盖（中训），然后用小而精的人工集定调（SFT），接着两道对齐（偏好优化 PO → 多奖励 RL），最后压步数（蒸馏）5。

§ 06STPO：堵住 DPO 的偷懒解法

偏好优化用 DPO7：拿"赢/输"成对样本，直接让模型更偏向赢家。但 DPO 只盯"赢减输"的差——模型能学会一个偷懒解法：把赢家、输家概率一起往下压，只要把输家压得更狠，差距照样拉大、loss 照样降。结果赢家概率也掉了，"为了拉开差距把好的也练差了"，这叫策略发散。Krea 的 STPO（报告没给全称）加一个辅助损失 + 改 DPO 公式，盯住赢家的绝对概率别掉，把这条偷懒路堵死。

取 β=1，参考模型对赢/输初始都给 logπ_ref = −2.0，隐式奖励 r = logπθ − logπ_ref 赢家 logπθ r(赢) 输家 logπθ r(输) 差=r赢−r输 σ(差) loss 好的一步 −1.5 +0.5 −3.0 −1.0 1.5 0.818 0.201 ✓ 赢家↑ 偷懒一步 −2.5 −0.5 −4.5 −2.5 2.0 0.881 0.127 ✗ 赢家↓ 但 loss 更低！ → DPO 只看"差"，偷懒步（赢家从 −2.0 掉到 −2.5）反而 loss 更小，它"更满意" → STPO 额外惩罚"r(赢) 掉成负数"，逼模型真把赢家顶上去，而不是靠压低赢家凑差距

✓ 自检：偷懒步的 loss（0.127）比好步（0.201）还低，可它把好图的概率练低了——这正是只看相对差的盲区。STPO 给赢家绝对概率上了把锁。详见 direct-preference-optimization。

§ 07多奖励 RL：四个裁判 + 把 prompt 拆成可验证项

RL 阶段用多奖励 GRPO 式方法9（GRPO：同 prompt 一组采样比相对好坏当 advantage，省掉 value 网络）。四个奖励模型一起推：通用美学 / 提示遵循 / 文字渲染 / 瑕疵与结构。其中提示遵循用 prompt 级 rubric 奖励10——借鉴 rubric-based evaluation，不让判官给一个笼统总分，而是把每个 prompt 拆成可验证的要求逐条判。整个 RL 不开 CFG11：让采样（rollout）和训练分布对齐、不额外开销，推理时再开。

§ 08Prompt Expansion：把短输入扩成富 caption，还得防塌成一种风格

模型训在信息密集的长 caption 上，可用户输入往往短、模糊、各人习惯还不一样。所以前面挂一个 prompt expander14 把短输入扩写成模型爱吃的富 caption。它也分两段训：

SFT：用另一个 LLM 从长 caption 反向造"用户 caption"（短、口语、半指令、故意省细节）配成训练对；还合成思考轨迹（中间一步"重建意图"能提升下游表现）。再做分布塑形——oversample 艺术性图扩大创意覆盖、给该写实的 prompt 加轻量"摄影介质"偏置；目标不是强加 house style，而是让扩写分布同时覆盖"艺术导演范"和"直白写实"两端。

RL：目标从"模仿目标 caption"转成"扩写要提升出图质量、同时保住用户意图"。用 GDPO（Generative DPO，DPO 的生成式变体）配多奖励：图级（出图质量/偏好）+ prompt 级可验证奖励（扩写忠不忠于原请求）+ 安全闸（稀疏约束，别引入没请求的/不可接受内容）。

这里有个必须正面对抗的失败模式——多样性塌缩：expander 会偷偷学会一种"安全、高奖励的 house style"（尤其当图级奖励占主导）。对策是加一个 DINOv3 嵌入多样性分，对同一组 prompt 的组内视觉多样性给奖励。报告说一旦这个权重调小，模型很快塌向单调；整个训练全程都得让多样性奖励保活才压得住。RL 的 prompt 还混了硬例（内部踩、bug 报告、手改 prompt，按失败类分桶），挑"难但不绝望"的——综合分中等、有真实方差，GDPO 才拿得到有效偏好信号。机制详见 prompt-expansion（含 GDPO 凑对流程）与 DINOv3 多样性奖励。

§ 09Style Reference：只借风格，别把人家的内容也搬来

风格参考让用户用文字生成、再拿一张或多张参考图导风格。Krea 想做到三点：多个风格平滑语义混合、每个参考强度连续可调、复杂风格的贴合度 SOTA。难就难在——"风格"和"内容"本来就没有清晰边界，最常见的翻车是内容泄漏15：参考图里的主体/物件直接钻进了结果图。数据还稀缺：风格迁移数据不像编辑数据那样能从视频里大量挖。Krea 的做法是一套新的自监督训练专训风格参考模块、再接一道偏好优化对齐，专门把内容泄漏压下去。机制详见 style-reference。

§ 10蒸馏与发布：TDM 压步数，出 K2 Turbo

最后压步数。Krea 比过 DMD / DMD2 / Decoupled DMD / piFlow / APT，选了 TDM（轨迹分布匹配）12，理由是好调、超参少。和只匹配终点分布的 DMD 不同，TDM 沿去噪轨迹的多个点都匹配分布，所以少步更稳、步数能灵活选。发布两套权重：K2 Raw（未蒸馏基座，给做微调/后训练的）和 K2 Turbo（引导蒸馏 + 时间步蒸馏双管，少步快出图）。

§ 11系统：在这种规模下，训练是怎么崩的

报告罕见地把基础设施摊开讲，而且很坦诚。调度用 Kubernetes 上的 Kueue（gang-scheduling 协调多机 + 借/借出/回收配额）；研究把 GPU 占满时，推理靠 Virtual Kubelet 自动外溢到集群外的别家算力；启动流水线自动避开坏节点，还有个叫 Packerman 的 operator 把 dev 这类非关键负载塞到坏节点上、给训练保住好节点。

最有料的是 observability 的几条反直觉结论16（全靠 DCGM 指标攒出来的）：GPU 温度过 75–78°C 就开始 throttle 拖累稳定；GPU 利用率会骗人——崩掉的卡反而显示 0%，所以他们改看张量核利用率当训练是否正常的首选信号；PCIe replay 突增常常是崩溃的前兆；而头号崩溃原因是 InfiniBand fabric 不稳（链路抖动、丢包、拥塞）。更现实的教训：很多崩溃是瞬态的——同样代码同样数据，两小时内反复崩，啥都不改重启就能稳跑 16 小时；128 卡以下很稳，可 GPU 一翻倍不稳定性涨得远超预期；在很大规模上，没有一次 run 跑过 24 小时不崩，而且常常无声无息（NCCL 超时，指标却全绿）。文件系统从 Ceph 换到 Weka 后故障骤降、性能同步提升，checkpoint 约 30 秒搞定。

数据仓他们自己写了个叫 krablet 的系统：PostgreSQL 分片 + "漏斗"服务异步批量化写入降锁争用，撑到 208 TB 元数据、每秒数万次竞争 UPSERT。最妙的是不上 Kafka/Ray/Spark，而是把 DAG 工作流当成数据库队列跑（靠 FOR UPDATE SKIP LOCKED）：失败的行自动排到队尾重试、worker 随时崩都不丢进度、动态加 worker 不用 reshard——用一句 SQL 换掉一整套流处理框架。

§ 12未来：把一堆碎片合成一个

报告结尾把路线图也写了。Scaling：上 MoE、用稀疏注意力做 native 2K–4K、用 NVFP4 预训练、换 Muon 优化器——并自认现在的模型训得还不够（undertrained）。两个更结构性的方向17：

MOPD（多教师 on-policy 蒸馏）：先训各领域的专家，再把它们的能力通过密集监督蒸进一个学生。好处是避免"调好这域、搞坏那域"的跨域能力打架，还能让不同 RL 团队各管一域、互不 regress——本质是把模型训练组织层面并行化。架构简化：现在要一串相互依赖的模型（VAE + DiT + 文本编码器 + prompt expander）才能跑，目标是把它们合成一个统一模型，让研究像 LLM 那样能拆开并行推进。再加新能力：稳健编辑、图像参考、native 2K/4K——因为用户早就在用自然语言/标签/JSON/bbox/指令/Markdown 各种花样下 prompt，模型得原生看懂。

§ 13定位：饱和之后比的是"多样"

Krea 2 在 Artificial Analysis 文生图榜前十、独立实验室里第二13。但这报告真正的赌注不在某个榜的名次，而在一套逆向品味：当画质和对齐都卷到天花板，差异就只剩"真实感"和"创意/多样"——而这两样必须在数据端就保住（别筛掉尾巴、别掺合成图），再靠后训练（STPO + 多奖励 rubric + 防风格塌缩的多样性奖励）轻轻拨向人类口味，而不是把它优化成一种讨喜默认。

Krea 2 和 Qwen-Image-Bench 是一体两面：后者在评测端量"创造性/多样性"（对齐饱和后差异在真实感和创意），Krea 2 在生产端保住这份多样性的来源——数据别筛掉尾巴 + 0 合成图。它跟 FLUX / SD3.5 / Qwen-Image（同走 MMDiT+flow matching，主干谱系上溯 DiT）骨架同源，差异全在数据哲学和后训练：STPO（治 DPO 的策略发散）、多奖励 GRPO + rubric 奖励、TDM 蒸出 K2 Turbo。一句话：当大家都在把图变得更讨喜，Krea 在确保"不讨喜但真实"的那部分别在源头就被删掉——多样性是要在数据端守住的。

  1. 探索性媒介 · Krea 的产品哲学：图像生成应表达力广（横跨多种审美）+ 可控（创作者能导航），而非把一切优化成同一张精致默认图。
  2. 审美收敛 · 技术饱和后顶尖模型画得越来越像（高饱和/糖水感）。Krea 2 的整个设计就是对抗这件事。

  4. 0 AI 合成图 · 预训练一张合成图都不掺。合成图"更好学"，会把模型往简单分布拽、给质量封顶（近亲繁殖）。
  5. 六段流水线 · 预训(256→512→1024) → 中训(~500 万维基概念) → SFT(小精人工集+模型合并) → 偏好优化 → 多奖励 RL → TDM 蒸馏。
  6. 架构配置 · MMDiT + flow matching；GQA + 门控 sigmoid 注意力；零中心 RMSNorm + QKNorm；SwiGLU 4×；3D 轴向 RoPE；文本编码 Qwen3-VL。
  7. DPO · 直接偏好优化 · 跳过奖励模型，拿赢/输成对样本直接对齐；隐式奖励 = β·log(πθ/π_ref)。
  8. STPO（策略发散修正） · Krea 自创的 DPO 变体（无全称）：加辅助损失盯住赢家绝对概率别掉，堵住"赢输一起压低凑差距"的偷懒解法。
  9. 多奖励 GRPO · 同 prompt 一组采样比相对好坏当 advantage（省 value 网络）；四奖励：通用美学/提示遵循/文字渲染/瑕疵结构。
  10. prompt 级 rubric 奖励 · 提示遵循不给笼统总分，把每个 prompt 拆成可验证要求逐条判，借鉴 LLM 训练里的 rubric 评测。
  11. RL 不开 CFG · 让采样(rollout)与训练分布对齐、省开销；classifier-free guidance 留到推理时再开。
  12. TDM · 轨迹分布匹配 · 沿去噪轨迹多个点都匹配分布（不只终点）→ 少步稳、步数灵活；从 DMD/DMD2/piFlow/APT 里选出，好调超参少。
  13. Artificial Analysis 定位 · 文生图榜前十、独立实验室中第二。报告原话的对外可观察定位。
  14. Prompt Expansion + 多样性塌缩 · 把短用户输入扩成富 caption 的前置模型；SFT+RL(GDPO 多奖励) 训。坑：会塌成一种高奖励 house style，靠 DINOv3 组内多样性奖励(全程保活)压住。
  15. 内容泄漏 · Style Reference · 风格参考的核心难题：参考图的主体/内容钻进结果图。Krea 用新自监督训练 + 偏好优化压低泄漏，支持多风格混合 + 强度连续可调。
