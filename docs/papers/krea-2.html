01 · The thesis它反对的不是“好看”，而是“只剩一种好看”

今天的文生图模型越来越可靠：结构稳、文字清楚、提示遵循也强。但越可靠，默认输出反而越像——高饱和、强对比、表面精致。Krea 把这个现象叫作审美收敛01。

对生产工具来说，一个稳定的默认风格很省事；对创作工具来说，它却会把探索空间压扁。创作者需要的不是每次都拿到“最稳妥的成片”，而是能在风格、情绪、构图和材质之间搜索。于是 Krea 把目标拆成两个互相拉扯的量：

Fig 01 · 宽广的分布与可导航的控制DIVERSITY × CONTROL
只优化一个审美打分器
→ 不同区域被挤成同一种默认输出
表达力：覆盖足够多的视觉概念可控性：准确得到目标结果
写实平面设计低保真时尚摄影插画实验影像

表达力决定模型能生成多少种视觉概念和风格，可控性决定结果能否准确符合用户意图。只追前者，模型变化很多但不听话；只追后者，模型听话，却总是生成相似的成片。Krea 2 的数据、后训练和产品接口，都围绕这对矛盾展开。

02 · Data不是“完全不过滤”，而是别拿一种口味统治数据

常见做法是先给每张图打美学分或画质分，再多采样高分图。问题是：打分器本身就有偏好。运动模糊可能被判成糊，低对比可能被判成不清楚，可它们也可能是有意的视觉语言。用一个打分器长期塑造训练分布，相当于让它替模型决定世界上什么值得存在。

按“好不好看”挑

  • 高分图反复出现
  • 长尾风格逐渐消失
  • 打分器偏见被写进模型
  • 最后不同 prompt 都长得相似
VS

按“能不能学明白”留

  • 去重复和过度代表的概念
  • 去掉 VLM 持续描述不准的样本
  • 去掉有害偏见、伪影与低分辨率难建模样本
  • 只要配文准确，不因“不讨喜”而删除

这里必须讲清两个限定。第一，Krea 在高分辨率阶段仍会用画质和美学分，但只删极差图，不按分数过采样“更美”的图。第二，报告说的是预训练图像混合不含 AI 生成图02，不是整条研发链都不用合成数据。

“0 AI 图”只说预训练

后面的偏好优化会造合成偏好对；prompt expander 也会造短用户 caption 与 thinking trace。它们属于后训练或另一个模块的训练，不是预训练图像混合。

“更好学”为什么反而危险

合成图往往来自较简单、较规则的模型分布，损失更容易下降。只要混入少量，模型也可能优先朝这块简单分布收缩，Krea 认为这会给最终质量设上限。

低分辨率预训练有数十亿级图像，昂贵 VLM 不可能逐张跑，于是先用文件、尺寸、宽高比、Laplacian16、RGB entropy、黑白像素比例等便宜过滤器。需要识别特定伪影14时，再用大 VLM 造伪标签，训练小于 1B 的 DINOv3 / SigLIP-2 分类器大规模执行。去重则组合 MD5、12×12 pHash 与 colorhash15；默认 8×8 pHash 不看颜色，误杀太多。

伪影：算法额外制造的错误，不等于所有“不好看”

伪影是图像采集、压缩、处理或生成过程额外制造出来的错误，不是画面本来想表达的内容。Krea 报告提到三类具体问题：预训练数据曾诱发纯色背景和边框错误；偏好优化的分布漂移会在后续训练中表现为高频伪影；RL 还要识别多指、畸形肢体和扭曲文字等结构错误。

高频伪影棋盘格、振铃光晕、散乱亮点；颜色或亮度在几个像素内反复突变。
结构伪影多指、肢体接错、文字扭曲；图像局部看似合理，整体结构却不成立。
不一定是伪影运动模糊、胶片颗粒、低对比可能是有意的视觉语言，不能只凭“糊”或“有噪点”删除。

pHash：先提取结构指纹，再数有几位不同

pHash 是“看起来像不像”的图像指纹，和 MD5 不同：同一张图即使缩放、轻微压缩或调整亮度，pHash 仍应比较接近。下面是常见 pHash 实现的流程；Krea 只披露使用 12×12 pHash，没有公开自己的代码。

DCT：不再逐个记像素，改为记录不同尺度的明暗变化

DCT 全称离散余弦变换。它不负责缩小图片，而是换一种方式记录同一张图：输入按“每个像素有多亮”记录，输出按“不同尺度的明暗变化各有多少”记录。这里的频率不是播放速度，而是亮度在空间里变化得有多快。

低频 · 变化慢大片区域缓慢变亮或变暗，主要对应整体亮度、大色块与粗轮廓。
高频 · 变化快相邻像素反复明暗切换，主要对应细边缘、文字、纹理与噪点。

pHash 只保留低频，是为了比较主要构图，同时尽量忽略 JPEG 压缩、少量噪点和轻微锐化带来的高频变化。12×12 pHash 的常见流程同理：48×48 灰度图 → DCT → 左上 12×12 → 144 位指纹。Krea 没公开它是否采用同样的缩放倍数。

默认 8×8 只有 64 位，结构相近的不同图片更容易碰巧得到相近指纹；它又只看灰度，一张红色海报和版式相同的蓝色海报也可能很接近。colorhash 则把图转到 HSV，统计黑色、灰色和不同色相各占多少：它擅长看整张图的配色分布，却基本不记录颜色位置。因此 12×12 pHash 补结构细节，colorhash 补颜色。Krea 没公开二者的判重阈值和合并规则，不能断言它用逻辑与还是加权距离。

这里的 Laplacian 是图像的二阶差分，不是概率论里的拉普拉斯分布。常见 3×3 核会对每个像素计算“上下左右之和 − 中心像素×4”：

03 · Captions & coverage模型吃进去的不是图，而是“图 + 对这张图的解释”

一张图留不留下，只决定模型能不能见到它；caption 写得准不准，才决定模型能不能把视觉现象和文字概念绑在一起。

图片侧OCR 先抽可见文字，再加入相机参数、已知实体等元数据。
→
Captioner把 OCR + 元数据 + 图像合成信息密集的长描述，再由便宜 LLM 改写成多种长度与格式。

长 caption 提供的监督更密，报告观察到它能让收敛更快、训练 loss 更低；但真实用户会写短句，所以训练仍保留短、中 prompt。这个折中很重要：长描述教会模型世界里有什么，短描述保证用户不必写论文才能出图。

预训练先广泛学习数据中的视觉概念，中训再有针对性地补足能力覆盖04：团队用 FAISS 跑层级 k-means，先把图像空间分成大类，再继续拆小簇，防止头部概念吃掉算力。VLM 查看每个簇中心附近的代表图并给簇命名或标记，人工再复核被标红的簇；簇内用 SigLIP 相似度做语义去重。

过滤阈值会随阶段变化

低分辨率先用便宜规则清掉极端样本；升到高分辨率后，才更关心画面复杂度、文字密度和最低画质。阈值不是一张固定清单，而是随 256/512/1024 训练目标调整。

Wikipedia 还要先筛一遍

英文 Wikipedia 条目先按 PageRank 排序并保留前 90%，再结合 Wikidata 去掉难以通过图像表达的主题。剩余约 500 万个概念才进入 caption 全文检索、稀有实体保护和采样后复查。

视觉聚类更容易把“都穿球衣、都在球场上”的图片放进同一个运动员簇，却不保证每位具体球员都能留下。假设这个簇有 10,000 张图片，其中某位知名球员只有 10 张：

注意力怎么选：MLA 略好，GQA 更合算

这里比较的不是“有没有注意力”，而是每个注意力头怎样保存和共享 K/V 表征。GQA 让多组 Query 共用较少的 K/V 头；MLA 则先把 K/V 压进一个更紧凑的 latent，再投影回来。低分辨率消融里 MLA 略有优势，但计算更贵。图像扩散在推理时每一步都会整段重算，没有自回归 LLM 那种长期复用的 KV cache，因此 MLA 最擅长的“缓存压缩”在这里兑现不了多少。团队最终选择了质量损失很小、工程路径更成熟的 GQA。

SwiGLU 替代 GeLU

MLP 扩展倍数保持 4×，只把激活函数换成带门控的 SwiGLU。不同规模下都更好，而且已有成熟的高效 kernel，所以这是一项几乎没有额外工程负担的改动。

Sigmoid 门控主要换稳定性

gated sigmoid attention 没带来显著画质跃升，但 loss 和梯度范数更平稳。对一次要跑很久的大模型训练，少量参数换来更少的训练抖动，本身就是收益。

单流、双流、混合流

单流让图文 token 共用注意力与 MLP；双流只在注意力里交互，其他权重分开；混合流前 1/3 用双流、后 2/3 合并。混合流略好，但差距很小，最后选择结构更简单的单流。

标准 residual 胜在没有副作用

Laurel 在 residual 支路加入低秩瓶颈，理论上能增加表达能力，但消融没有看到明显改善。团队因此保留普通残差连接，把 NOBLE、mHC 等更激进方案留给以后。

时间步条件不是“告诉模型现在第几步”这么简单

同一个带噪 latent 在不同时间点含义不同：靠近噪声端时，模型应先决定大构图；靠近图像端时，模型主要修纹理。时间步条件18就是把这个“当前噪声程度”送进每个 block。常见 AdaLN 做法是：每个 block 都用一套 MLP，根据时间步产生 scale、shift 和 gate；在大模型里，这些调制 MLP 合计可占 20–30% 参数。

完全删掉时间条件

省参数最彻底，但模型不知道眼前状态处于轨迹哪一段，效果明显变差。这说明时间信息不能省，只能换一种更便宜的注入方式。

把时间写成 token

团队试过把正弦时间编码做成 4–16 个 token，与文字、图像 token 一起参加注意力。256px 可用；升到 512/1024 后落后，而且继续增加 token 收益递减。

最终：每块一个可学习 bias

保留时间条件，但不再让每个 block 都养一套大 MLP。这样把省下的容量给主干 Transformer，同时避免高分辨率下 time token 传不进去的问题。

这项消融告诉我们什么

低分辨率小实验只能证明“方案能跑”，不能保证它随 token 数和分辨率一起扩展。Krea 在 256px 看见 time token 可行，却在 512/1024 主动推翻了它。

位置编码：一次漂亮的零样本结果，后来被长期训练推翻

最终采用 3D axial RoPE：把每个注意力头的维度分给帧、纵向位置和横向位置；纯图片只有一帧，文字 token 的三轴位置设为 0。团队还比较了 Golden Gate、MRoPE、归一化 RoPE 和 partial RoPE。低分辨率下都没有显著优势。

最值得记的是 partial RoPE：它只旋转注意力头前一半维度。把 256px 模型直接零样本拉到 512px 时，它看起来更好，也更少出现物体重复；但真正继续进行高分辨率训练后，标准 3D axial RoPE 反而超过它。短期外推表现和长期可训练性不是一回事。

VAE、文本编码器、归一化与优化器：主干外面的四个上限

VAE 决定细节上限

团队以 FLUX.1 AE 为起点，比较 Qwen Image VAE、DC-AE、FLUX 2 AE 和内部 VAE。DC-AE 可做到最高 32× 空间压缩，但重建误差会直接封住细节上限；Qwen 与 FLUX 2 收敛更快、重建更稳。内部 VAE 加了 DINOv3 语义对齐和轻量扩散损失，表现有竞争力，但时间有限，主模型采用已验证的外部 VAE。

Qwen3-VL 不只取最后一层

T5-XXL 在对比中仍很强，最终选 Qwen3-VL 是为了更丰富的图文输入与多语言能力。模型用浅层注意力汇总多层隐藏状态，让 DiT 自己选择“词面细节”还是“高层语义”；再加一个轻量双向 Transformer，缓解自回归 LLM 只能看左侧 token 的偏置。

归一化追求便宜且稳定

LayerNorm 换成 RMSNorm 几乎不伤质量；最终使用 zero-centered RMSNorm，并对可学习参数施加 weight decay。更省计算的 Derf 在实验中有不可忽略的退化，因此没有采用。

Muon 不是无条件优于 AdamW

团队通过 Dion 实现 Muon19，并用 Moonlight 的 RMS matching 对齐更新尺度。它早期更快，长训却更差且会出现 loss/梯度尖峰。排除 MMDiT 第一层和最后一层线性层、再加入 Nesterov momentum 后，它才在低分辨率和高分辨率上稳定超过 AdamW。由于验证时间不足，最新预训练仍用 AdamW。

容易画错的一点

报告汇总表把 Qwen Image VAE 与 FLUX 2 VAE 都列进最终实践，但正文说的是早期放大模型先用 Qwen Image VAE，较大模型后来采用 FLUX 2 VAE。不是一次前向把两套 VAE 串起来。

05 · Rectified flow模型到底在预测什么？先把一条直线学会

报告只用一句话写过：最终模型用标准 rectified-flow loss 和 v 参数化。下面把这句话展开。按最常见的记号，先在真实图像 latent x₀ 和随机噪声 ε 之间连一条直线：

Step 1 · 在图像与噪声之间取一个中间点
x_t=(1-t)x_0+t\varepsilon
Step 2 · 目标速度就是这条直线的方向
v^*=\frac{\mathrm d x_t}{\mathrm d t}=\varepsilon-x_0
Step 3 · 让模型在任意中间点都预测对方向
\mathcal L_{\mathrm{RF}}=\mathbb E_{x_0,\varepsilon,t}\!\left[\left\lVert v_\theta(x_t,t,c)-v^*\right\rVert_2^2\right]
\left\lVert [0.4,-0.2]\right\rVert_2^2=0.4^2+(-0.2)^2=0.20
x₀真实图片经 VAE 压缩后的 latent；下标 0 表示图像端，不是乘以 0。ε与 x₀ 形状相同的随机噪声，通常从标准高斯分布采样。t ∈ [0,1]直线上的位置；t=0 是图像端，t=1 是噪声端。xₜ按当前 t 混出的中间状态；下标 t 表示“时刻 t 的 x”。dxₜ/dtt 增加一点时，xₜ 每单位会变化多少，也就是速度。v*正确的目标速度；星号表示训练答案，不是乘法。c文字条件，也就是 prompt 的编码。θ12B DiT 的全部可训练参数。vθ(·)参数为 θ 的 DiT 输出的速度预测；括号里是它收到的输入。LRFRectified Flow 损失；越小表示预测速度越接近目标。E[·]对很多组 x₀、ε、t 求平均；下标列出随机抽取的量，E 不是另一个网络。‖z‖₂²按“下面的 2，再上面的 2”读：下标 ₂ 指 L2 范数，先开平方；上标 ² 再平方，正好消掉平方根，最终就是每个元素的平方和。

整句翻译：随机取一张图、一个噪声和一个时间点，让模型预测速度；算预测速度与正确速度每个元素的平方误差，再对大量样本取平均。

一个从头算到底的标量例子（真实模型里每个量都是高维张量）
给定x₀=2
ε=−1
t=.25
混合xₜ=.75×2
+.25×(−1)
=1.25
目标v*=ε−x₀
=−1−2
=−3
损失若 v̂=−2.6
L=(−2.6+3)²

v̂ 读作“v hat”，帽子表示估计值；这里就是 vθ(xₜ,t,c) 的数值。模型预测得比目标“向左少了一点”，梯度就把 −2.6 往 −3 推。推理时从噪声端出发，按模型给出的反方向多次积分，最后回到图片。

这套公式是标准 rectified flow 的解释，不是 Krea 独创。Krea 的贡献在于用什么架构、数据和后训练把这条速度场训得更宽、更稳、更可控。

预训练不是把同一配方放大三次

256px · 先学世界的粗结构大部分训练 FLOPs 花在低分辨率。第一轮临时加入 iREPA，让视觉表征更快成形；之后移除，避免模型长期被外部表征目标牵着走。8-bit 训练采用 tensorwise scaling：一个张量共用缩放尺度。LOW COST · WIDE COVERAGE

iREPA 的作用可以理解成“训练初期给表征一个方向提示”，不是永久增加一条生成条件。报告也试过 TREAD，但在其设置下收益有限。两者都属于训练辅助项，不改变最终推理结构。

同一个时间采样分布，不能原封不动跨分辨率

训练时不会平均抽所有噪声强度，而是用 shifted logit-normal 分布调整更常看到哪些时间点。分辨率升高后，图像 token 和细节结构变多，团队会增大 shift，并在每个分辨率重新搜索最合适的训练 timeshift；为了让不同 checkpoint 的产品行为可比较，推理 timeshift 保持固定。也就是说，训练时可以改变“课程重点”，上线时不随意移动用户的控制旋钮。

Warmup–stable–decay

学习率先从小值升上去，长时间保持稳定，最后再衰减。这样既避免开局更新过猛，也把大部分训练预算放在稳定学习阶段。

PMA 代替常驻 EMA

PMA11把若干预训练 checkpoint 合并，效果接近 EMA，却不用训练全程在显存里维护一套影子权重。不同合并方法差别不大；增加 checkpoint 数量、缩短采样间隔，对下游有轻微帮助。

中训负责最后一次“补课”

在进入审美对齐前，集中补高分辨率、特定领域、世界知识和文字渲染。报告把它视为最后一个适合新增基础能力的阶段，因为后续目标会越来越偏向取舍和控制。

SFT 小数据也能改默认风格

每个领域只用小规模人工精选数据。数据量够用后，质量远比继续堆规模重要。它重点修正早期 checkpoint 的高饱和与纹理问题，再把不同领域 checkpoint 合并成通用模型。

模型合并越到后期收益越小：SFT 初期的领域模型还像是在补不同短板；继续经过 PO/RL 后，每个模型都沿着自己的奖励方向移动，参数差异不再只是可叠加的“技能”，直接平均更容易互相抵消。

07 · Preference optimizationDPO 为什么能把赢家和输家一起压低？

偏好数据给同一个 prompt c 配两张图：人更喜欢的 y_w 与不喜欢的 y_l。DPO 不要求单独训练 reward model，而是比较当前模型与冻结参考模型对这两张图的相对偏好05：

标准 DPO 目标（用来解释 Krea 观察到的 divergence）
\mathcal L_{\mathrm{DPO}}=-\log\sigma\!\left(\beta\left[\Delta_\theta-\Delta_{\mathrm{ref}}\right]\right)
\Delta_\theta=\log p_\theta(y_w\mid c)-\log p_\theta(y_l\mid c)
\Delta_{\mathrm{ref}}=\log p_{\mathrm{ref}}(y_w\mid c)-\log p_{\mathrm{ref}}(y_l\mid c)
c两张图共用的 prompt。yw / yl偏好图 winner 与落选图 loser；w、l 只是英文缩写。pθ(y | c)当前模型在已知条件 c 时生成 y 的条件似然；竖线 | 读作“在已知……的条件下”。θ / refθ 是当前模型的可训练参数；ref 是冻结参考模型，不参加这一步更新。log自然对数。它把似然的除法变成减法；值越大或越不负，模型越偏向该图。Δ读作“差值”。Δθ 与 Δref 分别是当前模型和参考模型眼里的 winner−loser 差距。β > 0人为设置的偏好强度系数，不是学习率；负责放大或缩小差距变化。σ(z)sigmoid：1/(1+e−z)。z 是输入的差距分数，e≈2.718 是自然指数的底数；输出落在 0–1。−log(·)把“越接近 1 越好”改成要最小化的损失；−log(1)=0。LDPO最终要最小化的标量损失。

整句翻译：先算当前模型比参考模型多偏爱 winner 多少，再把这个改善映射成 0–1 的成功程度；改善越大，损失越接近 0。

关键是：这个式子只看差值，不单独检查 winner 自己有没有变得更可能。假设参考模型对两张图的 log-likelihood 都是 −2，因此 Δref=0：

直觉上好的更新winner: −2 → −1.5
loser: −2 → −3.0
Δθ = 1.5

赢家上升，输家下降。若 β=1，loss≈0.201。

DPO 也会喜欢的“偷懒更新”winner: −2 → −2.5
loser: −2 → −4.5
Δθ = 2.0

两张都下降，但输家降得更多。差距更大，loss≈0.127，反而更低。

Krea 在多种偏好数据混合上观察到这种 policy divergence：winner 与 loser 的似然同时下降，只是速度不同。如果两张图都不如当前模型分布，这未必坏；可偏好集不总满足这个前提。模型还会远离预训练分布，后续出现高频伪影。

STPO 的披露边界

报告只说 STPO 修改了原始 DPO 公式，并加入一个辅助损失来减轻 divergence。它没有给全称、完整公式、辅助项形式或消融。因此我们能准确解释“它要修什么”，但不能把“限制 winner 绝对似然”“加某种 KL”之类猜测写成 STPO 的真实公式。

PO 本身分两段：先大规模生成合成偏好对做初调，多数 pair 至少有一张来自当前策略（on-policy）；再只用内部人工标注做校准。标注者熟悉模型具体的优点、缺点和怪癖，减少“通用审美偏好”与这个模型实际问题之间的错位。

08 · Reinforcement learning四类奖励定义优化方向，prompt 池决定算力花在哪里

通用美学RL 主奖励。最终采用 pointwise Bradley–Terry reward model。
提示遵循按每个 prompt 的 rubric 逐项核验，而不是只给整体印象分。
文字渲染单独检查要写出的文字，避免被一般美学分掩盖。
瑕疵与结构专抓多指、肢体畸形、扭曲文字等 VLM 容易漏掉的问题。

美学 reward model 比过两条路：pointwise 模型用 Bradley–Terry 目标让 winner 分数高于 loser；pairwise VLM judge 同时看两图，直接输出偏好。后者能用上大模型推理，前者却更快、没有左右位置偏差，大规模 rollout 打分更划算，所以最终选 pointwise。

Style Reference：只借“怎么画”，别把“画了什么”也搬来

参考图里风格与内容纠缠在一起：你想借笔触、色调和材质，模型却可能把参考图的主体也带进结果，这叫内容泄漏08。Krea 的系统支持多张参考图、每张独立权重、连续强度与语义混合。报告披露的训练方法只有两层：新的自监督训练 + 偏好优化；没有公开模块结构、自监督任务和损失，所以这里不能再往下“脑补实现”。

10 · Timestep distillationTurbo 为什么能从 52 步缩到 8 步？

一个成熟 teacher 会用很多个小步逐渐去噪；少步 student 每一步都要跨过更大的噪声区间。如果训练只要求最终图片分布一致，student 的中间状态仍可能明显偏离 teacher。普通 DMD 主要匹配干净图分布；TDM09 把这种分布匹配扩展到多个时间步，让 student 的中间状态也接受约束。

Raw / Teacher · 很多小步每一步都短，路径细致；适合训练、LoRA 与研究。
→
Turbo / Student · 8 个大步不只终点对齐，若干时间步的分布也向 teacher 靠近。

Krea 比过 DMD、DMD2、Decoupled DMD、piFlow、APT，最后选 TDM，理由是：无数据、快、超参数少、容易调，而且能灵活蒸馏成不同步数。这里的“无数据”是说蒸馏不依赖原始训练集，不是说训练完全不采样。

11 · Infrastructure大规模训练不是“显卡越多越快”，而是故障面一起膨胀

报告有很长一部分讲基础设施，这不是跑题。数十亿图像的数据策展、长时间 12B 训练和 RL rollout 如果没有稳定系统，前面的方法根本落不了地。

训练并行PyTorch DTensor + torchtitan，FSDP2 与 Megatron 风格 TP；TP>2 时启用 async-TP。
研究吃满，推理外溢Kueue 负责多机 gang scheduling 与配额借还；训练占满本地 GPU 时，Virtual Kubelet 把线上推理迁到外部算力。
调度器不替业务兜底外部实例失败后，适配层把 pod 标为失败，让 Kubernetes / HPA 创建替代者，而不是自己再造一套恢复系统。
模型做宽一点更大的矩阵乘更容易掩盖预取延迟，也减少 all-gather / reduce-scatter 次数，NCCL 错误随之下降。
把故障当常态不追求永不失败，而是靠频繁 checkpoint 和快速恢复改善 MTBF / MTTR；一次 checkpoint 约 30 秒。
别只看 GPU 利用率坏卡可能显示 0%。团队更看张量核利用率、PCIe replay、NVLink 与 InfiniBand 指标。
最大的崩溃源InfiniBand fabric 不稳定是头号原因。少于 128 卡很稳；规模再翻倍，故障增长远超预期。
数据即队列krablet 用 PostgreSQL 分片 + funnel 批写，208TB 元数据；FOR UPDATE SKIP LOCKED 直接把 DAG 当可靠队列。

训练引擎：每一部分按自己的通信模式切

训练代码从 PyTorch 搭起，分布式抽象采用 DTensor / torchtitan。MMDiT 主干同时使用 FSDP2 与 Megatron 风格 Tensor Parallel：FSDP2 把参数、梯度和优化器状态分到数据并行卡上；TP 则把同一层的大矩阵乘拆到多张卡。TP 大于 2 时启用 async-TP，让通信尽量和计算重叠。Autoencoder 体量较小，直接在每卡复制；文本编码器和 MMDiT 才做分片。

为什么模型更宽、层数更少

第一，更大的单次矩阵乘更容易盖住 FSDP2 预取等待，而且层数少意味着 all-gather / reduce-scatter 次数少，触发 NCCL 故障的机会也少。第二，8-bit 训练每层都要量化和反量化，矩阵更大时这部分固定开销更容易被摊薄。

Kernel 与显存取舍

全模型用 torch.compile。注意力默认走 cuDNN，特殊 mask 再选 FlexAttention 或 FlashAttention 3。低分辨率只对昂贵 block 做选择性 activation checkpoint；高分辨率才全量重算，以计算换显存20。

训练数据先按形状排好

Parquet 每行记录图片引用、裁剪缩放参数、caption 和元数据。数据在进入 GPU 前预先裁剪、缩放，并按宽高比分桶打包，使一个 batch 内 padding 后形状一致，避免某些 CPU/GPU 因样本更大而拖慢整组。

为什么强调“可精确重放”

数据预先打乱后顺序扫描，样本打包结果可以原样重放。若某一步 loss 突然尖峰，团队能重新喂同一批图，而不是只看到“第 N 步坏了”却找不到当时的数据组合。

故障恢复没有引入复杂的 torch-ft 或 DiLoCo，而是优化两个更直接的量：多久才遇到一次失败，以及失败后多久能回来，也就是 MTBF / MTTR24。频繁保存 checkpoint、缩短启动时间，通常比让训练永不失败更现实。RL 的 reward 推理与策略训练已经拆成独立服务，但 rollout 与训练目前仍共享 GPU；报告把完全异步的 PipelineRL 留作后续。

一个集群同时跑研究和线上：先排整组，再排单个 Pod

研究训练和生产推理共用一个 Kubernetes 集群。Kueue21先在 Workload 层决定“这次多机训练能不能整组开跑”，默认调度器再决定每个 Pod 落到哪台机器。这种 gang scheduling 很关键：一个 128 卡任务如果只拿到 120 卡就开始，前面 120 卡只会空等最后 8 卡。

队列与配额研究队列可以借生产队列暂时不用的 GPU；高优任务到来时再回收。
整组准入所有训练 Pod 能一起满足，Workload 才进入可调度状态。
本地 GPU 优先训练尽量留在高速互联的物理集群；研究高峰可以占满。
推理外溢自研 Virtual Kubelet 把普通 Pod spec 翻译成外部算力供应商的部署请求22。
失败交还 K8s外部实例坏了就把 Pod 标记为失败，由 HPA 新建替代者；适配层不另造恢复控制面。

团队曾评估 Volcano、自定义 kube-scheduler 与 InterLink，但当时 Kueue + 默认调度器已够用，InterLink 也不适合其外部推理场景。自研 Virtual Kubelet 只保留 create、delete、exec 和启动恢复等必要原语。这里的重点不是“自研一定更好”，而是把供应商差异隔离在虚拟节点后面，让上层仍按 Kubernetes 的失败语义工作。

坏节点不一定完全坏：开发任务能忍，长训不能赌

运维先把异常节点清单转成 label，再用 affinity、taint 和 drain 控制任务去向。名为 Packerman 的内部工具会把开发机尽量塞到这些“有小毛病但还能用”的节点，把健康节点留给长训练。例如单卡长期比邻卡高 5–10°C，交互实验也许还能接受，数百卡同步训练却可能因一张卡降频拖慢所有卡。

监控什么：GPU_UTIL 只是线索，不是结论

指标正常时怎么看异常时说明什么
温度同机 GPU 应处在相近区间。持续超过约 75–78°C 可能降频或变得不稳；单卡明显偏热更值得查。
GPU_UTIL只表示采样窗口里是否有 kernel 在跑。崩溃后显示 0% 往往只是结果，不告诉你为什么崩，不能单独定位根因。
Tensor Core Active比 GPU_UTIL 更接近真正的矩阵计算强度；高分辨率通常更高。稳定下降可能指向慢卡；过热降频时也可能出现反常波动。
FB_USED显存占用应随同一训练配置保持一致。有些卡在分配阶段卡住约 5GiB，需要重启；不一定是普通 OOM。
XID / Remapped rowsXID 很少出现；remapped row 应长期稳定。一旦出现，XID 很有诊断价值；remapped row 增长常指向显存硬件问题。
PCIe / NVLink偶发少量 replay 未必有害。单卡 replay 突增常早于崩溃；团队另采集 NVLink CRC、replay 与 recovery 计数。
InfiniBand链路吞吐和错误应在各节点间相对均匀。这是团队遇到的最大崩溃来源；用设备级指标和 ibtop 缩小到具体链路。

最诚实的一条经验是：相同代码和数据可能连续两小时反复崩，什么都不改重启后却跑 16 小时；少于 128 GPU 的任务能稳定数天，规模继续放大后，故障增长远超线性预期。报告还明确说，他们的大规模 run 没有一次能连续超过 24 小时不失败。

存储与数据处理：把一张表同时当 DAG 和工作队列

Ceph 在这支团队的规模与负载下造成大量问题；换到 WEKA 后，故障、停机和 checkpoint 时间都明显改善，一次 checkpoint 约 30 秒。所有数据与产物放进同一文件系统，管理简单；但通过 IPoIB 访问存储可能加重 InfiniBand 压力。报告强调网络问题在换存储前就存在，因此不能把两者简单写成单一因果。

元数据系统 krablet23由多个 PostgreSQL 分片和 funnel 批写层组成，管理约 208TB 元数据，并承受每秒数万次互相竞争的 UPSERT。大对象仍在对象存储里，数据库只保留引用和处理状态，作为单一事实来源。

为什么 FOR UPDATE SKIP LOCKED 能把表变成队列 SELECT ... FROM jobs
WHERE done = false
ORDER BY last_tried_at
FOR UPDATE SKIP LOCKED;

工人 A 先锁住一批未完成行；工人 B 查询时自动跳过已锁行，领取下一批，因此两者不会重复处理。失败任务不进单独的死信队列，而是更新 last_tried_at 后重试，避免一个坏任务永远堵在队首。这样从 1 个扩到 1000 个 worker 不必重新分片，处理进度也能直接用 SQL 查询。

上层工具 pluck 提供类似全局 map 的接口：用 TABLESAMPLE 或统计分区挑任务，把 cloudpickle 序列化的 UDF 发给 worker。它支持 spot 实例、部分完成、不同样本不同 batch 代价，以及持续增量处理。团队未来考虑把元数据迁到对象存储上的 LSM tree，但这仍是路线，不是 Krea 2 当前已经采用的实现。

不要把经验写成通用定律

“Ceph 不好”“128 卡以上必崩”“WEKA 一定更快”都不是这份报告能证明的普遍结论。准确说法是：在 Krea 当时的硬件、网络、访问模式和集群规模下，他们观察到了这些结果。

12 · Future work下一代的目标：更大、更统一，也更像真正的创作接口

结尾不是普通的“以后扩大规模”，而是暴露了 Krea 2 当前最真实的四类债务：模型还没训够、不同能力会互相打架、生产链条由太多模型拼成、自然语言 prompt 也装不下真实用户意图。

继续扩规模尝试 MoE、稀疏注意力下的原生 2K–4K、NVFP4 预训练和 Muon。团队还直接承认当前模型 undertrained，更长训练本身仍可能带来收益。
MOPD：专家并行，能力汇总不同 RL 团队先训练领域专家，再用多教师 on-policy dense supervision 蒸进一个学生，目标是避免“这个域变好、另一个域退化”。团队说已内部验证 OPD / MOPD 有效，但没有公开实验。
把模型链合成一个现在要同时维护 VAE、DiT、文本编码器、prompt expander，甚至 style reference 与 upscaler。未来想统一成一个模型，让研究资源集中在同一主干上。
原生理解多种输入协议真实用户不只写自然语言，还会给 tags、JSON、bounding box、指令、视觉规范与 Markdown。Prompt expansion 只能缓解，底座最终应直接理解这些格式。

这里的 MOPD13还有一层组织意义：如果每个领域团队都能独立训练专家，最后统一蒸馏，就不必让所有人同时改一个 checkpoint、彼此担心回归。但“内部验证有效”与“已经公开证据”不是一回事，报告没有给出专家数量、dense supervision 形式或主结果。

术语表 · 点正文角标可直接查看

  1. 审美收敛 · 模型能力变强的同时，默认输出越来越像同一种高饱和、强对比、表面精致的风格。Krea 要保住的是可探索的视觉分布。
  2. 预训练 0 AI 图 · 只限定预训练图像混合。后训练仍会使用合成偏好对，prompt expander 也使用合成文本数据。
  3. SAE 视觉标签 · 在 SigLIP-2 嵌入上训练稀疏自编码器，让 VLM 给高激活特征命名，用这些特征筛明显伪影，不必每类问题都单训分类器。
  4. 中训 · 位于大规模预训练和高质量 SFT 之间，按领域与来源自上而下补高分辨率、长尾概念、世界知识和文字能力。
  5. DPO · 直接比较 winner / loser 相对参考模型的似然差。只盯差值会留下“两者一起下降、loser 降得更多”的解。
  6. Prompt 资源分配 · RL 不应平均花算力；太易、太难或组内没方差的 prompt 信号都弱，应优先难但可学的样本。
  7. Prompt Expansion · 把短、口语、欠明确的用户输入映射为模型更熟悉的长 caption，同时尽量不覆盖用户意图。
  8. 内容泄漏 · 本想只借参考图的笔触、色调、材质，结果把参考图的主体或物件也带进生成图。
  9. TDM · 不只匹配最终干净图分布，还把分布匹配铺到多个时间步，用来训练灵活的少步 student。
  10. Gated sigmoid attention · 报告观察到主要收益是训练稳定性，而非明显质量提升；额外参数和计算很少。
  11. PMA · 用若干预训练 checkpoint 做参数合并近似权重平均，效果接近 EMA，又省去常驻影子权重的显存。
  12. Rubric reward · 把每条 prompt 拆成对象、数量、关系、文字等可验证要求逐项判，不让整体印象分掩盖局部失败。
  13. MOPD · Multi-teacher On-Policy Distillation。不同领域专家产生在线密集监督，再把能力蒸进同一个学生；报告只给路线与内部验证结论，未给具体算法和结果。
  14. 伪影 · 图像采集、压缩、处理或生成过程额外制造的错误，例如边框、棋盘格、振铃、多指或扭曲文字。运动模糊和胶片颗粒也可能是有意风格，不能一概算伪影。
  15. pHash / colorhash · pHash 把灰度图的低频结构变成 bit 指纹，用汉明距离比较；colorhash 记录颜色分布但很少记录颜色位置。Krea 用 12×12 pHash 加 colorhash 降低 8×8 灰度 pHash 的误判。
  16. Laplacian · 图像的二阶差分，会放大边缘、细纹理和孤立噪点等局部突变。它适合做便宜初筛，但不能单独判断图片好坏。
  17. DCT · 离散余弦变换 · 把“每个像素有多亮”改写成“不同尺度的明暗变化各有多少”。输出左上角是整体亮度和粗结构等低频，越往右下越偏细边缘、纹理和噪点等高频。
  18. 时间步条件 · 告诉去噪网络当前状态离纯噪声或干净图有多远。相同的 latent 若处在不同时间点，模型该做的事不同：前期决定大结构，后期修局部细节。
  19. Muon · 面向矩阵参数的优化器，会先对动量更新做正交化，使不同方向的更新更均衡。它不是把 AdamW 名字换掉；哪些层使用、更新尺度怎样匹配、是否加 Nesterov momentum 都会显著影响稳定性。
  20. Activation checkpointing · 前向时不保存部分中间激活，反向时再重新算一遍。它用更多计算换更少显存；这里的 checkpoint 不是写到磁盘的模型存档。
  21. Kueue / gang scheduling · Kueue 先判断一整个分布式任务是否有足够配额；gang scheduling 要求所需 Pod 成组获得资源，避免只启动一半后集体空等。
  22. Virtual Kubelet · 在 Kubernetes 中伪装成节点的适配层。调度器仍提交普通 Pod，它再把 Pod 规格翻译成外部算力供应商的部署请求，并把状态同步回来。
  23. krablet / SKIP LOCKED · Krea 的分片 PostgreSQL 元数据系统。多个 worker 用行锁领取任务；SKIP LOCKED 让后来者跳过别人已经锁住的行，从而并行取走不同任务。
