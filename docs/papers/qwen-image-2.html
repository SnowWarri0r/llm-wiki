Qwen-Image-2.0 ·
生成与编辑，揉成一个模式。

Qwen Team, Alibaba · Qwen-Image-2.0 Technical Report · arXiv 2605.10730 · 2026

以前"从零生成"和"改一张图"是两套管线。Qwen-Image-2.0 把它们统一成一件事——都是"条件 → 目标"：生成时条件只有文字，编辑时条件里多拼一段原图 latent。同一个 MMDiT 全包。招牌是复杂文字渲染（尤其中文）和 1K token 长指令直出幻灯片/海报/信息图。

TL;DR

Qwen-Image-2.0（20B MMDiT）把生成 + 精确编辑统一进一个模式：核心是"条件→目标"建模，编辑时把原图 VAE latent 和文字嵌入拼接 Concat(ℰ_x, h_y)1 成一条多模态序列，喂同一个 MMDiT——没点名的区域天然照抄、只改指令点名处2。条件编码器是 frozen Qwen3-VL3；backbone 用 RMSNorm QK-Norm、纯乘性调制、SwiGLU、MSRoPE 跨模态位置；VAE 升级 f16c64（16× 压缩，1.0 是 8×）4。六段训练5 + 闭环 Data Flywheel；用 DMD 把 40 步老师蒸成 4-NFE 学生6。结果：LMArena T2I ELO 1168，全球 #9 / 中文 #1，超 Nano Banana；原生 2K 分辨率。

§ 01痛点：生成和编辑本是两套

过去文生图和"改图"常是分开的能力/模型：一个从噪声画新图，一个专门接收原图做局部修改。两套管线、两份权重、两种用法。Qwen-Image-2.0 的主张：它俩本质是同一件事——"给定条件，产出目标图"。区别只在条件里有没有那张原图。

§ 03架构：Qwen3-VL 读懂指令 · VAE 压原图 · MMDiT 联合

一条流水线串起来：

几个设计点：条件编码器是冻结的 Qwen3-VL（强 VLM 读懂指令，指令遵循更好，跟 Ideogram-4 那套"翻译官决定上限"同思路）；MMDiT 用 RMSNorm QK-Norm 防 attention 炸、SwiGLU、纯乘性调制（h' = αh，无加性 bias）、MSRoPE 给文字+图像算联合位置；VAE 升到 f16c64（16× 压缩，1.0 是 8×），残差自编码 + 语义对齐损失，PSNR 33.42 / SSIM 0.9225。

§ 04编辑为什么"只改点名处"

编辑的一致性不是靠额外的 mask，而是原图 latent 当 context 自然来的：模型一边去噪、一边盯着拼进来的原图 latent，没被指令点名的区域直接照抄原图、只在点名处改。

§ 05训练六段 · 蒸馏 · 招牌能力

阶段内容
① 256P文生图(T2I)预训练（8 道数据过滤）
② 256P文生图 + 图文生图(TI2I=编辑) 合训
③ 512P加合成数据
④ 512P/1024P混分辨率
⑤ 多分辨率512 / 1024 / 2048（原生 2K）
⑥ SFT高质量精选数据微调

外加一个闭环 Data Flywheel（数据飞轮）：把失败样本自动路由到 RL / 预训练 / prompt 工程 三条轨道，持续补数据。

蒸馏：40 步 → 4 步

用 DMD（Distribution Matching Distillation，分布匹配蒸馏） 把 40 步老师压成 4-NFE 学生（人像/风景/文字都保质），其中辅助的 fake-score 模型用 flow-matching 训。思路跟你看过的 Drifting（一步生成）/ DiffusionOPD（扩散蒸馏）一条线——都是"把多步推理压成少步"。

Qwen-Image-2.0 把这几块拧在一起：MMDiT + Qwen3-VL 条件编码器 + VAE latent(升 16×) + MSRoPE + QK-Norm + flow-matching/DMD 蒸馏。一句话：生成和编辑都是"条件→目标"，编辑只是条件里拼一段原图 latent；同一个 MMDiT 全包，没点名的地方天然照抄。（把这套推广到分层——任意子集图层当条件——就是 MRT。）

引用 / 术语表

  1. Concat(ℰ_x, h_y) · 条件-目标拼接 · 编辑时把原图 VAE latent ℰ_x 和文字嵌入 h_y 拼成一条多模态序列，喂同一个 MMDiT。生成则只有文字。统一两种任务的关键。
  2. 天然一致性 · 不用额外 mask：原图 latent 当 context，没被指令点名的区域逐像素照抄、只改点名处。
  3. frozen Qwen3-VL 条件编码器 · 冻结的 VLM 抽取文字(和图)的语义特征当条件。强 VLM → 指令遵循更好（同 Ideogram-4 思路）。
  4. VAE f16c64 · 16× 压缩（1.0 是 8×）、64 latent 通道；残差自编码 + 语义对齐损失。79M enc / 259M dec，PSNR 33.42 / SSIM 0.9225（重建还原度）。
  5. 六段训练 + Data Flywheel · 256P→512P→多分辨率(到 2K) 渐进→SFT；闭环数据飞轮把失败样本路由到 RL/预训练/prompt 工程补数据。
  6. DMD 蒸馏 · 4-NFE · Distribution Matching Distillation：把 40 步老师蒸成 4 次前向（NFE）的学生，让学生出图分布匹配老师。辅助 fake-score 用 flow-matching。同 Drifting/DiffusionOPD 的少步化线。
  7. MMDiT / MSRoPE · 多模态扩散 Transformer 做联合 condition-target 建模；MSRoPE 给文字+图像 token 算跨模态联合位置。详见 diffusion-transformer / M-RoPE。
