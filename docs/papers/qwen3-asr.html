FIELD NOTES · ASR · LLM 接耳朵

Qwen3-ASR ·
不从头训 ASR，给 LLM 接个耳朵。

Qwen Team, Alibaba · Qwen3-ASR Technical Report · arXiv 2601.21337 · 2026

Whisper 从零训一个 encoder-decoder，语言能力也得硬学。Qwen3-ASR 换个想法：语言别重学了——拿预训练的 Qwen3 LLM 当解码器，前面接一个音频编码器（耳朵）+ projector（翻译插头），让音频"说成 LLM 听得懂的输入"。就是 minimind-o 那套 modality-projector，专做语音、规模拉满。

TL;DR

Qwen3-ASR 把 ASR 拆成 音频 → AuT 编码器1 → projector2 → Qwen3 LLM → 文字——和 modality-projector 模式（minimind-o）同骨架，只是解码器换成现成的语言大模型，post-train 自 Qwen3-Omni。AuT 对 128 维 Fbank 做 8× 下采样 → 12.5Hz token 率3，并用动态注意力窗口 1s~8s4——同一个模型既能 1 秒短块流式、又能 8 秒长上下文离线高精度。训练四段：AuT 预训练（~4000 万小时伪标注）→ Omni 预训练 → ASR SFT（学 I/O 格式 + context biasing5）→ RL 用 GSPO 磨鲁棒。亮点：prompt 塞热词定制转写（因为解码器就是会读 prompt 的 LLM）、52 种语言/方言、带 BGM 的整首歌、填槽式 NAR 时间戳6。LibriSpeech WER 1.63/3.38；带口音英语 16.07 完胜 Whisper-large-v3 的 21.30；0.6B TTFT 92ms。

§ 01思路：别重学语言，给 LLM 接耳朵

Whisper 是从零训一个 encoder-decoder：听音频、写文字，连"语言长什么样"都得从弱监督数据里硬学。Qwen3-ASR 的想法：语言这件事，世界上最强的一批模型（LLM）早就学透了——别重学，直接拿来当解码器。

所以它在一个预训练 Qwen3 LLM 前面接两块：一个音频编码器（耳朵）把声音变成 token，一个 projector（翻译插头）把音频 token 翻译进 LLM 的输入空间。LLM 本身就是写转写的那支笔，还自带强语言先验。整套是从 Qwen3-Omni（已经多模态）post-train 来的。

一句话对照

Whisper = 从头培养一个"既会听又会写"的译员；Qwen3-ASR = 雇一个文笔已经很好的写手（LLM），只教它读懂一种新输入格式（音频 token）——语言那半边几乎不用重学。这也是它能 prompt 里塞热词的根：解码器就是个会读指令的 LLM。

§ 02三段流水线 = modality-projector 模式

整条链就三块，正是你在 minimind-o 见过的 模态编码器 → projector → LLM：

Qwen3-ASR-1.7B = Qwen3-1.7B LLM + 300M 的 AuT（hidden 1024）；还有个 0.6B 小号（Qwen3-0.6B + 180M AuT）。

§ 03AuT 编码器：压成 12.5Hz + 动态窗口

这只"耳朵"有两手关键设计。

① 8× 下采样 → 12.5Hz token 率

输入是 128 维 Fbank（log-mel 频谱的近亲，每秒上百帧）。AuT 把它8 倍下采样，压成每秒 12.5 个音频 token。为什么压：LLM 不需要每 10ms 的细节，它要的是接近"音节/词"粒度的块——少 token = 又快又省、还更好喂进 LLM。

§ 04训练四段

阶段干啥数据
① AuT 预训练学通用音频表示（可变窗口下）~4000 万小时伪标注（中英为主）
② Omni 预训练底座 = Qwen3-Omni（多模态理解打底）3T token 音/视/文
③ ASR SFT风格迁移到 ASR-only I/O 格式 + 学 context biasing较小的多语数据
④ RL（GSPO）磨噪声鲁棒 + 转写稳定~5 万条（35% 中英 / 35% 多语 / 30% 功能）

第④段的 GSPO 是 GRPO 的近亲——把重要性比率从 token 级提到序列级，低方差更稳（见 GSPO 概念页）。

§ 05亮点 · context biasing + 时间戳 + 结果

最有意思的卖点：因为解码器是个会读 prompt 的 LLM，你能把热词/背景知识塞进 system prompt，定制转写。专有名词、人名、行话——Whisper 这种从头训的 enc-dec 很难做到。

其它亮点：52 种语言/方言（30 语言 + 22 中文方言）含内建语种识别；对带 BGM 的整首歌鲁棒、能转歌词；时间戳由 Qwen3-ForcedAligner 给——把强制对齐改成"填槽"，用 [time] 特殊 token，非自回归一次性预测所有时间戳（RTF≈0.001，极快）。

Qwen3-ASR 把这几块拼起来：modality-projector（同骨架，minimind-o）+ Whisper（对照：从头 enc-dec vs LLM 接耳朵）+ Fbank/log-mel（输入）+ GSPO/GRPO（RL 阶段）。一句话：语言交给预训练 LLM，只新训"耳朵+插头"；解码器是 LLM 这一点，顺手解锁了 prompt 热词定制。

引用 / 术语表

  1. AuT 编码器 · 那只"耳朵"，AED（attention encoder-decoder）式的独立预训练音频模块。1.7B 版 300M / hidden 1024；把声音变成音频 token 喂给 LLM。
  2. projector · "翻译插头"，把音频 token 映射进 LLM 的输入嵌入空间。就是 modality-projector 那块（minimind-o 同款）。
  3. 8× 下采样 → 12.5Hz · 把 128 维 Fbank 的每秒上百帧压成每秒 12.5 个 token。LLM 不要每 10ms 细节，要接近音节粒度的块 → 少 token、又快又好喂。
  4. 动态注意力窗口 1s~8s · 同一个编码器，注意力窗口可在 1 秒（短块 → 流式低延迟）和 8 秒（长上下文 → 离线高精度）之间切。一个模型两种模式。
  5. context biasing · 把热词/背景知识塞进 system prompt 定制转写。只因解码器是会读 prompt 的 LLM 才做得到；Whisper 那种从头 enc-dec 难。
  6. Qwen3-ForcedAligner · 填槽 NAR 时间戳 · 强制对齐=文字已知、只求每字时间（卡拉OK逐字对时间）。把它重构成"槽位填充"，每字插 [time] token，非自回归一次性并行预测全部时间戳 → 不累积误差。RTF≈0.001，比 WhisperX/NFA 累计偏移降 67~77%。
  7. GSPO（RL 阶段） · GRPO 的近亲：重要性比率从 token 级提到序列级，低方差更稳。Qwen3-ASR 第④段训练用它磨鲁棒。
