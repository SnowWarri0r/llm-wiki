FIELD NOTES · VISION · TRANSFORMER

ViT ·
把图切成词.

Dosovitskiy · Beyer · Kolesnikov · et al. · Google Brain · 2021

把图切成 16×16 的方块，每块当一个"词"，剩下的事全交给 Transformer。CNN 在视觉的护城河，被一篇论文填平了。

TL;DR

2017 年 Transformer 在 NLP 平推了 RNN1，但视觉一直是 CNN 的天下。问题不在 Transformer 不行，在像素当 token 算不动。ViT 的办法很粗暴 —— 把图切成 16×16 的 patch2，每个 patch 拉平投影后当一个 token，剩下的就是标准 Transformer encoder3。代价是丢掉 CNN 的归纳偏置4（局部性、平移不变、层次结构），换来更纯净的架构。结果：数据少时 ViT 输（ImageNet-1k）；数据多时 ViT 完胜（JFT-300M）5。视觉也吃 scaling law6。

§ 01痛点 · 像素当 token 算不动

2017 年 Transformer 出来后，NLP 端的旧时代基本结束 —— RNN/LSTM 被推平，BERT、GPT 一统江湖。视觉端的人也想跟进，但卡在一个最朴素的问题上：

最朴素的想法不行

一张 224×224 的 RGB 图 = 50,176 个像素。每个像素当一个 token，喂进 self-attention？
Self-attention 是 O(N²)，N = 50,176 → 算 25 亿次注意力。算力直接爆。

所以 2017-2020 这几年，视觉里用 Transformer 的论文全是"混搭" —— 留一段 CNN 做特征提取（把 224×224 压成 14×14 的 feature map），再接 attention。CNN 和 Transformer 各分半。

没人敢直接拿原图喂 Transformer。因为只要这一步成立，CNN 的护城河就没了。这就是 ViT 做的事 —— 它没改 Transformer 本身，只是把"什么是 token"这个定义改了。

§ 03Patch embedding · 切完拉平投影

切块这一步在工程上怎么实现？看 PyTorch 代码就 4 行8：

patch_embed = nn.Conv2d(
    in_channels=3,           # RGB
    out_channels=768,        # hidden dim
    kernel_size=16,          # patch size
    stride=16,               # 不重叠
)
# 输入: (B, 3, 224, 224) → 输出: (B, 768, 14, 14)
x = patch_embed(image).flatten(2).transpose(1, 2)   # (B, 196, 768)

关键洞察：这里的 Conv2d 不是真的在做卷积特征提取。stride=kernel_size=16 意味着卷积窗口不重叠 —— 它其实就是 "切块 + 拉平 + linear" 这三步的一个紧凑实现。可以理解成 整个 ViT 网络里唯一的一个卷积层，只用一次，用完就再也不见。

跟 BERT 比一下就一目了然：

BERT:  字符串 → 切词 → token_id → Embedding 查表 → (B, L, 768)
ViT:   图像 → 切 patch → patch_pixels → Linear 投影 → (B, L, 768)
                                              ↑
                                       唯一的差别在这一层

"图像和文本都是 token 序列，区别只是 token 怎么来的" —— 这个抽象是 ViT 留给世界最重要的礼物。后来所有的多模态（CLIP / DALL-E / Flamingo / LLaVA）都是顺着这一步走出来的9。

§ 04归纳偏置 · CNN 的祖传家产被扔了

切 patch 这一步在工程上简单，但代价不小 —— ViT 把 CNN 经过 8 年沉淀下来的"祖传家产"全扔了。具体扔了三样东西4：

老员工 vs 新员工

归纳偏置就像老员工的经验：少数据时是宝（不用培训直接上手）、大数据时是包袱（过时的判断拖累学习更精细的规律）。这是 ViT 引出的大问题 —— 数据多到什么程度，扔掉偏置才划算？下一节给答案。

§ 05scaling · 数据量决定胜负

论文里最震撼的一张图，是把 ViT 和 ResNet 在不同数据规模上对比。这条交叉点曲线定义了 2020 年之后视觉的范式10。

§ 06注意力学到了什么 · 自己长出 CNN

论文里还有一个有意思的发现 —— 训练好的 ViT，浅层 layer 的注意力居然自己长出了类似 CNN 的局部性，没人手写，模型从数据里自己学的11。

这条线连到 bitter lesson

Sutton 的 bitter-lesson 说过去 70 年 AI 历史一次次证明: 通用方法 + 大数据 + 大算力 打败 把人类知识写死进结构。ViT 是这个规律在视觉端的最经典案例 —— CNN 的"祖传家产"是人类知识，ViT 扔了它，靠 attention + 大数据自己学，而且学出来的东西比写死的更好。

§ 07遗产 · 它真正改变的是什么

ViT 在 2020 年发布时，圈内有两种声音：

• "只是大力出奇迹，没意思" —— 跟 GPT-3 出来时的反应一样
• "CNN 还能再苟一阵" —— 因为 ImageNet-1k 上 ViT 输了

但 ViT 真正的影响力不在分类准确率本身，而在三个更深的层面：

① 跨模态架构统一

ViT 之前，视觉 = CNN、语言 = Transformer、语音 = CNN+RNN，三家用不同的架构，没法直接对齐。ViT 之后，"图像也是 token 序列" 立起来了 —— 从此 CLIP、DALL-E、Flamingo、LLaVA 这些多模态都顺着这条路走9。文本 token 和图像 token 共享同一个 Transformer，跨模态学习就是给两边 token 做对齐。没有 ViT 就没有 GPT-4V。

② CNN 时代结束

2021 年起，顶会论文里"纯 CNN backbone" 越来越少。Swin Transformer (用层次窗口) / MAE (BERT 思路套视觉) / DINOv2 / SAM —— 全是 ViT 的变种。CNN 没有死，但它从"主流"变成了"工具" —— 在算力受限、数据少的场景还有用，在前沿研究里已经退场。

③ 视觉吃 scaling law

ViT 证明的最深远的事是: 视觉跟语言一样，越大越好、越多越好，看不到尽头。这个结论之后，视觉领域开始进入"大数据 + 大模型"的疯狂 scaling 时代 —— DINOv2 用 142M 张图、SAM 用 11M 张图 + 1B mask、Stable Diffusion 用 LAION-5B 五十亿图文对。这条 scaling 路线，是 ViT 第一个打开的。

引用 / 术语表

  1. Transformer 推平 NLP · 2017 Vaswani et al. 提出 Transformer。3 年内 RNN/LSTM 在 NLP 几乎全被替换，BERT/GPT 主导一切。视觉端的人看着眼馋。
  2. 16×16 patch · ViT 论文标题就是 "An Image is Worth 16×16 Words"。224×224 图切成 14×14 个 patch = 196 token。patch 大小是工程妥协：再小算不动 (O(N²))，再大丢细节。详见 patch-embedding。
  3. 标准 Transformer encoder · 跟 BERT 几乎完全一致 —— 12 层 multi-head self-attention + MLP + LayerNorm + 残差连接。ViT 没改 Transformer 本身，只改了输入端的 token 怎么来。
  4. 归纳偏置 · inductive bias · 模型架构里"预先写死"的对世界的假设。CNN 的三大假设：局部性、平移不变、层次结构。少数据时强偏置帮模型省培训成本，大数据时强偏置反而是限制。详见 inductive-bias。
  5. JFT-300M · Google 内部的 3 亿张图数据集 (不公开)。论文核心结论 —— ViT 在 JFT-300M 预训练后微调下游任务时反超 ResNet。意味着 ViT 的赢面需要 普通研究者拿不到的数据规模。后来 LAION-5B 这种开源大数据集才让 ViT 路线对学术界开放。
  6. 视觉的 scaling law · ViT 证明视觉跟语言一样满足 scaling law —— 数据越多、模型越大、训练越久，效果越好且看不到饱和。详见 scaling-laws。
  7. ViT 的完整 pipeline · 切 patch → 拉平 → linear 投影 → 加位置编码 → 加 [CLS] token → 标准 Transformer encoder → 取 [CLS] 输出 → MLP 分类头。整个网络只有第一步 "切+linear" 是 ViT 自创的，后面照搬 BERT。
  8. Conv2d 实现 patch embedding · 工程上把 "切块 + 拉平 + linear" 三步合并成一个 stride=16 的 Conv2d。它不是真的卷积特征提取，只是这三步的紧凑实现。整个 ViT 网络里这是唯一的卷积层。
  9. 跨模态架构统一 · ViT 把"图像 = token 序列"这个抽象立起来后，文本 token 和图像 token 可以进同一个 Transformer，跨模态学习变成"给两边 token 做对齐"。CLIP / DALL-E / Flamingo / LLaVA 全是顺着这条路走的。
  10. 交叉点曲线 · 论文最核心的图：数据量从小到大，CNN 和 ViT 的曲线相交于 ImageNet-21k 附近。交叉点前 CNN 赢、交叉点后 ViT 赢，而且 ViT 的曲线斜率更陡 (扩展性更好)。
  11. 注意力的自然层次 · 训练好的 ViT 浅层 attention 自己学出了"只看附近 patch"的局部性 —— 跟 CNN 写死的卷积核行为一样，但是学出来的而非写死的。深层 attention 才扩散到全图。证明 ViT 不是不会用局部性，而是它能自己判断什么时候该用。
