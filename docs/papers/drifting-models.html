Drifting ·
把迭代从推理时，搬进训练时。

Mingyang Deng, He Li, Tianhong Li, Yilun Du, Kaiming He · Generative Modeling via Drifting · arXiv 2602.04770 · 2026

扩散把"反复迭代"放在推理时——生成一张图要跑 250 步去噪。Drifting 反过来：用一个"吸引真数据 + 排斥自己"的漂移场，在训练时一点点把生成分布推向真实分布；等它们重合、场归零，生成器就练成了。推理只要一步。ImageNet 256 一步 FID 1.54。

TL;DR

Drifting Model 是一种新生成范式：扩散/flow 把复杂的"噪声→数据"映射拆成推理时的多步迭代（250+ NFE）；Drifting 把迭代挪到训练时，推理就只剩一步（1-NFE）。靠一个漂移场1 V = V⁺(吸引真数据 p) − V⁻(排斥自生成 q) 推动样本。关键反对称2 V_{p,q} = −V_{q,p}：当 q=p 时场归零 → 平衡，生成器恰好停在"骗过数据"那刻。训练是个不动点3：把"漂移后的自己"stopgrad 冻成靶子去追，loss = ‖f(ε) − stopgrad(f(ε)+V)‖² = 最小化漂移量 ‖V‖²。场用核函数4 在预训练特征空间里算（吸近处真样本、斥近处假样本，像粒子间的力）。像没有判别器的 GAN5。结果：ImageNet 256 一步 FID 1.54（latent）/ 1.61（pixel），SOTA 单步。

§ 01痛点：迭代卡在推理时

扩散 / flow-matching 的本事，是把"噪声 → 数据"这个超复杂的映射，拆成一长串小步：每步只去掉一点噪声，沿着学好的 ODE/SDE 爬。好处是每步好学；代价是生成一张图要在推理时跑很多步（几十到几百次网络前向，NFE 高），又慢又贵。

各种"少步采样"（DDIM、蒸馏、consistency）都在事后压缩这条推理链。Drifting 换了个根上的想法：能不能压根不在推理时迭代——把迭代这件事，整个搬到训练时去？

§ 02漂移场：吸引真数据 + 排斥自己

把生成器吐出的样本想成一群粒子。Drifting 给每个粒子设计一个漂移场 V，两股力合成：

  • 吸引 V⁺：把粒子往真实数据 p 拉（靠近真样本）。
  • 排斥 V⁻：把粒子从当前生成分布 q 推开（别都挤在自己现在这个团里 → 散开、补上没覆盖到的地方）。
V_{p,q}(x) = V⁺_p(x) − V⁻_q(x)
           = 吸向真数据      − 斥离自生成

§ 03反对称 → 平衡：q=p 时场归零

怎么知道"练好了"该停？靠漂移场的反对称性质：

V_{p,q}(x) = − V_{q,p}(x)        对所有 x 成立
⟹  当 q = p 时，V_{p,q}(x) = 0     （场处处归零 → 平衡）

直白说：生成分布 q 一旦和真实分布 p 重合，吸引和排斥正好抵消，粒子不再被推动。生成器恰好停在"q 骗过 p"那一刻——这就是天然的收敛信号，不用判别器、不用额外的停止准则。

§ 04训练即演化：不动点 + stopgrad

那"漂移"具体怎么训进网络？用一个不动点技巧。每一步：算出漂移场会把当前样本推到哪（f(ε)+V），把这个"漂移后的自己"冻住（stopgrad，不回传梯度）当靶子，让生成器输出去追它：

L = E_ε ‖ f_θ(ε) − stopgrad( f_θ(ε) + V_{p,q}(f_θ(ε)) ) ‖²
  = E_ε ‖ V(f_θ(ε)) ‖²            # 等价于最小化漂移量

妙处：随着训练一步步走，样本的隐式轨迹 xᵢ₊₁ = xᵢ + V(xᵢ) 把整个分布 q 一点点漂向 p。优化器的迭代，本身就是分布的演化——这正是扩散在推理时用 ODE solver 干的事，被搬到了训练循环里。训练完，V≈0、q≈p，生成器 f_θ(ε) 一次前向直接出图。

# Algorithm 1（简化）
e = randn(M, C)
x = f(e)                          # 生成样本
V = compute_V(x, y_真样本, y_neg=x)  # 吸引-排斥力场（核函数）
x_drift = stopgrad(x + V)         # 漂移后的靶子，冻住
loss = mse(x, x_drift)            # 追上去

§ 05它像谁 · 结果

这味道很像 GAN，但没有判别器：GAN 用一个学出来的判别器给"真/假"打分、推动生成器；Drifting 把这股推力换成核函数算的漂移场——吸引近处真样本、排斥近处自生成样本（k(x,y)=exp(−‖x−y‖/τ)），不用训练对抗网络，排斥自己那项天然防 mode collapse。场在预训练特征空间（latent-MAE）里算，省掉直接在高维像素上较劲。

对照扩散 / flowDrifting
迭代发生在推理时（固定 ODE/SDE 轨迹）训练时（优化器迭代 = 分布演化）
推理步数多步（250+ NFE）一步（1-NFE）
收敛信号—漂移场归零（q=p）
ImageNet 256 FID多步才到1.54（latent）/ 1.61（pixel）· 一步

Drifting 接 ODE/SDE 那条线但走相反方向：扩散把"噪声→数据"的迭代放推理时（多步慢）；Drifting 把它放训练时，推理一步。机制上像无判别器的 GAN / 分布匹配，靠吸引-排斥漂移场 + 反对称平衡（q=p 场归零）+ stopgrad 不动点三件套。一句话：别在生成时反复爬，训练时就把生成器调到"一步到位"。

引用 / 术语表

  1. 漂移场 V_{p,q} · 推动生成粒子的力场，V = V⁺(吸向真数据 p) − V⁻(斥离自生成 q)。loss 等于最小化 ‖V‖²。
  2. 反对称 → 平衡 · V_{p,q}=−V_{q,p}，所以 q=p 时场处处归零。生成分布骗过真实分布的那刻，粒子停止漂移 = 收敛信号。
  3. stopgrad 不动点 · 把"漂移后的自己"f(ε)+V 冻住当靶子，让生成器回归去追。训练迭代 xᵢ₊₁=xᵢ+V 就是分布演化。
  4. 核漂移场 · 用 k(x,y)=exp(−‖x−y‖/τ) 算吸引/排斥，在预训练特征空间（latent-MAE）里做，像粒子间物理力。
  5. vs GAN · 同样"吸真斥假"，但不用判别器、不打对抗；排斥自生成那项天然防 mode collapse。也可看作分布匹配蒸馏的连续场版本。
  6. 1-NFE / 训练时演化 · NFE=网络前向次数。扩散推理几百次；Drifting 把迭代搬进训练，推理只 1 次前向。ImageNet 256 一步 FID 1.54。
