Adam ·
既记方向，又调步长。

Kingma & Ba · Adam: A Method for Stochastic Optimization · arXiv 1412.6980 · 2014

SGD 拿一个全局学习率走天下，在"窄沟"地形里横着来回弹、顺着沟爬得慢。Adam 给梯度记两个滑动平均——一个记方向（动量）、一个记大小（方差），用前者除后者，每个参数都拿到一个自己的、合适的步长。

TL;DR

Adam = ADAptive Moment estimation，给梯度记两个 EMA：一阶动量 m1（梯度的 EMA，记方向、抹平抖动）和二阶 v2（梯度平方的 EMA，记大小）。更新量 = lr · m̂ / (√v̂ + ε)——分子是平滑方向，分母按"最近梯度多大"把步长归一化 → 每个参数自己的自适应学习率3。因为 m、v 从 0 起步早期偏小，再除 (1−β^t) 做 bias correction4。默认 β1=0.9、β2=0.999、ε=1e-8。= 动量（抹抖）+ RMSprop（自适应步长）合体，几乎不用调参就稳，是 Transformer/LLM 的默认优化器（常用 AdamW5）。

§ 01SGD 为什么抖 / 慢

普通 SGD：θ ← θ − lr · g，一个全局学习率对所有参数一视同仁。问题出在 loss 地形常常是"窄沟"——某些方向陡（一动就过头）、某些方向平（要走很远）。同一个 lr：陡方向横着来回弹，平方向爬得龟速。

根子：一个 lr 没法同时适配陡和平。Adam 用两招分别治这两个病。

§ 02动量 · 一阶 EMA 记方向

第一招治"来回弹"：不直接用瞬时梯度，而用梯度的 EMA——动量 m。一致的方向会累积（越走越快），来回抖的方向正负相消（被抹平）。像一个有惯性的重球滚下沟，不跟着每一步噪声乱跳。

m_t = β1·m_{t−1} + (1−β1)·g_t      # β1=0.9，梯度的 EMA = 平滑后的方向

§ 03RMSprop · 二阶 EMA 调步长

第二招治"陡平不一"：再记一个梯度平方的 EMA——v，它衡量每个参数最近梯度有多大。更新时除以 √v：梯度一直很大的参数（陡方向）步长被缩小，梯度一直很小的（平方向）步长被放大 → 每个参数都拿到自己的合适步长。这就是 RMSprop。

v_t = β2·v_{t−1} + (1−β2)·g_t²     # β2=0.999，梯度平方的 EMA = 最近多大
步长 ∝ 1 / √v_t                    # 大梯度→小步、小梯度→大步（逐参数）

"大的变小、小的变大"换个角度才直观：每个参数都除以"它自己平时的梯度大小"，于是全被归到同一档。魔法在于分子 m̂ ≈ g、分母 √v̂ ≈ |g|，一除把梯度自己的大小约掉：步 ≈ lr·m̂/√v̂ ≈ lr·(±1)——步子大小跟梯度多大无关，只剩方向。举两个参数：

参数 A（陡）参数 B（平）
每步梯度 g1000.01
v = EMA(g²) → √v≈ 100≈ 0.01
SGD 步 lr·glr·100（冲过头）lr·0.01（爬）
Adam 步 lr·g/√vlr·100/100 = lr·1lr·0.01/0.01 = lr·1

SGD 下 A 的步是 B 的 1 万倍；Adam 下两个都 ≈ lr·1。A 的 100 除以它自己的 100→1，B 的 0.01 除以它自己的 0.01→也是 1。像 z-score 标准化（√v ≈ 梯度的标准差，除掉它=换算成"几个标准差"），也像按各自基线打分——绝对分不重要，相对自己平时才重要。（用 EMA 而非当前 |g| 是为了稳；而且分子带符号、分母只看大小，所以来回抖的方向 m 相消、比值<1 → 自动走小步。）

§ 04Adam = 两者合 + bias correction

Adam 就是把这两个 EMA 合起来：分子用动量 m̂（平滑方向）、分母用 √v̂（最近大小）。还有一道修正：m、v 都从 0 起步，头几步被"0 的旧值"往下拖、偏小，所以各除一个 (1−β^t) 把它补回来（t 大了这项 →1，自动消失）。

m_t = β1·m + (1−β1)·g            # 一阶：方向
v_t = β2·v + (1−β2)·g²           # 二阶：大小
m̂ = m_t/(1−β1ᵗ),  v̂ = v_t/(1−β2ᵗ)   # bias correction（修早期偏小）
θ ← θ − lr · m̂ / (√v̂ + ε)        # 默认 β1=0.9, β2=0.999, ε=1e-8

Adam 是 EMA 的"两连击"——同一个 新=m·旧+(1−m)·新值 的滚动平均，一个滚梯度（方向）、一个滚梯度平方（大小）。AdamW（解耦 weight decay）是它在 Transformer/LLM 时代的默认形态：把权重衰减从梯度里拆出来单独做，正则更干净。要记的一句：m 管"往哪走"，√v 管"走多大"，bias correction 管"刚起步别太怂"。

引用 / 术语表

  1. 一阶动量 m · 梯度的 EMA（β1=0.9）。平滑方向：一致方向累积、来回抖动相消。等价于带惯性的重球。
  2. 二阶矩 v · 梯度平方的 EMA（β2=0.999）。衡量每个参数最近梯度多大；更新时除 √v 把步长归一化。来自 RMSprop。
  3. 自适应学习率 · 不是全局一个 lr，而是每个参数按 1/√v 各拿一个有效步长：梯度大的参数走小步、梯度小的走大步。
  4. bias correction · 偏差修正 · m、v 初始化为 0，头几步被旧的 0 拖得偏小。各除 (1−βᵗ) 补回来；t 增大该项 →1，影响自动消失。
  5. AdamW · Adam + 解耦 weight decay：把权重衰减从梯度更新里拆出来单独做，正则更正确。Transformer/LLM 的默认优化器。
  6. SGD / 动量 / RMSprop · 谱系：SGD（裸梯度·全局 lr）→ +动量（一阶 EMA 抹抖）→ RMSprop（二阶 EMA 调步长）→ Adam（两者合）。
