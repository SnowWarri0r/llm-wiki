CLIP ·
图文对齐到同一空间.

Radford · Kim · Hallacy · Ramesh · Goh · Sutskever · et al. · OpenAI · 2021

不再标"这张图是哪一类"，直接从互联网爬 4 亿个"图 + caption"配对训练。图像和文本在同一向量空间里对齐 —— 这是所有现代多模态的起点。

TL;DR

ImageNet 那套"人工标 1000 类"的训练方式贵且穷 —— 一张图一个 label, 信号被压扁1。CLIP 反过来 —— 从网上爬 4 亿个图文对2, 用对比学习3训一个图像 encoder + 一个文本 encoder, 让两边的 embedding 对齐到同一空间4。训完后想分什么类不用 fine-tune —— 写句话 "a photo of a {class}" 喂进去就行5。Zero-shot 27 个数据集平均跟 ResNet-50 finetune 持平。真正的意义不在 benchmark, 在于它立起来的那个图文对齐空间 —— DALL-E / Stable Diffusion / LLaVA / GPT-4V 全靠它6。

§ 01痛点 · 一张图一个 label 太浪费

2020 年之前, 视觉训练的标准流程是 ImageNet 范式:

1. 雇人标几千几万张图 (每张图标一个 class)
2. 训 CNN, 输出 1000 类的概率分布
3. 想换任务 (医学图 / 卫星图 / 商品图) → 重新标 + 重新训

浪费在哪

看下面这张图。它的 ImageNet label 是 "golden_retriever"。但这张图实际包含的信息远多于此 —— "金毛犬 / 在草地上 / 阳光 / 戴红色项圈 / 转头看镜头 / 年轻"。所有这些信息全被压扁成了一个 label, 监督信号极度稀疏。

同一时间, NLP 那边 GPT-2/3 已经证明: 不用人工标, 直接从互联网原文学, 反而能学到更通用、更可迁移的表示7。视觉为什么不能也来这一套？

问题: 视觉没有"原文" —— 图不能像维基百科一样直接读。

OpenAI 注意到一个事实: 互联网上有数十亿张图配着 caption（alt 文本、Instagram 配文、Flickr 描述、新闻图说）。这些 caption 不是为训练写的, 是网友自然产生的"图说话"。这就是图像的"原文"。

§ 02架构 · 双塔 + 末端对齐

CLIP 的工程设计 —— 两个独立的 encoder, 各跑各的, 最后只在末端算一个点积4:

关键设计:

• 两边 encoder 完全独立, 训练和推理都互不相干
• 各自最后一层投影到同一维度 (512), 然后 L2 normalize 让点积 = cosine similarity
• 所有跨模态交互只发生在最后那一个点积 —— 没有 cross-attention, 没有 fusion layer

为啥不用一个大 Transformer 直接吃图+文本

有两种主流多模态架构: Dual-tower (CLIP) 和 Single-tower (LLaVA / GPT-4V)。前者推理高效 (一边可缓存)、训练目标干净, 但表达力受限; 后者表达力高 (能生成、能问答) 但推理贵。实际系统经常混合用 —— CLIP 当视觉前端 + LLM 当推理后端。详见 dual-tower-architecture。

§ 03对比学习 · 把整个 batch 当训练单元

CLIP 的训练目标 —— 这是真正的"魔法所在"3:

核心机制 —— 三句话讲完:

• 一个 batch 里 N 个图文对, 都是网上天然配对的 (image_i 跟 text_i 是网友写在一起的)
• 对角线那 N 对是正样本 (image_i ↔ text_i)
• 非对角线 N²-N 对是负样本 (image_i ↔ text_j where j≠i) —— 用 batch 内其他文本当负样本, 不用单独构造8

# 简化伪代码
img_embeds = image_encoder(images)        # (N, 512)
txt_embeds = text_encoder(texts)          # (N, 512)

# normalize 到单位向量
img_embeds = img_embeds / img_embeds.norm(dim=-1, keepdim=True)
txt_embeds = txt_embeds / txt_embeds.norm(dim=-1, keepdim=True)

# 算 N×N 相似度矩阵
sim = img_embeds @ txt_embeds.T * temperature.exp()    # temperature 可学

# 标签就是 [0, 1, 2, ..., N-1] (对角线位置)
labels = torch.arange(N)

# 两边对称做 cross entropy
loss = (F.cross_entropy(sim, labels) +
        F.cross_entropy(sim.T, labels)) / 2
为什么 batch 要这么大

CLIP 用 batch_size = 32768。原因: 对比学习的"难度"取决于负样本数量。batch=32 时, 每个图只跟 31 个其他文本对比, 很容易学; batch=32768 时, 模型必须学到极精细的语义区别 (比如 "金毛" vs "拉布拉多") 才能跑赢这 32767 个负样本。负样本越多, 模型被逼着学得越细。

工程代价: 这要求分布式 + gradient checkpointing + 大显存。CLIP 还学了一个可学的温度参数9控制 softmax 尖锐度。后来 SigLIP12 (2023) 用 sigmoid loss 替代 softmax-style, 据说能在小 batch 上跑得动, 目前 (2024-2025) 是开源多模态事实标准。

§ 04数据 · 4 亿对子的力量

CLIP 真正的"魔法"不在算法, 在数据规模2:

WIT (WebImageText) 是 OpenAI 自己爬的数据集, 没有公开。但 LAION-400M / LAION-5B 后来开源复刻了这条路 —— 现在所有开源 CLIP / 开源 Stable Diffusion 都是建立在 LAION 之上。这是社区给 OpenAI 闭源 WIT 的最大回应。

跟 gpt-3 是同一个故事 —— 数据规模 + 弱监督 > 数据精标 + 强监督。GPT-3 证明语言可以这么训, CLIP 证明视觉-语言对齐也可以这么训。Bitter lesson 再次胜出。

§ 05Zero-Shot · 分类变成"哪句话最配"

训完之后, 用法极其优雅 —— 不用 fine-tune, 不用任何标注。想分什么类, 把类名包成一句话喂进去就行5:

这个用法看起来简单, 但背后的逻辑是革命性的:

传统做法: 类别 = 模型最后一层权重的 1000 个 row, 训完就定死
CLIP 做法: 类别 = 你写的几句话, 想换就换

Prompt engineering 来到视觉

既然类别用文本描述, "怎么描述"也成了变量。CLIP 论文实验显示:

• {class} 只写类名 → ImageNet 64.2%
• a photo of a {class} → 66.7%
• a photo of a {class}, a type of pet → 68.3%
• 80 个模板 ensemble → 70.7%

为啥? 网上的 caption 大多是完整句子, 不是孤立的词。CLIP 的文本 encoder 对完整句子更熟。视觉模型的效果, 现在要靠文本提示词的写法10 —— 这就是 open vocabulary11 的全部含义: 类别集合不再由模型权重定义, 由用户用自然语言定义。

§ 06下游 · 谁在用 CLIP

CLIP 真正的影响力远超它自己的 benchmark。2022 年之后整个生成 AI 浪潮, 底层都依赖 CLIP 立起来的那个 joint embedding space6:

① 生成模型的 conditioning

DALL-E 2 / Stable Diffusion 都用 CLIP 文本 encoder 把 prompt 转成 conditioning 向量, 喂给 diffusion / flow matching 模型生成图。没有 CLIP 就没有"a photo of an astronaut riding a horse"这种 text-to-image 能力。

② LLM 的视觉前端

LLaVA / GPT-4V / Qwen-VL 用 CLIP 图像 encoder 把图转成 visual token, 然后跟文本 token 一起喂给 LLM。所有"看图问答"产品的视觉前端基本都是 CLIP。

③ 大数据集的过滤器

LAION-5B 自己就是用 CLIP 过滤出来的 —— 爬一堆原始图文对, 用 CLIP 算"图文是否匹配", 留下高匹配的, 扔掉噪声。这是个非常 meta 的递归: 训出来的 CLIP 用来筛数据, 筛出来的数据训下一代 CLIP / SD。

④ Open-vocabulary 检测 / 分割

传统目标检测固定类别 (COCO 80 类)。OWL-ViT / OpenSeed / Grounded-SAM 把 CLIP 文本端跟 detection / segmentation 头对齐, 实现 "找出图中所有 '红色花瓶'" 这种自由查询。检测也变 open vocabulary 了。

§ 07历史定位 · 视觉的 GPT-3 时刻

CLIP 在 2021 年发布时, 圈内反应是分层的:

• 视觉传统派: "Zero-shot 跑不过 finetune, 意义不大"
• 多模态人: "终于有可用的 joint embedding 了"
• 生成式人: "这是 DALL-E 想要的那个 prior"

2022 年 Stable Diffusion 公开发布后, 大家才反应过来 —— CLIP 不只是个分类器, 它是整个生成 AI 浪潮的地基。

CLIP 跟 GPT-3 是 OpenAI 同期的两条主线, 共享同一个哲学:

这两条线汇合后, 才有了 2022-2024 的 ChatGPT / Stable Diffusion / GPT-4V / Sora 等等等。CLIP 是视觉端的 GPT-3 时刻。

引用 / 术语表

  1. ImageNet 范式 · 2009-2020 视觉训练的标准做法：人工标注几百万张图, 每张一个 class label, 训 CNN 输出 K 类概率分布。优点是干净, 缺点是贵 + 类别固定 + 单一监督信号信息量太低 (一张图配一个 label 信息浪费)。
  2. WIT · WebImageText 400M · OpenAI 自己从互联网爬的图文对数据集, 4 亿对子, 没开源。后来 LAION 社区复刻成 LAION-400M / LAION-5B 开源。所有现代开源多模态 (Stable Diffusion / OpenCLIP) 都建在 LAION 上。
  3. 对比学习 · contrastive learning · 不预测具体类别, 只学"两个东西配不配"。CLIP 用 in-batch negatives —— batch 里其他文本当负样本, 不用单独构造。一个 batch 同时贡献 N² 个判断。详见 contrastive-learning。
  4. Dual-tower architecture · 双塔 · 两个独立 encoder (图像 + 文本), 各自最后投影到同一维度, 跨模态只在末端点积发生。推理时一边可以预先 encode 缓存, 适合检索类应用。详见 dual-tower-architecture。
  5. Zero-shot classification · 不 fine-tune, 不看任何下游标注 —— 把类名包成 "a photo of a {class}" 喂给文本 encoder, 跟图像 embedding 算相似度, 最高的就是预测。类别由文本定义, 不由权重定义。详见 zero-shot-image-classification。
  6. CLIP joint embedding space · CLIP 真正的价值不在它自己的分类准确率, 而在它训出来的那个"图像和文本对齐"的向量空间。DALL-E / Stable Diffusion / LLaVA / GPT-4V 等所有后续多模态都依赖这个空间。
  7. GPT-3 弱监督路线 · OpenAI 在 2020 年用 GPT-3 证明: 不用人工标注, 直接拿互联网原文做 next-token prediction, 学出来的表示能 zero-shot 任何 NLP 任务。CLIP 是这条思路在视觉端的复制 —— 弱监督 + 大数据 + 大模型 = 通用表征。详见 GPT-3 paper。
  8. In-batch negatives · 批内负样本 · 对比学习的核心工程技巧。一个 batch 有 N 个正样本对, 其他 N-1 个文本就自然成为每个图的负样本。N² 个判断, N 个正例 + N²-N 个负例, 不需要单独构造负样本。Batch 越大负样本越多, 模型被逼着学到更细的语义区别。
  9. Temperature parameter · 温度参数 · CLIP 损失里有一个可学的温度标量, 控制 softmax 的尖锐度。初始化 0.07 左右, 训练中模型自己调。温度低 → softmax 尖锐 → 强调最近的正样本; 温度高 → softmax 平坦 → 让梯度信号更均匀。
  10. Prompt engineering · 提示词工程 · 既然 CLIP 类别用自然语言描述, 怎么写描述也成了变量。{class} 只 64.2%, a photo of a {class} 66.7%, 多模板 ensemble 70.7%。NLP 的 prompt engineering 在 CLIP 这里复刻了。
  11. Open vocabulary · 类别不由模型权重定义, 由文本定义 —— 想加什么新类写句话就行, 不需要重训模型。这是 CLIP 给视觉带来的最大概念跃迁: 从"模型决定类别集" 到"用户用自然语言定义类别集"。
  12. SigLIP · sigmoid-loss CLIP · 2023 Google 提出的 CLIP 变种, 用 sigmoid loss 替代 softmax-style 对比损失。优势: 对 batch size 不敏感 (小 batch 也能训), 对负样本不需要 N² 那么多。是当前 (2024-2025) 开源多模态的事实标准。
