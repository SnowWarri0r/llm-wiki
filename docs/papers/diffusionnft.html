DiffusionNFT ·
在唯一的前向上做 RL。

清华 / NVIDIA 等 · Online Diffusion Reinforcement with Forward Process · arXiv 2509.16117 · ICLR 2026

给扩散模型做在线 RL 的老路,是把反向采样切成多步 MDP 再套 GRPO——却继承一身毛病。DiffusionNFT 抓住一个事实:前向加噪只有一条、反向去噪有无数条,于是把 RL 直接做在前向上。用奖励切正负样本,正负之差就是该走的方向;整件事退化成一个监督式 flow-matching 损失。

TL;DR

扩散的似然算不出来1,所以没法照搬 GRPO。FlowGRPO 的绕法是把反向去噪切成多步马尔可夫过程(每步高斯可算概率),但落三个坑:锁死一阶 SDE solver、偏离前向过程、CFG 要两套模型2。DiffusionNFT 换思路——前向加噪唯一、反向去噪无数,把 RL 搬回前向 flow matching3:用奖励 r 把样本软切成正分布 π⁺∝r·π_old / 负分布 π⁻∝(1−r)·π_old4,正负速度之差 Δ 就是隐式的强化引导方向(CFG 不过是它的离线版)6。一个模型隐式同时装正负速度,RL 揉进一个监督损失7:免似然、任意 solver、只要干净图。SD3.5-M 上 GenEval ~1k 步逼 0.98,比 FlowGRPO 快 3–25×10。

§ 01似然墙：扩散为什么不能照搬 GRPO

RL 微调 LLM 顺手,是因为策略梯度要的"模型给这个样本的概率"唾手可得:每个 token 输出的就是概率分布,一条回答的概率 = 逐 token 概率连乘。奖励一来,乘上对数概率的梯度就更新。policy gradient / PPO / GRPO 都吃这口。

扩散没有这种现成概率1。一张图是从噪声经几十步去噪来的,"模型生成这张图的概率"要对所有路径积分,只能靠概率流 ODE 或 SDE 变分界昂贵地近似。这就是似然墙。主流绕法(FlowGRPO / DanceGRPO):把反向采样看成多步马尔可夫决策过程,每步状态转移是个高斯——高斯概率可算,于是能套 GRPO。但这条路有代价。

§ 02关键洞察：前向只有一条，反向有无数条

DiffusionNFT 的转身就一句话3:前向加噪过程是唯一且良定义的(图一点点变成噪声,只此一条);反向去噪有无数条路(同一团噪声能还原出无数种采样轨迹)。 FlowGRPO 偏要在那"无数条反向"里挑一条来训,才落得锁死 solver、偏离前向。既然前向唯一,那就把 RL 做在前向上——而前向正是 flow matching 的训练地盘,等于把 RL 接回最朴素的监督训练。于是三个好处一起来:免似然估计、数据可用任意黑盒 solver 采、只存干净图不存采样轨迹9。

§ 03奖励切正负：两堆之差就是该走的方向

怎么在前向上注入奖励?给每张采来的图打个奖励 r∈[0,1],按 r 把样本软切成两堆4:正分布 π⁺∝r·π_old(高奖励权重大)、负分布 π⁻∝(1−r)·π_old(低奖励权重大)。两堆各自对应一个去噪速度,正速度减负速度得到方向 Δ——它指向"怎么改能更高分",就是强化引导6。

§ 04一个模型隐式装正负，RL 揉进监督损失

不真去训两个模型——这套"奖励切正负 + 前向监督"的做法就是负样本感知微调(NFT)5。一个 velocity 网络 v_θ 通过参数插值同时表示正负两副动态(都围绕冻结的旧模型 v_old)7:正速度 v_θ⁺=(1−β)v_old+β·v_θ、负速度 v_θ⁻=(1+β)v_old−β·v_θ。损失是纯监督 flow matching(不是 policy gradient)8:L=E[ r·‖v_θ⁺−v‖² + (1−r)·‖v_θ⁻−v‖² ]——高奖励样本(r→1)把正速度拉准、低奖励(r→0)把负速度拉准,两股梯度合起来把 v_θ 推向 Δ 指示的方向。

数字例子 · 正负速度对称夹住 v_old

取标量简化(真实是逐维速度向量):旧模型 v_old=1.0、模型当前输出 v_θ=1.4、引导强度 β=0.5。

v_θ⁺ = (1−0.5)·1.0 + 0.5·1.4 = 0.5 + 0.7 = 1.2     (正速度)
v_θ⁻ = (1+0.5)·1.0 − 0.5·1.4 = 1.5 − 0.7 = 0.8     (负速度)

自检①  v_θ⁺ + v_θ⁻ = 1.2 + 0.8 = 2.0 = 2·v_old   → 正负永远对称夹住 v_old ✓
自检②  v_θ⁺ − v_θ⁻ = 0.4 = 2β·(v_θ − v_old) = 2·0.5·0.4 ✓   → 缝隙 = 离旧模型多远
最优解  v_θ* = v_old + (2/β)·Δ ，小小的改进方向 Δ 被 2/β=4 放大着学进去

✓ 含义:模型只动一个量 v_θ,正速度抬到 1.2、负速度对称压到 0.8。采样时只取被引导的正速度 v_θ⁺——等于把强化引导内建进了一个模型,不再要 CFG 的两套模型/两遍前向。这也是它跟 DPO 的亲缘:都靠正负对比把 RL 折成类监督损失,只是 DPO 在 LLM 对数似然上、NFT 在扩散速度场上。

§ 05结果：又快又强，CFG-free 还赢 CFG

在 SD3.5-Medium(2.5B,LoRA r32,512²)上训12。GenEval 从无 CFG 的 0.24、开 CFG 的 0.63,到 FlowGRPO 跑 5k+ 步的 0.95;DiffusionNFT ~1.7k 步就 0.94、~1k 步逼 0.98,墙钟比 FlowGRPO 快 3–25×10。而且它训的是个 CFG-free 的 omni-model:起点因为没 CFG 引导而很低,却一路反超开 CFG 的基线,PickScore / HPSv2.1 / Aesthetics / ImageReward 全面更高,OOD 上还压过更大的 SD3.5-Large 和 FLUX.1-dev11。

DiffusionNFT 把 LLM 的 GRPO/PPO/DPO 那条 RL 线接到了扩散上,但绕开了"反向离散化"的弯路——靠的是 NFT(奖励切正负、前向 flow matching、单模型隐式参数化)和 强化引导(把 CFG 重新理解成"离线版的奖励引导")。它跟 DPO 是表亲(正负对比、RL 折成类监督),跟 Krea 2 的多奖励 RL 是同领域不同路。一句话:反向去噪千条路、前向加噪只一条;在那唯一的前向上做 RL,似然墙、solver 限制、CFG 两模型一次性全绕开——正负样本之差就是该走的方向。

引用 / 术语表

  1. 似然墙 · 策略梯度要"模型给样本的概率",LLM 逐 token 现成、扩散只能靠概率流 ODE / SDE 变分界昂贵近似;所以没法直接套 GRPO。
  2. FlowGRPO 的三个坑 · 把反向采样切成多步 MDP 以套 GRPO,代价:① 耦死一阶 SDE solver;② 偏离前向、易退化成级联高斯;③ CFG 变两模型优化。
  3. 前向唯一 / 反向无数 · 前向加噪只一条且良定义;反向去噪有无数采样轨迹。NFT 把 RL 放到唯一的前向上,接回 flow matching 的监督训练。
  4. 奖励软切正负 · 给样本打 r∈[0,1],正分布 π⁺∝r·π_old(高奖励权重大)、负分布 π⁻∝(1−r)·π_old(低奖励权重大)。
  5. NFT · 负样本感知微调 · 在前向过程上、用正负对比把扩散在线 RL 折成加权监督 flow-matching 损失;免似然、只要干净图。
  6. 强化引导 / Δ · 正速度减负速度 Δ=v⁺−v⁻,指向"怎么改更高分"。CFG 的 cond−uncond 是它的离线固定版;两者都是"差向量当引导"。
  7. 隐式参数化 v_θ± · 单模型 v_θ 同时表示正负:v_θ⁺=(1−β)v_old+β·v_θ、v_θ⁻=(1+β)v_old−β·v_θ,对称夹住 v_old(相加=2v_old)。
  8. 监督损失（非 policy gradient） · L=E[ r·‖v_θ⁺−v‖²+(1−r)·‖v_θ⁻−v‖² ];最优解 v_θ*=v_old+(2/β)·Δ。RL 信号靠 r vs 1−r 的对比权重揉进监督目标。
  9. 免似然 · 任意 solver · 只存干净图 · 训练与采样解耦:数据可用任意黑盒 solver(ODE/高阶) 采,不存轨迹只存 x₀,全程不估似然。
  10. 3–25× 效率 · 墙钟比 FlowGRPO 快 3–25×;GenEval ~1k 步逼 0.98,FlowGRPO 5k+ 步才 0.95。
  11. CFG-free omni-model · 训出单个不依赖 CFG 的模型,起点低却反超开 CFG 基线;OOD 上压过更大的 SD3.5-Large、FLUX.1-dev。
  12. SD3.5-M / LoRA r32 · 基座 Stable Diffusion 3.5 Medium(2.5B)、512²、LoRA(α=64,r=32)。奖励含 GenEval/OCR(规则) + PickScore/HPSv2.1/Aesthetics/ImageReward(模型)。
