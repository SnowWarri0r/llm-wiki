ODE vs SDE ·
跟着箭头走，还是边走边被踹。

Ordinary / Stochastic Differential Equation · 流模型与扩散模型的底层语言

ODE：给你一张"风场地图"，每点都有箭头说往哪走——丢颗弹珠顺着滑，同起点跑一万次走同一条路（确定）。SDE：同一张风场，但每一瞬间额外随机踹你一脚——同起点跑两次，两条不同的路（随机）。流模型走 ODE，扩散模型走 SDE，而它俩其实是同一团概率云的两种看法。

TL;DR

ODE1：dx/dt = f(x,t)——"现在在哪，就决定下一步往哪走多快"，跟着速度场（风场）积分出唯一轨迹，确定（同起点→同一条路）。SDE2：dx = f·dt + g·dW——ODE 再加一项每瞬间的随机踹（dW3 布朗运动），随机（同起点→每次不同→一团落点）。两项分别叫drift（漂移，确定推力）和 diffusion（扩散，噪声）。关键桥4：每个扩散 SDE 都配一个概率流 ODE，两者每一刻的分布完全相同——SDE 追一个抖动的个体，ODE 追整团密度的平滑水流，同一团云。落点：flow-matching = 纯 ODE；diffusion = SDE（也能当 ODE 跑 → DDIM 少步快采）。数值解：ODE 用 Euler，SDE 用 Euler–Maruyama6，噪声乘 √dt 不是 dt7。

§ 01ODE · 跟着风场走的确定轨迹

先把符号翻译成大白话：给定你现在在哪，有条规则告诉你下一瞬间往哪挪、挪多快。

dx/dt = f(x, t)
现在位置变化的速度 = 一个只跟"当前位置和时间"有关的函数

最好的类比是风场 / 流场地图：空间里每个点都钉着一个小箭头，说"在这儿就往这个方向、这个速度走"。丢一颗弹珠进去，它顺着箭头滑，整条轨迹靠一路积分（累加每步的小位移）得出。f 就是那个速度场。

核心性质：完全确定。同一个起点跑一万次，走出来一模一样的路——没有任何随机。像跟着 GPS 一句句指令走，或弹珠在形状固定的山谷里滚到底。

放进你学的

flow-matching 学的就是这个速度场 f；采样时从噪声出发解一个 ODE 一路积分到数据（"probability flow ODE"）。所以 flow-matching 的采样本质就是"在学好的风场里放颗弹珠，看它滑到哪"。

§ 02SDE · ODE 再加每步随机踹一脚

SDE 就是 ODE，但每一瞬间额外加一个随机抖动：

dx  =  f(x,t)·dt   +   g(t)·dW
       └ drift 漂移 ┘     └ diffusion 扩散 ┘
       跟 ODE 一样的确定推力   每一刻灌进的随机噪声
  • drift（漂移）：那个确定推力，跟 ODE 那项一模一样——"大方向往哪走"。
  • diffusion（扩散）：dW 是每瞬间的随机踹一脚（布朗运动 / 维纳过程，一连串微小高斯抖动），g(t) 是踹多狠。

类比是醉汉走夜路回家：大体朝家走（drift 的确定拉力），但每一步都在随机踉跄（diffusion 的噪声）。于是——同一个起点跑两次，走出两条不同的路；你拿到的不是一条轨迹，是一团可能落点（一个分布）。也像溪水里的叶子：水流整体冲它下去（drift）+ 湍流不停晃它（diffusion）。

放进你学的

扩散模型就是这个：前向过程是个 SDE，drift 把图往 0 拉、diffusion 不停灌高斯噪声，把图糊成纯噪声；反向过程也是个 SDE（把噪声一点点抽走）——那就是采样。它学的是 score 函数（噪声该往哪去），而不是 flow-matching 直接学的速度场。

§ 03桥 · 同一团云的两种看法

这是最值钱的一点。Song 等人（2021）证明：每个扩散 SDE，都对应一个"概率流 ODE"，两者在每个时刻的分布完全一样。于是同一个模型，采样可以走两条路。

类比：一群人散场出体育场。SDE 盯着某一个人在人潮里乱撞的抖动轨迹；ODE 盯着整团人群密度的平滑水流（平均的洋流）。两者描述的是同一团正在散开的云——一个追抖动的个体、一个追整体的河。

采样走哪条SDE 路（随机）ODE 路（确定）
走法每步注入新噪声纯跟着箭头走
同起点结果每次不同 → 多样、更稳每次一样 → 可复现、可反推
步数 / 速度慢（要小步补偿噪声）快（能少步）· DDIM 走这条
附赠噪声每步重随机 → 冲掉累积误差可逆 → 精确似然 + latent 插值/编辑

所以不是 ODE 全面赢：SDE 的噪声能纠错、给更多样更鲁棒的样本，ODE 确定、可逆、能少步快采。一句话——多样性/鲁棒 ↔ 速度/可逆 的取舍。（这条线在 ML 落地上更细的对比见 ode-vs-sde 概念页：训练目标、推理步数、为什么 SD3/Flux 都转向 flow-matching。）

§ 04怎么数值解 · 用真数字砸实

"解"这俩方程，计算机做不到连续积分，只能切成一小步一小步地累加。光看公式太抽象，我们拿一个具体例子，把数字一步步走出来。

设定（两边共用）

取一个最简单的 drift：f(x) = −x（把值往 0 拉，像去噪——离 0 越远拉得越狠）。步长 dt = 0.5，噪声强度 g = 2，所以 √dt = √0.5 ≈ 0.707。两边都从 x = 10 出发。

① ODE → Euler：每步 x += f(x)·dt

没有噪声，每步就是"当前位置 + 速度×步长"。f(x)·dt = (−x)(0.5)，等于每步把 x 砍掉一半：

步当前 xdrift = f(x)·dt = −x·0.5新 x
010.00−5.005.00
15.00−2.502.50
22.50−1.251.25
31.25−0.6250.625 → 平滑奔 0

关键：这张表是写死的。跑 1 次、跑 1 万次，第一步永远 10→5、第二步永远 →2.5。确定，没有"运气"。

② SDE → Euler–Maruyama：x += f·dt + √dt·g·z

多一项噪声：√dt·g·z = 0.707×2×z = 1.414·z，其中 z 是每步现抽的标准正态随机数 N(0,1)。同样从 10 出发，抽到的 z 不同，路就不同。跑两次看看：

跑 A · 步起点 xdrift −x·0.5这步抽到 z → 噪声 1.414·z新 x
010.00−5.00z=+0.8 → +1.136.13
16.13−3.07z=−0.3 → −0.422.64
22.64−1.32z=+1.1 → +1.562.88
跑 B · 步起点 xdrift −x·0.5这步抽到 z → 噪声 1.414·z新 x
010.00−5.00z=−1.2 → −1.703.30
13.30−1.65z=+0.5 → +0.712.36
22.36−1.18z=−0.7 → −0.990.19

对比就出来了：第一步，ODE 永远落 5.00；SDE 跑 A 落 6.13、跑 B 落 3.30——同一个起点、同一条规则，只因为每步那个随机 z 不同，整条路就岔开了。把无数次跑 SDE 的落点收集起来，就是一团分布。

为什么噪声乘 √dt 不是 dt

把步长 dt 砍一半（0.5→0.25）：drift 每步的贡献也老实砍一半（−x·0.25）；但噪声项只缩到 √0.25/√0.5 = 1/√2 ≈ 0.71 倍——比 drift 顽固。这正是布朗运动的签名：随机游走走的距离按 √时间 长，不是按时间长（走 4 倍步数，确定位移 ×4，但随机扩散只 ×2）。所以 SDE 想算准，步长得切得比 ODE 更碎，这也是扩散采样比流模型慢的根。

§ 05一句话收口

ODE = 风场里放颗弹珠，确定地滑出一条路；SDE = 同一个风场，但每步随机踹一脚，滑出一团落点。两者由"概率流 ODE = 扩散 SDE 的同分布替身"这座桥连起来：flow-matching 站 ODE 这头（直接学速度场、少步快采），diffusion 站 SDE 那头（学 score、噪声纠错更稳），而它能切到 ODE 模式（DDIM）借速度。数值上 ODE 用 Euler、SDE 用 Euler–Maruyama，记牢那个 √dt——它是"随机比确定更顽固"的全部来历。

引用 / 术语表

  1. ODE · 常微分方程 · dx/dt=f(x,t)。给定起点 + 速度场，轨迹唯一确定。类比：风场里放颗弹珠顺箭头滑。flow-matching 的采样就是解它。
  2. SDE · 随机微分方程 · dx=f·dt+g·dW。ODE 加一项每瞬间的随机踹，结果随机。同起点跑两次两条路。扩散模型的前向/反向都是它。
  3. drift / diffusion / dW · drift=确定推力（同 ODE）；diffusion=噪声强度 g；dW=布朗运动/维纳过程，一串微小高斯抖动。
  4. 概率流 ODE（桥） · Song 2021：每个扩散 SDE 配一个 ODE，两者每一刻边际分布相同。SDE 追抖动个体、ODE 追密度水流，同一团云。
  5. score 函数 · ∇log p_t(x)，"概率密度上升最快的方向"。SDE/扩散学它来构造反向漂移；对照 flow-matching 直接学速度场。
  6. Euler / Euler–Maruyama · 数值解法。ODE：x+=f·dt。SDE：x+=f·dt+√dt·g·z（z∼N(0,1)）。后者多了随机项。
  7. √dt 缩放 · 噪声项乘 √dt 而非 dt。源于随机游走"按 √时间扩散"：步长砍半，drift 贡献砍半但噪声只缩 1/√2。SDE 要更小步才算准。
  8. DDIM · 把训练好的扩散模型当概率流 ODE 来确定性采样，少步、可复现、可做 latent 插值。即"扩散借 ODE 的速度"。
