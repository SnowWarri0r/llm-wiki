Interactive video world modeling

动作、记忆与实时反馈.

Jiuming Liu · Chaojun Ni · Mengmeng Liu · Chensheng Peng · Fangjinhua Wang · Sitian Shen · Marc Pollefeys · Masayoshi Tomizuka · Ayush Tewari · Per Ola Kristensson

一次生成视频像提前写好剧本；交互式世界模型则要在你每次操作后继续演，而且不能忘记刚才发生过什么。

TL;DR

交互式 world model01 是一个闭环03生成器：历史说明世界走到哪里，当前动作决定下一步往哪走。现有方法主要在动作注入、长期记忆与少步/缓存加速之间做取舍。

SECTION 00先别背模型名：想象你走进房间再回头

你看见红沙发，按 W 前进；向右转后看见书架；继续走，再回头。模型要同时做到动作生效、沙发还在、反馈及时。

one-shot video control 与 multi-shot interaction04 的区别是：前者在开拍前一次给完提示词与轨迹，后者每轮都能接收新动作。后者不是简单循环前者，因为每轮生成误差都会进入下一轮历史，而且每轮都有延迟预算。

SECTION 01定义：历史告诉模型“到了哪”，当前动作告诉它“往哪走”

论文先用 POMDP02 列出世界的五样东西。这个名字先不用背，五个字母只是一张清单：

c_t文字、声音、布局、相机轨迹等额外条件
pφ模型给各种合理未来分配概率

h当前视觉特征

⊙对应位置逐元素相乘

动作调制的一般写法：

若 h=[2,4]、γ(a)=[1.2,.5]、β(a)=[.1,−.2]，则 h′=[1.2×2+.1,.5×4−.2]=[2.5,1.8]：动作放大第一维，同时压低第二维。

SECTION 04长期一致性：从“带最近照片”到“维护一张世界地图”

最简单的历史条件去噪器可记作：

εθ扩散去噪器输出的噪声或去噪方向，不是概率
o_t当前要生成的观察
o_≤t−1到上一时刻的历史观察
a_≤t−1到上一时刻的动作

竖线只表示“生成当前观察时把历史当条件”；整条式子不是条件概率。最近帧很直观，但固定窗口会把更旧的世界状态挤出去。

RELIC 把短期未压缩 KV cache 与长期压缩空间 memory 分开；Spmem 把静态点图与动态参考帧分开；MosaicMem 混合显式几何和隐式视频记忆。没有一种存储同时免费获得几何、动态、无限容量和实时更新。

SECTION 05Forcing 到底 force 什么：训练时拿哪种历史当上下文

普通 Teacher Forcing 每步都给真历史，部署却只能看自己输出。小错回流并累积，这叫 exposure bias08。各种 Forcing 方法主要在改变“训练时给过去帧什么”。

Solaris 先用 Diffusion Forcing 把双向多人模型改成因果生成器，再用 Checkpointed Self Forcing 在自己的生成历史上训练。

SECTION 06为什么记得越多，模型可能越不听新动作

先定义教学旋钮 λ∈[0,1]：λ 是历史权重，1−λ 是当前动作权重。下面只是解释冲突，不是论文训练公式：

λ=.8 时历史 / 动作各占 80% / 20%；降到 .3 后动作变强，旧场景也更容易被改写。Astra 给条件帧加随机噪声，故意削弱历史影响；Context Forcing 让带 memory 的 student 在自生成长轨迹上学习平衡。

因此只看 FID 会误判：几乎复制上一帧的模型可能非常“稳”，却完全没有执行动作。世界模型评测必须同时包含动作跟随。

允许非线性时间对齐后比较两条路径整体差异。

新动作是否产生预期状态变化。

换视角后空间结构是否保持。

MEMORY ↑

离开再回来，旧对象与布局是否恢复。

反馈速度；需同时报告硬件、分辨率和采样步数。

SECTION 09研究趋势：沿通用性、动态性与感官三条轴扩张

AXIS 01专家 → 通用

从一个游戏或驾驶域，走向多场景 foundation world model；动作语义统一和数据清洗随之更难。

AXIS 02静态单人 → 动态多人

不只让相机移动，还要维护对象变化、其他智能体行动和交互后果。

AXIS 03视觉 → 多感官

文字、声音、力、遥操作加入闭环；不同模态的控制频率和精度不同。

SECTION 10五个未解难题：从数据一直到用户界面

01动作标注

真实机器人与驾驶数据贵；latent action 又可能混入相机和外观偏差。

02反事实因果

“看起来合理”不等于动作真的导致正确后果，安全场景尤其危险。

03小时级一致

需要短期动作细节与长期地点、身份、世界状态的分层记忆。

04物理理解

模仿像物理的运动，不等于懂刚体、软体、力与碰撞。

05直接操作10

让对象可见、操作可逆、反馈即时，而不是每次靠长 prompt 描述。

SECTION 11以后读任何新世界模型，先问这三句

QUESTION 01动作怎样进入？

开场给一次还是每轮更新？文字、token、数值位姿还是显式渲染控制？

QUESTION 02历史存在哪里？

最近帧、KV cache、latent memory、显式 3D，还是根本没有持久状态？

QUESTION 03速度怎样得到？

少步蒸馏、缓存、并行、量化？报告是否同时给硬件、分辨率和端到端延迟？

再检查评测是否覆盖动作、记忆、几何、画质与闭环任务。若只给 FID/FVD，它仍更像“可控视频生成质量报告”，不足以单独证明世界模型能力。

术语落点

  1. world model：根据当前观察与动作预测世界之后怎样变化的模型。
  2. POMDP：智能体只能看到部分世界状态的决策过程；相机画面不等于完整世界。
  3. closed loop：模型输出会影响下一动作或成为下一轮输入，错误也会反馈回来。
  4. one-shot / multi-shot：条件一次给完，或生成途中可反复加入新动作。
  5. latent memory：不保存原始像素，而保存编码后的历史视觉 token 与状态线索。
  6. Surfel：带位置、法线、颜色等信息的局部表面元素，可用于索引历史视角。
  7. 3DGS：用许多带位置、形状、颜色与透明度的 3D 高斯表示场景并快速渲染。
  8. exposure bias：训练总看真历史，推理却看自己的不完美输出，由此产生输入分布错位。
  9. distillation：把慢而强的 teacher 能力迁移到少步、更快的 student。
  10. direct manipulation：对象和动作可见，操作快速可逆，并立即得到反馈的交互原则。
