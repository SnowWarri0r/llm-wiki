FIELD NOTES · POSITION ENCODING

Rotary Position
Embedding.

Jianlin Su · Yu Lu · Shengfeng Pan · et al. · 2021

不在输入层加位置向量，而是在每次 attention 时按位置旋转 Q 和 K —— 让点积天然包含相对位置信息。LLaMA、Mistral、Qwen 全在用。

TL;DR

Transformer 的 attention 对顺序无感1。原版 sinusoidal PE 在输入层一次性"加"位置，往深走信号就淡了2。RoPE 换了一种做法 —— 在每层 attention 计算时，按位置旋转 Q 和 K3。旋转之后做点积，结果里天然包含两个 token 的相对位置4。零额外参数，每层都有位置信号，远距离自动衰减，跟 KV cache5 完美兼容。2024-2025 几乎所有新 LLM 都用 RoPE 或它的变体。

§ 01为什么需要新的 PE

Attention Is All You Need 那篇用 sinusoidal PE 解决"attention 不认识顺序"的问题2。做法是给每个 token 的 embedding 加一个位置向量，然后丢进 Transformer。

这套方案用了好几年，但三个问题越来越明显：

1 · 位置信息随层数稀释。位置向量只在第 0 层加了一次，过了几层 attention + FFN 之后，位置信号在 residual stream 里被语义信号淹没。模型后半段几乎不知道 token 在哪。

2 · 编的是绝对位置。模型收到的是"你在 position 5""你在 position 8"，要自己做减法推出"你俩差 3 个"。能学会，但不直接。而自然语言里"差几个位置"几乎总是比"在第几个位置"更有意义4。

3 · 长上下文外推差。训练时见过 512 长度，推理时给 2048 就开始抖。虽然 sin/cos 理论上能外推，实际效果不好。后来的 Learned PE（GPT-2 / BERT）直接训 embedding，更灵活但完全不能外推 —— 没见过的 position id 没有对应 embedding。

RoPE 是第三条路 —— 不加不偏，而是旋转3。它不改 attention 公式，只在算 Q·K^T 之前把 Q 和 K 各自按位置转一下。

§ 02核心直觉 · 时钟指针

忘掉所有数学，先建立一个直觉。

想象你手里有两根指针，一根代表 token A（在句子里第 m 个位置），一根代表 token B（第 n 个位置）。

RoPE 做的事极简 —— 把 A 的指针转 m 圈，把 B 的指针转 n 圈。

现在你量两根指针之间的角度差：m − n 圈 = 相对位置4。

你不需要知道 m 和 n 各自是多少 —— 光看两根针的夹角就知道它们差几个位置。而且不管绝对位置怎么变（position 5 vs 8 跟 position 105 vs 108），差 3 个位置的指针夹角永远一样。

这就是"旋转编码相对位置"的全部直觉。剩下的只是把这个 2D 指针扩展到 d 维。

§ 03怎么做 · 多频率旋转

d 维向量不是"整体旋转"（那没意义）。实际做法：把 d 维拆成 d/2 对 2D 子空间6，每对用不同的频率旋转。

d_model = 64  →  32 对 2D 子空间

每对 i 的旋转频率:  θ_i = 1 / 10000^(2i / d)

position m 的 token:
  第 i 对子空间旋转角度 = m × θ_i

→ 低频对(i 小) 转得慢, 对长距离位置敏感
→ 高频对(i 大) 转得快, 对短距离位置敏感
📐 频率公式拆解

θ_i = 1 / 10000^(2i/d) 跟 sinusoidal PE 用的是同一套频率。
i=0 → θ = 1（最低频，转一圈要经过整个序列）
i=d/2-1 → θ ≈ 1/10000（最高频，每走一步就转很多）
区别只是 sinusoidal PE 把 sin/cos 加到 embedding 上，RoPE 把 sin/cos 做成旋转矩阵乘到 Q/K 上。

→ 完整数学符号表：math-symbols

就像音乐的多声部 —— 低频鼓点标记大拍（段落距离），高频钹声标记小拍（相邻词距离）。所有频率合在一起，给每个位置一个多尺度的唯一指纹，跟 sinusoidal PE 的多频率思路一脉相承7。

实际代码里不会真的做矩阵乘法旋转（太慢），而是用 sin/cos 直接做元素级重排8：

# 对 Q/K 的第 i 对 (x_even, x_odd):
out_even = x_even × cos(m × θ_i) − x_odd × sin(m × θ_i)
out_odd  = x_even × sin(m × θ_i) + x_odd × cos(m × θ_i)

# 就是 2D 旋转公式 · 两次乘法 + 两次加法 per pair
# 全程 element-wise · 跟 Flash Attention 完美兼容

§ 04为什么管用

点积天然含相对位置

2D 旋转是正交变换9 —— 保持向量长度不变，只改方向。所以：

Rotate(Q, m) · Rotate(K, n)
= Q · K × cos((m−n) × θ)  +  交叉项 × sin((m−n) × θ)
       ↑                              ↑
  原始语义相似度完好保留      多出一个只依赖 (m−n) 的位置项
📐 这个公式在说什么

旋转之后做点积，结果分两部分：
1) Q 和 K 原来的语义相似度（"这两个词意思有多像"）—— 旋转没碰它
2) 一个只跟 (m−n) 有关的项（"这两个词差几个位置"）—— 旋转加进来的
两部分叠加 = attention score 同时看语义 + 相对位置。不需要额外参数或架构修改。

V 不旋转 —— 位置信息只在 attention score 里起作用（决定"关注谁"），不影响 value 的内容（"拿什么"）。

远距离自动衰减

RoPE 还有一个好性质 —— 当 |m−n| 越大（两个 token 越远），位置对 attention score 的贡献自动变弱。

直觉用拔河想：d/2 个频率 = d/2 根绳子，每根给一个 cos 值（+1 到 -1 之间）。距离近时所有 cos 都接近 +1（小角度 cos ≈ 1），32 根绳子朝同一方向拉 → 合力大 → 位置信号强。距离远时每根绳子的 cos 值正负各异（有的 +0.3，有的 -0.7），朝左朝右互相抵消 → 合力趋近于零 → 位置信号弱。距离越远，绳子方向越"各说各话"，加起来越接近零。

结果：RoPE 自带一个软窗口效应10 —— 不需要显式 mask，模型就会天然更关注近处。这跟语言的直觉一致 —— 离你越远的词对你影响越小。

§ 05历史定位 · 静悄悄改变了整个行业

RoPE 作者 Jianlin Su（苏剑林）是中国开发者11。这篇论文可能是中文 NLP 社区对全球 LLM 基础设施最大的单篇贡献。

2024-2025 年几乎所有新 LLM 都选了 RoPE：

为什么 RoPE 赢了 ALiBi

ALiBi 理论上也很优雅（线性距离惩罚 + 天然外推）。但实际训练中 RoPE 在长上下文质量上表现更好 —— 它的多频率旋转能捕获比线性衰减更丰富的距离模式。而且 RoPE + NTK/YaRN 的上下文扩展方案在工程上更成熟、社区支持更广。

RoPE 之后的发展方向是上下文长度扩展。原始 RoPE 训练 4K 上下文，直接推理 32K 会挂。但只要调整旋转频率（NTK-aware scaling / YaRN / Dynamic NTK），就可以把窗口拉到 128K 甚至更长。这条线目前还在快速演进12。

引用 / 术语表

  1. Attention 对顺序无感 · Transformer 的 self-attention 是置换不变的（permutation invariant）—— 把 token 打乱，Q·K^T 算出来的分数矩阵只是行列跟着重排，不会变化。所以不加位置信息的话，"dog bites man" 和 "man bites dog" 对模型来说是一样的。详见 self-attention。
  2. Sinusoidal PE · 原版位置编码 · Attention Is All You Need 提出的方案：用不同频率的 sin/cos 函数给每个位置一个唯一的 d 维向量，加到 embedding 上。优点是零参数、理论上可外推；缺点是只在输入层注入一次、编的是绝对位置。详见 positional-encoding。
  3. 旋转 Q/K · RoPE 的核心操作。不是给 embedding 加向量，而是在每层 attention 计算前，把 Q 向量按 position×频率 旋转，K 同理。旋转后做标准点积。数学上等价于在点积里插入一个只依赖相对位置的项。详见 rotary-position-embedding。
  4. 相对位置 vs 绝对位置 · "你在第 5 个位置"是绝对位置；"你跟那个词差 3 个位置"是相对位置。自然语言里，相对距离几乎总是比绝对地址有意义 —— 把整个句子平移 10 个位置，语义不变。RoPE 通过旋转让点积天然感知相对距离。详见 relative-position-encoding。
  5. KV Cache · 自回归推理的缓存机制。已经算过的 token 的 K/V 存起来，下一步只算新 token 的 Q/K/V。RoPE 跟它完美兼容 —— cache 里存的就是旋转后的 K，新 token 的 Q 也旋转过，直接点积就行。详见 kv-cache。
  6. 2D 子空间分解 · d 维向量不是整体旋转（那在高维没有唯一定义），而是拆成 d/2 个二维平面，每个平面独立旋转。就像把一个长向量想象成 d/2 个时钟，每个时钟转不同频率。
  7. 多频率 = 多尺度位置指纹 · 跟 sinusoidal PE 的多频率设计一脉相承 —— 低频捕捉长距离关系（段落级），高频捕捉短距离关系（词级）。合在一起给每个位置一个唯一的"频率指纹"。区别是 sinusoidal PE 把这个指纹加到 embedding，RoPE 把它做成旋转角度。
  8. 元素级实现 · 真实实现不会构造旋转矩阵做矩阵乘法（太慢），而是直接用 sin/cos 值做 element-wise 的重排：out_even = x_even×cos − x_odd×sin, out_odd = x_even×sin + x_odd×cos。跟 Flash Attention 完全兼容，几乎没有额外开销。
  9. 正交变换 · 保角性 · 2D 旋转是正交变换（orthogonal transformation）—— 保持向量长度不变，保持两个向量之间的夹角不变。这意味着旋转不会破坏 Q 和 K 的原始语义相似度，只是在点积里多加一个位置相关的偏移。
  10. 远距离自动衰减 · 拔河类比：d/2 个频率 = d/2 根绳子。距离近时所有绳子朝同一方向拉（cos 都接近 +1），合力大；距离远时正负各异互相抵消，合力趋近于零。RoPE 自带这个"软窗口"效应，不需要显式 mask。跟语言的直觉一致 —— 离你越远的词影响越小。
  11. 苏剑林 · Jianlin Su · RoPE 的第一作者，中国 NLP 开发者。这篇论文是中文 NLP 社区对全球 LLM 基础设施的重要贡献 —— 2024 年几乎所有新 LLM 的位置编码都用 RoPE 或其变体。
  12. 上下文扩展 · NTK-aware / YaRN · RoPE 训练 4K 上下文直接推理 32K 会崩。后续方法通过调整旋转频率来扩展窗口：NTK-aware scaling 按信息论原则重新分配高/低频带宽，YaRN 进一步做分段插值。实践中可以从 4K 扩到 128K+。这条线还在快速演进。
