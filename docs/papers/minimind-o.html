FIELD NOTES · 端到端 Omni · 从 0 实现

MiniMind-O ·
0.1B 也能听、看、说。

jingyaogong · ~0.1B 端到端 Omni · 全开源（代码/权重/数据/报告）· 2026-05-05

把语音真正纳入 Omni，最省的做法是 ASR→LLM→TTS 级联——但中间多一次转文字，延迟、语气、情绪都受损。MiniMind-O 的赌注：让语音和文本在 hidden state 层直连，单一 0.1B 权重同时听文/音/图、说出流式语音，还能从第一行代码读起自己训。

TL;DR

当前公开最小的完整 Omni1 实现之一（minimind-3o ~113M，moe 版 ~315M-A115M）。骨架是 Thinker–Talker 双路径2：Thinker 读文/音/图出语义文本，Talker 在 Thinker 的中间层3条件上、用 MTP6 一次预测 8 层 Mimi codes4（RVQ5 神经码），Mimi 解码器增量还原 24kHz 流式语音。语音/图像由冻结的 SenseVoice / SigLIP28 编码，经两层 projector7 注入隐空间。音色走 in-context 克隆9 + CAM++10，配 VAD11 实时打断、近似双工。训练不拆复杂预训练，按数据流 T2A→A2A→I2T 逐步接入。单卡 3090 约 2 小时跑通 mini 数据全链路。

§ 01一张图看懂 · 听看说一条链

一个权重同时干三件事：听（语音/文本）、看（图像）、说（流式语音）。关键是哪些训、哪些冻——可训练主体只有 Thinker + Talker + 两个 projector，加起来才 ~0.1B；语音/图像编码器和 Mimi codec 都是冻结的外部旁路（合计 ~425M，不算进 active）。

§ 02Thinker–Talker · 想的说的分两路

要让一个 0.1B 小模型既听懂图文语音、又说得自然，塞一条链会互相拖累：语言理解要抓语义，语音生成要逐帧铺声学。Thinker–Talker2 把这两件事解耦——Thinker 是编剧（想清楚说什么），Talker 是配音演员（把台词演成声音），配音员不必重读剧本背景。

最关键的一刀：Thinker 传给 Talker 的不是 embedding、也不是最后一层，而是中间层3。embedding 语义太薄；最后一层被 LM head 往"预测下一个文本 token"过度塑形，丢了适合发声的信息；中间层上下文和跨模态已融好、又没被输出目标带偏，最适合当语音条件。默认 bridge_layer = 层数//2 − 1。

解耦的红利：语言理解和跨模态融合都压在 Thinker，Talker 只在语义条件上渲染声音，所以 Talker 能做得很小，整条 Omni 链路压在 0.1B。详见 thinker-talker。

§ 03多模态进来 · 冻结编码器 + projector

让纯文本主干"看图听音"，不用重训：借现成冻结编码器抽特征，再训一个小 projector7 把特征翻译成 LLM 隐空间的向量，灌进序列里的占位符。LLM 是只懂中文的专家，SigLIP2 / SenseVoice 是懂"图像语""语音语"的人——你只配个翻译官，不让专家重学外语。

projector 小到可忽略（vision 1.18M / audio 0.99M），因为重活都被冻结编码器干完了，它只做"换坐标系"。详见 modality-projector。

§ 04Talker · MTP 一次出 8 层 Mimi codes

Talker 要把语义状态变成可解码的音频。Mimi4 是 RVQ5 神经码：每帧 8 层 codebook（第 1 层粗轮廓，往后逐层补残差），12.5 Hz、24 kHz。要是像普通自回归一个 code 一个 code 吐，序列瞬间 8 倍长。MTP6 的办法：每个解码步并行吐多层，配延迟错位让残差依赖沿对角线保住（不是粗暴丢掉），序列不膨胀。

为控制 0.1B 里的参数，8 层不各配一整套：共享主体 + 每层一个轻量 codebook adapter，既留住各层分布差异、又不复制参数。详见 multi-token-prediction 与 rvq-codec。

§ 05训练 · 按数据流逐步接入

不拆复杂的多阶段预训练，而是按数据流逐步接入能力，三段都是 SFT。同一条样本里放文本 token + 8 路 audio-code stream；回复开始后才算文本/音频 loss，reference 与 conditioning 区只提供条件、不作重构目标。

§ 06结果 · 小而完整 · 端到端 vs 级联

0.06B 主干不可能在知识/推理/长回答上跟成规模系统比，所以不追综合榜，而是落到可复现的局部评估。最有意思的是 Talker hidden 消融：小不等于划算——512/384 短句还行，中长句更容易漏词、重复、读音漂移；768 维留下来是因为它和 MiniMind 主干同维、能用 Thinker 后 4 层初始化，参数仍 ~0.1B，一致性却稳得多。

另一条线是端到端 vs 级联12：级联（ASR→LLM→TTS）中间多一次转文字，延迟、语气、情绪都受损；MiniMind-O 让语音和文本在 hidden state 层直连，把交互信息留在模型里。音色走 in-context 克隆9（参考音频编成 voice prompt 当上下文 + CAM++ embedding，换音色只换条件、不改权重），目前能区分男女声/语调/部分韵律，但还到不了"一段参考稳定复刻"的产品级。

放进 wiki 这条线

这是你 fish-speech 那条主线的"麻雀版"：Mimi=RVQ 神经码、Talker 处理多码本、音色靠参考条件，全是熟面孔，只是缩到 0.1B 且把"听看说"凑齐。值得记的两个判断：① 语音生成的瓶颈不在语言端而在多码本输出端（所以 Thinker 能小、Talker 不能太小）；② 条件取中间层是个可迁移的小技巧——embedding 太薄、末层被输出目标污染，中间层最干净。

引用 / 术语表

  1. Omni 模型 · 单一权重同时处理多模态输入（文/音/图）并输出文本与流式语音，目标是自然的实时语音交互（GPT-4o、Qwen-Omni、Moshi 一脉）。MiniMind-O 是其 ~0.1B 的从零开源实现。
  2. Thinker–Talker · 把"理解+思考"（Thinker，语言主干出语义文本）和"发声"（Talker，渲染音频 codes）解耦成两条路。源自 Qwen-Omni。详见 thinker-talker。
  3. bridge 中间层 · Thinker 传给 Talker 的条件取自中间层（默认 层数//2−1），不取 embedding（语义薄）也不取末层（被 LM head 往 next-token 过度塑形）。中间层上下文+跨模态已融、又未被输出目标污染。
  4. Mimi codec · Kyutai 的神经音频编解码器：8 层 codebook、12.5 Hz、24 kHz。把波形量化成离散 code 供模型预测，再增量解码回波形。MiniMind-O 里冻结。
  5. RVQ · 残差向量量化 · 多层 codebook，每层量化上一层的残差：第 1 层粗轮廓，往后逐层补细节。Mimi 是 RVQ 神经码。详见 rvq-codec。
  6. MTP · Multi-Token Prediction · 每个解码步并行吐多层 codebook，配延迟错位（delay pattern）让 RVQ 残差依赖沿对角线保住，既不膨胀序列也不丢依赖。共享主体 + 轻量 codebook adapter。详见 multi-token-prediction。
  7. modality projector · 两层 MLP，把冻结编码器的特征投影到 LLM 隐空间、填进占位符 token（audio 512→768、vision 768→768）。详见 modality-projector。
  8. SenseVoice / SigLIP2 · 冻结的外部编码器：SenseVoice-Small 抽 16kHz 语音特征（234M）、SigLIP2 把 256×256 图编成 64 patch token（94.55M）。只抽特征，不参与训练。
  9. in-context voice cloning · 参考音频先编成 voice prompt 当上下文条件喂给 Talker，换音色只替换条件，不微调权重、不改 prompt。详见 voice-cloning-reference。
  10. CAM++ speaker embedding · 192 维说话人向量，给 Talker 更稳的说话人约束，配合参考 codes 一起控音色。
  11. VAD · barge-in · 语音活动检测，推理时检测用户插话即可实时打断（barge-in），实现边听边答的近似双工。详见 vad。
  12. 端到端 vs 级联 · 级联 = ASR→LLM→TTS 串起来，中间转一次文字，丢语气/情绪、加延迟。端到端 = 语音文本在 hidden state 直连，把交互信息留在模型里。MiniMind-O 走端到端。
