Generative12
Pre-Training.

Radford · Narasimhan · Salimans · Sutskever · 2018-06

第一个把 Transformer + 大规模 LM 预训练 + 任务 finetune 做成一套方法。当时被 4 个月后的 BERT 盖过去，但 decoder-only + causal LM 这条路最终通往 GPT-3 / ChatGPT。

TL;DR

2018 上半年，OpenAI 押了一注：用 Transformer decoder 在大量无标注文本上学"预测下一个词"，能不能压过所有 NLP 任务专门方法？

核心三件事：

  • 预训练 · 12 层 Transformer decoder + BookCorpus2 7000 本书 + causal LM
  • 架构 · 只用 decoder（带 causal mask 的 self-attention）
  • 下游适配 · 不为每个任务设计新 head，而是把任务结构编码进输入文本（prompting 的雏形）

结果：9/12 项 NLP benchmark7 SOTA。4 个月后 BERT 抢了头条；6 年后 GPT 路线赢了一切。

§ 012018 上半年的赌局

2017 末 Transformer 出来，2018 大家都在试它能干什么。当时 NLP 的现状：

  • 标注数据稀缺 —— 情感分析 SST-2 才 6.7K 训练样本，从头训 transformer 必过拟合
  • 每个任务一个架构 —— 分类要分类 head，蕴含要双输入 encoder，相似度要交叉特征
  • 已有的预训练方法（word2vec / GloVe / ELMo）能给一些静态/上下文相关 embedding，但只能当"特征提取器"，不能 finetune

OpenAI 押的注：把 Transformer 当成语言模型来训，然后 finetune 到各种任务。语言模型 = "给定前缀预测下一个词"，是个无监督任务，能利用无限的文本数据。

为什么是这个时间点

核心赌博在两条假设上：(1) Transformer 这种新架构能在 LM 任务上 scale；(2) 用 LM 学到的表征能迁移到下游任务（不是只对生成有用）。两个假设 2018 时都没人证明过。GPT-1 是第一个端到端做这件事的 —— 而且代价不大：8 卡机器一个月级别就能复现8，2018 任何研究生都能动手。

§ 02预训练 · Causal Language Model

每个位置预测下一个 token，全句训练。Causal mask 保证每个位置只能看自己和之前的 token（否则就作弊了）。

为什么这个目标能教出通用能力

要把"今天天气真..." 后面的词猜对，模型必须懂语法（接形容词）、词频（"好"比"碳酸氢钠"概率高）、常识（夏天接"热"、冬天接"冷"）、甚至浅推理（前文说"下雪"则接"冷"）。一个朴素的"猜下词"任务，逼着模型把整个世界知识压进权重 —— 这是 LLM 一切能力的源头。

对比 BERT 的 MLM：BERT 随机 mask 15% 位置预测被遮的词。CLM 是每个位置都预测下一个 —— 训练信号密度差 6 倍多1。同等 wall-clock 下 CLM 让模型"做题"次数远多于 MLM，部分解释了 decoder-only 系 scale 起来更顺。

§ 03架构 · Causal Mask 是怎么回事

GPT-1 用 Transformer 的 decoder 部分，跟 BERT 几乎一样的 block，只在 attention 上加了 causal mask5 —— 让每个位置看不到未来。

具体到代码层面，causal mask 就是在 softmax 之前的 attention scores 矩阵的上三角加上 -∞：

scores = Q @ K.T / sqrt(d_k)        # [seq, seq]
mask = torch.triu(torch.ones_like(scores), diagonal=1).bool()
scores.masked_fill_(mask, float('-inf'))
attn = softmax(scores, dim=-1)      # 上三角变 0
out = attn @ V

架构其他部分跟 BERT 同款：残差连接 + LayerNorm + Multi-Head Attention + FFN。差别只在一个 mask。但训练目标 + 这个 mask 决定了一切：能不能自回归生成6、能不能双向理解、能不能 in-context learning。

§ 04怎么用 · Input Transformations

预训练完了 117M 参数的模型 —— 然后呢？怎么 finetune 到分类、蕴含、相似度、多选 QA 这些任务？

BERT 的做法是每个任务一个 head。GPT-1 走了一条更激进的路：不改架构，只改输入。

⟨s⟩ ⟨$⟩ ⟨e⟩ 这三个特殊 token 是新加的 —— 在 finetune 时学。每个位置的 hidden state 通过一个共用的小线性层得到分类 logit。架构完全不变，只是输入文本被不同的方式排列。Finetune 时还混了个 auxiliary LM loss3（任务 loss + LM loss 加权和）让模型不忘掉预训练学到的语言能力。

这是 prompting 的雏形

"用文本格式编码任务结构"这个 idea 在 GPT-1 里已经成型，后来这套技巧有了名字 —— prompting11。演化路线：

  • GPT-1 · 用特殊 token 编码任务结构，仍需 finetune
  • GPT-2 · 用自然语言 prompt 触发任务（"TL;DR:" 让模型做摘要）4
  • GPT-3 · few-shot in-context learning · 在 prompt 里给几个例子就行，连 finetune 都不要
  • ChatGPT · "system + user + assistant" 三角色格式

一条直接的演化线。GPT-1 的 input transformations 是这条线的起点。

§ 05跟 BERT 的对照

GPT-1 和 BERT 是 2018 同期、同 pretrain+finetune 范式的两个分叉。当年 BERT 更耀眼，6 年后 GPT 系赢了一切。看一眼各项对比：

事后看的关键差异

2018 看 BERT 赢；2020 GPT-3 出来后看 GPT 路线赢。差异在两件事：

1 · 训练 = 推理。Decoder-only 训练时就是 "predict next token"，推理时也是。一致性让 scaling 顺利。BERT 训练用 MLM（mask 填空），推理时却用别的 head（分类 / 抽取）—— 推理跟训练任务不一致，scale 上去后 finetune 阶段成为瓶颈。

2 · Generation 是更通用的能力。GPT-3 发现 decoder-only 模型 scale 大了之后能用 prompt 触发任何任务，包括分类（"Is this positive or negative? Text: X. Answer:"）。BERT 系做不到，因为它没法把"分类"做成"生成下一个 token"。

§ 06真正的遗产

GPT-1 当年的 benchmark 数字现在看来不重要 —— 117M 模型早被超越了几个数量级，也不具备 zero-shot 触发任务的能力10（那要等 GPT-2/3 把 scale 推上去）。它留下来的三件事：

  1. Causal LM 作为预训练范式。所有现代 LLM（GPT / Claude / Llama / Qwen）都在 GPT-1 这套训练目标上 scale
  2. Decoder-only 架构。砍掉 encoder 这件事 GPT-1 第一个端到端验证9
  3. 用输入格式表达任务结构。后来叫"prompting"，再后来叫"in-context learning"。GPT-1 已经在做这事，只是名字还没普及

真正改变历史的不是这篇 paper 的 benchmark，是它的thesis：causal LM + scale → 通用语言能力。这个 thesis 后来被 GPT-2 / GPT-3 一路推到极致，最终诞生 ChatGPT。BERT 的双向 + MLM 路线则在嵌入 / 检索领域活下来，但在前沿模型里已经退场。

为什么这个 thesis 当时不被普遍接受

2018 大家觉得"预测下一个词"这种任务能学到的能力有限 —— 它能学到表面统计规律（语法、词频），但不可能教出推理。BERT 的"双向理解"被视为更"深刻"的目标。GPT-1 当时 9/12 SOTA 的成绩说明思路可行，但不够震撼。直到 GPT-3 用 175B 参数和 in-context learning 证明这条路不仅可行还更强，整个领域才换边。scale 把表面规律变成了能力 —— 这是当年没人预见到的。

§ 07训练 vs 推理 · Causal Mask 的双重作用

Causal mask（三角矩阵）在 GPT 里同时解决两件事：

训练时：让所有位置并行计算 —— 位置 i 只看 0..i-1（causal mask 挡住后面的）。每个位置预测下一个 token，全部同时出 loss。一个 batch 就算完整个序列。

推理时：自回归逐 token 生成。每步只算一个新 token 的 Q/K/V，跟 KV cache 里存的之前所有 token 的 K/V 做 attention。不重复计算。

Causal mask 让训练和推理"看到的东西一样" —— 每个位置在训练时被 mask 挡住后面、推理时还没生成后面，信息量对称。这跟 BERT（训练见 [MASK] 推理不见）不同，GPT 没有 train-inference distribution shift 问题。

引用 / 术语表

  1. 训练信号密度 · CLM 每个位置都贡献 loss，MLM 只在 mask 位置（约 15%）贡献。同等数据量下 CLM 让 transformer 看到的"做预测"次数大约多 6 倍 —— 这是 decoder-only 系训练效率高的部分原因。
  2. BookCorpus · GPT-1 用的预训练数据集，11038 本电子书，约 800M 单词。当时算"大规模"但跟 GPT-3 的 Common Crawl（数千亿 token）相比微不足道。BookCorpus 后来因版权问题逐渐淡出 LLM 训练数据。
  3. Auxiliary LM loss · GPT-1 在 finetune 阶段同时优化下游任务 loss + 语言模型 loss（加权和）。让模型在 finetune 时不"忘了"预训练学到的语言能力。后来 ChatGPT 的 SFT 阶段、Instruction tuning 也用类似思路。
  4. "TL;DR:" prompt · GPT-2 论文展示的 zero-shot 摘要技巧 —— 给一段文章后面接 "TL;DR:"，模型会自动生成摘要。证明大 LM 已经在预训练里"看过"这种格式无数次，能在没 finetune 的情况下 触发任务。Input transformation 路线的天然延续。
  5. Causal mask · 在 attention scores 矩阵的上三角（j > i）位置加 -∞，让 softmax 后变 0。物理含义：query 位置 i 在算 attention 时看不见 key 位置 j 的内容（如果 j > i）。详见 causal-language-model、self-attention。
  6. Autoregressive generation · 自回归生成 · 推理时每生成一个 token 就把它接到输入后面，再 forward 一次。一个 token 一个 token 出，无法并行。是 decoder-only 模型推理慢的根本原因 —— 后来的 speculative decoding / KV cache 等优化都是绕这件事的。
  7. 12 项 benchmark · GPT-1 论文 finetune 了 12 个 NLP 数据集：分类（SST-2、CoLA）、相似度（MRPC、QQP、STS-B）、NLI（MNLI、QNLI、SNLI、RTE）、问答（RACE、Story Cloze）。9 项 SOTA。
  8. BookCorpus + LM = 117M 训练几天能搞定 · GPT-1 训练只用 8 GPU 一个月级别。对比 GPT-3 训练用 1024+ GPU 几个月，规模差 1000+ 倍。2018 的 GPT-1 任何研究生都能复现，是它思路明确但成本不高的产物。
  9. Decoder-only 的最终胜出 · 2020 GPT-3 + in-context learning 出来后，业界开始大规模转向 decoder-only。原因在 §05 列了：训练 = 推理一致、scaling 平稳、生成天然。BERT 系在 2024 之前已基本退出大模型前沿。
  10. 为什么 GPT-1 不能 zero-shot · 117M 太小。预训练时没"看过"足够多的任务格式，模型不具备从 prompt 推断任务的能力。同样 thesis 在 GPT-2（1.5B）和 GPT-3（175B）才浮现 zero-shot / in-context learning 能力。这是 scaling 带来的 emergent property。
  11. Prompting · 提示工程 · 通过设计输入文本格式来引导 LLM 输出特定类型结果的技巧。从 GPT-1 的 input transformations → GPT-3 的 few-shot in-context learning → ChatGPT 的 chat template，是一条直接演化线。详见 input-transformations。
  12. Generative vs Discriminative pre-training · 论文标题里的 Generative 是核心 —— 把预训练任务设计成"生成下一个 token"。对比 BERT 的 discriminative pretraining（"判断这个位置该是什么 token"）。GPT 系赢了之后，生成式预训练成为统一范式。
