PPO ·
一行 clip 撑起 ChatGPT.

Schulman · Wolski · Dhariwal · Radford · Klimov · OpenAI · 2017

策略梯度最头疼的"步子大就崩"问题, PPO 用一行 torch.clamp 干掉了。简单到 200 行能复现, 稳到撑起了 RLHF, 也就是 ChatGPT 真正会聊天的那一步。

TL;DR

强化学习里 policy gradient1 走太大一步策略就崩, 老数据立刻不能用。TRPO2 用 KL 散度精确约束步长, 但需要共轭梯度 + 线搜索, 复杂。PPO 的回答: 不约束 KL, 直接 clip 新老策略的概率比值3。一行 torch.clamp(r, 1-ε, 1+ε) 让 loss 在"该停的方向"梯度归零, 软性 trust region。代价是不精确, 收获是简单、稳、能跟 Adam 配。顺手解锁的是同一批 rollout 跑 K 个 epoch4, sample efficiency 翻倍。2017 出, 2022 撑起 InstructGPT/ChatGPT 的 RLHF5。

先打底

如果"rollout / state / action / 分布变化 / KL / 各种 loss 走势"这些词让你发懵, 先读 RL 直觉打底 · 给懂 LLM 不懂 RL 的人 —— 它用 LLM 自回归把这套术语全翻译成你已经会的东西, 再回来看这页会顺很多。

§ 01RL 基本设定 · agent ↔ env 一来一回

跟监督学习"猜对答案给奖、猜错给罚"不同, 强化学习的训练信号是迟来的 —— 你玩完一整局才知道输赢, 而不是每一步都有 ground truth6。

目标: 让 agent 学一个 π(a|s) 使得长期累计 reward 最大。策略梯度的核心想法是把 π 当成可微分的神经网络, 直接 gradient ascent。

loss = -log π(a_t | s_t) × R_t       # 跟监督学习 cross-entropy 几乎一样
                       └─ "label 权重", 这一步之后的总 reward

意译: "在状态 s_t 选了 a_t 之后总收益是 R_t, 就把 P(选 a_t) 朝 R_t 的方向推"。R_t 高 → 推大, R_t 低 → 推小。详见 policy-gradient。

§ 02痛点 · 步子大就崩

策略梯度有个致命缺陷 —— 步子大小决定生死7。具体崩盘场景:

1. 用老策略 π_old 跑一批 rollout 收集 (s, a, R) 数据
2. 用这批数据算 gradient
3. 走一步, 得到 π_new
4. 但如果 π_new 离 π_old 太远, 那批数据描述的"在 π_old 下哪些 action 好"对 π_new 已经不再适用
5. 拿着失效数据继续算梯度, 越调越偏, 几步后训练崩盘

TRPO (2015) 解过这个 —— 加 KL 散度约束 KL(π_new ‖ π_old) ≤ δ 精确限制每步。理论漂亮, 但工程上要做共轭梯度 + 线搜索, 慢、复杂、不亲深度学习。

§ 03核心 · 一行 clip 干掉所有麻烦

PPO 的回答3: 不用 KL 约束, 不用共轭梯度, 就一行 torch.clamp。

定义新老策略的概率比:

r_t(θ) = π_new(a_t | s_t) / π_old(a_t | s_t)

  r = 1.0  ←  新老完全一样
  r = 1.5  ←  新策略把这个 action 概率提了 50%
  r = 0.5  ←  新策略把这个 action 概率降了 50%

PPO 的 loss:

L_CLIP = E_t [ min( r_t · A_t,  clip(r_t, 1-ε, 1+ε) · A_t ) ]
                                          ↑
                                  ε 一般 0.2

其中 A_t 是 advantage8 —— "这个 action 比平均好多少"。实际工程里 A_t 用 GAE11 算 (λ=0.95 是标配)。看不懂没事, 下面这张图把它讲透:

为什么 min 函数

min(r·A, clip(r)·A) 取"更不利于增加 loss"的那个。当更新是"善意方向"(好 action 增、坏 action 减): clip 起作用, 防贪心。当 r 已经跑过边界且要修正回来: 让原 r·A 起作用, 不阻碍修正。这个 min 设计让 clip 只在该停的方向停, 是 PPO 论文最微妙也最关键的一个细节。

§ 04顺手白送 · 数据重用 K 个 epoch

Vanilla 策略梯度: 一批 rollout 只能用一次, 算完梯度立刻扔。
PPO: 一批 rollout 可以反复跑 K 个 epoch (K=3~10), sample efficiency 翻好几倍4。

为什么这能 work: clip 保证了 π_new 离 π_old 不会跑出 [1-ε, 1+ε] 范围, 那批 rollout 数据相对 π_new 还"近似准确"。这是 PPO 工程价值的核心 —— sample efficiency 翻倍, 同样 wall-clock 时间能训出更好的策略。

§ 05杀手级应用 · RLHF 训出 ChatGPT

PPO 2017 年出, 在游戏 / 机器人控制圈里跑了 5 年。2022 年 OpenAI 用 PPO 训了 InstructGPT, 把"会预测下一个 token 的 GPT-3" 变成了"会按人类偏好聊天的 ChatGPT"5。这是 PPO 最重要的一笔遗产。

KL penalty 防 reward hacking

没有 KL penalty 时, LLM 会过拟合 RM 的 bug —— "RM 喜欢长回答" → LLM 学会啰嗦; "RM 给 'Certainly!' 高分" → LLM 每个回答开头都加 "Certainly!"。加 β × KL(LLM ‖ SFT) 限制 LLM 偏离 SFT 模型太远, 是 RLHF 工程最关键的细节之一9。

§ 06遗产 · GRPO · DPO · 跟它们的关系

PPO 之后的 RL alignment 演化, 几条主要路线都跟它有渊源:

GRPO (DeepSeek 2024)

PPO 的 LLM 友好简化版10。干掉了 value network, 改用一组样本 (一个 prompt 采样 N 个回答) 的相对得分直接当 advantage:

# PPO 标准做法
A_t = GAE(rewards, V_critic(state))    # 需要训 critic 估 V

# GRPO 简化做法
A_t = (r_i - mean(group)) / std(group) # 一组样本里相对位置

省一个网络, 训得更稳。DeepSeek R1 (2025) 用 GRPO 做 reasoning 训练取得突破。详见 grpo concept

DPO (2023)

Direct Preference Optimization12 跳过 reward model + PPO, 直接用对比学习的思路一步训 LLM:

loss = -log σ( β · log[π(y_w|x) / π_ref(y_w|x)] - β · log[π(y_l|x) / π_ref(y_l|x)] )
                          ↑ 偏好的回答                       ↑ 不偏好的回答

没有 RM, 没有 PPO 循环, 一个 cross-entropy 一样的 loss 训完。工程上更稳, 但不能 online 探索 —— 适合"用现成偏好数据对齐", 不适合需要 RL 探索的场景 (比如 reasoning)。

当前生态

2024-2025 的格局大致是:

• 对话对齐 (chat models, helpfulness, harmlessness): DPO 多, 因为偏好数据现成 + 工程稳
• Reasoning 训练 (R1 / o1 系): GRPO 多, 因为 reward signal 来自正确性 (可验证), 需要 online RL 探索
• 需要严格 KL 控制 + 明确 reward 的场景: PPO 经典版仍在用

所以 PPO 没死, 它分裂成了几条更专的路线 —— 但 clipped surrogate + advantage 这套 core idea, 是 DPO/GRPO 都建立在其上的基础。

§ 07它真正改变的是什么

PPO 作为 2017 年的 RL 算法本身的影响:

• RL 落地门槛大幅降低 —— 之前 RL 是研究领域的事, PPO 之后中等团队也能跑出能用的策略
• OpenAI Five (Dota 2 击败人类职业战队), AlphaStar (StarCraft II 顶级水准) 都基于 PPO 系
• 机器人控制, locomotion (Boston Dynamics 风格的步态学习), simulated humanoid 全靠 PPO

但 PPO 的最大遗产明显是 RLHF —— 没有 PPO 这套稳定可调的 policy gradient, OpenAI 不可能在 2022 年底把 ChatGPT 训出来。ChatGPT 之后的 LLM 应用爆炸, 算上一笔 PPO 的功劳。

从工程哲学层面: PPO 体现的"粗暴近似打败精确解"也成了深度学习里反复出现的规律 ——

• TRPO (精确 KL 约束) → PPO (粗暴 clip): clip 赢
• Attention with mask (精确因果约束) → causal mask (粗暴下三角): mask 赢
• Adam (精确学习率自适应) → SGD + momentum (粗暴动量): 各有领地
• Bayesian deep learning (精确不确定性) → Monte Carlo dropout (粗暴 sample): dropout 赢

PPO 的 clip 是这个规律的一个标杆案例。

引用 / 术语表

  1. Policy gradient · 策略梯度 · RL 训练的基础方法 —— 把策略 π(a|s) 当成神经网络, 用 reward 当 loss 的方向梯度 ascent。跟监督学习的 cross-entropy 几乎同一个公式, 唯一区别是 label 换成了 reward 数值。详见 policy-gradient。
  2. TRPO · Trust Region Policy Optimization · PPO 的前身 (Schulman 2015)。加 KL 散度约束 KL(π_new ‖ π_old) ≤ δ 精确限制每步策略变化。理论漂亮但工程上要做共轭梯度 + 线搜索, 慢、复杂、不亲深度学习框架。
  3. Clipped surrogate objective · PPO 的核心创新。用 clip(r, 1-ε, 1+ε) 限制新老策略概率比值, 把 trust region 约束直接写进 loss。一行 torch.clamp 干掉 TRPO 的所有复杂。详见 clipped-surrogate-objective。
  4. K-epoch data reuse · PPO 的副产品 —— 因为 clip 保证 π_new 不跑远, 同一批 rollout 可以反复跑 K 个 epoch (K=3~10) 而不让数据失效。Sample efficiency 比 vanilla PG 高 K 倍。这是 PPO 工程价值的核心。
  5. RLHF · Reinforcement Learning from Human Feedback · PPO 在 LLM 上的杀手级应用 —— 用人类排序偏好训 reward model, 再用 PPO 让 LLM 最大化 reward。InstructGPT (2022) 用这套把 GPT-3 变成 ChatGPT。详见 rlhf。
  6. RL 训练信号迟来 · 监督学习每个样本都有 ground truth, RL 只在一整局结束才知道输赢。这是 RL 难训的根本原因 —— 训练信号稀疏且滞后, 还得 attribute 给中间某个 action。
  7. 步子大就崩 · Vanilla 策略梯度的致命缺陷。数据用 π_old 收集, 梯度算 π_new, 一旦 π_new 跑远, 数据立即失效, 训练发散。所有现代 RL 算法 (TRPO/PPO/GRPO/DPO) 都在解这个。
  8. Advantage function · 优势函数 · A(s,a) = Q(s,a) - V(s) = "这个 action 比当前局面的平均预期好多少"。比 raw return 更稳的训练信号, 对 reward 偏移不敏感。GAE 是估计 A 的标配。详见 advantage-function。
  9. KL penalty · 防 reward hacking · RLHF 第 3 步里, 把 reward 改成 reward - β·KL(LLM ‖ SFT_frozen)。防止 LLM 为了 RM 高分把所有回答都搞成"啰嗦版/markdown 版/Certainly! 开头"。β 一般 0.01-0.1。是 RLHF 工程最关键的细节之一。
  10. GRPO · Group Relative Policy Optimization · DeepSeek 2024 提出的 PPO 简化版 —— 干掉 value network, 用一组样本的相对得分直接当 advantage。R1 (2025) 的 reasoning 训练用它取得突破。详见 grpo。
  11. GAE · Generalized Advantage Estimation · Schulman 2015 提出的 advantage 估计方法 —— 在 TD (1-step) 和 MC (whole episode) 之间用 λ 插值。λ=0.95 是 PPO 标准配置。算 GAE 比 clip 本身对训练效果影响还大。
  12. DPO · Direct Preference Optimization · 2023 提出, 跳过 reward model 跟 PPO 循环, 直接用对比学习的思路从偏好数据一步训出对齐的 LLM。比 PPO 简单稳定, 但不能 online 探索, 现在 chat 模型对齐多数选 DPO; reasoning 训练仍用 PPO/GRPO。
