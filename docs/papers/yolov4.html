TL;DR / BUILD SHEET

YOLOv4 没换掉 YOLOv3 的 anchor 三尺度 head，而是用 CSPDarknet53 重做 backbone1，用 SPP/PAN 重做 neck2，再把训练期 Freebies 与推理期 Specials 分开筛选。COCO 608 输入达到 43.5 AP / 65.7 AP50 / 47.3 AP75；V100 表中是 62 FPS。

§ 01 / The accounting rule先问一招记哪本账，再问要不要装

YOLOv4 面对的不是“没技巧可用”，而是技巧多到容易乱装。论文把候选分成两袋：Bag of Freebies3只改变训练；Bag of Specials4会进入推理图，但只允许付少量速度成本。这个分类把“精度涨了”拆成更实用的问题：训练贵了多少？部署慢了多少？GPU 真跑多快？

别被近似名字绕住CIoU 是训练时回归框的损失；DIoU-NMS 是推理时去掉重复框的后处理。Mish 和 Mosaic 又分别跨越了 backbone / detector 的清单，所以不能只记“两袋各有什么”，还要记它在哪一段生效。

§ 02 / ArchitectureBackbone、neck、head：只换前两段

现代检测器可以拆成三段：backbone 从像素抽特征；neck 汇合不同层级；head 把特征变成框和类别。v4 的最终装车单是 CSPDarknet53 → SPP + modified PAN → YOLOv3 head。它仍是一阶段检测，也仍在三张特征图直接出框。

CSP：只让一部分通道反复深加工

Cross-stage partial connection5把一个 stage 分成加工支路和旁路。它不是把一半特征丢掉，而是让两路在末尾重新 concat，减少全部通道反复走同一串残差块产生的重复梯度。

SPP 拉宽视野，PAN 补上回程路

SPP 在 13×13 深层特征上保留原特征直通支路，再并排做 5×5、9×9、13×13 最大池化6；PAN 再让语义从 13→26→52 下发，让位置从 52→26→13 回流。v4 把原 PAN 的 shortcut 加法改成 concat，先保留两路，再交给卷积混合。SAM 也从 spatial-wise attention 改成 point-wise attention，在每个位置、每个通道上做逐点门控；论文没有给这项结构修改单独做严格消融。

§ 03 / Training freebies四图拼接与自我攻击：训练难一点，部署不背债

Mosaic：一张样本混进四种背景

Mosaic7把四张图缩放、裁剪后拼成一张。它让物体脱离惯常背景，缩放也会增加小目标出现频率。标注框必须跟着做同样的缩放、平移、裁剪。

SAT：按模型自己的弱点现场出题

Self-Adversarial Training8不是拿对抗图去做安全测试，而是把攻击阶段直接塞进训练：第一段不更新权重，只改输入，让目标变难看见；第二段固定这张难图，正常更新模型把目标找回来。

§ 04 / Assignment & coordinates边界更好学，一个真值也能多派几名负责者

消除网格敏感：0.99 不再要求巨大 logit

YOLOv3 的格内偏移是 sigmoid(t_x)。越靠近 0 或 1，logit 越极端、梯度越小，这就是论文说的 grid sensitivity10。公开实现改成 s·sigmoid(t_x)-0.5(s-1)，把可预测范围稍微撑出格子。

多 anchor 正样本：阈值以上都可以学

v3 默认只奖励形状 IOU 最好的一个 anchor。v4 仍保留最佳者，同时让其他形状 IOU 超过阈值的 anchor 也学习。遗传搜索给出的阈值是 0.213；这改变的是正样本覆盖，不是推理候选数。

§ 05 / Box lossCIoU：覆盖、中心、形状一张账单

普通坐标误差把 x/y/w/h 分开罚；IOU 又只看重叠。CIoU11把框的三种肉眼错误合起来：1-IoU 管盖住多少，中心距离管整体偏移，长宽比项管形状。

L_CIoU = 1-IoU + ρ²(b,b_gt)/c² + αv v = 4/π² · [atan(w_gt/h_gt)-atan(w/h)]² α = v / (1-IoU+v)

b,b_gt 是两框中心；ρ² 是中心距离平方；c² 是最小外接框对角线平方；w,h 与 w_gt,h_gt 是预测和真值宽高；v 衡量长宽比差；α 根据当前重叠情况调节形状项。

§ 06 / Training record论文默认、遗传搜索、当前 cfg，不是同一套数

ImageNet backbone 筛选先训练 8,000,000 step：总 batch 128、mini-batch 32，初始学习率 0.1，poly 衰减，warm-up 1,000 step，momentum 0.9，weight decay 0.005。BoS 对照沿用同一预算；BoF 对照额外增加 50% step，所以分类表里的提升并非完全等训练时长。

论文 COCO 默认实验：500,500 step，初始学习率 0.01，在 400k / 450k 各乘 0.1；momentum 0.9，weight decay 0.0005；总 batch 64，单卡 mini-batch 4 或 8，并做多尺度训练。Cross mini-Batch Normalization9只在同一个 batch 的多个 mini-batch 之间汇总统计，不需要跨 GPU 同步。

遗传搜索另得到：学习率 0.00261、momentum 0.949、anchor IOU 阈值 0.213、框损失归一系数 0.07。当前 GitHub master 的 yolov4.cfg 又写学习率 0.0013、subdivisions 8、Mosaic、CIoU。论文最终配方列 DIoU-NMS，当前 cfg 却是 greedynms。复现时必须钉住具体配置与 commit，模型名本身不是可执行规范。

§ 07 / Ablation最重要的负结果：热门组件装上也会掉点

分类实验先说明 BoF + Mish 能把 CSPDarknet53 从 Top-1 77.2 / Top-5 93.6 推到 78.7 / 94.8。但更高分类精度不自动等于更好检测：CSPResNeXt50 的分类预训练变强，检测反而从 42.4 轻微掉到 42.3；CSPDarknet53 才从 42.4 升到 43.0。

mini-batch 结论也要窄读：带 BoF/BoS 的 CSPDarknet53 在 mini-batch 4 与 8 时是 41.6 vs 41.7 AP，只差 0.1；另一套不带这些策略的 CSPResNeXt50 是 37.1 vs 38.4，差 1.3。它说明这套配方在该实验里更稳，不等于 batch size 永远无所谓。

§ 08 / Results & limits严格定位和小目标一起补上，但功劳没法逐件拆清

YOLOv4-608 在 COCO test-dev 得到 43.5 AP / 65.7 AP50 / 47.3 AP75 / 26.7 small AP。论文按 Maxwell、Pascal、Volta 分开列 FPS，避免把不同 GPU 混成一张绝对榜；它的结论更适合读成 Pareto 前沿12——同样精度谁更快、同样速度谁更准。

没有错误类型画像论文没有按类别拆定位、分类、背景误检，只能从 AP50/AP75 与尺寸分桶读总体表现。
消融不是完整因果账许多实验跨骨干、输入尺寸或同时改多项，无法给每个组件精确分功。
仍有 anchor 与 NMS正样本阈值、anchor 形状和推理去重都要维护，后续 YOLOX 会转向 anchor-free。
配置必须版本化论文清单与当前仓库 cfg 已有差异；复现“YOLOv4”必须同时保存代码和配置版本。
一句话带走YOLOv4 的真正遗产不是一长串缩写，而是一套工程筛选法：先拆 backbone / neck / head，再把训练账与推理账分开，最后用真实 GPU 和完整消融决定哪些零件留下。

引用 / 术语表

  1. Backbone · 从输入图像提取多层特征的主干；YOLOv4 使用 CSPDarknet53。
  2. Neck · 位于骨干与检测头之间，汇合不同尺度特征；见 SPP + PAN。
  3. Bag of Freebies · 增加训练成本但不改变推理图的技巧，见两类成本账。
  4. Bag of Specials · 付少量推理成本换精度的模块或后处理。
  5. CSP · Cross-Stage Partial；部分特征深加工，部分走旁路，末尾拼接。
  6. Receptive field · 一个特征位置能看见的输入范围；见感受野。
  7. Mosaic · 四图缩放裁剪后拼成一张训练画布，见Mosaic 增强。
  8. SAT · Self-Adversarial Training；先改输入藏目标，再用难图正常训练。
  9. CmBN · Cross mini-Batch Normalization；只在单个 batch 内跨 mini-batch 汇总统计。
  10. Grid sensitivity · sigmoid 靠近格子边界时需要极端 logit、梯度又变小的问题。
  11. CIoU · 同时计入重叠、中心距离和长宽比的框损失，见完整手算。
  12. Pareto frontier · 无法在不牺牲速度的前提下继续提精度，或不牺牲精度继续提速度的一组最优折中点。
