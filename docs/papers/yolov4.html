TL;DR / BUILD SHEET

YOLOv4 沿用 YOLOv3 基于 anchor 的三尺度检测头，用 CSPDarknet53 换掉主干网络，再用 SPP/PAN 重搭特征融合层。训练期技巧和部署时仍要执行的模块分开筛选。608 输入时达到 43.5 AP；论文报告在 V100 上可达到 62 FPS。

先把这页最常见的 12 个词钉住点击任意卡片，右侧直接弹出解释
BACKBONE主干网络把图片变成多层特征，不负责最后报框。 NECK特征融合层把深层语义和浅层位置线索汇到一起。 HEAD检测头把融合后的特征翻译成框、置信度和类别。 FEATURE MAP / CHANNEL特征图与通道一摞响应网格；每个通道偏向识别一种模式。 STRIDE / PADDING步长与补边一个管窗口每次走几格，一个管边缘补几圈。 CONCAT / SHORTCUT拼接与相加拼接先保留两路；相加当场合成一路。 ANCHOR预设框每个位置先摆几种宽高模板，网络只学怎么校准。 AP / AP50 / AP75检测精度既罚误报也罚漏报；数字后的 50/75 是命中门槛。 FPS / BFLOPS实测速度与理论运算量一个是真跑多少帧，一个是大约算了多少次。 BATCH / MINI-BATCH一批数据怎么拆总 batch 是一次更新合计用多少张图，mini-batch 是每次实际放进显存多少张图。 LOGIT / SIGMOID原始分数与 0～1 映射网络先吐任意实数，再用 sigmoid 压成概率或格内偏移。 IOU / NMS框重叠与去重IOU 衡量两个框有多重合；NMS 用它删掉重复框。

§ 01 / The accounting rule先问一招记哪本账，再问要不要装

YOLOv4 不缺候选技巧，真正难的是该留谁。论文先分两本账：Bag of Freebies 只在训练时花钱；Bag of Specials 部署后还要执行，因此真会拖慢推理。这样一来，“精度涨了”还不算回答完：训练多花了多少？上线后慢了多少？GPU 实测能跑多快？

§ 02 / Architecture主干、融合层、检测头：YOLOv4 主要改前两段

目标检测器可以拆成三段：主干网络（backbone）把像素变成多层特征；特征融合层（neck）把不同深度、不同尺寸的特征汇在一起；检测头（head）再把它们变成框、置信度和类别。YOLOv4 的组合是 CSPDarknet53 → SPP + modified PAN → YOLOv3 head。它仍是一阶段检测：一遍前向，就在 52×52、26×26、13×13 三张特征图上直接预测框。

CSP：只让一部分通道反复深加工

像工厂流水线：只让一半零件进返工车间反复打磨，另一半原样直送打包区，最后两批合流。省下的正是"全部零件都返工"的冤枉功——精度不怎么掉，算力却省一截。

CSP（Cross-Stage Partial）就是这么干：先把一个 stage（一组保持同样分辨率的网络层）分成两路，一路穿过残差块反复加工，另一路走短路，最后再拼接回来。它不是丢掉一半特征，而是避免所有通道都重复跑完整条加工线；两路梯度也不用挤在同一条路径里。

SPP：窗口看得更远，但特征图仍是 13×13

池化（pooling）做的事很简单：拿一个 k×k 窗口，在里面取最大值或平均值。特征图会不会缩小，要看步长和补边，不能看到 pooling 就默认它会下采样。YOLOv4 让 5×5、9×9、13×13 三个最大池化分支都读取同一份 13×13×512 输入。它们的步长都是 1，补边分别是 2、4、6。

输出高度按 Hout=floor((H+2p−k)/s)+1 计算：H 是输入高度，k 是窗口大小，p 是单侧补边，s 是步长。5×5 分支代入后是 (13+2×2−5)/1+1=13；9×9 和 13×13 也都是 13。因此这四路不是连续把图越缩越小，而是在同一批空间位置上收集四档上下文。

PAN：SPP 管“看多远”，PAN 管“三层怎么互通”

SPP 只在 13×13 这一层扩大视野；PAN 才负责 13×13、26×26、52×52 三层之间传消息。13→26→52 把深层的类别语义送到细网格，52→26→13 再把浅层的精确位置送回粗网格。YOLOv4 还把原 PAN 的相加（shortcut）改成拼接（concat）：相加会立刻把两路同通道压成一个数；拼接则先完整保留两路，再让后续卷积决定怎么混。

SAM：从“一个位置共用一扇门”改成“每个元素各有一扇门”

先看直觉。原来的 spatial-wise attention 给每个空间位置一个权重；同一位置上的所有通道共用它。YOLOv4 改成 point-wise attention 后，每个通道、每个位置都有自己的权重。同一个位置上，“轮廓”通道可以保留，“纹理”通道可以压低。

再看公式。F∈R^(C×H×W) 是输入特征，C 是通道数，H,W 是空间尺寸；A 是 0～1 的门控权重，Y 是加权后的输出。原版先沿通道做平均池化和最大池化，得到一张 1×H×W 门控图：Y[c,h,w]=F[c,h,w]×A_sp[h,w]。改版则生成 C×H×W 门控：Y[c,h,w]=F[c,h,w]×A_pt[c,h,w]。差别就落在最后一个下标 c：改版能区分同一位置上的不同通道。

实现版本提醒论文 Fig. 5 明确画了 modified SAM，官方 sam_layer.c 也支持逐元素相乘；但官方仓库 master@59596d7 的标准 yolov4.cfg 没有 [sam]，SAM 只出现在实验配置里。因此，照论文画结构图和照默认 cfg 复现，不一定得到同一张网络图。

§ 03 / Training freebies四图拼接与自我攻击：训练难一点，部署不背债

Mosaic：四张图拼成一张训练样本

Mosaic先把四张图分别缩放、裁剪，再拼到同一张画布上。这样，同类物体会被塞进不同背景；物体缩小后，小目标也更常出现。图像怎么缩放、平移、裁剪，标注框就必须跟着做一模一样的坐标变换。

SAT：按模型自己的弱点现场出题

Self-Adversarial Training（SAT）不是拿对抗图做安全测试，而是让模型现场给自己出难题。第一段冻结模型，只改输入图像，把原本看得见的目标藏起来；第二段固定这张难图，再更新模型，把目标重新找回来。

§ 04 / Assignment & coordinates边界位置更好学，一个真值也能交给多个预设框

消除网格敏感：想预测 0.99，不必再把原始分数推得很大

YOLOv3 先让网络输出任意实数 t_x，再用 sigmoid 把它压到 0～1，作为目标中心在格子里的偏移。问题是：越靠近 0 或 1，所需的原始分数（logit）越大，梯度却越小，这就是网格敏感。公开实现改成 s·sigmoid(t_x)-0.5(s-1)：t_x 仍是网络原始输出，s 是把范围向两边撑开的缩放系数；s=1 就退回 YOLOv3，图中用 s=1.2。

一个真值可以交给多个 anchor 学

YOLOv3 默认只让形状 IOU 最高的那个预设框（anchor）负责学习。YOLOv4 仍保留最佳者，但其他 anchor 只要超过阈值，也能一起当正样本。遗传搜索给出的阈值是 0.213。它会让训练时多出几个负责者，却不会增加推理时生成的候选框数量。

§ 05 / Box lossCIoU：覆盖、中心、形状一张账单

普通坐标误差分别处罚 x、y、w、h；IOU 又只看两框重叠多少。CIoU 把三种一眼能看出的错误放进同一张账单：1-IoU 管覆盖不足，中心距离管整框偏到哪，长宽比项管框是不是太扁或太高。

L_CIoU = 1-IoU + ρ²(b,b_gt)/c² + αv v = 4/π² · [atan(w_gt/h_gt)-atan(w/h)]² α = v / (1-IoU+v)

b,b_gt 是预测框和真值框的中心；ρ² 是两中心距离的平方；c² 是能同时包住两框的最小外接框，其对角线长度平方；w,h 是预测框宽高，w_gt,h_gt 是真值框宽高；atan(w/h) 把长宽比变成角度，两个角度相差越大，v 越大；α 决定这一笔形状误差现在该占多大权重。

§ 06 / Training record论文设置、搜索结果、当前 cfg：三组数字别揉在一起

先把训练词汇对齐：一个 step 是更新一次权重；总 batch 是这次更新合计用了多少张图；mini-batch 是每次真正塞进显存多少张图。学习率、warm-up、momentum、weight decay则分别控制每步走多远、开局怎么慢慢提速、历史梯度保留多少、权重收缩多强。

ImageNet 主干筛选训练了 8,000,000 step：总 batch 128、mini-batch 32，初始学习率 0.1，poly 衰减，warm-up 1,000 step，momentum 0.9，weight decay 0.005。BoS 对照沿用这套训练量；BoF 对照多训练了 50%。因此分类表里的 BoF 提升，不是“训练时长完全相同，只多装一个技巧”的纯对照。

论文的 COCO 默认实验训练 500,500 step：初始学习率 0.01，在 400k 和 450k step 各缩小 10 倍；momentum 0.9，weight decay 0.0005；总 batch 64，单卡 mini-batch 4 或 8，并做多尺度训练。Cross mini-Batch Normalization（CmBN）只汇总同一个 batch 内几个 mini-batch 的统计量，不要求多张 GPU 互相同步。

遗传搜索又找到另一组数：学习率 0.00261、momentum 0.949、anchor IOU 阈值 0.213、框损失归一系数 0.07。当前 GitHub master 的 yolov4.cfg 则写着学习率 0.0013、subdivisions 8、Mosaic、CIoU。论文最终配方列的是 DIoU-NMS，当前 cfg 用的却是 greedynms。想复现结果，只记“YOLOv4”四个字不够，代码 commit 和 cfg 都要一起保存。

§ 07 / Ablation消融实验：热门组件装上去，也可能掉点

消融实验就是一次只换一个零件，看看成绩怎么变。分类实验里，BoF + Mish 把 CSPDarknet53 从 Top-1 77.2 / Top-5 93.6 提到 78.7 / 94.8；但分类更准，不保证检测也更准。CSPResNeXt50 的分类预训练变强后，检测反而从 42.4 AP 掉到 42.3；CSPDarknet53 才从 42.4 升到 43.0。

mini-batch 实验只能得出一个有限结论：带 BoF/BoS 的 CSPDarknet53 在 mini-batch 4 和 8 时是 41.6 vs 41.7 AP，只差 0.1；另一套不带这些策略的 CSPResNeXt50 是 37.1 vs 38.4，差 1.3。也就是说，这套配方在这组实验里对小 mini-batch 更稳；不能外推成“batch size 永远无所谓”。

§ 08 / Results & limits高门槛定位和小目标都变好，但提升没法逐件分账

YOLOv4-608 在 COCO test-dev 得到 43.5 AP / 65.7 AP50 / 47.3 AP75 / 26.7 small AP。AP50 把 IOU≥0.50 算命中，AP75 要求 IOU≥0.75，因此更考验框得准不准。论文还把 Maxwell、Pascal、Volta 三代 GPU 的 FPS 分开列，避免硬凑一张绝对榜。更稳妥的读法是看速度—精度前沿：精度差不多时谁更快，速度差不多时谁更准。

没有详细错误分类论文没有分别统计定位错、类别错和背景误报；只能从 AP50/AP75 与大中小目标分数看总体变化。
消融没有把每项功劳完全拆开不少实验同时换了骨干、输入尺寸或多个技巧，没法给每个模块分出一笔精确收益。
仍要维护 anchor 与 NMS训练要选 anchor 形状和正样本阈值，推理还要用 NMS 去重；后来的 YOLOX 才转向 anchor-free。
代码和配置必须一起存论文清单与当前仓库 cfg 已经有差异。想复现“YOLOv4”，模型名、代码 commit、cfg 缺一不可。
一句话带走YOLOv4 留下的不是一长串缩写，而是一套实用的选件方法：先把检测器拆成主干、融合层和检测头；再把训练成本与部署成本分开算；最后看真实 GPU 速度和消融结果，决定哪些零件值得留下。

引用 / 术语表

  1. Backbone · 主干网络 · 图片先过这里，逐层变成特征图。它负责提取边缘、纹理、部件和语义，不负责最后报框；YOLOv4 用的是 CSPDarknet53。
  2. Neck · 特征融合层 · 主干和检测头之间的中转站。它把浅层清楚的位置信息与深层强的类别语义对齐、合并；YOLOv4 用 SPP + PAN。
  3. Bag of Freebies · 训练期技巧 · 只在训练时多花算力，导出的推理网络不多一层。“免费”只指部署延迟，不是训练不要钱；见两类成本账。
  4. Bag of Specials · 推理期组件 · 部署后仍要执行的模块或后处理。它们通常能换来精度，但 FPS 也会真受影响。
  5. CSP · 分流再合流 · 把通道分两路：一路反复加工，一路走短路，末尾再拼回来。不是砍掉一半特征，而是避免所有通道重复跑完整条加工线。
  6. Receptive field · 感受野 · 一个输出格往回追，最多能参考原图多大区域。感受野大，说明它能借更远的上下文判断；不等于当前卷积核本身很大。见完整解释。
  7. Mosaic · 四图拼接 · 把四张图随机缩放、裁剪后拼成一个训练样本，框坐标跟着变；既混背景，也让小目标更常出现。见坐标例子。
  8. SAT · 自对抗训练 · 先冻结权重、修改输入，把模型本来认得出的目标藏起来；再固定难图、更新权重，把目标重新找回。它是训练增强，不是部署时的安全攻击。
  9. CmBN · 跨小批次归一化 · 普通 BN 要估计每个通道的均值和方差；mini-batch 太小时容易估不准。CmBN 会汇总同一个总 batch 内几个 mini-batch 的统计量，但不跨 GPU。
  10. Grid sensitivity · 网格边界难学 · sigmoid 越靠近 0 或 1 越平。目标中心若贴着格子边缘，网络就得输出很大的原始分数，梯度反而更小，学起来费劲。
  11. CIoU · 三项框损失 · 同时检查覆盖够不够、中心偏多远、长宽比像不像。它不是只看重叠；见从 4×2 真值框手算到 0.73167。
  12. Pareto frontier · 速度—精度前沿 · 一组各有所长、无法直接淘汰任何一个的方案：若想比其中一个更准，就得牺牲速度；若想更快，就得牺牲精度。读检测榜时应沿这条前沿比较，而不是只找单项第一。
  13. Pooling · 池化 · 用窗口汇总邻域：最大池化取最强响应，平均池化取平均水平。会不会缩图取决于步长和补边；见3×3 滑窗与 SPP 手算。
  14. Spatial-wise / point-wise attention · spatial-wise 每个空间位置只有一个门，所有通道共用；point-wise 则给每个 (通道, 高, 宽) 元素单独一扇门。见双通道 2×2 手算。
  15. Head · 检测头 · 检测器的输出层。它把融合后的特征翻译成候选框位置、是否有物体以及类别分数；YOLOv4 沿用 YOLOv3 的三尺度检测头。
  16. Feature map / channel · 特征图与通道 · 特征图不是原图，而是一摞响应网格。每个通道可以理解成一种模式探测器：有的对竖边敏感，有的对纹理或部件敏感。见卷积如何生成特征图。
  17. Stride / padding · 步长与补边 · stride 决定窗口每次移动几格；padding 在边缘外补值。步长大通常会缩图，合适的补边能保持宽高；见卷积与池化。
  18. Concat / shortcut · 拼接与相加 · concat 沿通道把两路并排保留，通道数会变多；shortcut/add 把两路同位置直接相加，通道数不变。前者更自由，后者更省。
  19. Anchor · 预设框 · 每个网格位置先摆几种常见宽高模板，网络不用凭空生成框，只预测中心和尺寸该怎样校准。见YOLOv2 的完整解码例子。
  20. AP / AP50 / AP75 · 检测精度 · AP 同时惩罚误报和漏报。AP50 把 IOU≥0.50 算命中，AP75 要求 IOU≥0.75，后者更考验框得准不准；COCO AP 还会平均多个 IOU 门槛。AP-small 只统计面积较小的目标。见完整计算。
  21. FPS / BFLOPs · 实测速度与理论运算量 · FPS 是机器一秒实际处理多少帧；BFLOPs 是一次前向大约做多少十亿次浮点运算。算得少可能仍跑得慢，因为显存访问、并行度和算子实现也会卡速度。
  22. Batch / mini-batch / subdivisions · 总 batch 是更新一次权重合计用了多少张图；mini-batch 是每次真正塞进显存多少张；subdivisions 表示要把总 batch 拆成多少小份再累积梯度。
  23. Logit / sigmoid · 原始分数与压缩函数 · 网络先输出可正可负、没有范围限制的原始分数 logit；sigmoid 再把它压到 0～1。越贴近 0 或 1，曲线越平，梯度越小。
  24. IOU / NMS · 框重叠与去重 · IOU 是交集面积÷并集面积，用来量两个框有多重合；NMS 保留高分框，再删除和它过度重叠的重复框。greedynms 用 IOU 判断该不该删；DIoU-NMS 还把两框中心距离算进去。
  25. Ablation · 消融实验 · 从完整方案里一次拿掉或换掉一个零件，其他条件尽量不动，再看分数变化。若一次改了多项，就不能把差值全算到某一个模块头上。
  26. 训练旋钮与搜索 · 学习率管每步走多远；warm-up 让开局从小步慢慢提速；momentum 累积一部分历史梯度；weight decay 持续把权重往小处拉。poly 和余弦退火是两种逐步降低学习率的曲线；遗传搜索则保留表现较好的超参数组合，再做随机变异，反复筛选。
  27. CutMix / DropBlock / 标签平滑 · 三种训练期正则化：CutMix 把一张图的矩形区域换成另一张图，并按面积混合标签；DropBlock 连片遮住特征图，逼模型别只依赖一个局部；标签平滑不把正确类写死成 100%，避免模型过度自信。
  28. Mish · 激活函数 · 卷积先算出任意实数，Mish 再做一层平滑的非线性变换，让网络不只是多层线性运算的叠加。它和 ReLU 干的是同类工作，但负数一侧不会直接全部截成 0。
  29. MiWRC · 多输入加权残差连接 · 普通残差连接把一路旧特征加回当前层；MiWRC 允许多路旧特征一起汇入，并给每一路学习一个权重。网络可以自己决定更信哪条旧路径。
  30. RFB / ASFF / Gaussian YOLO · 消融候选 · RFB 用多种卷积窗口和膨胀率扩大感受野；ASFF 学习不同尺度特征该各占多大权重；Gaussian YOLO 让框坐标同时预测不确定性。它们在这页的作用是说明“候选听起来合理，组合后仍可能掉分”。
  31. Top-1 / Top-5 · 分类命中率 · Top-1 只看模型得分最高的类别是否正确；Top-5 只要正确答案落在得分最高的五类里就算命中。它们评的是分类，不是检测 AP。
  32. MSE · 均方误差 · 预测值与真值先相减、再平方、最后取平均。它能逐项罚坐标差，却不像 CIoU 那样直接判断两个框是否盖住同一块区域、中心是否对齐、形状是否相似。
