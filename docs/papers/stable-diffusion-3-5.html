Stable Diffusion 3.5 ·
文字和图像，坐到同一张桌子。

Stability AI · 基于 SD3《Scaling Rectified Flow Transformers》· arXiv 2403.03206 · 模型 2024-10 开源

老文生图把文字当"调料"从旁边塞进图像主序列。SD3/3.5 把文字 token 和图像 token 拼成一条序列、过同一个注意力——但两种模态各用各的权重(MMDiT)。再换掉加噪去噪那套、改用近乎直线的整流流少步采样，加 QK 归一化稳住大规模训练。开源 Large 8B / Medium 2.5B。

TL;DR

SD3.5（开源权重，2024-10）是 SD3 那套架构的产品化版：整流流1（不走 DDPM 加噪去噪，学一条噪声↔图的近乎直线路径 → 少步出图）+ MMDiT2（文字 + 图像 token 拼成一条序列共享注意力，但两种模态各用各的一套权重，双向对齐 → 长 prompt / 排版大涨）。条件靠三个文本编码器 CLIP-L + CLIP-G + T5-XXL3（T5 可丢省显存）。3.5 相比 3 的关键补丁：QK-Norm4 稳训 + MMDiT 块里双注意力层。还在 16 通道 VAE latent 空间画。家族：Large 8B / Large Turbo（蒸馏 ~4 步）/ Medium 2.5B（MMDiT-X，消费级能跑）5。

§ 01痛点：文字只是"塞进来的调料"

老的文生图（SD1.5 / SDXL 那代）主干是 U-Net + cross-attention：图像是主角，一条图像特征序列从头走到尾；文字编码后从旁边通过 cross-attention "喷"进每一层。文字像炒菜时从锅边淋进去的调料——能影响味道，但它自己不在锅里翻炒，跟图像不是平等的两方。

后果很实在：长 prompt 顾此失彼（"红帽子的猫站在写着 OPEN 的牌子旁"——颜色、主体、文字招牌容易串）、拼写/排版差（生成的英文单词缺胳膊少腿）。根子在文字和图像的信息流不对等。SD3 的主张：让两种模态平起平坐。

§ 02不走加噪去噪：整流流的"直线少步"

先换掉生成的底层范式。传统扩散（DDPM）是"图 → 一点点加噪 → 纯噪声"，生成时反着一步步去噪，路径弯弯绕绕、要几十步。整流流（Rectified Flow）直接学一条噪声 ↔ 图之间近乎直线的路径（就是你已学的 flow-matching 那套）——路径越直，采样需要的步数越少。

SD3 还在训练时用 logit-normal 时间步采样：不平均地采整条轨迹，而是偏向中段（噪声和信号势均力敌、最难学的那段），论文说这是"把噪声采样偏向感知上重要的尺度"。Turbo 版进一步用蒸馏把步数压到 ~4 步（同 DMD 那条少步化线）。

§ 03主干 MMDiT：同一张桌子，各带各的笔记本

这是 SD3/3.5 最值钱的创新。把"图像主角 + 文字调料"的不对等关系，换成两种模态平起平坐：图像 token 和文字 token 拼成同一条序列，一起过同一个自注意力——但每种模态各用各的一套权重（各自的 QKV 投影、各自的 MLP）。

类比：cross-attention 像说两种语言的人靠传话筒隔房间递纸条；MMDiT 像让他们坐到同一张桌子上——各自带自己的笔记本（独立权重，因为图像和文字的统计规律本就不同），但能直接对视、双向交流（共享同一个自注意力）。文字和图像的对齐因此强得多，这正是 SD3/3.5 在长指令理解和文字渲染上明显变好的根。

SD3.5 在此之上还把 MMDiT 块里的单注意力换成双注意力层（Medium 版叫 MMDiT-X：在前 13 层额外加自注意力），进一步改善多分辨率一致性和整体连贯。

§ 04三个文本编码器 + QK-Norm 稳训

三编码器并联喂条件：CLIP-L + CLIP-G 给"图文对齐过的语义"，T5-XXL 给"长句、细节、拼写"的纯语言理解。T5 很吃显存，推理时可以临时丢掉只用两个 CLIP 省内存（掉一点细节服从）。

3.5 相比 3 的关键稳定性补丁是 QK 归一化。大模型训练时，注意力的 Q·K 点积容易随规模数值爆炸：logit 飙到很大 → softmax 饱和成 one-hot → 梯度消失、训练崩。在算 attention 前先把 Q、K 各自归一化，把这个隐患摁住。

QK-Norm 现在是大 Transformer 的标配（你已学的 qk-rmsnorm 是同一招，Qwen-Image-2.0 也用）。再加上还在 16 通道 VAE 压缩空间里做扩散（不在像素上直接画，省算力），整套就齐了。

放进 wiki 这条线

SD3.5 把你学的几块拧在一起：flow-matching/整流流（少步采样）+ MMDiT（文字图像同序列、双权重）+ CLIP/T5 文本编码 + QK-Norm（稳训）+ VAE latent + DMD 蒸馏（Turbo）。一句话：把"文字当调料"换成"文字和图像坐同一张桌子"，再用直线少步的整流流跑完。同样吃 MMDiT 的还有 Qwen-Image-2.0 / MRT。

引用 / 术语表

  1. 整流流 · Rectified Flow · 不走 DDPM 加噪去噪，而是学一条噪声↔数据之间近乎直线的速度场（flow-matching 家族）。路径越直，采样步数越少。SD3 还用 logit-normal 时间步采样把训练算力偏向轨迹中段。
  2. MMDiT · 多模态扩散 Transformer · 文字 + 图像 token 拼成一条序列、共享同一个自注意力实现双向对齐，但每种模态各用各的一套 QKV/MLP 权重。SD3 的招牌创新，长 prompt 服从与排版大涨的根。
  3. 三文本编码器 · CLIP-L + CLIP-G（图文对齐语义）+ T5-XXL（长句/细节/拼写的纯语言理解）。三者输出拼起来当条件；T5 吃显存，可丢掉省内存。
  4. QK-Norm · 算 attention 前对 Q、K 各自归一化，防止 logit 随规模数值爆炸 → softmax 饱和 → 训练崩。SD3.5 相比 SD3 的关键稳定性补丁，现已是大 Transformer 标配。
  5. SD3.5 家族 / MMDiT-X · Large 8B、Large Turbo（蒸馏 ~4 步）、Medium 2.5B。Medium 用 MMDiT-X：在 MMDiT 前 13 层额外加自注意力，改善多分辨率一致性，目标消费级硬件可跑。
