01 · WHY JOINT先视频后配音，究竟少了什么

`T2V → V2A` 只能让声音服从已经定稿的画面；`T2A → A2V` 只能让画面服从已经定稿的声音。问题不是第二个模型不够强，而是第一步一旦交卷，另一模态再也改不回去。

Fig 01 · 串行生成与联合生成ONE-WAY vs TWO-WAY

串行：先交卷，再补另一科

视频先定嘴型和碰撞时刻 → 音频只能追；或音频先定节奏 → 画面只能追。反馈箭头断掉。

联合：两份草稿反复对齐

台词修嘴型，人物位置修音源，碰撞画面修 Foley17 时刻，房间材质修混响；两边还在去噪，都能改。

同一个事件通常同时产生视觉和声音。LTX-2 想建模的是文本条件下的联合分布，而不是把两个条件分布串起来01。不过“架构允许双向依赖”不等于论文已经用互信息或似然证明它完美学到了真实联合分布。

03 · ONE DUAL-STREAM BLOCK一层里，先各自思考，再交换答案

Fig 03 · 每个 block 的四道工序SELF → TEXT → A/V → FFN
① Self-Attention视频内部建空间与运动；音频内部建语音、音乐、环境声的连续性。
② Text Cross-Attn两条流分别读取为自己整理过的 prompt 条件。
③ A ↔ V Cross-Attn沿共同时间轴双向取信息，是同步发生的位置。
④ FFN每个 token 在通道维做非线性变换，再进下一层。

RMSNorm04 穿插在各子层之前稳定尺度。当前官方实现还先保存交换前的 `video/audio` 快照：A→V 与 V→A 都读取同一份旧状态，第二个方向不会偷看到第一个方向刚写回的结果。这让一层内的双向交换保持对称。

论文 vs 实现论文正文没有列 block 数；当前官方 `ltx-core` README 与代码给出 48 层。本文把它标为实现补充，而不是伪装成论文表格里的公开训练配置。

04 · TEMPORAL ALIGNMENT画面中的撞击，怎么查到同一刻的声音

视频 query 读取音频 K/V，就是 A→V；音频 query 读取视频 K/V，就是 V→A05。

A_{V\leftarrow A}=\operatorname{softmax}\!\left(\frac{Q_VK_A^\top}{\sqrt d}\right)V_A
Q_V视频 token 提出的查询：我现在需要什么声音线索？K_A音频 token 的索引，用来与 query 比较。V_A匹配后真正取回的音频内容。dQ/K 的共同通道宽度；除以 √d 防止点积过大、softmax 过早饱和。

softmax([1,3,1])≈[.107,.787,.107]。若三个音频 value 的碰撞强度为 `[.2,1,.1]`，加权输出约 `.819`。这只演示一次 attention 的算术。

视频 self-attention 使用 3D RoPE06 编码 `(x,y,time)`，音频 self-attention 使用 1D 时间 RoPE；跨模态时只保留时间。原因很朴素：声音没有画面里的 `x,y`。跨模态位置编码只负责“是不是同一刻”，该看嘴、车还是手仍由内容相似度决定。

05 · CROSS-MODALITY ADALN自己的进度调制，对方的进度开门

联合生成时两条流可能噪声程度不同。V2A 甚至可以让给定视频保持干净 `σ_video=0`，只去噪音频。交叉注意力必须知道：谁现在更可靠07。

h'_V=h_V+g(\sigma_A)\odot\operatorname{CrossAttn}\!\left(\operatorname{AdaLN}(h_V;\sigma_V),\operatorname{AdaLN}(h_A;\sigma_A)\right)
h_V,h_A交换前的两条隐藏状态。σ_V, σ_A视频与音频当前噪声级别。AdaLN用本流 σ 算 scale/shift，先把自己整理到适合交换的尺度。g(σ_A)由对方噪声进度算出的写回门；⊙ 是逐通道相乘。

06 · DEEP TEXT CONDITIONING为什么不能只拿 Gemma 最后一层

冻结的 Gemma 3-12B08 会产出所有 decoder 层的隐藏状态。论文的出发点是：较早层可能保留更多字词与音素线索，较深层偏向复杂语义；准确语音两者都需要。

H\in\mathbb{R}^{B\times T\times D\times L}\;\longrightarrow\;\widetilde H\in\mathbb{R}^{B\times T\times(DL)}\;\xrightarrow{\;W\;}\;C\in\mathbb{R}^{B\times T\times D_{target}}
Bbatch 大小。T文本 token 数。DGemma 每层隐藏宽度。L抽取的层数。W把所有层拼接后的高维特征压回目标维度的可学习矩阵。

08 · MODALITY-SPECIFIC LATENTS视频压空间与时间，音频压时频结构

视频：让 DiT 不必面对每个像素

09 · FLOW MATCHING两条流训练时，究竟在预测什么16

x_\sigma=(1-\sigma)x_0+\sigma\epsilon,\qquad v^*=\epsilon-x_0,\qquad \mathcal L=\lVert v_\theta(x_\sigma,\sigma,c)-v^*\rVert_2^2
x₀VAE 编码后的干净视频或音频 latent。ε与 latent 同形状的随机噪声。σ0 到 1 的插值位置；0 靠近数据，1 靠近噪声。xσ模型训练时看到的带噪 latent。v*直线路径的真实速度。由 dxσ/dσ 求导得到 ε−x₀。vθ双流 DiT 预测的速度。‖·‖²₂每一维误差先平方，再全部求和。

补充材料只明确“video/audio latents 用 flow-matching velocity loss”。它没有披露两项如何加权、σ 如何采样、optimizer、batch 或训练步数。推理时从两条噪声 latent 出发，用 Euler 等求解器反向积分，最后各自由 VAE 解码。

10 · MODALITY-CFG“符合文字”和“彼此同步”可以分开调

论文原式用 `t` 表示 text，容易和 diffusion timestep 混淆；这里改成 `c`。`m` 表示另一模态13。

\widehat M=M(x,c,m)+s_{text}[M(x,c,m)-M(x,\varnothing,m)]+s_{cross}[M(x,c,m)-M(x,c,\varnothing)]
M(x,c,m)文本与另一模态都给全的预测。M(x,∅,m)拿掉文本，用差值测文本造成的变化。M(x,c,∅)拿掉另一模态，用差值测跨模态造成的变化。s_text放大文本方向。s_cross放大音视频互相对齐的方向。

  • Gemma 全层特征、thinking registers

最后的判断：LTX-2 最有价值的设计原则是统一生成不等于抹平模态差异。视频与音频使用各自合适的 latent、容量和位置编码，只把真正共同的时间关系做成高频通信接口。速度优势有实数支撑；质量 SOTA、thinking token 收益与可复现训练配方，则需要更多公开证据。

14 · GLOSSARY点过的术语，在这里展开

  1. 联合分布：一次描述视频与音频共同出现的概率关系。它允许两边同时受同一个事件约束，而不是先固定一个再条件生成另一个。
  2. VAE：把高维原始信号压成连续 latent，并能再解码回来。这里视频和音频各用一套。
  3. DiT：在 latent 上运行的 Diffusion Transformer；LTX-2 用它预测 flow velocity。
  4. RMSNorm：用向量平方均值的平方根统一尺度，不减均值；用来稳定深层网络的激活。
  5. Cross-Attention：query 来自一组 token，key/value 来自另一组；因此能用视频问题检索音频，反之亦然。
  6. RoPE：按位置旋转 Q/K，使点积自然带上相对位置。3D 表示 x/y/time，1D 只表示时间。
  7. Cross-modality AdaLN：用本流噪声进度调制交换前的特征，再用对方噪声进度控制交叉信息写回量。
  8. Gemma 3-12B：冻结的 decoder-only 多语言 LLM；LTX-2 读取它所有层的隐藏状态作为文本特征来源。
  9. Text Connector：Gemma 后面的双向 Transformer，重新混合整句，并分别产出视频/音频条件。
  10. Thinking Tokens / Registers：替换 padding 的可学习隐藏槽，可聚合上下文；不是自然语言思维链。
  11. Causal Video VAE：压缩空间与时间，且每个时间位置只依赖现在和过去，便于流式处理。
  12. Vocoder：把 mel-spectrogram 合成为可播放波形的神经网络；不负责理解 prompt。
  13. Modality-CFG：分别拿掉文本、拿掉另一模态，抽出两条预测差值，再用两个 scale 放大。
  14. Standard CFG：常写作 uncond + s(cond−uncond)，放大条件相对无条件造成的预测变化。
  15. Blend Window：重叠 tile 中随位置变化的融合权重，让一块逐渐淡出、邻块逐渐淡入，避免接缝。
  16. Flow Matching：构造数据到噪声的路径，让网络回归路径速度；推理时从噪声反向积分到数据。
  17. Foley：与画面动作同步的拟音，如脚步、衣物摩擦、杯子碰桌面。
  18. Mel-Spectrogram：按时间展示各频带能量的图，频率轴按人耳感知尺度压缩。
