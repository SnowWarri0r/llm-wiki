把高保真 TTS 做成跟标准 LLM 同构 —— 一篇带架构图与术语注解的中文解读。

TTS 行业过去 5 年的命题一直是同一道不可能三角：音质要高 ↔ 模型要大 ↔ 推理要快。fish-speech S2 的招数是把音频生成做成跟标准 LLM 结构上同构 —— 一旦同构，所有 LLM 推理基建白嫖；分两层 AR 把高采样率 token 流的成本降下来。这是一篇把 Dual-AR / RVQ / GRPO 三件套讲清楚的中文版，配三张可播放的动画图，加一份术语词典。

§ 01 · the bottleneck不可能三角

第三代 TTS（神经声学 + vocoder pipeline）的标准做法是把"说话"切成三段独立模型：phonemizer（文本 → 音素）→ 声学模型（音素 → mel 频谱）→ vocoder（频谱 → 波形）。每段单独训练，对齐靠规则，三段都受限于上下两端的接口。

第四代是 neural codec4 + LLM 路线（VALL-E / Voicebox / XTTS / fish-speech 都在此列）：

  • codec：可学的"音频压缩器"，把 waveform 压成离散 token 序列
  • LLM：transformer 在 token 序列上做自回归
  • 解码：codec 反推 waveform

这条路漂亮在端到端可微 + 跟文本 LLM 结构一致，但有个根本矛盾叫"不可能三角"：

维度想要代价
音质高保真codec 帧率高 → token rate 高（每秒上千 token）
容量大模型每 token 要过的参数多
速度实时 / 流式token_rate × params = 单帧计算量，必须 < 实时

三个全要 = 现实里推不动。第四代 TTS 团队各自的招数其实都在这三角里找妥协点。fish-speech S2 的解法：分两层 AR，把"大模型每帧只算一次，小模型每帧算多次" —— 见下一节。

背景 · 为什么不用 diffusion

diffusion-based TTS（Voicebox / NaturalSpeech 3 等）走的是连续路径：模型不预测离散 token 而是学一个速度场，多步采样到目标 mel。质量天花板高，但推理要 ODE 求解 N 步，难做到极低 TTFA（首音延迟）。codec + AR 路线推一步出一帧，延迟可控 —— 这是 fish-speech 选 AR 而不是 flow matching 的关键工程取舍。

§ 02 · the core trickDual-AR · 慢 + 快

S2 用了一个主从两层的自回归结构：

  • 慢 AR (4B) · 沿时间轴每帧吐 1 个主语义 codebook —— 决定这一帧"说什么"
  • 快 AR (400M) · 在同一帧上顺序吐 9 个残差 codebook —— 决定"怎么发"（音色 / 咬字 / 气声）

RVQ codec 把音频压成 10 路 codebook、~21 Hz 帧率（约 47ms 一帧）。慢 AR 算少次但算重；快 AR 算多次但便宜。乘下来推理预算守得住。下图把这个时序展开看：

关键工程取舍 · 跟 LLM 同构

慢 AR 和快 AR 都是 decoder-only transformer，跟标准 LLM 结构上一致。这意味着 SGLang9 的全部推理优化（paged KV cache8 / RadixAttention10 / CUDA graph11 / continuous batching）直接复用，不用 codec-aware 自研推理栈。这是 fish-speech 速度上能甩开其他第四代 TTS 的根本原因之一。

§ 03 · the codecNeural Codec · 编 / 量 / 解

fish-speech 用的"codec"不是 MP3 那种规则编解码。它是一整套神经网络 —— encoder、codebook、decoder 三件套联合训练。这一节先把整体架构画清楚，再讲 RVQ 怎么把"一个 codebook"扩展成"10 层 codebook"。

03.1Neural codec 全貌

三个组件 + 一条 reconstruction loss：

  • Encoder（卷积下采样栈）把 16 kHz 波形压成 ~21 Hz 的连续向量序列（每 ~47ms 一个 d 维向量）
  • Codebook（一个 K=1024 大的查找表，每条 entry 是个 d 维向量）把每个连续向量 snap 到最近的码字 → 它的索引是一个整数 token
  • Decoder（转置卷积上采样栈）拿 token 查表得到向量，再还原成 16 kHz 波形

训练时三件套同时优化，loss = 重建误差（让 decoder 输出尽量像 encoder 输入）+ VQ 损失（commitment + codebook update）。codebook 不是预先定的，是跟着学出来的 —— 训完之后码字恰好覆盖人类语音的统计结构，这是 neural codec 最神奇的地方。

03.2RVQ · 把 codebook 摞 10 层

单个 codebook 对音频来说不够 —— 要么 K 太小（量化误差大）要么 K 太大（参数爆且训不起来）。RVQ 的招：

  • 第 1 层：用 codebook₁ 粗略量化输入向量 x，留下残差 r₁
  • 第 2 层：用 codebook₂ 量化 r₁，留下更小的残差 r₂
  • 第 N 层：用 codebookₙ 量化 rn-1，rn 已经接近 0

每层独立学自己的 codebook。"叠"出来的总效果跟一个超大 codebook 等价，但参数和训练成本都低得多。类比 JPEG 渐进式编码：先一个低分辨率版本，再叠细节层。播放下图看一帧完整的残差量化过程：

跟 dMel（TML 用的）的区别

RVQ 是学出来的 codebook，跨数据集自动适配，表达力强 —— 但需要联合训练 encoder + decoder + codebook（VQ-VAE 风格，带 commitment loss）。

dMel 是规则离散化 Mel 频谱（每个频带按预设格点切），无需训 codebook，简单可控 —— 但只适合做输入侧（音频 → token 喂 transformer）。要做输出侧（token → 音频）还得另接一个连续模型，TML 选了 flow matching。fish-speech 输入输出全用 RVQ，端到端一致。

§ 04 · the alignmentGRPO · 组相对 RL

预训练完的 S2 还要做 post-training alignment，让它语义更准、情感 tag 更服从、音色更像参考。用的是 GRPO6（Group Relative Policy Optimization），DeepSeek-Math 那篇提出来的，DeepSeek-R1 也用。

GRPO 是 PPO7 的简化版：砍掉 critic（value network），用同 prompt 一组采样之间的相对 reward 当 advantage。直觉：考试不需要绝对分数，只看你在班里的排名。

04.1四维 reward 信号

优雅之处 · self-rewarding

Reward model = 数据清洗 / 标注用的同一套模型。传统 RLHF 的痛点：reward model 训在另一份数据上，跟 pre-train 数据分布有 gap → policy 学到的是"如何讨好 reward model"而不是"真正变好"。fish-speech 把"挑数据 / 给标签"和"打分"用同一套模型做，distribution mismatch 自动消失。

§ 05 · the plumbing推理工程

"跟 LLM 同构"这条不只是好看的口号，它直接换成了具体的工程数字。H200 单卡上：

  • TTFA（time-to-first-audio）~100ms · 用户点回车到第一帧音频出来
  • RTF（real-time factor）0.195 · 生成 1 秒音频只要 ~200ms，远超实时 5×
  • 3000+ acoustic tokens/s · 持续吞吐量 · RTF 始终 < 0.5

能做到这数字，靠的是白嫖整套 SGLang 优化：

  • Paged KV cache8 · 多个 session 共享 GPU 内存池，碎片化最小化
  • Continuous batching · 不等整批跑完，新请求随时插入
  • CUDA graph11 · 把固定计算流图编译成 GPU 原生 graph，省 launch 开销
  • RadixAttention10 · prefix sharing —— 相同 prompt 前缀的 KV cache 自动复用

如果 fish-speech 自研一套 codec-aware 推理引擎，这些优化每条都要重新实现一遍。"跟 LLM 同构"省的就是这部分天文级工程成本。

§ 06 · the controls控制与多语言

06.115000+ 自由文本 tag

不是固定枚举，是开放词汇。常用的有：

[pause] [emphasis] [laughing] [inhale] [chuckle] [tsk] [singing]
[excited] [laughing tone] [interrupting] [chuckling] [excited tone]
[volume up] [echo] [angry] [low volume] [sigh] [low voice] [whisper]
[screaming] [shouting] [loud] [surprised] [short pause] [exhale]
[delight] [panting] [audience laughter] [with strong accent]
[volume down] [clearing throat] [sad] [moaning] [shocked]

但更牛的是自由描述：[whisper in small voice] / [professional broadcast tone] / [pitch up] 都能写。模型在大规模带标注语料上学的是"自然语言 → 声学特征"的映射，不需要枚举所有 tag。

06.280+ 语言不靠 phonemizer

Tier语言含义
1ja / en / zh主力 · 量大、质量最高
2ko / es / pt / ar / ru / fr / de覆盖主流 · 质量接近 Tier 1
3+70+ others · sv / it / tr / no / nl / cy / eu / ca / da / ...能用 · 含 sa / la / yi 等小语种

关键是不需要 phonemizer / G2P。第三代 TTS 每种语言都要一套"文本 → 音素"的规则或小模型，新增语言成本高。fish-speech 直接把 raw 文本 token 喂主模型，音素映射隐式从 codec + audio 监督中学出来。这又是一例 把外挂规则吃进权重 的范式。

§ 08 · what to keep个人解读

  1. "跟 LLM 同构"是这篇里最值得偷的设计抉择。直接复用所有 LLM 推理基建 → 工程 ROI 极高。任何"新模态 + transformer"的项目都该问一句：能不能把核心结构跟标准 LLM 对齐？
  2. Dual-AR 的本质是把单一 AR 的 token 流上的成本摊到两层。慢的算少次但算重，快的算多次但便宜。这个 trick 不限于音频 —— 任何"高 token rate + 大模型"场景都可以借鉴：视频 token、机器人控制信号、HDR 图像 RGB stream 全是。
  3. GRPO 的 self-rewarding（reward model = annotation model）是 RLHF 漂移问题的优雅解。值得在自己的项目里复制：如果你已经有一套数据清洗的小模型，复用它当 reward model 而不是单独训。
  4. RVQ vs flow matching 是个工程取舍而不是理论优劣。RVQ 适合服务化 TTS（白嫖 LLM infra），flow matching 适合实时双向交互（端到端可微 → 跟主 transformer 联训）。详见跨源 topic audio-tokenization-rvq-vs-flow（待做 bespoke）。
  5. 跟 TML interaction models 是不同维度的"双层"。fish-speech Dual-AR 是生成路径上的主从（慢/快沿 codec 层级分工）；TML 的 dual-model 是控制流上的前后台（沿时间尺度分工）。两个值得对照学。

§ ▼ · vocabulary术语词典

正文里带波浪底的词都对应这里一条。按出现顺序编号：

01 AR  (autoregressive)

"自回归"。模型生成时第 t 步的输出依赖前 t-1 步的输出（往回看）。语言模型、TTS、视频生成几乎都是 AR 范式。AR 推理是串行的（每步等上步结果），所以推不动是常态。Dual-AR 的"两层"是结构上的，每层内部还是 AR。

02 codebook  / VQ

VQ = Vector Quantization。把连续向量映射到最近邻的离散"码字"。codebook 就是这堆码字组成的查找表（比如 16384 个 1024 维向量）。VQ-VAE 把 encoder 输出连续向量量化到 codebook，VQ 离散索引就是后续 transformer 的 token。RVQ 是多层串联的 VQ。

03 RVQ codec

Residual Vector Quantization。多层 codebook 串联，第 k 层量化前 k-1 层留下的残差。fish-speech S2 用 10 层、~21 Hz 帧率。整套（encoder + N 层 codebook + decoder）联合训练，loss = 重建误差 + commitment loss + codebook loss（VQ-VAE 风格）。前几层贡献最大，后几层补细节。

04 neural codec

"用神经网络学出来的音频压缩器"。传统编解码（MP3 / Opus）用规则 + 信号处理；neural codec 用 encoder + 量化（一般是 RVQ）+ decoder 联合训练。代表：SoundStream / Encodec / DAC / fish 的自研 codec。它的"token"是 LLM 时代音频模态的事实接口。

05 主语义 codebook

RVQ 第 1 层 codebook。承载的是音频里最"宏观"的信息：大致是什么音、什么音节、什么韵律。fish-speech 让慢 AR（4B）专门预测这一层，因为这是决定"说什么"的核心。后续 9 层 codebook 由快 AR（400M）顺序生成，决定"怎么发"。

06 GRPO

Group Relative Policy Optimization。PPO 的简化版：不要 critic（value network），只用同 prompt 一组采样的相对 reward 算 advantage。来自 DeepSeek-Math 论文，DeepSeek-R1 / fish-speech S2 等都用。优点：少一个跟 policy 一样大的网络 → 训练成本砍一半 + 更稳定。

07 PPO

Proximal Policy Optimization。OpenAI 2017 提的 on-policy RL 算法，是 ChatGPT 那波 RLHF 的工作马。需要训一个 value network（critic）做 baseline。优点是稳；缺点是 critic 训练 + 内存开销大。GRPO 砍掉了 critic 用组内 normalize 代替。

08 Paged KV cache

vLLM 那篇论文（Kwon 2023）提出的核心创新。传统 KV cache 给每个 session 预留连续大块 GPU 内存 → 内存碎片严重，多 session 并发吃亏。Paged KV 借鉴 OS 虚拟内存的思路：把 cache 切成固定大小的 page，按需分配，多 session 共享内存池。GPU 显存利用率从 ~30% 提到 90%+。

09 SGLang

LMSys 团队（vLLM 之后）做的下一代 LLM 推理引擎。在 vLLM 基础上加 RadixAttention、co-runtime DSL、更激进的图优化。fish-speech S2 选 SGLang 是因为他们专门做了对 multi-modal / dual-AR 结构的优化。

10 RadixAttention

SGLang 的核心优化。多个请求共享相同 prompt 前缀时，传统做法每个请求各算各的 KV，浪费。RadixAttention 用一棵 radix 树管理所有 session 的 KV cache，相同前缀自动共享。对 TTS 场景特别有用：很多请求会共用 system prompt 或 voice reference 前缀。

11 CUDA graph

NVIDIA 提的优化：把一段固定的 CUDA kernel 调用序列编译成"原生 graph"一次性 launch，省去每个 kernel 单独的 launch 开销（~5μs/次）。小模型 / 低延迟场景受益最大。LLM 推理（每步几十次 kernel）用了能省 10-20% 延迟。

12 TTFA  (time-to-first-audio)

"出第一帧音频的延迟"。用户输入 → 第一帧音频被解出来的时间。TTS 场景最重要的体验指标。fish-speech S2 在 H200 上 ~100ms —— 已经低到人耳几乎感知不到延迟。流式播放靠这条数字。

13 RTF  (real-time factor)

生成 1 秒音频所需的实际计算时间 / 1 秒。RTF < 1 = 比实时快。fish-speech S2 RTF 0.195 表示 1 秒音频只要 195ms 算完。RTF 越小越好，但太小没意义（不会带来体验提升）；RTF 越接近 1 越省 GPU。
