FIELD NOTES · 在线向量量化 · 随机旋转

TurboQuant ·
把未知,旋成已知。

Google Research · Online Vector Quantization with Near-optimal Distortion Rate · arXiv 2504.19874 · ICLR 2026

在线量化的死结:边来边压、看不到数据统计,不知道量化格子该切在哪。TurboQuant 一个几何洞察破局——先随机旋转向量,它每个坐标就服从一个固定已知的分布,于是最优量化器能离线预制、在线零成本套用。逼近信息论下界只差 ~2.7×。

TL;DR

KV cache 压缩和向量检索都要在线 / data-oblivious 量化:向量边生成边压,没法先扫全量数据算统计再定格子1。TurboQuant 的招:先随机旋转——单位向量旋转后每个坐标服从固定已知的集中 Beta 分布(挤在 ±1/√d 附近)、且高维近独立2,于是离线就能设计好一个最优标量量化器、在线只旋转+套,不看数据3。各坐标各自最优复合起来 ≈ 向量最优,逼近信息论下界只差 ≈2.7×5。再补一手:MSE 最优量化器会让内积估计有偏,后接 1-bit QJL 残差纠成无偏6。KV 上 3.5-bit 无损、2.5-bit 微降,≥6× 省内存、H100 attention 快 8×11。

§ 01痛点：在线量化看不到数据,只能盲切格子

量化就是"把连续值塞进有限几个格子"。切格子要切在数据密集处才省 bit——离线场景可以先扫一遍全量数据、看清分布再切。但很多场景是在线 / data-oblivious 的1:KV cache12 的 key/value 向量边生成边存、向量检索的索引也想"建索引近零成本",都没机会先看全量数据统计。不知道数据往哪偏,只能保守地均匀切,浪费 bit、失真大。

§ 02招一：随机旋转,把分布从"未知"变"已知"

核心不是压得更狠,是把未知变已知2。几何事实:把任意向量乘一个随机正交矩阵(旋一下,长度不变),坐标就被搅匀成各向同性——原来能量可能全堆在某几个坐标(不知道堆哪),旋转后每个坐标统计上都长一个样、服从同一个算得出来的分布。分布已知,就能离线把最优标量量化器设计好;在线只需旋转 + 套这个预制器3,零数据依赖。

数字例子 · 旋转后坐标全挤到 ±1/√d

单位向量(长度 1)随机旋转后,每个坐标平方 x_i² 服从 Beta(1/2,(d−1)/2),期望 E[x_i²]=1/d,所以典型幅度 ≈ 1/√d——维度越高,所有坐标越挤在这层薄壳里:

d=4:    每坐标 ~ ±1/√4  = ±0.5     例:旋后 [0.5,−0.5,0.5,−0.5]，长度 √(4·0.25)=1 ✓
d=100:  每坐标 ~ ±1/√100= ±0.1     每个都在 0.1 上下,同一个窄分布
d=1000: 每坐标 ~ ±0.0316           越高维越集中

✓ 旋转前坐标可能一个 0.98、其余≈0(能量堆一处,量化器不知道堆哪);旋转后每个坐标都是同一个窄分布,`E[x_i²]=1/d` 是拿到手就能算的常数、不看任何数据——一个离线预制的量化器对所有坐标、所有输入都好使。

为什么坐标平方服从 Beta,均值为什么正好 1/d

先说 Beta 是什么:它专门描述"一个 0~1 之间的占比"。两个形状旋钮 Beta(a,b),均值 = a/(a+b);a 小、b 大时,占比挤在 0 附近。代入 Beta(1/2,(d−1)/2):均值 = (1/2)/(d/2) = 1/d——期望 1/d 就这么直接算出来。

为什么坐标平方是个"占比":随机旋转单位向量 = 在单位球面上均匀撒一个点。而均匀撒点的标准造法是"每个坐标各抽一个高斯数,再把整个向量缩放到长度 1",于是 x_i² = g_i² / (所有 g² 之和)——就是第 i 个坐标占总能量的份额。总能量固定是 1(单位向量),所以每个 x_i² 必然落在 0~1、是个占比 → 天然是 Beta。参数里的 1/2 对应分子"这一个坐标"、(d−1)/2 对应"其余 d−1 个坐标"。

均值 1/d 还有个更快的对称论证:d 个坐标地位完全对称,它们的平方加起来恒等于 1(单位向量),d 份平均分 → 每份 1/d,根本不用 Beta 也能看出来。

d=4,抽到 g=[2,1,0,1]：总能量 = 2²+1²+0²+1² = 6
x_i² = g_i²/6 → [0.667, 0.167, 0, 0.167]，加起来 = 1 ✓（这一次坐标1占大头）
换很多组随机 g 再平均 → 每个坐标平方都收敛到 1/4 = 0.25 → 幅度 √0.25 = 0.5 = 1/√4 ✓

§ 03高维近独立 → 每坐标各自最优 ≈ 向量最优

旋转还带来第二个礼物:高维下不同坐标近独立(几乎不相关)。所以不用去解"整个向量怎么量化最优"这个难题,只要每个坐标各自用它那个一维最优标量量化器(bin 量化的最优版9),复合起来就 ≈ 整个向量的最优4。TurboQuant 证明:这套逼近信息论失真下界(任何量化器的理论极限),只差一个小常数 ≈2.7×5——所有 bit 宽、所有维度都成立。

§ 04招二：MSE 最优会让内积有偏,QJL 残差纠回无偏

量化有两种"好":MSE 好(压得像原样)和内积无偏(拿压完的向量算点积,平均不偏)。麻烦是:MSE 最优的量化器,内积估计往往系统性偏小——量化误差跟向量结构相关,不是随机噪声,是每次都往同方向偏,加再多样本也纠不掉6。检索 / attention 靠内积排序,系统偏差会歪掉排名。

TurboQuant 两阶段各管一段:阶段 1 用 MSE 量化器压主体(低失真);阶段 2 对残差 r=x−x̂ 做 1-bit QJL(符号随机投影,内积无偏)7,把那点系统偏差换成能被平均掉的方差。

数字例子 · 有偏 vs 无偏,差在能不能纠回

真内积 ⟨x,q⟩ = 1.00
只用 MSE 量化器(有偏)：每次都 ≈ 0.90 → 平均还是 0.90,系统偏小 0.10,样本再多也停 ✗
加 QJL 残差(无偏)：0.90 + {单次乱跳、期望 0.10} → 多次平均 → 1.00 ✓

✓ 关键区别:偏差(bias)是加样本纠不掉的系统错;方差(variance)是加样本能平均掉的随机抖8。MSE 量化器给的是前者(致命),QJL 残差把它换成后者(可收敛)。最终估计既借 MSE 的低失真、又拿回内积的无偏。

QJL 到底怎么做的 · 只存符号(1 bit)+ 缩放常数 = 无偏内积

上面说 QJL 是"无偏"的,但它到底怎么算出来的?省法出奇地狠。先给一堆随机方向 r(高斯或 ±1 向量),每个方向就是一道随机的"分界线"。存向量 x 时只记它落在分界线哪一侧——一个符号 b=sign(⟨r,x⟩),1 bit,全部信息就这么点。

估内积 ⟨x,q⟩ 时:把 query 也投到同一个 r 上得 ⟨r,q⟩(query 是现来现用、不压,留全精度),乘上存下来的符号 b,再乘一个只跟维度有关的固定缩放常数 c,对多个随机方向取平均:

存 x:   每个方向只存 1 bit    b_i = sign(⟨r_i, x⟩)
估内积: c · (1/m) · Σ b_i · ⟨r_i, q⟩   对 m 个随机方向平均
        →  E[估计] = ⟨x, q⟩     无偏(c 是与维度有关的常数)

为什么"只留符号"还能把内积估准?把 query 拆成两部分看:跟 x 同方向的分量,每个方向的符号都一致地把它拾起来累加(不抵消);跟 x 垂直的分量,方向随机、正负相消平均掉。于是单个方向噪声极大(符号只有 ±1、信息稀薄),但期望正好落在真内积上——多加几个方向一平均就往真值收,每方向只花 1 bit,还不引入任何系统偏差。这就是它能拿来纠 MSE 残差偏差的底气:补进去的是无偏的东西。

数字例子 · 符号项平均出真内积

真内积 ⟨x,q⟩ = 0.80
单个随机方向的符号项(乘缩放常数后)噪声极大:
  方向1: +2.1   方向2: −0.5   方向3: +1.3   方向4: +0.3   … 单次乱跳,但无系统偏
取 4 个平均: (2.1 − 0.5 + 1.3 + 0.3) / 4 = 0.80   ✓ 逼近真值,方向越多越准

✓ 对比一个有偏的估计器可能永远停在 0.72(系统偏小,加样本纠不回);QJL 无偏,所以能靠"多加方向"收敛到真值——这正是 TurboQuant 拿它做残差纠偏、既省 bit 又不留系统偏差的原因。

§ 05结果与落地

把随机旋转 + 每坐标最优标量量化 + MSE/QJL 两阶段拼起来,得到一个逼近理论最优、又完全 data-oblivious 的在线量化器。落到 KV cache 和向量检索10:

指标TurboQuant
KV 量化质量3.5 bit/通道无损,2.5 bit/通道微降
KV 内存≥6× 省
attention 速度H100 上 快 8×
离下界≈2.7×(所有 bit 宽/维度)
向量检索recall 超现有 product quantization,建索引成本近零
放进 wiki 这条线

TurboQuant 把几条线拧一起:随机旋转(把未知分布变已知,data-oblivious 的真杠杆)+ 最优标量量化(每坐标各自最优)+ QJL 残差(治 MSE 最优→内积有偏)。落地 KV cache 压缩与 向量检索,attention 加速接 FlashAttention 那条线。一句话:随机旋转不是压得更狠,是把"我不知道数据长啥样"变成"我精确知道每个坐标的分布"——于是最优量化器离线预制、在线零成本套用。

引用 / 术语表

  1. 在线 / data-oblivious 量化 · 向量边来边压,拿不到全量数据统计再定量化格子(KV cache、检索索引都是)。对比离线可先扫数据看清分布。
  2. 随机旋转 → 分布已知 · 乘随机正交矩阵旋转(长度不变),单位向量每坐标平方服从 Beta(1/2,(d−1)/2)、挤在 ±1/√d 附近,分布固定已知且高维近独立。
  3. 离线预制 · 在线套用 · 分布已知 → 离线设计好最优标量量化器;在线只旋转+套,零数据依赖。旋转的价值=把未知变已知,不是压更狠。
  4. 高维近独立 → 每坐标各自最优 · 旋转后不同坐标近独立,不必解"整向量最优",每坐标用一维最优标量量化(bin 量化最优版)复合即可。
  5. ≈2.7× 信息论下界 · 论文给出任何量化器的失真理论极限,TurboQuant 在所有 bit 宽/维度下只差一个小常数(≈2.7×)。
  6. MSE 最优 → 内积有偏 · 重建最像的量化器,内积估计系统性偏(通常偏小),量化误差与向量相关 → 加样本纠不掉。检索/attention 排序会被歪。
  7. QJL 残差纠偏 · 阶段1 MSE 压主体,阶段2 对残差 r=x−x̂ 做 1-bit 符号随机投影(内积无偏),把系统偏差换成可平均掉的方差 → 低 MSE 且内积无偏。
  8. 偏差 vs 方差 · 偏差=系统性错、加样本纠不掉;方差=随机抖、加样本能平均掉。两阶段的本质是把致命的偏差换成可收敛的方差。
  9. 最优标量量化(bin) · 一维上把格子切在概率密度高处使失真最小(Lloyd-Max 那类)。旋转后坐标分布已知,这个能离线一次算好。
  10. 向量检索 / product quantization · 把高维向量压成离散码做快速召回;TurboQuant recall 超现有 PQ(见 FAISS),且建索引近零成本。
  11. 落地数字 · KV 3.5-bit 无损 / 2.5-bit 微降;≥6× 省 KV 内存;H100 attention 快 8×。
  12. KV cache · LLM 推理缓存每层 key/value 避免重算;越长上下文越吃显存,是量化压缩的主战场,天然 online。
