SOLARIS.

Georgy Savva · Oscar Michel · Daohan Lu · Suppakit Waiwitlikhit · Timothy Meehan · Dhairya Mishra · Srivats Poddar · Jack Lu · Saining Xie

一个人按下 W，只改自己画面还不够。队友也必须在同一时刻看到你移动。Solaris 研究的就是：怎样让两路生成视频像来自同一个世界。

TL;DR

给视频 latent01 增加玩家维，让两位玩家的视觉 token 进入共享 self-attention；再从双向单人、双向多人、因果多人训练到 Checkpointed Self Forcing，最终根据两套键鼠动作同步续写两路画面。

SECTION 00先看完整系统：部署时只有一个因果生成器

训练时有高质量双向 teacher 和因果 student；推理时保留 student。四阶段不是四套互不相关的模型，而是一条能力迁移路线。

先纠正一个误解：Solaris 不是维护方块坐标、天气和背包的 Minecraft 引擎。它维护的是一段联合视频 latent；“同一个世界”来自模型学到的跨视角统计关系。

SECTION 01多人难点：同一件事必须同时出现在两台相机里

玩家 A 放一块火把时，A 要看到放置动画和物品栏减一；玩家 B 若站在旁边，也要在正确位置看到火把。若只分别运行两个单人模型，两边很容易各编各的。

Frame concat 基线把两路画面沿通道拼接。它能看到两路像素，却没有 player ID 与专门的跨玩家信息交换。论文中它的 Movement VLM 分数更高，但会在 no-op 时凭空制造动作，且五项 FID 都更差。

SECTION 02先造数据：画面和动作必须共享同一时钟

SolarisEngine 把一个逻辑玩家拆成 controller bot 与 camera bot。前者适合自动化操作，后者用官方 Java 客户端渲染真实第一人称画面；服务器插件和时间戳再把二者对齐。

数据覆盖 building、combat、movement、mining；多数 episode 为 128–512 帧，即 6.4–25.6 秒。Superflat 与 Normal 世界约各半。完整动作包括走、跳、冲刺、yaw/pitch、攻击、使用、放置、挖掘和物品栏切换，但不含打开背包与原始鼠标轨迹。

SECTION 05四阶段训练：先会玩，再学多人，再学流式

因果 student 的窗口为 6 个 latent frame；冻结 3D VAE 每个 latent 对应 4 个真实帧，所以最大滚动 KV cache07 上下文是 24 帧。前三阶段学习率均为 1e-4；Self Forcing generator / critic 分别为 3e-6 与 3e-7。

SECTION 06Self Forcing 解决的不是画质公式，而是训练—推理错位

Self Forcing 让 student 在训练时就用自生成历史向前滚动，再由 teacher / distribution matching 纠正整段输出。这让输入更像部署，却带来巨大的反向传播显存：滑窗不断重叠，每个窗口都保存一份计算图。

SECTION 09结论边界：像一个世界，还没有永久的世界状态

01 · TWO PLAYERS

张量形式可扩到更多玩家，但实验固定 P=2；更多玩家的质量和显存没有证据。

02 · NO PERSISTENT MEMORY

玩家离开彼此视野后，共享上下文会消失，两条轨迹开始分叉。

03 · TWO INITIAL FRAMES

世界由两张初始帧指定，没有永久方块表、坐标表或背包状态。

04 · SYNTHETIC DATA

数据全来自自动 Minecraft bot，与真实人类动作和视觉分布有差距。

05 · PARTIAL METRICS

FID 不懂动作，VLM 会误判；没有真实玩家可玩性盲测。

06 · SPEED UNREPORTED

模型可因果滚动，但论文没有给端到端 FPS、硬件与交互延迟表。

准确定位：Solaris 证明了多位玩家的画面与动作可以作为联合视频状态训练，并用 Checkpointed Self Forcing 把长 rollout 的训练显存从 O(LtLs) 降到 O(Lt)；它还没有替代显式多人游戏引擎。

术语落点

  1. latent：VAE 压缩后的连续特征图。模型在这里去噪，再由 VAE 解码回像素。
  2. DiT：Diffusion Transformer，用 Transformer 处理带噪视觉 token 的生成主干。
  3. bidirectional：同一段中的 token 可以互相读取，适合整段去噪，不适合在线逐帧生成。
  4. causal：当前位置只能读取过去，保证推理时不会偷看未来。
  5. Diffusion Forcing：训练时给不同时间位置施加不同噪声，让扩散模型学会自回归状态组合。
  6. Self Forcing：训练时用模型自己的 rollout 当历史，缩小 teacher-forcing 与部署输入的差距。
  7. KV cache：保存历史 token 已计算的 Key / Value，生成下一帧时避免重算全部过去。
  8. FID：比较真实与生成图像特征分布的距离，越低越好，但不直接判断动作是否正确。
  9. VLM as judge：让视觉语言模型回答可验证问题；能测语义任务，也继承裁判模型的误差。
  10. stop-gradient：前向仍使用该数值，反向不沿生成它的旧路径传梯度；Solaris 会在第二遍重算新的可微路径。
