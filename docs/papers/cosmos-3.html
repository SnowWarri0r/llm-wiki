FIELD NOTES · 世界模型 / Physical AI

理解、生成
行动 · 一个模型

NVIDIA · Cosmos 3 Technical Report · 2026-06 · GTC Taipei

以前要拼 VLM + 视频生成 + 世界模拟 + 动作模型四个模型。Cosmos 3 用 MoT 把它们装进一个，统一处理语言/图像/视频/音频/动作五模态。

TL;DR

给 Physical AI（机器人/自驾）做基础模型，要同时会"理解世界"和"生成/模拟世界、产出动作"。以前这是四个割裂模型（VLM 看懂 + 视频生成 + 世界模型模拟 + VLA 出动作）。Cosmos 3 用 Mixture-of-Transformers 把它们合一：每个 transformer 层带两套权重——reasoner 塔处理自回归（AR）token（理解，因果注意力），generator 塔处理扩散 token（生成，双向注意力）；两塔都从 Qwen3-VL 初始化，在同一次联合注意力里相遇对齐。靠重排 token 就能切换语言/文生图/文生视频+音频/图生视频/视频 transfer/动作等模式，一套权重 subsume 了一堆专用模型。三档 Edge 4B / Nano 16B / Super 64B，被评为最强开源文生图 + 图生视频（Artificial Analysis）和最强策略模型（RoboArena）。

§ 01以前要拼一串模型，现在一个

让家用机器人"晚饭后收拾餐桌"，按老范式它得缝合一串模型：VLM 找碗+规划、VLA/世界-动作模型10出动作、世界模型模拟评估未来会怎样。四个割裂模型拼在一起，既笨重又难一致。Cosmos 3 的主张：理解和生成本就耦合——要决定动作就得先想象后果（生成），判断后果好不好（理解）——所以该用一个模型把它们统一。

§ 02MoT：每层两套权重，一塔推理、一塔生成

核心是 Mixture-of-Transformers。一个 transformer 层同时干"理解"（语言/推理，要因果注意力）和"生成"（高保真视频，要双向注意力）会打架——两种工作模式对权重要求不同。MoT 干脆给每种模式配一套完整权重（"塔"）：AR token 走 reasoner 塔，扩散 token 走 generator 塔4。两塔都从 Qwen3-VL 初始化，继承语言+视觉底子再学生成。

§ 03联合注意力：推理保因果，生成看全部

两塔权重虽分开，却在同一次注意力里相遇5，这是统一的关键。规则两条：AR token 只因果看 AR（保住像 LLM 那样的文本生成）；扩散 token 用全双向注意力看 [AR; 扩散] 的全部（能看到文字 prompt + 所有条件和生成 token，保持时空一致）。而且 AR 永不被扩散 token 更新——条件通路的因果完整性不破。

§ 04一套权重，重排 token 就换任务

所有模式都是同一条序列 [AR 前缀 | 扩散目标] 的不同摆法6：只给 AR、不激活扩散 → 就是个 VLM；AR 放文字、扩散放噪声图 → 文生图；扩散里先放干净的条件帧再放噪声目标 → 图生视频；条件换成边缘/深度控制图 → 视频 transfer；目标换成动作 token → 策略。一套权重 subsume 了一堆专用模型（这就是 omnimodal9）。

§ 05结果与规模

后训练版同时拿下三个"最强开源/最强"：Artificial Analysis 评的最强开源文生图 + 图生视频，RoboArena 评的最强策略模型8——理解、生成、动作三头都打到 SOTA。三档规模 Edge 4B / Nano 16B / Super 64B，注意总参≈2× 稠密底座（因为每层两塔），但每个 token 只过一塔，激活量≈单塔。

放进 wiki 这条线

Cosmos 3 是 世界基础模型 这条线给 Physical AI 做的集大成。它的发动机 MoT 是 unified-transformer（一条序列、混合注意力同时理解+生成）的升级——把权重拆成两塔，跟 MoE（FFN 专家、为省算力）是两回事。generator 塔就是 DiT + 扩散一族；两塔都从 Qwen3-VL 初始化。跟 Lumine 同属具身，但 Lumine 纯 AR 学策略、Cosmos 3 把生成交给扩散塔。一句话：理解要因果、生成要双向，硬塞一套权重会打架——那就给每种工作模式一套完整权重(塔)，但共用一次联合注意力让它们对齐；于是一个模型把 VLM、视频生成、世界模拟、动作策略全收了。

引用 / 术语表

  1. Mixture-of-Transformers · MoT · 每层两套权重（塔）：reasoner 塔处理 AR（理解，因果）、generator 塔处理扩散（生成，双向），token 按模态/任务路由，但共用一次联合注意力。
  2. Physical AI · 具身智能 · 在真实物理世界感知-推理-行动的 AI（机器人/自驾），动作后果不可逆，理解与生成天生耦合，主要靠仿真/合成数据训练。
  3. 世界基础模型 · WFM · 学"世界接下来会怎么变"的大模型，给当前+动作能模拟未来，当 Physical AI 的训练场/合成数据工厂/策略评估器。
  4. reasoner 塔 / generator 塔 · MoT 一层里的两套独立权重；前者跑自回归理解、后者跑扩散生成；都从 Qwen3-VL 初始化。
  5. dual-stream 联合注意力 · AR token 只因果看 AR（保文本生成）；扩散 token 双向看 [AR;扩散] 全部（看得到 prompt + 所有 token）；AR 永不被扩散更新。
  6. 生成模式 / token 排列 · 同一序列 [AR 前缀 | 扩散目标] 的不同摆法：纯 AR=VLM、文字+噪声图=文生图、加干净条件帧=图生视频、控制图=transfer、动作 token=策略。
  7. Edge / Nano / Super · 三档：4B（底座 2B）/ 16B（底座 8B，Qwen3-VL 8B）/ 64B（底座 32B，Qwen3-VL 32B）。总参≈2×底座（两塔），激活≈单塔。
  8. 三头 SOTA · 最强开源文生图 + 图生视频（Artificial Analysis）、最强策略模型（RoboArena）——理解/生成/动作三头同时打到顶。
  9. omnimodal · 五模态（语言/图像/视频/音频/动作）统一处理+生成；subsume 了 VLM、视频生成、世界模拟、世界-动作模型。
  10. VLA / 世界-动作模型 · 出动作的模型（Vision-Language-Action / World-Action Model）；Cosmos 3 把动作当一种可生成模态，跟视频平起平坐。
