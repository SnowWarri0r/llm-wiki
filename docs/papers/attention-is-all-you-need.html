Attention
Is All You Need.

2017 年那篇砍掉 recurrence、用纯 attention 把 NLP 推上 scaling 轨道的奠基论文 —— 一篇带交互式 self-attention 演示与术语注解的中文解读。

原文 / Vaswani et al. · Google Brain arxiv:1706.03762 → NeurIPS 2017 讲解 / Claude

现在所有大模型（GPT / Claude / Gemini / Qwen / Llama / fish-speech / TML interaction model）都是这一篇的后代。它的核心主张是一句几乎不像论文标题的话："注意力，是你所需要的全部" —— 抛弃 2014-2017 年间 NLP 标配的 RNN/LSTM，用纯 attention 堆出来的网络，又快、又能学长程依赖、又能 scale。这页的目的是把 self-attention 这个核心机制讲到能在脑子里跑一遍，配上 5 张可播放的图。

§ 01 · the bottleneckRNN 的天花板

2017 年之前，NLP 处理序列的标准是 RNN1（包括 LSTM2 / GRU）。机器翻译、语言建模、问答、命名实体识别 —— 全是 RNN 的天下。但它有三个让人受不了的痛点：

  1. 串行瓶颈：RNN 必须 t=0 → t=1 → t=2 ... 一个 timestep 接一个，**前一步算完才能算下一步**。GPU 有几千个核能并行，但 RNN 在 sequence 维度上根本用不上。
  2. 长程依赖衰减：序列里第 50 个 token 想看第 1 个 token 的信息，得让信号在 RNN 隐状态里流 49 步。中间每一步乘一个矩阵 + 过激活函数 → 梯度沿路衰减或爆炸（LSTM 缓解但没根治）。
  3. 难 scale：参数量上去之后训练不稳定 —— 时间维度越长、梯度路径越深、问题越严重。

2014-2017 有过几次尝试：Bahdanau attention（给 RNN decoder 加 attention 看 encoder 输出，仍是 RNN 主体）、ConvS2S / ByteNet（用空洞卷积把长程依赖压到 O(log N) 跳）。但都没真正解决"序列建模必须串行"的诅咒。

这篇论文的标题就是答案：把 RNN 完全拿掉，纯 attention + FFN 堆出来的网络反而 work better。

§ 03 · the trickSelf-Attention 机制

这是整篇论文最核心、最值得脑子里跑一遍的部分。一句话：

序列里每个 token，用 Q（"我在问什么"）、K（"我能提供什么 key"）、V（"我能提供什么 value"）三个向量 —— 跟所有其他 token 的 K 算相似度，softmax 当权重，加权求和 V → 这个 token 的输出。

为什么除 √dk

当 dk 大（论文 dk=64），Q·KT 的 dot product 数值很容易爆到 +50 / -50 量级。softmax 进入饱和区，绝大部分位置 weight ≈ 0、一两个 ≈ 1 —— 梯度直接消失。除 √dk 把 scores 的方差控回 1 量级，softmax 在线性区，梯度健康。

这条不是"装饰"，是论文最重要的工程细节之一。后来所有 transformer 变种都保留 √dk 缩放。

§ 04 · the expansionMulti-Head

一个 self-attention 头只能学一种关系模式。但一个句子里同时有句法、语义、共指、词性各种关系。论文的招数：跑 h 次（论文 h=8），每次用不同的 WQ / WK / WV 投影 —— 每个 head 学不同的模式。

关键数字：

  • 输入 dmodel = 512
  • 每个 head 的 dk = dv = 512 / 8 = 64
  • 所以多头 不是免费扩张能力，是"同样参数预算下，把 512 维 attention 切成 8 个 64 维 attention"
  • concat 后再过一个线性 WO 把 8 × 64 = 512 拼回 dmodel
for head in [0..7]:
  Q_h = x @ W_Q[h]    # 各自的投影
  K_h = x @ W_K[h]
  V_h = x @ W_V[h]
  head_out[h] = softmax(Q_h @ K_h.T / sqrt(64)) @ V_h

output = concat(head_out) @ W_O

§ 05 · the hackPositional Encoding

Self-attention 有个让人尴尬的性质：对位置不敏感。把序列里 K 和 V 的行打乱，Q·KT 算出来的"行内分布"完全一样 —— 模型分不清 "dog bites man" 和 "man bites dog"。

论文的招数：给每个位置加一个唯一的"位置向量"，跟 token embedding 直接相加。位置向量用 sin/cos 不同频率组成：

PE(pos, 2i)   = sin(pos / 10000^(2i / d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i / d_model))

input = embedding(token) + PE(pos)
📐 公式拆解

pos = token 在序列里的位置（0, 1, 2, ...）
i = 维度下标（每个位置向量有 d_model 维，i 跑 0 到 d_model/2-1）
10000^(2i/d_model) = 频率控制 · i 小时频率高（变化快），i 大时频率低（变化慢）
→ 偶数维放 sin，奇数维放 cos，跨多个频率的组合形成"位置指纹"。低维高频快变（区分相近位置），高维低频慢变（区分远距位置）。
看不懂 sin/cos/exp 这些符号？查 math-symbols。

现代变种

论文的 sinusoidal PE 是 OG 方案，但外推效果一般。现代 LLM 大多换了：

  • Learned PE（GPT-2）·把 PE 当 embedding 训。简单，但不能外推到训练时没见过的长度
  • RoPE（GPT-J / LLaMA / Qwen）·把位置编码进 Q/K 的旋转里，相对位置天然就有。当下主流
  • ALiBi（BLOOM）·在 attention scores 上加位置距离的线性偏置。可外推
  • NoPE·完全不要 PE，让大模型自己学。小模型不行，大模型 OK

§ 06 · the architecture完整架构

把上面所有组件拼成完整网络。论文 base 版：Encoder 6 层 + Decoder 6 层。每一层都是"multi-head attention + position-wise FFN + residual + LayerNorm"的标准块。

06.1每个子层里发生什么

"sub-layer" 这词在论文里反复出现。一个 sub-layer 的完整流程：

def sublayer(x, fn):
    # fn = multi_head_attention 或 ffn
    return LayerNorm(x + Dropout(fn(x)))

Residual（x + fn(x)）让梯度有"直通车" —— 不管 fn 学得多差，x 都能直接传到下一层。LayerNorm 把激活值标准化，让训练数值稳定。Dropout 正则化。三件套是深层 transformer 能训得动的关键。

06.2FFN 是什么

每个 attention 子层后面跟一个 position-wise FFN —— 看起来不起眼，实际占模型一半以上参数：

FFN(x) = max(0, x · W_1 + b_1) · W_2 + b_2
       # 两层 MLP, dim 是 d_model=512 → d_ff=2048 → d_model=512
       # 中间膨胀到 4 倍, 再压回来

"Position-wise" 是说每个位置独立过这个 FFN（不像 attention 是位置间交互）。可以理解为 attention 负责"位置间信息聚合"，FFN 负责"每个位置自己的 non-linear 变换"。两者轮流交替。

§ 07 · the legacy改变了什么

2017 这篇出来时，作者大概没想到它会成为接下来 10 年所有大模型的母体。粗略时间线：

年事件用了 Transformer 哪部分
2017Attention Is All You NeedEncoder + Decoder (本文)
2018GPT-1（OpenAI）纯 Decoder + causal mask · "decoder-only" 范式诞生
2018BERT（Google）纯 Encoder + Masked LM 训练
2019GPT-2Decoder-only 放大到 1.5B
2020GPT-3175B 参数 · in-context learning emerge
2020Vision Transformer (ViT)Transformer 跨界 CV，把图切 patch 当 token
2022ChatGPTDecoder-only + RLHF · 大众破圈
2023+Claude / Gemini / Qwen / Llama全是 decoder-only transformer 变种

Decoder-only 路线最后赢了 —— GPT 们证明：只要数据和参数够大，不需要 encoder + cross-attention 也能学好。fish-speech S2 Pro 的 Dual-AR 也是 decoder-only；TML interaction model 也是。整个现代 LLM 圈基本上是这一篇的延长线。

原论文 BLEU 数字（次要但记下）

WMT 2014 En→De：BLEU 28.4（比当时 SOTA 高 2 分以上）；En→Fr：BLEU 41.8。训练 8 GPU 3.5 天。这些数字 2017 年很惊人但今天已经没人关心 —— transformer 真正的价值是它开辟了 scaling 的可能，不是这两个 BLEU。

§ 08 · what to keep个人解读

  1. 最核心的认知升级：从"序列建模 = 沿时间步串行处理 + 记忆状态"切到"序列建模 = 一次性看全所有位置 + 学位置间的注意力"。这是一个完全不同的归纳偏置（inductive bias）—— 它放弃了 RNN 的"时间结构先验"，换来了"任意两个位置都能 O(1) 交互"的灵活性。
  2. "O(N²) 但可并行"赢"O(N) 但串行"。论文出来时大家担心 N² 内存/计算炸，但 GPU 上的 N² 是同时算的 N² 元素，wall-clock 上反而比 RNN 的 O(N) 串行快。这个"用并行算力换 wall-clock"的范式在后来 GPU 集群时代越来越值得。
  3. 结构简洁 + 易 scale 是底层优势。Transformer 的整个网络就两种子层（attention + FFN），加上 residual / LayerNorm 这些通用组件。没有任何"特殊先验" —— 所以容易加大、容易并行训练、容易跨模态（语言 → 视觉 → 音频 → 视频 → 多模态都用同一个骨架）。
  4. "Decoder-only" 是事后才明朗的选择。论文本来设计的是 encoder-decoder（适合翻译这种 seq2seq 任务）。GPT 系列证明：用 decoder + causal mask + 海量 next-token prediction 训练，模型既能"理解"（encoder 的活）又能"生成"（decoder 的活）。结构上更简单，且 scaling 性更好。
  5. Positional encoding 是工程妥协，不是终极方案。论文用 sinusoidal PE 是没办法 —— 给 attention 补一个位置感。后续 RoPE / ALiBi 等改进证明这个位置注入方式可以更优雅。任何"attention 的位置信息怎么搞"是后续 10 年还在被改的方向。

§ ▼ · vocabulary术语词典

正文里带波浪底的词都对应这里一条。按出现顺序编号：

01 RNN  (recurrent neural network)

"循环神经网络"。每个时间步 t 用 h_t = f(h_{t-1}, x_t) 更新一个隐状态，序列处理是串行的（必须 t 算完才能 t+1）。1980s 提出，但训练长序列梯度爆炸/消失。LSTM / GRU 缓解了这条，但仍然串行。被 Transformer 全面取代。

02 LSTM  (long short-term memory)

"长短期记忆网络"。1997 年 Hochreiter & Schmidhuber 提出，给 RNN 加了"门"（input / forget / output gate）控制信息流，缓解长程梯度衰减。2013-2017 NLP 的主力模型。Transformer 出来后逐渐淡出，但在某些序列任务（语音识别、时间序列预测）还活着。

03 softmax

归一化函数：softmax(x)_i = exp(x_i) / Σ_j exp(x_j)。把任意实数向量映射到正且和为 1 的概率分布。Self-attention 的 weights 就是 softmax 后的。"饱和区"指 softmax 在某些极端 input 下输出几乎都是 0 或 1，梯度消失 —— 所以要 √d_k 缩放。

04 attention head pruning

"头剪枝"。Multi-head attention 训完后，研究表明不是所有 head 都重要 —— 剪掉一半 head 性能基本不掉。这意味着多头里有冗余。Pruning 是模型压缩的常见手段。代表论文：Michel et al. 2019 "Are Sixteen Heads Really Better Than One?"

05 causal mask

"因果掩码"。Decoder 自注意力时只允许每个位置看自己和之前的位置（不偷看未来），用一个上三角全 −∞ 的矩阵加到 attention scores 上，softmax 后这些位置 weight = 0。Decoder-only LLM（GPT 系列）整个模型都用 causal mask 训练，做下一 token 预测。

06 FFN  (feed-forward network)

两层 MLP：linear → ReLU → linear。在 Transformer 里每个 attention 子层后面跟一个 position-wise FFN，每个位置独立过 FFN（不像 attention 是位置间交互）。中间层维度通常是输入的 4 倍（论文 d_model=512 → d_ff=2048）。占 transformer 一半以上参数。

07 LayerNorm

"层归一化"。对每个样本的特征维度做标准化（mean=0, var=1），然后乘可学的 γ + β 还原。让深层网络训练数值稳定（输入分布不会随层数漂移到爆掉）。BatchNorm 是另一种归一化（在 batch 维度），适合 CNN；Transformer 用 LayerNorm 因为它对序列长度和 batch size 不敏感。

08 residual connection

"残差连接"。output = x + sublayer(x)。来自 ResNet (He et al. 2015)，让深层网络的梯度可以"直通"反传，避免衰减。Transformer 每个子层都套着 residual，所以 6 / 12 / 24 / 96 层都能稳定训。

09 embedding

"嵌入"。把离散 token（词表索引 0..V）映射到 d_model 维连续向量的查找表。论文 d_model=512。embedding 矩阵跟整个模型联合训练。也是后来 LLM 把"world model"压在里面的主要载体 —— 词表里相近的概念在 embedding 空间里距离近。

10 BLEU

"双语评估替补"。机器翻译质量指标（0-100），算生成译文跟参考译文的 n-gram 重叠率。2002 年提出，至今仍是翻译 benchmark 标准。论文 WMT 2014 En→De 28.4 BLEU 当时 SOTA。现代 LLM 在翻译任务上 BLEU 通常 35+。

11 in-context learning

"上下文学习"。GPT-3 论文提出的能力：模型在 prompt 里给几个示例（few-shot），就能在没有梯度更新的情况下做新任务。这是 transformer + scaling 的 emergent property，RNN/LSTM 几乎没有这个。Transformer 的 attention 机制天然支持"从 prompt 里抽取模式"，可能是它的根因。

12 decoder-only

"纯解码器"。GPT 系列开始的范式：去掉原 Transformer 的 encoder + cross-attention，只保留 decoder + causal mask，用 next-token prediction 训整个网络。证明这种简化结构同样能学好（甚至更好），且更容易 scale。当下所有大语言模型都是 decoder-only 变种。
