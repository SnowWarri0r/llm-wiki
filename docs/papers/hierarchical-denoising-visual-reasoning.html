HDR 不能修改已经播放的 RGB 帧。“可修改”只发生在最终细节尚未提交的层级 latent01 中。

流式自回归扩散02 的联合概率可写成：

p(\mathbf z\mid c)=\prod_{i=1}^{N}p(z_i\mid z_{<i},c)
zᵢ第 i 个干净时间 latent。z<i已经生成并写入历史的所有位置。c条件，例如第一帧。∏把每一步条件概率相乘，得到整段序列的概率。

`z₂` 一旦包含错误转弯，后面的概率都显式以它为条件。HDR 要解决的就是这个“太早提交”。

04 · LAYER-WISE FLOW MATCHING每一层都在学同一个“该往哪走”

\begin{aligned}v^t_{\ell,i}&=(1-t)v^0_{\ell,i}+t\epsilon\\u^t_{\ell,i}&=\frac{dv^t_{\ell,i}}{dt}=\epsilon-v^0_{\ell,i}\end{aligned}
ℓ, i第 ℓ 层、第 i 个节点。v⁰干净层级 latent。ε同形状标准高斯噪声。t0 到 1 的线性路径位置；0 靠数据，1 靠噪声。vᵗ训练时送给模型的带噪节点。uᵗ正确速度；直接对插值式求导得到。

先把“目标速度减去模型预测”记成单节点误差 Δ，再对节点和层求平均：

\begin{aligned}\Delta_{\ell,i}&=(\epsilon-v^0_{\ell,i})\\&\quad-\nu_\theta(v^t_{\ell,i},t,h_{\ell,i},c)\\\mathcal L_{\mathrm{HDR}}&=\sum_{\ell=1}^{L}\lambda_\ell\frac{1}{N_\ell}\\&\quad\sum_{i=1}^{N_\ell}\mathbb E_{t,\epsilon}\!\left[\|\Delta_{\ell,i}\|_2^2\right]\end{aligned}
Δℓ,i当前节点的目标速度与预测速度之差。νθVideo DiT 的速度预测，θ 是全部可训练参数。hℓ,iSHAP 为当前节点选出的层级上下文。c第一帧等条件。E对很多随机 t、ε 取平均。λℓ层损失权重；论文没有公开数值。‖·‖²₂每维误差平方后相加。

所以训练目标没有从“视频推理”突然变成另一套公式：每个节点仍做标准 flow matching04，差别是输入上下文包含父层规划，损失覆盖整棵树。

06 · SHAP第 6 个细节点具体能看见谁

SHAP（Sparse Hierarchical Attention Pattern）06 不让一棵树变成全连接图。对 `(ℓ,i)`，它只选同层前一个、父节点、父节点左右邻居；每层第一个节点直接读取第一帧。

\begin{aligned}\mathcal A(\ell,i)={}&\mathbf1[i>1]\{(\ell,i-1)\}\\&\cup\mathbf1[i=1]\{x_{ref}\}\\&\cup\mathbf1[\ell>1]\{\pi,\operatorname{left}(\pi)\}\\&\cup\mathbf1[\ell>1]\{\operatorname{right}(\pi)\}\end{aligned}
1[条件]条件成立就加入后面的集合，否则跳过。(ℓ,i−1)同层前一个节点。xref干净第一帧；按式子直接接到每层首节点。π上一层父节点。left/right父节点的左右邻居；越界项省略。

节点先按层摊平成一维序列：`φ(ℓ,i)=i+Σ_{r<ℓ}Nᵣ`。在 `1/2/4/8` 小树中，L4-6 的序号是 `6+(1+2+4)=13`。父层已经先生成，同层只读前一个，因此 mask07 不会偷看未来。

生成完成的节点把 K/V 写进共享 KV cache08。SHAP 决定当前 query 从 cache 读哪几项；cache 负责不重算，稀疏 mask 负责少读取，二者不是一回事。

迷宫 progress 用生成路线与最短正确路线的最长公共子序列（LCS）比例；一笔画则是 `.5×覆盖率 + .3×合法转移率 + .2×不出界比例`。所以 `89.56 average progress`12 不能读成“89.56% 视频完全成功”，完全成功对应 `60.29 success`13。

训练配方Wan2.2-5B-TI2V 初始化；六类任务各 3,000 条；125 帧、224²；100k steps；8 GPU 约 48h；AdamW，LR 2e−6，betas (0,.999)，WD .01；bf16、gradient checkpointing、hybrid FSDP17；总 batch 8；推理 CFG16=3。

最终判断：HDR 改进的是提交决策的时机。普通流式模型一边规划一边锁死；双向扩散一直保留修改权但每轮重算整段；HDR 用稀疏时间树先确定大方向，到了最细层才形成连续输出。

这套方法成立需要三件事同时存在：层级表示提供中间规划空间，粗层少去噪避免过早确定，SHAP 控制新增节点的通信成本。只增加层数，不做不确定性分配和稀疏连接，很可能只是增加 token 与 prefill。

13 · GLOSSARY正文术语，点击后直接查看

  1. Latent：VAE 把像素视频压缩后得到的连续特征。它仍包含画面和时间信息，但不是可直接播放的 RGB 帧。
  2. 流式自回归扩散：从左到右生成视频 latent，每个位置内部仍用扩散或 flow 采样；已生成历史进入 KV cache。
  3. Video VAE：把视频压到较小的时空 latent 网格，并从 latent 解码回帧。HDR 的 32 个细时间节点不等于 32 个 RGB 帧。
  4. Flow Matching：构造数据与噪声之间的路径，让网络回归路径速度；推理时沿预测速度从噪声走回数据。
  5. Entropy-matched schedule：作者按每层有效时间支撑的幂分配去噪步数；这里是设计近似，并非直接测量熵后求得。
  6. SHAP：Sparse Hierarchical Attention Pattern；只保留同层前一步、父节点、相邻父节点和层首参考帧的结构化 mask。
  7. Attention Mask：一个 0/1 可见性矩阵。1 表示当前 query 可以读取该 key，0 表示屏蔽，不参与 softmax。
  8. KV Cache：保存过去 token 在每层算出的 Key/Value，避免后续生成时重新投影；不自动减少当前 query 要读取的历史长度。
  9. Action Token：把连续机器人动作块投影成 Transformer 可处理的向量；它预测的是可执行控制量，不是文字动作描述。
  10. Cross-Attention：query 来自一组 token、key/value 来自另一组。HDR-WAM 用它让视频去噪读取已经允许的动作组。
  11. RoboDojo：包含 42 个模拟机器人交互任务的 benchmark，按泛化、精度、长程、记忆和开放任务分组。
  12. Average Progress：任务专用的部分完成分，不是生成质量分；各任务用路线重合、合法转移、终局距离等规则计算。
  13. Success：严格的完整任务通过率。必须同时满足过程合法和最终目标，单条样本只记 0 或 1。
  14. OOD：Out-of-Distribution；评测布局不同于常见训练模式，例如真实迷宫中斜放或堆叠积木。
  15. Prefill：流式输出开始前一次性建立上下文与 KV cache 的阶段。HDR 的 16.19s 发生在这里。
  16. Classifier-Free Guidance：用条件预测与去条件预测的差值放大条件影响；HDR 主实验 scale 为 3.0。
  17. FSDP：把参数、梯度和优化器状态切到多张 GPU；hybrid full-sharding 在节点内外采用分层切分/复制策略。
  18. Inverse Dynamics Model：输入相邻视觉状态，预测什么机器人动作能造成这次状态变化；把生成视频转成控制命令。
