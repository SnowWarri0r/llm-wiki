DiffusionOPD ·
先各请专家，再沿学生的路蒸馏。

ali-vilab · DiffusionOPD: A Unified Perspective of On-Policy Distillation in Diffusion Models · arXiv 2605.15055 · 2026

给扩散模型做多奖励对齐（美学 + 文字 OCR + 指令遵循）时，多个奖励一起用 RL 会互相打架、级联训会遗忘。DiffusionOPD 换个路子：先各训一个单任务专家老师，再沿学生自己走出来的轨迹把老师们蒸进一个学生——而且扩散去噪是高斯马尔可夫链，每步 KL 有闭式解，直接塌成均值的 MSE。

TL;DR

把 RL（PPO/GRPO）对齐扩散，多奖励一起搞会出事：联合 RL 任务互相打架（cross-task interference）、简单任务抢戏；级联 RL 灾难性遗忘。DiffusionOPD 用 On-Policy Distillation1：①Stage 1 各任务先用 RL 训一个专家老师（单任务探索解耦）；②Stage 2 沿学生自己 rollout 的轨迹，把老师们蒸进一个统一学生2。关键数学甜头：扩散去噪 = 高斯马尔可夫链3（来自反向 SDE），两个同协方差高斯的 KL 有闭式解4，每步 on-policy KL 直接塌成学生 vs 老师去噪均值的 MSE：L = Σⱼ ‖μ_s − μ_t‖²/(2σⱼ²)——干净、没 RL 方差。结果：平均归一化 0.929 vs 级联 0.851 / 多任务 RL 0.763，更快更省，美学/OCR/GenEval 全 SOTA。

§ 01痛点：多奖励对齐扩散，怎么搞都别扭

给文生图扩散模型"调好"——更美、能正确渲染文字、更听指令——本质是 RLHF/RL 对齐：拿奖励信号把模型往人喜欢的方向推。一个奖励还好，同时要好几个就难了：

  • 联合多奖励 RL（几个奖励加一起一起优化）：不同任务给的优化方向互相矛盾（cross-task interference），还有难易不均——简单任务的梯度抢戏，难任务学不动。
  • 级联 RL（一个任务训完再训下一个）：要多阶段、精调度，且灾难性遗忘——学后面的把前面的本事忘了。

DiffusionOPD 的思路：把"单任务探索"和"多任务整合"拆开——探索交给各自的专家老师，整合用一次干净的蒸馏。

§ 02On-Policy Distillation：在学生自己走到的状态上学老师

先讲核心招式 OPD，它从 LLM 那边借来。普通（off-policy）蒸馏是老师生成数据、学生照抄固定数据集——问题是学生一旦走到自己没练过的状态，不会救（训练/推理分布不一致）。

On-policy 蒸馏反过来：学生自己 rollout 生成整条轨迹，然后在学生实际走到的那些状态上，去匹配老师的下一步。这样监督是每步稠密的（对比 RL 只有稀疏的终局奖励），而且专治学生自己的薄弱状态。

§ 03两阶段：先各请专家，再蒸进一个学生

把上面的招式套成一个干净的两阶段流程：

Stage 1：每个任务各用 RL 训一个【专家老师】   # 单任务探索，互不干扰
Stage 2：沿【学生自己 rollout】的轨迹            # on-policy
         把 M 个老师一起蒸进【一个统一学生】     # 整合，不打架

§ 04数学甜头：扩散去噪的 KL 塌成 MSE

为什么这套在扩散上特别顺？因为扩散去噪本身就是一条高斯马尔可夫链——来自反向 SDE，每一步把当前样本按一个高斯分布挪到下一步，而且这个高斯的协方差只由噪声 schedule 定、跟策略无关。

于是 on-policy 蒸馏要算的"学生每步 vs 老师每步"的 KL 散度，落到两个同协方差高斯上，有干净的闭式解——直接退化成两个均值差的平方（MSE）：

KL( N(μ_s, σ²I) ‖ N(μ_t, σ²I) ) = ‖μ_s − μ_t‖² / (2σ²)
                                    └ 同协方差 → 只剩均值之差的平方 ┘

把它逐时间步求和，就是 DiffusionOPD 的训练目标（SDE 版带 1/σ，ODE 版更干脆）：

L_OPD = E[ Σⱼ ‖μ_s(x_tⱼ; θ) − μ_t(x_tⱼ)‖² / (2σⱼ²) ]     # SDE 版
L_OPD-ODE = E[ Σⱼ (1/2)‖μ_s(x_tⱼ; θ) − μ_t(x_tⱼ)‖² ]      # ODE 版

注意 x_tⱼ 是学生自己 rollout 走到的状态（on-policy），在那上面只让学生的一步去噪均值去贴老师的。没有 RL 的采样方差、没有奖励建模——一个稳的 MSE。

# 训练循环（简化）
for 每个任务 m:
    用当前学生 rollout 出轨迹 {x_tⱼ}
    L_m = Σⱼ ‖μ_s − μ_t^(m)‖² / (2σⱼ²)   # 贴第 m 个老师
    L_total += L_m
一次反传更新 θ                              # 跨任务累加 → 均衡整合

DiffusionOPD 把好几条线拧到一起：RL/PPO/GRPO 对齐（但用蒸馏绕开多任务 RL 的打架与遗忘）+ on-policy 蒸馏（学生自己走、稠密每步监督，从 LLM 借来）+ 扩散=反向 SDE 的高斯马尔可夫链 + 同协方差高斯 KL=MSE。一句话：探索交给各任务的专家老师，整合用"沿学生自己轨迹、贴老师去噪均值"的一个干净 MSE。

引用 / 术语表

  1. On-Policy Distillation（OPD） · 学生自己 rollout 生成轨迹，在学生实际走到的状态上匹配老师的下一步。每步稠密监督；对比 off-policy（照抄老师固定数据，学生偏了不会救）和 RL（稀疏终局奖励）。
  2. 两阶段解耦 · Stage 1 各任务用 RL 训专家老师（探索解耦）；Stage 2 沿学生轨迹把老师们蒸进一个学生（整合不打架、不遗忘）。
  3. 扩散 = 高斯马尔可夫链 · 去噪过程来自反向 SDE，每步是一个高斯转移，协方差只由 noise schedule 定、与策略无关 → 才能有闭式 KL。
  4. 同协方差 KL = MSE · KL(N(μ_s,σ²I)‖N(μ_t,σ²I)) = ‖μ_s−μ_t‖²/(2σ²)。每步 on-policy KL 蒸馏塌成学生 vs 老师去噪均值的 MSE，无 RL 方差。
  5. cross-task interference / 遗忘 · 多奖励联合 RL 的两大病：任务优化方向矛盾、简单任务抢戏；级联 RL 则灾难性遗忘。OPD 用解耦+蒸馏避开。
  6. 结果 · 平均归一化 0.929 vs 级联 0.851 / 多任务 RL 0.763；美学/OCR/GenEval 全 SOTA，训练更快更省。
