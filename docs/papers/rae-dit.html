RAE-DiT ·
在"看懂图"的空间里作画。

NYU 等 · Scaling Text-to-Image Diffusion Transformers with Representation Autoencoders · arXiv 2601.16208 · 2026

主流 latent 扩散先单训一个 VAE 把图压成没语义的低维"压缩坐标"。RAE 改成:拿一个冻结的预训练表征编码器(SigLIP-2 / DINOv2)当 latent 空间、只训一个轻量解码器,让扩散直接在高维语义空间里跑。收敛快 4×、抗过拟合(VAE 64 epoch 崩、RAE 稳到 256)、越 scale 越赢。

TL;DR

RAE(Representation Autoencoder)把扩散的 latent 从 VAE 压缩 latent 换成冻结表征编码器的高维语义 latent1:编码器用 frozen SigLIP-2 / DINOv2(256 token×1152 维),只训一个轻量解码器还原像素2;扩散(DiT + flow matching)直接在这个"已看懂图"的空间里跑。一个**必须**的技术:维度相关噪声调度(高维 latent 要按 α=√(m/n) 移时间步,去掉 GenEval 49.6→23.6)3。规模放大后配方反而简化:宽 diffusion head、噪声增强解码都变得没必要4。结果:比 FLUX VAE 基线收敛快 4.0×;VAE 微调 64 epoch 过拟合崩、RAE 稳到 256;0.5B→9.8B 全程 RAE>VAE5。

§ 01痛点：VAE 的 latent 是"压缩坐标"，没语义

主流做法:先单训一个 VAE,把图压成低维 latent(常 <100 维),再让扩散在里头跑。问题是 VAE 是为"忠实还原"训的——它的 latent 轴没有语义含义(纯压缩编码),还在压缩时丢掉细节。扩散在这种"压缩坐标系"里学,慢,而且高质量数据微调时容易过拟合。

RAE 的主张:别再单训压缩器了。已经有一堆"看懂图"的模型(DINOv2、SigLIP)——直接拿它们当 latent 空间。

§ 02RAE：冻结语义编码器 + 只训解码器

Representation Autoencoder:编码器 = 冻结的预训练表征模型(SigLIP-2 So400M / DINOv2 / WebSSL,就是自监督/对比学出来"看懂图"的那类),输出 256 token × 1152 维的高维语义 latent;只训一个轻量解码器把这堆语义 embedding 还原成像素(L1 + LPIPS + Gram + 对抗损失,7300 万图)。扩散直接在这个语义空间里跑。

类比:VAE 像把图存成 zip 压缩包(紧凑但坐标没意义、有损);RAE 像直接在一个"已经理解图像"的概念坐标系里作画——"附近 = 语义相似",扩散学起来顺,再用解码器把概念渲染回像素。

§ 03唯一不能省的：维度相关噪声调度

RAE 的 latent 是语义特征:N=256 token × d=1152 维,有效维度 m = N×d = 294,912,比 VAE latent 大得多(参考维度 n=4096)。高维空间有个反直觉的事实:同一个名义时间步,维度越高、相对信噪比越高——全局/语义结构能在更多坐标上"平均"出来、噪声被摊薄。直接套 VAE 那套 schedule,在高维 latent 上就长期欠噪(模型见到的目标太干净),训练信号全错位。

为什么偏偏是 √(m/n)——先讲动机,再验算

很容易把因果讲反:不是"先拍脑袋选了 α=√(m/n),碰巧逐维 SNR 缩了 m/n 倍"。真实的链条是反的——先定下"逐维 SNR 必须缩 m/n 倍"这个目标,再反解出 α 只能是 √(m/n)。所以 n/m 是推出来的目标、α 是为命中目标定义出来的,两个都不是凑的。分两步看。

① 目标里的 n/m 哪来的:高维"白捡" m/n 倍有效信噪比。 想象干净信号的某个全局/语义量是同一个值 s,摊在每个维度上,每维各挂一份独立噪声、方差 σ²。要把 s 估出来,最优做法是跨维度平均——维度越多,噪声被平均掉得越狠:

n 维：平均 n 个噪声维 → 噪声方差 σ²/n → 全局 SNR = s²/(σ²/n) = n·s²/σ²
m 维：平均 m 个噪声维 → 噪声方差 σ²/m → 全局 SNR = m·s²/σ²

m 维 / n 维 = (m·s²/σ²)/(n·s²/σ²) = m/n   ← 同样的每维噪声,换到高维白捡 m/n 倍有效 SNR

也就是说:啥都没做,光是把 latent 从 n 维换到 m 维,去噪任务就免费变简单了 m/n 倍。要让高维的去噪难度跟 n 维参考一样(否则模型在高维偷懒、长期欠噪、学不到东西),就得主动把逐维 SNR 砍掉 m/n 倍抵消。目标就此钉死:SNR_每维(t_m) = (n/m)·SNR_每维(t_n)。

② 为什么开根号、不是直接拿 m/n 当 α:因为 SNR 是平方量。 flow matching 的加噪是 x_t=(1−t)x+tε,逐维信噪比 SNR(t)=((1−t)/t)²(信号系数²÷噪声系数²)。想让这个平方量变成原来的 n/m,里头的幅度比 (1−t)/t 只能变 √(n/m)(平方之后才是 n/m)。把"幅度比要缩的倍数"记成 1/α,于是 α=√(m/n)——√ 纯粹来自"SNR 是幅度的平方"。

动机讲完了。下面是验算(方向跟动机相反:假设 α=√(m/n) 和移位公式,代进去确认逐维 SNR 真的精确缩了 m/n 倍。两个方向互为充要):

记 D = 1+(α−1)·t_n ，移位公式：t_m = α·t_n / D

① 算分子 1−t_m：
   1 − t_m = 1 − α·t_n/D = (D − α·t_n) / D
   而 D − α·t_n = 1 + (α−1)·t_n − α·t_n
                = 1 + α·t_n − t_n − α·t_n = 1 − t_n      （α·t_n 抵消）
   ⟹ 1 − t_m = (1 − t_n) / D

② 分母就是  t_m = α·t_n / D

③ 两者相除（D 上下抵消）：
   (1−t_m)/t_m = [(1−t_n)/D] ÷ [α·t_n/D] = (1−t_n)/(α·t_n) = (1/α)·(1−t_n)/t_n

④ 代进 SNR(t) = ((1−t)/t)²（平方一下）：
   SNR(t_m) = ((1−t_m)/t_m)² = [(1/α)·(1−t_n)/t_n]² = (1/α²)·SNR(t_n)

⑤ 因为 α = √(m/n) ⟹ α² = m/n ⟹ 1/α² = n/m：
   ⟹ SNR(t_m) = (n/m)·SNR(t_n)

验算闭环:step ⑤ 落回 SNR(t_m)=(n/m)·SNR(t_n),跟 ① 钉下的目标一字不差。所以这个 shift 把逐维信噪比在每个 t 上都精确缩小 m/n 倍,刚好抵消高维白捡的那 m/n 倍,让"有效去噪难度"跨维度对齐。

m = N×d = 256×1152 = 294,912    n = 4096    α = √(m/n) = √72 ≈ 8.49
移位 t_m = α·t_n /(1+(α−1)·t_n)：
  t_n=0.1 → t_m=0.485    逐维 SNR  81.0  → 1.125     (÷72)
  t_n=0.5 → t_m=0.895    逐维 SNR  1.00  → 0.0139    (÷72)
  t_n=0.8 → t_m=0.971    逐维 SNR  0.063 → 0.00087   (÷72)
每个 t 都精确 ÷72 = ÷(m/n) ✓

✓ 自检:名义"中点" t=0.5(逐维 SNR=1)在这个 294,912 维 latent 里,其实要用到 t≈0.9 那么多的噪声(SNR≈0.014)才对得上"等效难度"。这是论文里唯一不能省的技术:加了 GenEval 49.6 / DPG-Bench 76.8,去掉直接掉到 23.6 / 54.8(腰斩)。公式本身出自 Zheng et al. 的 logit-SNR 时间移位,RAE 把移位量 α 锚到了维度比。

§ 04越 scale 越简单 + 抗过拟合

有意思的是:模型放大后,一堆小模型需要的补丁都变得没必要了——

设计小模型放大后(>2.4B)
宽 diffusion head0.5B 时 +11.2 GenEval没必要(主干容量够了)
噪声增强解码训练早期有用没必要(只是早期正则)
维度相关噪声调度必须必须(唯一不能省)

另一个大优势是抗过拟合:高质量数据微调时,VAE 模型 64 epoch 后就崩(训练 loss 塌向 0、生成质量掉),而 RAE 稳到 256 epoch 还在涨——高维 + 语义结构提供了隐式正则。

放进 wiki 这条线

RAE 要替的是 VAE latent;编码器就是 DINOv2 / SigLIP 那类表征模型;主干仍是 DiT + flow matching。它跟 PiD 是一对:PiD 换解码端(latent→像素用扩散补)、RAE 换编码端(用语义编码器当 latent 空间),都在挑战"VAE 是不是扩散的最佳地基"。一句话:别单训没语义的压缩器,把扩散搬进一个已经"看懂图"的高维语义空间——快、稳、越大越赢,代价是噪声调度按维度重算。

引用 / 术语表

  3. 维度相关噪声调度 · 高维 latent 加同样噪声相对腐蚀更小,须按 α=√(m/n) 把时间步往"更多噪声"挪,补回信噪比。唯一不能省:去掉 GenEval 49.6→23.6。
  4. scale 后配方简化 · 宽 diffusion head(小模型 +11.2,>2.4B 没用)、噪声增强解码(只早期正则)放大后都没必要。模型越大,recipe 越简单。
