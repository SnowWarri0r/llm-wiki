CNN ·
一把小尺子，滑过整张图。

卷积神经网络 · LeNet(1989) → ResNet → 被 ViT 填平护城河 · foundational

把图拉平喂全连接，参数上千万、还学不会"猫挪个位置还是猫"。CNN 的全部聪明就一句：用一个小核滑遍全图、一路共用同一套权重 —— 把"局部性 + 平移不变"这两条视觉先验，直接焊进网络结构。

TL;DR

CNN 的原子操作是卷积1：一个小核2滑过整张图，每停一处做逐元素相乘求和得一个数，滑遍 → 一张 feature map4。两条先验焊进结构——局部连接（核只看小窗口）+ 权重共享5（一套核滑遍全图 → 参数暴降、天生平移不变6）。一层放很多核抓不同模式，层层堆叠把感受野8从"看个边"撑到"看整只猫"，配 pooling7 下采样高效扩张。这三条就是 CNN 的归纳偏置9：数据少时是宝、大数据时代被 ViT10 用纯注意力填平，但卷积仍是最省最快的视觉底座。

§ 01卷积 · 核滑过每个位置

原子操作就一句：一个小核，盖住一块 patch，逐元素相乘再求和 = 一个输出数；右移一格再来，走完一行换下一行。下图核滑遍 5×5 输入的 9 个位置，每停一处填一个输出格，最后拼成 3×3 的 feature map——核的权重一路不变。

几个旋钮：stride（步长）一次滑几格，2 就跳着走、输出更小；padding边上补 0 控尺寸。输出边长 ≈ (W − K + 2P) / S + 1。详见 convolution。

§ 02为什么不直接全连接 · 局部 + 共享

把 224×224 的图拉平喂全连接（MLP），第一层每个神经元连 ~5 万个输入，几百个神经元就上千万参数；更糟的是位置写死——猫在左上角学的权重，猫挪到右下角白学。卷积用两刀砍掉这个浪费。

两刀：局部连接（核只看小窗口，吃"局部性"先验）+ 权重共享（同一套核滑遍全图，吃"平移不变"先验）。参数从上千万降到"几个核 × 9 个数"，而且猫在画面哪都用同一套核认得出。详见 convolution、inductive-bias。

§ 03多核 → 层级特征

一个核只抓一种局部模式（比如竖边）。所以一层放很多核（64、128…），各抓一种，输出一摞 feature map。下一层的核在这摞上再滑，把简单模式组合成复杂的——边缘 → 纹理 → 部件 → 物体，越深越抽象。

初学最容易揉成一团的三件事，分清就顺了：核（检测一种模式，一层内放很多个 → 多通道，横轴）、层（一组核扫一遍，层层往下堆 → 深度，纵轴）、pooling（夹在层之间、对整摞 feature map 一起下采样，不绑某个核）。看一条 shape 流：

§ 04pooling + 感受野 · 看得越来越宽

单个 3×3 卷积只看原图一小块，怎么认出需要全局的"猫"？靠层层叠加把视野撑大——这个"一个输出能看到原图多大一片"就是感受野8。光堆卷积，感受野线性长；插一刀下采样（pooling7 或 stride=2 卷积）把分辨率砍半，感受野就指数级往上窜。

Max pooling 在小窗口取最大（"有没有这特征"，丢"精确在哪"），降计算、扩感受野、还带一点平移鲁棒。现代骨干常用 stride=2 卷积替掉它，但末端的 global average pooling 几乎人人还用——把每张 feature map 压成一个数，替掉老式 CNN 末端那坨全连接。详见 pooling、receptive-field。

§ 05谱系 · LeNet → ResNet → 被 ViT 填平

CNN 的发展史就是"怎么把卷积网络做得更深"，直到 Transformer 从另一条路杀进视觉。

放进 wiki 这条线

CNN 和 Transformer 是"先验 vs 通用"的两端。卷积把"局部性+平移不变"焊进结构 → 数据少时高效、省参、收敛快；注意力（self-attention）几乎不带空间先验、一层即全局 → 要更多数据喂，但天花板更高。ResNet 是 CNN 做深的终点也是 Transformer sublayer 模板的起点；ViT 用 patch-embedding 把图切成 token、用纯注意力填平了 CNN 的视觉护城河。这正是 inductive-bias 那条主线：归纳偏置在小数据是宝、大数据是包袱。

引用 / 术语表

  1. 卷积 · convolution · 小核滑过整张图，每处把覆盖的 patch 跟核逐元素相乘求和得一个数。CNN 的原子操作。详见 convolution。
  2. 核 · kernel / filter · 一个小权重矩阵（如 3×3 = 9 个可学的数），代表一种要检测的局部模式。一层放很多个，各抓一种。
  3. stride / padding · stride = 核一次滑几格（2 = 下采样）；padding = 边上补 0 控制输出尺寸。输出边长 ≈ (W−K+2P)/S+1。
  4. feature map · 一个核滑遍全图得到的输出图，记录"这种模式在每个位置的响应强度"。一层多核 → 多张 feature map 摞成输出通道。
  5. 权重共享 · weight sharing · 同一套核滑遍全图、所有位置共用，相比全连接参数暴降，也是平移不变的来源。
  6. 平移不变 · translation invariance · 目标在画面哪个位置都用同一套核检测 → 猫挪个地方还是猫。卷积焊进结构的视觉先验之一。
  7. 池化 · pooling · 小窗口取 max/avg 把 feature map 缩小，留"有没有特征"丢"精确在哪"。降计算、扩感受野、带一点平移鲁棒。详见 pooling。
  8. 感受野 · receptive field · 深层一个输出最终由原图多大一块决定。堆卷积线性长，下采样让它指数级扩张。详见 receptive-field。
  9. 归纳偏置 · inductive bias · 架构里焊进的先验。CNN 的是局部性+平移不变：小数据时是宝、大数据时成束缚。详见 inductive-bias。
  10. ViT · 护城河被填平 · ViT 把图切成 patch token、用纯 self-attention（一层即全局），在大数据上打平/超过 CNN，证明视觉不一定要卷积。详见 vit。
