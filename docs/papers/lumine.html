看着屏幕，把原神主线打通。

Tan et al. (ByteDance Seed 等) · An Open Recipe for Building Generalist Agents in 3D Open Worlds · arXiv 2511.08892 · 2025

一个 VLM 当大脑，直接吃游戏画面像素、吐键盘鼠标操作，端到端打通《原神》5 小时蒙德主线，零样本迁移到《鸣潮》《崩铁》。三个关键招：动作当文本 token 生成、看一眼预测多步(action chunking)、该想才想(hybrid thinking)。全程纯模仿人类录像，零 RL。

TL;DR

Lumine 用 Qwen2-VL-7B 当大脑，直接从屏幕像素 → 键鼠操作，端到端跑完《原神》5 小时主线、零样本迁移其他游戏。三招：① 动作即文本1——没有专门动作头，键鼠操作当普通 token 吐出来（鼠标位移 + 33ms 按键片）；② action chunking2——5Hz 看、30Hz 动，一次推理预测 6 个动作块；③ hybrid thinking3——只在关键转折点触发 <|thought|> 推理，平时反射式出招。训练是 2424 小时人类录像的纯模仿学习（预训练→指令跟随→推理三段），零 RL4。记忆 = context 当记忆(滑窗 20 帧)。靠 W8A8 量化+流式输出+投机解码做到 25.3× 延迟优化5 实时跑。

§ 01痛点：开放世界，纯 RL 练不动

3D 开放世界游戏（原神这种）对 agent 是地狱级：任务长达数小时、要探索 + 战斗 + 解谜 + 跟 NPC 对话、还得实时操作键鼠。你正在玩的 Mario RL 那套（强化学习，自己试错 + reward 信号）在这儿基本失灵——reward 太稀疏（5 小时才通关一次，中间没人给分）、状态空间太大，纯试错几乎不可能练出来。

Lumine 的选择：不试错，抄人类作业。站在一个预训练 VLM 的语言 + 视觉先验上，用模仿学习看人怎么玩就学怎么玩，再加"该想才想"的推理。下面三招是它能实时跑长任务的关键。

§ 02第一招：动作就是"文本"，没有动作头

最反直觉的设计：它不给 VLM 加任何专门的"动作输出层"，而是让模型把键鼠操作当普通文字 token 直接吐出来——和写句子一模一样的自回归预测。

一段动作长这样："92 0 0 ; Shift W ; Shift W ; F W ; F"——鼠标右转 92 单位、按 Shift+W 冲刺、按 F+W 开宝箱。鼠标位移是 [-1000,1000] 的数，按键是分号隔开的 33ms 时间片。把整个游戏操作翻译成一种"语言"，VLM 就能用它最擅长的"预测下一个 token"来产出，视觉语言模型的先验白白捡来用。

§ 03第二招：看慢动快 · action chunking

难点：它每 200ms 才看一帧（5Hz），但游戏要 30Hz（每 33ms 一个动作）才流畅。慢感知怎么喂出快操作？答案——一次推理预测未来 6 个动作块（6×33ms≈200ms），看一眼就把接下来 200ms 的连续操作一口气规划好。

§ 04第三招：该想才想 · hybrid thinking

每步都长篇内心独白太慢。Lumine 学会了只在"关键转折点"才触发推理（用 <|thought_start|>…<|thought_end|> 包起来）——环境突变、旧计划失效、冒出新目标时才停下来想；平时条件反射直接出动作。要是连续 100 步没想过，就强制进入思考模式自救（防卡死）。

§ 05训练：2424 小时人类录像，纯模仿三阶段

和你 Mario RL 最大的分野——完全没有强化学习、没有 reward，就是行为克隆（看人怎么玩，学着复现）：

阶段数据学什么
预训练1731h 原始录像行为克隆，学基本操作"肌肉记忆"
指令跟随200h（自动打标）把操作绑到自然语言指令上
推理15h（人工标内心独白）学长程自主决策（hybrid thinking）

第二阶段省人工的巧办法：先训一个小 Qwen2-VL-2B 分类器（165h 人工标的 38 类活动）去给海量录像自动打活动标签，再用 GPT-4.1 在转折点生成多样的自然语言指令。这样 200h 指令数据不用全靠人标。

跟 RL 的路线对照

维度Mario RL（你在玩的）Lumine
学法强化学习：试错 + reward纯模仿：抄人类录像，无 reward
大脑小 CNN 策略网络7B VLM（语言+视觉先验）
动作离散按键，网络直接输出当文本 token 生成，可 chunking
推理不"想"，纯反射关键点才"想"（hybrid thinking）

一句话：RL 是"从零试错练手感"，Lumine 是"站在 VLM 先验上抄人类作业 + 该想才想"。开放世界 reward 太稀疏，纯 RL 几乎练不出 5 小时主线，所以走模仿。

§ 06真·实时：25.3× 延迟优化

要 200ms 出一帧的 6 个动作块，工程上压榨到极致：流式输出（每个分号块一好就执行，把 200ms 死线松成每块 33ms）+ 投机解码（利用固定分隔符免草稿模型）+ KV-cache 复用（StreamingLLM + attention sink）+ W8A8 量化（SmoothQuant，权重激活都 INT8）+ 4 卡张量并行 + CUDA graph 融合。合起来 25.3× 延迟降低：不带推理时一帧 ~114ms，稳稳塞进 200ms 窗口。

放进 wiki 这条线

Lumine 把几块拧在一起：VLM（Qwen2-VL 当大脑）+ action chunking（看慢动快）+ 模仿学习（替代 RL）+ context 当记忆（同 MemGPT 思路）+ W8A8 量化/KV-cache 做实时。一句话：把游戏操作翻译成"语言"，让一个 VLM 看图说"话"（话=键鼠），抄人类录像学会，该想才想。跟纯 RL 的 Mario 是两条路。

引用 / 术语表

  1. 动作即文本 · 不加专门动作头，键鼠操作编码成 token 串（鼠标位移 [-1000,1000] + 分号隔开的 33ms 按键片），VLM 用自回归"预测下一 token"产出。复用语言/视觉先验，免从零训控制网络。
  2. Action Chunking · 一次推理预测未来多步动作（Lumine: 6 块 × 33ms）。用 5Hz 的慢感知喂出 30Hz 的快控制；也让动作更连贯、少抖。机器人 VLA（ACT/RT 系）通用招。
  3. Hybrid Thinking · 该想才想 · 每步决定"直接出动作"还是先生成 <|thought|> 推理。只在关键转折点想（省算力），卡 100 步没想则强制思考自救。
  4. 纯模仿学习 · 三阶段 · 2424h 人类原神录像：预训练 1731h 行为克隆 → 指令跟随 200h（小分类器+GPT-4.1 自动打标）→ 推理 15h（人工标内心独白）。零 RL。
  5. 25.3× 延迟优化 · 流式输出 + 投机解码 + KV-cache 复用(StreamingLLM) + W8A8 量化(SmoothQuant) + 4 卡张量并行 + CUDA graph。无推理一帧 ~114ms。
