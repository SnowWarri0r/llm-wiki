§ 01 / The representation problem连续坐标好生成，离散连接关系才是硬骨头

一张三角 mesh 就存成两张表。顶点表每行是一个点的 xyz 坐标：第 0 行 (0.31,-0.42,0.08)、第 1 行 (0.55,0.10,-0.20)……面表每行三个整数，但这三个数不是坐标，是顶点表的行号——像三个"门牌号"。所以面 (17,42,91) 读作"把顶点表里第 17、42、91 号这三个点连成一个三角形"。

难点就在这两张表脾气相反：顶点坐标是连续数，加一点噪声、去一点噪声都平滑，扩散/flow 模型最擅长这个；可面表里的门牌号 42 改成 43 立刻指向了完全不同的另一个点，没有"改一半"的中间态——它是离散开关，不是连续量。传统路线要么先生成隐式表面再抽网格，要么把坐标和门牌号全切成离散 token 交给自回归模型一个一个吐。MeshFlow 换招：把"谁连谁"也改写成连续数，让模型全程只在连续空间里玩，最后一步才切成 0/1 的边。它对输入也不挑，不强制是封闭流形。

§ 02 / Continuous topology把“有没有边”变成两枚连续向量的匹配分数

顶点 i 除了坐标 v_i，还随身带两样东西：外法线 n_i（待会儿管朝向）和一枚边嵌入 h_i。边嵌入就是一串数字——把它想成模型给每个顶点发的一张"交友名片"，名片上写什么是学出来的，不是 xyz 坐标。要不要在 i、j 之间连一条边，只看两张名片合不合得来。

怎么算"合不合得来"？把 h_i 和 h_j 逐位相乘、再全部加起来——这一步叫内积，结果是一个数。数越大，说明两张名片在这个学出来的空间里越"同向、越投缘"，越该连边；过阈值 τ 就连，不过就不连：

h_i = (0.8, 0.5, −0.2) 两张"名片"（学出来的边嵌入） h_j = (0.9, 0.3, −0.1) 该连的一对： score(i,j) = 0.8·0.9 + 0.5·0.3 + (−0.2)·(−0.1) = 0.72 + 0.15 + 0.02 = 0.89 > τ=0.6 → 连边 ✓ h_k = (−0.6, 0.1, 0.7) 不该连的一对： score(i,k) = 0.8·(−0.6) + 0.5·0.1 + (−0.2)·0.7 = −0.48 + 0.05 − 0.14 = −0.57 < 0.6 → 不连 ✗

凭什么"投缘=该连"这条规矩成立？因为训练时的边对比损失就在逼着模型把真边那对名片的内积推高、把非边那对推低（见 §03 的五笔损失）。练完之后，模型已经把该连的顶点在名片空间里摆得彼此同向、不该连的摆得彼此背离——这个"名片空间"是模型自己雕出来的，不是几何位置。这就是为什么离散的"连/不连"能被一串连续数字表达出来。

代码里的小拐弯真实实现把上面的内积做了个"有正有负"的变体（承 SpaceMesh 血统，代码里叫 signed spacetime score：名片的一部分维度算加、一部分算减）。直觉完全一样——一个可学的相似度分数，过线连边；只是这个分数不是普通欧氏距离，所以判据是 score > τ（越大越像边），不是"距离越小越连"。这点在论文正文和代码里写反过，见 §10。

法线：同一个三角形有正反两面，谁来定？

连边只决定"哪三个点围成一个三角形"，但同一个三角形有正反两面——从外面看和从里面看，渲染出来的明暗完全相反。定正反的是顶点外法线 n_i：每个顶点自己预测一支"朝外的箭头"，指向物体表面的外侧。

给定三角形 (i,j,k)，拿它的两条边做叉乘（cross product）能得到一支垂直于这个三角形的箭头——它的"几何法线"。但叉乘方向取决于你按 i→j→k 还是 i→k→j 绕，两个顺序算出的箭头正好相反。到底哪个才朝外？拿它和三个顶点自己预测的外法线平均一比：同向说明绕对了；反向（内积为负）就把 j、k 调个位——绕序一反，叉乘的箭头就翻到另一面，正反面就摆正了。

三角形 a=(0,0,0) b=(1,0,0) c=(0,1,0) 边1 = b−a = (1,0,0) 边2 = c−a = (0,1,0) 几何法线 = 边1 × 边2 = (0,0, +1) ← 指向 +z 若三个顶点预测的外法线平均朝 −z： (0,0,+1)·(0,0,−1) = −1 < 0 → 绕反了 → 交换 b,c 交换后 边1 = c−a、边2 = b−a： 新几何法线 = (0,0, −1) ← 翻到 −z，和预测一致 ✓

整个过程一个坐标都不改，只动"三个顶点写下来的先后顺序"，形状纹丝不动，只是把可能翻过来的那些面正过来。

§ 03 / MeshVAETokenMerge 先缩短序列，再让短 query 回看全部顶点

逐个顶点跑 DiT 太贵，先用 MeshVAE 把顶点序列压短。先认几个字母：B=一批几个 mesh，N=一个 mesh 有几个顶点，D=每个顶点用多长的向量描述。所以 [B,N,D] 就是"B 个 mesh、每个 N 个顶点、每个顶点一条长 D 的向量"。编码端先把坐标、法线、邻居坐标做 Fourier 编码拼成这条长 D 的向量。

TokenMerge 干的事一句话：把"顶点很多"换成"每个顶点的向量更长"，总信息量不搬走。做法是把连续 4 个顶点首尾拼接成一条 4 倍长的向量，再用一层 Linear 压回原长：

假设 4 个顶点，每个用长度 2 的向量（D=2）： 顶点1 = [1, 2] 顶点2 = [3, 4] 顶点3 = [5, 6] 顶点4 = [7, 8] ① 拼接（token 轴 → 通道轴）：4 条向量首尾接成 1 条长 8 的向量 [1,2, 3,4, 5,6, 7,8] ← 4 个 token 变 1 个，一个数都没丢 ② Linear(8 → 2)：学一组权重，把这 8 个数搅拌压回长度 2 → [q₁, q₂] ← 这就是 1 枚"短 query"

关键全在第 ① 步是"拼接"不是"取平均"。取平均会把四个顶点糊成一团（信息真丢了）；拼接是把四份原料原封不动摆进一条长向量，再让线性层学着怎么搅拌——搅法是训练出来的，可以有取有舍，而不是粗暴对半分。所以序列短了 4 倍，信息还在通道里。

光靠这条压出来的短 query 还嫌信息不够，于是再用交叉注意力让每枚短 query 回头"问"全部 N 个原始顶点（原始顶点留着当 key/value）：谁需要哪个顶点的细节，就把它抓回来。所以"压短"不等于扔掉另外四分之三。解码端做反向的 TokenSplit，把 N/4 重新摊回 N。

§ 04 / Face recovery一堆候选边怎样变回三角 mesh

解码器输出顶点、法线、边嵌入和有效顶点 mask。先删掉 padding；再枚举边 (i,j) 的共同邻居 k，若 ij、jk、ki 三条边都在，就收下三角面；最后用法线校正绕序。论文还描述小边界环补洞，但公开 pipeline 默认 fill_holes=False。

§ 05 / Rectified flow从噪声往回走：每一步同时移动全部 latent

这一节讲压好的 latent 到底怎么"长"出来。先建直觉：想象一屋子乱飞的灰尘（纯噪声），你要把它整理成一座雕像（干净数据）。Flow 学的就是"每一粒灰、在每个时刻、该往哪个方向、多快地飘"——这个"方向 + 快慢"就叫速度场。把整段路切成很多小步，每步都按预测的速度挪一点，灰尘就整体流成雕像。

"整流（rectified）"是让这条路走直线：训练时干脆把干净 latent x₀ 和一团高斯噪声 ε 用一条直线连起来，直线上的速度自始至终恒定，就是 ε−x₀（终点减起点）。直线最好学、也最好走，采样时用很少几步就能到头。DiT 的活儿：看着当前这个带噪点、当前时刻 t 和点云条件 c，把该走的速度预测出来。

x(t) = (1−t)x₀ + tε 直线上 t 时刻的点（t=0 是数据，t=1 是噪声） u = ε − x₀ 目标速度；这条直线上从头到尾不变 L = ||vθ(x(t), t, c) − u||² 训练：让预测速度贴近目标速度

拿数字走一遍：x₀=[2,−1]、ε=[−2,3]、t=.25，代入得中间点 x(t)=[1,0]，目标速度 u=ε−x₀=[−4,4]。若 DiT 预测成 [−3.5,3.7]，两维平方误差相加就是 .5²+.3²=.34，训练就是把这个数往 0 压。采样时反过来：从纯噪声出发，每步按预测速度往回挪 Δt，走到 t=0 就是干净 latent，交给 VAE 解码成 mesh。

先澄清一个常搞混的点flow 本身就不是自回归。自回归（MeshGPT 那路）是把面切成 token、吐完第 1 个才能吐第 2 个，输出越长串行等待越久；而 flow 一步就把整串 latent 的速度一起算出来，天生并行。所以"并行生成"不是 MeshFlow 发明的技巧，而是选了 flow 这条路自带的——MeshFlow 真正的功劳在前几节：把 mesh 改写成 flow 吞得下的连续 latent，flow 才用得上。

§ 06 / Geometry conditioning32³ 体素不是拿来建模，而是统一位置口径

到这里 DiT 会"从噪声凭空长网格"了，但你总得告诉它长成什么形状。办法：扔给它一团点云（物体表面的一堆散点）当参照——像给雕塑师递一张参考照片，它照着这团点把网格生出来。这就是"几何条件"。

麻烦出在参照本身：训练时这团点取自美术师做的网格顶点（锐边、拐角上点特别密），推理时取自在表面"按面积均匀撒"的采样点。同一只兔子、两种撒点法，点的疏密完全不同。模型要是死记了"训练时顶点爱扎堆在哪"，一到推理换了撒法就认生。

解法是把精确坐标"打成粗格子"：像把 GPS 的精确经纬度粗化成"你在哪个街区"——只记街区、不记门牌号。把空间切成 32×32×32 的小立方格（体素），每个点只报"我落在哪一格"。这样两种撒点法都落进同一批格子，疏密差异被抹平。再用 3D RoPE 把这个"街区编号"写进注意力，让模型只关心"点大致在哪块空间"，不纠结"具体是哪个点、怎么采出来的"。代价：喂进去的几何被粗化了。

§ 07 / Full pipeline六个零件，拼成一条流水线

前面六节把零件一个个拆开了；这节把它们接回一条流水线。看整体最省事的办法是分两条道：推理是模型真正干的活——只给一团点云，从噪声端到端生成出网格；训练是这套参数怎么学出来的。

§ 09 / Experiments它赢在表示短和生成快，不是每一项误差都全场最低

Chamfer Distance 与 Hausdorff Distance都比较两组表面采样点；越小越好。CD 看平均最近距离，HD 只盯最坏点。它们看几何位置，却不知道边是否接对、面是否翻转。

消融：TokenMerge 的优势不只是“少几个 token”

边 F1同时检查乱连和漏连：precision 低说明多连了边，recall 低说明漏了边，两者缺一都会把 F1 拉低。

§ 10 / Limits还没解决的短板,和复现要当心的地方

复现坑 · 论文和代码对不上的三处不是挑字眼，这三处会真的改变结果：① 连边阈值主文写 d≤τ 连边，但边损失、算法和代码提交 55f56f6 都用 d>τ——因为代码里的 d 是"越大越像边"的带符号分数，不是普通距离。② DiT 规模主文 18 层 / 427M，补充材料 16 层 / 895M，两套冲突，gated 配置暂时裁不了。③ 补洞论文说修补小边界环，公开 pipeline 默认关闭。复现时把论文 v2 + 代码 55f56f6 + 模型配置一起记下来。

仍没解决的事情

只会三角面不生成美术拓扑常用的 quad，也没有 UV 与纹理。
指标看不全拓扑CD/HD 低不代表无洞、无翻面或好编辑。
私有训练分布60 万 artist meshes 无法由外部完整复建。
错误边会生错误面三条误边凑成三环，恢复器照样接受。
图像条件不是替代品DINOv3 图像可选，但发布流程仍需要点云提供几何位置。
全注意力仍贵并行采样解决串行延迟，没有消除大 token 数的注意力成本。
一句话带走MeshFlow 真正的新意不是“把 DiT 用到 3D”，而是先发明一种 DiT 吃得下的 mesh：坐标描述形状，边嵌入描述连接，法线负责朝向。表示问题一旦理顺，VAE 压缩和 Flow 并行生成才顺理成章。

术语 / 直接解释

  1. Topology / 拓扑 · 只关心顶点怎样连接成边和面；顶点坐标相同，连法不同，拓扑就不同。见连续 Mesh 连接表示。
  2. Autoregressive / 自回归 · 后一个 token 必须等前一个 token 生成完；输出越长，串行链越长。
  3. Implicit surface / 隐式表面 · 不直接存三角面，而用 SDF 或 occupancy 回答空间点在表面哪里；通常再用 Marching Cubes 抽 mesh。
  4. Edge embedding / 边嵌入 · 每个顶点携带的连续向量；两向量的连接分数过阈值就恢复一条离散边。
  5. Vertex normal / 顶点外法线 · 指向表面外侧的单位向量；MeshFlow 用它决定三角面的正反朝向。
  6. MeshVAE · 把完整 mesh 压成较短随机 latent，再从 latent 重建顶点、法线、边和 mask。KL 项让 latent 接近高斯。见KL-VAE。
  7. TokenMerge / TokenSplit · 在 token 轴与通道轴之间重排信息，先缩短序列，再反向摊开。见完整数值例。
  8. Cross-attention / 交叉注意力 · query 与另一组 token 的 key/value 对话；这里短 query 用它回看完整顶点输入。
  9. Mask / padding · 不同 mesh 顶点数不同，batch 要补到同长度；mask 标出哪些槽位是真顶点。
  10. Rectified Flow · 在数据与噪声之间取直线，训练网络预测沿线速度。见Flow Matching。
  11. DiT · 用 Transformer 处理带噪 latent 并预测去噪速度；每层包含 self-attention 与 FFN。见Diffusion Transformer。
  12. Voxel / 体素 · 三维空间里的“小立方格”；这里只用于把点坐标粗化，不用它重建表面。
  13. 3D RoPE · 分别把 x、y、z 位置编码成旋转相位，直接影响 attention 的相对位置关系。见体素化 3D RoPE。
  14. Edge F1 · 预测边的 precision 与 recall 的调和平均；既处罚乱连，也处罚漏连。
  15. CD / HD · CD 看双向平均最近距离，HD 看最坏最近距离；两者都不直接检查拓扑。见手算例。
  16. Watertight manifold · 没有开口、没有一条边被异常多张面共用的封闭表面；MeshFlow 的输入表示不强制满足它。
  17. 内积 / Inner product · 两条等长向量"逐位相乘再全部相加"得到的一个数；越大代表两向量越"同向"。这里用它当"两枚边嵌入合不合得来"的分数。
