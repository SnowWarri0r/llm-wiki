BERT
double-edged.

Devlin · Chang · Lee · Toutanova · NAACL 2019 best paper

Transformer 之后的第一个分叉：只要 encoder。用 masked LM 让一个模型一次性看到双向上下文做预训练 —— 然后给所有理解类任务 finetune。NLP 一夜之间换了套打法。

TL;DR

GPT-1 当年用 Transformer 的 decoder 部分做了 left-to-right 预训练。但理解任务 —— 情感、问答、指代 —— 都天生需要双向上下文。

核心诡计：随机 mask 15% 的 token，让模型从剩下所有位置（左右都行）还原。绕开了"双向看自己"的悖论。

这同时也立住了pretrain → finetune 范式 —— 一个 backbone，11 项 benchmark 通吃。

§ 012018 的分叉路口

2017 Transformer 出来，2018 OpenAI 用它的 decoder 搞出 GPT-1，证明"Transformer + 因果 LM pretrain + 任务 head finetune"能打 NLP 一片。但 GPT-1 只能看左边上下文 —— 这对生成类没问题，对理解类是天生的短板。更早的 ELMo12 用双向 LSTM 拼接试图绕过，但两个方向互不通气，不是真正的双向 attention。

看一句歧义句子："她从河岸边走过，看到 [bank] 上飘着的纸船"。判断这个 bank 是河岸还是银行 —— 你需要看右边的 "飘着的纸船"。GPT-1 看不到，BERT 看得到。

为什么是 attention 的 mask

BERT 和 GPT 的架构几乎一模一样 —— 都是 transformer 堆叠。唯一的差别是 attention 矩阵上有没有那个三角形的 causal mask。BERT 把它去掉了 —— 然后所有难题都在"怎么训练这玩意"上。

§ 02MLM · 训练双向的诡计

直接训"双向预测下一个词"会出现自我打脸：模型能看到右边，包括它自己。BERT 的解法 MLM6 是先把目标 token 遮起来，模型看不到答案，然后再训双向。输入先经过 WordPiece tokenizer9 切成子词，再送进 transformer。

为什么不是 100% 换成 [MASK]

因为 finetune 和 inference 时输入里没有 [MASK] 这个特殊 token。如果训练时 100% 用 [MASK]，模型会形成"看到 [MASK] 才动脑"的坏习惯 —— 训推不一致2。10% 随机词 + 10% 保留原词 是为了让模型在每一个 token 位置都保持高质量表征。

§ 03NSP · 给段间关系一个 pretrain 信号

MLM 是 token 级监督。但问答、自然语言推理这些任务关心的是两段文本的关系 —— 它们承接吗？支持吗？矛盾吗？BERT 想给段间关系一个预训练任务，于是搞了 NSP7，配套引入了 [CLS] / [SEP] / segment embedding 三件套8。

⚠ 假洞察

RoBERTa (2019) 实验证明 NSP 没用甚至有害 —— 因为随机抽的负样本通常话题完全不同，模型靠"主题词共现"就能 99% 判对，根本没在学段落级逻辑。后来主流 BERT 变种（RoBERTa / DeBERTa）都去掉了 NSP，把那点算力用在更长 MLM 序列上。BERT 论文里唯一被证伪的设计3。

§ 04只要 encoder · 架构对比

2017 Transformer 是 encoder + decoder（翻译 seq2seq）。2018 BERT 切下来 encoder，GPT 切下来 decoder，分叉成两条路。

三种切法 —— Transformer 保留两端做翻译；BERT 只留 encoder 把双向用足；GPT 只留 decoder 押注生成。架构上几乎一样的 block，全靠训练目标决定走向。

§ 05Pretrain → Finetune · 范式革命

2018 之前 NLP 大多任务都是从头训：取 word2vec / GloVe，搭 LSTM/CNN，标几千条数据训一个。BERT 把这套换成了：

关键点：backbone 共享，head 极小。一次昂贵 pretrain，N 次便宜 finetune。这套范式后来主导 NLP 五年，直到 GPT-3 的 in-context learning 把它从前沿挤掉。

这条路现在还活着

对话 / 生成被 GPT 系拿下了，但嵌入 / 检索 / RAG 几乎全是 BERT 后代 —— BGE / E5 / Cohere Embed / OpenAI text-embedding 都建立在 encoder-only + 双向表征的优势上4。decoder-only 算 embedding 慢且不双向。

§ 06当时炸场，今天的遗产

BERT 发布时把 11 项 NLP benchmark 全推到新 SOTA。GLUE 综合分11 从 GPT-1 的 72.8 跳到 BERT-large 的 82.1，SQuAD 1.1 EM 从 81.8 → 87.4。一夜之间，所有 NLP 论文的 baseline 都变成"BERT + 某种 finetune"。

但 BERT 真正持久的影响不是 benchmark 分数，是它把"pretrain 一个大模型再到处 finetune"做成了行业默认。GPT 路线后来用 prompt + ICL 替代了 finetune，但"先 pretrain 一个 backbone"这件事再也没倒退过5。

两条路的总账

2018-2022 NLP 是 BERT 的天下：理解任务通吃，每个公司都有自己的 BERT-finetuned 模型。2022 之后 GPT-3.5 / ChatGPT 出来，几乎所有"prompt 一个 LLM 就行"的场景把 BERT finetune 路径取代了 —— 包括分类、信息抽取、问答。BERT 系仍在嵌入 / 检索里活着，但前沿任务上 decoder-only 赢了。不是 BERT 错了，是 decoder-only 在规模和泛化上意外地强。

引用 / 术语表

  1. Bidirectional context · 双向上下文。在 BERT 里，每个位置的 self-attention 能看到序列里所有其他位置（包括右边）。GPT 因为加了 causal mask 看不到右边。这是 BERT 整篇论文的核心优势。详见 self-attention。
  2. Train-inference 分布漂移 · 训练时输入里有 [MASK]，但 inference 时没有。如果训练 100% 用 [MASK]，模型只在见到 [MASK] 时才"上心"，其他位置表征质量下滑 —— finetune 时会发现下游任务输入根本不会有 [MASK]，结果性能掉。80/10/10 是缓解这个问题的工程妥协。
  3. RoBERTa · A Robustly Optimized BERT Pretraining Approach · Liu et al. 2019。同 BERT 架构，做了三件事：去掉 NSP、训练序列加长、训练数据扩 10×。所有指标都涨。证明 NSP 不仅没用还在浪费算力。
  4. RAG 嵌入模型为何还在用 BERT 系 · embedding 任务需要每个 token 都看双向才能产生稳定的语义压缩。decoder-only 模型每个位置只看左边，需要额外 trick（最后一个 token 池化 + bidirectional fine-tune）才能做嵌入，且通常慢于同尺寸 encoder。RAG 检索侧 BGE / E5 / Cohere-Embed 都是 BERT 后裔。
  5. "先 pretrain 一个大模型再到处用"成了行业默认 · 2018 之后任何新任务的起点都是"载入哪个 backbone"，而不是"从零设计"。BERT 把这事变成 commonsense。后来 GPT-3 用 prompt 替代 finetune，再到 ChatGPT 用 RLHF 替代专项 finetune —— "pretrain → 适配"这件事从未改变。
  6. MLM · Masked Language Model · BERT 的核心预训练任务。详见 masked-language-model。
  7. NSP · Next Sentence Prediction · 两段 A、B；用 [CLS] 表征预测 B 是不是 A 在原文里的下一段。被证伪。详见 next-sentence-prediction。
  8. [CLS] / [SEP] / segment embedding · 三个特殊 token 和一个 0/1 embedding。[CLS] 在句首，hidden state 用于 sentence-level 分类；[SEP] 分隔 segment；segment embedding 标记 token 属于 A 还是 B。这套约定影响了后续所有 encoder-only 模型。
  9. WordPiece tokenizer · BERT 用的子词分词器，基于贪心最长匹配。把不在词表里的词拆成子词（如 "playing" → "play" + "##ing"）。后来 BPE / SentencePiece 更主流，但 BERT 当年用 WordPiece。
  10. BERT-base vs BERT-large · base 是 12 层 / 768 hidden / 12 head / 110M params；large 是 24 层 / 1024 hidden / 16 head / 340M params。论文给了同 hyperparam 下两个尺寸都更优，立住了"参数多更好"的趋势 —— 也是 GPT-3 / 现代 LLM scaling 的伏笔。
  11. GLUE benchmark · General Language Understanding Evaluation。9 项 NLP 理解任务（NLI、相似度、情感等）的综合 leaderboard。2018-2019 大家都靠它比 BERT 变种。后来 SuperGLUE、BIG-bench 取代了它。
  12. ELMo · 2018 前的"双向"半成品 · Peters et al. 2018。用两个独立的 LSTM 一个左到右一个右到左，最后拼接两边的 hidden state。两个方向互不通气，跟 BERT 在 transformer 内做真正的双向 self-attention 是两件事。BERT 让 ELMo 立刻过时。
