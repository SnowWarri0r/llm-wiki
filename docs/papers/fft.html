FFT ·
把波拆成纯频率，还拆得飞快。

Cooley & Tukey · An Algorithm for the Machine Calculation of Complex Fourier Series · Math. Comp. · 1965（高斯 1805 已先得）

傅里叶变换把一段信号拆成一堆纯频率的强度——像棱镜把白光散成彩虹。朴素地算要 N² 次，慢到跑不动；FFT 用"偶/奇折半 + 单位根对称"的分治，把它压到 N·log N。这一下，让 MP3、JPEG、MRI、音频频谱全都跑得动了。

TL;DR

FFT = 快速算 DFT1（离散傅里叶变换）的算法。DFT 做的事：把 N 个时域采样 → N 个频率强度，告诉你"每个频率占多少"（像棱镜拆光、耳朵拆和弦）。它测频率的办法是拿信号去和"转速 k 的小转子"对齐——频率对上就越堆越大（共振），对不上就正负抵消≈0。朴素算每个输出都要扫全部输入 → N² 次。FFT 把 N 点按下标拆成偶半 + 奇半，整体 DFT 能由两个半边 DFT 拼出来，递归到底 → N·log N4。关键是单位根2的对称性（转半圈反号）让一次乘法喂两个输出——那个 ± 小图案叫蝴蝶5，中间乘的旋转因子叫 twiddle3。搞 ML 为什么在意：卷积定理6（时域卷积 = 频域逐点乘）+ 音频 mel 频谱就是反复做 FFT。

§ 01傅里叶变换在算什么：棱镜拆光

先把"傅里叶变换"这四个字翻译成大白话：把一段随时间起伏的波，拆成一堆纯正弦波之和，并告诉你每个频率占多少。

最好的类比是棱镜：白光看着是一团，穿过棱镜散成彩虹——其实白光本来就是各种频率的光混在一起，棱镜只是把它们按频率排开。或者想钢琴和弦：你按下 do-mi-sol，空气里只有一条复杂波形，但耳朵能把它拆回三个音。傅里叶变换干的就是棱镜/耳蜗这件事——从混合波形反推出成分频率。

数字版叫 DFT（离散傅里叶变换）：喂 N 个时域采样点，吐 N 个数，第 k 个数说"频率 k 的成分有多强"（还带个相位）。

§ 02它怎么"测"一个频率：转圈圈

那 DFT 是怎么知道某个频率在不在的？直觉这样建：一个纯频率 = 一个匀速绕圈转的点（数学上就是复指数 e^(iθ)，单位圆上转的小箭头）。

DFT 测频率 k 的办法，是把信号缠到一个圆盘上、缠绕的转速设成 k，再看缠完之后这堆点的"重心"跑哪去了：

  • 转速正好等于信号里某个频率 → 每一圈都对齐，所有质量堆到一边，重心明显偏离圆心 → 这个频率响应很大（共振）。
  • 转速对不上 → 点均匀撒一圈，正负互相抵消，重心留在圆心 ≈ 0 → 这个频率不在。

所以一次 DFT，就是拿 N 个不同转速的转子，各自跟整条信号对一遍，量出每个频率的重心偏移。问题随之而来：每个转子都要扫一遍全部 N 个采样点，N 个转子 → N×N 次乘加。

§ 03为什么慢，又怎么变快：N² → N·log N

朴素 DFT：N 个输出频率，每个都把 N 个输入加权求和 → N² 次乘加。音频 N=100 万的话就是 10¹² 次，单机跑一次要按分钟甚至小时算，完全没法实时。

FFT 的偷懒（Cooley–Tukey 1965，其实高斯 1805 就写过）是一招经典分治：把 N 个采样按下标拆成偶数位一半、奇数位一半。神奇之处——整条信号的 DFT，可以由"偶半边的 DFT"和"奇半边的 DFT"拼出来，每半边只有 N/2 大。然后对每半边递归地再拆。跟归并排序同一个套路：

T(N) = 2·T(N/2) + O(N)   →   N·log N
       └ 拆两半 ┘  └ 拼接 ┘     （拆 log N 层，每层合并 O(N)）

省的到底是什么？拆到底后，偶半边、奇半边各自的结果会被反复复用——同一个中间结果，能同时拼出两个不同频率的输出。这个"算一次、用两次"的复用，乘以 log N 层，就是 N² 塌成 N·log N 的全部来源。下一节看清楚这个复用长什么样。

§ 04蝴蝶与 twiddle：一次乘喂两个输出

为什么偶/奇拆分能拼回来、还能复用？靠单位根的对称性。转子 w = e^(−2πi/N)（绕一圈的小步），它有个关键性质：转过半圈正好反号，w^(k+N/2) = −w^k。

于是对一对输出频率 k 和 k+N/2，公式长这样（E=偶半边结果，O=奇半边结果）：

X[k]       = E[k] + w^k · O[k]      ← 加
X[k+N/2]   = E[k] − w^k · O[k]      ← 减（因为 w 转半圈反号）

注意：那个乘法 w^k · O[k] 只算一次，一个加、一个减，就同时出了两个输出。这就是"算一次用两次"。中间乘的旋转因子 w^k 叫 twiddle factor；这个 ± 一对的数据流小图案叫 蝴蝶（butterfly）——两进两出、交叉得像翅膀。堆 log N 层蝴蝶，就是一整个 FFT。

三个实现细节 · 拿 N=8 砸实

① 为什么非得 2 的幂（radix-2）。折半要能一路折到底：8 → 4 → 2 → 1 干净。但 6 折一次到 3 就卡住——3 是奇数，没法再对半切成两个整数半边。遇到非 2 幂长度有三条路：补零凑到 8（zero-pad）、按 6=2×3 走 mixed-radix、或用 Bluestein 把任意长度变成一个卷积再拿 2 幂 FFT 算。

② bit-reversal 到底重排成什么样。把下标 0–7 反复按偶/奇拆到底，叶子顺序变成 0 4 2 6 1 5 3 7。这串数正好是把下标的二进制位左右翻过来：比如 1=001 翻成 100=4，3=011 翻成 110=6。所以实现里常先按这张表把输入重排一遍，再就地叠蝴蝶：

③ IFFT 怎么"变回去"。逆变换用同一套蝴蝶，只改两处：twiddle 的旋转方向取反（共轭，e^(+…) 而不是 e^(−…)），最后每个数除以 N。举个最小例子——常数信号 [1,1,1,1] 做 FFT → [4,0,0,0]（只有"直流"那一根亮，因为它压根没有任何起伏的频率）；把 [4,0,0,0] 喂回 IFFT、跑同样的蝴蝶、最后除以 4 → 又老老实实变回 [1,1,1,1]。

§ 05拿 x=[1,2,3,4] 真算一遍

前面都是图和道理，这节拿一个具体输入 x = [1, 2, 3, 4]，把 N=4 的 FFT 一步步蝴蝶算到底，再跟朴素 N² 对答案。

先备好 N=4 的 4 个转子（twiddle）。w = e^(−i2π/4) 就是"每次转 90°（顺时针）"，所以：

w⁰ = 1     （不转）
w¹ = −i    （转 90°）
w² = −1    （转 180°）
w³ = +i    （转 270°）
带 i 别慌：i 只是"转一个直角"的记号，× (−i) = 顺时针转 90°

Step 1 · 按偶/奇拆成两半

x=[1,2,3,4] → 偶位 [x₀,x₂]=[1,3]、奇位 [x₁,x₃]=[2,4]。

Step 2 · 各做一个 2 点 DFT（最简单的蝴蝶，twiddle=w⁰=1，纯加减）

半边top = 两数相加bottom = 两数相减
偶 [1,3]E₀ = 1+3 = 4E₁ = 1−3 = −2
奇 [2,4]O₀ = 2+4 = 6O₁ = 2−4 = −2

Step 3 · 拼起来（外层蝴蝶，奇路乘 twiddle）

规则：X[k] = E[k] + wᵏ·O[k]，X[k+2] = E[k] − wᵏ·O[k]。

k · twiddle加 → X[k]减 → X[k+2]
k=0 · w⁰=1X₀ = 4 + 1·6 = 10X₂ = 4 − 1·6 = −2
k=1 · w¹=−iX₁ = −2 + (−i)(−2) = −2+2iX₃ = −2 − (−i)(−2) = −2−2i

对答案 · 朴素 N² 直接硬算

朴素 DFT 每个输出 X[k] = Σ x[n]·wⁿᵏ，4 个数全加权求和：

X₀ = 1 + 2 + 3 + 4                     = 10
X₁ = 1 + 2(−i) + 3(−1) + 4(i) = 1−2i−3+4i = −2+2i
X₂ = 1 + 2(−1) + 3(1) + 4(−1)  = 1−2+3−4   = −2
X₃ = 1 + 2(i) + 3(−1) + 4(−i)  = 1+2i−3−4i = −2−2i

一模一样。FFT 没偷工，只是把重复的加减/乘法重排复用了。

读结果 + 省了多少

X₀=10 是"直流"——就是 4 个数的总和（整体平均水平×4，没有任何起伏）；其余几个复数的大小就是对应频率成分的强度。
省了多少：朴素要算 4×4=16 个 x[n]·wⁿᵏ 乘加；FFT 这边只有 2 个 2 点 DFT + 2 次 twiddle 乘。N=4 省得还不明显，但每把 N 翻倍一次，朴素 ×4、FFT 只 ×2——到 N=1024 时朴素 ~100 万次、FFT ~1 万次。

§ 06搞 ML 为什么该懂：卷积定理 + 频谱

这不只是信号处理的老古董，它在你学的东西里反复出现：

  • 卷积定理——时域卷积 = 频域逐点相乘。所以一个贵的卷积可以"FFT → 逐元素乘 → IFFT"绕过去，大核卷积提速就靠这个。
  • 音频模型——mel 频谱就是把信号切成一段段窗口、每段做一次 FFT（STFT），再压到 mel 刻度。你跑过的 fish-speech、Whisper 的输入频谱全是这么来的。
  • 频域学习——FNO（傅里叶神经算子）直接在频域里学算子；RoPE 的旋转本质也是一套频率基。
放进 wiki 这条线

FFT 是分治（同 卷积/归并的"折半再合"思路）+ 单位根对称（转半圈反号 → 一次乘喂两个输出）两件事拼出来的。一句话记牢：傅里叶变换把波拆成频率（棱镜）；FFT 用偶/奇折半 + 蝴蝶复用，把 N² 砍成 N·log N。它是音频 频谱 和卷积加速背后的发动机。

引用 / 术语表

  1. DFT · 离散傅里叶变换 · 把 N 个时域采样 → N 个频率强度（每个带幅度+相位）。FFT 是快速计算 DFT 的算法，两者算的是同一件事，只是 FFT 快得多。
  2. 单位根 / 复指数 · w = e^(−2πi/N)，单位圆上等分的旋转步。关键对称性 w^(k+N/2) = −w^k（转半圈反号），正是偶/奇能拼回、能复用的根源。
  3. twiddle factor · 旋转因子 · 蝴蝶里乘在奇半边上的 w^k。一个蝴蝶只乘一次它，加减各取一次结果。
  4. 分治 · N·log N · 按下标拆偶/奇两半，整体 DFT 由两半 DFT 拼出，递归到底。T(N)=2T(N/2)+O(N) → N·log N，对比朴素 N²。
  5. 蝴蝶 · butterfly · 两进两出的基本单元：X[k]=E+w^k·O、X[k+N/2]=E−w^k·O，交叉如翅膀。堆 log N 层就是整个 FFT。
  6. 卷积定理 · 时域卷积 = 频域逐点相乘。于是卷积可走"FFT→逐点乘→IFFT"加速；大核尤其划算。也是 FFT 在 ML 里最常被用到的入口。
  7. bit-reversal · 位反转重排 · 递归偶/奇拆到底后，输入顺序正好是下标二进制位反过来的排列。就地 FFT 常先做一遍位反转，再叠蝴蝶。
  8. STFT · 短时傅里叶变换 · 把长信号切成一段段窗口、每段做 FFT，得到"时间×频率"的频谱图。音频模型的标准输入。
