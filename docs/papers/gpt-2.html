FIELD NOTES · 2019 · OpenAI · GPT-1 之后 8 个月

Language Models are
Multitask Learners.

Radford · Wu · Child · Luan · Amodei · Sutskever · 2019-02

同一架构放大 13 倍（117M → 1.5B），换上 WebText 数据，把 "prompt 触发任务" 第一次摆出来 —— 不用 finetune 也能跨任务工作。从 GPT-1 的"pretrain + finetune"演化到 GPT-2 的"pretrain + 直接 prompt"。

TL;DR

GPT-1 验证了"用 LM 预训练 + 任务 finetune"路线，但每个新任务还要 finetune 一次。GPT-2 押注：把 LM 做大到 1.5B，它在预训练阶段已经"看过"足够多的任务格式 —— prompt 一下就能触发，不用 finetune。

3 个变化：

  • 架构 scale 13× · 117M → 1.5B · 4 个 size 系列证明 LM loss 单调下降
  • WebText · 40GB 高质量网页 · 用 Reddit ≥ 3 karma 当过滤信号
  • Zero-shot transfer1 · 用 prompt 触发摘要 / 翻译 / QA / 分类，不改一个参数

论文标题就是 thesis："Language Models are Unsupervised Multitask Learners"2 —— LM 训练本身已经在做无数任务，多任务能力是预训练的天然副产品。

§ 01GPT-1 之后的下一步

GPT-1 在 2018-06 验证了 thesis 可行，但每个新任务还要 finetune。问题：

  • 每个任务一份 finetune 后的 weights，工程上麻烦
  • finetune 超参敏感，要为每个任务调一次
  • 模型大了之后 finetune 占用资源也大

OpenAI 押的新注：如果 LM 大到一定程度，预训练时已经"见过"足够多的任务格式 —— prompt 一下就能触发，根本不用 finetune。这个思路是 GPT-1 的 input transformations 的直接延续，只是把"特殊 token + finetune"换成"自然语言 prompt + 无 finetune"。

核心 thesis 升级

GPT-1 论文标题 "Improving Language Understanding by Generative Pre-Training" —— 还是 GPT-1 在帮 understanding tasks。

GPT-2 论文标题 "Language Models are Unsupervised Multitask Learners" —— LM 训练本身就是多任务学习，预训练就够了。论点向前迈了一大步。

§ 03Zero-shot · prompt 就能触发任务

1.5B 模型预训练完后，不改一个参数，用提示文本就能触发各种任务。这是 GPT-2 最具历史意义的发现。

为什么这能 work

WebText 里有大量这种格式的文本 —— 博客末尾的 "TL;DR:"、双语对照表里的 "english: ... french: ..."、问答论坛的 "Q: ... A: ..."、评论网站的 "Sentiment: ..."。预训练时模型见过几百万次后，自然学到"看到 TL;DR: 就要总结前文"。prompt 只是触发这层已经被压进权重的映射，不是教新东西。详见 language-modeling-as-multitask6。

⚠ 但 zero-shot 不完美

论文里 GPT-2 zero-shot 在很多 benchmark 上离 SOTA 还有距离：

  • 翻译 BLEU 5-10 分 · 专门 finetune 模型能到 30-40 分
  • SQuAD QA F1 55 · BERT 当年已经 91+
  • 分类任务多数低于 SOTA finetune 模型

但趋势在那里：从 124M 到 1.5B，zero-shot 性能稳定提升。GPT-2 没追上 SOTA 不算输 —— 它在论证一条新路。GPT-3 用 175B 把这条路推到能赢 SOTA 的程度。

§ 04WebText · 把人类注意力当数据 filter

GPT-1 用 BookCorpus 7000 本书，量小但干净。要 scale 到 1.5B，得有更多数据 —— 但不能用纯爬虫垃圾。OpenAI 想了个聪明 trick：

为什么这 trick 聪明

Reddit ≥ 3 karma 是个简陋但有效的启发式：

  • 用户点赞 = 一种 free 的人工标注（不用付钱标数据）
  • 外链到外站的帖子 = "我觉得这值得分享"语义 → 天然高质量
  • 能滤掉 SEO 垃圾、自动生成内容、刷量博客

本质是把人类注意力当成免费的数据质量 reviewer。这思路在后来的 RLHF / Constitutional AI 里又以不同形式出现 —— 让人当 reviewer 一直是 AI 训练成本的最大单点投入。WebText 本身因版权 + Reddit ToS 没开源12，开源社区做了 OpenWebText 复现版。

§ 06"Too dangerous to release" · staged release 的开端

GPT-2 发布时 OpenAI 做了个当年很有争议的决定：不一次性放出 1.5B 权重，怕被滥用生成假新闻 / spam。分阶段释放：

  • 2019-02 · 124M · 同时发论文
  • 2019-05 · 355M · 三个月后
  • 2019-08 · 774M · 半年后
  • 2019-11 · 1.5B · 9 个月后 · 完整 release

当时业界争论激烈：有人说作秀（"AI 模型能多危险？"），有人说慎重是必要的。回头看，分阶段评估社会影响这事是必要操作 —— Claude / GPT-4 / Llama 等后来 frontier model 都做类似流程。GPT-2 开了这个头。

回头看的判断

2019 当年 1.5B 模型能生成的内容相比之下没那么危险（写假新闻还得编 prompt 引导）。但这次实践本身有价值：

(1) 建立了"frontier model 不是发了就完"的概念，每次发布是一次 deliberate 评估
(2) 启动了 AI safety 社区对"模型公开 vs 封闭"的长期讨论
(3) 让 OpenAI 学会怎么做风险评估流程 —— 这套流程后来在 GPT-3 / GPT-4 / o1 / o3 上反复使用

真正的危险不是 GPT-2，是它预示了 GPT-3 / 4 这种确实有影响力的模型即将到来。GPT-2 的 staged release 是这个 trajectory 的演习。

§ 07真正的遗产

GPT-2 留下来的三件事：

  1. Thesis 钉死 · "Language Models are Unsupervised Multitask Learners"9 —— 后续每个 OpenAI / 大模型 paper 都在续这本账
  2. Scaling 行为可见 · 4 个 size 单调改善证明了"模型大 → 能力强"，为 2020 Scaling Laws 提供数据基础
  3. Prompt engineering 起点10 · "用文本格式触发任务"从这里开始成为正经能力，演化成 GPT-3 in-context learning 和 ChatGPT chat template

但 GPT-2 自己不是终点产品。它在 benchmark 上输给 finetune 的 BERT 系，工业界 2019-2020 也很少用 GPT-2 做实际任务。它的价值是论证一条路径 —— 而这条路径在 GPT-311 那里才真正赢。

把 GPT-2 放进 trajectory

GPT-1 论证可行 → GPT-2 论证 scale 有效 + thesis 表达清晰 → GPT-3 论证 175B 能 emergence in-context learning → ChatGPT 把这事产品化。GPT-2 是中间一环 —— 没它就没 Scaling Laws，没 GPT-3，没 ChatGPT。但单看 GPT-2 当时的 benchmark，看不出这事的重要性。这是为什么 trajectory 视角比 single-paper 视角更准：每篇 paper 不是孤立的成就，是路径上的一步。

引用 / 术语表

  1. Zero-shot transfer · 零样本迁移 · 预训练完的 LM 不改一个参数，直接在 prompt 里写任务格式让模型续写出答案。详见 zero-shot-transfer。
  2. "Language Models are Unsupervised Multitask Learners" · GPT-2 论文标题，也是核心 thesis：LM 预训练本身就在隐式做多任务学习。互联网文本充满隐式任务示例（TL;DR / Q&A / 翻译对 / Sentiment 标签），LM 训练时每个 token 位置预测下一词，相当于同时学几万种任务。详见 language-modeling-as-multitask。
  3. Perplexity · 困惑度 · LM 性能指标，越低越好。直觉：模型预测下一个词时"心里"在多少个候选词里犹豫。perplexity = 1 表示模型百分百确定下一个词；perplexity = 100 表示在 100 个候选词里平均犹豫。GPT-2 1.5B WebText perplexity 18.34 = 模型平均在 ~18 个候选里挑下一词。
  4. Scaling Laws · Kaplan et al. 2020 · OpenAI 后续把 GPT-2 系列的 scaling 行为形式化：LM 的 loss 跟参数量 / 数据量 / 算力满足 power law。给出了"模型大多少 + 数据多少 + 算力多少 = loss 多少"的经验公式，让 OpenAI 训练 GPT-3 时能预算式地决定 175B 这个数字。
  5. "TL;DR:" Trick · GPT-2 论文展示的 zero-shot 摘要技巧。把一段文章后面接 "TL;DR:" 然后让模型续写，会自动输出摘要。WebText 里"博客末尾 TL;DR:"格式见过几百万次，模型学到了这层映射。
  6. 隐式任务示例 · 互联网无标注文本里"碰巧"包含的任务样本。比如博客末的 "TL;DR:" 是摘要样本、双语对照表是翻译样本、问答论坛是 QA 样本。LM 预训练时这些隐式样本驱动模型学到对应任务能力 —— 不是显式标注，是数据分布自带的。详见 language-modeling-as-multitask。
  7. Reddit karma ≥ 3 启发式 · WebText 数据筛选的核心 trick。爬出现在 Reddit ≥ 3 karma 帖子里的外链页面 —— 把用户点赞当作"内容值得读"的免费信号。详见 webtext。
  8. Staged release · 分阶段释放 · GPT-2 不一次性放 1.5B 权重，分 4 阶段放（124M → 355M → 774M → 1.5B），每次评估社会影响后再决定下一步。当年争议大，但开了 AI 安全发布的先例 —— Claude / GPT-4 / Llama 后续都做类似流程。
  9. 为什么 thesis 钉死重要 · 一个领域的方向感来自于"接下来该往哪走"。GPT-2 论文标题把方向钉死："LM 即多任务学习"。所有 OpenAI 后续 paper 都在续这本账：scaling laws / GPT-3 / instruction following / RLHF / ChatGPT。BERT 系当时没立这种长期 thesis（"双向理解"是个静态目标），所以方向感比 GPT 路线弱。
  10. Prompt engineering · "用文本格式触发任务"演化出的实践。GPT-2 时还很粗糙，主要靠 "TL;DR:" "english:/french:" 这种关键词。GPT-3 + ChatGPT 时代演化成 system prompt / few-shot example / chain-of-thought reasoning 等更复杂技巧。详见 input-transformations。
  11. GPT-3 在哪赢 · GPT-2 zero-shot 多数 benchmark 输给 finetune 模型。GPT-3 (175B) 加入了 in-context learning（prompt 里塞几个例子）后，zero/few-shot 性能首次能跟 SOTA finetune 对标 —— 这才是 GPT 路线真正"赢"的拐点。GPT-2 是论证可行的中间一步。
  12. WebText 没开源 · OpenAI 因版权 + Reddit ToS 没公开 WebText 本身。开源社区做了 OpenWebText 复现版（自己重新爬一遍 Reddit ≥ 3 karma 外链）。这也开了"大模型训练数据不公开"的先例 —— 后来 Anthropic / Google 训练数据也都不公开。
