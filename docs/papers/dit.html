FIELD NOTES · 扩散生成主干

U-Net
退役.

William Peebles (UC Berkeley) · Saining Xie (NYU) · ICCV 2023 · arXiv 2212.09748

扩散去噪一直用 U-Net。DiT 证明纯 Transformer 在 latent patch 上去噪不但能用，还更会 scale——SD3 / FLUX / 视频生成全是它的后代。

TL;DR

把扩散模型的去噪 U-Net 整个换成 Transformer：在 VAE latent（256×256 图 → 32×32×4）上 patchify 成 token 序列，像 ViT 一样去噪。时间步 t 和类别 c 通过 adaLN-Zero 注入——从 (t+c) 算出每层归一化的缩放/平移，再加一个零初始化的门控让每块开训即恒等。比 cross-attention / in-context 都好(FID 19.47 vs 26.14)还更省。关键认知：DiT 是架构，flow matching / diffusion 是训练目标，两者正交——原版 DiT 预测噪声 ε(DDPM)，SD3/FLUX 把同一个 DiT 拿去预测速度就是 flow matching，架构一个字不改。算力越多 FID 越低，DiT-XL/2 在 ImageNet 256 拿下 FID 2.27。

§ 01把 U-Net 换成 Transformer

早期扩散（DDPM、Stable Diffusion）的去噪网络都是卷积 U-Net——带一堆局部性、多尺度的归纳偏置。但 Transformer 在别的领域早证明了"大数据 + 可 scale"更香。DiT 来验证：去噪能不能也换成纯 Transformer？做法跟 ViT 一个味道——把 VAE latent 切成 patch token，喂进标准 Transformer，让 self-attention 在 token 间传信息。patch 越小，token 越多、算力越大、FID 越低。块里的 FFN 原版用 GELU、现代 DiT 家族换成 SwiGLU。

§ 02adaLN-Zero：用时间和条件去拧每层旋钮

Transformer 没有 U-Net 那些现成的条件注入口，得自己设计怎么把 时间步 t + 类别 c 喂进去。DiT 的答案是 adaLN：不用固定的 LayerNorm 缩放 γ、平移 β，而是从 (t+c) 用一个小 MLP 算出 γ、β 去拧每层归一化。再加一手 adaLN-Zero——为每个残差分支多算一个门控 α 并零初始化，让每块开训时是恒等函数（同 resnet 的 identity 起点）。详见 adaptive-layernorm。

§ 03为什么是 adaLN-Zero：四种条件注入实测

DiT 把四种把条件喂进 Transformer 的办法摆一起比（DiT-XL/2 @400K 步，FID 越低越好）：把条件当额外 token 拼进序列（in-context）、加一层交叉注意力（cross-attention）、用条件算 γ/β（adaLN）、再加零门控（adaLN-Zero）。结论干脆——adaLN-Zero 最低，还最省算力。

§ 04关键：DiT 是架构，flow matching 是目标（正交）

这是最容易卡住的点：你在 DiT 结构里看不到 flow matching，因为它根本不在架构里。 DiT 只是个条件函数——吃进 (带噪 latent xt, 时间 t, 条件 c)，吐出同形状的一个张量。这个张量该是什么、以及怎么用它采样，才是 diffusion / flow matching 的事。原版 DiT 训它预测噪声 ε(DDPM)；SD3/FLUX 把同一个 DiT 训成预测速度 v，就是 flow-matching——架构一个字不改。adaLN 的时间条件，就是把网络和"轨迹走到哪了"接起来的桥。

一句话拆穿"模拟"

网络从头到尾只干一件事：给定你现在在哪(xt)、现在几点(t)，说一个方向。flow matching 规定"方向的目标 = x₁−x₀"且"用欧拉法把方向积分出来"；diffusion 规定"预测噪声 ε 再按公式反推"。同一个 DiT，换目标即换范式，这就是为什么结构图里看不到它。

§ 05Scaling：算力越多，FID 越低

DiT 用 Gflops 当横轴画 scaling：把模型加深加宽、或把 patch 调小（token 变多），算力越大，FID 单调往下，干净且可预测（四档 DiT-S/B/L/XL）。最大的 DiT-XL/2 在 class-conditional ImageNet 256×256 拿下 FID 2.27（cfg=1.5；不加 guidance 是 9.62），当时 SOTA。

放进 wiki 这条线

DiT 是 扩散用 Transformer 的源头：把去噪 U-Net 换成在 latent patch 上跑的 ViT，用 adaLN-Zero 注 t/c。它的两个后话——SwiGLU 换掉 GELU、flow matching 换掉 DDPM——都是后来 SD3 / FLUX 给 DiT 家族做的现代化，把它升级成 MMDiT 双流；Cosmos 3 的 generator 塔本质也是个 DiT。一句话：DiT 给扩散生成立了个"纯 Transformer、可 scale"的干净主干;而它能跑 diffusion 还是 flow matching,不在架构里,在你训它预测 ε 还是速度。

引用 / 术语表

  1. adaLN-Zero · 从 (t+c) 用 MLP 算每层归一化的缩放 γ/平移 β，再加零初始化门控 α 让每块开训即恒等。比 cross-attn/in-context 都好且省。
  2. patchify · 把 VAE latent（32×32×4）切成 p×p 的 patch，拉平投影成 token 序列喂 Transformer。p 越小 token 越多、算力越大、FID 越低（p=2→256 token）。
  3. 条件注入四法 · in-context(35.24) / cross-attention(26.14) / adaLN(25.21) / adaLN-Zero(19.47)，FID@400K，越低越好，adaLN-Zero 胜。
  4. 架构 ⊥ 目标 · DiT 是架构(条件函数 (xt,t,c)→输出)，diffusion(预测 ε)/flow matching(预测速度 v) 是训练目标 + 采样法。同一个 DiT 换目标即换范式。
  5. Gflops scaling · 算力(更深/更宽/更小 patch)↑ → FID↓，单调可预测。DiT-XL/2 = 118.64 Gflops。
  6. VAE latent · 用 Stable Diffusion 的 VAE 把 256×256×3 图压成 32×32×4 latent，DiT 在 latent 上去噪而非像素。
  7. DiT-S/B/L/XL · 四档：S(12 层,384)/B(12,768)/L(24,1024)/XL(28,1152)。XL/2 最大。
  8. FID 2.27 · DiT-XL/2 在 class-conditional ImageNet 256×256 的成绩(cfg=1.5)，当时 SOTA；不加 guidance 9.62。
  9. SwiGLU · 带门的 FFN(现代 DiT 家族用)。注意：原版 DiT 用的是 GELU，SwiGLU 是 SD3 之后的升级。
  10. classifier-free guidance · 采样时用条件/无条件两支外推，拉高与 prompt 的契合；DiT 靠它把 FID 从 9.62 提到 2.27。
