Black Forest Labs（原 Stable Diffusion / Latent Diffusion 核心作者）· FLUX.1 · 2024-08 开源权重

SD 原作者出走另起炉灶的下一代答卷。12B 整流流 Transformer，主干是"先双流 MMDiT、后单流"的混合架构；再叠两道正交的蒸馏——引导蒸馏(把 CFG 的两遍前向压成一遍) + 步数蒸馏(4 步出图)——做出 pro / dev / schnell 三档。

TL;DR

FLUX.1（Black Forest Labs，原 SD/LDM 作者，2024-08）是 12B 整流流 Transformer。架构招牌：先双流后单流的混合1——前段 双流 MMDiT 块（文字图像各用各的 FFN/AdaLN 对齐），后段 单流块（所有 token 共享权重、更省）。再叠两道正交蒸馏：引导蒸馏2（把 CFG 的每步 2 次前向压成 1 次，引导强度当输入喂进去）做出 dev；再 步数蒸馏到 4 步做出 schnell3。仍是整流流（flow-matching）+ T5/CLIP 文本编码 + VAE latent。三档：pro(闭源旗舰) / dev(引导蒸馏,非商用) / schnell(再步数蒸馏,Apache 2.0)4。底座具体数：19 双流块 + 38 单流块、hidden 3072(24 头×128)、RoPE 位置编码、16 通道 VAE(f8,比老 SD 的 4 通道厚 4×)5。

§ 01来头：SD 原作者的下一代

做 FLUX 的 Black Forest Labs，核心成员就是当年 Stable Diffusion / Latent Diffusion 论文的作者，从 Stability 出走后另起炉灶。所以 FLUX 是"SD 原班人马的下一代答卷"——2024-08 一出就把开源文生图的天花板顶上去。底子和 SD3.5 同源（整流流 + MMDiT + VAE latent），但有两个自己的招。

§ 02整体解剖：12B 是怎么搭起来的

先把零件摆清楚,后面两招才有地方挂。一句 prompt 进来要走三路条件,图在压缩后的 latent 里去噪,主干是 57 个 Transformer 块,最后解码回像素。

三路条件:T5-XXL11(给细粒度的文字语义,4096 维序列)+ CLIP-L(给一个全局 pooled 向量)管"画什么";时间步 t 和引导强度 w 过一个小 MLP 变成调制向量 vec,经 AdaLN12 拧每个块的归一化(跟 DiT 注入条件一个套路);图像那路在 VAE 压出来的 latent 里去噪。

数字例子 · VAE 把图压掉多少

FLUX 的 VAE 是 f8 + 16 通道7:空间每边下采样 8 倍、通道从 RGB 的 3 升到 16。一张 1024×1024 的图:

原图像素数   = 1024 × 1024 × 3  = 3,145,728 个数
压缩后 latent = 128  × 128  × 16 =   262,144 个数   (1024÷8=128)
压缩倍数     = 3,145,728 / 262,144 = 12×

✓ 自检:空间上 8×8=64 倍更少的格子,但每格通道厚了 16/3≈5.3 倍,净下来 64 ÷ 5.3 ≈ 12 倍。扩散全程在这 12 倍小的空间里跑,省的就是这块算力;通道开到 16(老 SD 只有 4)是为了少压点细节、把"压得狠"和"还原得清"平衡好。

§ 03招一：先双流对齐，再单流干完

你在 DiT / MMDiT 学过"单流 vs 双流"的分歧：双流（文字图像各用各的权重，只在 attention 合流）对齐强但参数多；单流（共享一套权重一条序列）省但表达弱。FLUX 两个都要，而且分前后。

前段双流：文字和图像还在"互相认识"，各带各的 FFN / AdaLN（各说母语）把两种模态对齐好。对齐之后切单流：它们已经融成一锅，再分两套权重就浪费，合成一条共享序列更省。先用贵的把齐对上，再用省的把活干完。

具体分配:19 个双流块在前、38 个单流块在后8(单流占了三分之二,印证"对齐是少数、干活是多数")。还有个省算力的细节:单流块里 attention 和 MLP 是并行算的——不是"先注意力、再前馈"串起来,而是从同一个归一化输入同时分出 QKV 和 MLP 两路、各算各的再相加(跟 GPT-J / ViT-22B 一个路子)。少一次中间归一化、两个大矩阵乘能拼成一次,实测更快且不掉质量。

§ 04招二：引导蒸馏 —— CFG 两遍压成一遍

普通文生图靠 CFG 提质量：每步要算两遍（一遍带 prompt、一遍空 prompt），再按引导强度 w 把差值外推 → 每步 2× 算力。FLUX.1-dev 把这事蒸进模型：把引导强度 w 当成一个输入喂进去，模型一遍前向就直接吐出"已经引导好"的结果。

引导蒸馏让 dev 在不掉质量的前提下省掉那一半算力（老师是闭源的 pro，dev 被训成"一遍就复现 pro 带 CFG 的结果"）。引导强度具体怎么"喂进去"? 跟时间步 t 走同一条路:guidance 标量过一个小 MLP 变成一个 embedding,加进调制向量 vec,再经 AdaLN 拧每个块。所以 dev 的 guidance 是模型的一个条件输入,跟标准 CFG 推理时在外面手动外推的 w 不是一回事——后者被蒸进了前者。

数字例子 · 被蒸掉的"外推"到底在算什么

把速度场想成方向向量,看标准 CFG 那两遍前向在干嘛(引导蒸馏要复现的就是这个结果):

v_uncond = [1, 0]        蒙眼:大致"画个合理的图"
v_cond   = [1, 0.5]      给"红色汽车":多了点往"红车"偏的分量
差值      = v_cond − v_uncond = [0, 0.5]   ← 纯粹因这句 prompt 多走的那部分
CFG(w=3): v = v_uncond + 3 × [0, 0.5] = [1, 1.5]   往"红车"方向使劲推

✓ 自检:注意结果 [1,1.5] 比 v_cond 的 [1,0.5] 还要"红车"——这是外推9(冲过条件解),不是在两者间插值,所以 w 太大会过饱和失真。标准 CFG 每步要跑两遍前向才能拿到这两个向量;dev 把"给定 w 直接输出这个外推结果"学进了一遍前向里。

§ 05招三：步数蒸馏 —— 50 步压到 4 步做 schnell

引导蒸馏省的是"每步两遍",但步数还在(整流流6采样仍要几十步积分)。schnell 再叠一道 DMD 步数蒸馏10,把出图压到 4 步。关键不是"让 4 大步去抄 50 小步的轨迹"(抄不像),而是只要求学生 4 步出来的图,整体分布跟老师 50 步的一样。

两道蒸馏因此互不打架(正交):引导蒸馏砍掉"每步的 2×"、步数蒸馏砍掉"步数本身"。一个管单步多贵、一个管走几步,各省各的,叠起来就是 schnell。DMD 那套"吸老师分布、斥自己分布"跟 Drifting 是同构的,细节见概念页。

§ 06三档：两道蒸馏怎么叠出来

把三档摆开看,就是"老师 + 两道正交蒸馏依次叠"的关系:

版本蒸了什么开源协议
FLUX.1 [pro]闭源旗舰（当老师）不开放（API）
FLUX.1 [dev]引导蒸馏：CFG 进模型，1 遍前向开放权重，非商用
FLUX.1 [schnell]在 dev 上再步数蒸馏 → 4 步出图Apache 2.0 全开放

跟 SD3.5 摆一起看

维度SD3.5FLUX.1
范式整流流 + MMDiT整流流 + MMDiT
架构全程双流（+双注意力）先双流 后单流 混合
规模8B12B
少步/省算Turbo 蒸馏引导蒸馏(dev) + 步数蒸馏(schnell)
出身StabilitySD 原作者新公司
放进 wiki 这条线

FLUX 把这几块重新排了次序：整流流 + MMDiT 双流（但只用在前段）+ 单流（后段）+ CFG 的引导蒸馏 + 步数蒸馏 + VAE latent。一句话：先双流对齐、再单流干完；引导蒸馏省每步的 2×，步数蒸馏省步数，两个正交叠成 schnell。跟 SD3.5 同源不同路（全双流 vs 先双后单），跟 Qwen-Image 都属 MMDiT 一族。

引用 / 术语表

  1. 先双流后单流 · 混合架构 · 前段 双流 MMDiT 块（文字图像各用各的 FFN/AdaLN 对齐），后段单流块（所有 token 共享权重）。先用贵的对齐、再用省的收尾。FLUX 跟全双流 SD3、全单流 Ideogram 的关键区别。
  2. 引导蒸馏 · guidance distillation · 把 CFG 的每步 2 次前向（条件+无条件）蒸成 1 次：引导强度当输入喂模型，直接吐"已引导"结果。dev 这么训出来（老师=pro）。省一半算力、不掉质量。
  3. 步数蒸馏 · schnell · 在 dev 基础上把采样步数蒸到 4 步。与引导蒸馏正交（一个省每步算力、一个省步数）。schnell = 两道都叠。
  4. pro / dev / schnell 三档 · pro 闭源旗舰当老师；dev 引导蒸馏、开放权重非商用；schnell 再步数蒸馏、4 步、Apache 2.0 全开放。都是 12B 整流流 Transformer。
  5. 架构具体数 · 19 双流块 + 38 单流块(单流占 2/3)、hidden 3072(24 头 × 头维 128)、RoPE 位置编码、16 通道 VAE(f8)。整体 12B 参数。
  6. 整流流 · rectified flow · 把噪声→数据的路径学成尽量直的线;直线少拐弯,所以少步也能采得准。FLUX 的训练目标(flow matching 一种)。
  7. VAE · f8 + 16 通道 · 空间每边压 8 倍、通道 3→16;1024² 图压成 128×128×16 ≈ 少 12× 的数。扩散全程在这小空间跑;16 通道(老 SD 才 4)是为了少丢细节。
  8. 双流块 / 单流块 · 双流=文图各一套权重、只在 attention 合流(对齐强但贵);单流=合并共享权重且 attention 与 MLP 并行算(省)。FLUX = 19 双 + 38 单。
  9. CFG 外推 · v_uncond + w·(v_cond − v_uncond),把"因 prompt 多走的那部分"放大。是外推(冲过条件解)非插值,w 太大会过饱和。被引导蒸馏吸进一遍前向。
  10. DMD 分布匹配 · 步数蒸馏不抄老师轨迹,只让学生少步出图的分布 ≈ 老师;teacher_score − fake_score(吸老师·斥自己),跟 Drifting 同构。
  11. 双文本编码 · T5-XXL 给 4096 维细粒度序列 + CLIP-L 给全局 pooled 向量,两路一起管"画什么"。
  12. AdaLN 调制注入 · 时间步 t 和引导 w 过 MLP 成调制向量 vec,经 adaptive LayerNorm 拧每个块的归一化——这是 DiT 注入条件的标准做法。
