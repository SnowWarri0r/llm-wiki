自然语言说明任务与解码规则；稀疏符号答案写成文本，像素对齐答案生成图像，语义与像素并存时输出文字图例+彩色 mask。一个 Bagel-7B-MoT 因此覆盖四大视觉族，不新增任务专用预测头。

§ 01 / Reframing统一的不是 backbone，是答案出口

先说清楚"head（输出头）"是啥。一个视觉大模型通常有一个公用的"身子"（特征抽取器，负责把图看懂），但检测、分割、深度、姿态这些任务后面各接一条专门的"尾巴"——把看懂的东西翻译成"框""mask""深度图"。这些尾巴就叫 head，每种任务一条，结构、损失、解码、后处理全得单独设计；新加一个任务，就得再焊一条尾巴。

SenseNova-Vision1 把这事倒过来想：一个统一多模态模型本来就已经会干两件事——写文本和生成图像。那能不能把所有视觉任务的答案，都翻译成这两种它天生就会的输出？检测框？"写"成一串文字坐标。深度图？"画"成一张灰度图。这样一条尾巴都不用加，全靠模型原有的"嘴"（文本）和"画笔"（图像）。统一的不是把网络主干换掉，而是把答案的出口从"各接一颗 head"改成"要么写、要么画"。

这条路线接在四类前人工作之后：Pix2Seq / Unified-IO 把任务序列化，但稠密图很别扭；SAM、Depth Anything、VGGT 在单一任务族里很强，却没有共同输出空间；Marigold、DICEPTION 把稠密感知变成图像生成，却难写 OCR、框与相机参数；LISA、X-SAM 等 MLLM 能理解语言，稠密结果仍经专用 decoder。SenseNova 的差异是把符号、像素与混合答案同时放回 UMM 原生空间。

§ 02 / Routing rule答案稀疏就写，答案稠密就画

怎么决定一个任务该"写"还是该"画"？看答案长什么样，不看任务叫什么名字：

· 答案稀疏（几个数、几个词就说完）——比如"图里有一个人，框在这里 [坐标]"。这种变长的小记录，用结构化文本写最省事，像列清单。
· 答案稠密（图里每一个像素都得有个值）——比如深度图，几十万个像素每个都要一个"多远"。这种用图像画最自然，因为图像本来就是一格一格的像素，天生对齐空间位置。
· 两者都要（既要说"是谁"、又要精确到像素）——就先写文本、再画图，按固定顺序拼起来。

"VAE latent" 先记这一点模型画图不是一个像素一个像素地涂，而是先在一张压缩过的小画布（VAE latent）上画，再由 VAE 解码器把小画布"解压"成完整大图。所以下文说"生成图像"，实际动作是"在 latent 这张小画布上生成"。mask、深度、法线、点图都在这块画布上学着画。

文本 marker 只是边界，不是新 head

<p> 划出类别、指代表达或 OCR 文本；<bbox>、<point>、<kpt>、<polygon> 分隔不同坐标字段。生成后由 parser 恢复成 benchmark 记录。检测、GUI、OCR 与关键点共享 text generation，却靠 instruction 和 schema 分开。

§ 03 / Decodable text640×480 的框，完整走一遍编码与解码

“能写出来”不够，输出还得确定地还原。可解码表示4是这篇论文真正的工程轴心。设图像宽 W=640、高 H=480，人物框像素坐标为 [64,96,448,384]。

b_norm = [x₀/W, y₀/H, x₁/W, y₁/H] = [64/640, 96/480, 448/640, 384/480] = [0.100, 0.200, 0.700, 0.800]

x₀,y₀ 是左上角，x₁,y₁ 是右下角；横坐标除以宽，纵坐标除以高，让不同分辨率共享 `[0,1]` 空间。论文保留三位小数。解码时分别乘回宽高，正好得到原框。

§ 04 / Dense encodings深度、法线、mask、3D 点都变成“另一张图”

稠密任务的答案和输入图一样大、一样按像素排布，所以"画一张图"是最自然的出口。套路就一句话：一个像素上的量，只要不超过 3 个数，就能当颜色画出来——因为一个像素的颜色本来就是 3 个数 (R,G,B)。

· 深度：每个像素 1 个数（多远）→ 存成灰度（0–255，一个通道就够）。
· 法线：每个像素是一支"朝向"箭头，3 个数 (nₓ,nᵧ,n_z)，每个落在 −1~1 → 线性搬到 0~255 分别塞进 R、G、B（例如 nₓ=0 → 128）。所以法线图看着五颜六色，其实每种颜色就是那一点的朝向。
· 分割 mask：前景涂固定色、背景涂固定色，画成一张双色图。
· point map：每个像素存它在 3D 空间里的坐标 (X,Y,Z)，3 个数 → 同样塞进 RGB。
表面是四种任务，在这套协议里全成了"画一张 RGB 图"，模型的画笔一套通用。

逆深度6为什么合理？`1m→2m` 的 `1/d` 差 0.5，`2m→4m` 只差 0.25，更多动态范围留给近处结构。但“图像形式”也带来生成误差：边缘可能模糊、RGB 可能漂移，必须解码后再按 AbsRel、角度误差或 IoU 评价。

§ 05 / Mixed response颜色图例负责“是谁”，mask 负责“在哪里”

多区域分割不能只吐一张彩色图：RGB 本身不知道对应 road、person 还是 instance-7。模型先用 <color> 绑定语义和颜色，再渲染 mask，这就是颜色图例掩码7。

d(c,S) = minₛ∈S ||c-s||₂ 候选 c 到"已选集合里最近那个"的距离 c_next = argmax_c d(c,S) 挑一个"最近邻都还很远"的候选

S 是已经选好的颜色集合；c 是候选 RGB。规则就是每次挑"离现有颜色最远"的，让颜色两两拉开、不容易看混。距离是普通三通道欧氏距离，拿数字走一遍：已选红 (255,0,0)、绿 (0,255,0)，

蓝 (0,0,255)：到红 = √(255²+0+255²)=√130050 ≈ 360.6，到绿也 360.6 → 最近邻距离 360.6 黄 (255,255,0)：到红 = √(0+255²+0)=255，到绿 = 255 → 最近邻距离 255 360.6 > 255 → 蓝"离得更开"，先选蓝

颜色拉开能减少实例混淆，但生成器仍可能写错颜色、或把边界画糊——所以最终还得解码后按 IoU 之类的指标核对。

§ 06 / Multi-view geometry点图走图像，相机走 token

这节最容易懵，先把几个词拆开：

· 多视图（multi-view）：同一个场景从好几个角度各拍一张（最多取 10 张），目标是从这些 2D 照片反推出 3D。
· 参考坐标系：3D 位置总得有个"原点"当基准。这里干脆拿第一帧当基准，后面所有帧的 3D 点都换算到"以第一张照片为原点"的坐标里，大家才对得齐。
· 内参（intrinsics）：描述这台相机"镜头"的几个数（焦距、画面中心）。有了它 + 某个像素的深度，就能把"屏幕上第 (u,v) 个像素、距离 d 米"反算成"相机眼前的一个 3D 点"。
· 外参（extrinsics）：这台相机本身摆在哪、朝哪（旋转 + 平移）。用它把"相机眼前的 3D 点"搬到"以第一帧为原点"的统一坐标里。
· point map（点图）：一张和照片一样大的图，但每个像素存的不是颜色，是那个点的 3D 坐标 (X,Y,Z)——把 3D 结果又画回一张 RGB 图（承接 §04 的"3 个数当颜色"），交给模型的画笔生成。

串起来就是：像素 + 深度 + 内参 → 相机眼前的 3D 点 → 外参搬到第一帧坐标 → 全序列一起做中心化和缩放（归一化进一个单位立方体，好让不同场景尺度可比）→ 每个点的 XYZ 当 RGB 画成 point map。深度无效的像素（比如天空，没法定位）就沿相机视线推到立方体最远的壁上（skybox），省得为它们单开一条处理分支。

point map 是一路输出；另一路是相机自己的位姿（它在哪、朝哪）。位姿很稀疏，所以用文本 token 写：

· 朝向用四元数（quaternion）——4 个数表示一个 3D 旋转的紧凑写法，比三个欧拉角稳（不会"万向锁"卡死）。(0,0,0,1) 就是"没转"。
· 位置拆成"方向 + 距离"：单位方向 u=(.6,0,.8) 再乘尺度 1.25m，得平移 t=(.75,0,1) 米（图里 pose ledger 那四格算的就是这个）。
· 这些小数怎么变 token？乘 1000 取整，落进 <-1000>…<1000> 这 2001 个数值 token 里；解码时再除 1000、乘尺度还原成米。

完整记录是 <frame><quat>...</quat><offset>...</offset><scale>...</scale></frame>。相机位姿 token 化的好处是相机几何也能用同一张"嘴"写出来；代价是格式、方向单位、旋转合不合法，都可能在自回归生成里写错。

§ 07 / CorpusSN-VC-50M：50M 条协议化监督，不是 50M 张独立原图

SN-VC 是完整可复现 corpus：公共标注能直接转换的部分给 source list、prompt、规则与样例；额外生成或整理的子集作为 SN-VC-50M 发布。重叠 benchmark 保留官方 split，并排除评测图与标注。

§ 08 / Joint training文字算交叉熵，图像 latent 学速度场

一个模型要同时练好两张"嘴"，就得两套 loss 一起上：写文本用交叉熵（预测下一个词，写对了 loss 低）；画图用 rectified flow（在"噪声 → 目标图"的直线上，学"该往哪个方向、多快地走"，即速度场，走到头就是要画的 latent）。两套目标共享同一套参数，只是各算各的 loss，加权相加。

模型从 Bagel-7B-MoT SFT。SN-VC 与 VQA、text-to-image、image-to-image 混采，后者是 capability-preserving mixture10，用来减轻视觉专训对通用理解和生成的侵蚀。VAE visual encoder 冻结，其他模块与 connector 更新；SigLIP2 输入上限在论文中提到 980。

联合目标可写成 L=λ_text·L_text+λ_image·L_image。论文没有报告两个权重；当前脚本才设 CE 0.25、MSE 默认 1.0。交叉熵11与 rectified flow12共享参数，却仍是两种不同目标。

§ 09 / Experiments强项、平手和专家差距要同时放在桌上

论文覆盖四大任务族，不能把不同 benchmark 的 F1、mIoU、AbsRel、角度误差和 pose AUC 平均成一个“总分”。更可靠的读法是分别看每个任务族的相对位置。

协议警告Vision Banana 的 ReasonSeg 使用 Gemini 辅助；其 depth 是 absolute-depth protocol，而 SenseNova 使用 affine-invariant。论文已标成 reference-only，不能拿这些格子直接判胜负。

§ 10 / Convergence先学像图像编辑的，再学长列表与跨视图几何

论文把每项指标除以最终 step 的值，只比较相对收敛进度。depth / normal 最快；普通 detection / generic segmentation 居中；multi-view reconstruction、referring segmentation 与 camera pose 更慢；dense detection 最慢。

§ 11 / Composition & errors组合能力是真的，误差串联也是真的

最有趣的探针没有新增训练协议：把 referring segmentation 的 <p> 与 structured task 的文本坐标 <point> 拼起来，模型能从 <p><point>[.372,.641]</point></p> 直接生成 binary mask。它说明分散任务学到的坐标、指代和 mask 能重新组合，但只有定性案例，没有成功率 benchmark。

其他探针也保留失败：十六进制颜色大体跟随但 RGB 与边界不总精确；透明、反射、视觉错觉会让 depth/normal 跟外观线索走；不同 CFG 会把组合输出偏向 depth-like、normal-like 或 segmentation-like。论文将这些称为 qualitative and imperfect 是恰当的。

§ 12 / What remains“统一接口”还不是“专家已被消灭”

缺核心架构消融没有同数据下专用 head vs 原生生成通道的对照，无法把全部收益归因于统一形式。
缺 mixture 因果账VQA、T2I、I2I 各自对保能力和视觉任务的贡献没拆开。
伪标签上限未分离MoGe-2 与 LingBot-Depth 参与造数据，教师覆盖面与学生上限没有系统隔离。
生成成本没进主表长文本列表与 flow sampling 的 latency / memory 没与一次前向专用 head 对齐比较。
组合任务仍是定性能成功不等于稳定；缺样本量、失败率、校准性与统一 benchmark。
专用先验仍有价值SAM/Mask2Former 与 DepthAnything3 在多项 mask / geometry 指标保持优势。
一句话带走SenseNova-Vision 的遗产不是“一个模型刷完所有榜”，而是把任务设计从“接哪颗 head”改成“答案能否变成文本、图像或混合响应，并确定地解码回来”。它换来统一训练与跨任务组合，也把格式错误、生成延迟、颜色漂移和专用几何先验缺失变成下一阶段的硬问题。

引用 / 术语表

  1. Unified multimodal generation · 用同一 UMM 的文本与图像生成空间表达不同视觉任务，见概念页。
  2. Structured text target · 用 marker 分隔 label、bbox、point、OCR、keypoint 等可解析字段。
  3. VAE latent · 图像经冻结 VAE encoder 压缩后的连续表示；mask、depth、normal、point map 都在这里学习生成。
  4. Decodable representation · 生成结果能经确定逆变换恢复成 benchmark 记录，见可解码视觉表示。
  5. Dense prediction · 每个像素都要预测值的任务，如深度、法线、分割与 point map。
  6. Inverse depth · 深度取倒数 `1/d`，让近处占更多数值动态范围；论文再将其归一化成灰度。
  7. Color-legend mask · 文本声明 region↔RGB，图像按颜色铺 mask，见颜色图例掩码。
  8. Skybox invalid mapping · 把无效深度点沿相机射线推到归一化单位立方体远端，避免另开 mask 训练分支。
  9. Camera-pose tokenization · 四元数、方向和厘米尺度离散成专用 token，见完整数值例。
  10. Capability-preserving mixture · 视觉任务 SFT 时混入 VQA、T2I、I2I，减轻基础理解与生成能力退化。
  11. Cross-entropy · 文本 next-token 目标，正确 token 概率越高 loss 越低，见交叉熵。
  12. Rectified flow · 在数据 latent 与噪声的直线路径上回归速度，见flow matching。
