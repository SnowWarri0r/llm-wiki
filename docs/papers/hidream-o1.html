FIELD NOTES · 统一图像生成 · 无 VAE

HiDream-O1-Image ·
扔掉 VAE 和文本编码器。

HiDream.ai · Pixel-level Unified Transformer · 8B / 200B+ · arXiv 2605.11061 · 2026-05

主流文生图是拼出来的：VAE 压图、独立文本编码器读字、生成器画画——三套语义空间各说各话。HiDream-O1 的赌注：把原始像素、文本、任务条件全塞进一个 transformer 的同一条 token 序列，不要 VAE、不要独立文本编码器，画之前还先"想"一遍版面。

TL;DR

一个原生统一的图像生成基座：Pixel-level Unified Transformer (UiT)1。最反直觉的两刀——没有 VAE2（扩散直接在原始 RGB 像素上做，patch embedding 替掉 VAE 的压缩、去掉 latent 瓶颈）、没有独立文本编码器（backbone 直接是 LLM，8B 从 Qwen3-VL6-8B 初始化）。像素 + 文本 + 条件三类 token 进同一条序列，混合注意力3：文本/条件走 causal、生成像素走 full。名字里的 O1 = 一个 Reasoning Prompt Agent4（Gemma），画之前先推理版面/属性/物理/文字。训练 flow matching + perceptual5、512→1024→2048 三段、后训 SFT + RLHF(GRPO)7。8B 在 GenEval 0.90（超 Qwen-Image 0.87），一套架构吃 T2I/编辑/personalization/storyboard。

§ 01一张图看懂 · 一个 transformer 吃下全部

传统 pipeline 是三个独立件拼的：VAE + 文本编码器 + 生成器。HiDream-O1 把它们塌成一个——原始像素、文本、条件图都变成 token，进同一个 Unified Transformer，出来反 patch 成图。VAE 和独立文本编码器都没了。

§ 02No VAE · 直接在像素上扩散

主流是 latent diffusion：先用 VAE 把图压 48× 成 latent，再在 latent 上跑 DiT。省算力，但 VAE 是个独立冻结模型 + 信息瓶颈——压进 latent 丢掉的高频（尤其文字、小结构）再也回不来。HiDream 直接在原始像素上扩散，让 patch embedding 干压缩这活。

详见 pixel-space-diffusion、kl-vae（被替掉的那个）。

§ 03Unified Transformer · 混合注意力

三类 token 进同一条序列，但注意力 mask 分两种：文本/条件 token 是"给定要理解的"→ 走 causal（像 LLM 顺着读 prompt）；生成像素是"要一起去噪的"→ 走 full（像 DiT 每个 patch 互看）。一条流里两种 mask = 把 LLM 和 DiT 缝进一个 transformer。

backbone 直接是 LLM（8B 从 Qwen3-VL-8B 初始化），文本走它原生词表 embedding——这就是"没有独立文本编码器"。详见 unified-transformer。

§ 04O1 · 画之前先"想"一遍

名字里的 O1 来自一个 Reasoning-Driven Prompt Agent（基于 Gemma）。你给一句模糊的话，它先显式推理——版面怎么摆、主体什么属性、物理逻辑对不对、文字怎么渲染——再把想清楚的细化 prompt 喂给 UiT 去画。像 ideogram 的 "magic prompt"，但带 o1 式的显式思维链。

§ 05训练 · 结果 · 对照 ideogram

训练目标 = flow matching（预测速度场）+ perceptual（LPIPS + DINO）。论文给的理由是：像素扩散细节够好但长程语义连贯弱，perceptual 用深层语义特征补全局连贯（不是教科书那套"防 L2 模糊"）。三段渐进 512 → 1024 → 2048（基础 T2I+语言+多模态理解 → 加编辑/personalization → 超高分辨率精修），后训 SFT + RLHF(GRPO) 优化 OCR/美学/指令遵循/推理质量。8B 从 Qwen3-VL-8B-Instruct 初始化，另有 200B+ 的 Pro。

放进 wiki 这条线 · 两种"统一"哲学

HiDream-O1 和 ideogram-4 是文生图"统一"的两条路，对照着看最清楚：
· ideogram：latent DiT（要 VAE）+ Qwen3-VL 当外挂文本编码器 + 结构化 JSON caption 控版面。统一在"监督形态"。
· HiDream：无 VAE 像素空间 + 文本编码器收进主干（backbone 即 LLM）+ 推理 agent 先想后画。统一在"架构"。
两者都用 Qwen3-VL，但角色完全不同（一个外挂编码、一个当主干）。共同味道还是那条主线——把拼起来的 pipeline 收进一个端到端的权重，少一道接口少一处信息损失。

引用 / 术语表

  1. UiT · Pixel-level Unified Transformer · 把原始像素、文本、任务条件映进同一条 token 序列、用一个 transformer 处理的统一生成架构。无 VAE、无独立文本编码器。详见 unified-transformer。
  2. 无 VAE · 像素空间扩散 · 扩散直接在原始 RGB 上做，patch embedding 替掉 VAE 的压缩、去掉 latent 瓶颈；token 数靠大 patch 控住。详见 pixel-space-diffusion。
  3. 混合注意力 · hybrid attention · 一条序列里文本/条件 token 用 causal mask（像 LLM 读 prompt）、生成 token 用 full attention（像 DiT 双向去噪）。把 LLM 和 DiT 缝进一个 transformer。
  4. Reasoning-Driven Prompt Agent · 基于 Gemma 的"thinking"，对模糊 query 先显式推理版面/属性/物理/文字，再输出细化 prompt 喂给 UiT。名字里 O1 的来历；后训用高质量推理轨迹 SFT。
  5. flow matching + perceptual · 训练目标：flow matching 预测速度场（见 flow-matching）+ LPIPS/DINO 感知损失（在预训练网络特征空间里比图，见 perceptual-loss）。论文说后者补的是像素扩散缺的长程语义连贯（细节本来就够好），不是防 L2 模糊。
  6. Qwen3-VL backbone · 8B 版从 Qwen3-VL-8B-Instruct 初始化，文本走它原生词表 embedding，所以不需要外挂文本编码器。详见 qwen3-vl。
  7. RLHF · GRPO · 后训用 GRPO 做 RLHF，优化 OCR 准确率、美学、指令遵循、推理质量。详见 grpo。
  8. GenEval / CVTG-2K / GEdit · 评测基准：GenEval 综合组合生成、CVTG-2K 图内文字渲染、GEdit 指令编辑。8B 在这几项打平/超更大模型。
