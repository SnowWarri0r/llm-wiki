分割
万物

Kirillov 等 (Meta AI / FAIR) · ICCV 2023 · arXiv 2304.02643

把 NLP 的"基础模型 + 提示 + 零样本"搬进图像分割：给一个点/框/文字，就把对应的东西抠出来，还零样本迁移到没见过的图和任务。

TL;DR

分割模型一直是"一个数据集训一个专才"，SAM 想做分割界的基础模型。三步棋：① 把任务重定义成 可提示分割——给任意 prompt（点/框/掩码/文字）返回一个合理掩码，哪怕有歧义也输出 3 个候选 + IoU 排序；② 架构是重编码器 + 轻解码器：ViT-H 图像编码器每张图只跑一次，提示+掩码解码器每换 prompt 只重跑、~50ms 可交互；③ 用数据引擎让模型帮人标、滚雪球造出 SA-1B：11M 图 / 1.1B 掩码（99.1% 全自动，比之前任何数据集多 400×）。结果：一个模型不微调，靠 prompt 组合就能零样本干边缘检测、物体提议、实例分割、文字→掩码。它的图像编码器后来被 DeepSeek-OCR 借去当 DeepEncoder 的前半段。

§ 01分割为什么没有"基础模型"

NLP 早就靠基础模型 + prompt 通吃下游：训一个大模型，用 prompt 把它"组合"进翻译、问答、摘要，不用每个任务重训。视觉分割却还停在"一个数据集训一个专才、换领域就重标重训"。卡点是数据——像素级标注比打标签贵几个数量级，没有互联网规模的掩码，就训不出基础模型。先有鸡还是先有蛋。

SAM 同时解开这两环：任务侧提出可提示分割让模型能被 prompt 组合，数据侧用数据引擎让模型自己造数据。两者还互相成就——正因为解码轻、能实时交互，标注飞轮才转得起来。

§ 02三件套：重编码器跑一次，轻解码器随便重跑

SAM 由三块组成，关键是重/轻拆分：重型 图像编码器（ViT-H，MAE 预训练，约 6.3 亿参数）每张图只跑一次，出一张 64×64×256 的 embedding；提示编码器（点/框/掩码/文字）+ 轻量掩码解码器（2 层 transformer、双向注意力）每换一个 prompt 只重跑这截，~50ms 在浏览器里就出掩码。这不只是性能优化——正是这个 50ms 让标注员能实时点几下出掩码，下一节的数据飞轮才转得动。

§ 03把歧义当 feature：一个点，三个掩码

可提示分割1有个绕不开的拧巴：你点在「穿格子衬衫的人」的衬衫上，到底要衬衫、这个人、还是这群人？以前的模型被逼着猜一个。SAM 的任务定义网开一面——"返回至少一个合理掩码就算对"，并让模型一次吐 3 个（整体/部分/子部分）9+ 每个的 IoU 自评分，默认取最高、下游也能各取所需。把歧义当 feature，不当 bug。

§ 04数据引擎：让模型帮自己造数据

没有互联网规模的掩码，就训不出基础模型。SAM 的破法是数据引擎——模型帮人标、标出的数据再训模型、模型更强又标得更多，三阶段滚雪球。前两阶段靠人推飞轮，第三阶段模型已经够强，撒手全自动。

§ 05SA-1B：前所未有的规模

飞轮的产出是 SA-1B：1100 万张图、11 亿个掩码，其中 99.1% 全自动生成，比之前任何分割数据集多 400×。质量也守住了——抽查 94% 的掩码跟专业人工修正的 IoU>90%。规模本身就是 SAM 能"分割万物"的底气。

§ 06零样本：一个模型，prompt 组合出多任务

有了可提示模型 + 海量训练，SAM 就能像 LLM 那样被组合进下游：零样本迁移到没专门训过的任务，全靠 prompt。把检测框当 prompt 就做物体提议、让检测器吐框 SAM 抠掩码就做实例分割、借 CLIP 文本编码就做文字→掩码——一个模型，不微调。

放进 wiki 这条线

SAM 跟 CLIP 是一对姊妹：CLIP 把"图文对齐 + 零样本分类"做成基础模型，SAM 把"可提示分割 + 零样本迁移"做成基础模型，骨架都是 ViT。它的真正杠杆是 数据引擎——模型帮自己造数据，这跟 Whisper 弱监督、Unlimited-OCR 自动标注是同一个母题10。而它的图像编码器（SAM-ViT）后来被 DeepSeek-OCR 的 DeepEncoder 借去当前半段——SAM 是 Unlimited-OCR 那条 OCR 压缩线的上游。一句话：把分割改写成"给提示返回掩码"的通用任务,用轻解码器换来实时交互、再用实时交互喂起数据飞轮,造出 400× 的掩码——一个模型零样本分割万物。SAM 只抠"哪一块",不认识"是什么",语义留给下游接。

引用 / 术语表

  1. 可提示分割 · Promptable Segmentation · SAM 的核心任务：给任意 prompt（点/框/掩码/文字）返回至少一个合理掩码，哪怕有歧义。让模型能像 LLM 被 prompt 一样组合进下游。
  2. 图像编码器 · ViT-H · MAE 预训练的 ViT-H（约 6.3 亿参数），14×14 窗口注意力 + 4 个全局块，输出 64×64×256 embedding。重，每张图只跑一次。
  3. 提示编码器 · 把点/框（稀疏，位置编码）、掩码（稠密，卷积）、文字（借 CLIP 文本编码）转成 prompt 向量。轻。
  4. IoU 预测 · 模型对自己每个掩码估一个"跟真值重合多少"的分，用来给 3 个候选排序、挑最好的那个。
  5. 数据引擎 · Data Engine · 模型帮人标→标注训模型→模型更强的飞轮。三阶段（辅助手动→半自动→全自动）滚出 SA-1B。
  6. SA-1B · 11M 图 / 1.1B 掩码，99.1% 全自动，比之前任何分割数据集多 400×；94% 掩码与专业修正 IoU>90%。
  7. 掩码解码器 · 2 层 transformer，双向注意力（prompt↔图像 embedding 互查），上采样出掩码；轻到 ~50ms 在浏览器里跑，是可交互的关键。
  8. 零样本迁移 · 不微调，靠 prompt 组合干没专门训过的任务：边缘检测、物体提议、实例分割、文字→掩码。承自 CLIP 的精神。
  9. 歧义的 3 个掩码 · 一个点可能指整体/部分/子部分，SAM 一次输出 3 个候选，训练时只对 IoU 最高的算 loss，3 个 head 自然分化成三种粒度。
  10. 数据引擎 vs 弱监督 · 都是"大规模数据靠模型自己造"的母题（见 Whisper / Unlimited-OCR），SAM 强调模型在环 + 人逐步退场。
