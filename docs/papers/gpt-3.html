FIELD NOTES · 2020 · OpenAI · 175B · in-context learning

Language Models are
Few-Shot Learners.

Brown · Mann · Ryder · Subbiah · ... · Amodei · 31 作者 · NeurIPS 2020

同架构再放大 116 倍 到 175B，发现一件之前没人预见到的事：prompt 里给几个例子，模型就能现学新任务。这是 in-context learning，整个 ChatGPT 时代由此开始。

TL;DR

GPT-2 1.5B 展示 zero-shot 触发任务可行，但准确度不够。OpenAI 押更大的注：scale 推到 175B + 在 prompt 里给几个示例，看会怎样。

结果发现 3 件大事：

  • In-context learning1 · 不改一个权重，prompt 里给 K 个例子就能让模型现学任务
  • Emergent abilities2 · 算术 / 推理 / 复杂任务在小模型 ≈ 0 分，13B-175B 之间突然抬头
  • Scaling laws 实证3 · 8 个 size 的系统数据证明 LM loss 满足 power law · 模型大小可以"算"出来

论文当时没引爆 —— benchmark 不及 finetune SOTA。2 年后 ChatGPT 引爆了它。GPT-1 的 thesis 一路演化到这里完成闭环。

§ 01把同架构再放 100 倍 · 175B 是怎么来的

架构上 GPT-3 跟 GPT-2 几乎一样 —— 同款 Transformer decoder + causal LM。唯一架构改动：部分层用 sparse attention4（dense / sparse 交替）省算力。

真正的变化在 size —— 一次性训了 8 个 size 系列7 从 125M 到 175B 全套跑 benchmark：

历史镜头里：GPT-1 (117M) 跟 GPT-3 small 一档，GPT-2 (1.5B) 跟 GPT-3 XL 一档。**OpenAI 把 GPT-2 当 "下限" 测试了系列，往上推到 175B 看天花板**。这是 scaling laws 当时押的赌：参数翻 100×，能力质变。

训练规模 · 不是普通研究者能复现的

GPT-3 175B 训练数据：~300B token（Common Crawl 过滤 + WebText2 + Books + Wikipedia）。单次训练算力 $4.6M6（按 2020 v100 价格估算）。"普通研究者训不动" 时代正式开始 —— 这给后来 Llama / Mistral / DeepSeek 等开源模型路线创造了反弹空间。

§ 02In-context learning · 论文真正的 surprise

175B 模型预训练完后，OpenAI 发现一个之前没预见到的能力：在 prompt 里给几个 (input, output) 示例，模型能从中"抽规则"应用到新输入。不需要 finetune，不需要任何梯度更新。

关键观察：权重一个不动。zero → one → few-shot 的性能提升完全来自prompt 里多了几个例子。模型在 forward 过程中从这些例子里"现学"了任务规则。

为什么这能 work · 至今没完全理解

主流假设：

  • Pattern matching · 模型在 attention 里识别 prompt 里的 (x, y) 对，在新 x 出现时去 attend 已有 (x, y) 推断答案
  • Implicit gradient descent5 · attention layer 内部相当于在做小规模的 "meta-learning"
  • Function approximation · transformer 学到的不是具体任务，是"从示例里抽函数"的元能力

无论哪种解释，实证现象都是：scale 上去后 ICL 能力浮现，prompt 里给例子能稳定提升性能。

⚠ 不及 finetune 极致 SOTA · 但赢在工程门槛

GPT-3 few-shot 不是全面碾压 finetune 模型。很多任务 finetune 仍领先 5-15 个点。但开发成本差异巨大：

  • finetune · 标几千条数据 + 训练 + 部署 · 工程一周
  • few-shot · 写几个例子塞 prompt · 5 分钟

"够好 + 极快" 赢了 "最好 + 极慢"8。这是 LLM 时代的真正颠覆点 —— BERT 系 finetune 范式从此在工业界退场，因为 ROI 不如 prompt。

§ 03Emergence · 小模型 ≈ 0 大模型突然能

论文里另一个重要观察：某些任务的性能曲线不是平滑上涨，而是"hockey stick" —— 小模型接近随机，到某个 size 阈值后突然跳到高分。

为什么 emergence 这事重要

它意味着你不能从小模型外推到大模型能力。13B 跑不通的任务 → 175B 可能能跑通。LLM 路线图很大程度上变成"凑齐 size 解锁新能力"，这是为什么 Anthropic / OpenAI / Google 仍在不停烧钱做更大模型。

2023 Anthropic 提出反驳：用错了 metric（exact-match）让平滑曲线看着像突变。改用 log-scale 或 token-level metric 后 emergence 变平滑。但reasoning / tool-use / 长上下文记忆这类复合能力仍表现出真正的非线性 jump，不只是 metric artifact。

§ 04Scaling laws · 算出来的 175B

175B 不是拍脑袋拍的数字。它来自 OpenAI 自己 2020-01 发布的 scaling laws 论文（Kaplan et al.），用大量训练实验拟合出的经验公式：

loss = a / (params)^α + b / (tokens)^β + c

→ 给定算力预算 C，最优 (params, tokens) 组合是固定比例
📐 公式拆解 · power law

loss = 语言模型预训练 loss（perplexity 取 log）· 越低越好
params = 模型参数量 · tokens = 训练数据 token 数
α, β = power law 指数 · 实验拟合出大约 0.05-0.1 之间
a, b, c = 拟合常数 · 跟数据集和架构有关
→ loss 跟 1/params 和 1/tokens 都呈幂律下降（不是线性，是 sub-linear，但永远在跌）。意思是模型越大 + 数据越多 = loss 一定会再降一点，没看到上限。
什么是幂律 / 指数 / sub-linear？查 math-symbols。

OpenAI 用这个公式算出：在 GPT-3 那个时间点的算力预算下，175B 参数 / 300B token 是最划算的配置。然后照着做。

2 年后 DeepMind 重新跑了实验（Hoffmann et al. 2022, "Chinchilla"9），发现 OpenAI 原版 scaling laws低估了数据需求：

Chinchilla 70B 在同算力下跑赢了 GPT-3 175B。结论：之前所有大模型（包括 GPT-3）都数据不足。

2022 之后所有大模型（Llama / Claude / GPT-4）都按 Chinchilla 配比训练：先定参数量，再配 20× 的 token 数据。GPT-3 留下了 scaling 思路，但具体的"最优配比"被 Chinchilla 修正了一次。

§ 05真正的遗产

GPT-3 当年的论文没引爆。benchmark 不及 finetune SOTA，业界看了觉得"挺大但好像没什么用"。真正的爆炸点在 2 年后：

  1. 2022-03 · OpenAI 用 RLHF 把 GPT-3 调成InstructGPT（能听指令）10
  2. 2022-11 · 在 InstructGPT 之上加 chat template + 优化对话体验 = ChatGPT
  3. 2022-12 · ChatGPT 一周破百万用户。普通人第一次意识到 GPT-3 的威力

这条引爆路径的所有交互本质上都是 in-context learning —— "system prompt + user message + assistant continues" 就是个三段式 prompt 模板。GPT-3 的 ICL 是这一切的根基能力。

把 GPT-3 放进 trajectory

GPT-1 论证 pretrain+finetune 可行 → GPT-2 论证 scale 有效 + thesis 清晰 → GPT-3 论证 ICL emerges at scale → ChatGPT 把 ICL 产品化引爆全球。

同样的看 paper 看不出重要性 · 看 trajectory 才看得到：

  • GPT-1 2018 当时 9/12 SOTA 不算震撼
  • GPT-2 2019 当时 zero-shot 不及 finetune
  • GPT-3 2020 当时 few-shot 仍不及 finetune SOTA

三篇都被当时低估，但 trajectory 在: finetune → zero-shot → few-shot ICL → product。每一步看似小幅改进，组合起来是范式革命。BERT 系没立这种长期 trajectory，所以方向感弱，最终输了。这就是 trajectory 视角 vs snapshot 视角的差别11。

2026 视角 · ICL 还在演化

GPT-3 的 ICL 是静态 few-shot —— 你给几个例子，模型现学。今天的 LLM 在此基础上加了：

  • Chain-of-thought · 在 prompt 里让模型"先思考再回答"，激活更深的 reasoning 能力
  • Tool use / function calling · prompt 里描述工具，模型决定调用 → 把"现学"扩展到"现用工具"
  • Long-context ICL · 100K-1M token 上下文里塞下整本书 + 多任务示例
  • Reasoning models (o1 / Claude 3.7 thinking) · 在 ICL 之上嵌套 chain-of-thought + RL 训练 → 内化"思考"过程

每代都是ICL 的扩展12，不是替代。GPT-3 立的范式至今没被打破。

引用 / 术语表

  1. In-context learning · ICL · 上下文内学习 · 模型权重不变，在 prompt 里给 K 个 (input, output) 示例，模型从示例里抽规则应用到新输入。GPT-3 的核心 contribution。详见 in-context-learning。
  2. Emergent abilities · 涌现能力 · 某些任务在小模型上接近 0 分，到某个 size 阈值后突然跳到接近完美。论文用 hockey stick 曲线展示。后续 Anthropic 2023 反驳部分 emergence 是 metric artifact。详见 emergent-abilities。
  3. Scaling laws · 标度律 · Kaplan et al. 2020 · LM loss 跟参数 / 数据 / 算力满足 power law。给出"模型大小可以算出来"的经验公式。175B 这个数字是公式推算的最优配置。后被 Chinchilla 修正。详见 scaling-laws。
  4. Sparse attention · 稀疏注意力 · GPT-3 用 alternating dense/sparse layers 节省算力。奇数层 full attention，偶数层 strided sparse attention。后来被 FlashAttention 部分取代。详见 sparse-attention。
  5. Implicit gradient descent · ICL 的一种理论解释 · attention layer 内部相当于在做小规模 in-weight optimization · 形式上有证明（Akyürek et al. 2022, von Oswald et al. 2023）· 工程上待充分验证。
  6. $4.6M 训练成本 · GPT-3 单次训练用 v100 GPU 按 2020 价格估算的成本。"普通研究者训不动" 时代的标志性数字。给后续开源大模型（Llama / Mistral / DeepSeek）的"低成本路线"创造了反弹空间。
  7. 8 size 系列 · GPT-3 训了 125M / 350M / 760M / 1.3B / 2.7B / 6.7B / 13B / 175B 八个尺寸，系统验证 scaling laws。每个 size 都跑全套 benchmark。这是为什么 emergence 曲线能画出来 —— 没有连续的 size 数据点，"涌现"看不到。
  8. "够好 + 极快"赢"最好 + 极慢" · finetune 可能性能更高，但工程门槛是几天/几周；few-shot ICL 性能略低，但门槛是 5 分钟。在有效市场下，ROI 更优的赢。这是 LLM 时代取代 BERT 时代的根本商业逻辑。
  9. Chinchilla 修正 · Hoffmann et al. 2022 · DeepMind 重新跑 scaling 实验，发现 OpenAI 原版公式低估了数据需求。新结论：每 1 参数配 ~20 token 才是 compute-optimal。GPT-3 175B/300B = 1/1.7 严重不足。Llama / Claude / GPT-4 都按 Chinchilla 比例训。详见 scaling-laws。
  10. RLHF 引爆链 · 2022 InstructGPT → ChatGPT · GPT-3 + RLHF + chat template = ChatGPT。GPT-3 是能力，RLHF 是对齐，chat template 是产品形态。三者缺一不可。论文 GPT-3 是这条引爆链的第一块基石。
  11. Trajectory 视角 vs Snapshot 视角 · 单看 GPT-1/2/3 任意一篇 paper 的 benchmark 都看不出它有多重要。三篇连起来看一条 trajectory（finetune → zero-shot → few-shot ICL）才能看到范式革命。BERT 系单点 paper 当时分数都高，但缺乏长期 trajectory，最终方向感不如 GPT 系。这是判断长期 thesis 是否有 staying power 的关键视角。
  12. ICL 的扩展 · CoT / 工具 / 长上下文 / reasoning · 2022-2026 LLM 演化全部建立在 GPT-3 的 ICL 之上 · Chain-of-thought 让 ICL 触发深思考 · Function calling 让 ICL 触发工具调用 · Long-context 让 ICL 容纳更多信息 · Reasoning models 在 ICL 之上加 RL 训练。GPT-3 立的范式至今没被打破，只是被层层扩展。
