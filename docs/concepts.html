拿一个小窗口（卷积核）滑过整张图，每停一处就把窗口盖住的像素跟核**逐元素相乘再求和**得到一个数 —— 滑遍全图得到一张新图（feature map）。

先把相邻的 `r` 个顶点 token 拼成一个宽 token，再用线性层压回隐藏维；解码时把通道反向拆回 `r` 个 token。

不让网络从零猜完整框；先在每个位置摆几种预设宽高，再回归中心和尺寸的相对变化。

Freebies 只加训练成本；Specials 用少量推理成本换精度。

Batch Normalization · BN

对一个 mini-batch 在每个 channel 内做归一化（减均值/除方差），让中间层的分布**不要剧烈漂移**，间接稳住训练。

Camera Pose Tokenization · 相机位姿 Token 化

把旋转四元数、平移方向和厘米尺度量化成 `[-1000,1000]` 的专用 token，让语言解码器像写结构化文本一样写相机位姿，再按确定公式还原成米制平移。

MLconceptupdated 2026-07-13auto HTML ✓

Color-Legend Mask · 颜色图例掩码

文字先声明“哪个区域对应哪个 RGB”，生成图再用这些颜色铺满像素；文字负责语义与实例身份，图像负责精细边界。

MLconceptupdated 2026-07-13auto HTML ✓

CIoU Loss · 重叠、中心、长宽比三笔账一起算

框没盖好、中心偏了、形状不对，CIoU 都会分别处罚。

MLconceptupdated 2026-07-13auto HTML ✓

Cross-Entropy · 交叉熵 · −log(你押对的概率)

模型对一堆候选输出一个概率分布，交叉熵只看一件事：**你给"正确答案"分了多少概率，取 −log**。押对的概率越高 loss 越低，越低 loss 暴涨。分类 / 预测下一个 token 的标准损失。

MLconceptupdated 2026-07-13auto HTML ✓

CSP · 特征分两路，别让所有通道重复深加工

一部分特征过残差块，一部分走旁路，末尾再拼接融合。

MLconceptupdated 2026-07-13auto HTML ✓

Darknet-53 · 把残差接进 YOLO 骨干

用 1×1/3×3 卷积、stride-2 下采样和残差捷径，换来接近 ResNet-152 的分类精度与约两倍速度。

MLconceptupdated 2026-07-13auto HTML ✓

Decodable Vision Representation · 可解码视觉表示

把视觉标注编码成模型容易生成的文本或图像，同时保留一条确定的逆变换，使生成结果还能还原成 benchmark 要的框、点、深度、法线或 3D 坐标。

MLconceptupdated 2026-07-13auto HTML ✓

维度聚类 · 用数据挑 anchor 形状

把训练集真值框的宽高做 k-means，但距离不用欧氏距离，而用 1−IOU。

MLconceptupdated 2026-07-13auto HTML ✓

层级分类 · 沿标签树逐层乘概率

不把所有类别塞进一个互斥 softmax；每层只在兄弟类之间选择，叶子概率由根到叶一路相乘。

MLconceptupdated 2026-07-13auto HTML ✓

IOU · 两个框重叠多少(交集 ÷ 并集)

衡量两个框有多重合的一个 0~1 数:它们的**交集面积 ÷ 并集面积**。完全重合=1,完全不沾=0。检测里用它判"预测框框得准不准",也用它给重叠框去重。

MLconceptupdated 2026-07-13auto HTML ✓

Mosaic · 四张训练图拼成一张画布

随机缩放、裁剪四张图，再拼成一个带四组标注的训练样本。

MLconceptupdated 2026-07-13auto HTML ✓

多标签分类 · 一个框可以同时叫“女人”和“人”

每个类别独立做 sigmoid，不再用 softmax 强迫所有类别互斥。

MLconceptupdated 2026-07-13auto HTML ✓

多尺度检测 · 同一张图铺粗中细三层探测网

一次前向在多张不同分辨率的特征图上直接出框，让小、中、大物体各用合适的网格。

MLconceptupdated 2026-07-13auto HTML ✓

多尺度训练 · 一套权重适配多档输入

训练时不断切换输入分辨率，让同一个全卷积检测器在部署时能按速度预算换挡。

MLconceptupdated 2026-07-13auto HTML ✓

NMS · 一堆重叠框里,只留最有把握的那个

检测器对同一个物体常吐出好几个重叠框;非极大值抑制(NMS)按置信度排序,留下最高分那个,把和它重叠太多([[iou-intersection-over-union]] 超阈值)的其它框全删掉,一物一框。

MLconceptupdated 2026-07-13auto HTML ✓

单阶段检测 · 一次前向直接出框,不先提候选

两阶段检测先"提一堆候选框、再逐个分类";单阶段(YOLO 这类)把这两步拍成一次前向——一个网络直接从整张图回归出所有框 + 类别,快得多。

MLconceptupdated 2026-07-13auto HTML ✓

Pooling · 池化 · 用一个窗口汇总邻域

Pooling 让一个窗口里的多个数变成一个数；**窗口怎么移动**由 kernel、stride、padding 决定，所以池化可以下采样，也可以像 YOLOv4 的 SPP 一样只扩大视野、不改变宽高。

MLconceptupdated 2026-07-13auto HTML ✓

Promptable Segmentation · 可提示分割

把分割重新定义成"给任意提示（点/框/掩码/文字）返回一个合理掩码"，哪怕提示有歧义也得给出至少一个说得通的答案。

MLconceptupdated 2026-07-13auto HTML ✓

Receptive Field · 感受野 · 一个输出能"看到"原图多大一片

深层一个神经元的值，最终是由原图上**多大一块区域**决定的 —— 这块区域就是它的感受野。卷积堆得越深、池化越多，感受野越大，从"看一个边"长到"看整只猫"。

MLconceptupdated 2026-07-13auto HTML ✓

Residual Connection · 残差连接 / Skip Connection

在每个 block 里加一条 `+ x` 的"快车道"，让网络默认输出 = 输入，layer 只学**跟默认值的差**。

MLconceptupdated 2026-07-13auto HTML ✓

Spatial-wise vs Point-wise Attention · 一张共享地图 vs 每个元素自己的门

原 spatial-wise attention 对每个空间位置只给一个权重，所有通道共享；YOLOv4 所说的 point-wise attention 给 <code>C×H×W</code> 中每个元素一个权重，所以同一位置的不同通道可以一升一降。

MLconceptupdated 2026-07-13auto HTML ✓

SPP + PAN · 同一位置看四档范围，再让高低层双向汇合

YOLOv4 的 SPP 不把 13×13 变小，而是在每个位置并排收集原点、5×5、9×9、13×13 四档上下文；PAN 再让深层语义向细网格下发、细位置向粗网格回流。

MLconceptupdated 2026-07-13auto HTML ✓

读懂 2×2 矩阵 · 两列就是两个基向量落到哪

拿到任何 2×2 矩阵,把它竖着看成两列:第 1 列 = x 基向量 (1,0) 变换后落到的点、第 2 列 = y 基向量 (0,1) 落到的点。单位方块就变成这两列箭头张成的平行四边形,`det` = 这个平行四边形的面积(体积缩放倍数)。

MLconceptupdated 2026-07-06auto HTML ✓

条件 VAE · 学一个潜变量,把数据编进去再还原(带条件)

VAE 学一个中间"潜变量 z":编码器把数据压进 z(而且逼 z 服从一个规整分布)、解码器从 z 还原数据。条件 VAE 就是让这套过程**带上条件**(比如文本),训练目标是最大化 ELBO = 还原得像 − 潜变量别偏离先验太远。

MLconceptupdated 2026-07-05auto HTML ✓

MAS · 动态规划找文本↔音频的单调对齐

文本有 N 个音素、音频有 T 帧(T 远大于 N),谁对应谁不知道;MAS 用动态规划找一条**单调**(不回头)的对齐——每个音素占连续的若干帧、依次铺满全部 T 帧——且让整体似然最大,不需要外部对齐器。

MLconceptupdated 2026-07-05auto HTML ✓

归一化流 · 用可逆变换把简单分布掰成复杂分布

拿一串**可逆**的变换,把一个简单分布(高斯)一步步掰成任意复杂的分布;因为可逆,既能正向采样、又能反向精确算出任一点的概率(靠变量替换 + 雅可比行列式)。

MLconceptupdated 2026-07-05auto HTML ✓

随机时长预测器 · 给每个音素的时长一个"分布",而不是一个定值

同一句话可以说得快或慢、这个字拖长那个字带过——节奏是一对多的。与其给每个音素预测一个固定帧数,VITS 用一个**基于流的时长预测器**输出时长的**分布**,推理时采样出不同节奏,更自然、有变化。

MLconceptupdated 2026-07-05auto HTML ✓

DualCodec · 低帧率 + 语义增强的双流音频码

把 24kHz 音频压成很少的离散码(12.5 帧/秒),而且第一个码本不是普通声学码、是从自监督模型蒸馏来的**语义码**——一条流管"说什么"、另一条流管"听起来啥样",低比特率还保音质。

MLconceptupdated 2026-07-02auto HTML ✓

NAR 掩码并行生成 · 像完形填空一样"一次性填声"

不逐帧自回归吐 token,而是先把要生成的码全部盖住(掩码),一次性**并行**猜出全部、再迭代几步把没把握的重猜——生成语音从"逐帧慢慢来"变成"整段一起填",快很多。

MLconceptupdated 2026-07-02auto HTML ✓

RVQ Codec · 残差向量量化

把音频压成 **N 层堆叠的 codebook**，第 k 层负责前 k-1 层量化剩下的"误差"。
