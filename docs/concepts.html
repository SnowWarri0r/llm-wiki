FlashAttention · 注意力的"省搬运"算法（精确，非近似）

标准注意力把 N×N 的大分数矩阵吐进慢显存(HBM)再读回来,**卡在内存搬运上**。FlashAttention 把 Q/K/V **切成小块、只在快内存(SRAM)里一块块算**,靠**在线 softmax** 拼出和一次算全**一模一样**的结果——大矩阵从不落地。结果:2~4× 快、显存从 O(N²) 降到 O(N),且**输出逐位相同**。

条件概率 / 链式法则 / 贝叶斯 · 拿观测更新信念

**条件概率** `P(A|B)` = 在 B 已知的前提下 A 的概率。**链式法则**把联合概率拆成一串条件 `P(x₁,x₂,…)=P(x₁)P(x₂|x₁)…`——这正是语言模型逐 token 生成的根。**贝叶斯** `P(因|果)=P(果|因)·P(因)/P(果)` 帮你**反过来推**:用观测把"先验信念"更新成"后验信念"。

协方差 / 多元高斯 · 数据散成什么形状的椭圆

方差说一个维度"散多开";**协方差矩阵 Σ** 说多个维度"各自散多开 + 两两一起动不动"——它描述一团数据的**椭圆形状**。妙处:**Σ 的特征向量 = 这个椭圆的轴 = PCA 主成分**(又是 [[eigenvector]] 那条不转的轴),多元高斯就是这个椭圆形状的钟形云 `N(μ, Σ)`。

MLconceptupdated 2026-06-18auto HTML ✓

点积与投影 · dot product · 衡量"方向多一致 + 多长"

两个向量点积 `a·b = Σ aᵢbᵢ = |a||b|cosθ`——**对应分量乘起来加总**,几何上等于"方向多一致(cosθ)× 各自多长"。它是相似度、注意力分数、检索打分、整个矩阵乘法的最小积木。

MLconceptupdated 2026-06-18auto HTML ✓

特征向量 / 特征值 · 被矩阵作用后"只缩放、不转向"的方向

矩阵 `W` 作用在向量上一般又转又拉;但有几个**特殊方向**,`W` 作用后**方向纹丝不动、只被拉长或缩短**——这些方向叫**特征向量**,拉伸倍数叫**特征值 λ**(`Wv = λv`)。它们是这个矩阵"最自然的骨架轴",[[svd]]、PCA、稳定性分析全建在它上。

MLconceptupdated 2026-06-18auto HTML ✓

熵 / KL 散度 · 多不确定 · 用错分布多浪费多少

**熵** = 一个分布平均有多"不确定 / 惊讶"(`H = −Σ p·log p`)。**KL 散度** = 拿一个错分布 Q 去描述真分布 P 时**多花的惊讶** = 交叉熵 − 熵,恒 ≥0、相等才 0。都是 `−log` 在打底,跟 [[cross-entropy]] 一家。

MLconceptupdated 2026-06-18auto HTML ✓

梯度 / 反向传播 · 往哪调参数能让 loss 变小

**梯度** = 每个参数"动一点点,loss 变多少"拼成的向量,指向**上坡最陡**的方向;训练就沿**负梯度**下坡(`参数 ← 参数 − lr·梯度`)。**反向传播** = 用链式法则从 loss 往回**一层层**把每个参数的梯度算出来,复用中间结果不重算。

MLconceptupdated 2026-06-18auto HTML ✓

KL-VAE · 把图压进 latent 的那层地基

一个轻度正则的自编码器，把 512×512×3 的像素图压成 64×64×4 的小 latent —— **扩散模型不在像素上画画，在这个压缩空间里画，省 48 倍计算**。

MLconceptupdated 2026-06-18auto HTML ✓

范数 / 正则化 · 衡量"权重多大"并罚它别太大

**范数**把一个向量"整体多大"压成一个数:**L2** = 欧氏长度 `√Σxᵢ²`(直线距离)、**L1** = 绝对值和 `Σ|xᵢ|`(街区距离)。**正则化** = 在 loss 里加一项权重的范数当"罚款",逼权重别长太大 → 防过拟合;**L2(weight decay)** 让权重均匀缩小,**L1** 逼出稀疏(很多权重正好 = 0)。

MLconceptupdated 2026-06-18auto HTML ✓

Representation Autoencoder · RAE · 拿"看懂图的编码器"当扩散 latent

不再为扩散单训一个 [[kl-vae]] 压缩器,而是**拿一个冻结的预训练表征编码器(SigLIP-2 / DINOv2)当 encoder、只训一个轻量 decoder 还原像素**;扩散直接在这个**高维语义 latent** 里跑。比 VAE 收敛快、抗过拟合、越 scale 越赢。

MLconceptupdated 2026-06-18auto HTML ✓

采样 / 解码策略 · 从概率分布里挑下一个 token

模型每步用 [[softmax]] 吐出**整个词表上的概率分布**;"解码"就是怎么从里面挑下一个 token([[tokenization]]):**greedy**(取最大,稳但呆)/ **按概率采样** + **温度**调随机 / **top-k、top-p** 砍长尾防胡说 / **beam search** 同时搜多条。这一步直接决定生成是稳重还是天马行空。

MLconceptupdated 2026-06-18auto HTML ✓

Softmax · 一组实数压成概率分布 · 温度调软硬

把一组实数(logits)变成**和为 1 的概率**:每个先取 `exp` 再除以总和。指数让"大的更大",但小的也留一点(不像 argmax 一刀切)。**温度 T** 是软硬旋钮:T 小→尖锐(接近 argmax)、T 大→平(接近均匀)。

MLconceptupdated 2026-06-18auto HTML ✓

SVD · 奇异值分解 · 任何矩阵 = 转一下·各轴拉伸·再转一下

[[eigenvector]] 只对方阵、还不一定正交。SVD 是它对**任意矩阵**的推广:任何 `W` 都能拆成 **`W = U Σ Vᵀ`**——一组**入方向** `V`、各自拉伸 `Σ`(对角线是奇异值 σ≥0)、再落到一组**出方向** `U`。σ 排序就是各方向的强度;**留前 r 个 = 最佳低秩近似**,这是 [[matrix-rank]] / 压缩 / PCA 的同一台机器。

MLconceptupdated 2026-06-18auto HTML ✓

Tokenization / BPE · 文本怎么切成 token 喂给模型

模型不直接吃字符也不吃整词,而是吃**子词 token**。**BPE**(Byte-Pair Encoding)从单个字符起步,**反复把"出现最多的相邻一对"合并成一个新 token**,直到词表到设定大小。结果:常见词合成一个 token、罕见词拆成几个、没见过的也能用碎片拼出来。

MLconceptupdated 2026-06-18auto HTML ✓

DMD 蒸馏 · NFE · 把 40 步压成 4 步

把多步扩散老师蒸成少步学生，目标不是抄老师的轨迹，而是让学生的**出图分布**匹配老师的分布；NFE（跑几次网络）从几十降到几步 → 近实时生成。

MLconceptupdated 2026-06-17auto HTML ✓

3D Gaussian Splatting · 3DGS · 用一堆高斯椭球表示场景

不用网格/体素,把一个 3D 场景表示成空间里**几百万个带颜色、透明度、形状朝向的小高斯椭球**;渲染时把它们**"泼/拍(splat)"到屏幕上按深度叠起来**——又快(实时)又能从任意新角度看(新视角合成)。

MLconceptupdated 2026-06-17auto HTML ✓

引导蒸馏 · guidance distillation · 把 CFG 的两遍前向压成一遍

把 [[classifier-free-guidance]](CFG)那个"每步算两遍再外推"的过程**蒸进模型**:让模型把**引导强度当输入**,**一遍前向**就直接吐出"已经引导好"的结果 → 省掉每步那一半算力,质量不掉。

MLconceptupdated 2026-06-17auto HTML ✓

LoRA · 低秩适配 · 不动底模，只训一个小增量

微调时**冻结底模权重不动**,只在旁边训一个**低秩的小增量矩阵** `ΔW = B·A`(秩 r 很小);推理时用 `W + BA`。几十 MB 就给大模型加一个专门技能,可插拔、一个底模能挂很多个。

MLconceptupdated 2026-06-17auto HTML ✓

矩阵的秩 · rank · 一个变换"真正有几个独立旋钮"

秩 = 一个矩阵里**真正独立的方向有几个**。一个 d×k 的大矩阵,秩可能只有几——说明它表面上参数很多,**实际能表达的东西只活在一个很小的子空间里**。"低秩"= 冗余多、可压缩;这正是 [[lora]]、SVD、PCA、注意力瓶颈背后那同一件事。

MMDiT · 多模态扩散 Transformer · 文字图像坐同一张桌子

文字 token 和图像 token **拼成同一条序列、过同一个自注意力**实现双向对齐——但**每种模态各用各的一套权重(QKV 投影 + MLP)**。是 SD3/FLUX 那代文生图的主干，长 prompt 服从和文字排版大涨的根。

MLconceptupdated 2026-06-17auto HTML ✓

像素扩散解码器 · 把 latent→像素的解码改成条件扩散

扩散在 latent 里画完后，用来把 latent 摊回像素的那个**确定性 VAE 解码器**，换成一个**以 latent 为条件、直接在高清像素上去噪的扩散模型**——解码从"忠实还原"升级成"生成式补细节 + 顺手超分"。

MLconceptupdated 2026-06-17auto HTML ✓

Action Chunking · 一次推理预测多步动作

看一眼 → **一口气预测未来好几步动作**，而不是"看一眼只走一步"。让慢的感知/推理频率喂出快的控制频率，还顺带让动作更连贯、少抖、少累积误差。

MLconceptupdated 2026-06-16auto HTML ✓

AI 服务器内存层级 · HBM / LPDDR5X / DDR5 / CXL / 光互连 · 快但小↔慢但大

AI 机架里的内存不是一种东西，而是一条**"越快越贵越小、越慢越便宜越大"的阶梯**——HBM 紧贴 GPU、SOCAMM(LPDDR5X) 在 CPU 边上管容量、再往外是 DDR5 / CXL 内存池 / 光互连共享池。理解这条阶梯，才看得懂"NVIDIA 砍内存"那种新闻到底砍的是哪一层。

模仿学习 · 行为克隆 · 抄人类作业

不靠试错 + reward，而是**看专家(人类)怎么做，学着复现**——把"看到的状态 → 专家的动作"当成监督学习的输入输出对训练。最简单的形态叫**行为克隆(Behavior Cloning)**。

MLconceptupdated 2026-06-16auto HTML ✓

量化 · WxAy · 用更少比特存数字和算

用**更少的比特**存权重、做计算，省显存 + 提速度。命名 **WxAy** = 权重 x 位、激活 y 位;**W8A8**(都 8 位)能让矩阵乘直接跑在 INT8 上、吞吐翻倍,但激活的"离群值"会坑精度,要靠 SmoothQuant 之类的招。

MLconceptupdated 2026-06-16auto HTML ✓

闭式 KL · 同协方差高斯 KL = 均值差²

"闭式"= 有个干净公式能直接代数算（对立面是撒随机点模拟着估）；两个**同协方差高斯**的 KL 散度就有闭式解，直接塌成两个均值差的平方（MSE）。

MLconceptupdated 2026-06-15auto HTML ✓

Deep Research · 深度研究系统 · 派一个会上网的研究员

你给一个问题，它**自己拆子问题、规划路线、多轮上网搜读、交叉核对、最后综合成一篇带引用的报告**——本质是一个**会用搜索工具的 [[memory-stream]] agent 循环**，跑几分钟、几十次工具调用，而不是普通 RAG 那种一问一答。

两把尺子量"重建图 vs 原图"差多少：PSNR 死磕逐像素误差（取对数变 dB，越高越好），SSIM 看局部亮度/对比度/结构（0~1，越接近 1 越像）。

把 RoPE 的"一个位置数字"升级成**三元组 (时间, 行, 列)** —— 让同一套 attention 既能数文字的先后，又能知道图像 patch 在画面的第几行第几列。

所有归一化做的事一样——把一组激活拉回均值 0、方差 1，再配可学的缩放 γ 和偏移 β；**唯一区别是：对<u>哪一组数</u>求那个均值/方差**。

先在小图上训（便宜、快，学全局构图/语义），再逐步升到大图（学细节/纹理）；不一上来就在 2K 从零训。

阿里 Qwen 团队的多模态大模型（图+文+视频）。[[ideogram-4]] 把它**当文本编码器**用——不是因为要看图，而是因为一个见过图文世界的大脑，比纯文本编码器更"懂"prompt 在说什么画面。

MLconceptupdated 2026-06-15auto HTML ✓

ReAct · Reason + Act · 想一步动一步

让 LLM **交替地"想一句(Reason) → 动一下(Act) → 看结果(Observe)"**，而不是憋一大段答案——下一步做什么取决于上一步看到了什么，这就是 agent 会用工具、能多轮调研的最小循环。

Video VAE · Wan-VAE · VAE 的视频版

图像 VAE 把一张图压成小 latent；视频 VAE 多压一个**时间**维——3D 因果卷积把空间和时间一起压、只看过去帧，靠特征缓存能编解码无限长视频。

MLconceptupdated 2026-06-15auto HTML ✓

Forced Alignment · 强制对齐 · 把已知文字钉到时间轴上

文字已经给你了（所以叫"强制"），只求每个字/词在音频里**什么时刻**出现（起止时间戳）——就是卡拉OK逐字高亮要算的那个"何时点亮"。

MLconceptupdated 2026-06-14auto HTML ✓

GSPO · Group Sequence Policy Optimization · 把重要性比率提到序列级

GRPO 在 **token 级**算重要性比率（每个 token 一个 `r`）；GSPO 把比率提到**整条序列**——因为每个 token 只采一次、token 级比率是高方差噪声、容易把训练带崩，而奖励本来就是给整条序列的。

MLconceptupdated 2026-06-14auto HTML ✓

Log-Mel Spectrogram · Fbank · 对数 Mel 频谱图

把 1D 音频波形转成 2D `[频道 × 时间]` 的能量矩阵 —— 跟人耳听到的"哪个时刻、哪个频段、有多响"对应。ASR 圈管它叫 **Fbank**（log mel filterbank energies），同一个东西。

MLconceptupdated 2026-06-14auto HTML ✓

Modality Projector · 把别的模态翻译进 LLM 的隐空间

一个小 MLP，把**冻结的外部编码器**吐出的语音/图像特征，投影成 LLM 听得懂的 hidden 向量，塞进序列里的占位符位置 —— 让一个纯文本 LLM 不改主干就能"看图听音"。

MLconceptupdated 2026-06-14auto HTML ✓

Convolution · 卷积 · 一把小尺子滑过整张图

拿一个小窗口（卷积核）滑过整张图，每停一处就把窗口盖住的像素跟核**逐元素相乘再求和**得到一个数 —— 滑遍全图得到一张新图（feature map）。

MLconceptupdated 2026-06-12auto HTML ✓

Cross-Entropy · 交叉熵 · −log(你押对的概率)

模型对一堆候选输出一个概率分布，交叉熵只看一件事：**你给"正确答案"分了多少概率，取 −log**。押对的概率越高 loss 越低，越低 loss 暴涨。分类 / 预测下一个 token 的标准损失。

MLconceptupdated 2026-06-12auto HTML ✓

Markov Chain · 马尔可夫链 · 只看现在、不看历史

一串状态，下一步**只取决于你现在在哪、跟怎么走到这儿无关**（无记忆）；每步用一条"转移核"规则跳。扩散去噪就是它的高斯版。

flow matching 用 ODE（确定性流），diffusion 用 SDE（随机性流）。差别是<strong>路径上有没有噪声项</strong>。

MLconceptupdated 2026-06-12auto HTML ✓

Score Function · ∇log p(x) · 对数概率梯度

对一个概率密度 p(x)，**score = ∇log p(x)** —— 指向"概率密度更大的方向"的向量。diffusion 模型学这个，flow matching 不学这个。

MLconceptupdated 2026-06-12auto HTML ✓

EMA · 指数滑动平均 · 一行就够

一行更新规则 `新 = m·旧 + (1−m)·新值`，一个数原地滚 —— 把抖动磨平、追出一条更稳的"趋势线"，不用存任何历史。

MLconceptupdated 2026-06-11auto HTML ✓

Elastic Inference · 一个模型，多档算力

一次训练出一**族**模型，推理时按手头算力**临时挑一档**（快但糙 ↔ 慢但好），不必为每个预算单独训一个模型。

**前台 always-on 低延迟模型** + **后台异步深推理模型**，共享 context，前台委托后台干慢活。

Early Fusion · 无独立编码器的早融合

各模态用**轻量预处理**直接进同一个 transformer，不用 Whisper / ViT 那种独立大编码器再拼。

KV Cache

Transformer 每层 attention 算过的 Key / Value 矩阵存下来，下一个 token 直接复用，不重算前面所有 token。

MLconceptupdated 2026-05-20auto HTML ✓

Micro-Turn · 200ms 微回合

把时间切成 200ms 一片，每片里**输入和输出同时发生**，模型看到的是一条单一交错 token 流。

MoE · Mixture of Experts

模型养着许多"专家"子网，每次推理一个 token 只激活其中少数几个 —— 总参大但计算路径小。

MLconceptupdated 2026-05-20auto HTML ✓

Multi-Head Attention · 多头注意力

把 attention 跑 **h 次**（论文 h=8），每次用不同的 Q/K/V 投影 → 每个 head 学不同的关系模式 → concat 后再线性。

Prefill / Decode

LLM 推理两阶段：**prefill** 一次性吃整段输入并行算所有 token，**decode** 一个一个往外吐。

RVQ Codec · 残差向量量化

把音频压成 **N 层堆叠的 codebook**，第 k 层负责前 k-1 层量化剩下的"误差"。

Transformer Architecture · 整体架构

Encoder 6 层 + Decoder 6 层，每层都是"multi-head attention + FFN + residual + layernorm"的标准块。Decoder 多一个 cross-attention 子层看 encoder 输出。
